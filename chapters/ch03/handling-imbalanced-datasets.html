Sampling Techniques

Chapter 3: Data Fundamentals Section 8
TL;DR

Can't use all data — too large, too expensive to label, or too imbalanced — sample. Sampling method decides if subset represents whole. Section covers random sampling, stratified sampling, systematic sampling, reservoir sampling for streams, importance sampling for rare events, and over/undersampling techniques (SMOTE, ADASYN) for handling class imbalance.

Why Sampling Matters

Got 500 million rows of transaction data. Training on all rows takes days. Need smaller, representative subset. Or got 100,000 images, labeling each costs $2. Budget covers only 5,000. Which 5,000? Or 99.5% of data is class "normal" and 0.5% is class "fraud." Model learns always predict "normal" and gets 99.5% accuracy — catches zero fraud.

All three problems are sampling problems. How you draw subset from larger population decides what model sees during training, and what model sees shapes what it learns. Biased sample makes biased model. Good sample is miniature version of reality.

Random Sampling

Random sampling gives every row equal probability of selection. Default. Baseline. Often enough. When dataset is large and reasonably homogeneous, random 10% gives representative subset without special machinery.

import pandas as pd
import numpy as np

# Simple random sample: 10% of the data
sample = df.sample(frac=0.10, random_state=42)

# Fixed size sample
sample_fixed = df.sample(n=5000, random_state=42)

# With replacement (bootstrap sampling)
bootstrap = df.sample(n=len(df), replace=True, random_state=42)
# Some rows appear multiple times, some not at all

Random sampling works when population is homogeneous. Breaks when important subgroups exist. If 0.1% of data represents rare disease, random 1% sample might contain zero examples. Next step: stratified sampling.

Stratified Sampling

Stratified sampling splits population into subgroups (strata) and samples from each stratum proportionally — or sometimes disproportionally, so rare groups get enough representation. Guarantees every stratum appears in sample.

from sklearn.model_selection import train_test_split

# Proportional stratified sampling
# Each class gets the same proportion in the sample as in the population
X_sample, _, y_sample, _ = train_test_split(
    X, y, train_size=0.1, stratify=y, random_state=42
)

# Verify proportions are preserved
print(f"Original: {np.bincount(y) / len(y)}")
print(f"Sample:   {np.bincount(y_sample) / len(y_sample)}")

# Disproportional stratified sampling (manual)
# Oversample the rare class to ensure adequate representation
rare = df[df['label'] == 1].sample(n=500, random_state=42)
common = df[df['label'] == 0].sample(n=500, random_state=42)
balanced_sample = pd.concat([rare, common]).sample(frac=1, random_state=42)

Use stratified sampling when data has meaningful subgroups with different sizes. Especially important for imbalanced classification, multi-class problems, and any case where performance estimates must stay reliable across all subpopulations.

Systematic Sampling

Systematic sampling selects every k-th item from ordered list, starting from random offset. Want 1,000 samples from 100,000 rows? Set k=100 and pick rows 0, 100, 200, ..., 99900 (with random start between 0 and 99).

# Systematic sampling
def systematic_sample(df, n_samples, random_state=42):
    """Select every k-th row from the dataframe."""
    rng = np.random.RandomState(random_state)
    k = len(df) // n_samples
    start = rng.randint(0, k)
    indices = np.arange(start, len(df), k)[:n_samples]
    return df.iloc[indices]

sample = systematic_sample(df, n_samples=1000)

Systematic sampling is simple and spreads selections evenly across dataset. Useful for data with natural ordering (e.g., time-sorted logs when you want even temporal coverage). Risk: if data has periodic pattern aligned with sampling interval, sample gets biased. Shuffling before systematic sampling removes risk but also removes advantage of even coverage.

Reservoir Sampling

What if you don't know total dataset size? Happens with streaming data — server logs arriving in real time, sensor readings from IoT devices, social media feeds. Can't compute len(df) because data hasn't finished arriving.

Reservoir sampling keeps fixed-size sample of k items from stream of unknown length, with guarantee that every item seen so far has equal probability of being in sample. Online algorithm — processes one item at time and never needs second pass.

import random

def reservoir_sample(stream, k, random_state=42):
    """Sample k items from a stream of unknown length.
    
    Each item in the stream has equal probability of being in the final sample.
    """
    rng = random.Random(random_state)
    reservoir = []
    
    for i, item in enumerate(stream):
        if i < k:
            reservoir.append(item)
        else:
            # Replace a random element with decreasing probability
            j = rng.randint(0, i)
            if j < k:
                reservoir[j] = item
    
    return reservoir

# Example: sample 100 items from a file too large to fit in memory
with open('huge_log.csv') as f:
    sample = reservoir_sample(f, k=100)

Math is elegant. When item i arrives (0-indexed), item enters reservoir with probability k/(i+1). If item enters, item replaces randomly chosen existing item. After n items pass, every item has exactly k/n probability of being in reservoir. No counting ahead. No multiple passes. Constant memory.

Reservoir sampling is essential when data arrives incrementally: real-time monitoring systems, large-scale log analysis, and streaming ML pipelines that need representative buffer.

Importance Sampling

Sometimes some regions of data distribution matter more than others. In reinforcement learning, rare but high-reward trajectories matter more than common boring ones. In risk modeling, tail events (0.01% catastrophes) matter more than 99.99% of normal days.

Importance sampling assigns non-uniform weights to data points, oversampling important regions and undersampling unimportant ones. To keep estimates unbiased, correct for sampling bias by dividing each point's contribution by its sampling probability.

import numpy as np

def importance_sample(data, weights, n_samples, random_state=42):
    """Sample with non-uniform probabilities, returning correction weights."""
    rng = np.random.RandomState(random_state)
    
    # Normalize weights to probabilities
    probs = weights / weights.sum()
    
    # Sample indices according to weights
    indices = rng.choice(len(data), size=n_samples, replace=True, p=probs)
    
    # Importance weights correct for the biased sampling
    # w_i = 1 / (n * p_i) where p_i is the sampling probability
    importance_weights = 1.0 / (n_samples * probs[indices])
    
    return data[indices], importance_weights

# Example: oversample extreme values in a distribution
values = np.random.exponential(scale=1.0, size=100000)
# Weight rare large values more heavily
weights = np.abs(values) + 0.1  # larger values get higher sampling probability
sampled, iw = importance_sample(values, weights, n_samples=1000)

# Unbiased mean estimate using importance weights
weighted_mean = np.average(sampled, weights=iw)
true_mean = values.mean()
print(f"True mean: {true_mean:.4f}, Importance-weighted estimate: {weighted_mean:.4f}")

Importance sampling is specialized technique. You'll see it in Monte Carlo methods, policy gradient reinforcement learning, and off-policy evaluation. For standard supervised learning, over/undersampling techniques below are more common.

Over/Undersampling for Imbalanced Data

Class imbalance is most common sampling challenge in applied ML. When one class vastly outnumbers another, classifiers take lazy path: always predict majority class. Getting 99.5% accuracy by ignoring all fraud is useless. Need model to learn minority class.

Random Undersampling and Oversampling

Simplest fix: random undersampling throws away majority class examples until classes balance. Random oversampling duplicates minority class examples until classes balance. Crude. Effective starting points.

from sklearn.utils import resample

# Separate classes
majority = df[df['label'] == 0]
minority = df[df['label'] == 1]

# Undersampling: reduce majority to match minority size
majority_down = majority.sample(n=len(minority), random_state=42)
df_under = pd.concat([majority_down, minority])

# Oversampling: duplicate minority to match majority size
minority_up = resample(minority, replace=True, n_samples=len(majority), random_state=42)
df_over = pd.concat([majority, minority_up])

Undersampling throws away potentially useful data. If you have 100,000 majority examples and 500 minority examples, undersampling leaves only 1,000 total samples. Big information loss. Oversampling creates exact duplicates, which can make model overfit specific minority examples.

SMOTE: Synthetic Minority Over-sampling

SMOTE (Synthetic Minority Over-sampling Technique) generates new synthetic minority examples instead of duplicating existing ones. For each minority sample, method finds k nearest neighbors in feature space, then creates new sample at random point along line connecting original sample to one neighbor.

# pip install imbalanced-learn
from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42, k_neighbors=5)
X_resampled, y_resampled = smote.fit_resample(X_train, y_train)

print(f"Before SMOTE: {np.bincount(y_train)}")
print(f"After SMOTE:  {np.bincount(y_resampled)}")
# e.g., Before: [9500, 500] → After: [9500, 9500]

SMOTE is more sophisticated than random oversampling because it creates new, slightly different examples instead of exact copies. This reduces overfitting to specific minority points. But SMOTE has real limits. It generates synthetic points by interpolating in feature space, which assumes space between two minority examples is also minority. In practice, not always true — generated points might land in majority class territory, creating noise that confuses classifier.

⚠️ Critical Rule

Always apply SMOTE (or any resampling) only to training set, never to validation or test set. If you resample before splitting, synthetic points derived from training data leak into evaluation set and give falsely optimistic results. Workflow: split first, then resample only training fold.

ADASYN: Adaptive Synthetic Sampling

ADASYN (Adaptive Synthetic Sampling) is a variation of SMOTE that focuses on hard-to-classify minority examples. It generates more synthetic samples near minority points that are surrounded by many majority points (the decision boundary) and fewer samples in regions where the minority class is already well-represented.

from imblearn.over_sampling import ADASYN

ada = ADASYN(random_state=42)
X_ada, y_ada = ada.fit_resample(X_train, y_train)

print(f"Before ADASYN: {np.bincount(y_train)}")
print(f"After ADASYN:  {np.bincount(y_ada)}")
# ADASYN may not produce exact balance — it adapts to difficulty

ADASYN's adaptive weighting is appealing in theory: spend your synthetic data budget where it matters most. In practice, it can generate noisy points in overlapping class regions, making the boundary even harder to learn. Compare SMOTE and ADASYN empirically on your specific dataset — there's no universal winner.

Combining Over and Undersampling

The best practical approach often combines both: oversample the minority with SMOTE, then clean up by removing majority points that are too close to synthetic minority points (or vice versa). The SMOTEENN and SMOTETomek methods from imbalanced-learn implement these hybrid strategies.

from imblearn.combine import SMOTEENN, SMOTETomek

# SMOTE + Edited Nearest Neighbors (removes ambiguous points)
smote_enn = SMOTEENN(random_state=42)
X_senn, y_senn = smote_enn.fit_resample(X_train, y_train)

# SMOTE + Tomek Links (removes close opposite-class pairs)
smote_tomek = SMOTETomek(random_state=42)
X_st, y_st = smote_tomek.fit_resample(X_train, y_train)

print(f"SMOTEENN result:  {np.bincount(y_senn)}")
print(f"SMOTETomek result: {np.bincount(y_st)}")

When Not to Resample

Resampling isn't always the answer. Modern gradient boosting frameworks (XGBoost, LightGBM) support scale_pos_weight or sample weights that achieve similar effects without modifying the data. This is often cleaner and avoids the artifacts of synthetic data generation.

from sklearn.ensemble import RandomForestClassifier
from xgboost import XGBClassifier
import numpy as np

# Class weights (built into sklearn classifiers)
rf = RandomForestClassifier(class_weight='balanced', random_state=42)
rf.fit(X_train, y_train)

# XGBoost: scale_pos_weight
n_neg = np.sum(y_train == 0)
n_pos = np.sum(y_train == 1)
xgb = XGBClassifier(scale_pos_weight=n_neg / n_pos, random_state=42)
xgb.fit(X_train, y_train)

Using class weights or sample weights is equivalent to cost-sensitive learning — misclassifying a minority example incurs a higher loss. This guides the optimizer toward caring about minority accuracy without fabricating data. Try class weights first. If that's not enough, add SMOTE. Always validate with metrics that reflect minority class performance (F1, precision-recall AUC) rather than raw accuracy.

ℹ️ Choosing a Sampling Strategy

Large balanced dataset: Random sampling is sufficient. Small dataset with rare subgroups: Stratified sampling preserves representation. Streaming data: Reservoir sampling maintains a fixed-size representative buffer. Imbalanced classification: Start with class weights, then try SMOTE if needed. Time series: Never shuffle — use temporal ordering. The right technique depends on your data structure and what "representative" means for your specific problem.
