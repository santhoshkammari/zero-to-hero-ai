Exploratory Data Analysis (EDA)

Chapter 3: Data Fundamentals 7 subtopics
TL;DR

Before you build any model, you need to actually look at your data. Exploratory Data Analysis is the practice of summarizing, visualizing, and questioning a dataset to understand its structure, spot problems, and form hypotheses. It catches missing values, weird distributions, and broken features before they silently wreck your model. Think of EDA as a conversation with your data — you ask questions, it answers with numbers and plots, and you decide what to do next.

1. Initial Data Inspection

Every EDA session starts the same way: open the dataset and figure out what you're looking at. How many rows? How many columns? What types are they? This sounds boring, but skipping it is how people spend three hours debugging a model only to discover a column was stored as text instead of numbers.

Shape, Columns, and Data Types

The shape of a DataFrame tells you its dimensions — rows and columns. The dtypes (data types) tell you how pandas is interpreting each column: int64 for integers, float64 for decimals, object for strings, datetime64 for dates. A column that should be numeric but shows up as object almost always contains dirty values — a stray comma, a "N/A" string, or an encoding issue. Catching this early saves you hours.

import pandas as pd
import numpy as np

df = pd.read_csv("housing.csv")

# Dimensions: (rows, columns)
print(df.shape)                # (20640, 10)

# Column names and types
print(df.dtypes)

# Quick look at the first and last rows
print(df.head())
print(df.tail())

# Random sample — better than head() for spotting patterns
print(df.sample(5, random_state=42))

info() and describe()

Two commands you'll run on literally every dataset. df.info() gives you a column-by-column summary: name, non-null count, and dtype. If the non-null count for a column is lower than the total row count, you have missing data. df.describe() computes summary statistics — count, mean, standard deviation, min, max, and the 25th/50th/75th percentiles — for every numeric column. Together, these two calls give you a surprisingly complete first picture.

# Compact summary: types, non-null counts, memory usage
df.info()

# Statistical summary for numeric columns
print(df.describe())

# Include categorical columns too
print(df.describe(include='all'))

Duplicates, Unique Values, and Value Counts

Duplicate rows are rows that appear more than once with identical values across all columns. Sometimes they're legitimate (two customers placing the same order), sometimes they're data pipeline bugs. Always check. Value counts show how often each unique value appears in a column — essential for understanding categorical features. A column with 50,000 unique values out of 50,001 rows is practically an ID column, not a useful feature.

# Check for duplicate rows
print(f"Duplicates: {df.duplicated().sum()}")

# Unique values per column
print(df.nunique())

# Value counts for a categorical column
print(df["ocean_proximity"].value_counts())

# Proportions instead of raw counts
print(df["ocean_proximity"].value_counts(normalize=True))

2. Univariate Analysis

Univariate analysis means examining one variable at a time. You're asking: what does this single column look like? Is it normally distributed? Skewed? Full of outliers? This is where you start plotting.

Distributions of Numerical Features

A histogram chops a numeric column into bins and counts how many values fall into each bin. It's the fastest way to see the shape of a distribution. A KDE plot (Kernel Density Estimate) smooths that histogram into a continuous curve, which can be easier to read when comparing multiple distributions. Most real-world data is not normally distributed — you'll see long tails, multiple peaks, and hard boundaries at zero.

import matplotlib.pyplot as plt
import seaborn as sns

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

# Histogram
axes[0].hist(df["median_income"], bins=50, edgecolor="black")
axes[0].set_title("Histogram: Median Income")

# KDE plot
sns.kdeplot(df["median_house_value"], ax=axes[1], fill=True)
axes[1].set_title("KDE: Median House Value")

# Combined histogram + KDE
sns.histplot(df["housing_median_age"], kde=True, ax=axes[2])
axes[2].set_title("Age Distribution")

plt.tight_layout()
plt.show()

Central Tendency and Spread

The mean is the arithmetic average. The median is the middle value when sorted — more robust to outliers than the mean. The mode is the most frequent value. Standard deviation measures how spread out values are from the mean; variance is its square. When the mean and median are far apart, your data is skewed. When standard deviation is large relative to the mean, your data is highly spread out. These aren't just academic numbers — a feature with near-zero variance is useless for prediction, and a feature with extreme outliers can dominate your model.

Skewness measures asymmetry. Positive skew means a long right tail (income data, for example). Negative skew means a long left tail. Kurtosis measures how heavy the tails are compared to a normal distribution. High kurtosis means more extreme outliers. Both are useful diagnostics, but don't over-index on them — a plot tells you more than a single number.

Categorical Feature Distributions

For categorical columns, bar plots replace histograms. You're looking at how many observations fall into each category. A category that contains 95% of the data gives your model almost no signal. Rare categories — values that appear only a handful of times — can cause problems too: your model might overfit to them, or they might disappear entirely in a train/test split. Consider grouping rare categories into an "Other" bucket if they individually represent less than 1-2% of the data.

3. Bivariate and Multivariate Analysis

Univariate analysis tells you about each feature in isolation. Bivariate analysis asks: how do two variables relate to each other? This is where you start finding the signal that actually helps prediction.

Scatter Plots and Box Plots

A scatter plot shows one numeric variable against another. You're looking for linear trends, clusters, or strange patterns. A box plot compares a numeric variable across categories — it shows the median, interquartile range, and outliers for each group. If the boxes look identical across groups, that categorical feature probably doesn't help predict the numeric one.

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Scatter: income vs house value
axes[0].scatter(df["median_income"], df["median_house_value"],
                alpha=0.1, s=5)
axes[0].set_xlabel("Median Income")
axes[0].set_ylabel("Median House Value")
axes[0].set_title("Income vs House Value")

# Box plot: house value by ocean proximity
sns.boxplot(x="ocean_proximity", y="median_house_value",
            data=df, ax=axes[1])
axes[1].set_title("House Value by Location")
axes[1].tick_params(axis='x', rotation=45)

plt.tight_layout()
plt.show()

Correlation Matrix and Heatmap

A correlation matrix computes the pairwise correlation between every numeric column. Pearson correlation measures linear relationships and ranges from −1 (perfect inverse) to +1 (perfect positive). Spearman correlation measures monotonic relationships — it works even when the relationship isn't a straight line. Kendall correlation is similar to Spearman but handles ties and small samples better. Pearson is the default and usually your first choice. Switch to Spearman when you suspect non-linear but still monotonic patterns.

# Correlation matrix
corr = df.select_dtypes(include=[np.number]).corr(method="pearson")

# Heatmap
plt.figure(figsize=(10, 8))
sns.heatmap(corr, annot=True, fmt=".2f", cmap="coolwarm",
            center=0, square=True, linewidths=0.5)
plt.title("Pearson Correlation Heatmap")
plt.tight_layout()
plt.show()

# Spearman for non-linear monotonic relationships
spearman_corr = df.select_dtypes(include=[np.number]).corr(method="spearman")

A word of caution: correlation does not mean causation, and zero correlation does not mean independence. Two variables can have zero Pearson correlation yet be perfectly related in a non-linear way. Always pair the matrix with scatter plots for the pairs that matter most.

Pair Plots, Grouped Statistics, and Cross-tabulation

A pair plot draws scatter plots for every pair of numeric columns, with histograms on the diagonal. It's a brute-force way to see everything at once. For datasets with more than about 8-10 numeric columns, pair plots get unwieldy — select the most promising features first.

# Pair plot for selected features, colored by a categorical variable
selected = ["median_income", "housing_median_age",
            "median_house_value", "ocean_proximity"]
sns.pairplot(df[selected], hue="ocean_proximity",
             diag_kind="kde", plot_kws={"alpha": 0.4, "s": 10})
plt.suptitle("Pair Plot", y=1.02)
plt.show()

Grouped statistics aggregate a numeric column by categories — the pandas groupby followed by .mean() or .median(). Cross-tabulation counts the overlap between two categorical variables, helping you spot associations. For understanding feature vs target relationships, group the target by each feature and look for separation: if the target's distribution changes meaningfully across groups, that feature has predictive power.

# Grouped statistics
print(df.groupby("ocean_proximity")["median_house_value"].agg(
    ["mean", "median", "std", "count"]
))

4. Data Quality Assessment

EDA isn't just about pretty plots. It's your main tool for finding data quality problems before they corrupt your model.

Missing Value Patterns

Missing values come in three flavors. MCAR (Missing Completely at Random) means the missingness has no relationship to any variable — rare in practice. MAR (Missing at Random) means the missingness depends on other observed variables — for example, younger people might skip the income question on a survey. MNAR (Missing Not at Random) means the missingness depends on the missing value itself — high earners refusing to report income. The type of missingness determines your imputation strategy. MCAR is safe to impute with mean or median. MAR benefits from model-based imputation. MNAR is dangerous — imputation can introduce systematic bias.

# Missing value counts and percentages
missing = df.isnull().sum()
missing_pct = (missing / len(df)) * 100
missing_report = pd.DataFrame({
    "missing_count": missing,
    "missing_pct": missing_pct
}).sort_values("missing_pct", ascending=False)
print(missing_report[missing_report["missing_count"] > 0])

# Visualize missing patterns
plt.figure(figsize=(10, 4))
sns.heatmap(df.isnull(), cbar=True, yticklabels=False, cmap="viridis")
plt.title("Missing Value Pattern")
plt.show()

Data Integrity, Inconsistencies, and Class Imbalance

Data integrity checks verify that values make physical or logical sense. Negative ages, future dates in historical records, prices of zero — these are red flags. Look for values that violate known constraints. Inconsistencies are subtler: the same entity recorded with different spellings ("New York" vs "new york" vs "NY"), units that change mid-dataset (meters vs feet), or timestamps in multiple time zones.

Class imbalance matters when your target variable is categorical. If 98% of your labels are one class and 2% are another, most models will just predict the majority class and report 98% accuracy — which is useless. Check the target distribution early so you can plan for resampling, weighted losses, or appropriate evaluation metrics.

Feature cardinality — the number of unique values a feature has — deserves attention too. A categorical column with thousands of unique values (like city names or user IDs) needs special handling: target encoding, hashing, or aggregation. Feeding raw high-cardinality categoricals into most models either blows up memory or produces garbage results.

5. Data Drift Detection During EDA

Data drift is when the statistical properties of your data change over time. Maybe customer behavior shifted during a pandemic, or a sensor started degrading. During EDA, you can detect drift by segmenting your data by time period and comparing distributions. Plot the same feature for different months or quarters side by side. If the histograms look different, you have drift. More formally, you can use the Kolmogorov-Smirnov test or Population Stability Index (PSI) to quantify distribution shifts. Drift doesn't always mean your data is broken — sometimes the world genuinely changed — but your model needs to know about it.

6. EDA for High-Dimensional Data

When you have 1,000+ features, the strategies above start to break down. You can't make a pair plot of a million scatter plots. Instead, focus on summary statistics at scale: compute the mean, variance, and missing-value percentage for every column, then sort by variance to find the near-constant features (drop them). Use correlation with the target to rank features by relevance. Dimensionality reduction techniques like PCA (Principal Component Analysis) or t-SNE project your high-dimensional data into 2D or 3D for visualization. PCA also tells you how many dimensions actually carry information — if the first 20 principal components explain 95% of the variance, the other 980 features are mostly noise. Automated tools, which we cover next, become essential at this scale.

7. Automated EDA Tools

Manually writing the code above for every dataset gets repetitive. Several libraries automate the entire EDA workflow into a single function call.

ydata-profiling (formerly pandas-profiling) generates an interactive HTML report covering distributions, correlations, missing values, duplicates, and more. It's the most comprehensive option. Sweetviz creates a visual comparison report — especially useful for comparing train and test sets side by side. D-Tale launches an interactive web UI where you can explore data, build charts, and export code — ideal for people who prefer clicking over coding.

# ydata-profiling (formerly pandas-profiling)
from ydata_profiling import ProfileReport

profile = ProfileReport(df, title="Housing EDA", explorative=True)
profile.to_file("housing_eda_report.html")

# Sweetviz
import sweetviz as sv

report = sv.analyze(df)
report.show_html("sweetviz_report.html")

# D-Tale (launches interactive browser UI)
# import dtale
# dtale.show(df)

The honest tradeoff: automated tools are great for a first pass, but they can't replace thinking. They'll show you every correlation and every distribution, but they won't tell you which ones matter for your problem. They also struggle with very large datasets — profiling a million-row DataFrame can take minutes or run out of memory. Use them to get oriented, then dig into the specifics manually.

EDA is not a checkbox you tick once. It's an iterative process. You inspect, you find something odd, you investigate, you clean, and then you inspect again. The goal is never a perfect dataset — it's a well-understood dataset. A model built on data you understand will always outperform one built on data you simply threw at an algorithm and hoped for the best.
