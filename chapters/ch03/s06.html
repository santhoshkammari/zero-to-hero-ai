Feature Selection

Chapter 3: Data Fundamentals Section 6
TL;DR

Feature engineering creates features. Feature selection decides which ones to keep. Irrelevant or redundant features add noise, slow training, and cause overfitting. This section covers three families of selection methods — filter (fast statistical tests), wrapper (search-based using model performance), and embedded (built into model training) — plus the distinction between feature selection and dimensionality reduction, stability concerns, and multi-objective selection strategies.

Why Feature Selection Matters

You've just engineered 200 features. Congratulations. Now here's the problem: many of them are useless. Some are random noise that happens to correlate with the target in your training set by accident. Some are near-duplicates of each other. Some measure real effects but are so noisy that they hurt more than they help. And some are genuinely predictive — but which ones?

Feature selection is the process of identifying and keeping only the features that contribute meaningfully to model performance. It's not optional. Unnecessary features increase computation time, make models harder to interpret, and — most importantly — degrade generalization. This is a direct consequence of the curse of dimensionality: as the number of features grows, the amount of data needed to fill the feature space grows exponentially. With too many features and not enough data, models memorize rather than learn.

Filter Methods

Filter methods rank features using statistical measures computed independently of any model. They're fast, scalable, and a good first pass for eliminating obvious dead weight.

Variance Threshold

The simplest filter: remove features with near-zero variance. If a column has the same value in 99% of rows, it carries almost no information. Variance Threshold drops any feature whose variance falls below a specified cutoff.

from sklearn.feature_selection import VarianceThreshold

# Remove features where >80% of values are identical
# For binary features, p(1-p) < 0.8 * 0.2 = 0.16
selector = VarianceThreshold(threshold=0.16)
X_filtered = selector.fit_transform(X)

# Check which features survived
kept_mask = selector.get_support()
print(f"Kept {kept_mask.sum()} of {X.shape[1]} features")

This catches columns that snuck through data cleaning — constants, near-constants, and features with a single dominant value. It won't catch features that have high variance but zero predictive power.

Correlation-Based Selection

Two highly correlated features carry redundant information. If feature A and feature B have a Pearson correlation of 0.97, keeping both wastes a degree of freedom and can destabilize models (especially linear regression, where multicollinearity inflates coefficient variance).

import pandas as pd
import numpy as np

# Compute correlation matrix
corr_matrix = X.corr().abs()

# Select upper triangle (avoid double-counting pairs)
upper = corr_matrix.where(np.triu(np.ones(corr_matrix.shape), k=1).astype(bool))

# Find features with correlation > 0.90
to_drop = [col for col in upper.columns if any(upper[col] > 0.90)]
X_reduced = X.drop(columns=to_drop)
print(f"Dropped {len(to_drop)} highly correlated features")

A common threshold is 0.85–0.95. When dropping one of a correlated pair, keep the one with higher correlation to the target or the one that's easier to interpret.

Mutual Information

Mutual information measures how much knowing one variable tells you about another. Unlike Pearson correlation, it captures non-linear relationships. If feature X and target Y have a complex curved relationship, correlation might be near zero while mutual information is high.

from sklearn.feature_selection import mutual_info_classif, mutual_info_regression

# For classification targets
mi_scores = mutual_info_classif(X, y, random_state=42)

# For regression targets
mi_scores = mutual_info_regression(X, y, random_state=42)

# Rank features by MI score
mi_ranking = pd.Series(mi_scores, index=X.columns).sort_values(ascending=False)
print(mi_ranking.head(10))  # top 10 most informative features

Mutual information is estimated from data, so it can be noisy with small datasets. It also doesn't account for redundancy between features — two features might each have high MI with the target but carry the same information.

Chi-Squared Test

The chi-squared test measures the dependency between two categorical variables. In feature selection, it tests whether each feature is independent of the target class. High chi-squared values mean the feature and target are related. It only works with non-negative features (counts, frequencies, binary indicators).

from sklearn.feature_selection import chi2, SelectKBest

# Select the top 10 features by chi-squared score
selector = SelectKBest(score_func=chi2, k=10)
X_chi2 = selector.fit_transform(X_nonneg, y)

# See the scores
scores = pd.Series(selector.scores_, index=X_nonneg.columns)
print(scores.sort_values(ascending=False).head(10))

ANOVA F-test

The ANOVA F-test measures whether the mean of a numerical feature differs significantly across target classes. It's designed for classification problems with continuous features — the complement of chi-squared. High F-values indicate that the feature separates classes well.

from sklearn.feature_selection import f_classif, SelectPercentile

# Keep top 25% of features by ANOVA F-score
selector = SelectPercentile(score_func=f_classif, percentile=25)
X_anova = selector.fit_transform(X, y)

# For regression targets, use f_regression instead
from sklearn.feature_selection import f_regression

ANOVA assumes linear relationships between features and the target. It misses non-linear patterns entirely. Pair it with mutual information for a more complete picture.

Wrapper Methods

Wrapper methods evaluate feature subsets by actually training a model and measuring performance. They're computationally expensive — training a model for every candidate subset — but they account for feature interactions that filter methods miss.

Forward and Backward Selection

Forward selection starts with zero features and greedily adds the one that improves performance the most, repeating until adding more features stops helping. Backward elimination starts with all features and removes the least useful one at each step. Both are greedy and won't find the globally optimal subset, but they're practical for moderate feature counts.

from sklearn.feature_selection import SequentialFeatureSelector
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100, random_state=42)

# Forward selection: add features one at a time
sfs_forward = SequentialFeatureSelector(
    model, n_features_to_select=10, direction='forward', cv=5
)
sfs_forward.fit(X, y)
selected_forward = X.columns[sfs_forward.get_support()]

# Backward elimination: remove features one at a time
sfs_backward = SequentialFeatureSelector(
    model, n_features_to_select=10, direction='backward', cv=5
)
sfs_backward.fit(X, y)
selected_backward = X.columns[sfs_backward.get_support()]

Recursive Feature Elimination (RFE)

Recursive Feature Elimination trains a model, ranks features by importance (e.g., coefficient magnitude or impurity-based importance), removes the least important features, and repeats. It's backward elimination with a model-driven ranking at each step.

from sklearn.feature_selection import RFECV
from sklearn.linear_model import LogisticRegression

model = LogisticRegression(max_iter=1000)

# RFECV: RFE with cross-validation to find optimal number of features
rfe = RFECV(estimator=model, step=1, cv=5, scoring='accuracy', min_features_to_select=5)
rfe.fit(X, y)

print(f"Optimal number of features: {rfe.n_features_}")
print(f"Selected: {X.columns[rfe.support_].tolist()}")

# Plot number of features vs CV score
import matplotlib.pyplot as plt
plt.plot(range(rfe.min_features_to_select, len(rfe.cv_results_['mean_test_score']) + rfe.min_features_to_select),
         rfe.cv_results_['mean_test_score'])
plt.xlabel('Number of features')
plt.ylabel('CV Accuracy')
plt.title('RFECV: Optimal Feature Count')
plt.show()

RFECV is one of the most practical wrapper methods. It automatically finds the right number of features and uses cross-validation to avoid overfitting the selection itself. The cost: it trains the model many times. For large datasets or complex models, this can be slow.

Embedded Methods

Embedded methods perform feature selection as part of model training. They're faster than wrapper methods (one training run instead of many) and more model-aware than filter methods. These are often the best default choice.

L1 Regularization (Lasso)

L1 regularization adds a penalty proportional to the absolute value of model coefficients. The mathematical effect is remarkable: it drives some coefficients exactly to zero, effectively removing those features from the model. Lasso regression (for regression) and L1-penalized logistic regression (for classification) perform automatic feature selection as a side effect of training.

from sklearn.linear_model import LassoCV, LogisticRegressionCV
import numpy as np

# Regression: Lasso with cross-validated regularization strength
lasso = LassoCV(cv=5, random_state=42)
lasso.fit(X, y)

# Features with non-zero coefficients are "selected"
selected = X.columns[lasso.coef_ != 0]
dropped = X.columns[lasso.coef_ == 0]
print(f"Lasso kept {len(selected)} features, dropped {len(dropped)}")

# Classification: L1-penalized logistic regression
lr_l1 = LogisticRegressionCV(penalty='l1', solver='saga', cv=5, max_iter=5000)
lr_l1.fit(X, y)
selected_cls = X.columns[np.any(lr_l1.coef_ != 0, axis=0)]

The strength of regularization (alpha in Lasso, C in logistic regression) controls how aggressive the selection is. Higher regularization = more features zeroed out. Cross-validation finds the sweet spot.

ElasticNet

ElasticNet combines L1 and L2 regularization. It gets Lasso's ability to zero out features while also handling correlated features better. When two features are highly correlated, Lasso arbitrarily keeps one and drops the other. ElasticNet tends to keep or drop them together, which is often more stable and interpretable.

from sklearn.linear_model import ElasticNetCV

enet = ElasticNetCV(l1_ratio=[0.1, 0.5, 0.7, 0.9, 0.95, 1.0], cv=5)
enet.fit(X, y)

print(f"Best l1_ratio: {enet.l1_ratio_}")  # ratio of L1 vs L2
print(f"Non-zero features: {np.sum(enet.coef_ != 0)}")

Tree-Based Feature Importance

Decision trees and their ensembles (random forest, gradient boosting) naturally rank features by how much they reduce impurity (Gini or entropy for classification, variance for regression) when used for splitting. Features that appear in many splits near the top of trees are "important."

from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.inspection import permutation_importance

# Impurity-based importance (fast but biased toward high-cardinality features)
rf = RandomForestClassifier(n_estimators=200, random_state=42)
rf.fit(X, y)
imp = pd.Series(rf.feature_importances_, index=X.columns).sort_values(ascending=False)
print(imp.head(15))

# Permutation importance (unbiased, slower)
# Measures how much performance drops when a feature's values are shuffled
perm_imp = permutation_importance(rf, X, y, n_repeats=10, random_state=42)
perm_df = pd.Series(perm_imp.importances_mean, index=X.columns).sort_values(ascending=False)
print(perm_df.head(15))
ℹ️ Impurity vs Permutation Importance

Impurity-based importance is fast but has known biases: it favors features with many unique values and features that are correlated with other important features. Permutation importance is more reliable because it measures actual performance impact, but it requires more computation. When in doubt, use permutation importance for final feature selection decisions.

Feature Selection vs Dimensionality Reduction

Feature selection and dimensionality reduction both reduce the number of input dimensions, but they work fundamentally differently. Feature selection picks a subset of original features and discards the rest. You keep "age" and "income" and drop "zip_code." The surviving features are interpretable — they're still age and income.

Dimensionality reduction (PCA, t-SNE, UMAP) creates new features that are combinations of the originals. PCA's first component might be 0.4 × age + 0.7 × income - 0.2 × zip_code. It's a new synthetic feature that captures maximum variance but isn't directly interpretable as any single real-world quantity.

from sklearn.decomposition import PCA
from sklearn.feature_selection import SelectKBest, f_classif

# Feature selection: keeps original features
selector = SelectKBest(f_classif, k=10)
X_selected = selector.fit_transform(X, y)  # shape: (n, 10)
# Each column IS an original feature

# Dimensionality reduction: creates new features
pca = PCA(n_components=10)
X_pca = pca.fit_transform(X)  # shape: (n, 10)
# Each column is a LINEAR COMBINATION of all original features

When do you use which? Feature selection when interpretability matters or when you need to understand which inputs drive predictions. Dimensionality reduction when you have very many correlated features and care about predictive performance more than interpretability. You can also use both — select features first, then reduce dimensionality on the survivors.

Stability of Feature Selection

Here's a dirty secret: run feature selection on slightly different subsets of your data, and you'll often get different features selected. This is the stability problem. It's especially bad with correlated features — if features A and B are nearly identical, the algorithm might pick A in one run and B in the next. Both are equally valid, but the inconsistency is concerning when you're trying to draw scientific conclusions about which features matter.

To assess stability, run your feature selection procedure inside multiple cross-validation folds or bootstrap samples. Count how often each feature is selected. Features that appear in 90%+ of runs are genuinely important. Features that appear in 40% of runs are borderline.

from sklearn.utils import resample
from sklearn.feature_selection import SelectKBest, f_classif
import numpy as np

n_bootstrap = 50
selection_counts = np.zeros(X.shape[1])

for i in range(n_bootstrap):
    X_boot, y_boot = resample(X, y, random_state=i)
    selector = SelectKBest(f_classif, k=10)
    selector.fit(X_boot, y_boot)
    selection_counts += selector.get_support().astype(int)

stability = pd.Series(selection_counts / n_bootstrap, index=X.columns)
print("Features selected in >80% of bootstrap runs:")
print(stability[stability > 0.8].sort_values(ascending=False))

ElasticNet and ensemble importance tend to be more stable than single-filter methods because they account for feature correlations. If stability matters for your application (medical research, regulatory models), use ensemble approaches and report confidence intervals on feature importance.

Multi-Objective Feature Selection

In practice, you rarely optimize for accuracy alone. You might need a model that's accurate, fast to inference, and uses only features available at prediction time. Or you might want the smallest feature set that achieves 95% of the full model's performance — trading a tiny bit of accuracy for massive simplicity.

A practical approach: plot the number of features against cross-validated performance. You'll typically see a curve that rises steeply at first (adding the most important features), then plateaus. The "elbow" of that curve — where adding more features gives diminishing returns — is often a good operating point.

from sklearn.feature_selection import RFECV
from sklearn.ensemble import GradientBoostingClassifier

# RFECV naturally produces this curve
rfe = RFECV(
    GradientBoostingClassifier(n_estimators=100, random_state=42),
    step=1, cv=5, scoring='f1_weighted', min_features_to_select=1
)
rfe.fit(X, y)

# The optimal n_features_ is where CV score peaks
# But you might prefer fewer features if the score plateau is flat
print(f"Optimal: {rfe.n_features_} features, score: {rfe.cv_results_['mean_test_score'][rfe.n_features_ - 1]:.4f}")
⚠️ Practical Workflow

Start with filters (variance threshold + correlation) to cheaply remove junk. Then use embedded methods (tree importance or Lasso) to rank survivors. Finally, if you need precise feature counts, apply RFECV with your actual model. Always validate that the selected features generalize — run feature selection inside cross-validation, not before it. Selecting features on the full dataset and then cross-validating gives you an optimistically biased estimate of performance.
