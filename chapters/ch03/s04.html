Feature Selection & Data Splitting

Chapter 3: Data Fundamentals Selection · Splitting · Sampling
Section Overview

This section combines Feature Selection and Data Splitting & Sampling into one comprehensive reference.

Feature Selection

TL;DR

You've engineered 200 features. Many are noise, some are redundant, and a few are gold. Feature selection finds the ones worth keeping. Three families: filter (fast stats), wrapper (search with model feedback), embedded (built into training). The right choice depends on dataset size and how much compute you can afford.

Why It Matters

Unnecessary features increase training time, degrade generalization, and make models harder to interpret. This is the curse of dimensionality: as features grow, the data you need to fill the space grows exponentially. With too many features and not enough data, models memorize rather than learn.

Filter Methods — Fast First Pass

Filter methods rank features using statistical measures independent of any model. Run these first to eliminate obvious dead weight.

from sklearn.feature_selection import VarianceThreshold, mutual_info_classif
import numpy as np

# 1. Remove near-constant features
selector = VarianceThreshold(threshold=0.01)
X_filtered = selector.fit_transform(X)
print(f"Kept {selector.get_support().sum()} of {X.shape[1]} features")

# 2. Drop highly correlated pairs
corr = pd.DataFrame(X_filtered).corr().abs()
upper = corr.where(np.triu(np.ones(corr.shape), k=1).astype(bool))
to_drop = [col for col in upper.columns if any(upper[col] > 0.90)]

# 3. Mutual information — captures non-linear relationships (unlike Pearson)
mi = mutual_info_classif(X, y, random_state=42)
top_features = np.argsort(mi)[-20:]  # top 20 by MI

Wrapper Methods — Search with Model Feedback

Wrapper methods train a model on subsets of features and use performance to guide search. More accurate than filters, but computationally expensive.

from sklearn.feature_selection import RFECV
from sklearn.ensemble import RandomForestClassifier

# Recursive Feature Elimination with cross-validation
rfe = RFECV(
    estimator=RandomForestClassifier(n_estimators=100, random_state=42),
    step=1, cv=5, scoring="f1", min_features_to_select=5
)
rfe.fit(X_train, y_train)
print(f"Optimal features: {rfe.n_features_}")
X_selected = rfe.transform(X_train)

RFECV is the go-to wrapper method. It trains, removes the weakest feature, retrains, and repeats — using CV to find the optimal number. The cost: with 200 features and 5-fold CV, you're training ~1,000 models.

Embedded Methods — Selection During Training

Embedded methods get feature importance for free during model training. No extra search loop.

from sklearn.feature_selection import SelectFromModel
from sklearn.linear_model import LassoCV
from sklearn.ensemble import RandomForestClassifier

# L1 (Lasso) — drives irrelevant feature coefficients to exactly zero
lasso = LassoCV(cv=5, random_state=42).fit(X_train, y_train)
important = np.where(lasso.coef_ != 0)[0]
print(f"Lasso kept {len(important)} features")

# Tree importance — built into Random Forest / XGBoost
rf = RandomForestClassifier(n_estimators=200, random_state=42).fit(X_train, y_train)
importances = rf.feature_importances_

# SelectFromModel wraps any estimator with feature_importances_ or coef_
selector = SelectFromModel(rf, threshold="median")
X_selected = selector.fit_transform(X_train, y_train)

Which Method When

MethodSpeedAccuracyBest For
Variance ThresholdInstantLow (only catches constants)First pass cleanup
Correlation filterFastModerateRemoving redundant features
Mutual InformationFastGood (captures non-linear)Ranking features by relevance
RFECVSlowHighFinding optimal subset when compute allows
L1 / LassoFastGoodLinear models, automatic sparsity
Tree importanceFast (byproduct of training)GoodWhen using tree-based models anyway

Practical recipe: Start with variance threshold + correlation filter (30 seconds). Then use mutual information or tree importance to rank what's left. If you have compute budget, finish with RFECV on the top candidates.

Feature Selection vs. Dimensionality Reduction

Feature selection removes features. PCA and other dimensionality reduction methods transform them into new composite features. Selection preserves interpretability (you know which original features survived). Reduction often captures more variance but produces features you can't name. Use selection when interpretability matters. Use reduction when you just need lower dimensions.

What You Should Now Be Able To Do

  • Use RFECV when compute budget allows and explain why it's more accurate than filters
  • Leverage L1 regularization and tree importance as embedded selection methods
  • Choose between feature selection and dimensionality reduction based on whether interpretability matters
  • Explain the curse of dimensionality and why more features isn't always better

Data Splitting & Sampling

TL;DR

How you split your data determines whether your performance estimates are trustworthy. How you sample determines what your model sees during training. A careless split can make a mediocre model look brilliant. Wrong sampling can make a model blind to rare events. This section covers train/val/test splits, stratification, time-based splits, group splits, cross-validation, and class imbalance handling — all the ways you can partition data correctly (or screw it up).

Train / Validation / Test — Three Distinct Jobs

Training set — the model learns from these examples. Validation set — you tune hyperparameters against this. Test set — your final, unbiased performance number. Touch it once. Report it honestly.

Common ratios: 70/15/15 or 80/10/10. With millions of rows, 1-5% for val/test is fine. With small datasets, skip the validation set and use cross-validation instead.

from sklearn.model_selection import train_test_split

# Two-step split: carve out test first, then split remainder
X_temp, X_test, y_temp, y_test = train_test_split(X, y, test_size=0.15, random_state=42)
X_train, X_val, y_train, y_val = train_test_split(X_temp, y_temp, test_size=0.176, random_state=42)
# 0.176 of 0.85 ≈ 0.15 of total

Critical rule: once you look at test results and go back to change your model, the test set is no longer unbiased. It's become a second validation set. Guard it like a sealed envelope.

Stratified Splitting

When 95% of your data is class A and 5% is class B, a random split can give you a fold with zero class B examples. Stratified splitting preserves the class distribution in every set.

# Always use stratify for classification
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# Stratified K-Fold
from sklearn.model_selection import StratifiedKFold
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
for train_idx, val_idx in skf.split(X, y):
    X_fold_train, X_fold_val = X[train_idx], X[val_idx]

For regression, stratify on binned target values to ensure each fold has a similar distribution.

Time-Based Splitting

With temporal data, random splits let the model peek at the future. Always split on time: train on the past, validate on the near future, test on the further future.

# Never shuffle time series — chronological split
df = df.sort_values("date")
train = df[df["date"] < "2023-01-01"]
val   = df[df["date"].between("2023-01-01", "2023-06-30")]
test  = df[df["date"] >= "2023-07-01"]

# TimeSeriesSplit for cross-validation (expanding window)
from sklearn.model_selection import TimeSeriesSplit
tscv = TimeSeriesSplit(n_splits=5)
for train_idx, val_idx in tscv.split(X):
    pass  # each fold: train on all data before the val window

Group-Based Splitting

When observations are grouped (multiple samples per patient, multiple reviews per user), random splitting leaks information — the model sees the same patient in both train and test, learning patient-specific patterns rather than generalizable ones.

from sklearn.model_selection import GroupKFold

# Ensures all samples from the same group stay in the same fold
gkf = GroupKFold(n_splits=5)
for train_idx, val_idx in gkf.split(X, y, groups=df["patient_id"]):
    pass

Cross-Validation

With small datasets, a single train/val split is too noisy — the val performance varies wildly depending on which examples end up where. K-fold cross-validation uses every example for both training and validation across K rounds, giving you a more stable estimate.

from sklearn.model_selection import cross_val_score

scores = cross_val_score(model, X, y, cv=5, scoring="f1")
print(f"F1: {scores.mean():.3f} ± {scores.std():.3f}")

5-fold is the default. 10-fold is more stable but 2× more expensive. Leave-one-out is theoretically appealing but practically too slow for anything beyond tiny datasets.

Handling Class Imbalance

When 99% of your data is "normal" and 1% is "fraud," the model learns to always predict "normal" and gets 99% accuracy — while catching zero fraud. Three approaches:

Resampling

from imblearn.over_sampling import SMOTE
from imblearn.under_sampling import RandomUnderSampler
from imblearn.pipeline import Pipeline as ImbPipeline

# SMOTE — synthesize new minority samples (not just copies)
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X_train, y_train)

# Combined: oversample minority, undersample majority
pipeline = ImbPipeline([
    ("over", SMOTE(sampling_strategy=0.5)),
    ("under", RandomUnderSampler(sampling_strategy=0.8)),
])
X_balanced, y_balanced = pipeline.fit_resample(X_train, y_train)

SMOTE creates synthetic minority examples by interpolating between real ones. It's better than random oversampling (which duplicates existing examples and invites overfitting). Never apply SMOTE to validation or test data — only to training data.

Class Weights

# Most sklearn models support class_weight
from sklearn.ensemble import RandomForestClassifier
rf = RandomForestClassifier(class_weight="balanced", random_state=42)
# "balanced" automatically weights classes inversely proportional to frequency

Evaluation Metrics

With imbalanced data, accuracy is meaningless. Use precision, recall, F1, or AUC-ROC instead. See Chapter 4 for the full metrics deep-dive.

Data Leakage — The Silent Killer

Leakage happens when information from outside the training set bleeds into it, inflating your metrics. Common sources:

  • Fitting preprocessors on full data — if you fit a scaler on all data before splitting, the scaler has seen test data. Fit on train only.
  • Target encoding without CV folds — the encoding uses the target, which leaks future information.
  • Time leakage — using future data to predict the past.
  • Feature leakage — a feature that's computed from the target (e.g., "approved_amount" when predicting "loan_approved").

The telltale sign: suspiciously high validation accuracy that craters in production. If your model looks too good, check for leakage first.

What You Should Now Be Able To Do

  • Implement a proper train/val/test split and explain each set's purpose
  • Use stratified splitting for classification and binned stratification for regression
  • Apply time-based and group-based splits when random splitting would leak information
  • Handle class imbalance with SMOTE, class weights, and appropriate metrics
  • Identify the four common sources of data leakage
  • Choose between a held-out validation set and cross-validation based on dataset size
