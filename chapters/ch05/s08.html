Generalized Linear Models (GLMs)

Chapter 5: Supervised Learning 4 subtopics ~15 min read
TL;DR

Linear regression and logistic regression look like completely different algorithms, but they're both special cases of the same framework: Generalized Linear Models. A GLM connects a linear combination of features to the predicted outcome through a link function, and assumes the response follows a distribution from the exponential family. Change the distribution and the link, and you get a different model — Gaussian for continuous data, Binomial for binary, Poisson for counts, Gamma for positive-valued, and more. Understanding GLMs means you stop memorizing individual algorithms and start reasoning about what your data actually looks like.

Why GLMs Exist

Here's the situation. You learn linear regression first. Then you learn logistic regression and it feels like a completely different thing — there's a sigmoid, you're predicting probabilities, the loss function changes. Then someone mentions Poisson regression for count data and you think, great, yet another separate algorithm to memorize.

GLMs are the unifying insight. John Nelder and Robert Wedderburn published the framework in 1972, and the key idea is elegant: all these models share the same structure. You have a linear combination of your features (the linear predictor), you push it through a function that maps it to the right scale for your response variable (the link function), and you assume the noise in your data follows a specific distribution from a family called the exponential family. That's the whole framework. Swap the distribution and the link function, and you recover linear regression, logistic regression, Poisson regression, and many more — all as special cases.

The Framework

Exponential Family Distributions

The exponential family is a class of probability distributions whose density can be written in a specific canonical form: f(y | θ, φ) = exp[(yθ − b(θ))/a(φ) + c(y, φ)]. That looks intimidating, but what matters is this: the Gaussian (normal), Binomial, Poisson, Gamma, Inverse Gaussian, and Negative Binomial distributions all fit this form. So do many others. If your response variable follows one of these distributions, you can build a GLM for it.

Why does this matter? Because the exponential family form gives you nice mathematical properties for free. The mean of the distribution is E[Y] = b′(θ) — the derivative of that b(θ) function. The variance is Var(Y) = a(φ) · b″(θ). This means the variance is tied to the mean through a known function, which is exactly what happens in real count data (higher counts, more spread) and in proportions (variance peaks at 0.5). The exponential family makes these relationships explicit and mathematically tractable.

The Three Components

Every GLM has three components. First, the random component: you pick a distribution from the exponential family for your response variable Y. Second, the systematic component (also called the linear predictor): η = β₀ + β₁x₁ + β₂x₂ + ⋯ + βₚxₚ. This is just a weighted sum of your features — the same thing you'd compute in ordinary linear regression. Third, the link function g(·): it connects the mean of the response μ = E[Y] to the linear predictor through the equation g(μ) = η. The link function is what lets you model outcomes that live on different scales — probabilities between 0 and 1, non-negative counts, strictly positive values — using the same linear machinery.

Link Function and Canonical Link

The link function g maps the mean μ to the linear predictor η: g(μ) = η. Its inverse g⁻¹ maps back: μ = g⁻¹(η). You can technically choose any monotonic differentiable function as a link. But each exponential family distribution has a canonical link — the link function that makes the math simplest by setting the linear predictor equal to the natural parameter θ of the distribution. For the Gaussian, the canonical link is the identity (μ = η). For the Binomial, it's the logit (log(μ/(1−μ)) = η). For the Poisson, it's the log (log(μ) = η). Using the canonical link isn't required, but it simplifies estimation and guarantees certain nice properties like concavity of the log-likelihood.

Variance Function

The variance function V(μ) describes how the variance of the response changes with its mean. In linear regression, you assume constant variance — V(μ) = 1 (the variance doesn't depend on μ). In Poisson regression, V(μ) = μ — the variance equals the mean, which is exactly what you observe in count data. In logistic regression, V(μ) = μ(1−μ), peaking at μ = 0.5. The variance function is what makes GLMs honest about heteroscedasticity. Instead of assuming equal variance everywhere and hoping for the best, you model the variance structure directly.

Common GLMs — And How They Connect

Gaussian with Identity Link → Linear Regression

Take a Gaussian distribution for Y, use the identity link g(μ) = μ, and you get: μ = β₀ + β₁x₁ + ⋯. That's ordinary linear regression. The response is continuous, the variance is constant, the link does nothing — just passes the linear predictor straight through as the mean. This is the simplest GLM, and the one you already know.

Binomial with Logit Link → Logistic Regression

Take a Binomial distribution for Y (each observation is 0 or 1), use the logit link g(μ) = log(μ/(1−μ)), and you get: log(p/(1−p)) = β₀ + β₁x₁ + ⋯. Invert the logit and you recover the sigmoid: p = 1/(1 + exp(−η)). That's logistic regression. The link function is what keeps predicted probabilities between 0 and 1. The variance function μ(1−μ) captures the fact that uncertainty is highest when the predicted probability is near 0.5.

💡 Key Insight

Linear regression and logistic regression differ in only two choices: the distribution (Gaussian vs. Binomial) and the link function (identity vs. logit). Everything else — the linear predictor structure, the estimation approach, the testing framework — is the same.

Poisson with Log Link → Count Data

Your response is a non-negative integer — number of customer complaints per week, number of accidents at an intersection, number of species in a habitat. The Poisson distribution is the natural starting point. The canonical link is the log: log(μ) = η, which means μ = exp(η). The log link ensures that the predicted mean is always positive, which makes sense for counts. The variance equals the mean (V(μ) = μ), capturing the fact that higher counts tend to have more variability.

Gamma with Inverse Link → Positive Continuous

The Gamma distribution models positive continuous responses where the variance increases with the mean — think insurance claim amounts, survival times, or manufacturing defect sizes. The canonical link is the inverse: g(μ) = 1/μ. In practice, the log link is often used instead because it's more interpretable. The variance function is V(μ) = μ², meaning the standard deviation is proportional to the mean (constant coefficient of variation).

Negative Binomial → Overdispersed Counts

Real count data is often overdispersed — the variance is larger than the mean, violating the Poisson assumption. The Negative Binomial distribution adds an extra parameter that allows the variance to exceed the mean: Var(Y) = μ + μ²/k, where k controls the overdispersion. When k → ∞, it reduces to Poisson. In practice, if your Poisson model shows a deviance much larger than the residual degrees of freedom, try a Negative Binomial instead.

Tweedie → Zero-Inflated Continuous

The Tweedie distribution is a power family with variance function V(μ) = μᵖ. By varying the power parameter p, you get different distributions: p = 0 gives Gaussian, p = 1 gives Poisson, p = 2 gives Gamma, p = 3 gives Inverse Gaussian. The interesting case is 1 < p < 2: a compound Poisson-Gamma distribution that produces exact zeros mixed with positive continuous values. This is exactly what you see in insurance claims (many months with $0 claims, some months with positive amounts) and rainfall data (many days with zero rain). Tweedie models handle this naturally without needing a separate zero-inflation mechanism.

Estimation

Maximum Likelihood via IRLS

GLMs are estimated by maximum likelihood estimation (MLE). You write down the log-likelihood of the data under your chosen distribution, then find the parameter values β that maximize it. Unlike linear regression, there's no closed-form solution — you need an iterative algorithm. The standard approach is Iteratively Reweighted Least Squares (IRLS). At each iteration, IRLS constructs a weighted least-squares problem where the weights depend on the current parameter estimates, solves it, updates the estimates, and repeats. It's essentially Newton-Raphson applied to the score equations, and it converges quickly in practice — usually in 5–25 iterations.

Deviance and Model Comparison

Deviance is the GLM equivalent of the residual sum of squares. It measures how far your model is from a perfect (saturated) model that fits each observation exactly. Deviance = 2 × [log-likelihood(saturated model) − log-likelihood(your model)]. Smaller deviance means better fit. You can compare nested models by looking at the difference in deviance, which follows a chi-squared distribution under the null hypothesis. This is the GLM version of the F-test in linear regression.

Wald, Score, and Likelihood Ratio Tests

Three classical tests for individual coefficients or groups of coefficients. The Wald test asks: is the estimate far from zero relative to its standard error? It's the z-test you see in model summary output. The Score test (Rao test) only requires fitting the null model — useful when the full model is expensive. The Likelihood Ratio (LR) test compares the log-likelihoods of nested models directly. In large samples, all three give similar results. In small samples, the LR test is generally most reliable. The Wald test can misbehave when coefficients are large (Hauck-Donner effect in logistic regression).

Quasi-Likelihood Methods

Sometimes you know the mean-variance relationship but don't want to commit to a full distributional assumption. Quasi-likelihood lets you specify only the link function and the variance function, without specifying the complete distribution. You lose the ability to compute a true likelihood or AIC, but you still get consistent parameter estimates and valid standard errors (adjusted for the estimated dispersion). This is common in overdispersed Poisson and binomial models — you fit a quasi-Poisson or quasi-binomial model that handles the extra variance without switching distributions.

Poisson Regression in Practice

Let's fit a Poisson GLM with statsmodels. Suppose you have count data — number of insurance claims per policyholder — and you want to model it as a function of age and vehicle type. 

import numpy as np
import pandas as pd
import statsmodels.api as sm
from statsmodels.formula.api import glm

# Simulated count data: insurance claims
np.random.seed(42)
n = 500
age = np.random.uniform(20, 70, n)
is_suv = np.random.binomial(1, 0.4, n)

# True model: log(mu) = -1.5 + 0.02*age + 0.6*is_suv
eta = -1.5 + 0.02 * age + 0.6 * is_suv
mu = np.exp(eta)
claims = np.random.poisson(mu)

df = pd.DataFrame({
    'claims': claims,
    'age': age,
    'is_suv': is_suv
})

# Fit Poisson GLM
model = glm('claims ~ age + is_suv',
            data=df,
            family=sm.families.Poisson()).fit()

print(model.summary())
print(f"
Deviance: {model.deviance:.2f}")
print(f"Pearson chi2 / df: {model.pearson_chi2 / model.df_resid:.3f}")

The output summary table gives you coefficient estimates, standard errors, z-values, and p-values — all from the Wald test. The coefficients are on the log scale because of the log link. To interpret: exp(0.02) ≈ 1.02, meaning each additional year of age increases the expected claim count by about 2%. The Pearson chi-squared divided by the residual degrees of freedom should be near 1.0 for a well-fitting Poisson model. If it's much larger, you have overdispersion and should consider a Negative Binomial or quasi-Poisson. 

# Check for overdispersion and try Negative Binomial if needed
disp_ratio = model.pearson_chi2 / model.df_resid
print(f"Dispersion ratio: {disp_ratio:.3f}")

if disp_ratio > 1.5:
    print("Overdispersion detected — fitting Negative Binomial")
    from statsmodels.genmod.families import NegativeBinomial
    nb_model = glm('claims ~ age + is_suv',
                    data=df,
                    family=NegativeBinomial()).fit()
    print(nb_model.summary())

Diagnostics

Residual Types

GLMs have several types of residuals because the raw residuals (y − μ̂) don't have constant variance. Pearson residuals divide by the standard deviation: (y − μ̂) / √V(μ̂). Deviance residuals are derived from the contribution of each observation to the total deviance — they tend to be more symmetrically distributed. Anscombe residuals apply a transformation designed to make them approximately normal. In practice, deviance residuals are the most commonly used for diagnostic plots.

Leverage and Influence

Just like linear regression, some observations can have outsized influence on the fitted model. The hat matrix in GLMs is a weighted version of the linear regression hat matrix. The diagonal elements (leverages) tell you how much each observation influences its own fitted value. Cook's distance combines leverage and residual size to flag observations that, if removed, would substantially change the fitted model. Always check for high-influence points, especially in smaller datasets.

Overdispersion Detection

For Poisson and binomial models, the mean-variance relationship is fixed by the distribution. If the actual variance in your data exceeds what the model predicts, you have overdispersion. The simplest diagnostic: compute the Pearson chi-squared statistic divided by the residual degrees of freedom. Values near 1 are fine. Values above 1.5–2 suggest overdispersion. Consequences of ignoring it: your standard errors will be too small, your p-values too optimistic, and you'll declare effects significant when they're not. Fixes: use quasi-likelihood (multiply standard errors by √φ̂), switch to Negative Binomial, or use robust standard errors.

Goodness-of-Fit: AIC and BIC

The Akaike Information Criterion (AIC) balances model fit against complexity: AIC = −2·log-likelihood + 2k, where k is the number of parameters. Lower is better. The Bayesian Information Criterion (BIC) penalizes complexity more heavily: BIC = −2·log-likelihood + k·log(n). BIC tends to select simpler models than AIC, especially with large sample sizes. Both are useful for comparing non-nested models on the same data — something the deviance test can't do. One caveat: AIC and BIC require a proper likelihood, so they're not available for quasi-likelihood models.

⚠ Common Pitfall

When comparing GLMs with different link functions or distributions, AIC values are only comparable if they're computed on the same response variable with the same scale. You cannot compare AIC from a Gaussian model with AIC from a Poisson model unless the likelihood contributions are on the same scale. Compare models within a family or use cross-validation instead.

Choosing the Right GLM

The decision tree is straightforward. What type is your response variable? Continuous and roughly symmetric → Gaussian. Binary (0/1) → Binomial. Non-negative integers → Poisson (or Negative Binomial if overdispersed). Positive continuous → Gamma. Positive continuous with exact zeros → Tweedie. After choosing the distribution, start with the canonical link. If the model fits well and residuals look reasonable, you're done. If not, try an alternative link or a more flexible distribution. The beauty of the GLM framework is that all these choices are modular — you're turning knobs within a single coherent system, not switching between unrelated algorithms.
