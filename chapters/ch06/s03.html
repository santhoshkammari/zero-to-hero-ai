Anomaly Detection

Chapter 6: Unsupervised Learning Finding the weird stuff in your data

The Fundamental Challenge

Every interesting anomaly detection problem shares the same headache: anomalies are rare and diverse. Fraudsters invent new tricks. Manufacturing defects appear in forms nobody anticipated. Network intrusions exploit zero-day vulnerabilities. You can't collect a representative training set of "all the bad stuff" because the bad stuff keeps changing shape.

So you flip the problem. Instead of learning what anomalies look like, you learn what normal looks like — and flag anything that doesn't fit. That's the unifying idea behind every method in this section.

💡 Why not just use a classifier?

A model that always predicts "normal" on fraud data achieves 99.97% accuracy. Accuracy is useless here. You need methods designed for extreme class imbalance — or better yet, methods that don't need labels at all.

Statistical Methods (Quick Pass)

Z-Score

Compute z = (x − μ) / σ. Flag anything beyond ±3σ. Works for univariate, roughly Gaussian data. You already know this one.

Mahalanobis Distance

Z-score breaks down when features are correlated. A person who's 195 cm and 60 kg might look fine on each feature individually, but the combination is unusual — tall people tend to weigh more. Mahalanobis distance fixes this by incorporating the covariance matrix: D = √((x − μ)ᵀ Σ⁻¹ (x − μ)).

Think of it as measuring "how many standard deviations away" but in the shape of the data's ellipse, not a circle. Points with large Mahalanobis distances are anomalous even when no single feature looks extreme. The catch: you need a reliable covariance estimate, which falls apart with more features than samples or when outliers contaminate the covariance itself.

Isolation Forest

This is your default. Reach for it first unless you have a good reason not to.

The Intuition: Anomalies Are Easy to Isolate

Pick a random feature. Pick a random split point between its min and max. You've just cut the data in two. Keep splitting recursively — like building a random decision tree — until every point sits alone.

Here's the key insight: if a point is far from the crowd, a random cut will separate it quickly — maybe in 2 or 3 splits. A normal point buried in a dense cluster needs dozens of splits before it's alone. Path length (number of splits to isolate a point) becomes the anomaly score. Short path → anomaly. Long path → normal.

Build a forest of these random trees, average the path lengths, and you have a robust anomaly score. No density estimation, no distance calculations, no distributional assumptions. It runs in O(n) time and handles high-dimensional data gracefully.

Production-Ready Code

import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.datasets import make_blobs

# Synthetic data: 500 normal points + 20 anomalies
np.random.seed(42)
X_normal, _ = make_blobs(n_samples=500, centers=1,
                          cluster_std=1.0, random_state=42)
X_anomalies = np.random.uniform(low=-6, high=6, size=(20, 2))
X = np.vstack([X_normal, X_anomalies])
true_labels = np.array([1]*500 + [-1]*20)  # 1=normal, -1=anomaly

# Fit and predict
iso = IsolationForest(
    n_estimators=200,        # more trees → more stable scores
    contamination=0.05,      # expected anomaly fraction
    random_state=42
)
preds = iso.fit_predict(X)   # 1=normal, -1=anomaly
scores = iso.decision_function(X)  # lower = more anomalous

# Evaluate
true_anom = true_labels == -1
caught = (preds[true_anom] == -1).sum()
print(f"Caught {caught}/{true_anom.sum()} true anomalies")
print(f"Total flagged: {(preds == -1).sum()}")

The contamination Parameter

This is the one knob that matters. It tells the model what fraction of your training data is anomalous, and it shifts the decision threshold accordingly. Set it too high and normal points get flagged. Too low and anomalies slip through.

In practice, use domain knowledge: "fraud is roughly 0.1% of transactions" → contamination=0.001. If you have no idea, use "auto" and tune the threshold on the raw decision_function scores instead.

Local Outlier Factor (LOF)

Isolation Forest treats the whole dataset uniformly. But what if your normal data has clusters of wildly different densities — a tight downtown cluster and a sparse suburban cluster? A point between them might fool Isolation Forest but LOF catches it.

LOF compares each point's local density to the density of its neighbors. If your neighborhood is much sparser than your neighbors' neighborhoods, you're the odd one out. Scores near 1.0 are normal. Scores well above 1.0 are suspicious.

from sklearn.neighbors import LocalOutlierFactor

lof = LocalOutlierFactor(
    n_neighbors=20,          # neighborhood size
    contamination=0.05,
    novelty=False            # True if scoring new unseen data
)
lof_preds = lof.fit_predict(X)
lof_scores = lof.negative_outlier_factor_  # more negative = more anomalous

caught_lof = (lof_preds[true_anom] == -1).sum()
print(f"LOF caught {caught_lof}/{true_anom.sum()} true anomalies")
⚙️ novelty=True vs False

novelty=False: LOF scores the training data itself (transductive). Call fit_predict.

novelty=True: Train on clean data, then call predict on new incoming points. This is what you want in production — train on yesterday's known-good data, score today's stream.

When LOF beats Isolation Forest: datasets with clusters of varying density. When it doesn't: high-dimensional data (distance metrics degrade) or very large datasets (LOF is O(n²) for neighbor search without tree structures).

One-Class SVM

One-Class SVM learns a tight decision boundary around normal data. It maps points into a high-dimensional kernel space (usually RBF) and finds the hyperplane that separates the data from the origin with maximum margin. Anything on the wrong side of the boundary is flagged.

The nu parameter (ν) controls the expected outlier fraction — set nu=0.05 and the model allows up to 5% of training points to fall outside the boundary. It works well on small datasets with clean boundaries, but training is O(n²)–O(n³), so it doesn't scale to millions of rows. For large data, stick with Isolation Forest.

Autoencoder-Based Detection

Train a neural network autoencoder on normal data only. It learns to compress and reconstruct normal patterns. At inference, anomalies produce high reconstruction error because the network has never seen anything like them.

This approach shines on complex, high-dimensional data (images, sensor streams) where hand-crafted features fall short. The tricky part is setting the reconstruction error threshold — too low and you drown in false positives; too high and you miss real anomalies. We'll dig deeper into autoencoders in the deep learning chapters.

Evaluation: Harder Than You Think

You can't just use accuracy. A model that never flags anything is 99.97% "accurate" on fraud data. Here's what actually works:

Precision@k

Your fraud team can review 200 alerts per day. If 180 of the top 200 scored points are real fraud, that's Precision@200 = 0.9. This is the metric that maps directly to operational reality.

PR-AUC over ROC-AUC

ROC-AUC looks great even for mediocre models when anomalies are rare — the false positive rate stays tiny in absolute terms. Precision-Recall AUC is more honest because it focuses on how well you do among the things you flag, not across the whole dataset. When your positive class is 0.1% of the data, PR-AUC is the metric you want.

🎯 Rule of thumb

If your anomaly rate is below 1%, report PR-AUC. If stakeholders need one number, give them Precision@k where k matches the team's daily review capacity.

Which Method When?

Method Best When Avoid When Scales To
Z-score / Mahalanobis Few features, roughly Gaussian Skewed, multimodal, high-dim Any size
Isolation Forest General default — no assumptions Variable-density clusters Millions of rows
LOF Variable-density clusters High dimensions, huge datasets ~100K rows
One-Class SVM Small data, clean boundary Large data, messy boundaries ~10K rows
Autoencoder Complex patterns, images, sequences Small data, no DL infra Any size (GPU)
