Nice to Know

Chapter 6: Unsupervised Learning Interesting but not essential — awareness level

These topics are real and show up in the wild, but they're not the tools you'll reach for daily. Skim for awareness — you'll know when to look them up.

Association Rules (Apriori, FP-Growth)

Classic market basket analysis: "customers who bought diapers also bought beer." The Apriori algorithm finds frequent itemsets by pruning candidates level-by-level; FP-Growth does it faster with a compressed tree structure. You measure rules with three metrics:

  • Support — how often items appear together: P(X ∩ Y)
  • Confidence — conditional probability: P(Y | X)
  • Lift — does the combo beat random chance? confidence / P(Y). Lift > 1 means there's a real association.

When you'd use it: Retail analytics, recommendation engines in e-commerce, and ML interview questions. Less common in modern pipelines — collaborative filtering and embeddings have mostly taken over.

Topic Modeling (LDA)

Latent Dirichlet Allocation treats each document as a mixture of topics, and each topic as a mixture of words. It's a generative probabilistic model — imagine each document was created by picking topics from a distribution, then picking words from each topic. You reverse-engineer those hidden distributions from the observed text.

For years, LDA was the way to discover themes in a text corpus. It still works, but it's been largely superseded by embedding-based approaches and LLMs that can just tell you what a document is about. Worth knowing for legacy systems and understanding the history of NLP.

BERTopic

The modern answer to topic modeling. BERTopic generates document embeddings (via sentence-transformers), reduces dimensions with UMAP, clusters with HDBSCAN, and then extracts topic representations using c-TF-IDF. The result: coherent, interpretable topics without the bag-of-words assumptions of LDA.

When you'd use it: If you need topic modeling in 2024+, start here. It handles short texts, supports dynamic topics over time, and integrates with any embedding model. pip install bertopic and you're five lines of code away from results.

Non-negative Matrix Factorization (NMF)

Factorize a matrix V ≈ W × H with the constraint that all values in W and H are non-negative. This forces a parts-based decomposition — components can only add, never subtract. In a document-term matrix, each "topic" is a weighted combination of words (all positive), which makes the output far more interpretable than PCA.

When you'd use it: Topic modeling (an alternative to LDA), recommender systems, and audio source separation. Also shows up as a quick-and-dirty feature extractor for non-negative data like images or spectrograms.

Spectral Clustering — The Math Behind It

We covered spectral clustering in the clustering section — build a similarity graph, use its eigenvectors, cluster in that space. If you want the theory: it's about the graph Laplacian (L = D − W), and the eigenvectors of L encode the optimal normalized cuts of the graph. The Fiedler vector (second-smallest eigenvector) gives you the best binary partition.

When you'd dig deeper: When you're dealing with non-convex clusters where K-Means fails, or when you need to understand why your spectral clustering results look the way they do. The theory connects beautifully to random walks and electrical networks.

BIRCH, CURE, and Mini-Batch Methods

When your data doesn't fit in memory, you need clustering algorithms designed for scale. BIRCH builds a compact tree structure (the CF-tree) that summarizes the data in a single pass — ideal for very large datasets. CURE represents each cluster with multiple scattered points instead of a single centroid, which helps with non-spherical shapes. Mini-Batch K-Means approximates regular K-Means by updating centroids on small random batches.

When you'd use them: Millions of data points, streaming data, or limited memory. Mini-Batch K-Means is in scikit-learn and is often "good enough" — try it before reaching for the others.

Fuzzy C-Means

Like K-Means, but each point gets a degree of membership in every cluster instead of a hard assignment. If a data point sits between two clusters, Fuzzy C-Means says "60% cluster A, 40% cluster B" rather than forcing a choice. Similar in spirit to GMMs, but without assuming Gaussian distributions.

When you'd use it: Image segmentation (especially medical imaging), control systems, and situations where hard boundaries don't make sense. Niche but still alive in specific domains.

Deep Clustering (DEC, DeepCluster)

The idea: use deep networks (typically autoencoders) to learn representations that are simultaneously good for reconstruction and clustering. DEC (Deep Embedded Clustering) pretrains an autoencoder, then fine-tunes the encoder by minimizing KL divergence against a sharpened target distribution. DeepCluster alternates between clustering the current representations and using those cluster assignments as pseudo-labels for training.

When you'd use it: When traditional clustering on raw features fails and you suspect there's a better representation hiding in the data. Active research area — we'll see more of this in the deep learning chapters.

None of these are things you need to master right now. But when someone mentions "lift" in a product meeting, or a legacy codebase has an LDA pipeline, or an interviewer asks about fuzzy clustering — you'll know what they're talking about and where to look.
