EM Algorithm & Gaussian Mixtures

Chapter 6: Unsupervised Learning Soft clustering · Hidden variables · Interview favorite

The Chicken-and-Egg Problem

You've got data that clearly came from multiple groups — the scatter plot practically screams "there are three blobs here." But nobody labeled which point belongs to which blob. If you knew the labels, estimating each group's mean and variance would be trivial. If you knew the means and variances, assigning labels would be trivial. You know neither.

That circular dependency is the hidden-variable problem, and it's everywhere. Customer segments you can't observe. Topics that generated words in a document. Regulatory pathways behind gene expression. The Expectation-Maximization algorithm breaks the deadlock with a beautifully simple idea: guess one, solve the other, repeat.

The EM Loop

EM alternates between two steps until convergence. That's it. The entire algorithm fits on a napkin.

E-Step: "Who Probably Belongs Where?"

Given the current parameter estimates (means, variances, mixing weights), compute responsibilities — the probability that each data point was generated by each component. These are soft assignments: point 42 might be 73% cluster A and 27% cluster B. You're using Bayes' theorem here, nothing fancier.

For a Gaussian mixture with K components, the responsibility of component k for point x_i is:

r_ik = π_k · N(x_i | μ_k, Σ_k) / Σ_j [π_j · N(x_i | μ_j, Σ_j)]

Where π_k is the mixing weight (prior probability of component k) and N(·) is the Gaussian density. Points near a component's mean get high responsibility for that component. Points in overlap regions split their responsibility.

M-Step: "Given Those Assignments, What Are the Best Parameters?"

Treat the responsibilities as fractional counts and re-estimate everything. The new mean of component k is a weighted average of all data points, where the weights are the responsibilities toward k. The new covariance is the weighted scatter matrix. The new mixing weight is the average responsibility.

N_k  = Σ_i r_ik                          # effective number of points in cluster k
μ_k  = (1/N_k) · Σ_i r_ik · x_i          # weighted mean
Σ_k  = (1/N_k) · Σ_i r_ik · (x_i - μ_k)(x_i - μ_k)ᵀ   # weighted covariance
π_k  = N_k / N                            # updated mixing weight

Each M-step is just weighted maximum likelihood — the same formulas you'd use if you had hard labels, except weighted by soft assignments.

The Intuition That Matters

💡 Why EM works

If you knew the hidden labels → parameter estimation is trivial (just compute group statistics). If you knew the parameters → inferring labels is trivial (just apply Bayes' rule). EM alternates between these two "easy" problems, bootstrapping its way from a random guess to a good solution. Each iteration is guaranteed to increase (or maintain) the data log-likelihood.

EM Generalizes K-Means

This is the interview insight people miss. K-Means is a special case of EM where:

  • Responsibilities are hard (0 or 1) instead of soft
  • All covariances are spherical and equal (σ²I)
  • Mixing weights are uniform

K-Means' "assign to nearest centroid" step is a degenerate E-step where the closest cluster gets responsibility 1.0 and everything else gets 0. The "recompute centroids" step is an M-step with those hard assignments. GMMs relax all three constraints, which is why they can find elliptical clusters, handle clusters of different sizes, and give you uncertainty estimates.

Gaussian Mixture Models in Practice

GMMs are the poster child for EM and the form you'll use 90% of the time. Think of them as "clustering with soft assignments and shape." Each cluster is a Gaussian with its own mean, covariance, and weight.

Covariance Types

The covariance type controls cluster shape and the number of free parameters. This choice matters enormously for both quality and computational cost.

Type Shape Parameters per Component (d dims) When to Use
full Arbitrary ellipsoids d(d+1)/2 Enough data, clusters have different orientations
tied Same ellipsoid, different centers d(d+1)/2 shared Clusters differ mainly in location
diag Axis-aligned ellipsoids d Features roughly independent, moderate data
spherical Spheres (= K-Means shape) 1 Little data, or clusters truly round

full is the most expressive but needs O(d²) parameters per component — it can overfit in high dimensions. diag is the practical sweet spot for many problems. Start there and upgrade to full only if you have the data to support it.

When GMMs Beat K-Means

K-Means struggles when clusters are elliptical, overlap significantly, or differ in size. GMMs handle all three naturally. They also give you probabilistic membership — "this customer is 60% segment A, 40% segment B" — which is often more useful than a hard label. And because GMMs produce a proper density model, you can compute the likelihood of new points, making them useful for anomaly detection too.

sklearn: The Code You'll Actually Write

from sklearn.mixture import GaussianMixture
import numpy as np

# Fit a GMM with 3 components
gmm = GaussianMixture(
    n_components=3,
    covariance_type='full',    # 'full', 'tied', 'diag', 'spherical'
    n_init=10,                 # run 10 times, keep best
    init_params='k-means++',   # smart initialization
    random_state=42
)
gmm.fit(X)

# Soft assignments (responsibilities)
probs = gmm.predict_proba(X)   # shape: (n_samples, 3)

# Hard assignments
labels = gmm.predict(X)

# Log-likelihood of new data (useful for anomaly detection)
scores = gmm.score_samples(X_new)

# Learned parameters
print(gmm.means_)              # (3, n_features)
print(gmm.covariances_)        # shape depends on covariance_type
print(gmm.weights_)            # (3,) — mixing proportions

Model Selection: How Many Components?

Unlike K-Means' elbow method, GMMs give you principled model selection via information criteria. Both penalize model complexity, but BIC penalizes more harshly and tends to select simpler models.

import matplotlib.pyplot as plt

bics, aics = [], []
K_range = range(1, 10)

for k in K_range:
    gm = GaussianMixture(n_components=k, covariance_type='full',
                          n_init=5, random_state=42).fit(X)
    bics.append(gm.bic(X))
    aics.append(gm.aic(X))

plt.plot(K_range, bics, 'o-', label='BIC')
plt.plot(K_range, aics, 's--', label='AIC')
plt.xlabel('Number of components')
plt.ylabel('Information criterion')
plt.legend()
plt.title('Pick the elbow (BIC is more conservative)')
plt.show()

BIC (Bayesian Information Criterion) is usually the right default — it's consistent, meaning it picks the true K as n→∞. AIC tends to overfit with more components.

Convergence: The Good and the Bad

The Guarantee

Every EM iteration increases (or at least doesn't decrease) the observed data log-likelihood. This is provable — it falls out of Jensen's inequality. So EM always converges to something.

The Catch

That "something" is a local optimum, not necessarily the global one. The likelihood surface for mixture models is riddled with local maxima and saddle points. Two runs with different initializations can give very different answers.

The fix is simple and effective: run EM multiple times with different random initializations and keep the result with the highest log-likelihood. That's exactly what n_init=10 does in sklearn. It's brute force, but it works.

⚠️ Singularities

If a Gaussian component collapses onto a single data point, its variance goes to zero and the likelihood goes to infinity. This is a degenerate solution, not a useful one. Sklearn handles this by adding a small reg_covar (default 1e-6) to the diagonal of each covariance matrix. If you see convergence warnings, increase it.

The ELBO: What EM Is Really Optimizing

Maximizing the log-likelihood directly is intractable because the log sits outside a sum over hidden variables. EM sidesteps this by optimizing a lower bound called the Evidence Lower BOund (ELBO). The E-step tightens the bound (makes it touch the true likelihood at the current parameters). The M-step maximizes the tightened bound. Together, they guarantee monotonic improvement.

If you go on to study variational autoencoders or variational inference, you'll meet the ELBO again — it's the same idea, just applied to more complex models where the M-step can't be computed in closed form.

Beyond GMMs: Where Else EM Shows Up

Missing data imputation: treat missing values as hidden variables. EM iterates between imputing expected values (E-step) and re-estimating the data model (M-step). This is how some implementations of multivariate imputation work under the hood.

Hidden Markov Models: the Baum-Welch algorithm is EM applied to sequential data. The E-step computes forward-backward probabilities; the M-step re-estimates transition and emission parameters. If you've ever used speech recognition or biological sequence analysis, you've used EM.

Practical Tips

Initialization

Bad initialization → bad local optimum. Always use init_params='k-means++' (sklearn's default) or run K-Means first and use those centroids as starting means. Random initialization from the data is acceptable but needs more restarts.

Covariance Regularization

In high dimensions with limited data, full covariances will overfit or become singular. Options: use diag, increase reg_covar, or reduce dimensionality with PCA before fitting the GMM.

Scaling

GMMs are distance-based models. Features on wildly different scales will dominate the covariance estimates. StandardScaler before fitting is almost always the right move.

A Complete Pipeline

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.mixture import GaussianMixture

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('pca', PCA(n_components=20, random_state=42)),
    ('gmm', GaussianMixture(
        n_components=5,
        covariance_type='full',
        n_init=10,
        random_state=42
    ))
])

pipe.fit(X_train)
soft_labels = pipe.predict_proba(X_test)
hard_labels = pipe.predict(X_test)
