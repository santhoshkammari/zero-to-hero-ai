Dimensionality Reduction

Chapter 6: Unsupervised Learning Compression · Visualization · Feature engineering

The Problem: Too Many Columns, Not Enough Signal

You've got a dataset with 500 features. Maybe it's gene expression data with 20,000 columns and 200 patients. Maybe it's user behavior with 300 engineered features. Either way, you're drowning in dimensions, and most of them are either redundant (annual_salary and monthly_pay × 12) or noise (that sensor column that drifts randomly).

Three things break when dimensions get high. Speed — distance-based algorithms like k-NN scale terribly with feature count. Overfitting — 500 features and 1,000 rows means your model has room to memorize garbage. Visualization — you can't plot 500 axes. You can barely fake 3.

There's also the curse of dimensionality: in high-dimensional space, every point is roughly equidistant from every other point. Distances lose meaning, and algorithms that rely on distance fall apart. Dimensionality reduction shoves your data back into a space where geometry works again.

The tools split into two camps: compression (PCA, LDA — create features for downstream models) and visualization (t-SNE, UMAP — create 2D pictures for your eyeballs). Mixing these up is the #1 mistake people make.

PCA: The Foundation

PCA finds the directions of maximum variance. That's the one-sentence summary, and it carries more weight than most people realize.

The Intuition — It's Just a Rotation

Picture a cloud of data points in 3D. It's not a sphere — it's shaped like a cigar, elongated along one axis. If you could only keep one number per point, you'd want their position along that long axis. That axis captures the most spread in the data. That's the first principal component.

The second PC? The direction of maximum remaining variance, perpendicular to the first. Third PC? Same idea, perpendicular to both. PCA doesn't warp or stretch — it rotates your coordinate system so the axes are ranked by how much information (variance) they carry. Then you drop the low-variance axes.

The key mental model. PCA = change of basis. Same data, better coordinate system. The new axes are ordered by importance, so you can chop off the tail without losing much.

The Math — Covariance Matrix → Eigendecomposition

Start with your centered data matrix X (n × p — samples by features, mean-subtracted). The covariance matrix is:

C = (1 / (n - 1)) * Xᵀ X    # shape: (p × p)

Entry C[i, j] tells you how much feature i and feature j vary together. Diagonal entries are variances. Off-diagonals are covariances — the redundancy PCA exploits.

Now eigendecompose: C = VΛVᵀ. The columns of V are eigenvectors — these are your principal component directions. Λ is diagonal with eigenvalues — each one tells you how much variance that direction captures. Sort by eigenvalue descending, keep the top k, and you've got k principal components.

That's it. The eigenvectors of the covariance matrix are the principal components. The eigenvalues tell you how important each one is.

SVD: What Actually Runs Under the Hood

Nobody actually computes Xᵀ X. For p = 10,000 features, that's a 10,000 × 10,000 matrix — expensive to form, numerically dodgy to decompose. Instead, sklearn applies Singular Value Decomposition directly to X:

X = U Σ Vᵀ

The columns of V are exactly the PC directions. The singular values σᵢ relate to eigenvalues by λᵢ = σᵢ² / (n − 1). Same answer, better numerics, and you never build the covariance matrix.

Randomized SVD goes further — it uses randomized linear algebra to approximate only the top k singular vectors. For a million-feature dataset where you need 50 components, this is orders of magnitude faster. Sklearn uses it automatically when you set svd_solver='randomized' (or 'auto', which picks it when n_components is small relative to features).

How Many Components? The Scree Plot

The explained variance ratio for component k = its eigenvalue divided by the sum of all eigenvalues. If PC1 explains 40%, PC2 explains 25%, and PC3 explains 15%, then three components capture 80% of the total information.

A scree plot shows these values in order. You're looking for an elbow — where the curve flattens and each additional component gives diminishing returns. Everything past the elbow is mostly noise.

The pragmatic rule: keep enough components for 90-95% cumulative variance. Then validate on your downstream task.

import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_digits

X, y = load_digits(return_X_y=True)
print(f"Original shape: {X.shape}")  # (1797, 64)

# ALWAYS standardize before PCA
X_scaled = StandardScaler().fit_transform(X)

# Fit with all components to inspect the variance breakdown
pca_full = PCA().fit(X_scaled)
cumvar = np.cumsum(pca_full.explained_variance_ratio_)

# How many components for 95% variance?
n_95 = np.argmax(cumvar >= 0.95) + 1
print(f"Components for 95% variance: {n_95}")  # ~28

# Now reduce for real
pca = PCA(n_components=n_95)
X_reduced = pca.fit_transform(X_scaled)
print(f"Reduced: {X_reduced.shape}")  # (1797, 28) — less than half the features

# Sanity check: reconstruction error
X_back = pca.inverse_transform(X_reduced)
mse = np.mean((X_scaled - X_back) ** 2)
print(f"Reconstruction MSE: {mse:.4f}")  # small = good
Standardize first! PCA maximizes variance. If feature A ranges 0–1,000,000 and feature B ranges 0–1, PCA will think feature A dominates — not because it's more informative, but because it has bigger numbers. StandardScaler before PCA. Every time. No exceptions.

When to Use PCA

Feature compression before modeling. You have 500 features and a linear model. Reduce to 50, keep 95% of variance, train 10x faster with no meaningful accuracy loss. This is PCA's bread and butter.

Denoising. Reconstruct from the top k components and you've filtered out the high-frequency noise that lived in the discarded dimensions. Image denoising with PCA actually works surprisingly well.

Multicollinearity removal. Ridge regression handles collinearity, but PCA eliminates it by construction — PCs are orthogonal. Feed PCA output to a model and collinearity is physically impossible.

PCA ≈ Linear Autoencoder

This is one of those connections that clicks once and stays with you forever: a single-layer autoencoder with linear activations and MSE loss learns the exact same subspace as PCA. The encoder learns to project down, the decoder learns to project back up, and the reconstruction loss forces it to capture maximum variance. They converge to the same solution.

Add nonlinear activations and you get something strictly more powerful. But the linear case and PCA are mathematically identical. PCA is the theoretical baseline for all learned compressions.

PCA in a Real Pipeline

from sklearn.pipeline import Pipeline
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

# The right way: PCA inside the pipeline so it only sees training folds
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('pca', PCA(n_components=0.95)),  # keep 95% variance (auto-selects n)
    ('clf', RandomForestClassifier(n_estimators=100, random_state=42))
])

scores = cross_val_score(pipe, X, y, cv=5, scoring='accuracy')
print(f"Accuracy: {scores.mean():.3f} ± {scores.std():.3f}")

# Compare: full features, no PCA
pipe_full = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', RandomForestClassifier(n_estimators=100, random_state=42))
])
scores_full = cross_val_score(pipe_full, X, y, cv=5, scoring='accuracy')
print(f"Full features: {scores_full.mean():.3f} ± {scores_full.std():.3f}")

Passing n_components=0.95 (a float) tells PCA to automatically keep however many components hit 95% explained variance. Clean, no manual tuning needed.

t-SNE: The Visualization Microscope

t-SNE makes gorgeous 2D plots where clusters jump off the screen. It is a visualization tool. Not a feature engineering method. Not a preprocessing step. A way to look at high-dimensional data. Tattoo this on your forearm before reading further.

How It Works (Intuitively)

Step 1 — Build a probability distribution in high-D. For each point, compute how likely it is to "pick" every other point as a neighbor, using a Gaussian kernel. Close points get high probability. Far points get near-zero. This gives you a probability distribution over pairs that captures the local neighborhood structure.

Step 2 — Do the same in 2D, but with a twist. Place points randomly in 2D. Compute pairwise similarities there too — but use a Student's t-distribution (one degree of freedom) instead of a Gaussian. This is the key innovation.

Step 3 — Make the two distributions match. Minimize the KL divergence between the high-D distribution and the low-D distribution via gradient descent. Points shuffle around in 2D until neighbors in high-D are also neighbors in low-D.

The Crowding Problem and Why the t-Distribution

This is the insight that made t-SNE work where earlier methods (SNE) didn't.

In 100 dimensions, a single point can have dozens of equidistant neighbors — the surface area of a hypersphere is enormous. In 2D, you can only pack ~6 equidistant neighbors around a point before they start colliding with other clusters. Trying to preserve all those neighborhoods with a Gaussian in 2D creates a traffic jam — everything smushes into the center. This is the crowding problem.

The t-distribution has heavier tails than the Gaussian. Moderately distant points in high-D get pushed much further apart in low-D. This creates breathing room — clusters can spread out and separate. The heavy tails literally give the layout space to work.

Perplexity: The One Knob That Matters

Perplexity (typical range: 5–50) roughly controls "how many effective neighbors each point considers." Think of it as a soft k for the neighborhood size.

Low perplexity (5–10): tight focus on very local structure. You'll see small, fragmented clusters — possibly splitting real groups into sub-clusters. High perplexity (30–50): broader view, more global context. Clusters are chunkier and their relative positions are more stable.

Golden rule. Run t-SNE at perplexity 5, 30, and 50. If a cluster appears in all three, it's probably real. If it only appears at perplexity 5, it might be noise.

Practical Gotchas — Read This Before Interpreting Any t-SNE Plot

Cluster sizes are meaningless. The algorithm can inflate or squash clusters arbitrarily. A big cluster isn't necessarily more spread out in high-D.

Distances between clusters are meaningless. Two well-separated blobs might be nearly identical in the original space. The gap tells you nothing.

Only cluster existence and membership are reliable. "These points group together" — trustworthy. "This cluster is far from that one" — not trustworthy.

Never use t-SNE output as features for a model. The embedding is optimized for visual aesthetics, not predictive power. The axes have no consistent meaning. Different random seeds produce different layouts.

Run it multiple times. If the structure changes dramatically across runs (with PCA init it's better, but still), be skeptical.

from sklearn.manifold import TSNE
from sklearn.datasets import load_digits

X, y = load_digits(return_X_y=True)

tsne = TSNE(
    n_components=2,
    perplexity=30,
    learning_rate='auto',  # sklearn auto-tunes this
    init='pca',            # much more stable than random init
    random_state=42,
    n_iter=1000            # default; increase if you see noisy results
)
X_2d = tsne.fit_transform(X)

# Plot with matplotlib:
# plt.scatter(X_2d[:, 0], X_2d[:, 1], c=y, cmap='tab10', s=5, alpha=0.7)
# Note: axes are meaningless — don't even label them

Barnes-Hut approximation brings complexity from O(n²) to O(n log n). Good enough for ~100k points. Beyond that, use UMAP.

UMAP: The Modern Default

UMAP showed up in 2018 and immediately became what most people reach for instead of t-SNE. It's faster. It preserves more global structure. And — crucially — you can use its output as features for downstream models, something t-SNE can't do.

How It Works (Without the Topology Lecture)

UMAP's theoretical foundation involves fuzzy simplicial sets and Riemannian geometry. In practice, here's what it does:

1. Build a weighted nearest-neighbor graph in high-D. For each point, find its n_neighbors closest points. Assign edge weights based on distance (closer = stronger). Each point's distances are normalized relative to its own local scale, so dense regions and sparse regions are treated equally.

2. Build a similar graph in low-D. Start from a random (or spectral) initialization in 2D. Compute edge weights there too.

3. Optimize the low-D layout so the two graphs match. Minimize the cross-entropy between the high-D and low-D edge weights using SGD. Attractive forces pull connected points together; repulsive forces push unconnected points apart.

The use of cross-entropy (instead of t-SNE's KL divergence) is a big deal — KL divergence only penalizes "things that should be close but aren't," while cross-entropy also penalizes "things that should be far apart but aren't." This is why UMAP preserves global structure better.

The Two Knobs

n_neighbors (default 15): how many neighbors define "local." Small values (5–15) emphasize fine-grained local structure. Large values (50–200) capture more global topology — you'll see how clusters relate to each other. This is the single most important parameter.

min_dist (default 0.1): how tightly points can pack in the embedding. Set to 0.0 for dense, well-separated clusters. Set to 0.5+ if you want to see the continuous topology within clusters rather than hard boundaries.

# pip install umap-learn
import umap
from sklearn.datasets import load_digits

X, y = load_digits(return_X_y=True)

# Visualization (2D)
reducer = umap.UMAP(
    n_neighbors=15,
    min_dist=0.1,
    n_components=2,
    metric='euclidean',
    random_state=42
)
X_2d = reducer.fit_transform(X)

# Feature engineering (higher-D — this is what t-SNE can't do)
reducer_feat = umap.UMAP(n_components=10, random_state=42)
X_features = reducer_feat.fit_transform(X)
# Now feed X_features into any classifier

# Supervised mode — uses labels to nudge the embedding
reducer_sup = umap.UMAP(n_components=2, random_state=42)
X_sup = reducer_sup.fit_transform(X, y=y)
# Class boundaries become sharper — great for exploration

LDA: Supervised Dimensionality Reduction

Everything above is unsupervised. Linear Discriminant Analysis uses class labels to find the directions that best separate classes. Different goal, different math.

Fisher's Criterion

LDA maximizes the ratio of between-class scatter to within-class scatter. Between-class scatter S_B measures how far apart the class means are. Within-class scatter S_W measures how spread out points are within each class. You want directions where classes are far apart (big S_B) and internally tight (small S_W).

Formally: find w that maximizes wᵀS_Bw / wᵀS_Ww. This is a generalized eigenvalue problem, and you get at most C − 1 components (where C = number of classes). Binary problem? One dimension. Ten classes? At most nine. That's a hard ceiling.

from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.datasets import load_digits

X, y = load_digits(return_X_y=True)

lda = LinearDiscriminantAnalysis(n_components=9)  # 10 digit classes → max 9
X_lda = lda.fit_transform(X, y)
print(f"LDA shape: {X_lda.shape}")  # (1797, 9)

# These 9 dimensions are OPTIMIZED for separating the 10 digit classes
# Often outperforms 28 PCA components for classification despite having fewer dims

LDA assumes Gaussian classes with equal covariance matrices. When those assumptions roughly hold, it's remarkably effective — often better than PCA with fewer components because it targets class separation, not variance. When they don't hold (curved boundaries, wildly different class shapes), it underperforms.

Name collision alert. LDA in dimensionality reduction = Linear Discriminant Analysis (Fisher, 1936). LDA in NLP = Latent Dirichlet Allocation (Blei, 2003). Completely different algorithms. Context always disambiguates, but be aware when reading papers.

ICA: The Cocktail Party Solver

PCA finds uncorrelated directions. ICA finds statistically independent sources. The distinction matters: uncorrelated means zero covariance. Independent means knowing one tells you absolutely nothing about the others. Independence is a much stronger condition.

The Classic Setup

Three people talking at a party. Three microphones, each recording a different mix: mic1 = 0.7·voice_A + 0.2·voice_B + 0.1·voice_C. ICA takes the three mixed signals and recovers the three original voices — without knowing the mixing weights. This is blind source separation.

The trick: mixed signals tend toward Gaussian (Central Limit Theorem). So ICA finds directions that maximize non-Gaussianity — the most non-Gaussian direction corresponds to a recovered original source. The FastICA algorithm does this efficiently with a fixed-point iteration.

When to use ICA: EEG/MEG signal processing (separating brain signals from eye blinks), audio separation, financial factor discovery. When not to: general dimensionality reduction. ICA requires the number of sources to equal the number of observations, and the sources must be non-Gaussian. It's a specialist tool.

from sklearn.decomposition import FastICA
import numpy as np

# Simulate the cocktail party: 3 sources, 3 mixtures
np.random.seed(42)
t = np.linspace(0, 1, 1000)
s1 = np.sin(2 * np.pi * 5 * t)           # sine wave
s2 = np.sign(np.sin(2 * np.pi * 3 * t))  # square wave
s3 = np.random.laplace(size=1000)         # noise burst
S = np.c_[s1, s2, s3]

A = np.array([[1, 0.5, 0.3], [0.5, 1, 0.7], [0.3, 0.7, 1]])  # mixing matrix
X = S @ A.T  # observed mixtures

# ICA recovers the sources (up to sign and scale)
ica = FastICA(n_components=3, random_state=42)
S_recovered = ica.fit_transform(X)
# S_recovered columns ≈ original sources (possibly reordered/sign-flipped)

Autoencoders: Nonlinear PCA (Preview)

An autoencoder is a neural network that compresses input through a bottleneck and tries to reconstruct the original. The bottleneck forces it to learn a compressed representation. With linear activations and MSE loss — you get PCA. With nonlinear activations — you get something strictly more powerful that can capture curved manifolds PCA misses.

The key variants in one breath: denoising autoencoders corrupt input and learn to clean it (forces robust features). Variational autoencoders (VAEs) add a probabilistic layer and can generate new data. Sparse autoencoders encourage most neurons to be off per input (sparse features).

The tradeoff is real: autoencoders need architecture design, GPU training, and give you uninterpretable latent dimensions. PCA gives you eigenvalues you can explain in a meeting. Use autoencoders when linear methods visibly fail and you have enough data. We'll cover them properly in the deep learning chapters.

And the meta-advice: dimensionality reduction is a preprocessing step, not a final answer. The right evaluation is downstream performance. Run PCA → train model → measure metric. Run UMAP → train model → measure metric. Whichever wins, wins. Elegance of mathematics is irrelevant.

Putting It All Together

"""
Full workflow: explore → reduce → model
Shows PCA, UMAP, and LDA head-to-head on the same dataset.
"""
import numpy as np
from sklearn.datasets import load_digits
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline

X, y = load_digits(return_X_y=True)

# --- Baseline: all 64 features ---
pipe_full = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', GradientBoostingClassifier(n_estimators=100, random_state=42))
])
score_full = cross_val_score(pipe_full, X, y, cv=5).mean()

# --- PCA: keep 95% variance ---
pipe_pca = Pipeline([
    ('scaler', StandardScaler()),
    ('pca', PCA(n_components=0.95)),
    ('clf', GradientBoostingClassifier(n_estimators=100, random_state=42))
])
score_pca = cross_val_score(pipe_pca, X, y, cv=5).mean()

# --- LDA: max 9 components ---
pipe_lda = Pipeline([
    ('scaler', StandardScaler()),
    ('lda', LinearDiscriminantAnalysis(n_components=9)),
    ('clf', GradientBoostingClassifier(n_estimators=100, random_state=42))
])
score_lda = cross_val_score(pipe_lda, X, y, cv=5).mean()

print(f"Full features (64D): {score_full:.3f}")
print(f"PCA (~28D):          {score_pca:.3f}")
print(f"LDA (9D):            {score_lda:.3f}")
# LDA often wins here — 9 dimensions that are PURPOSE-BUILT to separate digits
