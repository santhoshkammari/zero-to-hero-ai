TL;DR

ViT proved pure self-attention вҖ” no convolutions вҖ” can match or beat CNNs on images when data is large enough. Swin fixed ViT scaling with windowed attention, making transformers practical for detection and segmentation. CLIP put image and text encoders in shared embedding space, enabling zero-shot classification. DINOv2 learned strong visual features from images alone through self-distillation. Hybrid architectures like ConvNeXt blurred CNN-transformer boundary. SAM made segmentation promptable and universal. Era of training one model per task is ending. Modern vision uses foundation models you adapt, not architectures you build from scratch.

The Moment Everything Changed

Long time, convolutions felt right for images. Locality, translation equivariance вҖ” physical sense. Then results piled up. ViT beat CNNs on ImageNet. Swin Transformer dominated detection and segmentation benchmarks. CLIP did zero-shot classification competitive with fully supervised models. DINOv2 learned features from raw images that transferred almost anywhere. Evidence got too strong to ignore.

Vision Transformer: An Image Is Worth 16Г—16 Words

For decades, assumption was simple: images need convolutions. Local connectivity, weight sharing, hierarchical receptive fields вҖ” standard tools for spatial data. In 2020, Dosovitskiy and colleagues at Google Brain asked heretical question: what if we use no convolutions at all? What if transformer crushing NLP benchmarks goes straight onto images?

Problem: transformers eat token sequences, and images are not sequences. Image is 2D pixel grid. A 224Г—224 RGB image has 150,528 values. Feed each pixel as token, attention matrix becomes 50,176 Г— 50,176 вҖ” about 2.5 billion entries. Not workable.

Patch Embedding вҖ” Turning Images Into Sequences

Fix: treat patches, not pixels, as tokens. Chop image into non-overlapping patches. Each patch becomes token. Like cutting photo into small squares, then reading left-to-right, top-to-bottom like sentence.

Take 224Г—224 image. Divide into 16Г—16 patches. Result: 14Г—14 grid вҖ” 196 patches total. Each patch is 16 Г— 16 Г— 3 = 768 values for RGB. Flatten patch into vector, run linear projection, get D-dimensional embedding. Now you have 196 tokens, each representing one image patch.

Two more pieces matter. First, positional embeddings вҖ” learned vectors added to each patch token to tell model where patch lives in image. Without them, transformer cannot know patch 1 is top-left and patch 196 is bottom-right; it sees unordered bag of patches. ViT uses learned 1D positional embeddings, not sinusoidal encodings from NLP transformers, and they work well. Model infers 2D spatial structure from data.

Second, special [CLS] token is prepended to sequence, borrowed from BERT. This token attends with all 196 patches and aggregates whole-image information. After final transformer layer, [CLS] output goes to classification head. Model says: "summary of this image lives here."

Why It Works (When It Has Enough Data)

Critical ViT insight: CNNs have strong inductive biases baked in. Locality: pixel relates most to nearby pixels. Translation equivariance: cat in top-left should be processed same way as cat in bottom-right. These assumptions give CNNs head start. They do not need to learn nearby pixels are correlated вҖ” architecture already knows.

ViT has almost none of these priors. It starts from blank slate. Every patch can attend to every other patch from layer one. Model must learn locality, spatial relationships, and feature hierarchies entirely from data. On small datasets, CNNs crush it. Original paper says this plainly: trained on ImageNet-1k alone (1.28 million images), ViT underperforms ResNets.

Give it enough data вҖ” ImageNet-21k (14M images) or JFT-300M (300M images) вҖ” and story flips. Lack of inductive bias becomes strength. Model is not boxed in by architectural assumptions about which relationships matter. It learns distant attention when useful, local focus when useful, and whatever feature hierarchy data demands. With enough examples, ViT surpasses CNNs on same benchmarks.

The Quadratic Problem

Self-attention computes NГ—N attention matrix, where N is token count. For ViT with 196 patches, matrix is 196Г—196 вҖ” manageable. But a 1024Г—1024 image with 16Г—16 patches gives 4,096 tokens and 4096Г—4096 attention matrix. Double image resolution, attention cost quadruples. This quadratic scaling is ViT's core limit. Swin Transformer exists because of it.

Swin Transformer: Making Vision Transformers Practical

ViT proved transformers can work for vision. But real-world use had two big problems. First, quadratic attention cost made high-resolution images impractical. Second, it produced single-scale feature map вҖ” all tokens at same resolution. CNNs naturally produce multi-scale features, like ResNet /4, /8, /16, /32 maps. Detection and segmentation need that to find both tiny and large objects.

Swin Transformer (Liu et al., 2021) fixed both problems with two ideas that now feel inevitable: windowed attention and hierarchical feature maps.

Windowed Attention вҖ” From Global to Local

Instead of letting every patch attend across whole image, Swin restricts attention to local windows. Split feature map into non-overlapping windows, say 7Г—7 patches. Self-attention runs inside each window independently. A 49-token window gives 49Г—49 attention matrix вҖ” same size no matter how large full image gets. Complexity drops from quadratic in image size to linear.

But now global context is gone. Patches in one window cannot see neighboring window. Fix: shifted windows. In alternating layers, window grid shifts by half window width. Patches separated in one layer share window in next layer, creating cross-window information flow.

Building analogy: regular windowed attention lets people talk only inside room. Shifted windows move walls. People separated before end up in same room and carry old information with them.

Hierarchical Feature Maps

Second innovation: Swin builds multi-scale feature maps like CNN. It starts with 4Г—4 patches вҖ” small, high-resolution tokens вҖ” then progressively merges patches between stages. Adjacent 2Г—2 patch groups get concatenated and projected down. Spatial resolution halves and channel depth doubles at each stage, producing feature pyramid:

  Stage 1:  56 Г— 56 tokens, C channels       (1/4 resolution)
  Stage 2:  28 Г— 28 tokens, 2C channels      (1/8 resolution)
  Stage 3:  14 Г— 14 tokens, 4C channels      (1/16 resolution)
  Stage 4:   7 Г—  7 tokens, 8C channels      (1/32 resolution)

Same multi-scale structure as ResNet. Result: Swin drops straight into existing detection and segmentation frameworks вҖ” Feature Pyramid Networks, Mask R-CNN, UPerNet вҖ” anywhere CNN backbones were used. Then it beat CNNs on those benchmarks.

Swin became de facto vision backbone for dense prediction tasks. Swin v2 later scaled design to 3 billion parameters and 1536Г—1536 resolution, proving approach could grow.

DeiT: Making ViT Practical Without Massive Data

ViT depended on hundreds of millions of training images. Real barrier. Most teams do not have JFT-300M lying around. Facebook Research's DeiT (Data-efficient Image Transformer, Touvron et al., 2021) showed ViT could train effectively on ImageNet-1k alone вҖ” with smarter setup.

Key idea: knowledge distillation. Train strong CNN teacher, like RegNet. Then train ViT student to match teacher outputs. DeiT adds special distillation token beside [CLS] token. [CLS] token trains against ground-truth labels in standard way. Distillation token trains against teacher soft predictions вҖ” learning not only answer, but teacher uncertainty across classes.

Dual supervision вҖ” hard labels from dataset plus soft knowledge from CNN вҖ” gives ViT student enough signal to learn without 300 million images. Strong data augmentation, like RandAugment, random erasing, Mixup, and CutMix, filled rest. DeiT matched ViT-on-JFT performance using only ImageNet-1k, making vision transformers broadly accessible.

Neat part: CNNs, with strong inductive biases, learn efficiently from limited data. Transformers, with minimal biases, can surpass CNNs when data is huge. DeiT lets CNN inductive biases flow into transformer through distillation. Best of both worlds.

The Hybrid Question: CNN or Transformer вҖ” Or Both?

Once ViT showed pure attention can work for vision, and Swin showed it can scale, obvious question appeared: do we even need to choose between convolutions and attention? Maybe best architecture uses both.

ConvNeXt вҖ” A CNN That Learned From Transformers

ConvNeXt (Liu et al., 2022) completely reframed CNN-vs-transformer debate. Authors started with standard ResNet-50 and modernized it step by step using design ideas from Swin Transformer, measuring impact of each change:

Larger kernels: 7Г—7 instead of 3Г—3. Inverted bottleneck blocks: wide вҶ’ narrow вҶ’ wide, like transformers' MLP. LayerNorm instead of BatchNorm. GELU instead of ReLU. Fewer activation and normalization layers. "Patchify" stem instead of traditional conv-pool-conv stem.

Result: pure CNN вҖ” no attention at all вҖ” matching Swin Transformer across classification, detection, and segmentation. ConvNeXt message is blunt: CNN-transformer gap was never mainly about attention versus convolution. It was about training recipes, architectural details, and scaling strategy. Modernize CNN with transformer-era lessons, performance gap disappears.

CoAtNet and True Hybrids

CoAtNet (Dai et al., 2021) takes explicit hybrid approach: use convolution layers early, where local features matter most, then switch to transformer layers later, where global context matters. Intuition: early layers extract edges, textures, and local patterns вҖ” convolutions do this efficiently. Later layers reason about distant image parts вҖ” attention excels there.

This staged approach gets CNN data efficiency in early layers, where model must learn local patterns anyway, and transformer flexibility in later layers, where global reasoning adds most value. CoAtNet reached state-of-the-art ImageNet results with less data than pure ViT needed.

The Real Takeaway

CNN-versus-transformer debate is mostly settled: answer depends on constraints. Need best accuracy and massive data? Pure transformer. Need data efficiency and fast inference? CNN or hybrid. Need drop-in backbone for detection or segmentation? Swin or ConvNeXt. Architecture matters less than training recipe, data, and scale.

CLIP: When Vision Learned to Read

For most of supervised computer vision history, workflow was simple: collect images, label them at high cost, train model on labels, hope it generalizes. If task changed вҖ” from classifying animals to classifying vehicles вҖ” you needed new labels and new training run. Each task lived in its own silo.

CLIP (Contrastive Language-Image Pre-training, Radford et al., 2021) broke that pattern. Instead of class labels, CLIP trains on image-caption pairs вҖ” 400 million scraped from internet. Example: image of golden retriever on beach paired with caption "a golden retriever playing on a sandy beach." No class taxonomy. No label ontology. Only images and natural-language descriptions.

The Training Loop

CLIP has two encoders. Image encoder вҖ” ViT or ResNet вҖ” maps images to vectors. Text encoder вҖ” Transformer вҖ” maps text to vectors of same dimension. Training is contrastive: take batch of N image-text pairs, compute cosine similarity between every image and every text, push matching pairs on diagonal of NГ—N matrix together, push non-matching pairs apart.

Loss is symmetric cross-entropy applied row-wise вҖ” for each image, which text matches? вҖ” and column-wise вҖ” for each text, which image matches? After training, both encoders live in shared embedding space. Images and text describing same thing land near each other.

Zero-Shot Classification

Here paradigm shifts. Suppose you want to classify images into categories never seen in training вҖ” say bird species. Traditional supervised learning needs labeled bird dataset and training run. CLIP needs text descriptions of classes: "a photo of a cardinal," "a photo of a blue jay," "a photo of a robin." Encode text. Encode image. Compute cosine similarities. Highest similarity wins.

# Zero-shot classification with CLIP вҖ” no training data needed
import clip, torch
from PIL import Image

model, preprocess = clip.load("ViT-B/32", device="cuda")

image = preprocess(Image.open("mystery_bird.jpg")).unsqueeze(0).to("cuda")
texts = clip.tokenize([
    "a photo of a cardinal",
    "a photo of a blue jay",
    "a photo of a robin"
]).to("cuda")

with torch.no_grad():
    img_emb = model.encode_image(image)
    txt_emb = model.encode_text(texts)
    img_emb /= img_emb.norm(dim=-1, keepdim=True)
    txt_emb /= txt_emb.norm(dim=-1, keepdim=True)
    similarity = (img_emb @ txt_emb.T).softmax(dim=-1)

# similarity might be [0.03, 0.91, 0.06] вҶ’ blue jay

CLIP zero-shot accuracy on ImageNet competes with fully supervised ResNet-50 trained on 1.28 million labeled ImageNet images. Model that never saw ImageNet labels ends up in same ballpark as model trained specifically on ImageNet. Wild result.

Why CLIP Became the Foundation of Everything

CLIP influence goes far beyond classification. Its text encoder became language-understanding backbone of Stable Diffusion. Its image encoder вҖ” or Google's improved SigLIP variant, which swaps softmax loss for more scalable sigmoid loss вҖ” serves as vision backbone for multimodal LLMs like LLaVA and GPT-4V. Its contrastive framework inspired open-vocabulary object detection and visual search in vector databases. Understand CLIP, and you understand foundation of modern vision-language AI.

Prompt Engineering for Vision

CLIP is sensitive to class phrasing. "A photo of a dog" works much better than "dog" alone, because training data was image-caption pairs, not single-word labels. For better accuracy, average embeddings from multiple prompt templates: "a photo of a dog," "a picture of a dog," "a bright photo of a dog." Serious CLIP deployments usually use this prompt ensemble trick.

DINOv2: Vision Without Words

CLIP needs image-text pairs. 400 million of them. That means heavy internet scraping and many assumptions about caption quality. Question: can you learn equally strong visual features from images alone вҖ” no captions, no labels, only raw pixels?

That is DINOv2 (Oquab et al., Meta, 2023). It learns visual representations through self-distillation вҖ” teacher-student setup where teacher is slowly updated copy of student.

How Self-Distillation Works

Take image. Create multiple augmented views вҖ” different crops, scales, color distortions. Feed some views to student network and other, often larger and less distorted, views to teacher network. Student job: produce embeddings matching teacher embeddings for same image despite different views. Teacher weights update as exponential moving average (EMA) of student weights. Slow change gives stable target.

Why DINOv2 Matters

DINOv2 features are remarkably general. Freeze backbone, add linear layer on top, and you get strong performance on image classification, semantic segmentation, depth estimation, and instance retrieval вҖ” all without fine-tuning backbone. Closest thing we have to universal vision feature extractor.

DINOv2 especially shines on dense prediction tasks вҖ” segmentation, depth, surface normal estimation. CLIP, trained on image-level alignment with text, produces features tuned for understanding what is in image. DINOv2, trained on view consistency across spatial crops, preserves fine-grained spatial information telling you where things are. They are complementary. With no labeled data, use CLIP for text-related tasks and DINOv2 for spatial tasks.

SAM: Segment Anything

SAM (Segment Anything Model, Kirillov et al., Meta, 2023) applies foundation-model philosophy directly to segmentation. Goal: one model that can segment any object in any image, prompted by point click, bounding box, or even text description.

Architecture has three pieces. Heavyweight ViT image encoder processes image once and produces dense feature map. Lightweight prompt encoder converts user prompts вҖ” points, boxes, masks вҖ” into embeddings. Lightweight mask decoder combines image features with prompt embeddings to predict segmentation masks. Image encoder runs once per image. Prompt encoder and mask decoder are fast enough for interactive use.

SAM represents philosophical shift. Traditional segmentation models learn specific categories like person, car, tree. The Foundation Model Paradigm

Step back from individual models and pattern appears. Modern vision has converged on two-phase paradigm that would have looked bizarre decade ago:

Phase 1: Pre-train broadly. Train massive model on massive dataset using general-purpose objective. CLIP uses contrastive image-text matching. DINOv2 uses self-distillation. SAM uses promptable segmentation. Model learns rich, transferable representations.

Phase 2: Adapt cheaply. Take pre-trained backbone and adapt it to specific task. Maybe add linear head for linear probing, fine-tune a few layers, or use model zero-shot. Key point: you should not need to train vision backbone from scratch again. Heavy lifting happens once, then whole community shares it.

Why this works so well is still not fully settled. Main theory: large-scale pre-training on diverse data forces model to learn general visual concepts вҖ” edges, textures, shapes, objects, scenes, spatial relationships вҖ” instead of task-specific shortcuts. But honest answer is simpler: nobody fully understands why representations from self-distillation or contrastive learning transfer so well across wildly different downstream tasks.

When to Reach for What

Practical engineer question is not "which architecture is theoretically best?" It is "which model do I grab for this problem?" Here is how landscape maps to real decisions:

Classify images, no labeled data CLIP zero-shot Describe classes in text, compare embeddings
Classify images, some labeled data DINOv2 or CLIP backbone + linear probe Frozen foundation features + thin task head

Interactive or zero-shot segmentation SAM Point/box prompts, works across domains

Fast inference on edge/mobile EfficientViT or MobileNetV3 Designed for low latency and small memory

The Interview Lens

Modern vision shows up constantly in senior ML interviews. Questions separating candidates are not about memorizing architectures. They are about understanding trade-offs. These are threads interviewers pull on:

ViT vs. CNN inductive bias. ViT underperforms CNNs on small datasets because CNNs have locality and translation equivariance baked in. They do not need to learn nearby pixels are related. ViT must learn that from scratch, which costs data. But when data is abundant, those same inductive biases can become constraints limiting what model can learn.

Attention complexity. ViT self-attention is O(NВІ) in number of patches. Swin windowed attention is O(N) with respect to image size, or O(MВІ) per window where M is fixed. That is why Swin can handle high-resolution images where vanilla ViT cannot. This is not abstract theory. It directly determines what resolution production system can process.

Supervised vs. self-supervised vs. contrastive. Traditional ImageNet training is supervised вҖ” you need 1.28 million labeled images. CLIP is contrastive вҖ” you need image-text pairs but no class labels. DINOv2 is self-supervised вҖ” you need images and nothing else. Each paradigm produces features with different strengths. Supervised features optimize for labeled task. CLIP features capture language-aligned semantics. DINOv2 features preserve spatial structure. Knowing when each fits shows depth.

The foundation model trade-off. Do not fine-tune whole backbone for every task because catastrophic forgetting can destroy general features. Linear probing preserves them but limits adaptation. Practical answer usually sits in middle: fine-tune last few layers, or use adapters, or use LoRA. Interviewers want understanding of this spectrum from linear probing to full fine-tuning, and when each choice makes sense.
