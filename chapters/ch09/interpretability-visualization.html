TL;DR

CNN makes prediction — but what did it look at? Build interpretability from scratch: start with single pixel's gradient, construct saliency maps, discover why they're noisy, fix with SmoothGrad and Integrated Gradients, graduate to Grad-CAM spatial heatmaps. Use occlusion sensitivity as brute-force sanity check, peer inside neurons with feature visualization, test human-defined concepts with TCAV, examine attention maps in Vision Transformers. Running example: debugging skin-lesion classifier that's suspiciously good — because interpretability isn't decoration. It's difference between model that works and model that cheats. For model-agnostic methods (SHAP, LIME, counterfactuals), see Chapter 18.

Treated interpretability as afterthought for too long. Train model, check accuracy, glance at confusion matrix, move on. Model said "malignant." Score 0.94. Good enough? Then saw Grad-CAM heatmap of one "best" model — hotspot was on ruler in corner of dermoscopy image. Not lesion. Ruler. Model learned rulers appear more often in images dermatologists thought worth measuring — measuring correlates with malignancy. 94% accuracy was a lie.

Changed how I think about every model. Interpretability isn't nice-to-have academic exercise. Only way to know whether model learned right thing or convenient thing. Tools in this section = X-ray machine for X-ray machines — look inside network, see what it actually learned.

The Question Every Model Must Answer

Trained CNN to classify skin lesions from dermoscopy images. Hits 96% accuracy on test set. Stakeholder — dermatologist with 20 years experience — asks one question: "What is model looking at when it says melanoma?"

Can't answer that → model doesn't ship. Not because dermatologist is difficult — model achieving 96% by detecting rulers, color cards, or ink marks is worse than no model. Gives confident wrong answers on images without those artifacts. In medicine, confident wrong answers kill people.

Problem interpretability tools solve: take billions of multiplications inside network, produce something human can look at and say "Yes, focusing on lesion border irregularity" or "No, focusing on hair follicles."

Call our skin-lesion classifier DermNet throughout — ResNet-50 fine-tuned on 25,000 dermoscopy images across 7 lesion types. Every technique applied to DermNet, so you see each tool doing real work on same model.

Saliency Maps — The Simplest Window

Most basic question about neural network's decision: "Which input pixels, if wiggled slightly, would change output most?" Precise mathematical answer — gradient of output with respect to input.

Concrete with DermNet: feed dermoscopy image. Network outputs score for each of 7 lesion classes. Pick predicted class — say, melanoma — ask PyTorch to compute how score changes if we nudge each of 224×224×3 = 150,528 input values by tiny amount. Gradient magnitude tells sensitivity to that pixel.

tensor.requires_grad_(True)
output = model(tensor)
score = output[0, predicted_class]
score.backward()

# Each pixel now has a gradient — take absolute value, collapse RGB channels
saliency = tensor.grad.data.abs().squeeze()   # [3, 224, 224]
saliency, _ = saliency.max(dim=0)             # [224, 224]

Gives us saliency map — pixel-level heat map of sensitivity. Term introduced by Simonyan et al. (2013), one of oldest neural network visualization techniques still in use. Idea: pixels with large gradients = ones model "cares about" most.

Catch: raw saliency map is noisy. Speckled. Individual pixels light up like static on old TV. Map doesn't show clean regions — scattered bright dots everywhere. Hard to tell if model looks at lesion or at image sensor noise.

Not a bug. Fundamental limitation. Gradients are local — tell what happens if you move single pixel by infinitesimal amount, holding everything else fixed. Real features span many pixels; gradient at any one pixel = noisy estimate of broader pattern.

Fixing the Noise — SmoothGrad and Integrated Gradients

Noisy saliency map frustrating, but underlying idea — use gradients to measure importance — is sound. Noise is problem, not principle. Two techniques fix it differently; understanding both matters because they reveal different things about model.

SmoothGrad takes brute-force approach. Instead of one saliency map, compute fifty. Each time, add small random Gaussian noise to input before computing gradients. Average all fifty maps. Noise differs each time so speckle averages out, but consistent signal — regions model truly depends on — survives.

import torch

def smooth_grad(model, tensor, target_class, n_samples=50, noise_level=0.15):
    mean_grad = torch.zeros_like(tensor)
    for _ in range(n_samples):
        noisy = tensor + noise_level * torch.randn_like(tensor)
        noisy.requires_grad_(True)
        score = model(noisy)[0, target_class]
        score.backward()
        mean_grad += noisy.grad.data.abs()
    return (mean_grad / n_samples).squeeze().max(dim=0).values

Improvement dramatic. Where vanilla map was speckled static, SmoothGrad produces coherent interpretable regions. For DermNet, smoothed map clearly highlights lesion boundary — exactly where dermatologists look for irregularity.

Integrated Gradients (Sundararajan et al., 2017) takes more principled approach. Instead of adding noise, asks: "What's path from nothing to this image, how much does each pixel contribute along that path?" Starts with baseline — typically black image (all zeros) — interpolates in small steps toward actual input. At each step, computes gradient. Final attribution per pixel = average gradient across all steps × difference between input and baseline.

Why bother? Integrated Gradients satisfies two axioms raw gradients don't. Sensitivity: if feature matters to prediction, gets non-zero attribution. Implementation invariance: two networks with identical outputs always get identical attributions, regardless of internal architecture. These properties matter when you need to trust explanations, not approximate them.

The Baseline Trap

Integrated Gradients requires baseline — "nothing" you interpolate from. Black image works for natural photos, but medical imaging where black regions carry meaning (dark background in X-rays) → black baseline produces misleading attributions. For dermoscopy, per-channel dataset mean as baseline gives more reliable results. Baseline choice not minor detail — can completely change explanation.

Grad-CAM — From Pixels to Regions

Saliency maps, even smoothed varieties, answer at pixel level. But dermatologist asking "what is model looking at?" doesn't want to hear about pixel (147, 203). Wants "irregular border in upper-left quadrant." Need to zoom out from pixels to regions.

Gradient-weighted Class Activation Mapping — Grad-CAM — single most important interpretability tool in CNN practitioner's toolkit. Selvaraju et al. (2017), became default visualization method for good reason: fast, works on any CNN without modification, produces class-specific heatmaps matching human intuition about what model should look at.

Core insight: deep inside CNN, last convolutional layer produces stack of feature maps — 2,048 in ResNet-50, each a 7×7 spatial grid. Each feature map responds to different learned pattern: one fires on irregular borders, another on brown pigment, another on particular texture. Maps still carry spatial information — know where pattern occurred — but semantically rich enough to relate to specific classes.

Grad-CAM figures out which of those 2,048 feature maps matter for target class via gradients. Backpropagate target class score; for each feature map, compute average gradient across all spatial positions. Average becomes importance weight for that channel. Multiply each feature map by weight, sum all up, pass through ReLU (only care about features that positively influence class) → 7×7 heatmap of which spatial regions mattered most. Upscale to 224×224, overlay on original image → see exactly where model was looking.

import torch
import torch.nn.functional as F

# Hook into the last conv layer to capture activations and gradients
activations, gradients = {}, {}

def save_act(module, inp, out):
    activations['val'] = out.detach()

def save_grad(module, grad_in, grad_out):
    gradients['val'] = grad_out[0].detach()

target_layer = model.layer4[-1]
target_layer.register_forward_hook(save_act)
target_layer.register_full_backward_hook(save_grad)

# Forward pass, then backprop the target class
output = model(tensor)
model.zero_grad()
output[0, target_class].backward()

# The Grad-CAM recipe:
# 1) Global average pool the gradients → one weight per channel
weights = gradients['val'].mean(dim=[2, 3], keepdim=True)

# 2) Weighted sum of feature maps, then ReLU
cam = (weights * activations['val']).sum(dim=1, keepdim=True)
cam = F.relu(cam)

# 3) Upscale to input size
cam = F.interpolate(cam, size=(224, 224), mode='bilinear', align_corners=False)
cam = cam.squeeze().numpy()
cam = (cam - cam.min()) / (cam.max() - cam.min() + 1e-8)

Ran this on DermNet's melanoma predictions — story immediately clear. Correctly diagnosed images: heatmap over lesion, specifically over asymmetric borders and color variegation (textbook dermoscopic features). "Ruler images" that fooled me: heatmap firmly on measurement artifact. Two minutes of Grad-CAM revealed what week of metric-staring missed.

Variant: Grad-CAM++ uses second-order gradients for better localization when multiple instances of same object in one image. In practice, vanilla Grad-CAM sufficient for most debugging workflows.

Grad-CAM limitation: resolution. 7×7 grid means heatmap inherently coarse — can tell "upper-left quadrant" but not "this specific cluster of 20 pixels." Need pixel-level attribution? Back to Integrated Gradients or methods below.

Occlusion Sensitivity — The Brute-Force Sanity Check

All methods so far rely on gradients — peek inside model's computational graph. Occlusion sensitivity does something refreshingly dumb: cover up part of image, check if model's confidence drops.

Slide grey square across DermNet's input, one position at a time. Each position: run partially occluded image through network, record confidence for target class. Where confidence drops most = region carrying most important information. No hooks, no gradients, no assumptions about model internals.

import numpy as np

def occlusion_map(model, img_tensor, target_class, patch=32, stride=8):
    _, _, H, W = img_tensor.shape
    base = model(img_tensor).softmax(1)[0, target_class].item()
    heatmap = np.zeros((H, W))
    count = np.zeros((H, W))

    for y in range(0, H - patch + 1, stride):
        for x in range(0, W - patch + 1, stride):
            masked = img_tensor.clone()
            masked[:, :, y:y+patch, x:x+patch] = 0.5
            score = model(masked).softmax(1)[0, target_class].item()
            heatmap[y:y+patch, x:x+patch] += (base - score)
            count[y:y+patch, x:x+patch] += 1

    return heatmap / np.maximum(count, 1)

Appeal: method is model-agnostic. Works on CNNs, transformers, even model behind API where you only get predictions. No weight access needed — model treated as pure black box.

Cost: speed. 224×224 image with 32×32 patch, stride 8 → ~576 forward passes per image. Fine for inspecting handful of suspicious predictions. Impractical for large-scale analysis — why Grad-CAM (one forward + one backward pass) remains default for routine work.

Why occlusion still matters: when Grad-CAM and occlusion agree — both highlight same region — confidence that model genuinely uses that region goes way up. Gradient-based methods can sometimes mislead (more later), but occlusion can't lie. Covering region tanks prediction → that region was carrying signal. Period.

Feature Visualization — What Neurons Dream About

Everything so far: explaining specific prediction ("Why melanoma for this image?"). Feature visualization asks different question: "What has this neuron learned to recognize, across all images?"

Technique, popularized by Chris Olah (2017 Distill article), is activation maximization. Pick neuron — say, channel 137 in ResNet-50's layer3. Start with random noise image. Use gradient ascent (maximizing, not minimizing) to iteratively modify image so neuron fires as strongly as possible. After few hundred iterations, resulting image shows what neuron "wants to see."

Results striking. Early-layer neurons dream of edges and color gradients — expected from Gabor-like filters. Mid-layer neurons produce textures: honeycomb, fur, scales, repeating geometric motifs. Deep-layer neurons produce recognizable objects/parts — eyes, wheels, dog faces — though often surreal, like hallucination. Not coincidence: Google's DeepDream built on exactly this technique — run activation maximization on entire image rather than noise, network's learned features amplified into psychedelic eye-covered landscapes that went viral in 2015.

Without regularization, activation-maximized images look like high-frequency noise that happens to excite neuron. Adding jitter (random shift by few pixels each step), small rotations, and penalizing high-frequency patterns produces more natural, interpretable images. Regularization isn't cosmetic — acts as prior for "images that actually occur in real world."

For DermNet, feature visualization revealed something interesting. Several deep neurons learned to respond to specific dermoscopic patterns — blue-white structures, pigment network, regression areas — corresponding almost exactly to features dermatologists trained to look for. Reassuring. But other neurons clearly learned to respond to circular border of dermoscope itself — fire whenever they see characteristic dark vignette, regardless of lesion inside. Kind of insight no accuracy metric would reveal.

Concept Activation Vectors — Thinking in Human Terms

Feature visualization shows what neurons respond to, but interpreting dream-like images requires guesswork. Testing with Concept Activation Vectors — TCAV (Kim et al., 2018) — flips script. Instead of "what does this neuron represent?", asks "how important is this human-defined concept to model's prediction?"

Mechanism: pick concept — say, "irregular border." Collect images that exhibit irregular borders and set that don't. Pass both through network, extract activations from chosen layer. Train simple linear classifier (logistic regression or linear SVM) to separate "has irregular border" from "doesn't." Normal vector of classifier's decision boundary becomes Concept Activation Vector — direction in activation space corresponding to your concept.

TCAV score answers: "For images model classifies as melanoma, what fraction get higher melanoma score when activations nudged in 'irregular border' direction?" Score 0.85 → model strongly influenced by border irregularity for melanoma. Score 0.50 (random) → concept doesn't matter. Test "ruler present" and get high TCAV score → caught model using shortcut.

TCAV's power: speaks language of domain experts. Instead of showing dermatologist heatmap asking "does this look right?", say: "Model's melanoma predictions 82% influenced by border irregularity, 74% by color variegation, 15% by ruler presence." Conversation clinician can engage with.

Attention Maps in Vision Transformers

Vision Transformers (ViTs) split images into patches — typically 16×16 pixels — process as sequence with self-attention. Architecture gives interpretability almost free: attention weights right there, telling which patches model attends to.

with torch.no_grad():
    out = vit_model(tensor, output_attentions=True)

# Last layer, all heads, CLS token attending to image patches
attn = out.attentions[-1]               # [batch, heads, patches, patches]
cls_attn = attn[0, :, 0, 1:]           # CLS → image patches, all heads
cls_attn = cls_attn.mean(dim=0)        # average across heads
grid = int(cls_attn.shape[0] ** 0.5)
attn_map = cls_attn.reshape(grid, grid) # spatial map

Individual attention heads often show surprisingly clean specialization. In ViT models trained on dermoscopy: heads that consistently attend to lesion borders, heads attending to color patches, heads attending to image center regardless of content. Visualizing heads separately more informative than averaging.

Attention ≠ Attribution

Trap worth flagging — comes up in interviews. Attention maps show where model looks, not what causes prediction. Head might attend strongly to region, but downstream layers might completely ignore that information. For rigorous causal attribution in ViTs, use Attention Rollout (multiply attention matrices across all layers to trace information flow from input to output) or apply Grad-CAM to ViT's patch embeddings. Raw single-layer attention is suggestive, not conclusive.

The Sanity Check That Shook the Field

In 2018, Adebayo et al. published "Sanity Checks for Saliency Maps" — made many people uncomfortable. Experiment devastatingly simple: take trained model, randomize its weights (destroying everything learned), re-run saliency method. If explanation changes dramatically → method sensitive to what model learned. If looks roughly same → method showing input image structure, not model's reasoning.

Several popular methods failed. Guided Backpropagation — produces visually stunning, sharp explanations — produced nearly identical outputs before and after weight randomization. Effectively performing edge detection on input, no connection to model's decision process. Pretty, but meaningless.

Grad-CAM fared better but not perfectly. Changes when weights randomized (good), but some cases changes smaller than expected. Takeaway: Grad-CAM isn't useless, but no single attribution method should be trusted in isolation. Grad-CAM + occlusion agree → confident. Disagree → investigate further.

Still use Grad-CAM as first tool. But after Adebayo paper, always corroborate with at least one other method before claiming what model "learned." Extra five minutes has caught problems more than once.

The Debugging Playbook

These tools aren't decorations for paper figures. Diagnostic instruments. Here's how they fit into real debugging workflow, DermNet as running example.

Model predicts wrong on image you expected right. Start with Grad-CAM. Heatmap highlights background instead of lesion → model learned shortcut. Maybe all melanoma training images from one hospital with distinctive skin marker → model learned "skin marker = melanoma." Fix: data-side — diversify sources, augment aggressively, crop irrelevant regions.

Accuracy suspiciously high. DermNet hitting 99% should trigger alarm bells, not celebration. Run Grad-CAM on batch of 50–100 correctly classified images. Heatmaps cluster on watermarks, ruler marks, or dermoscope vignette rather than lesion → data leakage. Strip metadata, crop margins, re-evaluate on images from hospital not in training set.

One class consistently underperforms. Overall accuracy fine, but "dermatofibroma" keeps confused with "benign keratosis." Run feature visualization on neurons most active for each class. Look similar → model hasn't found discriminative features. Need more training examples of underperforming class, or harder negative mining to force learning subtle differences.

Stakeholder asks "can you prove it's not using race/skin-tone?" TCAV question. Define concept "dark skin tone" with example images, compute concept activation vector, check TCAV score for each diagnosis class. Score significantly above 0.5 → model's predictions influenced by skin tone — fairness problem needing fix before deployment.

The Clever Hans Problem

Named after horse in 1900s Berlin that appeared to do arithmetic but read trainer's unconscious body language. CNNs do same thing. Famous example: model trained to distinguish wolves from huskies learned to detect snow in background — wolves photographed in snow, huskies on grass. Model never learned what wolf looks like. Learned what snow looks like. Grad-CAM would've caught this in minutes. Lesson: never deploy model whose Grad-CAM you haven't checked.

The Toolbox — Libraries and Practicalities

Don't need to implement Grad-CAM from scratch for production. Ecosystem matured significantly.

Captum (Meta) — most comprehensive PyTorch library. Implements Integrated Gradients, Layer Conductance, Grad-CAM, SHAP, dozens more — consistent API where you pass model and input, specify target layer, get attributions back. Includes visualization utilities for publication-quality overlays.

pytorch-grad-cam — laser-focused on CAM variants. Only need Grad-CAM, Grad-CAM++, Score-CAM and relatives? Smaller, faster to start. Handles multi-GPU models, works with CNNs and ViTs.

SHAP and LIME — model-agnostic explanation libraries covered in Chapter 18. Both work with image models — SHAP via DeepExplainer and superpixel segmentation, LIME via local perturbation — but scope extends beyond vision.

Practical tip: always run at least two methods on same image before drawing conclusions. Grad-CAM says "model looking at lesion" but occlusion says "confidence doesn't change when covering lesion" → something wrong. Gradient-based method more likely misleading than brute-force one.

Started with simplest question — "which pixels have largest gradients?" — discovered answer was noisy. Smoothed with SmoothGrad, built principled version with Integrated Gradients. Zoomed out pixels→regions with Grad-CAM, verified trust with brute-force occlusion. Went deeper: peering inside neurons with feature visualization, bridging human concepts to model internals with TCAV, watching ViT attention, learning from Adebayo sanity checks that some explanations less trustworthy than they look.

Next time model hits impressive accuracy, instead of celebrating and shipping — open Grad-CAM, run across batch of predictions, and look. Model right for right reasons worth ten models right by accident — only way to tell difference is look under hood.
