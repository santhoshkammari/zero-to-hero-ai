This section covers topics that did not fit neatly into CNN architectures, detection, or transfer learning but appear constantly in papers, job descriptions, and production systems. Video understanding, tracking, pose estimation, 3D reconstruction, visual search, super-resolution, plus architectural blocks that power much of this.

Before start: deep familiarity not needed. If you know what convolutional neural network does and roughly how object detection works, you have enough. We build each concept from ground up.

To keep things concrete, use one running scenario: building wildlife monitoring system for forest reserve. Cameras mounted on trees record animals 24/7. Job: build computer vision pipeline that turns raw footage into useful intelligence — what species are present, how many individuals, what they are doing, where they go. Every topic here solves one real part of that puzzle.

Optical Flow

Wildlife cameras record video. First question: what moved? Not which objects sit in frame — that comes later — but which pixels shifted between frames, and where they went.

That is optical flow. Given two consecutive frames, it produces dense motion map — vector (dx, dy) for every pixel, showing where pixel moved. If nothing moved, all vectors are zero. If deer walked left, pixels on deer point left. If camera jiggled, everything gets small uniform shift.

Used to think optical flow was relic from classical computer vision that deep learning made obsolete. Wrong. Optical flow survives because it captures motion itself, separate from object identity. Detection model can tell you deer exists in frame 12. Optical flow can tell you something in that region moved 4 pixels left between frames 11 and 12, without knowing what it is.

Classical approaches — Lucas-Kanade (1981) and Farneback (2003) — assume nearby pixels move together and solve for motion that best explains brightness changes between frames. Good in simple settings. Weak on large motions, occlusion, and textureless regions.

Modern standard: RAFT (Recurrent All-Pairs Field Transforms, Teed & Deng, 2020). Interview version: both frames pass through shared CNN to produce feature maps. RAFT builds all-pairs correlation volume — for every pixel in frame 1, it computes similarity with every pixel in frame 2. Result: massive 4D tensor. If images are 128×128, volume is 128×128×128×128. Brute-force answer to "where might each pixel have gone?"

RAFT does not stop at one lookup. It keeps flow estimate and iteratively refines it through recurrent unit (ConvGRU). Each iteration looks up current estimate in correlation volume, uses local context, nudges flow toward better answer. Like focusing camera: each iteration sharpens picture. After 12–32 iterations, flow field converges.

Key insight: RAFT does not try to predict perfect flow in one shot. It allows error, then corrects iteratively. That refinement lets it handle large motions that break single-pass methods.

For wildlife system, optical flow is foundation layer. It tells us "something moved there" before detector runs. It feeds video stabilization, frame interpolation, and — next — tracking.

Limitation: optical flow knows nothing about objects. Pixel-level bookkeeping. Two deer side by side with overlapping motion vectors? Optical flow does not know two animals exist. Need different tool.

Multi-Object Tracking

Wildlife cameras detect animals in single frames with object detector like YOLO. Frame 14 has three boxes: deer, deer, fox. Frame 15 also has three: deer, deer, fox. Hard question: which deer in frame 15 matches which deer from frame 14? Did left one stay left, or did they swap?

This is multi-object tracking (MOT). Detection answers "where are things?" in one frame. Tracking answers "which thing is which?" across frames. Difference matters. Detection gives headcount per frame. Tracking gives individual trajectories — deer #7 walked from waterhole to tree line over 40 seconds.

Dominant approach is tracking-by-detection. Two clean steps. First, run detector independently on every frame. Second, link detections across frames into consistent identities. Nice decoupling: upgrade detector or linker independently.

Linking is where things get interesting. Border-crossing analogy: cars arrive every minute. Job is deciding which cars are new and which you saw before. You can use position — blue car at gate 3 last minute, blue car at gate 4 now, probably same car. Or use appearance for stronger match, like checking license plate.

SORT (Simple Online and Realtime Tracking, Bewley et al., 2016) uses position only. For each tracked object, SORT keeps Kalman filter — mathematical model that predicts next position from recent velocity. When new detections arrive, SORT uses Hungarian algorithm to find optimal matching between predicted positions and actual detections, minimizing total distance. Unmatched predictions become lost tracks. Unmatched detections become new tracks.

SORT is fast and works well when objects move predictably. Weak point: occlusion. If deer walks behind tree for three frames, track dies. When deer reappears, SORT creates new identity. Deer #7 becomes deer #12 for no good reason.

DeepSORT (Wojke et al., 2017) fixes this by adding appearance model — license-plate matching. Each detected box passes through small CNN that produces appearance embedding, compact vector describing what object looks like. During matching, DeepSORT uses both where object should be (Kalman prediction) and what it looks like (embedding similarity). Deer reappears from behind tree? Appearance embedding still matches stored embedding for deer #7, so identity survives occlusion.

Tradeoff: speed. Running CNN for every detection in every frame adds real latency, especially in crowded scenes.

ByteTrack (Zhang et al., 2022) takes cleverer path. Most trackers keep only high-confidence detections. ByteTrack says low-confidence detections, which other trackers throw away, are valuable for occluded objects. Partially hidden deer might get 30% confidence — too low for most trackers, still useful for ByteTrack. It runs association in two rounds: first match tracks with high-confidence detections, then match remaining tracks with low-confidence ones. No appearance CNN. State-of-the-art results.

ByteTrack's key idea is simple: do not throw away uncertain detections. Human tracker would do same. Blurry glimpse still helps maintain mental model.

For wildlife system, tracking turns anonymous detections into named trajectories. Deer #7 visited waterhole at dawn, stayed 12 minutes, left northeast. That is conservation data. That is what researchers need.

Tracking tells us where each animal goes. Next question: what are they doing? Need body structure.

Pose Estimation

Knowing deer #7 sits at coordinates (340, 220) is useful. Knowing deer #7 has head down, front legs bent, and drinking posture means behavior understanding. Pose estimation detects body keypoints — joints like elbows, knees, wrists, ankles, nose, and ears — and connects them into skeleton.

Two main strategies find these keypoints. Distinction matters in practice.

Top-down works like spotlight. First detect each individual with bounding box (YOLO, Faster R-CNN, any detector). Then crop each box and run specialized keypoint network on crop. Each animal gets close-up analysis. HRNet (High-Resolution Network, Sun et al., 2019) is gold standard here. Unlike networks that downsample hard and later recover detail, HRNet keeps high-resolution feature maps through whole forward pass. Result: precise keypoint localization. It never loses fine spatial detail needed to place joint exactly.

Bottom-up works like floodlight. Instead of finding individuals first, it detects all keypoints in whole image at once — every knee, elbow, nose, from every animal. Then it groups them: this knee, hip, and shoulder belong to one skeleton. OpenPose (Cao et al., 2017) pioneered this with Part Affinity Fields (PAFs) — vector fields encoding direction and strength of connection between adjacent body parts. If knee and hip have strong PAF between them, they belong to same individual.

Spotlight vs floodlight tradeoff is real. Top-down methods (HRNet) are more accurate per individual but slow down as scene gets crowded — each person needs separate forward pass. Bottom-up methods (OpenPose) run once regardless of crowd size but can struggle during grouping when bodies overlap. For wildlife cameras with 2–5 animals per frame, top-down works well. For stadium crowd, bottom-up is better.

MediaPipe (Google, 2019) deserves mention because it runs in real time on phone, not because it is most accurate. MediaPipe is top-down pipeline optimized hard for speed — lightweight detector, then small regression network instead of heatmaps. Less precision. Better deployment practicality. Good trade for prototyping or mobile apps.

For wildlife system, pose estimation turns tracking data ("deer #7 moved here") into behavior data ("deer #7 is drinking," "deer #7 is running"). Conservation biologists can ask questions like "how much time do deer spend at waterhole?" or "do animals show stress behaviors near hiking trails?" without watching thousands of hours of footage.

Pose estimation gives posture snapshot in one frame. Animal behavior unfolds over time — drinking motion is sequence of poses, not one pose. To understand actions across frames, need models that reason about video.

Video Understanding

Everything so far processes one or two frames at time. Optical flow uses pairs. Detection and pose estimation use single frames. But understanding what is happening in video — grazing vs running vs fighting — needs many frames together. Question: how do you feed temporal image sequence into model?

Most intuitive answer: extend familiar 2D convolutions into time dimension. Standard 2D kernel slides across height and width. 3D convolution adds third axis — height, width, and time (frames). Where 2D kernel might be 3×3 pixels, 3D kernel might be 3×3×3 — three pixels in each spatial direction and three frames deep. This lets kernel detect patterns spanning space and time, like paw moving downward across three consecutive frames.

C3D (Tran et al., 2015) first showed straightforward 3D ConvNets, stacked like VGG, could learn strong spatiotemporal features for action recognition. I3D (Carreira & Zisserman, 2017) improved this by taking 2D architectures pretrained on ImageNet and "inflating" them to 3D — copying 2D kernel weights along time axis to bootstrap 3D network. That reuses years of image-classification progress for video.

Limitation of 3D convolutions is same as 2D convolutions, worse: they are local. 3×3×3 kernel sees only 3 frames of context. To capture 5-second action at 30fps, you need 150 frames of context, which means very deep networks or very large kernels. Same frustration that pushed NLP toward attention. Same solution appears in video.

Video Transformers like TimeSformer (Bertasius et al., 2021) and ViViT (Arnab et al., 2021) replace 3D convolutions with self-attention over space and time. They split video into patches, like Vision Transformers do for images, but each patch now has spatial position and temporal position. Attention can directly relate patch in frame 1 to distant patch in frame 90 without passing signal through 89 intermediate layers. TimeSformer factors attention — first across space within each frame, then across time — making computation tractable.

More recent: VideoMAE (Tong et al., 2022) brings self-supervised pretraining to video. It masks 90% of video patches — much more aggressive than image masking — and trains model to reconstruct them. High masking works because video has huge temporal redundancy; neighboring frames are nearly identical. Result: strong video representations from unlabeled footage. Very relevant for wildlife system with terabytes of raw video and limited labels.

Two-stream approach matters as historical landmark. Two-stream networks (Simonyan & Zisserman, 2014) run RGB frames through one CNN and precomputed optical flow maps through second CNN, then fuse outputs. Appearance stream knows what things look like. Motion stream knows how they move. Elegant split. It dominated video understanding for years before 3D convolutions and transformers absorbed both jobs into one model.

For wildlife system, video understanding turns "there is deer in frame" into "deer #7 is engaged in territorial display behavior." That leap — pixels to behavior — makes system useful to researchers.

Visual Search & Retrieval

Wildlife system has millions of animal detections after months of operation. Researcher photographs animal at another site and asks: "Have you seen this individual before?" We must search millions of stored images and find most similar ones. This is visual search. Engine is embedding space.

Core idea: take image, pass it through CNN or Vision Transformer backbone, extract feature vector from one of final layers. This vector — usually 128 to 2048 numbers — is image embedding. Think fingerprint. Two photos of same deer from different angles produce similar fingerprints. Deer and fox produce very different ones.

Raw features from ImageNet-pretrained backbone are decent embeddings, but they were trained for classification, not similarity. To build strong embedding space — similar images close together, dissimilar images far apart — you need metric learning: training with loss function designed for distance relationships.

Contrastive loss uses pairs. Take two images. If similar (same species, same individual), loss penalizes large distance between embeddings. If dissimilar, it penalizes small distance, but only up to margin — once far enough apart, loss stops caring. You sculpt embedding space: pull similar things together, push different things apart, but not infinitely far apart.

Triplet loss uses triplets: anchor image, positive (similar to anchor), negative (different from anchor). Loss says distance from anchor to positive should be smaller than distance from anchor to negative by at least margin. Formula: L = max(d(anchor, positive) - d(anchor, negative) + margin, 0). If positive is already closer than negative by more than margin, loss is zero — no gradient, nothing to fix. Hard part is mining good triplets. Random triplets are usually too easy; negative is already far away, model learns nothing. Hard negative mining — choosing negatives close to anchor — makes triplet loss work.

ArcFace (Deng et al., 2019) takes different path that dominates face recognition and is spreading elsewhere. Instead of pairs or triplets, it modifies classification loss itself. It normalizes feature embeddings and classification weights to unit length, so dot product becomes cosine of angle between them. Then it adds fixed angular margin to correct class angle before softmax. Result: embeddings where same-class samples sit farther from decision boundaries. In embedding space, classes form tighter, more compact clusters.

Why does angular margin often beat Euclidean margin in practice? Best explanation: in high-dimensional spaces, cosine similarity is more stable than Euclidean distance. Distances concentrate in high dimensions, making absolute distances less meaningful, while angles stay discriminative.

Once embeddings are good, retrieval becomes nearest-neighbor search. Store embeddings in vector database — FAISS, Milvus, or Pinecone for managed cloud — and query by cosine or L2 similarity. In wildlife system, researcher photo gets embedded, then we retrieve 10 most similar embeddings in database, which map to 10 most visually similar sightings. If same individual appeared before, it should rank near top.

Same machinery powers reverse image search, "find similar products" on e-commerce sites, and face verification in security systems. Backbone, loss function, vector database — whole pipeline.

Limitation: embedding quality is everything. Mediocre embedding space makes whole system unreliable. Training strong embeddings also needs carefully curated pairs or triplets, which can be expensive to label.

Image Super-Resolution

Some wildlife cameras are cheap. They capture grainy, low-resolution images, especially at night when infrared sensor activates. Researchers want to zoom in on distant animal and see enough detail to identify species, maybe individual. Image super-resolution trains model to upscale low-resolution image into higher-resolution version with plausible fine detail.

Standard approach is GAN-based. ESRGAN (Enhanced Super-Resolution GAN, Wang et al., 2018) uses generator to upscale image and discriminator to judge whether output looks like real high-resolution photo. Generator learns to add texture, sharpen edges, and synthesize detail that fools discriminator. Real-ESRGAN (Wang et al., 2021) extends this to real-world degradation — not clean benchmark downsampling, but messy blur, noise, and compression artifacts from actual cameras.

Results can be stunning. Blurry text becomes readable. Faces gain recognizable features. Fur texture appears where only smudge existed. But important truth: model is hallucinating that detail. It generates plausible-looking pixels from training patterns. It does not recover information lost during capture. If pixel was blurred beyond recognition, no neural network can know original value. It can only guess what looks realistic.

This distinction matters anywhere accuracy matters. For wildlife blog post, hallucinated fur texture is fine. For identifying individual animal by unique spot pattern, hallucinated spots are actively misleading. In medical imaging, hallucinated detail could cause false diagnoses. In forensic imaging, it could fabricate evidence.

Super-resolution is often presented as image "enhancement," as if model recovers hidden ground truth. Not true. It makes educated guess. Often very good guess, sometimes visually indistinguishable from reality — still guess. Know this. Calibrate trust accordingly.

For wildlife system, super-resolution helps make distant captures easier to inspect. Researcher gets cleaner image. But species identification and individual re-identification should happen on raw images, or minimally processed ones, not hallucinated outputs.

Neural Radiance Fields & Gaussian Splatting

Different challenge for wildlife system: research team wants 3D map of forest habitat. They walked area and took photos from many angles. Goal: turn 2D photos into navigable 3D model.

Neural Radiance Fields (NeRFs) (Mildenhall et al., 2020) solved this with striking quality. NeRF represents 3D scene not as mesh or point cloud, but as neural network — small MLP with 5D input and output describing scene properties. Input is spatial position (x, y, z) plus viewing direction (θ, φ). Output is color (r, g, b) and volume density (σ) — opacity at that point. Solid surfaces have high density. Empty air has zero density. Semi-transparent objects like smoke or glass sit in between.

To render pixel in new view, shoot ray from camera through pixel into scene, sample many points along ray, query network for each point's color and density, then composite front-to-back with differentiable volume rendering. Dense points block ray; first dense thing ray hits dominates pixel color. Because whole rendering process is differentiable, you can train network by backpropagating through rendering — compare rendered images to real photos, reduce difference.

Give NeRF 50–100 photos with known camera positions, and it learns photorealistic novel views from almost any angle. Paper results were jaw-dropping. Intuition usually points to positional encoding — mapping coordinates through sinusoidal functions so network can learn high-frequency detail — plus implicit smoothness of neural networks.

Catch: speed. Rendering one pixel needs hundreds of network evaluations along ray. Rendering full image takes seconds. Fine for offline use. Bad for real-time exploration.

3D Gaussian Splatting (Kerbl et al., 2023) fixes speed by dropping neural-network scene representation entirely. Instead of implicit function, it represents scene as millions of tiny fuzzy 3D ellipsoids — 3D Gaussians. Each Gaussian has position, shape (orientation and extent), color, and opacity. To render view, project all Gaussians onto camera plane — "splatting" — and alpha-composite them. No per-ray neural network evaluation. No volume sampling. Result: real-time rendering at 60+ frames per second.

Quality in 2024 rivals, sometimes beats, NeRFs while rendering orders of magnitude faster. Training is faster too — instead of gradient descent through neural network, you optimize parameters of individual Gaussians to match training views. Tradeoff: storage. Millions of Gaussians need more memory than compact neural network.

For wildlife system, 3D reconstruction lets researchers build navigable habitat models, measure terrain features, or plan camera placement from desk. Starting new 3D reconstruction project today? Begin with Gaussian Splatting.

Deformable Convolutions

Now shift from applications to architectural building blocks — components you can plug into existing networks to improve them.

Standard convolution samples pixels on fixed rectangular grid. A 3×3 kernel always samples same 9 relative positions: top-left, top-center, top-right, and so on. This rigidity is strength and weakness. Strength: efficient and translation-equivariant. Weakness: nature is not rectangular.

Imagine wildlife camera captures snake curving through grass. Standard 3×3 kernel sees small square patch. It may catch part of body, but rectangular sampling grid does not follow snake's curve. It needs many layers of context to understand full shape.

Deformable convolutions (Dai et al., 2017) fix this by making sampling grid learnable. For each output location, network predicts set of 2D offsets — one (Δx, Δy) per sampling point. Kernel's 9 sampling positions shift by those offsets before feature extraction. Offsets are real-valued, not integer pixels, so network uses bilinear interpolation to sample fractional positions.

Crucial detail: offsets come from extra convolutional layer beside main convolution. It takes same input features and produces 2×k×k offset map (k is kernel size). During training, backpropagation flows through convolution weights and offset-prediction layer. Network learns what features to extract and where to look.

Visualize sampling positions on snake image. Nine points stretch and curve along body instead of sitting in rigid square. Kernel learns to deform itself to match feature geometry.

Deformable DETR (Zhu et al., 2021) applies same idea to attention — instead of attending to all positions uniformly, it learns to attend to small set of offset positions around each query. Result: more efficient attention and more relevant focus.

In wildlife system, deformable convolutions help detect animals of many shapes and orientations — coiled snake, bird with spread wings, deer partly occluded by branches — without specialized augmentation for every geometric case.

Squeeze-and-Excitation Blocks

Convolutional layer produces many channels — each channel acts like feature detector for different pattern. One channel may respond to horizontal edges, another to green textures, another to circular shapes. Typical convolution treats all channels equally. But for any image, some channels matter much more than others. Forest scene does not need "brick texture" channel at full volume.

Squeeze-and-Excitation (SE) blocks (Hu et al., 2018) add simple mechanism that lets network learn which channels matter for each input. This is channel attention.

Process has two steps. First, squeeze: each channel's full spatial feature map is reduced to one number with global average pooling. If you start with 7×7×512 feature map, you get 512-dimensional vector. Each number summarizes one channel's overall activation. This is "what is present?" summary.

Second, excitation: summary vector passes through small two-layer fully connected bottleneck network — first down to, say, 512/16 = 32 neurons (with ReLU), then back up to 512 neurons (with sigmoid). Output is 512 values between 0 and 1 — one per channel. These are learned importance weights. Multiply each channel by its weight, and channels that matter for current input get amplified while unhelpful ones get suppressed.

Bottleneck matters because it keeps SE block lightweight. Going from 512 to 32 and back forces network to learn compressed, generalizable channel relationship instead of memorizing arbitrary per-channel weights. Parameter count added by SE block is negligible compared with layers it modifies.

SENet won 2017 ImageNet competition, improving top-1 accuracy by about 1% across many architectures at minimal computational cost. Since then, SE blocks or close variants have been added to EfficientNet, RegNet, MobileNetV3, and many more. This is classic neural-architecture free lunch — drop in, get small consistent gain, move on.

For wildlife system, SE blocks help classification backbone focus on species-relevant features while suppressing irrelevant background textures. In forest scene full of green foliage, block learns to turn down "green texture" channels and turn up "animal shape" channels, adapting attention to current input.

Stepping Back

We started at pixel level with optical flow — raw motion between frames. Then tracking, where detections gain persistent identities. Then pose estimation for body posture, and video understanding for behavior over time. After that came visual search with learned embeddings, super-resolution for low-quality captures, 3D reconstruction with NeRFs and Gaussian Splatting, and two architectural blocks — deformable convolutions and SE blocks — that improve networks behind all of it.

Next time you see job posting asking for "experience with multi-object tracking" or paper citing "deformable attention," goal is simple: no more nod-and-change-subject. Have concrete mental model of what happens under hood. Then dig into implementation details with confidence.
