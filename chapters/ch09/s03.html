Feature Maps & Receptive Field

Chapter 9: CNNs & Computer Vision 5 subtopics
TL;DR

CNNs build a hierarchy: early layers detect edges and textures, middle layers combine them into parts and patterns, deep layers recognize whole objects. The receptive field — how much of the original image a single neuron "sees" — grows with depth. Understanding this hierarchy and how to calculate receptive fields tells you why deep networks work, why certain architectures struggle with large objects, and how attention mechanisms like squeeze-and-excitation improve channel interactions.

What's actually happening inside a CNN? When you train a network on ImageNet, the first conv layer doesn't learn "cat" or "dog" — it learns edges. Horizontal lines, vertical lines, color gradients, blobs. These are the building blocks. Layer by layer, the network composes these simple detectors into increasingly abstract representations. Understanding this hierarchy is the key to understanding why CNNs work — and why they sometimes fail.

Hierarchical Feature Learning

Low-Level Features: Edges and Textures

The first convolutional layer — the one directly touching the raw pixels — almost universally learns edge detectors and color filters. Visualize the 64 learned 3×3 or 7×7 kernels of a trained ResNet, and you'll see oriented edges at various angles, color-opponent filters (red vs. green, blue vs. yellow), and frequency detectors. These look strikingly similar to the Gabor filters that neuroscientists found in the primary visual cortex (V1). The network rediscovers what evolution spent millions of years optimizing.

This happens regardless of the training task. Whether you train on ImageNet classification, medical imaging, or satellite photos, the first-layer features look nearly identical. That's a profound observation — it means low-level vision is task-independent, which is the entire foundation of transfer learning.

Mid-Level Features: Parts and Patterns

By layers 3–5, the network combines edges into more complex patterns. You start seeing texture detectors (brick patterns, fur textures, stripes), corners and junctions, small parts like eyes or wheels, and repeated motifs. These features are still somewhat general — a "circular edge" detector is useful for both eyes and wheels — but they're starting to become task-specific.

The mid-level features are where the magic of compositionality happens. A horizontal edge plus a vertical edge at a specific relative position makes a corner. Many corners in a grid make a checkerboard texture. These compositions happen automatically through the stacking of convolution and non-linearity layers — the network doesn't need explicit instructions to build this hierarchy.

High-Level Features: Objects and Scenes

The deepest layers develop feature maps that respond to whole objects or semantic concepts. A single channel in the last conv layer of a trained network might fire strongly whenever there's a dog face, regardless of breed, pose, or lighting. Another channel might detect wheels. Another might respond to text. These are highly task-specific and abstract.

This hierarchy — edges → textures → parts → objects — is not something we designed. It emerges from training. The loss function says "classify this image correctly," and gradient descent discovers that the best strategy is to build features bottom-up. This emergent structure is one of the most beautiful results in deep learning.

Receptive Field

The receptive field of a neuron is the region of the original input image that can influence that neuron's value. A neuron in the first conv layer with a 3×3 kernel sees only a 3×3 patch — its receptive field is 3×3. A neuron in the second conv layer (also 3×3) sees a 3×3 region of the first layer's output, which in turn covers a 5×5 region of the original input. So the receptive field of the second layer is 5×5.

Receptive Field Calculation

For a stack of convolutional and pooling layers, the receptive field grows according to a simple recursive formula. For each layer, starting from the top (deepest) and working back to the input:

# Receptive field calculation (working backwards from top layer)
# For layer i with kernel_size k_i and stride s_i:
#   r_i = r_{i+1} + (k_i - 1) * j_i
#   j_i = j_{i+1} * s_i
# where r is receptive field size and j is "jump" (stride product)

def compute_receptive_field(layers):
    """Each layer is (kernel_size, stride). Returns RF at each layer."""
    n = len(layers)
    r = [1] * (n + 1)  # RF starts at 1 for the output
    j = [1] * (n + 1)  # jump starts at 1

    # Work backwards
    for i in range(n - 1, -1, -1):
        k, s = layers[i]
        j[i] = j[i + 1] * s
        r[i] = r[i + 1] + (k - 1) * j[i + 1]

    return r[0]  # RF at input

# Example: three 3×3 conv layers with stride 1
print(compute_receptive_field([(3,1), (3,1), (3,1)]))  # 7

# With a 2×2 max pool (stride 2) in between
print(compute_receptive_field([(3,1), (2,2), (3,1)]))  # 8

# VGG-style: conv-conv-pool-conv-conv-pool
vgg_block = [(3,1), (3,1), (2,2), (3,1), (3,1), (2,2)]
print(compute_receptive_field(vgg_block))  # 24

Here's the intuition: each layer adds (kernel_size - 1) pixels of context, but those pixels are measured in the coordinate system of the layer below. Stride multiplies the coordinate system — a stride-2 pooling layer means each pixel in the next layer corresponds to 2 pixels in the previous layer. So a 3×3 conv after a stride-2 pool adds (3-1) × 2 = 4 input pixels of context, not just 2.

Why Receptive Field Matters

If your task requires understanding large-scale structure — like distinguishing a lake from an ocean, or recognizing that a group of people are playing soccer — your network needs a large receptive field. If the receptive field of your final feature map doesn't cover the relevant object, the network physically cannot detect it. This is why very deep networks, dilated convolutions, and large-kernel designs all exist: they're different strategies for growing the receptive field.

Effective Receptive Field

The theoretical receptive field tells you the maximum region a neuron could be influenced by. But not all pixels in that region contribute equally. The effective receptive field — the region that actually has a significant impact — is much smaller, typically concentrated in the center.

Research by Luo et al. (2016) showed that the effective receptive field follows a Gaussian distribution: central pixels have outsized influence, while border pixels contribute almost nothing. For a network with theoretical receptive field of 100×100, the effective receptive field might only be about 30×30. This means that even "deep" networks with large theoretical receptive fields may struggle with truly global patterns.

This finding motivates several architectural choices. Skip connections (ResNet) help by creating shortcuts that allow gradient to flow more evenly, making the effective RF closer to the theoretical RF. Self-attention (Vision Transformers) provides global receptive field from the very first layer, sidestepping the issue entirely. And techniques like dilated convolutions expand the effective RF more aggressively than standard convolutions by spreading sampling points across a wider area.

Channel Interactions

So far we've focused on spatial patterns — what a filter detects within a 2D region. But there's another dimension: channels. Each feature map is one channel in a multi-channel tensor, and how channels interact matters enormously.

A standard convolution naturally mixes channels — each output channel is a weighted sum across all input channels. But it does so with a single set of spatial weights. What if some channels are more important than others for a given image? A photo of a car might benefit from emphasizing "wheel" and "window" channels while suppressing "fur texture" channels.

Squeeze-and-Excitation (SE) Block

The Squeeze-and-Excitation (SE) block learns to adaptively recalibrate channel importance. It works in three steps. First, "squeeze": global average pooling collapses each channel's spatial dimensions to a single number, producing a channel descriptor of shape (batch, C). Second, "excitation": two fully-connected layers (with a bottleneck) process this descriptor and output per-channel weights via a sigmoid. Third, "scale": the original feature map is multiplied channel-wise by these learned weights.

import torch.nn as nn

class SEBlock(nn.Module):
    def __init__(self, channels, reduction=16):
        super().__init__()
        self.squeeze = nn.AdaptiveAvgPool2d(1)
        self.excitation = nn.Sequential(
            nn.Linear(channels, channels // reduction, bias=False),
            nn.ReLU(inplace=True),
            nn.Linear(channels // reduction, channels, bias=False),
            nn.Sigmoid()
        )

    def forward(self, x):
        b, c, _, _ = x.shape
        # Squeeze: (b, c, h, w) → (b, c)
        w = self.squeeze(x).view(b, c)
        # Excitation: (b, c) → (b, c) channel weights
        w = self.excitation(w).view(b, c, 1, 1)
        # Scale: element-wise multiply
        return x * w

The SE block adds very few parameters (two small FC layers) but provides consistent accuracy improvements. Plugging SE blocks into ResNet-50 improved ImageNet top-1 accuracy by about 1% with negligible compute overhead. SE blocks were integrated into EfficientNet, MobileNetV3, and many other architectures.

CBAM: Convolutional Block Attention Module

CBAM extends the SE idea to both channel and spatial dimensions. First, a channel attention module (similar to SE) computes per-channel weights. Then, a spatial attention module computes per-pixel weights using a small convolution on the channel-pooled features. The two attention maps are applied sequentially, allowing the network to focus on both "what" (which channels matter) and "where" (which spatial locations matter).

Feature Visualization Techniques

How do we know what features each layer learns? Several visualization techniques make the network's internal representations interpretable.

Filter visualization directly plots the learned kernel weights. For the first layer, this is straightforward — the kernels operate on RGB pixels, so you can display them as tiny images. For deeper layers, the kernels operate on abstract feature maps, making direct visualization less informative.

Activation maximization asks: "what input image maximally activates a given neuron?" Starting from random noise, you optimize the input using gradient ascent to maximize a specific neuron's activation. The resulting images reveal what the neuron "wants to see." Early layers produce simple edge patterns; deep layers produce trippy, textured compositions of the objects and concepts the neuron detects.

Gradient-weighted Class Activation Mapping (Grad-CAM) produces a heatmap showing which regions of the input image were most important for a specific prediction. It computes gradients of the target class score with respect to the feature maps in the last conv layer, then weights each feature map by its average gradient. The result is a coarse heatmap overlaid on the original image. If the network classifies an image as "dog," Grad-CAM typically highlights the dog's face and body.

Practical Feature Inspection

When debugging a CNN that's performing poorly, visualize intermediate feature maps and Grad-CAM heatmaps. If the network is "looking" at the wrong part of the image — say, the background instead of the object — it's a strong signal of dataset bias (e.g., all dogs in your dataset appear on grass, so the network learned "grass = dog"). Feature visualization is not just academic; it's a debugging tool.

Understanding feature maps and receptive fields gives you the mental model to reason about CNN behavior. Why does a shallow network struggle with large objects? Small receptive field. Why do fine-grained classification tasks need higher resolution? Because the discriminative features (subtle texture differences between bird species) exist at small spatial scales and get lost by aggressive pooling. Why does transfer learning work? Because the feature hierarchy — edges, textures, parts — is universal across tasks. These aren't just theoretical concepts; they're the lens through which you diagnose and improve real models.
