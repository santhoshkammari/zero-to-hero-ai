Transfer Learning

Chapter 9: CNNs & Computer Vision The practical skill · Fine-tuning · Feature extraction · Model hubs
TL;DR

Never train a vision model from scratch. Load a pretrained backbone, replace the head, and fine-tune. Feature extraction (frozen backbone) works with tiny datasets. Gradual unfreezing with discriminative learning rates is the gold standard for serious projects. The timm library gives you 700+ pretrained models through a single API. Master this one technique and you'll outperform most custom architectures with a fraction of the compute.

If there's one rule in modern computer vision, it's this: never train from scratch. Someone has already trained a model on millions of images — use their features as your starting point. This single technique will save you weeks of training time and give you better results than you'd ever get alone.

I've been doing this for two decades and I can count on one hand the number of times training from scratch was the right call. Transfer learning isn't a shortcut — it's the standard operating procedure. Every winning Kaggle solution, every production vision system, every state-of-the-art result starts with a pretrained model. If you remember nothing else from this chapter, remember this.

Why Transfer Learning Works

Neural networks learn features in a hierarchy. The early layers of any CNN trained on natural images learn remarkably similar things, regardless of what the network was ultimately trained to classify:

  • Layer 1–2: Edges, corners, color gradients — the visual alphabet
  • Layer 3–5: Textures, patterns, repeated motifs — visual words
  • Layer 6–10: Parts of objects — eyes, wheels, fur patches — visual phrases
  • Final layers: Whole objects and their spatial relationships — visual sentences

A model trained on ImageNet has already learned what an edge looks like. Why would you re-learn that? Those first few layers are universal feature detectors. They're useful whether you're classifying cats, detecting tumors, or reading license plates. The deeper you go, the more task-specific the features become — but even the mid-level features (textures, shapes) transfer surprisingly well across domains.

This hierarchy is the key insight. When you do transfer learning, you're reusing the universal features and only learning the task-specific parts on top. Instead of learning from billions of pixel values, you're learning from rich, meaningful feature representations. That's why you can train a good classifier on 200 images when training from scratch would need 200,000.

The math behind it: Yosinski et al. (2014) measured feature transferability layer by layer. They found that the first three layers of a CNN transfer almost perfectly between tasks. Performance drops only when you get to the last few layers, which encode task-specific combinations. This isn't a hack — it's a fundamental property of how neural networks decompose visual information.

Strategy 1: Feature Extraction (Frozen Backbone)

The simplest form of transfer learning. Take a pretrained model, throw away its classification head, freeze all the weights, and train only a new head for your task. The pretrained model becomes a fixed feature extractor — a black box that converts images into rich feature vectors.

import torch
import torch.nn as nn
import torchvision.models as models

# Load a pretrained ResNet-50
model = models.resnet50(weights='IMAGENET1K_V2')

# Freeze ALL parameters — no gradients, no updates
for param in model.parameters():
    param.requires_grad = False

# Replace the classification head
# ResNet-50's fc layer: 2048 → 1000 (ImageNet classes)
# We replace it with: 2048 → num_classes
num_classes = 5
model.fc = nn.Linear(2048, num_classes)
# The new layer's parameters have requires_grad=True by default

# Only the new fc layer will be updated during training
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total = sum(p.numel() for p in model.parameters())
print(f"Training {trainable:,} of {total:,} parameters ({100*trainable/total:.2f}%)")
# Training 10,245 of 25,567,957 parameters (0.04%)

You're training 0.04% of the network. That's it. The other 25.5 million parameters stay exactly as ImageNet left them. Training is blazing fast — often a few minutes on a GPU — because you're only computing gradients for the final layer.

When to Use Feature Extraction

  • Tiny dataset (under 1,000 images) — not enough data to fine-tune without overfitting
  • Similar domain — your images look roughly like natural photos (animals, objects, scenes)
  • Quick prototyping — you need a baseline in 10 minutes, not 10 hours
  • Limited compute — you don't have a GPU or are working on a laptop

A More Expressive Head

A single linear layer is fine for simple tasks, but sometimes you need more capacity in the head:

# Multi-layer head with dropout for regularization
model.fc = nn.Sequential(
    nn.Linear(2048, 512),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(512, num_classes)
)

# For models that use nn.classifier (like EfficientNet, MobileNet):
# model.classifier = nn.Sequential(...)
Watch the architecture: Different pretrained models name their classification head differently. ResNet uses model.fc. VGG and DenseNet use model.classifier. EfficientNet uses model.classifier (a Sequential). Always inspect the model architecture with print(model) before replacing layers — guessing will break things silently.

Strategy 2: Fine-Tuning (The Real Work)

Feature extraction treats the backbone as a black box. Fine-tuning opens that box and adjusts the weights. This is more powerful but more dangerous — you can improve the features for your task, or you can destroy the pretrained knowledge entirely. The difference comes down to technique.

Full Fine-Tuning

Unfreeze everything and train the entire model end-to-end. Simple, but the nuclear option.

# Full fine-tuning: unfreeze all layers
model = models.resnet50(weights='IMAGENET1K_V2')
model.fc = nn.Linear(2048, num_classes)

# Everything is trainable (requires_grad=True by default)
# Use a MUCH lower learning rate than training from scratch
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)  # NOT 1e-2!

The critical detail: use a learning rate 10–100× lower than you'd use for training from scratch. The pretrained weights are already in a good region of the loss landscape. A large learning rate will yank them out of that region and destroy the learned features. Think of it like this: you're making careful adjustments to a finely tuned instrument, not building one from raw materials.

Full fine-tuning works best when you have a large dataset (10K+ images) and enough compute for multiple epochs. With small datasets, you'll overfit to the training data and lose the generalizable pretrained features — the worst of both worlds.

Gradual Unfreezing: The Gold Standard

This is the technique I use on every serious project. Instead of unfreezing everything at once, you unfreeze the model in stages, starting from the head and working backward toward the input. Each stage lets the newly unfrozen layers adapt while the still-frozen layers provide stability.

import torch.optim as optim

model = models.resnet50(weights='IMAGENET1K_V2')
model.fc = nn.Linear(2048, num_classes)

# Phase 1: Train only the head (1-3 epochs)
for param in model.parameters():
    param.requires_grad = False
for param in model.fc.parameters():
    param.requires_grad = True

optimizer = optim.Adam(model.fc.parameters(), lr=1e-3)
# ... train for a few epochs ...

# Phase 2: Unfreeze layer4 (the last residual block)
for param in model.layer4.parameters():
    param.requires_grad = True

optimizer = optim.Adam([
    {'params': model.fc.parameters(),     'lr': 1e-3},
    {'params': model.layer4.parameters(), 'lr': 1e-4},
], lr=1e-4)
# ... train for a few more epochs ...

# Phase 3: Unfreeze layer3
for param in model.layer3.parameters():
    param.requires_grad = True

optimizer = optim.Adam([
    {'params': model.fc.parameters(),     'lr': 1e-3},
    {'params': model.layer3.parameters(), 'lr': 1e-5},
    {'params': model.layer4.parameters(), 'lr': 1e-4},
], lr=1e-5)
# ... train for a few more epochs ...

# Phase 4 (optional): Unfreeze layer2 and layer1
# Only if you have enough data and the domain is very different

Why does this work so well? Two reasons:

  1. The head adapts first. When you unfreeze deeper layers, the gradients flowing back through them are meaningful — they're optimizing for your task, not for random initial weights in the head.
  2. Early layers need less change. Edges and textures are universal. You almost never need to significantly modify layer1 or layer2. Gradual unfreezing naturally respects this hierarchy.

Discriminative Learning Rates

The complement to gradual unfreezing. Instead of one learning rate for the whole model, assign different rates to different layer groups — lower rates for early layers (don't disturb universal features), higher rates for later layers (adapt to your task).

# Discriminative learning rates in one shot
model = models.resnet50(weights='IMAGENET1K_V2')
model.fc = nn.Linear(2048, num_classes)

# Group parameters by depth
param_groups = [
    {'params': model.conv1.parameters(),  'lr': 1e-6},  # barely touch
    {'params': model.bn1.parameters(),    'lr': 1e-6},
    {'params': model.layer1.parameters(), 'lr': 1e-5},
    {'params': model.layer2.parameters(), 'lr': 1e-5},
    {'params': model.layer3.parameters(), 'lr': 1e-4},
    {'params': model.layer4.parameters(), 'lr': 1e-4},
    {'params': model.fc.parameters(),     'lr': 1e-3},  # learn fast
]

optimizer = optim.Adam(param_groups)

# Common pattern: exponential decay from head to stem
# base_lr * decay_factor^(num_groups - group_index - 1)
# With base_lr=1e-3 and decay=0.1:
# head: 1e-3, layer4: 1e-4, layer3: 1e-5, layer2: 1e-6, ...

The exponential decay pattern is elegant: each layer group gets a learning rate that's 10× smaller than the one above it. The head learns aggressively, the mid-layers adjust cautiously, and the early layers barely move. This single trick alone can improve your results by 2-5% accuracy over a flat learning rate.

Choosing Your Strategy

Here's the decision matrix I've used on hundreds of projects. It's not a formula — use it as a starting point, then validate with your data.

Your Dataset Size Similar to ImageNet? Recommended Strategy Learning Rate
Small (<1K images) Yes Feature extraction only 1e-3 (head only)
Small (<1K images) No Feature extraction + heavy augmentation 1e-3 (head only)
Medium (1K–10K) Yes Fine-tune last 1–2 layer groups 1e-4 backbone, 1e-3 head
Medium (1K–10K) No Gradual unfreezing + discriminative LR 1e-5 to 1e-3 (layered)
Large (>10K images) Yes Full fine-tuning, low LR 1e-4 flat
Large (>10K images) No Full fine-tuning or train from scratch 1e-4 (fine-tune) or 1e-2 (scratch)

"Similar to ImageNet" means your images are natural photographs — things you'd take with a phone camera. Dogs, food, landscapes, products, buildings. If your images are medical scans, satellite imagery, spectrograms, or microscopy, that's a different domain — the pretrained features still help, but less reliably.

Practical Best Practices

These are the things that separate a transfer learning tutorial from a production-quality pipeline. I've learned each one the hard way.

1. Freeze BatchNorm Layers

This one catches everyone. BatchNorm layers maintain running statistics (mean and variance) computed during pretraining on ImageNet's 1.2 million images. When you fine-tune, if BatchNorm is in training mode, it recomputes those statistics from your mini-batches — which might be 32 images from a dataset of 500. The statistics become noisy garbage, and your model's performance collapses.

def freeze_batchnorm(model):
    """Set all BatchNorm layers to eval mode.
    Call this AFTER model.train() in your training loop."""
    for module in model.modules():
        if isinstance(module, (nn.BatchNorm2d, nn.BatchNorm1d)):
            module.eval()

# In your training loop:
model.train()
freeze_batchnorm(model)  # override BN layers back to eval
for images, labels in dataloader:
    # ... training step ...
The silent killer: Forgetting to freeze BatchNorm is the #1 reason fine-tuning "doesn't work." Your training loss will look fine, but validation accuracy will be 5–15% worse than expected. Always freeze BN when fine-tuning with small datasets. Only unfreeze BN if you have 50K+ images and are doing full fine-tuning.

2. Use the Right Preprocessing

Pretrained models expect inputs normalized with the same statistics used during pretraining. For ImageNet models, that's a specific mean and standard deviation per channel. Get this wrong and your features are meaningless.

from torchvision import transforms

# ImageNet normalization — use this for ANY ImageNet-pretrained model
imagenet_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.485, 0.456, 0.406],
        std=[0.229, 0.224, 0.225]
    ),
])

# For training, add augmentation BEFORE normalization
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ColorJitter(brightness=0.2, contrast=0.2),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225]),
])

3. Data Augmentation Matters MORE with Transfer Learning

You're fine-tuning on fewer images than the model was originally trained on. Every image needs to count. Aggressive augmentation — random crops, flips, color jitter, rotation — artificially expands your dataset and prevents the fine-tuned layers from memorizing specific training examples.

Think of it this way: if you have 500 images and train for 20 epochs, the model sees each image 20 times. With augmentation, it sees 20 different versions of each image. That's the difference between memorization and generalization.

4. Monitor for Catastrophic Forgetting

Catastrophic forgetting happens when fine-tuning destroys the useful pretrained features. Signs to watch for:

  • Training loss drops quickly but validation loss stalls or increases
  • Performance on similar-but-different tasks degrades sharply
  • Early stopping triggers much sooner than expected

Prevention: lower learning rates, gradual unfreezing, fewer epochs, and more regularization (dropout, weight decay). If you see forgetting, you're fine-tuning too aggressively — dial back the learning rate or freeze more layers.

5. Use a Learning Rate Finder

Don't guess the learning rate. Use a learning rate range test to find the sweet spot:

# Using torch-lr-finder (pip install torch-lr-finder)
from torch_lr_finder import LRFinder

model = models.resnet50(weights='IMAGENET1K_V2')
model.fc = nn.Linear(2048, num_classes)

criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-7)

lr_finder = LRFinder(model, optimizer, criterion, device="cuda")
lr_finder.range_test(train_loader, end_lr=1e-1, num_iter=100)
lr_finder.plot()  # look for the steepest downward slope
lr_finder.reset()

Pick the learning rate where the loss is decreasing most steeply — usually about 10× before the minimum. For fine-tuning, this typically lands between 1e-5 and 1e-3.

Pretrained Model Hubs

You're not limited to the handful of models in torchvision. Modern model hubs give you hundreds of pretrained architectures with consistent APIs.

timm: The Secret Weapon

timm (PyTorch Image Models) is Ross Wightman's library with 700+ pretrained vision models. If you do any serious computer vision work, you'll use it constantly. Consistent API, every architecture you can name, and weights pretrained on ImageNet-1K, ImageNet-21K, and other large-scale datasets.

import timm

# List all available models (there are a LOT)
all_models = timm.list_models(pretrained=True)
print(f"{len(all_models)} pretrained models available")

# Search for specific architectures
efficientnets = timm.list_models('efficientnet*', pretrained=True)
print(efficientnets[:5])
# ['efficientnet_b0', 'efficientnet_b1', 'efficientnet_b2', ...]

# Create a model with a custom number of classes — it handles the head for you
model = timm.create_model('efficientnet_b0', pretrained=True, num_classes=10)

# Need the feature extractor only? Get the embedding vector.
model = timm.create_model('efficientnet_b0', pretrained=True, num_classes=0)
# num_classes=0 removes the head, output is the pooled feature vector

# Get the right transforms for any model
data_config = timm.data.resolve_model_data_config(model)
val_transform = timm.data.create_transform(**data_config, is_training=False)
train_transform = timm.data.create_transform(**data_config, is_training=True)
Pro tip: timm handles the preprocessing automatically. Each model knows its own input size, normalization stats, and interpolation mode. Use timm.data.create_transform() instead of manually specifying ImageNet stats — it's correct for every model, even those not trained on ImageNet.

torchvision

The standard library. Fewer models but rock-solid. Good for ResNet, EfficientNet, ViT, and the classics.

# torchvision with the modern weights API
from torchvision.models import efficientnet_b0, EfficientNet_B0_Weights

weights = EfficientNet_B0_Weights.IMAGENET1K_V1

# Replace the classifier
model.classifier[1] = nn.Linear(1280, num_classes)

Complete Fine-Tuning Pipeline

Let's put it all together. Here's a production-quality fine-tuning pipeline with gradual unfreezing, discriminative learning rates, frozen BatchNorm, and proper validation.

import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import timm

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# 1. Create model with timm
model = timm.create_model('resnet50', pretrained=True, num_classes=10)
model = model.to(device)

# 2. Data transforms (timm handles preprocessing)
data_config = timm.data.resolve_model_data_config(model)
train_transform = timm.data.create_transform(**data_config, is_training=True)
val_transform = timm.data.create_transform(**data_config, is_training=False)

train_dataset = datasets.ImageFolder('data/train', transform=train_transform)
val_dataset = datasets.ImageFolder('data/val', transform=val_transform)
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True, num_workers=4)
val_loader = DataLoader(val_dataset, batch_size=32, shuffle=False, num_workers=4)

criterion = nn.CrossEntropyLoss()

def freeze_batchnorm(m):
    for module in m.modules():
        if isinstance(module, (nn.BatchNorm2d, nn.BatchNorm1d)):
            module.eval()

def train_one_epoch(model, loader, optimizer, criterion):
    model.train()
    freeze_batchnorm(model)
    running_loss, correct, total = 0.0, 0, 0
    for images, labels in loader:
        images, labels = images.to(device), labels.to(device)
        optimizer.zero_grad()
        outputs = model(images)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
        running_loss += loss.item() * images.size(0)
        correct += (outputs.argmax(1) == labels).sum().item()
        total += labels.size(0)
    return running_loss / total, correct / total

@torch.no_grad()
def evaluate(model, loader, criterion):
    model.eval()
    running_loss, correct, total = 0.0, 0, 0
    for images, labels in loader:
        images, labels = images.to(device), labels.to(device)
        outputs = model(images)
        loss = criterion(outputs, labels)
        running_loss += loss.item() * images.size(0)
        correct += (outputs.argmax(1) == labels).sum().item()
        total += labels.size(0)
    return running_loss / total, correct / total

# ── Phase 1: Train head only ──
for param in model.parameters():
    param.requires_grad = False
for param in model.get_classifier().parameters():
    param.requires_grad = True

optimizer = optim.AdamW(model.get_classifier().parameters(), lr=1e-3, weight_decay=0.01)

for epoch in range(3):
    train_loss, train_acc = train_one_epoch(model, train_loader, optimizer, criterion)
    val_loss, val_acc = evaluate(model, val_loader, criterion)
    print(f"Phase 1 | Epoch {epoch+1} | Train Acc: {train_acc:.3f} | Val Acc: {val_acc:.3f}")

# ── Phase 2: Unfreeze last block + head ──
for param in model.layer4.parameters():
    param.requires_grad = True

optimizer = optim.AdamW([
    {'params': model.get_classifier().parameters(), 'lr': 1e-3},
    {'params': model.layer4.parameters(),           'lr': 1e-4},
], weight_decay=0.01)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=5)

for epoch in range(5):
    train_loss, train_acc = train_one_epoch(model, train_loader, optimizer, criterion)
    val_loss, val_acc = evaluate(model, val_loader, criterion)
    scheduler.step()
    print(f"Phase 2 | Epoch {epoch+1} | Train Acc: {train_acc:.3f} | Val Acc: {val_acc:.3f}")

# ── Phase 3: Unfreeze more layers if needed ──
for param in model.layer3.parameters():
    param.requires_grad = True

optimizer = optim.AdamW([
    {'params': model.get_classifier().parameters(), 'lr': 5e-4},
    {'params': model.layer4.parameters(),           'lr': 5e-5},
    {'params': model.layer3.parameters(),           'lr': 1e-5},
], weight_decay=0.01)
scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=5)

for epoch in range(5):
    train_loss, train_acc = train_one_epoch(model, train_loader, optimizer, criterion)
    val_loss, val_acc = evaluate(model, val_loader, criterion)
    scheduler.step()
    print(f"Phase 3 | Epoch {epoch+1} | Train Acc: {train_acc:.3f} | Val Acc: {val_acc:.3f}")

When Transfer Learning Hurts: Negative Transfer

Transfer learning isn't magic. It can actually hurt performance when the source and target domains are too different. This is called negative transfer, and it's more common than people admit.

When to Suspect Negative Transfer

  • Your images look nothing like natural photos. Medical X-rays, satellite imagery, spectrograms, scientific visualizations — these are structurally different from ImageNet photos. The edge detectors still help, but the mid-level features (fur textures, car parts) are useless or misleading.
  • Fine-tuning performs worse than training from scratch. If a randomly initialized model outperforms your fine-tuned one given enough training data, the pretrained features are actively interfering.
  • You see early saturation. The model quickly reaches a mediocre accuracy and then flatlines, as if the pretrained features create a ceiling the model can't break through.

What to Do About It

Situation Solution
Moderate domain shift Use only early layers (freeze after layer2), train deeper layers from scratch
Large domain shift Find a domain-specific pretrained model (e.g., models pretrained on medical images)
Very large dataset + different domain Train from scratch — you have enough data and the pretrained features don't help
Small dataset + different domain Self-supervised pretraining on your unlabeled data, then fine-tune 
# Partial transfer: use only early layers, reinitialize deeper ones
model = models.resnet50(weights='IMAGENET1K_V2')

# Keep pretrained weights for layer1 and layer2
# Reinitialize layer3, layer4, and fc from scratch
for name, module in model.named_children():
    if name in ('layer3', 'layer4', 'fc'):
        for param in module.parameters():
            if param.dim() > 1:
                nn.init.kaiming_normal_(param)
            else:
                nn.init.zeros_(param)

model.fc = nn.Linear(2048, num_classes)

Knowledge Distillation

Knowledge distillation is transfer learning's deployment-friendly cousin. You have a large, accurate model (the teacher) that's too slow for production. You train a small, fast model (the student) to mimic the teacher's outputs — not just the hard labels, but the soft probability distributions that encode the teacher's "dark knowledge."

import torch.nn.functional as F

def distillation_loss(student_logits, teacher_logits, labels,
                      temperature=4.0, alpha=0.7):
    """Combine soft targets from teacher with hard labels."""
    # Soft targets: student tries to match teacher's probability distribution
    soft_loss = F.kl_div(
        F.log_softmax(student_logits / temperature, dim=1),
        F.softmax(teacher_logits / temperature, dim=1),
        reduction='batchmean'
    ) * (temperature ** 2)

    # Hard targets: standard cross-entropy with true labels
    hard_loss = F.cross_entropy(student_logits, labels)

    return alpha * soft_loss + (1 - alpha) * hard_loss

# Training loop sketch
teacher_model.eval()
student_model.train()

for images, labels in train_loader:
    images, labels = images.to(device), labels.to(device)

    with torch.no_grad():
        teacher_logits = teacher_model(images)

    student_logits = student_model(images)
    loss = distillation_loss(student_logits, teacher_logits, labels)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Why temperature? At temperature=1 (normal softmax), the teacher's output for a cat image might be [0.98, 0.01, 0.01] — almost all the information is in the top class. At temperature=4, those logits get softened to something like [0.60, 0.22, 0.18], which reveals that the teacher thinks this cat looks somewhat like a dog and a bit like a fox. That's the "dark knowledge" — the relationships between classes that the student can learn from.

When to use distillation: You need to deploy on mobile/edge devices. You have a ViT-Large that's 95% accurate but takes 200ms per image. Distill it into a MobileNetV3 that's 92% accurate but takes 5ms. Three percent accuracy for 40× speedup is usually a great trade.

Domain Adaptation

Domain adaptation is what you reach for when your training data (source domain) and deployment data (target domain) come from different distributions, but the task is the same. Classic example: you have 100K labeled product photos taken in a studio (source), but in production, users upload blurry phone photos in variable lighting (target).

The core idea: learn features that are discriminative for the task but invariant to the domain. If the feature extractor can't tell whether an image came from the source or target domain, it's learned features that generalize across both.

Key Techniques

  • Adversarial domain adaptation (DANN): Add a domain classifier that tries to distinguish source from target. Train the feature extractor to fool the domain classifier via a gradient reversal layer. The feature extractor learns domain-invariant representations.
  • Self-training on target domain: Train on labeled source data, generate pseudo-labels for unlabeled target data using the model's predictions, then retrain on both. Iterate. Simple but surprisingly effective.
  • Feature alignment: Minimize the distance between source and target feature distributions using MMD (Maximum Mean Discrepancy) or CORAL (CORrelation ALignment).
# Pseudo-label self-training (simplest domain adaptation)
# Step 1: Train on source (labeled) data
model = timm.create_model('resnet50', pretrained=True, num_classes=num_classes)
# ... train normally on source data ...

# Step 2: Generate pseudo-labels for target data
model.eval()
pseudo_labels = []
confidences = []
with torch.no_grad():
    for images in target_loader:
        logits = model(images.to(device))
        probs = torch.softmax(logits, dim=1)
        conf, preds = probs.max(dim=1)
        pseudo_labels.append(preds)
        confidences.append(conf)

# Step 3: Keep only high-confidence predictions (threshold: 0.9+)
pseudo_labels = torch.cat(pseudo_labels)
confidences = torch.cat(confidences)
mask = confidences > 0.9
print(f"Kept {mask.sum()} of {len(mask)} target samples ({100*mask.float().mean():.1f}%)")

Domain adaptation is a deep topic with active research. For most practical purposes, combining a good pretrained model with data augmentation that mimics the target domain (adding blur, changing lighting, adjusting colors) gets you 80% of the way there without any special machinery.
