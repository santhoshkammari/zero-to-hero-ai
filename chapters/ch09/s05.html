Transfer Learning

Chapter 9: CNNs & Computer Vision 8 subtopics
TL;DR

Transfer learning reuses a model pre-trained on a large dataset (like ImageNet) for a new task. Feature extraction freezes the pre-trained backbone and trains only a new classifier head. Fine-tuning unfreezes some or all layers and trains them with a small learning rate. This works because early CNN features (edges, textures) are universal. Use differential learning rates, freeze BatchNorm, and start with feature extraction before gradually unfreezing. Transfer learning is the single most impactful technique for getting good results with limited data.

Here's the reality of applied deep learning: you almost never train from scratch. Training a ResNet-50 on ImageNet from random initialization takes days on 8 GPUs and requires 1.3 million labeled images. Most real-world projects have a few thousand images, a single GPU, and a deadline. Transfer learning bridges this gap by starting from a model that already knows how to see.

The insight is straightforward. As we discussed in Section 3, CNNs learn a hierarchy: edges → textures → parts → objects. The early layers — edge detectors, texture recognizers — are nearly identical regardless of the task. A horizontal edge is a horizontal edge whether you're classifying cats, detecting tumors, or reading satellite imagery. Only the deep layers specialize for the specific task. So why not reuse the general features and only retrain the task-specific parts?

Feature Extraction: Frozen Backbone

Feature extraction is the simplest form of transfer learning. You take a pre-trained network, remove the final classification layer, freeze all the remaining weights (no gradient updates), and add a new classification head for your task. The pre-trained network becomes a fixed feature extractor — it converts your images into rich feature vectors, and you only train the new head.

import torch
import torch.nn as nn
import torchvision.models as models

# Load a pre-trained ResNet-50
model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2)

# Freeze ALL parameters
for param in model.parameters():
    param.requires_grad = False

# Replace the final FC layer (originally 2048 → 1000 for ImageNet)
# with a new head for our task (e.g., 5 classes)
num_classes = 5
model.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(2048, num_classes)
)
# Only model.fc parameters will be updated during training

# Verify: count trainable vs frozen parameters
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total = sum(p.numel() for p in model.parameters())
print(f"Trainable: {trainable:,} / {total:,} ({100*trainable/total:.1f}%)")
# Trainable: 10,245 / 23,518,277 (0.04%)

Feature extraction is fast (you're training less than 1% of the parameters), resistant to overfitting (the frozen backbone can't memorize your small dataset), and often surprisingly effective. For many tasks — medical imaging, satellite imagery, industrial inspection — just slapping a linear classifier on top of ImageNet features gets you 80-90% of the way to the best possible accuracy.

When does feature extraction struggle? When your target domain looks very different from ImageNet. ImageNet is natural photographs of everyday objects. If your target domain is X-ray images, microscopy, or spectrograms, the pre-trained features may be too "natural-photo-specific" to transfer well. In those cases, you need fine-tuning.

Fine-Tuning Strategies

Fine-tuning unfreezes some or all of the pre-trained layers and trains them with a small learning rate. The key word is "small." These weights are already in a good region of the loss landscape — you want to gently nudge them toward your task, not blast them with large gradient updates that destroy the pre-trained features.

Gradual Unfreezing

The safest approach is gradual unfreezing. Start with everything frozen except the new head. Train for a few epochs. Then unfreeze the last conv block and train for a few more epochs. Then unfreeze the second-to-last block, and so on. Each time you unfreeze more layers, you let the network adapt deeper features to your task — but only after the head has stabilized, reducing the risk of destructive gradient updates to the backbone.

# Gradual unfreezing example with ResNet-50

# Phase 1: Train only the head (all backbone frozen)
model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2)
for param in model.parameters():
    param.requires_grad = False
model.fc = nn.Linear(2048, num_classes)

optimizer = torch.optim.Adam(model.fc.parameters(), lr=1e-3)
# ... train for 5 epochs ...

# Phase 2: Unfreeze layer4 (last residual block)
for param in model.layer4.parameters():
    param.requires_grad = True

optimizer = torch.optim.Adam([
    {'params': model.fc.parameters(), 'lr': 1e-3},
    {'params': model.layer4.parameters(), 'lr': 1e-4},  # 10× smaller LR
])
# ... train for 5 more epochs ...

# Phase 3: Unfreeze layer3
for param in model.layer3.parameters():
    param.requires_grad = True

optimizer = torch.optim.Adam([
    {'params': model.fc.parameters(), 'lr': 1e-3},
    {'params': model.layer4.parameters(), 'lr': 1e-4},
    {'params': model.layer3.parameters(), 'lr': 1e-5},  # even smaller
])
# ... train for 5 more epochs ...

Discriminative Learning Rates

Discriminative learning rates (also called differential or layer-wise learning rates) assign different learning rates to different parts of the network. Early layers get very small learning rates (their features are general and shouldn't change much), while later layers and the new head get larger learning rates (they need more adaptation).

A common scheme is geometric decay: if the head uses learning rate lr, each preceding layer group uses lr × decay^n where n is the distance from the head. With decay=0.1, the head trains at 1e-3, the last block at 1e-4, the second-to-last at 1e-5, and so on. This was popularized by the ULMFiT paper and is standard practice in fine-tuning.

# Discriminative learning rates with layer-wise LR decay
def get_param_groups(model, base_lr=1e-3, lr_decay=0.1):
    """Assign geometrically decaying LRs from head to early layers."""
    param_groups = []

    # Head: highest LR
    param_groups.append({
        'params': model.fc.parameters(),
        'lr': base_lr
    })

    # Layer groups from deep to shallow
    layer_groups = [model.layer4, model.layer3, model.layer2, model.layer1]
    for i, layer in enumerate(layer_groups, 1):
        param_groups.append({
            'params': layer.parameters(),
            'lr': base_lr * (lr_decay ** i)
        })

    # Stem (conv1, bn1): smallest LR
    param_groups.append({
        'params': list(model.conv1.parameters()) + list(model.bn1.parameters()),
        'lr': base_lr * (lr_decay ** 5)
    })

    return param_groups

# Unfreeze everything, but with differential LRs
for param in model.parameters():
    param.requires_grad = True

param_groups = get_param_groups(model, base_lr=1e-3, lr_decay=0.1)
optimizer = torch.optim.AdamW(param_groups, weight_decay=0.01)

Fine-Tuning Best Practices

Freeze BatchNorm Layers

This is a subtle but important detail. BatchNorm layers maintain running statistics (mean and variance) computed over the training data. During fine-tuning on a small dataset, these statistics can drift wildly because each batch is tiny and non-representative. The fix: keep BatchNorm layers in eval mode even during training, so they use the pre-trained running statistics instead of recomputing from your small batches.

# Freeze BatchNorm during fine-tuning
def freeze_batchnorm(model):
    for module in model.modules():
        if isinstance(module, (nn.BatchNorm2d, nn.BatchNorm1d)):
            module.eval()  # use running stats, don't update them
            for param in module.parameters():
                param.requires_grad = False

# Call after model.train() in your training loop
model.train()
freeze_batchnorm(model)  # keeps BN in eval mode

Data Augmentation Matters More

With small datasets, the augmentation strategy often matters more than the architecture or learning rate schedule. Use aggressive augmentation: random crops, horizontal flips, color jitter, RandAugment, or Mixup/CutMix. These effectively multiply your dataset size and prevent the fine-tuned model from memorizing the few training examples.

Pre-Trained Model Hubs

You don't need to train your own backbones. Two major hubs provide hundreds of pre-trained models:

torchvision.models ships with PyTorch and includes all the classics: ResNet, VGG, EfficientNet, MobileNet, ViT, Swin Transformer, and more. Each comes with pre-trained weights on ImageNet. The API is clean and well-documented. For most projects, this is your starting point.

timm (PyTorch Image Models by Ross Wightman) is the power user's choice. It provides over 700 model architectures with thousands of pre-trained weight variants, including models trained on ImageNet-21K, LAION, and other large datasets. Many of these achieve higher accuracy than the torchvision defaults because they were trained with modern recipes (longer training, stronger augmentation, label smoothing).

# torchvision: simple and official
import torchvision.models as models
resnet = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2)
effnet = models.efficientnet_b0(weights=models.EfficientNet_B0_Weights.IMAGENET1K_V1)

# timm: massive collection with modern training recipes
import timm

# List available models matching a pattern
print(timm.list_models('convnext*'))
# ['convnext_base', 'convnext_large', 'convnext_small', 'convnext_tiny', ...]

# Load with pre-trained weights, custom number of classes
model = timm.create_model('convnext_tiny', pretrained=True, num_classes=5)

# timm handles the head replacement for you — just set num_classes
# It also provides the correct preprocessing transforms
data_config = timm.data.resolve_model_data_config(model)
transforms = timm.data.create_transform(**data_config, is_training=True)

When Transfer Learning Helps — and When It Hurts

When It Helps

Transfer learning works best when: (1) your target dataset is small (under 10,000 images), (2) your target domain is similar to the pre-training domain (natural images → natural images), and (3) you need fast iteration (days of fine-tuning instead of weeks of training from scratch). In these conditions, transfer learning typically provides 10-30% accuracy improvement over training from scratch.

Negative Transfer

Negative transfer occurs when the pre-trained features actually hurt performance on the target task. This happens when the domains are too different — ImageNet features may not help with medical ultrasound images, since the visual primitives (edges, textures) look fundamentally different. In extreme cases, fine-tuned models perform worse than training from scratch.

To assess domain gap, compare the statistics of your data to the pre-training data. If your images have very different resolution, color distributions, or visual structure, consider: (1) using only very early pre-trained layers (edges transfer better than object-specific features), (2) training from scratch with a smaller model, or (3) using self-supervised pre-training on your own data first.

Catastrophic Forgetting

Catastrophic forgetting is when fine-tuning overwrites the useful pre-trained features. This happens most often when you fine-tune aggressively (large learning rate, many epochs) on a small dataset. The model memorizes the fine-tuning data but loses the general representations it learned during pre-training. Gradual unfreezing, differential learning rates, and early stopping all help mitigate this.

Domain Adaptation

Domain adaptation addresses the scenario where your source and target domains have different distributions but the same task. For example, training on synthetic rendered images (source) and deploying on real photographs (target). The gap between domains — different lighting, textures, noise characteristics — causes a distribution shift that degrades model performance.

Techniques include adversarial domain adaptation (train a domain discriminator that the feature extractor tries to fool, forcing domain-invariant features), style transfer (make source images look like target images before training), and self-training (use the model's confident predictions on unlabeled target data as pseudo-labels for further training).

Few-Shot Transfer

What if you have only 5 or 10 examples per class? Standard fine-tuning tends to overfit catastrophically with so little data. Few-shot transfer strategies include:

Freezing the entire backbone and training only a simple classifier (nearest-centroid, cosine similarity, or logistic regression) on the extracted features. With a strong backbone, this can work surprisingly well — the features are already so good that a linear classifier suffices.

Using prototypical networks or metric learning approaches that learn an embedding space where same-class examples are close and different-class examples are far apart. At inference, you classify by finding the nearest class prototype in embedding space.

Applying heavy augmentation — with 10 images, every augmented variant matters. Techniques like CutMix, which splices patches from different training images together, are particularly effective because they effectively create new compositions from your tiny dataset.

# Few-shot feature extraction with a frozen backbone
import torch.nn.functional as F

# Extract features from your few-shot support set
model.eval()
features = []
labels = []
with torch.no_grad():
    for img, label in few_shot_loader:
        # Get features before the FC layer
        feat = model.avgpool(model.layer4(model.layer3(
            model.layer2(model.layer1(model.maxpool(model.relu(
            model.bn1(model.conv1(img)))))))))
        features.append(feat.flatten(1))
        labels.append(label)

features = torch.cat(features)
labels = torch.cat(labels)

# Compute class prototypes (mean feature per class)
prototypes = {}
for c in labels.unique():
    prototypes[c.item()] = features[labels == c].mean(dim=0)

# Classify a new image by nearest prototype
def classify(query_feat, prototypes):
    dists = {c: F.cosine_similarity(query_feat.unsqueeze(0),
              proto.unsqueeze(0)) for c, proto in prototypes.items()}
    return max(dists, key=dists.get)

Knowledge Distillation

Knowledge distillation is a related technique where a large "teacher" model trains a smaller "student" model. Instead of training the student on hard labels (one-hot class vectors), you train it to match the teacher's soft predictions — the full probability distribution across all classes.

Why soft labels? A teacher's output of [0.7, 0.2, 0.1] for [cat, dog, fox] carries more information than the hard label [1, 0, 0]. The soft label says "this is a cat, but it looks a bit like a dog." This "dark knowledge" helps the student learn better representations. A temperature parameter T softens the distribution — higher T produces more uniform distributions that reveal more inter-class relationships.

Knowledge distillation is how you deploy large models in production. Train a massive EfficientNet-L2 or ViT-L for maximum accuracy, then distill it into a MobileNet-V3 that runs in real-time on a phone. DeiT used knowledge distillation from a CNN teacher to train Vision Transformers effectively on ImageNet-scale data.

Transfer Learning Decision Flowchart

Small dataset, similar domain? → Feature extraction (freeze backbone, train head only).
Small dataset, different domain? → Feature extraction with very early layers only, or train from scratch with a small model.
Medium dataset, similar domain? → Fine-tune with discriminative learning rates and gradual unfreezing.
Medium dataset, different domain? → Fine-tune aggressively, possibly from a self-supervised pre-trained model.
Large dataset, any domain? → Training from scratch is viable, but fine-tuning from a pre-trained model still usually converges faster.

Transfer learning is arguably the most important practical technique in deep learning. It democratized computer vision — you don't need Google's dataset or compute budget to build a high-quality image classifier. A pre-trained backbone, a few hundred labeled examples, and thoughtful fine-tuning will get you remarkably far.
