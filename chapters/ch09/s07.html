Modern Vision

Chapter 9: CNNs & Computer Vision CLIP · Stable Diffusion · Multimodal LLMs · Vision-Language
TL;DR

The boundary between vision and language has collapsed. CLIP connects images and text in a shared embedding space, enabling zero-shot classification without any task-specific training. Diffusion models like Stable Diffusion generate images from text by learning to reverse a noise-adding process. Multimodal LLMs — GPT-4V, LLaVA, Gemini — can see and discuss images by projecting visual features into a language model's embedding space. GANs dominated generative vision for years but have been largely displaced by diffusion models. Self-supervised models like DINOv2 learn powerful visual representations without labels. Understanding what these models do and when to reach for them is more important than knowing every implementation detail.

The Vision-Language Convergence

The boundary between vision and language has collapsed. Modern models don't just see images — they understand them in terms of language. CLIP connects images and text. Stable Diffusion generates images from words. GPT-4V discusses photographs. This is where computer vision is heading.

For most of computer vision's history, vision systems and language systems lived in completely separate worlds. An image classifier output a class index. A text model output a sequence of tokens. The two never talked. The breakthrough came from a deceptively simple idea: what if we trained an image encoder and a text encoder to put matching images and descriptions near each other in the same embedding space? That idea — contrastive vision-language learning — unlocked everything that followed.

Let's walk through each piece.

CLIP: Contrastive Language-Image Pre-training

CLIP (OpenAI, 2021) is arguably the single most influential model in modern vision. The idea: train an image encoder and a text encoder simultaneously so that matching image-text pairs produce similar embeddings, while non-matching pairs produce dissimilar ones. That's it. No class labels. No bounding boxes. Just images paired with their text descriptions — 400 million of them scraped from the internet.

How CLIP Works

The training setup is beautifully simple. Take a batch of N image-text pairs. Encode all N images with the image encoder (a Vision Transformer or ResNet). Encode all N texts with the text encoder (a Transformer). Now you have N image embeddings and N text embeddings. The correct pairings lie on the diagonal of the N×N similarity matrix — image 1 should match text 1, image 2 should match text 2, and so on. Push matching pairs together, push all other pairs apart. That's contrastive learning.

  CLIP Training: Contrastive Learning
  ────────────────────────────────────

  Batch of N image-text pairs:

  Images                    Texts
  ───────                   ─────
  🖼️ dog on beach          "a dog playing on a sandy beach"
  🖼️ red sports car        "red sports car on highway"
  🖼️ mountain sunset       "sunset over snow-capped mountains"

  Image Encoder              Text Encoder
  (ViT-B/32)                 (Transformer)
       │                          │
       ▼                          ▼
  [img₁ emb]                [txt₁ emb]
  [img₂ emb]                [txt₂ emb]
  [img₃ emb]                [txt₃ emb]

  Cosine Similarity Matrix (N × N):
                  txt₁    txt₂    txt₃
       img₁  [  0.95    0.12    0.08  ]  ← maximize diagonal
       img₂  [  0.10    0.93    0.15  ]  ← minimize off-diagonal
       img₃  [  0.07    0.11    0.96  ]

Zero-Shot Classification

Here's where CLIP gets magical. Once trained, you can classify images into any set of categories — without any task-specific training data. Describe your classes in text, encode them, encode the image, and pick the closest text embedding. Want to classify animals? Encode "a photo of a cat," "a photo of a dog," "a photo of a bird." Encode your image. Cosine similarity tells you which class it is.

# Zero-shot classification with CLIP
import clip
import torch
from PIL import Image

model, preprocess = clip.load("ViT-B/32", device="cuda")

image = preprocess(Image.open("mystery_animal.jpg")).unsqueeze(0).to("cuda")

# Define classes as natural language — no training needed
texts = clip.tokenize([
    "a photo of a cat",
    "a photo of a dog",
    "a photo of a bird",
    "a photo of a fish"
]).to("cuda")

with torch.no_grad():
    image_features = model.encode_image(image)
    text_features = model.encode_text(texts)

    # Normalize and compute similarity
    image_features /= image_features.norm(dim=-1, keepdim=True)
    text_features /= text_features.norm(dim=-1, keepdim=True)
    similarity = (image_features @ text_features.T).softmax(dim=-1)

print(similarity)  # e.g., [0.02, 0.91, 0.05, 0.02] → it's a dog!

No labeled training data. No fine-tuning. Just text descriptions and cosine similarity. This is what "zero-shot" means — the model has never seen labeled examples of your specific task, yet it can perform it. CLIP's zero-shot accuracy on ImageNet is competitive with a fully supervised ResNet-50 that was trained on 1.2 million labeled images.

Why CLIP Matters So Much

CLIP isn't just a clever classifier. It's a foundation that everything else builds on:

  • Stable Diffusion uses CLIP's text encoder to understand text prompts
  • Open-vocabulary object detection uses CLIP embeddings to detect objects from text descriptions without fixed class sets
  • Visual search engines use CLIP to find images by text queries (or similar images by image queries)
  • Multimodal LLMs often start with CLIP's image encoder as their vision backbone
Prompt Engineering for Vision

CLIP is sensitive to how you phrase class descriptions. "A photo of a dog" works better than just "dog" because the training data was image-caption pairs, not single-word labels. For even better accuracy, use ensembles of prompts: "a photo of a dog," "a picture of a dog," "a drawing of a dog" — and average the text embeddings.

DINOv2: Self-Supervised Visual Features

CLIP learns from image-text pairs. But what if you don't have text at all — just images? DINOv2 (Meta, 2023) is a self-supervised vision foundation model that learns powerful visual representations from images alone. No labels. No captions. Just raw images and clever self-supervision.

The training uses a combination of self-distillation and masked image modeling. A student network sees partial views of an image (cropped, masked patches) and tries to match the representations of a teacher network that sees the full image. The teacher is updated as a moving average of the student — no separate labels needed. The result: features that capture fine-grained spatial information, textures, shapes, and semantic concepts — all without a single human annotation.

Why care about DINOv2? Because its features are remarkably versatile. Take a DINOv2 backbone, freeze it, and add a simple linear head — you'll get strong performance on classification, segmentation, depth estimation, and more. It's the closest thing we have to a "universal vision feature extractor." When you need visual features but don't have labeled data for your specific task, DINOv2 is your first call.

  CLIP vs. DINOv2 — Different Strengths
  ────────────────────────────────────────

  CLIP                              DINOv2
  ────                              ──────
  Learns from image-text pairs      Learns from images alone
  Great for text-guided tasks       Great for dense prediction
  (search, classification)          (segmentation, depth)
  Semantic understanding            Spatial + semantic features
  Needs captions                    Needs nothing but images
  Foundation for generation         Foundation for perception

Generative Vision: Diffusion Models

If CLIP is the understanding side of modern vision, diffusion models are the generation side. And they've completely transformed what's possible. Type "a cat astronaut riding a horse on Mars in the style of Van Gogh" and get a photorealistic (or artistic) image in seconds. That's diffusion.

The Core Idea

Diffusion models work by learning to reverse a noise-adding process. The forward process takes a clean image and gradually adds Gaussian noise over many steps until you're left with pure static — complete random noise. The reverse process trains a neural network (typically a U-Net or Transformer) to undo each noise step — to look at a noisy image and predict what it looked like one step less noisy. Once trained, generation is simple: start with pure random noise and iteratively denoise it. Each step removes a little noise, and after enough steps, a coherent image emerges.

  Diffusion Process
  ─────────────────

  Forward (training): Add noise step by step
  ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
  │ Clean  │ →  │ Slight │ →  │ More   │ →  │ Very   │ →  │ Pure   │
  │ image  │    │ noise  │    │ noise  │    │ noisy  │    │ noise  │
  │  🐱    │    │  🐱~   │    │  ~~?   │    │  ~~~~  │    │ static │
  └────────┘    └────────┘    └────────┘    └────────┘    └────────┘
    t=0           t=250         t=500         t=750         t=1000

  Reverse (generation): Remove noise step by step
  ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
  │ Pure   │ →  │ Vague  │ →  │ Shape  │ →  │ Almost │ →  │ Clean  │
  │ noise  │    │ blobs  │    │ forms  │    │ clear  │    │ image  │
  │ static │    │  ~~~~  │    │  ~🐱~  │    │  🐱~   │    │  🐱    │
  └────────┘    └────────┘    └────────┘    └────────┘    └────────┘
    t=1000        t=750         t=500         t=250          t=0

  Network learns: "Given this noisy image at step t, predict the noise"

Understanding the principles matters more than the math. The forward process adds noise. The reverse process learns to remove it. Conditioning on text guides what gets generated.

Latent Diffusion (Stable Diffusion)

Early diffusion models operated directly in pixel space — denoising 512×512×3 images. This works but it's painfully slow and memory-hungry. Latent Diffusion (the architecture behind Stable Diffusion) introduced a crucial optimization: work in a compressed latent space instead.

First, train a Variational Autoencoder (VAE) to compress images into a much smaller latent representation — typically 64×64×4 instead of 512×512×3. That's a ~48× compression. Then, run the entire diffusion process in this latent space. Finally, decode the denoised latent back to pixel space with the VAE decoder. Compressing to a latent space made diffusion practical — 64×64 latent instead of 512×512 pixels.

Classifier-Free Guidance

One of the most important practical controls in diffusion models is classifier-free guidance (CFG). During training, the model sometimes sees the text condition and sometimes doesn't (the condition is randomly dropped). At inference, you generate two predictions: one conditioned on the text prompt and one unconditional. The final prediction amplifies the difference between them:

output = unconditional + guidance_scale × (conditional - unconditional)

The guidance_scale controls the trade-off between quality and diversity. A scale of 1.0 means no guidance — you get diverse but often off-topic images. A scale of 7-12 (typical for Stable Diffusion) gives high-quality, prompt-faithful images. Push it to 20+ and you get oversaturated, artifact-heavy images that are too faithful to the prompt. Finding the right guidance scale is part art, part experimentation.

Practical Tip

Guidance scale of 7-8 is the sweet spot for most Stable Diffusion use cases. Below 5 and images drift from the prompt. Above 15 and they become oversaturated and artifact-heavy. When you see "CFG scale" in a diffusion UI, this is what it controls.

GANs: A Brief Retrospective

Before diffusion models took over, Generative Adversarial Networks (GANs) were the undisputed champions of image generation from 2014 to roughly 2020. The setup: a generator creates fake images, a discriminator tries to tell real from fake, and they train adversarially until the generator's fakes are indistinguishable from real images.

The highlights of the GAN era:

  • StyleGAN / StyleGAN2 (NVIDIA): Generated photorealistic human faces that don't exist. Introduced style-based generation with explicit control over coarse and fine features. thispersondoesnotexist.com ran on StyleGAN.
  • Pix2Pix: Paired image-to-image translation. Sketch → photo, satellite → map, day → night. Required paired training data.
  • CycleGAN: Unpaired image translation. Horse → zebra, summer → winter. No paired data needed — a breakthrough for domains where pairs don't exist.

GANs dominated 2014-2020. Diffusion models have largely replaced them for image generation. But GANs still win in specific niches:

  • Real-time generation: GANs generate in a single forward pass. Diffusion needs 20-50 steps.
  • Video super-resolution and enhancement: GAN discriminators produce perceptually sharper results.
  • Domain-specific tasks: When your domain is narrow (faces, medical images), a well-tuned GAN can still outperform generic diffusion models.

Multimodal LLMs: Language Models That Can See

The most exciting frontier in modern vision is multimodal LLMs — language models that can process and reason about images. GPT-4V, LLaVA, Gemini, Claude — these models can look at a photograph and have a conversation about it. They can answer questions, describe scenes, read text in images, interpret charts, spot anomalies, and even write code from screenshots.

The Architecture Pattern

The recipe is surprisingly consistent across all multimodal LLMs:

  1. Vision encoder — typically a CLIP-based ViT or SigLIP. Takes the image and produces a sequence of visual tokens (patch embeddings).
  2. Projection layer — a learned linear layer or small MLP that maps visual tokens into the LLM's embedding space. This is the bridge between vision and language.
  3. Large Language Model — a pretrained LLM (GPT-4, LLaMA, etc.) that processes the projected visual tokens alongside text tokens. From the LLM's perspective, image patches are just another kind of "word."

What Multimodal LLMs Unified

Image captioning, visual question answering (VQA), image-based reasoning, optical character recognition, document understanding, chart interpretation — these used to be separate tasks with separate architectures, separate datasets, and separate evaluation benchmarks. Each one had its own specialist model. Now they're all just prompting a multimodal LLM.

Want a caption? "Describe this image." Want VQA? "How many people are in this photo?" Want OCR? "Read the text in this receipt." Want reasoning? "What will happen next in this scene?" One model, one interface, infinite tasks. That's the power of unifying vision and language.

The LLaVA Recipe

LLaVA (Large Language and Vision Assistant) showed that you can build a competitive multimodal LLM with a surprisingly simple recipe: take a pretrained CLIP ViT, add a linear projection layer, connect it to a pretrained LLaMA, and fine-tune on visual instruction-following data. The whole thing can be trained on a single 8-GPU machine in about a day. This democratized multimodal AI research.

How the Pieces Connect

Modern vision isn't a collection of independent models — it's an ecosystem where the same components keep appearing in different configurations. Understanding the connections helps you see the big picture:

  Decision Guide:

CLIP is the connective tissue. Its text encoder powers Stable Diffusion's understanding of prompts. Its image encoder (or descendants like SigLIP) serves as the vision backbone for multimodal LLMs. Its contrastive training framework inspired countless follow-up models. If you understand CLIP, you understand the foundation of modern vision.

DINOv2 complements CLIP by excelling at spatial tasks. CLIP is trained on image-level alignment with text, so it's great for understanding what's in an image but less precise about where things are. DINOv2's self-supervised objective preserves fine-grained spatial information, making it better for segmentation, depth estimation, and other dense prediction tasks.

Diffusion models handle generation. They've replaced GANs as the default for text-to-image, image editing, inpainting, and style transfer. The combination of CLIP text understanding + latent diffusion + classifier-free guidance gives users unprecedented control over generated images.

Multimodal LLMs handle understanding. By connecting a vision encoder to a language model, they turn the "describe what you see" problem into a language modeling problem — and language models are very, very good at that.

When to Reach for What

The practical question isn't "how do these work internally?" — it's "which tool do I grab for my problem?" Here's the decision framework:

Task Reach For Why
Classify images, no labeled data CLIP zero-shot Text descriptions define your classes
Classify images, some labeled data CLIP/DINOv2 + linear probe or fine-tune Foundation model features + task-specific head
Search images by text query CLIP embeddings + vector DB Encode images and queries in same space
Find similar images CLIP or DINOv2 embeddings Nearest-neighbor in embedding space
Generate images from text Stable Diffusion / DALL-E / Flux Diffusion models dominate text-to-image
Edit or inpaint images Stable Diffusion (img2img, inpainting) Condition on both text and source image
Answer questions about an image Multimodal LLM (GPT-4V, Gemini) Natural language in, natural language out
Extract text from images (OCR) Multimodal LLM or dedicated OCR LLMs handle messy real-world text surprisingly well
Dense prediction (segmentation, depth) DINOv2 features + task head Best spatial features without labels
Real-time face generation StyleGAN Single forward pass, still the fastest
