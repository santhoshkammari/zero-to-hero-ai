CNN Interpretability & Visualization

Chapter 9: CNNs & Computer Vision 6 subtopics
TL;DR

Neural networks aren't black boxes — you just need the right flashlight. Filter visualization shows what low-level patterns each kernel detects. Activation maximization generates synthetic inputs that maximally excite a neuron. Grad-CAM paints a heatmap over the image showing where the network looked to make its decision. Saliency maps compute per-pixel importance via gradients. SHAP brings game-theoretic fairness to pixel attribution. Occlusion sensitivity brute-forces it: cover patches and watch the prediction wobble. Together, these tools turn "the model said cat" into "the model said cat because of these ear-shaped pixels."

You've built a CNN that hits 95% accuracy. Your manager asks: "Why does it classify this X-ray as pneumonia?" You shrug. That shrug is the interpretability problem, and it's not just academic — it's a regulatory, ethical, and debugging necessity. In medicine, a doctor needs to know whether the model looked at the lung opacity or the hospital's label stamp in the corner. In autonomous driving, an engineer needs to verify the model spotted the pedestrian, not the lamppost shadow. Interpretability tools don't just explain models — they catch bugs that accuracy metrics miss entirely.

This section walks through six families of techniques, each giving you a different lens into what's happening inside a CNN. We'll go from the simplest (literally looking at filter weights) to the most principled (SHAP values with theoretical guarantees). Every technique involves a tradeoff between computational cost, faithfulness to the actual model computation, and human readability. No single method is "best." Use them together.

Filter Visualization

Let's start with the simplest possible question: what does a single convolutional filter look like? A convolutional filter (or kernel) is just a small matrix of learned weights — typically 3×3 or 5×5. In the first layer, these weights operate directly on pixel values, so you can literally plot them as tiny images and see meaningful patterns.

First-layer filters in a trained CNN almost always learn edge detectors, color blobs, and simple textures. This isn't something we engineer — it emerges from the data. Horizontal edges, vertical edges, diagonal edges, color-opponent filters (red vs. green, blue vs. yellow). They look remarkably similar to the Gabor filters that neuroscientists found in the mammalian visual cortex decades ago. That convergence isn't a coincidence — both systems are optimized for natural images.

import torch
import matplotlib.pyplot as plt
from torchvision import models

# Load a pretrained ResNet and grab first-layer filters
model = models.resnet18(pretrained=True)
filters = model.conv1.weight.data.clone()  # shape: [64, 3, 7, 7]

# Plot the first 16 filters
fig, axes = plt.subplots(4, 4, figsize=(8, 8))
for i, ax in enumerate(axes.flat):
    # Normalize each filter to [0, 1] for display
    f = filters[i]
    f = (f - f.min()) / (f.max() - f.min())
    ax.imshow(f.permute(1, 2, 0).numpy())  # [H, W, C]
    ax.axis('off')
    ax.set_title(f'Filter {i}')
plt.suptitle('First-Layer Convolutional Filters')
plt.tight_layout()
plt.show()

For deeper layers, direct visualization stops being useful. A filter in layer 4 of a ResNet operates on 256-channel feature maps — you can't meaningfully display a 3×3×256 tensor as an image. Instead, you visualize what those filters respond to by examining feature maps (also called activation maps). Pass an image through the network, stop at a particular layer, and plot the output of each filter as a grayscale image. Bright regions indicate where that filter fired strongly. You'll see that deeper layers respond to increasingly complex patterns: textures in early layers, parts (eyes, wheels) in middle layers, and entire objects or scenes in late layers.

The Hierarchy Insight

This layer-by-layer progression from edges → textures → parts → objects is the fundamental insight of deep learning for vision. Each layer composes features from the previous layer. Filter and feature map visualization makes this hierarchy tangible and inspectable, not just theoretical.

Activation Maximization

Filter visualization shows you the weights. Activation maximization asks the inverse question: what input would make a particular neuron (or class output) maximally happy? Instead of optimizing the network's weights (training), you freeze the weights and optimize the input image. Start with random noise, compute the gradient of a target neuron's activation with respect to each pixel, and nudge the pixels in the direction that increases that activation. Repeat for hundreds of steps.

The result is a synthetic image that represents the neuron's "ideal stimulus" — the platonic form of what it has learned to detect. For a neuron in an early layer, you might get oriented stripes or checkerboard patterns. For a class output neuron (say "golden retriever"), you get a hallucinatory, fractal-like image that contains repeating golden retriever parts: fur textures, eye shapes, ear patterns, all woven together in a dreamlike way. Google's DeepDream was literally this technique applied at scale — it went viral in 2015 because the results were so eerily beautiful.

import torch
import torch.nn.functional as F
from torchvision import models, transforms

model = models.vgg16(pretrained=True).eval()

# Start from random noise image
input_img = torch.randn(1, 3, 224, 224, requires_grad=True)
optimizer = torch.optim.Adam([input_img], lr=0.05)

target_class = 207  # golden retriever in ImageNet

for step in range(300):
    optimizer.zero_grad()
    output = model(input_img)
    loss = -output[0, target_class]  # maximize this class score
    loss.backward()
    optimizer.step()

    # Optional: regularize to keep image natural-looking
    with torch.no_grad():
        input_img.data = torch.clamp(input_img.data, -2, 2)

Without regularization, activation maximization produces high-frequency noise that exploits the network's weights but looks meaningless to humans. Common regularizers include Gaussian blur every few steps, penalizing the L2 norm of the image, total variation loss (which encourages smoothness), and jittering (randomly shifting the image a few pixels before each forward pass). These regularizers act as a prior for "what natural images look like," and they dramatically improve the visual quality of the result.

The deeper insight here is conceptual: activation maximization reveals what the network has learned to consider "important." If a class neuron's maximized image contains checkerboard artifacts but no recognizable object parts, that's a red flag — the model might be relying on texture shortcuts rather than genuine shape understanding.

Grad-CAM and Grad-CAM++

Now for the most widely used interpretability tool in computer vision. Grad-CAM (Gradient-weighted Class Activation Mapping) answers the question: "Which spatial regions of the input image were most important for predicting class Y?" It produces a coarse heatmap that you overlay on the original image — bright regions are where the model "looked."

Here's how it works, step by step. Pick a target class (say "cat"). Do a forward pass. Compute the gradient of the cat class score with respect to the feature maps of the last convolutional layer. These gradients tell you how much each spatial location in each feature map matters for the "cat" prediction. Global-average-pool those gradients across the spatial dimensions to get one importance weight per feature map channel. Multiply each feature map by its importance weight, sum across channels, and apply ReLU (because we only care about features that positively influence the class). The result is a heatmap the size of the last conv layer's spatial output (typically 7×7 for ResNet), which you upsample to the original image size.

import torch
import torch.nn.functional as F
from torchvision import models, transforms
from PIL import Image
import numpy as np

model = models.resnet50(pretrained=True).eval()

# Hook to capture feature maps and gradients
feature_maps = {}
gradients = {}

def forward_hook(module, input, output):
    feature_maps['value'] = output

def backward_hook(module, grad_input, grad_output):
    gradients['value'] = grad_output[0]

# Register hooks on the last conv layer
target_layer = model.layer4[-1].conv3
target_layer.register_forward_hook(forward_hook)
target_layer.register_full_backward_hook(backward_hook)

# Preprocess image and forward pass
img = Image.open('cat.jpg').resize((224, 224))
tensor = transforms.ToTensor()(img).unsqueeze(0)
output = model(tensor)
target_class = output.argmax(dim=1).item()

# Backward pass for target class
model.zero_grad()
output[0, target_class].backward()

# Compute Grad-CAM
weights = gradients['value'].mean(dim=[2, 3], keepdim=True)  # GAP
cam = (weights * feature_maps['value']).sum(dim=1, keepdim=True)
cam = F.relu(cam)  # only positive influence
cam = F.interpolate(cam, size=(224, 224), mode='bilinear')
cam = cam.squeeze().detach().numpy()
cam = (cam - cam.min()) / (cam.max() - cam.min())  # normalize to [0, 1]

Grad-CAM has a limitation: it uses global average pooling of gradients, which treats every spatial location within a feature map equally when computing the channel importance weight. This means it can miss situations where only a small region of a feature map is critical. Grad-CAM++ addresses this by using a weighted combination of the positive partial derivatives, effectively computing a pixel-wise importance weight rather than a single scalar per channel. The result is better localization, especially when multiple instances of the same class appear in one image (two cats, for example). Grad-CAM++ highlights both; vanilla Grad-CAM might blur them together.

Faithfulness vs. Plausibility

A Grad-CAM heatmap can look convincing while being unfaithful to the model's actual reasoning. Research has shown that different interpretability methods sometimes disagree on what's important. Always cross-validate with multiple methods. If Grad-CAM says the model looks at the cat's ears but occlusion sensitivity says the background matters, you have a problem worth investigating.

Saliency Maps

Saliency maps are the simplest gradient-based attribution method. The idea: compute the gradient of the class score with respect to each input pixel. Pixels where a tiny change causes a large change in the output are "salient" — the model is sensitive to them. Take the absolute value of those gradients (we care about magnitude, not direction), and you have a per-pixel importance map.

The math is straightforward. For a model f and an input image x, the saliency map for class c is simply |∂f_c / ∂x|. For a color image with 3 channels, you typically take the maximum gradient magnitude across the RGB channels for each pixel, giving you a single-channel heatmap.

import torch
from torchvision import models, transforms
from PIL import Image

model = models.resnet50(pretrained=True).eval()

img = Image.open('cat.jpg').resize((224, 224))
tensor = transforms.ToTensor()(img).unsqueeze(0)
tensor.requires_grad_(True)

output = model(tensor)
target_class = output.argmax(dim=1).item()
output[0, target_class].backward()

# Saliency = max absolute gradient across color channels
saliency = tensor.grad.abs().squeeze().max(dim=0).values
saliency = saliency.detach().numpy()

Vanilla saliency maps tend to be noisy — they look like salt-and-pepper static with vague highlights. Several improved variants exist. SmoothGrad adds small amounts of Gaussian noise to the input multiple times, computes the saliency map for each noisy copy, and averages them. The noise washes out the random high-frequency gradient fluctuations while preserving the consistent signal. Typically 20–50 noisy samples are enough. Integrated Gradients takes a more principled approach: it computes the integral of gradients along a straight-line path from a baseline (usually a black image) to the actual input. This satisfies two desirable axioms — sensitivity (if a feature matters, it gets nonzero attribution) and implementation invariance (two functionally identical networks produce the same attributions). Integrated Gradients is often considered the "gold standard" among gradient-based methods for its theoretical properties.

The tradeoff: vanilla saliency is nearly free (one backward pass), SmoothGrad costs 20–50× more, and Integrated Gradients costs 50–300× more (depending on the number of interpolation steps along the path). For a quick debugging check, vanilla saliency is fine. For a publication or regulatory audit, use Integrated Gradients.

SHAP for Images

SHAP (SHapley Additive exPlanations) brings a rigorous, game-theoretic framework to the attribution problem. The core idea comes from cooperative game theory: each pixel (or superpixel) is a "player," the model's prediction is the "payout," and we want to fairly distribute the payout among the players based on their contribution. The Shapley value is the unique attribution method that satisfies four fairness axioms: efficiency (attributions sum to the total prediction), symmetry (identical contributors get equal credit), dummy (irrelevant features get zero), and linearity (attributions for combined models add up).

Computing exact Shapley values is intractable — you'd need to evaluate all 2ⁿ subsets of features, where n is the number of pixels. For a 224×224 image, that's 2^50176 evaluations. So in practice, SHAP for images works with superpixels: group adjacent pixels into ~50–200 regions using a segmentation algorithm like SLIC, then compute Shapley values over those regions. Each evaluation masks out a subset of superpixels (replacing them with a baseline like gray or blurred) and measures how the prediction changes.

import shap
import numpy as np
from torchvision import models, transforms
from PIL import Image

model = models.resnet50(pretrained=True).eval()

def predict(images):
    """Wrapper for SHAP: takes numpy array, returns predictions."""
    batch = torch.stack([transforms.ToTensor()(img) for img in images])
    with torch.no_grad():
        output = torch.softmax(model(batch), dim=1)
    return output.numpy()

img = np.array(Image.open('cat.jpg').resize((224, 224)))

# Use KernelSHAP with superpixel masking
masker = shap.maskers.Image("inpaint_telea", img.shape)
explainer = shap.Explainer(predict, masker, output_names=["cat", "dog"])
shap_values = explainer(np.expand_dims(img, axis=0), max_evals=500)

# Plot: red = positive contribution, blue = negative
shap.image_plot(shap_values)

The key advantage of SHAP over gradient methods: it measures the actual effect of removing features, not just the local sensitivity. Gradients tell you "if I wiggle this pixel infinitesimally, how does the output change?" SHAP tells you "if this region were absent, how much worse would the prediction be?" These can disagree. A pixel might have a zero gradient (the model is at a local plateau with respect to it) but a large SHAP value (removing the entire region collapses the prediction).

The downside is cost. Even with superpixels, KernelSHAP for images requires hundreds to thousands of model evaluations per image. For real-time applications, Grad-CAM takes milliseconds; SHAP might take minutes. Use SHAP when you need rigorous, auditable explanations — medical imaging, legal compliance, high-stakes debugging — and gradient methods for quick iteration.

DeepSHAP: The Fast Approximation

DeepSHAP combines SHAP values with DeepLIFT's backpropagation rules to approximate Shapley values in a single forward+backward pass. It's orders of magnitude faster than KernelSHAP but sacrifices some theoretical guarantees. For CNNs, it's often a good middle ground between speed and rigor.

Occlusion Sensitivity

Occlusion sensitivity is the brute-force approach to interpretability, and sometimes brute force is exactly what you want. The idea: slide a gray patch (say 32×32 pixels) across the entire image, one position at a time. At each position, run the model and record how the target class probability changes. If covering a particular region causes the probability to drop sharply, that region was important for the prediction.

The output is a heatmap the same size as the image, where each value represents the prediction drop when that location is occluded. Dark spots (big drops) mark critical regions. Bright spots (no change) mark irrelevant areas.

import torch
import numpy as np
from torchvision import models, transforms
from PIL import Image

model = models.resnet50(pretrained=True).eval()
preprocess = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])

img = Image.open('cat.jpg').resize((224, 224))
tensor = preprocess(img).unsqueeze(0)

with torch.no_grad():
    baseline_prob = torch.softmax(model(tensor), dim=1)
    target_class = baseline_prob.argmax().item()
    baseline_score = baseline_prob[0, target_class].item()

# Slide a gray patch across the image
patch_size = 32
stride = 8
heatmap = np.zeros((224, 224))
count = np.zeros((224, 224))

for y in range(0, 224 - patch_size + 1, stride):
    for x in range(0, 224 - patch_size + 1, stride):
        occluded = tensor.clone()
        occluded[0, :, y:y+patch_size, x:x+patch_size] = 0  # gray patch
        prob = torch.softmax(model(occluded), dim=1)[0, target_class].item()
        drop = baseline_score - prob
        heatmap[y:y+patch_size, x:x+patch_size] += drop
        count[y:y+patch_size, x:x+patch_size] += 1

heatmap /= np.maximum(count, 1)  # average overlapping regions

Occlusion sensitivity has two major strengths. First, it's completely model-agnostic — it doesn't require access to gradients, internal activations, or any knowledge of the architecture. It works on any model that takes an image and returns a probability. Second, it measures true causal importance: "when I remove this, does the prediction change?" Gradient methods measure sensitivity (infinitesimal perturbations), not actual removal.

The weakness is speed. For a 224×224 image with a 32×32 patch and stride 8, you need about (224/8)² ≈ 784 forward passes per image. That's workable for individual explanations but impractical for explaining an entire dataset. You can trade off resolution for speed by using larger strides or larger patches, but you lose spatial precision.

Patch size matters. A very small patch (4×4) gives high resolution but may not be large enough to disrupt the model — the surrounding context compensates for the missing information. A very large patch (64×64) reliably disrupts the model but gives coarse, blobby heatmaps. A common strategy: start with a coarse sweep (large patch, large stride) to find the important region, then do a fine sweep in that region only.

Putting It All Together: A Debugging Workflow

In practice, interpretability isn't about picking one technique. It's about triangulating with several. Here's a workflow that experienced practitioners use:

Start with Grad-CAM — it's fast and gives you a quick spatial overview of where the model is looking. If the heatmap highlights the correct object, you have initial confidence. If it highlights the background or an artifact, you've found a likely bug immediately.

Next, run saliency maps (preferably Integrated Gradients) to get pixel-level detail. Grad-CAM is coarse (7×7 upsampled); saliency maps show fine-grained attribution. If saliency highlights edges and textures within the object that Grad-CAM already identified, the evidence is converging.

For high-stakes decisions, run SHAP or occlusion sensitivity. These provide causal evidence — they measure what happens when features are actually removed, not just what the gradient says locally. If all four methods agree that the model is using the right region and the right features, you have strong evidence that the model is reasoning correctly.

If the methods disagree, that disagreement itself is informative. It might indicate that the model's decision boundary is complex in that region, or that the model is relying on subtle correlations that different methods expose differently. Disagreement means "investigate further," not "pick the answer you like."

Interpretability ≠ Explanation

These tools show you where the model looked, not why in a human-reasoning sense. A Grad-CAM heatmap on an X-ray might highlight a region near the lung — but you still need a radiologist to confirm whether the highlighted pattern corresponds to a medically meaningful finding. Interpretability assists human judgment; it does not replace it.
