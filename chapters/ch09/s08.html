Interpretability & Visualization

Chapter 9: CNNs & Computer Vision Grad-CAM · Saliency maps · What the network sees
TL;DR

Your CNN makes a prediction — but what did it look at? Interpretability tools answer this by producing heatmaps, saliency maps, and attention visualizations that highlight which regions of the input drove the output. Grad-CAM is the go-to: fast, class-specific, and works on any CNN without retraining. Saliency maps show pixel-level sensitivity. Occlusion sensitivity is simple but slow. For Vision Transformers, you can visualize attention maps directly. Use these tools to debug spurious correlations, explain predictions to stakeholders, and catch data leakage before it bites you in production.

Your model classified this X-ray as "pneumonia." The doctor asks: "What did it look at?" If you can't answer that, your model doesn't ship. Interpretability tools show you where the network is focusing — and whether it's cheating.

This isn't about philosophy or abstract notions of "explainability." It's about practical debugging. A model that achieves 99% accuracy by reading a hospital watermark in the corner is worse than useless — it's dangerous. The tools in this section let you catch that problem before deployment.

Grad-CAM: The Go-To

Gradient-weighted Class Activation Mapping (Grad-CAM) is the most widely used CNN interpretability method. It produces a coarse heatmap highlighting which spatial regions of the final convolutional layer were most important for a specific class prediction. Here's the recipe:

  1. Forward pass — run the image through the network, record the feature maps from the last conv layer.
  2. Backward pass — compute the gradient of the target class score with respect to those feature maps.
  3. Weight the channels — global average pool each gradient map to get one importance weight per channel.
  4. Weighted combination — multiply each feature map by its weight, sum across channels, apply ReLU.
  5. Upscale — resize the heatmap to the input image size and overlay it.
import torch
import torch.nn.functional as F
from torchvision import models, transforms
from PIL import Image

# Load a pretrained model
model = models.resnet50(pretrained=True).eval()

# Hook into the last conv layer
activations, gradients = {}, {}

def save_activation(module, input, output):
    activations['value'] = output.detach()

def save_gradient(module, grad_input, grad_output):
    gradients['value'] = grad_output[0].detach()

target_layer = model.layer4[-1]
target_layer.register_forward_hook(save_activation)
target_layer.register_full_backward_hook(save_gradient)

# Preprocess the image
img = Image.open("xray.jpg").resize((224, 224))
tensor = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406],
                         [0.229, 0.224, 0.225])
])(img).unsqueeze(0)

# Forward + backward for target class
output = model(tensor)
target_class = output.argmax(dim=1).item()
model.zero_grad()
output[0, target_class].backward()

# Compute Grad-CAM
weights = gradients['value'].mean(dim=[2, 3], keepdim=True)  # global avg pool
cam = (weights * activations['value']).sum(dim=1, keepdim=True)
cam = F.relu(cam)                          # only positive contributions
cam = F.interpolate(cam, size=(224, 224),
                    mode='bilinear', align_corners=False)
cam = cam.squeeze().numpy()
cam = (cam - cam.min()) / (cam.max() - cam.min() + 1e-8)  # normalize to [0, 1]

The resulting heatmap tells you exactly which region drove the classification. If the model classified a chest X-ray as pneumonia and the heatmap highlights the lung opacities — great, it's working. If it highlights the text label in the corner — you have a data leakage problem.

Why the Last Conv Layer?

Earlier layers detect low-level features (edges, textures) that aren't class-specific. The last conv layer has the best balance: features are still spatially localized but semantically rich enough to relate to specific classes. You can apply Grad-CAM to earlier layers — the heatmaps will be higher resolution but less class-discriminative.

Grad-CAM++ is a refinement that uses second-order gradients for better localization when multiple instances of the same class appear in the image. In practice, vanilla Grad-CAM works fine for most debugging tasks.

Saliency Maps

Saliency maps are the simplest gradient-based visualization. Compute the gradient of the output class score with respect to the input pixels. That's it.

# Saliency map — gradient of class score w.r.t. input
tensor.requires_grad_(True)
output = model(tensor)
score = output[0, target_class]
score.backward()

saliency = tensor.grad.data.abs().squeeze()  # shape: [3, 224, 224]
saliency, _ = saliency.max(dim=0)            # collapse color channels
saliency = (saliency - saliency.min()) / (saliency.max() - saliency.min())

The question a saliency map answers: "Which pixels, if changed by a tiny amount, would most affect the prediction?" High-gradient pixels are the ones the model is most sensitive to.

The catch: raw saliency maps are noisy. They highlight individual pixels rather than coherent regions, making them harder to interpret visually than Grad-CAM. Two common improvements:

  • SmoothGrad — add random noise to the input many times, compute the saliency map each time, and average. This smooths out the noise dramatically.
  • Integrated Gradients — interpolate between a blank baseline image and the actual input in small steps, compute gradients at each step, and average. This gives a theoretically grounded attribution that satisfies axioms like sensitivity and implementation invariance.

Attention Visualization (for Vision Transformers)

Vision Transformers (ViTs) split images into patches and process them with self-attention. This gives you interpretability almost for free — just visualize the attention maps.

# Pseudo-code for ViT attention extraction
# Most ViT implementations let you return attention weights

with torch.no_grad():
    output = vit_model(tensor, output_attentions=True)
    # attentions: list of [batch, heads, patches, patches]
    # one entry per transformer layer

# Visualize attention from [CLS] token to all patches
# in the last layer, averaged across heads
attn = output.attentions[-1]          # last layer
cls_attn = attn[0, :, 0, 1:]         # [heads, num_patches] — CLS to others
cls_attn = cls_attn.mean(dim=0)      # average across heads
patch_grid = int(cls_attn.shape[0] ** 0.5)
attn_map = cls_attn.reshape(patch_grid, patch_grid)

Each attention head focuses on different things. Some look at edges, some at textures, some at specific objects. Visualizing individual heads often reveals surprisingly interpretable specialization — one head might consistently attend to eyes in face images, another to backgrounds.

Attention ≠ Attribution

Attention maps show where the model looks, not necessarily what causes the prediction. A head might attend strongly to a region but downstream layers might ignore it. For rigorous attribution in ViTs, use methods like Attention Rollout (multiply attention matrices across layers) or apply Grad-CAM to the ViT's intermediate representations.

Occlusion Sensitivity

The most intuitive method. No gradients, no hooks — just brute force.

  1. Take the original image. Record the model's prediction confidence.
  2. Slide a grey (or black) patch across the image, one position at a time.
  3. At each position, record how the prediction confidence changes.
  4. Where confidence drops most → that region matters most.
import numpy as np

def occlusion_sensitivity(model, img_tensor, target_class,
                          patch_size=32, stride=8):
    """Slide a grey patch and record confidence drop."""
    _, _, H, W = img_tensor.shape
    baseline_score = model(img_tensor).softmax(1)[0, target_class].item()

    heatmap = np.zeros((H, W))
    count = np.zeros((H, W))

    for y in range(0, H - patch_size + 1, stride):
        for x in range(0, W - patch_size + 1, stride):
            occluded = img_tensor.clone()
            occluded[:, :, y:y+patch_size, x:x+patch_size] = 0.5  # grey

            score = model(occluded).softmax(1)[0, target_class].item()
            drop = baseline_score - score  # positive = region matters

            heatmap[y:y+patch_size, x:x+patch_size] += drop
            count[y:y+patch_size, x:x+patch_size] += 1

    heatmap /= np.maximum(count, 1)
    return heatmap

It's dead simple and model-agnostic — works on any architecture without modification. The downside is speed: for a 224×224 image with a 32×32 patch and stride 8, you're running ~576 forward passes. For a quick sanity check on a few images it's fine; for large-scale analysis, use Grad-CAM.

Feature Map Visualization

Sometimes you want to see what individual convolutional filters have learned. Hook into any layer and visualize the activations directly:

# Visualize feature maps from a specific layer
activation = {}

def hook_fn(module, input, output):
    activation['maps'] = output.detach()

handle = model.layer2[0].conv1.register_forward_hook(hook_fn)
_ = model(tensor)
handle.remove()

# activation['maps'] shape: [1, num_channels, H, W]
# Plot the first 16 channels as a grid
maps = activation['maps'][0, :16]  # [16, H, W]

Early layers show edge detectors and color blobs. Middle layers show texture patterns and parts. Deep layers show abstract, class-specific features that are harder to interpret visually but map closely to semantic concepts.

Method Comparison

Method Speed Resolution What It Shows Use When
Grad-CAM Fast (1 forward + 1 backward) Coarse (conv layer grid) Which regions matter for a class Default choice — first thing to try
Saliency Maps Fast (1 backward pass) Pixel-level (but noisy) Per-pixel sensitivity Quick sanity check on input sensitivity
SmoothGrad Medium (N backward passes) Pixel-level (smoother) Denoised pixel attribution When raw saliency is too noisy to read
Integrated Gradients Medium (N forward+backward) Pixel-level Theoretically grounded attribution When you need formal guarantees
Occlusion Sensitivity Slow (hundreds of forwards) Patch-level Which regions are necessary Model-agnostic debugging, small datasets
Attention Maps Free (already computed) Patch-level (ViTs) Where the model attends ViTs — quick visual inspection
Feature Maps Fast (1 forward pass) Layer-dependent What individual filters detect Understanding learned representations

The Debugging Workflow

These tools aren't just for pretty pictures in papers. They're debugging instruments. Here's when you reach for them:

Scenario 1: Model Predicts Wrong

Run Grad-CAM on the misclassified image. If the heatmap highlights the background instead of the object, your model learned a shortcut. The labels are leaking context — maybe all "outdoor cat" images have grass and all "indoor cat" images have sofas, and the model learned "grass vs. sofa" instead of "cat."

Fix: diversify your backgrounds, or use data augmentation (random crops, background replacement) to break the spurious correlation.

Scenario 2: Suspiciously High Accuracy

Your model hits 99.5% on a medical dataset. That's almost certainly too good. Run Grad-CAM on a batch of correct predictions. If the heatmaps cluster on watermarks, patient ID text, or scanner artifacts instead of anatomical features — you've found data leakage.

Fix: strip metadata, crop margins, and check whether accuracy holds on images from a different hospital or scanner.

Scenario 3: Model Fails on One Class

Accuracy is great overall, but one class consistently underperforms. Visualize feature maps for that class vs. well-performing classes. Maybe the failing class has high intra-class variance (many different appearances) or the model is confusing it with a visually similar class.

Fix: add class-specific augmentations, collect more diverse examples, or add hard-negative mining.

The Clever Hans Problem

Named after a horse that appeared to do arithmetic but was actually reading its trainer's body language. CNNs do the same thing. A famous example: a model trained to detect wolves vs. huskies learned to detect snow in the background — wolves were photographed in snow, huskies on grass. The model never learned what a wolf looks like. Grad-CAM would have caught this immediately.

Practical Tips

  • Always visualize before deploying. Run Grad-CAM on at least 50–100 images per class before calling a model "done." It takes 10 minutes and catches problems that metrics miss.
  • Use the right baseline. Integrated Gradients requires a baseline (usually a black image). A bad baseline gives meaningless attributions. For medical images, consider using the dataset mean as the baseline.
  • Libraries save time. Don't reimplement these from scratch in production. Use pytorch-grad-cam, captum (by Meta), or tf-explain (TensorFlow). They handle edge cases and support dozens of methods.
  • Multiple methods, same image. If Grad-CAM and occlusion sensitivity both highlight the same region, you can be much more confident that region truly matters.
  • Quantitative evaluation. For rigorous work, use insertion/deletion metrics — progressively reveal (or remove) the most important pixels and measure how quickly the prediction changes.

What You Should Now Be Able To Do

  • Generate a Grad-CAM heatmap — hook into the last conv layer, compute gradient-weighted feature maps, and overlay the result on the input image to see what drove a classification.
  • Compute a saliency map — backpropagate to the input pixels and visualize per-pixel sensitivity, using SmoothGrad or Integrated Gradients when raw gradients are too noisy.
  • Visualize ViT attention — extract attention weights from transformer layers and display which image patches the [CLS] token attends to.
  • Debug with occlusion sensitivity — systematically occlude image regions and map confidence drops to identify critical areas, especially when working with non-differentiable or black-box models.
  • Run a 3-step diagnostic — when a model misbehaves, use Grad-CAM to check for shortcut learning, spurious features, or class-specific failure modes before touching the architecture.
