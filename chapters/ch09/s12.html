Video & Temporal Vision

Chapter 9: CNNs & Computer Vision 12 subtopics
TL;DR

Images are frozen moments. Video adds the dimension of time, and understanding motion requires architectures that can reason across frames. 3D convolutions extend spatial filters into the temporal axis — C3D and I3D slide 3×3×3 kernels through space-time volumes. Two-stream networks split the problem: one CNN reads RGB frames for appearance, another reads optical flow for motion, and their outputs fuse. Video Transformers (TimeSformer, ViViT) tokenize video into space-time patches and let self-attention learn which frames and regions matter. Each approach trades compute for temporal understanding differently.

A single image can tell you "there's a person with their arm raised." Only video can tell you whether they're waving hello, throwing a ball, or hailing a taxi. The temporal dimension — how things change across frames — carries information that no single frame contains. Action recognition, anomaly detection in surveillance, sports analytics, sign language understanding, surgical video analysis: all require models that see through time, not just at a frozen instant.

The fundamental challenge is scale. A 10-second clip at 30 fps is 300 frames. Each frame at 224×224 resolution is 150,528 pixels. That's 45 million pixel values per clip — and that's a short, low-resolution video. Naive approaches (just flatten everything into one big tensor) blow up memory and compute budgets immediately. Every architecture in this section is, at its core, a strategy for efficiently extracting temporal information without drowning in data.

3D Convolutions: C3D and I3D

Regular 2D convolutions slide a filter across height and width. 3D convolutions add a third axis: they slide across height, width, and time simultaneously. A 3×3×3 kernel looks at a 3-pixel-tall, 3-pixel-wide, 3-frame-deep cube of the video. This means the filter can detect spatiotemporal patterns — things that have a particular shape and a particular motion direction, captured in a single learned filter.

Think about detecting a "waving hand." In any single frame, a hand could be in any position. But across three consecutive frames, a waving hand creates a specific spatiotemporal pattern: the hand shape moves left-right-left. A 3D filter can learn to detect exactly this kind of space-time pattern, just as a 2D filter learns to detect an edge or a texture.

C3D: The Pioneer

C3D (Convolutional 3D, 2015) was the first architecture to demonstrate that straightforward 3D convolutions could learn useful video features. The architecture is simple: stack 3D convolutional layers with 3×3×3 kernels, interspersed with 3D max pooling, and end with fully connected layers. The input is a clip of 16 consecutive RGB frames, each resized to 112×112 pixels. The key finding was that the intermediate features (before the classification head) served as surprisingly good general-purpose video descriptors — you could train C3D on one dataset and use its features for a completely different video task.

import torch
import torch.nn as nn

class SimpleC3D(nn.Module):
    """Simplified C3D-style block to illustrate 3D convolutions."""
    def __init__(self, num_classes=101):
        super().__init__()
        self.features = nn.Sequential(
            # Input: [B, 3, 16, 112, 112] = [batch, channels, time, H, W]
            nn.Conv3d(3, 64, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool3d(kernel_size=(1, 2, 2), stride=(1, 2, 2)),
            # Pool spatially but not temporally yet → [B, 64, 16, 56, 56]

            nn.Conv3d(64, 128, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool3d(kernel_size=2, stride=2),
            # Pool all dims → [B, 128, 8, 28, 28]

            nn.Conv3d(128, 256, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool3d(kernel_size=2, stride=2),
            # → [B, 256, 4, 14, 14]
        )
        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool3d(1),
            nn.Flatten(),
            nn.Linear(256, num_classes)
        )

    def forward(self, x):
        return self.classifier(self.features(x))

# 16-frame clip, 112×112 pixels
clip = torch.randn(2, 3, 16, 112, 112)
model = SimpleC3D(num_classes=101)
output = model(clip)  # [2, 101]

C3D's limitation: 3D convolutions are computationally expensive. A 3×3×3 kernel has 27 weight parameters per input-output channel pair, compared to 9 for a 2D 3×3 kernel. Every layer costs roughly 3× more compute. And the temporal receptive field grows slowly — even with 8 conv layers, you might only "see" 16–32 frames, which is less than a second of video.

I3D: Inflating 2D Knowledge into 3D

I3D (Inflated 3D ConvNets, 2017) solved the initialization problem brilliantly. Training 3D CNNs from scratch on video datasets is hard because video datasets are small compared to ImageNet. I3D's insight: take a 2D CNN pretrained on ImageNet (like Inception-V1), "inflate" all its 2D kernels into 3D by repeating the weights along the temporal dimension, and then fine-tune on video data. A pretrained 3×3 filter becomes a 3×3×3 filter by copying the same 3×3 weights across 3 time steps and dividing by 3 (to preserve the activation scale). This way, the inflated network starts with all the spatial knowledge from ImageNet and only needs to learn the temporal patterns.

I3D also showed that training on the Kinetics dataset (a large-scale video dataset with 400+ action classes) produced features that transferred well to other video tasks, establishing Kinetics as the ImageNet of video understanding. The two-stream version of I3D — one stream for RGB, one for optical flow — set new state-of-the-art results that stood for several years.

(2+1)D: A Practical Compromise

Factorized 3D convolutions, like R(2+1)D, split a 3×3×3 kernel into a spatial 1×3×3 convolution followed by a temporal 3×1×1 convolution. This has the same representational capacity but fewer parameters and easier optimization. It also adds a ReLU between the two steps, effectively doubling the nonlinearities. In practice, (2+1)D often outperforms full 3D while being cheaper to train.

Two-Stream Networks

The human visual system processes "what" and "where/how" through separate neural pathways — the ventral and dorsal streams. Two-stream networks (2014) took biological inspiration and applied it to video: one stream processes static appearance from individual RGB frames, and a second stream processes motion information from optical flow.

Optical flow is a 2D vector field that describes the pixel-level displacement between consecutive frames. For every pixel in frame t, optical flow tells you where that pixel moved to in frame t+1, as a (dx, dy) vector. Regions with large flow vectors are moving fast; regions with zero flow are static. Optical flow explicitly encodes motion — something that CNNs on raw RGB frames have to learn implicitly (and often struggle with).

The architecture is straightforward. The spatial stream is a standard 2D CNN (typically a ResNet or BN-Inception) that takes a single RGB frame as input and predicts the action. The temporal stream is the same architecture but takes a stack of L consecutive optical flow frames (typically L=10, meaning 10 pairs of dx/dy maps, giving a 20-channel input). Each stream produces a class prediction, and the final prediction is a weighted average (or learned fusion) of both.

# Pseudocode for two-stream architecture
class TwoStreamNetwork:
    def __init__(self):
        self.spatial_cnn = ResNet50(input_channels=3)   # RGB frame
        self.temporal_cnn = ResNet50(input_channels=20)  # 10 flow frames × 2 (dx, dy)
        self.fusion_weight = 0.5  # or learn this

    def predict(self, rgb_frame, flow_stack):
        spatial_score = self.spatial_cnn(rgb_frame)      # appearance
        temporal_score = self.temporal_cnn(flow_stack)    # motion
        return self.fusion_weight * spatial_score + (1 - self.fusion_weight) * temporal_score

The temporal stream alone achieves surprisingly strong results, demonstrating that motion is often more discriminative than appearance for actions. "Running" vs. "walking" have similar appearance but very different motion magnitudes. "Juggling" vs. "catching" involve the same objects but different motion patterns.

The main drawback: optical flow must be precomputed, and computing it is expensive. Traditional methods like TV-L1 take about 0.06 seconds per frame pair on a GPU. For a dataset with millions of frames, precomputation takes days and the stored flow fields consume massive disk space. This is why 3D CNNs eventually overtook two-stream networks — they learn temporal features end-to-end from raw RGB without needing precomputed flow. However, recent work like FlowNet and RAFT made optical flow computation much faster, and flow remains useful as an auxiliary training signal even when not used at inference time.

Video Transformers: TimeSformer and ViViT

Transformers revolutionized NLP. Vision Transformers (ViT) proved they work for images. The natural next question: can self-attention replace convolutions for video too? Video Transformers extend ViT to the temporal dimension, and the results are compelling — but the engineering challenges are severe because self-attention's cost is quadratic in the number of tokens, and video has a lot of tokens.

The Token Explosion Problem

A ViT for images splits a 224×224 image into 16×16 patches, giving 196 tokens. Self-attention cost is O(n²), so that's manageable. Now extend to video: 8 frames × 196 patches = 1,568 tokens. The attention matrix is 1,568 × 1,568 — about 2.5 million entries. Go to 32 frames? That's 6,272 tokens and a 39-million-entry attention matrix. Standard self-attention simply doesn't scale to raw video. Every video transformer architecture is fundamentally a strategy for making attention tractable over space-time volumes.

TimeSformer

TimeSformer (2021, Facebook AI) solves the scalability problem with divided space-time attention. Instead of computing attention over all space-time tokens simultaneously (which is O((T×N)²) where T is frames and N is patches per frame), TimeSformer splits each attention layer into two sequential steps:

First, temporal attention: each spatial patch attends only to the same spatial position across all frames. Patch (3,5) in frame 1 attends to patch (3,5) in frames 2, 3, …, T. This captures how a specific location changes over time. Cost: O(T²) per patch, applied N times.

Second, spatial attention: within each frame, all patches attend to each other, exactly like standard ViT. This captures spatial relationships within a single moment. Cost: O(N²) per frame, applied T times.

Total cost: O(T²×N + N²×T) instead of O((T×N)²). For T=8, N=196: divided attention costs about 340K operations vs. 2.5M for joint attention — roughly an 8× savings. The savings grow dramatically with more frames.

# Conceptual TimeSformer attention block
class DividedSpaceTimeAttention(nn.Module):
    def __init__(self, dim, num_heads):
        super().__init__()
        self.temporal_attn = nn.MultiheadAttention(dim, num_heads)
        self.spatial_attn = nn.MultiheadAttention(dim, num_heads)

    def forward(self, x, T, N):
        # x shape: [B, T*N, dim]
        B, _, D = x.shape

        # Temporal attention: each spatial position across frames
        x_t = x.reshape(B, T, N, D)
        x_t = x_t.permute(0, 2, 1, 3).reshape(B * N, T, D)
        x_t, _ = self.temporal_attn(x_t, x_t, x_t)
        x_t = x_t.reshape(B, N, T, D).permute(0, 2, 1, 3).reshape(B, T * N, D)
        x = x + x_t  # residual

        # Spatial attention: all patches within each frame
        x_s = x.reshape(B, T, N, D).reshape(B * T, N, D)
        x_s, _ = self.spatial_attn(x_s, x_s, x_s)
        x_s = x_s.reshape(B, T * N, D)
        x = x + x_s  # residual

        return x

ViViT: Four Flavors of Video Vision Transformer

ViViT (2021, Google) systematically explored four different strategies for handling video in transformers, forming a spectrum from most powerful to most efficient:

Model 1 — Spatio-temporal attention: Full joint attention over all space-time tokens. The most expressive but the most expensive. Practical only for short clips or small spatial resolutions.

Model 2 — Factorized encoder: First, a spatial transformer encodes each frame independently (producing one token per frame). Then, a temporal transformer processes the sequence of frame-level tokens. This is efficient because the temporal transformer operates on T tokens instead of T×N.

Model 3 — Factorized self-attention: Similar to TimeSformer's divided attention. Within each transformer layer, spatial attention and temporal attention are applied sequentially. Each layer processes both dimensions but never jointly.

Model 4 — Factorized dot-product attention: The most aggressive factorization. The attention computation itself is factored into separate spatial and temporal components, computed independently and combined. This is the cheapest but loses the most interaction between dimensions.

The paper found that Models 2 and 3 provided the best accuracy-efficiency tradeoff. Model 1 was impractical at scale. Model 4 was fast but sacrificed too much accuracy. In practice, most follow-up work adopted factorized approaches (similar to Models 2 and 3).

The Compute Reality Check

Video transformers achieve state-of-the-art accuracy but demand enormous compute. Training TimeSformer on Kinetics-400 takes ~1,500 GPU-hours on V100s. In production, processing a single 10-second clip can take 100–500ms. For real-time applications (30 fps), you still need efficient CNNs or lightweight transformer variants. Video transformers dominate benchmarks; efficient CNNs still dominate deployment.

Optical Flow and Motion Estimation

We mentioned optical flow as an input for two-stream networks, but it deserves its own discussion because it underpins much of temporal vision. Optical flow estimates per-pixel motion between consecutive frames. Classical methods like Lucas-Kanade and Horn-Schunck use variational optimization. Modern deep learning approaches treat it as a supervised regression problem.

FlowNet (2015) showed that a CNN can predict optical flow from a pair of frames end-to-end. RAFT (2020) is the current standard — it builds correlation volumes between frames and iteratively refines flow estimates using a GRU-based update operator. RAFT is accurate, fast enough for practical use, and generalizes well across datasets. Its key insight is that iterative refinement (start with a coarse estimate, gradually improve it) works better than direct one-shot prediction.

Optical flow is used beyond two-stream classification. Video interpolation (generating intermediate frames) relies on flow to warp pixels from known frames. Video stabilization estimates global flow and compensates for camera shake. Video object segmentation uses flow to propagate masks across frames. Any task that involves "tracking how things move" benefits from flow, whether computed explicitly or learned implicitly by the model.

Multi-Object Tracking

Detection tells you what's in each frame. Multi-object tracking (MOT) tells you which detection in frame t corresponds to which detection in frame t+1. The dominant paradigm is "tracking by detection": run a detector on every frame, then associate detections across time.

SORT (Simple Online and Realtime Tracking) uses a Kalman filter to predict where each tracked object will appear next, then matches predictions to detections using the Hungarian algorithm on IoU (intersection over union) overlap. It's fast but struggles when objects occlude each other because it relies solely on spatial overlap.

DeepSORT adds a learned appearance descriptor — a small CNN that extracts a feature vector for each detection. Matching uses a combination of spatial distance (Kalman prediction vs. detection) and appearance distance (cosine similarity of feature vectors). This lets DeepSORT re-identify objects after occlusion: "that person disappeared behind a pillar and reappeared — the appearance features match, so it's the same person."

More recent approaches like TrackFormer unify detection and tracking in a single transformer: tracked objects become "track queries" that persist across frames, and new detections become "detection queries." The transformer's attention mechanism implicitly handles association, occlusion, and appearance modeling. This is cleaner architecturally but requires more compute and training data than the detection-then-associate pipeline.

Frame Sampling Strategy Matters

Most video models don't process every frame. Common strategies: uniform sampling (pick 8–32 evenly spaced frames), dense sampling (process all frames in short clips), and temporal stride (skip every k frames). The right strategy depends on the task — counting reps in exercise needs dense sampling; classifying "cooking vs. driving" works fine with sparse sampling. Always think about what temporal resolution your task actually requires.
