Chapter 9: CNNs & Computer VisionSpecialized applications · Advanced topics

Nice to Know

You won't need most of these on day one. But you will encounter them in the wild — in papers, job descriptions, project requirements. This section gives you just enough context to recognize each topic and know where to dig deeper when the time comes.

Video Understanding

3D convolutions (C3D, I3D) extend the familiar 2D filter to include a time dimension — the kernel slides across height, width, and frames. Video Transformers like TimeSformer and ViViT take a different approach, using self-attention across spatial and temporal tokens. Two-stream networks hedge their bets by processing RGB frames and optical flow through separate pathways, then fusing the results.

When you'd need it: If your task involves temporal patterns — action recognition, anomaly detection in surveillance, sports analytics — start with a pretrained video model from the Kinetics-400 dataset.

Pose Estimation

The goal is to detect human body keypoints (joints like elbows, knees, wrists) from images or video. Top-down methods detect the person first with a bounding box, then estimate joints within it. Bottom-up methods detect all joints in the scene at once, then group them by person. Key tools: OpenPose, HRNet, and MediaPipe.

When you'd need it: MediaPipe runs in real-time on a phone — start there for prototyping. HRNet is more accurate for research benchmarks. If you're building a fitness app, a gesture interface, or motion capture on a budget, this is your starting point.

Visual Search & Retrieval

Extract feature embeddings from a CNN or ViT backbone, store them in a vector database (FAISS, Milvus, Pinecone), and query by cosine or L2 similarity. The key to good retrieval is the embedding space: metric learning objectives like triplet loss, contrastive loss, or ArcFace train the network so similar images land close together and dissimilar ones are pushed apart.

When you'd need it: This is how reverse image search, product recommendation ("find visually similar items"), and face verification systems work under the hood.

Neural Radiance Fields (NeRF)

NeRFs represent a 3D scene as a neural network that maps a 5D input — spatial coordinates (x, y, z) plus viewing direction (θ, φ) — to color and volume density. You train it on a collection of 2D photos with known camera poses, and it learns to render the scene from any new viewpoint via differentiable volume rendering.

When you'd need it: Give it ~50 photos of a scene and you get a 3D-renderable model. Gaussian Splatting is the faster, more recent successor — if you're starting fresh, look there first.

Optical Flow

Optical flow computes per-pixel motion vectors between consecutive frames — essentially answering "where did each pixel move?" Classic methods (Lucas-Kanade, Farneback) work in simple settings; modern deep learning methods like RAFT are far more robust and accurate. Flow maps are often used as an input feature to video understanding models.

When you'd need it: If you need to know what moved where between frames — motion estimation, video stabilization, frame interpolation — that's optical flow. RAFT is the current go-to.

Multi-Object Tracking (MOT)

Detection tells you where objects are in a single frame. Tracking tells you which object is which across frames. The dominant paradigm is tracking-by-detection: run a detector every frame, then associate detections using appearance features and motion prediction. SORT uses a Kalman filter for motion; DeepSORT adds a Re-ID embedding for appearance. ByteTrack achieves state-of-the-art by also utilizing low-confidence detections.

When you'd need it: Surveillance, autonomous driving, sports analytics — anywhere you need consistent object identities over time. Detection handles the WHERE, tracking handles the WHICH.

Image Super-Resolution

Train a model to upscale low-resolution images to higher resolution. ESRGAN and Real-ESRGAN are the standards — they use a GAN-based approach where a generator network upscales the image and a discriminator pushes it toward photorealistic detail. The results can look impressive, but remember: the model is hallucinating plausible detail, not recovering ground truth.

When you'd need it: Enhancing satellite imagery, medical scans, or old photos. Useful, but don't expect miracles — you can't create detail that was never captured.

Deformable Convolutions

Standard convolutions sample on a fixed rectangular grid. Deformable convolutions add learned 2D offsets to each sampling point, so the kernel can adapt its shape to the geometry of the object it's looking at. This makes them better at handling irregular shapes, varying scales, and geometric transformations without needing explicit data augmentation for every case.

When you'd need it: They show up inside modern detection heads (like Deformable DETR) and are useful whenever your objects have irregular shapes or extreme aspect ratios.

Squeeze-and-Excitation (SE) Blocks

A channel attention mechanism. First, squeeze: global average pool each channel to a single number. Then, excitation: pass through a small bottleneck FC network to learn per-channel importance weights. Finally, scale each channel by its learned weight. It's a plug-and-play module — drop it into any architecture.

When you'd need it: SE blocks improve most CNN architectures by ~1% top-1 accuracy at minimal computational cost. They were introduced in SENet (ImageNet 2017 winner) and are now baked into EfficientNet, RegNet, and many others.

The Point of This Section

You don't need to master any of these right now. When you encounter them in a project requirement or a paper, you'll know what they are and where to start. That awareness is the difference between "I've never heard of that" and "Yeah, I know the approach — let me look up the details."

What You Should Now Be Able To Do

  • Recognize the major specialized areas of computer vision (video, pose, 3D, tracking) and know roughly which tools or architectures belong to each.
  • Explain what optical flow, NeRF, and super-resolution are at a high level — and when each is the right tool for the job.
  • Distinguish between detection (single-frame WHERE) and tracking (cross-frame WHICH), and name the key algorithms for each.
  • Understand how visual search works end-to-end: backbone → embedding → vector database → similarity query.
  • Identify architectural building blocks like deformable convolutions and SE blocks, and know they're plug-and-play enhancements you can add to existing networks.
