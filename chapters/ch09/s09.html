Vision-Language Models

Chapter 9: CNNs & Computer Vision 7 subtopics
TL;DR

The world isn't just images or just text — it's both, all the time. Vision-language models (VLMs) learn to connect pixels and words in a shared representation space. CLIP aligns images and text via contrastive learning, enabling zero-shot classification without task-specific training. Diffusion models like Stable Diffusion use text embeddings to guide image generation. Captioning models (BLIP-2) bridge the modality gap with a Q-Former. Modern multimodal LLMs (LLaVA, GPT-4o, Gemini) treat images as visual tokens fed directly into a language model. The trend is clear: specialized single-modality models are giving way to unified architectures that see and speak natively.

Think about how you understand a photograph. You don't just see edges and textures — you see "a golden retriever catching a frisbee on a sunny beach." Your brain effortlessly connects visual patterns to language concepts. For decades, computer vision and natural language processing evolved as separate fields. Image classifiers output class IDs. Language models output word sequences. Neither could do what a toddler does: look at a picture and talk about it.

The shift started around 2021. Researchers realized that the internet is full of images paired with text — captions, alt-text, descriptions, comments. If you train a model on hundreds of millions of these pairs, it learns something remarkable: a shared understanding of what things look like and what they're called. This wasn't just a new model — it was a new paradigm. Instead of training a cat-vs-dog classifier, then a car-vs-truck classifier, then a flower classifier, you train one model that understands the relationship between any image and any text description. Task-specific models gave way to general-purpose multimodal models.

CLIP: Contrastive Language-Image Pre-training

CLIP (Contrastive Language-Image Pre-training), released by OpenAI in January 2021, changed how we think about vision models. The core idea is almost disappointingly simple: take an image encoder and a text encoder, and train them so that matching image-text pairs end up close together in a shared embedding space, while non-matching pairs are pushed apart.

OpenAI scraped 400 million image-text pairs from the internet (called WIT — WebImageText). For each training batch of N pairs, CLIP computes the image embedding for all N images and the text embedding for all N captions. This creates an N×N matrix of cosine similarities. The diagonal entries (matching pairs) should be high; everything else should be low. The training objective is the InfoNCE loss — a symmetric contrastive loss that pushes the model to maximize the similarity of the N correct pairs while minimizing the similarity of the N²−N incorrect pairs.

Why was this revolutionary? Three reasons. First, zero-shot transfer: CLIP can classify images into categories it has never been explicitly trained on, just by comparing image embeddings to text embeddings of class descriptions. Second, scale: training on 400M noisy internet pairs beat models trained on 1.2M carefully curated ImageNet labels. The signal in messy, abundant data outweighed the precision of small, clean datasets. Third, robustness: CLIP was far more robust to distribution shifts than ImageNet-trained models — it didn't overfit to the quirks of any single dataset.

Why Contrastive Learning Works So Well

Contrastive learning doesn't tell the model what a "dog" looks like directly. Instead, it says: "this image and this text should have similar representations, and this image and that unrelated text should not." The model figures out the visual concepts on its own. This is more like how children learn — through association, not explicit labels.

Zero-Shot Classification with CLIP

Here's the practical magic of CLIP. Say you want to classify images into "cat," "dog," and "bird" — but you have zero labeled training data. With CLIP, you simply encode the image, encode each label as text (typically using a prompt template like "a photo of a {class}"), and pick the label whose text embedding is closest to the image embedding. No fine-tuning. No training loop. Just a forward pass through both encoders and a cosine similarity comparison.

# Zero-shot image classification with CLIP (using open_clip)
import open_clip
import torch
from PIL import Image

# Load model and preprocessing
model, _, preprocess = open_clip.create_model_and_transforms(
    'ViT-B-32', pretrained='laion2b_s34b_b79k'
)
tokenizer = open_clip.get_tokenizer('ViT-B-32')
model.eval()

# Prepare image
image = preprocess(Image.open("photo.jpg")).unsqueeze(0)

# Define candidate classes with prompt templates
classes = ["cat", "dog", "bird", "car", "airplane"]
text_prompts = [f"a photo of a {c}" for c in classes]
text_tokens = tokenizer(text_prompts)

# Encode both modalities
with torch.no_grad():
    image_features = model.encode_image(image)
    text_features = model.encode_text(text_tokens)

    # Normalize embeddings to unit vectors
    image_features /= image_features.norm(dim=-1, keepdim=True)
    text_features /= text_features.norm(dim=-1, keepdim=True)

    # Cosine similarity → probabilities
    similarity = (100.0 * image_features @ text_features.T).softmax(dim=-1)

# Print results
for cls, prob in zip(classes, similarity[0]):
    print(f"{cls:>10s}: {prob.item():.1%}")

CLIP's zero-shot accuracy on ImageNet is about 76% — competitive with a supervised ResNet-50 that was trained on all 1.28 million ImageNet labels. On some datasets with distribution shifts (ImageNet-Sketch, ImageNet-V2), CLIP actually beats supervised models handily.

But it's not perfect. CLIP struggles with fine-grained distinctions (telling apart 200 bird species), counting objects ("an image with exactly three dogs"), understanding spatial relationships ("the cup is on top of the book"), and abstract or compositional concepts. The text prompts are also fiddly — "a photo of a {class}" works better than just "{class}" because CLIP was trained on natural language captions, not bare labels.

Image Generation from Text

If CLIP connects images and text in a shared space, the next question is obvious: can we go from text to a brand-new image? The answer launched an entire creative revolution.

DALL-E (OpenAI, 2021) was the first dramatic demonstration. It treated image generation as a sequence prediction problem: encode an image as a sequence of discrete tokens using a dVAE (discrete variational autoencoder), concatenate with text tokens, and let a Transformer predict the next token autoregressively. It worked, but generated images were blurry at high resolution.

DALL-E 2 (2022) took a different approach. It used CLIP embeddings as the bridge: encode the text with CLIP's text encoder, use a "prior" model to predict the corresponding CLIP image embedding, then use a diffusion model (called unCLIP) to generate an image conditioned on that predicted image embedding. The results were dramatically sharper because diffusion models excel at iterative refinement.

Stable Diffusion (Stability AI, 2022) democratized the approach. The key insight was latent diffusion: instead of doing the noisy diffusion process in pixel space (expensive), do it in the latent space of a pre-trained autoencoder. The text conditioning enters through cross-attention layers — at each denoising step, the model attends to the text embedding to guide what it generates. This made high-quality image generation possible on consumer GPUs.

The common thread across all three: text embeddings guide the generation process. CLIP (or a similar encoder) translates language into a vector that the image generation model can condition on. The text doesn't tell the model which pixels to paint — it provides a high-level semantic direction that the generative model iteratively refines into concrete pixels.

Image Captioning

Generation goes from text to image. Image captioning goes the other direction — from image to text. The classic architecture is an encoder-decoder setup: a vision encoder (CNN or ViT) produces a representation of the image, and a language model decoder generates a caption word by word, conditioned on that visual representation.

Early approaches used a ResNet or VGG to extract a feature vector, then fed it as the initial hidden state of an LSTM. This worked but produced generic captions ("a man is standing on a street"). The features were too compressed — a single vector can't capture all the spatial details in an image.

Attention-based captioning improved things. Instead of compressing the image to one vector, keep the full grid of spatial features and let the language model attend to different regions as it generates each word. When writing "dog," the model attends to the dog region; when writing "frisbee," it shifts attention to the frisbee.

BLIP (Bootstrapping Language-Image Pre-training) and BLIP-2 pushed this further. BLIP-2 introduced the Q-Former (Querying Transformer) — a lightweight transformer that sits between a frozen image encoder and a frozen large language model. The Q-Former has a set of learnable query tokens that extract the most relevant visual information from the image encoder's output. This bridges the modality gap: the image encoder speaks in visual features, the LLM speaks in text tokens, and the Q-Former translates between them. Crucially, BLIP-2 keeps both the image encoder and the LLM frozen, only training the Q-Former — making it efficient to train even with massive backbone models.

Visual Question Answering (VQA)

Visual Question Answering takes multimodal understanding a step further: given an image and a natural language question about that image, produce the correct answer. "What color is the car?" "How many people are sitting?" "Is the dog to the left of the cat?" This requires not just recognizing objects but understanding spatial relationships, counting, reading text in images, and reasoning about scene semantics.

The central challenge is multimodal fusion — how do you combine visual and textual information? There are three main strategies:

Early fusion concatenates visual tokens and text tokens into a single sequence before any processing. The model treats them as one unified input. This is simple and lets the model learn arbitrary cross-modal interactions from scratch, but it's computationally expensive because the self-attention over the combined sequence scales quadratically.

Late fusion processes each modality independently through its own encoder, then combines the final representations (by concatenation, element-wise product, or a small fusion network). This is efficient because each encoder is specialized, but it limits cross-modal interaction — the image encoder can't "ask questions" about the text while processing the image.

Cross-attention fusion is the sweet spot for many architectures. One modality's representation serves as queries, and the other serves as keys and values (or vice versa). This lets the text representation attend to relevant image regions, and optionally the image features attend to relevant words. DeepMind's Flamingo (2022) pioneered this approach at scale: it interleaves frozen vision encoder features into a frozen language model using special cross-attention layers called "Perceiver Resampler" blocks. Flamingo could handle multiple images and text interleaved in context, enabling few-shot visual learning.

Modern Multimodal LLMs

The most exciting recent development is the convergence of vision and language into unified large language models. The architecture pattern has become almost standardized:

A vision encoder (typically a ViT variant like CLIP's visual encoder or SigLIP) processes the image into a grid of feature vectors. A projection layer (sometimes a simple MLP, sometimes a more complex adapter like Q-Former) maps these visual features into the same embedding space as the LLM's text tokens. These visual tokens are then concatenated with the text tokens and fed into the LLM, which processes the combined sequence using its standard attention mechanism. The LLM generates text conditioned on both the visual and textual context.

LLaVA (Large Language and Vision Assistant, 2023) demonstrated that this simple recipe works remarkably well. Take a pre-trained CLIP ViT, a pre-trained LLaMA, connect them with a linear projection layer, and fine-tune on visual instruction-following data. The training happens in two stages: first align the projection layer on image-caption pairs, then instruction-tune the full model on multimodal conversations. LLaVA showed that you don't need billions of dollars — a clever training recipe with good data can produce strong multimodal models.

GPT-4V and GPT-4o (OpenAI, 2023–2024) represent the frontier of commercial multimodal models. GPT-4o is natively multimodal — trained end-to-end on text, images, and audio, rather than bolting separate encoders together. This native training means the model doesn't suffer from the modality gap that plagues adapter-based approaches.

Gemini (Google, 2023) follows a similar philosophy: natively multimodal from the ground up, trained on interleaved text, image, audio, and video data. The native approach enables more fluid reasoning across modalities — the model can interleave image understanding and text generation naturally within a single forward pass.

Visual Tokens Are Expensive

A single 336×336 image at patch size 14 produces 576 visual tokens. Higher resolution or multiple images can easily push the visual token count into the thousands. Since LLM attention is quadratic in sequence length, visual tokens significantly increase inference cost. This is why techniques like the Perceiver Resampler and Q-Former exist — to compress visual information into fewer tokens.

DINOv2: Self-Supervised Vision Foundation Model

Not every vision foundation model needs language. DINOv2 (Meta, 2023) learns powerful visual representations using only images — no text, no labels, no captions. It uses a self-supervised teacher-student framework with an exponential moving average (EMA) to update the teacher.

The setup: a student network and a teacher network, both ViTs. The student sees small local crops of an image. The teacher sees larger global crops. The student is trained to match the teacher's output distribution. The teacher's weights are updated as an EMA of the student's weights — it's a slowly-moving target that stabilizes training. This is combined with a self-distillation loss that encourages the student to produce consistent representations regardless of which crop it sees.

The result is remarkable: DINOv2 features, learned without any labels whatsoever, produce excellent results on downstream tasks like classification, segmentation, and depth estimation — often competitive with supervised models. The features are also surprisingly semantic: nearby patches that belong to the same object tend to have similar feature vectors, creating emergent segmentation without any segmentation training.

How do CLIP features compare to DINOv2 features? CLIP features are optimized for text alignment — they're great for tasks that involve language (retrieval, zero-shot classification) but can lose fine-grained spatial detail because the contrastive objective focuses on image-level semantics. DINOv2 features preserve more local, spatial information because the self-supervised objective operates on patches and crops. For dense prediction tasks (segmentation, depth), DINOv2 often wins. For text-related tasks, CLIP wins. Some recent models (like SigLIP + DINOv2 combinations) use both.

Multimodal Fusion Strategies

We touched on fusion strategies in the VQA section, but this topic deserves a deeper look. How you combine information from different modalities is one of the most consequential architectural decisions in multimodal systems.

Early fusion is the simplest: convert both modalities into token sequences, concatenate them, and feed the combined sequence into a single transformer. This is what modern multimodal LLMs do with visual tokens. The advantage is maximum flexibility — the model can learn arbitrary cross-modal interactions through self-attention. The disadvantage is cost: attention over the combined sequence is O((N_img + N_text)²), and image tokens can dominate the sequence length.

Late fusion processes each modality independently with specialized encoders, then combines the final representations. CLIP is a late fusion model — the image and text encoders never interact until the very end, where a cosine similarity compares the two output vectors. Late fusion is efficient and allows pre-training each encoder independently, but the lack of intermediate interaction limits the model's ability to reason about fine-grained relationships between modalities.

Cross-attention fusion is the middle ground. One modality's features serve as queries that attend to the other modality's features via cross-attention layers. This lets one stream "ask questions" of the other without merging them into a single sequence. Flamingo's Perceiver Resampler uses learned latent queries that cross-attend to the image features, producing a fixed number of output tokens regardless of image resolution. BLIP-2's Q-Former is similar: learnable query tokens cross-attend to both frozen image features and text tokens. Both approaches compress visual information into a manageable number of tokens before passing them to the language model.

Hybrid CNN-Transformer Architectures

While much of the VLM world has moved to pure ViT encoders, there's a parallel line of research asking: can we get the best of both CNNs and Transformers? CNNs have strong inductive biases for local spatial patterns — translation equivariance, locality — and are efficient at processing high-resolution images. Transformers excel at global reasoning and long-range dependencies. Hybrid architectures try to combine these strengths.

CoAtNet (Google, 2021) stacks depthwise convolution stages (for local feature extraction) followed by Transformer stages (for global attention). The convolution stages handle the early, high-resolution processing efficiently, while the Transformer stages handle the later, lower-resolution processing where global context matters most. CoAtNet achieved state-of-the-art ImageNet accuracy when it was released.

CvT (Convolutional Vision Transformer) introduces convolutions into the Transformer itself. Instead of the standard linear projection for creating keys, queries, and values, CvT uses depthwise separable convolutions. This injects local spatial structure into the attention computation at every layer, helping the model learn local patterns without needing enormous datasets.

LeViT (Meta, 2021) is designed for fast inference. It uses a CNN-like stem for initial feature extraction, then switches to Transformer blocks with attention biases that encode spatial locality. LeViT is 3–5× faster than similarly accurate ViT models, making it practical for mobile and edge deployment.

# Using a CLIP-based model for image-text similarity with Hugging Face
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import torch

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

image = Image.open("beach.jpg")
texts = [
    "a dog playing on the beach",
    "a cat sleeping on a couch",
    "a sunset over mountains",
    "a car parked in a garage"
]

inputs = processor(text=texts, images=image, return_tensors="pt", padding=True)

with torch.no_grad():
    outputs = model(**inputs)
    logits = outputs.logits_per_image  # shape: [1, 4]
    probs = logits.softmax(dim=-1)

for text, prob in zip(texts, probs[0]):
    print(f"{prob.item():.1%}  {text}")

When do hybrids outperform pure architectures? Generally in two scenarios: when training data is limited (the CNN's inductive biases help) and when inference speed matters (convolutions are faster than attention at high resolution). With unlimited data and compute, pure Transformers tend to win — which is why the largest models (GPT-4o, Gemini) are mostly Transformer-based. But for practical deployment with constrained resources, hybrids remain highly relevant.

Where This Is All Heading

The trend is unmistakable: separate vision and language models are merging into unified systems. Early models bolted encoders together with adapters. Current frontier models are natively multimodal — trained end-to-end on interleaved text, images, audio, and video. The adapter-based approach (LLaVA, BLIP-2) is still valuable for efficiency and modularity. But the endgame is models that perceive and reason across modalities as fluently as humans do — no bridges, no translation layers, just one model that sees, reads, listens, and thinks.
