Segmentation

Chapter 9: CNNs & Computer Vision Semantic · Instance · Panoptic · U-Net · SAM
TL;DR

Detection gives you boxes. Segmentation gives you pixel-perfect boundaries. Three flavors: semantic (classify every pixel), instance (separate individual objects), panoptic (both). Key architectures are FCN (first end-to-end pixel prediction), U-Net (encoder-decoder with skip connections — dominates medical imaging), DeepLab (atrous convolutions for multi-scale context), Mask R-CNN (Faster R-CNN + mask head for instance segmentation), and SAM (foundation model for promptable, zero-shot segmentation). Use Dice loss for class imbalance. Evaluate with mIoU.

Object detection draws boxes around things. That's useful, but a box is a rough approximation — it includes a lot of background and tells you nothing about shape. If you need to know the exact boundary of a tumor, the precise outline of each car in a self-driving scene, or which pixels belong to road versus sidewalk, you need segmentation.

Segmentation is pixel-level classification. Instead of one label per image (classification) or one label per bounding box (detection), segmentation assigns a label to every single pixel. The output isn't a class or a box — it's a map the same size as the input image, where each pixel carries a prediction.

Three Flavors of Segmentation

Not all segmentation tasks are the same. The field has converged on three well-defined task types, and choosing the right one depends on what you need to know about the scene.

Semantic Segmentation

Every pixel gets a class label. Period. All cats in the image are labeled "cat," all dogs are "dog," all background is "background." You know what is at every pixel, but you can't distinguish between individual objects of the same class. Two cats touching each other? They're one blob of "cat" pixels.

Use case: autonomous driving (road, sidewalk, building, sky), land-use mapping from satellite imagery, scene understanding.

Instance Segmentation

Each individual object gets its own mask. Cat 1, Cat 2, Cat 3 — each with a separate, pixel-perfect boundary. This is harder than semantic segmentation because the model must both classify pixels and group them into distinct object instances.

Use case: counting cells in microscopy, separating overlapping objects in robotics, any task where "how many" matters as much as "what."

Panoptic Segmentation

The unified version. Panoptic segmentation divides the scene into stuff (amorphous regions like sky, road, grass) and things (countable objects like cars, people, dogs). Stuff gets semantic labels. Things get instance-level masks. Every pixel in the image is accounted for — no gaps, no overlaps.

Use case: complete scene understanding for autonomous systems, robotics, and augmented reality. This is what you want when you need a full, unambiguous description of every pixel in a scene.

FCN: Fully Convolutional Networks

Before 2015, people did segmentation by classifying each pixel independently — slide a patch classifier across the image, one pixel at a time. Horrendously slow. The Fully Convolutional Network (Long et al., 2015) changed everything by making one simple observation: if you replace the fully connected layers in a classification CNN with 1×1 convolutions, the entire network becomes fully convolutional and can accept inputs of any size, producing a spatial output map in a single forward pass.

The problem: after multiple pooling layers, the output is much smaller than the input (e.g., 7×7 for a 224×224 input). FCN recovers the spatial resolution using transposed convolutions (learnable upsampling) to produce pixel-level predictions. The paper introduced skip connections from earlier layers (FCN-32s → FCN-16s → FCN-8s) to combine coarse, high-level semantics with fine, low-level spatial detail.

FCN was the proof of concept. It showed end-to-end trainable pixel prediction was possible. But its outputs were still fairly coarse. The architectures that followed — especially U-Net and DeepLab — refined the idea dramatically.

U-Net: The Workhorse Architecture

If there's one segmentation architecture you should know inside and out, it's U-Net (Ronneberger et al., 2015). Originally designed for biomedical image segmentation where labeled data is scarce, U-Net has become the default architecture for segmentation tasks across domains — from cell segmentation to satellite imagery to industrial inspection.

The Encoder-Decoder Structure

U-Net has a distinctive symmetric shape that gives it its name. The left side is the encoder (contracting path): a standard stack of convolutions and max-pooling that progressively reduces spatial resolution while increasing the number of feature channels. This path captures what is in the image — the semantic content. The right side is the decoder (expanding path): transposed convolutions that progressively increase spatial resolution while decreasing the number of channels. This path recovers where things are — the spatial detail.

Skip Connections: The Key Innovation

The magic of U-Net is the skip connections. At each level, the encoder's feature maps are concatenated (not added — concatenated) with the decoder's upsampled feature maps. This gives the decoder access to both high-level "what" information (from the upsampled deep features) and fine-grained "where" information (from the encoder at the same resolution). Without skip connections, the decoder has to hallucinate spatial detail from a heavily compressed bottleneck. With them, it can use the original high-resolution features directly.

This is why U-Net produces sharp, precise boundaries even when trained on small datasets. The spatial information doesn't need to survive the bottleneck — it takes a shortcut.

Why U-Net Dominates Medical Imaging

Medical imaging has two properties that make U-Net ideal:

  1. Very little labeled data. Getting an expert radiologist to annotate pixel-level masks is expensive. U-Net's architecture is data-efficient — the skip connections provide strong inductive biases that help it learn from hundreds (not millions) of examples.
  2. Precise boundaries matter. In medical imaging, a few pixels of error can mean the difference between "tumor is operable" and "tumor has invaded surrounding tissue." U-Net's skip connections preserve the spatial precision needed for clinical decisions.

Modern variants like U-Net++ (nested skip connections), Attention U-Net (learnable skip connection gating), and nnU-Net (auto-configured U-Net that adapts hyperparameters to any medical segmentation task) have extended the basic architecture. But the core idea — symmetric encoder-decoder with skip connections — remains the dominant paradigm.

DeepLab: Dilated Convolutions and Multi-Scale Context

U-Net recovers spatial detail through the decoder. DeepLab takes a different approach: what if we never lose spatial detail in the first place?

Atrous (Dilated) Convolutions

Standard convolutions with stride or pooling reduce spatial resolution to increase the receptive field. Atrous convolutions (also called dilated convolutions) expand the receptive field without downsampling. They insert gaps ("holes" — hence "à trous" in French) between kernel elements:

Standard 3×3 convolution (receptive field = 3×3):
  X X X
  X X X
  X X X

Dilated 3×3 convolution, rate=2 (receptive field = 5×5):
  X . X . X
  . . . . .
  X . X . X
  . . . . .
  X . X . X

Dilated 3×3 convolution, rate=4 (receptive field = 9×9):
  X . . . X . . . X
  . . . . . . . . .
  . . . . . . . . .
  . . . . . . . . .
  X . . . X . . . X
  . . . . . . . . .
  . . . . . . . . .
  . . . . . . . . .
  X . . . X . . . X

Same number of parameters. Much larger receptive field.
The network "sees" more context without losing resolution.

This means you can maintain a higher-resolution feature map throughout the network while still capturing long-range context. The tradeoff: higher memory usage (you're working with larger feature maps).

ASPP: Atrous Spatial Pyramid Pooling

Objects in images appear at different scales. A car nearby is huge; a car far away is tiny. ASPP handles this by applying atrous convolutions at multiple dilation rates in parallel, then combining the results. Each rate captures features at a different scale:

Each branch captures context at a different spatial scale.
Concatenation fuses all scales into a rich multi-scale representation.

DeepLabv3+ combines ASPP with a simple decoder module and uses a modified Xception or ResNet backbone with atrous convolutions in the later stages. The result: strong multi-scale context awareness with sharp boundary recovery. DeepLabv3+ remains a top-tier choice for semantic segmentation on natural images, particularly for driving scenes and outdoor environments.

Mask R-CNN: Instance Segmentation

Remember Faster R-CNN from the detection section? It generates region proposals, classifies them, and refines bounding boxes. Mask R-CNN (He et al., 2017) makes one elegant addition: a parallel branch that predicts a binary segmentation mask for each detected object.

Mask R-CNN Architecture:

Image ──▶ Backbone (ResNet + FPN) ──▶ Region Proposal Network
                                            │
                                            ▼
                                      RoI Proposals
                                            │
                                       RoIAlign
                                      ╱     │     ╲
                                    ╱       │       ╲
                              Class Head  Box Head  Mask Head
                              (softmax)  (regress)  (per-class
                                                     binary mask)

Key difference from Faster R-CNN: the Mask Head.
For each detected object, it predicts a 28×28 binary mask.

Two important details:

  1. RoIAlign replaces RoIPool. RoIPool used coarse quantization (rounding to integer coordinates), which introduced spatial misalignment — fine for classification but destructive for pixel-level masks. RoIAlign uses bilinear interpolation, preserving sub-pixel precision.
  2. Masks are per-class. For each proposal, the mask head predicts K binary masks (one per class), and only the mask for the predicted class is used. This decouples classification from segmentation, making both tasks easier to learn.

Mask R-CNN is a two-stage method (propose then refine), so it's not the fastest. But its accuracy on instance segmentation benchmarks was dominant for years, and it remains the starting point for many instance segmentation projects.

Segment Anything Model (SAM)

In 2023, Meta AI released SAM, and it changed the game for segmentation the way GPT changed the game for text. SAM is a foundation model for segmentation — trained on over 1 billion masks across 11 million images, it can segment anything without task-specific training.

Promptable Segmentation

SAM's key innovation is promptable segmentation. Instead of training a model for a specific set of classes, you give SAM a prompt — a point, a bounding box, a set of points, or a text description — and it returns a segmentation mask for whatever you pointed at.

Encode the image once. Generate masks interactively
with different prompts — in real time.

This design is deliberate. The image encoder (a massive Vision Transformer) is expensive but only runs once. The prompt encoder and mask decoder are lightweight, so you can try many different prompts interactively — click a point, get a mask instantly, click another point, refine the mask. This enables interactive annotation workflows that are orders of magnitude faster than manual polygon drawing.

Why SAM Changes Everything

  • Zero-shot segmentation. SAM segments objects it has never been explicitly trained on. No fine-tuning needed for most tasks. Give it an image of a cell, a satellite, a factory part — it segments it.
  • Annotation acceleration. Labeling segmentation masks manually takes minutes per image. With SAM, a single click generates a mask in milliseconds. This makes building segmentation datasets dramatically faster.
  • Foundation for downstream tasks. SAM's masks can feed into specialized classifiers, tracking systems, or 3D reconstruction pipelines. It handles the "where" while other models handle the "what."

SAM 2 (2024) extended the model to video, maintaining consistent segmentation masks across frames with a memory mechanism — moving from "segment anything in an image" to "segment anything in a video."

Loss Functions for Segmentation

Segmentation losses are trickier than classification losses because classes are almost always imbalanced. In a driving scene, 60% of pixels might be "road," 20% "sky," and the remaining 20% split among cars, people, signs, and everything else. A model that predicts "road" everywhere gets 60% accuracy.

Cross-Entropy (Per-Pixel)

The straightforward approach: treat each pixel as an independent classification problem and sum the cross-entropy losses. Works, but dominated by the majority class unless you add class weights:

# Standard per-pixel cross-entropy with class weights
weights = torch.tensor([0.5, 2.0, 3.0, 5.0, ...])  # rare classes get higher weight
loss_fn = nn.CrossEntropyLoss(weight=weights)

Dice Loss

Dice loss directly optimizes the overlap between the predicted mask and the ground truth. It measures how well two sets overlap, naturally handling class imbalance because it's based on the ratio of overlap to total area — not raw pixel counts:

Dice coefficient = 2 × |Prediction ∩ Ground Truth|
                   ─────────────────────────────────
                     |Prediction| + |Ground Truth|

Dice loss = 1 − Dice coefficient

Perfect overlap → Dice = 1.0 → Loss = 0.0
No overlap     → Dice = 0.0 → Loss = 1.0

Dice loss is especially important in medical imaging where the region of interest (a lesion, a tumor) might occupy 1-2% of the image. Cross-entropy would happily ignore those pixels. Dice loss forces the model to get the small region right.

Combined Loss

In practice, most segmentation pipelines use a combination: Loss = CE + λ × Dice. Cross-entropy provides stable per-pixel gradients. Dice loss ensures small regions aren't ignored. Together, they outperform either alone.

Evaluation Metrics

MetricFormulaWhat It MeasuresNotes
IoU (Jaccard) |P ∩ G| / |P ∪ G| Overlap between predicted and ground truth mask Per-class; range [0, 1]
mIoU Mean of per-class IoU Average segmentation quality across all classes The standard metric for semantic segmentation
Dice Score 2|P ∩ G| / (|P| + |G|) Overlap (equivalent to F1 at pixel level) Standard in medical imaging
Pixel Accuracy Correct pixels / Total pixels Raw accuracy Misleading with class imbalance — avoid as primary metric

IoU and Dice score are related: Dice = 2 × IoU / (1 + IoU). Dice is always ≥ IoU. For benchmarking, mIoU is the standard on datasets like PASCAL VOC, Cityscapes, and ADE20K. For medical imaging, Dice score is conventional.
