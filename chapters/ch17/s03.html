Multi-Armed Bandits

Chapter 17: Learning Theory & Advanced ML 5 subtopics
TL;DR

Multi-armed bandits formalize the exploration-exploitation dilemma: should you stick with what works or try something new? You have K slot machines (arms) with unknown reward distributions. Each pull gives you a reward, and you want to maximize your total payoff. Epsilon-greedy explores randomly. UCB1 explores optimistically using confidence intervals. Thompson Sampling explores probabilistically using Bayesian posteriors. Extensions like contextual bandits (LinUCB) power real-time decisions in recommendation systems, clinical trials, and ad placement — anywhere you must learn and act simultaneously.

You are the lead engineer at a streaming service. Your homepage can show one of five different recommendation layouts. Layout A has been performing well — a 5% click-through rate. But you just designed Layouts B through E. Any of them might be better. Do you keep showing Layout A (exploit what you know) or try the new ones (explore to learn)? Every time you show an inferior layout, you lose clicks. Every time you refuse to experiment, you might miss a layout that doubles engagement.

This is the multi-armed bandit problem, named after a gambler facing a row of slot machines ("one-armed bandits") with unknown payoff distributions. It is the simplest formulation of the exploration-exploitation tradeoff, and it has surprisingly deep theory and wildly practical applications.

Problem Setting

The Stochastic Bandit Model

In the stochastic bandit setting, you have K arms. Each arm i has a fixed but unknown reward distribution with mean μᵢ. At each round t = 1, ..., T, you pull one arm and observe a reward drawn from that arm's distribution. The best arm has mean μ* = max(μᵢ). Your goal is to minimize cumulative regret:

Regret(T) = T · μ* - Σₜ μ_{aₜ}

where aₜ is the arm you pulled at time t. Regret counts how much reward you lost by not always pulling the best arm.

import numpy as np

class BanditEnvironment:
    """K-armed stochastic bandit with Bernoulli rewards."""
    
    def __init__(self, true_means):
        self.means = np.array(true_means)
        self.K = len(true_means)
        self.best_mean = np.max(true_means)
        self.best_arm = np.argmax(true_means)
    
    def pull(self, arm):
        """Pull arm, get Bernoulli reward."""
        reward = float(np.random.rand() < self.means[arm])
        regret = self.best_mean - self.means[arm]
        return reward, regret

# 5 arms with different click-through rates
env = BanditEnvironment([0.05, 0.03, 0.08, 0.04, 0.07])
print(f"True means: {env.means}")
print(f"Best arm: {env.best_arm} with mean {env.best_mean}")
print(f"Gap (suboptimality) of each arm: {env.best_mean - env.means}")

Adversarial Bandits

In the adversarial bandit setting, the rewards are chosen by an adversary, not drawn from fixed distributions. This is relevant when competing against adaptive opponents (ad auctions, security). The achievable regret is higher: O(√(KT)) instead of O(log T). The EXP3 algorithm handles this setting, and we will see it later.

Algorithms

Epsilon-Greedy — The Simplest Baseline

Epsilon-greedy is the algorithm everyone tries first. With probability ε, pick a random arm (explore). With probability 1 - ε, pick the arm with the highest observed average reward (exploit). It works, but it has a problem: it explores at a constant rate forever, even after it has confidently identified the best arm. This gives O(ε·T) regret from unnecessary exploration.

import numpy as np

class EpsilonGreedy:
    """Epsilon-greedy bandit algorithm."""
    
    def __init__(self, K, epsilon=0.1):
        self.K = K
        self.epsilon = epsilon
        self.counts = np.zeros(K)
        self.values = np.zeros(K)
    
    def select_arm(self):
        if np.random.rand() < self.epsilon:
            return np.random.randint(self.K)
        return np.argmax(self.values)
    
    def update(self, arm, reward):
        self.counts[arm] += 1
        # Incremental mean update
        n = self.counts[arm]
        self.values[arm] += (reward - self.values[arm]) / n

def run_bandit_experiment(env, algorithm, T=10000):
    """Run bandit experiment and track cumulative regret."""
    cumulative_regret = 0
    regrets = []
    
    for t in range(T):
        arm = algorithm.select_arm()
        reward, instant_regret = env.pull(arm)
        algorithm.update(arm, reward)
        cumulative_regret += instant_regret
        regrets.append(cumulative_regret)
    
    return regrets

# Run epsilon-greedy
np.random.seed(42)
env = BanditEnvironment([0.3, 0.5, 0.7, 0.4, 0.6])
eg = EpsilonGreedy(5, epsilon=0.1)
eg_regrets = run_bandit_experiment(env, eg)
print(f"ε-greedy regret after 10K rounds: {eg_regrets[-1]:.1f}")

UCB1 — Optimism in the Face of Uncertainty

Upper Confidence Bound (UCB1) takes a smarter approach. Instead of random exploration, it selects the arm with the highest upper confidence bound — the estimated mean plus a bonus for uncertainty. Arms you have not tried much have wide confidence intervals, so they get explored. Arms you have tried a lot have narrow intervals, so only truly promising ones get selected.

The selection rule is: aₜ = argmax(μ̂ᵢ + √(2 ln t / nᵢ)), where μ̂ᵢ is the empirical mean and nᵢ is the number of pulls. The √(2 ln t / nᵢ) term is the exploration bonus — it shrinks as you pull an arm more (nᵢ grows) and grows as time passes (ln t grows, encouraging revisiting old arms).

class UCB1:
    """UCB1: Upper Confidence Bound algorithm.
    
    Achieves O(K * log(T) / Δ) regret where Δ is the gap.
    """
    
    def __init__(self, K):
        self.K = K
        self.counts = np.zeros(K)
        self.values = np.zeros(K)
        self.t = 0
    
    def select_arm(self):
        self.t += 1
        # Pull each arm once first
        for i in range(self.K):
            if self.counts[i] == 0:
                return i
        
        # UCB1 selection
        ucb_values = self.values + np.sqrt(2 * np.log(self.t) / self.counts)
        return np.argmax(ucb_values)
    
    def update(self, arm, reward):
        self.counts[arm] += 1
        n = self.counts[arm]
        self.values[arm] += (reward - self.values[arm]) / n

np.random.seed(42)
env = BanditEnvironment([0.3, 0.5, 0.7, 0.4, 0.6])
ucb = UCB1(5)
ucb_regrets = run_bandit_experiment(env, ucb)
print(f"UCB1 regret after 10K rounds: {ucb_regrets[-1]:.1f}")
print(f"Arm pull counts: {ucb.counts}")
# UCB1 heavily favors the best arm but keeps exploring others slowly

Thompson Sampling — The Bayesian Approach

Thompson Sampling is the oldest bandit algorithm (1933!) and often the best in practice. The idea is Bayesian: maintain a posterior distribution over each arm's mean. Each round, sample from each posterior and pull the arm with the highest sample. Arms with high uncertainty generate high samples occasionally, ensuring exploration. Arms with well-understood low means rarely generate high samples, so you stop wasting pulls on them naturally.

class ThompsonSampling:
    """Thompson Sampling with Beta posteriors (for Bernoulli rewards).
    
    Maintains Beta(α, β) posterior for each arm.
    Achieves near-optimal regret with elegant Bayesian reasoning.
    """
    
    def __init__(self, K):
        self.K = K
        self.alpha = np.ones(K)  # successes + 1 (Beta prior)
        self.beta = np.ones(K)   # failures + 1
    
    def select_arm(self):
        # Sample from each arm's posterior
        samples = np.random.beta(self.alpha, self.beta)
        return np.argmax(samples)
    
    def update(self, arm, reward):
        if reward > 0:
            self.alpha[arm] += 1
        else:
            self.beta[arm] += 1
    
    def get_posterior_stats(self):
        means = self.alpha / (self.alpha + self.beta)
        return means

np.random.seed(42)
env = BanditEnvironment([0.3, 0.5, 0.7, 0.4, 0.6])
ts = ThompsonSampling(5)
ts_regrets = run_bandit_experiment(env, ts)
print(f"Thompson Sampling regret after 10K rounds: {ts_regrets[-1]:.1f}")
print(f"Posterior means: {ts.get_posterior_stats().round(3)}")
print(f"True means:      {env.means}")

EXP3 — Adversarial Bandits

EXP3 (Exponential-weight algorithm for Exploration and Exploitation) handles adversarial bandits. It maintains probability weights over arms using multiplicative updates (like Hedge from online learning), but with a crucial twist: since you only observe the reward of the arm you pulled, you construct an unbiased importance-weighted estimate of all arms' rewards. This gives O(√(KT log K)) regret against any adversary.

class EXP3:
    """EXP3 for adversarial bandits.
    
    Uses importance-weighted reward estimates and multiplicative updates.
    Regret: O(sqrt(K * T * ln(K)))
    """
    
    def __init__(self, K, gamma=None, T=10000):
        self.K = K
        self.gamma = gamma or np.sqrt(K * np.log(K) / T)
        self.weights = np.ones(K)
    
    def get_probs(self):
        """Mixed strategy: (1-γ) * weights/sum + γ/K"""
        w_sum = self.weights.sum()
        return (1 - self.gamma) * self.weights / w_sum + self.gamma / self.K
    
    def select_arm(self):
        probs = self.get_probs()
        return np.random.choice(self.K, p=probs)
    
    def update(self, arm, reward):
        probs = self.get_probs()
        # Importance-weighted reward estimate
        estimated_reward = reward / probs[arm]
        # Only update the pulled arm (importance weighting handles the rest)
        self.weights[arm] *= np.exp(self.gamma * estimated_reward / self.K)

np.random.seed(42)
# Adversarial scenario: best arm changes over time
K, T = 5, 10000
exp3 = EXP3(K, T=T)
cumulative_regret = 0

for t in range(T):
    # Adversary: arm rewards shift every 2000 rounds
    phase = t // 2000
    rewards = np.array([0.3, 0.5, 0.7, 0.4, 0.6])
    rewards = np.roll(rewards, phase)  # best arm changes!
    
    arm = exp3.select_arm()
    reward = float(np.random.rand() < rewards[arm])
    exp3.update(arm, reward)
    cumulative_regret += rewards.max() - rewards[arm]

print(f"EXP3 regret in adversarial setting: {cumulative_regret:.1f}")

Extensions

Contextual Bandits — Personalized Decisions

In real applications, you have context. A recommendation is not "which layout for everyone?" but "which layout for this user, on this device, at this time?" Contextual bandits extend the standard setting: at each round, you observe a context vector xₜ, then choose an arm. The expected reward depends on both the arm and the context.

LinUCB — Linear Contextual Bandits

LinUCB assumes the expected reward is linear: E[r|x, a] = θₐᵀx. For each arm, it maintains a ridge regression estimate of θₐ along with a confidence ellipsoid. The selection rule is UCB-style: pick the arm with the highest upper confidence bound, computed using the linear model's uncertainty.

class LinUCB:
    """LinUCB for contextual bandits.
    
    Assumes linear reward: E[r|x,a] = θ_a^T x
    Uses ridge regression with confidence ellipsoids.
    """
    
    def __init__(self, K, d, alpha=1.0):
        self.K = K
        self.d = d
        self.alpha = alpha
        # Per-arm statistics
        self.A = [np.eye(d) for _ in range(K)]  # d×d design matrix
        self.b = [np.zeros(d) for _ in range(K)]  # d×1 response
    
    def select_arm(self, x):
        ucb_values = np.zeros(self.K)
        for a in range(self.K):
            A_inv = np.linalg.inv(self.A[a])
            theta_hat = A_inv @ self.b[a]
            # UCB = predicted reward + confidence width
            ucb_values[a] = theta_hat @ x + self.alpha * np.sqrt(x @ A_inv @ x)
        return np.argmax(ucb_values)
    
    def update(self, arm, x, reward):
        self.A[arm] += np.outer(x, x)
        self.b[arm] += reward * x

# Demo: contextual bandit for news recommendation
np.random.seed(42)
d = 5   # context dimension (user features)
K = 3   # number of articles to recommend

# True preference vectors for each article
theta_true = [np.array([1, 0, 0, 0, 0]),   # article 0: sports fans
              np.array([0, 1, 0, 0, 0]),   # article 1: tech readers
              np.array([0, 0, 1, 0.5, 0])] # article 2: science + politics

linucb = LinUCB(K, d, alpha=0.5)
cumulative_reward = 0

for t in range(5000):
    # Random user context
    x = np.random.randn(d)
    x /= np.linalg.norm(x)
    
    arm = linucb.select_arm(x)
    # True reward: linear + noise
    true_reward = theta_true[arm] @ x + 0.1 * np.random.randn()
    reward = max(0, min(1, true_reward))  # clip to [0,1]
    linucb.update(arm, x, reward)
    cumulative_reward += reward

# Check if LinUCB learned the right θ
for a in range(K):
    A_inv = np.linalg.inv(linucb.A[a])
    theta_hat = A_inv @ linucb.b[a]
    print(f"Arm {a}: true θ={theta_true[a]}, learned θ={theta_hat.round(2)}")

Other Extensions

Combinatorial bandits let you choose subsets of arms (think: selecting a slate of K items from N candidates). Best arm identification flips the objective: instead of maximizing cumulative reward, you want to identify the best arm with minimum samples. Non-stationary bandits handle drifting reward distributions, using techniques like sliding windows or discounted estimates.

Theoretical Results

The Lai-Robbins Lower Bound

The Lai-Robbins lower bound (1985) says: for any consistent bandit algorithm, the expected number of times you pull a suboptimal arm i must grow at least as fast as ln(T) / KL(μᵢ || μ*), where KL is the Kullback-Leibler divergence between the arm's distribution and the best arm's distribution. This gives a regret lower bound of Ω(Σᵢ Δᵢ · ln T / KL(μᵢ || μ*)).

UCB1 and Thompson Sampling both achieve logarithmic regret, matching this lower bound up to constant factors. The minimax regret (worst case over all problem instances) is Θ(√(KT)), achieved by algorithms that do not need to know the gaps.

def lai_robbins_lower_bound(true_means, T):
    """Compute Lai-Robbins lower bound on regret.
    
    Regret >= Σ_i (Δ_i / KL(μ_i, μ*)) * ln(T)
    For Bernoulli: KL(p,q) = p*ln(p/q) + (1-p)*ln((1-p)/(1-q))
    """
    mu_star = max(true_means)
    
    def kl_bernoulli(p, q):
        if p == 0: return -np.log(1 - q + 1e-10)
        if p == 1: return -np.log(q + 1e-10)
        return p * np.log(p / (q + 1e-10)) + (1-p) * np.log((1-p) / (1-q + 1e-10))
    
    lower_bound = 0
    for mu_i in true_means:
        if mu_i < mu_star:
            delta_i = mu_star - mu_i
            kl = kl_bernoulli(mu_i, mu_star)
            lower_bound += delta_i * np.log(T) / (kl + 1e-10)
    
    return lower_bound

# Compare theoretical lower bound vs actual algorithm performance
means = [0.3, 0.5, 0.7, 0.4, 0.6]
for T in [1000, 5000, 10000, 50000]:
    lb = lai_robbins_lower_bound(means, T)
    print(f"T={T:>6d}: Lai-Robbins lower bound = {lb:.1f}")

Applications

A/B Testing and Adaptive Experiments

Traditional A/B testing is wasteful — you split traffic 50/50 even when one variant is clearly better halfway through. Bandit-based adaptive experiments shift traffic toward the winning variant as evidence accumulates. This reduces regret (fewer users see the inferior variant) while still maintaining statistical validity. Companies like Google, Netflix, and Microsoft use bandit-powered experimentation platforms.

Recommendation Systems

Every "for you" feed is a contextual bandit problem. The context is the user profile and situation. The arms are items to recommend. The reward is engagement (click, watch, purchase). LinUCB and Thompson Sampling are workhorses here. The cold-start problem (new users with no history) is naturally handled by the exploration component — the algorithm tries diverse recommendations to learn the user's preferences quickly.

Clinical Trials

Adaptive clinical trials use bandits to allocate patients to treatments. As evidence accumulates that treatment A is better than B, more patients get assigned to A. This is ethically superior to fixed randomization — fewer patients receive the inferior treatment — while still producing statistically valid conclusions. Thompson Sampling is particularly popular because its Bayesian framework aligns naturally with the statistical analysis required by regulators.

Bandits Are the Bridge Between Theory and Practice

Multi-armed bandits sit at the sweet spot between theory and application. The theory is clean and elegant — tight regret bounds, matching lower bounds, clear algorithmic principles. And the applications are immediate — any system making sequential decisions under uncertainty benefits from bandit algorithms. If you work in industry and want to apply one piece of ML theory directly to production, start here. A/B testing platforms, recommendation engines, and ad systems all use bandits, and the gap between the textbook algorithm and the production implementation is smaller than almost any other area of ML.
