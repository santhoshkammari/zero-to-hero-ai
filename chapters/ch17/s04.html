Reinforcement Learning

Chapter 17: Learning Theory & Advanced ML 12 subtopics
TL;DR

Reinforcement learning teaches agents to make sequences of decisions by trial and error, maximizing cumulative reward over time. An agent observes states, takes actions, receives rewards, and learns a policy — a mapping from states to actions. Value functions estimate long-term reward. Dynamic programming solves known environments. Q-learning and SARSA learn from experience without a model. Deep Q-Networks scale to raw pixels. Policy gradient methods (REINFORCE, PPO) directly optimize the policy. Model-based RL (Dyna, MuZero) learns an environment model for planning. This section covers the full stack from tabular methods to modern deep RL, including offline RL, imitation learning, and multi-agent settings.

Consider a robot learning to walk. Nobody writes explicit rules for "move left leg 23 degrees forward, shift weight, extend right leg." Instead, you define a reward — distance traveled forward minus energy spent — and let the robot figure it out through thousands of tries. Most attempts end with the robot falling on its face. But gradually, through trial and error, it discovers gaits that no engineer would have designed. That is reinforcement learning.

RL is fundamentally different from supervised learning. There is no labeled dataset. The agent must generate its own training data by interacting with the environment. Actions have consequences that unfold over time — eating a mushroom in a game might not kill you for another 50 steps. The agent must figure out which past actions led to current rewards. This is called the credit assignment problem, and it is what makes RL hard.

Markov Decision Process (MDP)

The Mathematical Framework

A Markov Decision Process is a tuple (S, A, P, R, γ) that formalizes sequential decision-making. S is the set of states. A is the set of actions. P(s'|s, a) is the transition probability — the chance of landing in state s' after taking action a in state s. R(s, a) is the reward function. γ ∈ [0, 1) is the discount factor — how much you value future rewards relative to immediate ones.

The Markov property is the key assumption: the future depends only on the current state and action, not on the history of how you got there. The current state contains all relevant information. Chess satisfies this (the board position is all you need). Real-world tasks often violate it (you cannot see all relevant information), leading to Partially Observable MDPs (POMDPs).

import numpy as np

class GridWorldMDP:
    """Simple 4x4 grid world MDP.
    
    Agent starts at (0,0), goal at (3,3).
    Actions: 0=up, 1=right, 2=down, 3=left
    Reward: -1 per step (encourages finding shortest path), +10 at goal.
    """
    
    def __init__(self, size=4, gamma=0.9):
        self.size = size
        self.gamma = gamma
        self.n_states = size * size
        self.n_actions = 4
        self.goal = (size-1, size-1)
        self.actions = [(-1,0), (0,1), (1,0), (0,-1)]  # up, right, down, left
    
    def state_to_pos(self, s):
        return (s // self.size, s % self.size)
    
    def pos_to_state(self, r, c):
        return r * self.size + c
    
    def step(self, state, action):
        r, c = self.state_to_pos(state)
        dr, dc = self.actions[action]
        nr, nc = max(0, min(self.size-1, r+dr)), max(0, min(self.size-1, c+dc))
        next_state = self.pos_to_state(nr, nc)
        
        if (nr, nc) == self.goal:
            return next_state, 10.0, True
        return next_state, -1.0, False

env = GridWorldMDP()
print(f"States: {env.n_states}, Actions: {env.n_actions}, γ: {env.gamma}")

Policy and Value Functions

Policy — What to Do in Each State

A policy π maps states to actions (or distributions over actions). A deterministic policy π(s) gives one action per state. A stochastic policy π(a|s) gives a probability distribution over actions for each state. The goal of RL is to find the optimal policy π* that maximizes expected cumulative discounted reward.

Value Functions — How Good Is a State?

The state-value function V^π(s) = E[Σₜ γᵗ rₜ | s₀ = s, π] is the expected total discounted reward starting from state s and following policy π. The action-value function Q^π(s, a) = E[Σₜ γᵗ rₜ | s₀ = s, a₀ = a, π] includes the initial action choice. The advantage function A^π(s, a) = Q^π(s, a) - V^π(s) tells you how much better action a is compared to the average under π.

Bellman Equations — The Recursive Structure

The Bellman expectation equation says: the value of a state equals the immediate reward plus the discounted value of the next state. For V^π:

V^π(s) = Σₐ π(a|s) [R(s,a) + γ Σₛ' P(s'|s,a) V^π(s')]

The Bellman optimality equation replaces the policy average with a max:

V*(s) = maxₐ [R(s,a) + γ Σₛ' P(s'|s,a) V*(s')]

These recursive equations are the foundation of every RL algorithm. Dynamic programming solves them exactly. Model-free methods approximate them from samples.

Dynamic Programming Methods

Policy Iteration and Value Iteration

Policy Iteration alternates two steps: (1) policy evaluation — compute V^π for the current policy by solving the Bellman equation, and (2) policy improvement — update the policy greedily with respect to V^π. This converges to the optimal policy in a finite number of iterations.

Value Iteration combines both steps into one update: V(s) ← maxₐ [R(s,a) + γ Σₛ' P(s'|s,a) V(s')]. It converges to V* and the optimal policy can be extracted greedily. Both require full knowledge of the MDP (P and R), which is the main limitation.

def value_iteration(env, theta=1e-6):
    """Value Iteration: find optimal value function and policy.
    
    Requires full MDP model (transitions and rewards).
    Convergence guaranteed by contraction mapping theorem.
    """
    V = np.zeros(env.n_states)
    
    iteration = 0
    while True:
        delta = 0
        for s in range(env.n_states):
            if env.state_to_pos(s) == env.goal:
                continue
            
            v = V[s]
            # Bellman optimality update
            values = []
            for a in range(env.n_actions):
                s_next, reward, done = env.step(s, a)
                q = reward + (0 if done else env.gamma * V[s_next])
                values.append(q)
            V[s] = max(values)
            delta = max(delta, abs(v - V[s]))
        
        iteration += 1
        if delta < theta:
            break
    
    # Extract policy from V*
    policy = np.zeros(env.n_states, dtype=int)
    for s in range(env.n_states):
        q_values = []
        for a in range(env.n_actions):
            s_next, reward, done = env.step(s, a)
            q = reward + (0 if done else env.gamma * V[s_next])
            q_values.append(q)
        policy[s] = np.argmax(q_values)
    
    print(f"Converged in {iteration} iterations")
    
    # Display value function as grid
    V_grid = V.reshape(env.size, env.size)
    print("\nOptimal Value Function:")
    for row in V_grid:
        print(" ".join(f"{v:7.2f}" for v in row))
    
    # Display policy
    arrows = ['↑', '→', '↓', '←']
    print("\nOptimal Policy:")
    for r in range(env.size):
        row = ""
        for c in range(env.size):
            s = env.pos_to_state(r, c)
            if (r, c) == env.goal:
                row += "  ★  "
            else:
                row += f"  {arrows[policy[s]]}  "
        print(row)
    
    return V, policy

env = GridWorldMDP(size=4, gamma=0.9)
V_star, pi_star = value_iteration(env)

Model-Free Methods

Monte Carlo Methods

Monte Carlo (MC) methods learn from complete episodes. Run the policy, collect the trajectory, then update value estimates using the actual returns. No model needed. First-visit MC updates V(s) using the return from the first time s is visited in an episode. Every-visit MC uses every visit. MC has zero bias (it uses actual returns) but high variance (returns are noisy).

Temporal Difference Learning — The Middle Ground

TD learning combines the model-free nature of MC with the bootstrapping idea of DP. Instead of waiting for the episode to end, TD(0) updates after every step using:

V(s) ← V(s) + α [r + γ V(s') - V(s)]

The term r + γ V(s') is the TD target, and r + γ V(s') - V(s) is the TD error. TD bootstraps — it uses the current estimate V(s') as a stand-in for the true future value. This introduces bias but reduces variance compared to MC. The bias shrinks as V gets more accurate.

Q-Learning — Off-Policy TD Control

Q-learning is the most important model-free RL algorithm. It learns the optimal Q-function directly, regardless of what policy the agent follows (this is called off-policy learning). The update rule is:

Q(s, a) ← Q(s, a) + α [r + γ max_a' Q(s', a') - Q(s, a)]

The max over next actions is the key — it learns from the best possible action, even if the agent took a different one for exploration.

class QLearning:
    """Tabular Q-Learning with epsilon-greedy exploration."""
    
    def __init__(self, n_states, n_actions, alpha=0.1, gamma=0.9, epsilon=0.1):
        self.Q = np.zeros((n_states, n_actions))
        self.alpha = alpha
        self.gamma = gamma
        self.epsilon = epsilon
    
    def select_action(self, state):
        if np.random.rand() < self.epsilon:
            return np.random.randint(self.Q.shape[1])
        return np.argmax(self.Q[state])
    
    def update(self, s, a, r, s_next, done):
        """Q-learning update: uses max Q(s', a') regardless of actual next action."""
        target = r + (0 if done else self.gamma * np.max(self.Q[s_next]))
        self.Q[s, a] += self.alpha * (target - self.Q[s, a])

def train_qlearning(env, episodes=2000):
    agent = QLearning(env.n_states, env.n_actions)
    
    rewards_per_episode = []
    for ep in range(episodes):
        state = 0  # start state
        total_reward = 0
        
        for step in range(100):
            action = agent.select_action(state)
            next_state, reward, done = env.step(state, action)
            agent.update(state, action, reward, next_state, done)
            total_reward += reward
            state = next_state
            if done:
                break
        
        rewards_per_episode.append(total_reward)
    
    # Show learning progress
    for window_end in [100, 500, 1000, 2000]:
        window_start = max(0, window_end - 100)
        avg = np.mean(rewards_per_episode[window_start:window_end])
        print(f"Episodes {window_start}-{window_end}: avg reward = {avg:.2f}")
    
    return agent

env = GridWorldMDP()
agent = train_qlearning(env)

# Show learned policy
arrows = ['↑', '→', '↓', '←']
print("\nLearned Policy:")
for r in range(env.size):
    row = ""
    for c in range(env.size):
        s = env.pos_to_state(r, c)
        if (r, c) == env.goal:
            row += " ★ "
        else:
            row += f" {arrows[np.argmax(agent.Q[s])]} "
    print(row)

SARSA — On-Policy TD Control

SARSA (State-Action-Reward-State-Action) is Q-learning's on-policy cousin. Instead of using max Q(s', a'), it uses the Q-value of the action the agent actually takes:

Q(s, a) ← Q(s, a) + α [r + γ Q(s', a') - Q(s, a)]

This matters when exploration is risky. Q-learning learns the optimal policy assuming perfect exploitation, but SARSA learns the best policy given the actual exploration strategy. Near a cliff edge with ε-greedy exploration, SARSA learns to take a safe path (accounting for random moves), while Q-learning learns the optimal path right along the edge (ignoring that random exploration might push it off).

Deep Q-Networks (DQN)

From Tables to Neural Networks

Tabular Q-learning fails when the state space is huge (Atari games have ~10^67 possible frames). Deep Q-Networks (DQN) replace the Q-table with a neural network that takes a state as input and outputs Q-values for all actions. Two innovations made this work:

Experience Replay: Store transitions (s, a, r, s') in a buffer. Train on random mini-batches from the buffer. This breaks correlation between consecutive samples and reuses data efficiently.

Target Network: Use a separate, slowly-updated copy of the Q-network for computing targets. This stabilizes training by preventing the target from shifting every update.

class DQN_Concept:
    """Conceptual DQN implementation (pseudocode-level).
    
    Key insight: Q(s,a) ≈ neural_network(s)[a]
    """
    
    def __init__(self, state_dim, n_actions, buffer_size=10000):
        self.state_dim = state_dim
        self.n_actions = n_actions
        
        # In practice, these would be PyTorch/TF models
        self.q_network = f"Neural net: {state_dim} → {n_actions}"
        self.target_network = f"Copy of Q-network (updated slowly)"
        
        # Experience replay buffer
        self.buffer = []
        self.buffer_size = buffer_size
    
    def training_step_pseudocode(self):
        """The DQN training loop."""
        return """
        1. Observe state s
        2. Select action: a = argmax Q(s, ·) with ε-greedy
        3. Execute a, observe r, s'
        4. Store (s, a, r, s') in replay buffer
        5. Sample random mini-batch from buffer
        6. For each (sᵢ, aᵢ, rᵢ, s'ᵢ):
           target = rᵢ + γ * max_a' Q_target(s'ᵢ, a')
           loss = (Q(sᵢ, aᵢ) - target)²
        7. Gradient step on Q-network
        8. Every C steps: Q_target ← Q  (hard update)
        """

    def extensions(self):
        return {
            "Double DQN": "Use Q-network to SELECT action, target network to EVALUATE. "
                         "Fixes overestimation bias: target = r + γ Q_target(s', argmax_a' Q(s', a'))",
            "Dueling DQN": "Split network into V(s) and A(s,a) streams. "
                          "Q(s,a) = V(s) + A(s,a) - mean(A). Better for states where action doesn't matter.",
            "Prioritized Replay": "Sample transitions proportional to their TD error. "
                                  "Learn more from surprising experiences.",
            "Rainbow": "Combine all improvements: Double + Dueling + Prioritized + "
                      "n-step returns + distributional + noisy nets."
        }

The Deadly Triad

Combining function approximation + bootstrapping + off-policy learning creates instability. This is the deadly triad of RL. DQN has all three, which is why it needs experience replay and target networks to work at all. SARSA with function approximation avoids the off-policy component. Algorithms like CQL and conservative approaches explicitly address this instability.

Policy Gradient Methods

Why Optimize the Policy Directly?

Value-based methods (Q-learning, DQN) learn a value function and derive a policy from it. But some problems are better solved by optimizing the policy directly. Continuous action spaces (robotic control with precise torques) are hard for Q-learning (you would need to maximize Q over a continuous action space at every step). Stochastic policies are natural for some problems. Policy gradient methods handle both.

REINFORCE — The Score Function Estimator

The REINFORCE algorithm uses the log-derivative trick (score function): ∇J(θ) = E[Σₜ ∇log π_θ(aₜ|sₜ) · Gₜ], where Gₜ is the return from time t. The gradient pushes the policy to increase the probability of actions that led to high returns and decrease those that led to low returns.

import numpy as np

class REINFORCE_Concept:
    """REINFORCE policy gradient algorithm."""
    
    def __init__(self, state_dim, n_actions, lr=0.01):
        # Simple linear softmax policy: π(a|s) = softmax(θᵀs)
        self.theta = np.zeros((state_dim, n_actions))
        self.lr = lr
    
    def policy(self, state):
        """Softmax policy."""
        logits = state @ self.theta
        logits -= logits.max()  # numerical stability
        exp_logits = np.exp(logits)
        return exp_logits / exp_logits.sum()
    
    def select_action(self, state):
        probs = self.policy(state)
        return np.random.choice(len(probs), p=probs)
    
    def update(self, episode):
        """Update after complete episode using REINFORCE.
        
        ∇J ≈ Σ_t ∇log π(aₜ|sₜ) * Gₜ
        where Gₜ = Σ_{k=t}^{T} γ^{k-t} rₖ
        """
        states, actions, rewards = zip(*episode)
        T = len(rewards)
        gamma = 0.99
        
        # Compute returns (reward-to-go)
        returns = np.zeros(T)
        G = 0
        for t in reversed(range(T)):
            G = rewards[t] + gamma * G
            returns[t] = G
        
        # Normalize returns (variance reduction)
        returns = (returns - returns.mean()) / (returns.std() + 1e-8)
        
        # Policy gradient update
        for t in range(T):
            state = np.array(states[t])
            action = actions[t]
            probs = self.policy(state)
            
            # ∇log π(a|s) for softmax = x * (1_{a} - π)
            grad_log = np.outer(state, -probs)
            grad_log[:, action] += state
            
            self.theta += self.lr * grad_log * returns[t]

print("REINFORCE: Simple but high variance.")
print("Variance reduction techniques:")
print("  1. Baseline subtraction: use Gₜ - V(sₜ) instead of Gₜ")
print("  2. Reward-to-go: only use future returns, not past rewards")
print("  3. Normalize returns: zero mean, unit variance")

Actor-Critic Methods

Actor-Critic methods combine policy gradient (the actor) with value function estimation (the critic). The actor decides what to do. The critic evaluates how good the action was by estimating V(s) or Q(s,a). The advantage A(s,a) = Q(s,a) - V(s) replaces the return in the policy gradient, dramatically reducing variance.

PPO — The Industry Workhorse

Proximal Policy Optimization (PPO) is the most widely used policy gradient algorithm in practice. It prevents destructively large policy updates using a clipped objective:

L(θ) = min(rₜ(θ) Âₜ, clip(rₜ(θ), 1-ε, 1+ε) Âₜ)

where rₜ(θ) = π_θ(aₜ|sₜ) / π_old(aₜ|sₜ) is the probability ratio and Âₜ is the advantage estimate. The clipping prevents the policy from changing too much in one step — a simple but effective trust region. PPO is what trained ChatGPT (via RLHF), robotics policies at OpenAI, and game-playing agents at DeepMind.

def ppo_update_pseudocode():
    """PPO training step (conceptual)."""
    return """
    For each iteration:
      1. Collect trajectories using current policy π_old
      2. Compute advantages Â using GAE(λ):
         Â_t = Σ_{l=0}^{∞} (γλ)^l δ_{t+l}
         where δ_t = r_t + γ V(s_{t+1}) - V(s_t)
      3. For K epochs over the collected data:
         ratio = π_θ(a|s) / π_old(a|s)
         L_clip = min(ratio * Â, clip(ratio, 1-ε, 1+ε) * Â)
         L_value = (V_θ(s) - V_target)²
         L_entropy = -Σ π_θ(a|s) log π_θ(a|s)  # encourage exploration
         
         loss = -L_clip + c₁ * L_value - c₂ * L_entropy
         Update θ with gradient descent on loss
    """

Model-Based Reinforcement Learning

Learning a Model of the World

Model-based RL learns the transition dynamics P(s'|s, a) and reward function R(s, a), then uses this model for planning. The advantage is sample efficiency — you can simulate millions of trajectories in the learned model without interacting with the real environment. The risk is model error: if your learned model is wrong, the policy will exploit those errors.

Dyna-Q and Model Predictive Control

Dyna-Q combines model-free and model-based learning. After each real experience, it performs Q-learning updates both on the real transition and on several simulated transitions from the learned model. This gets you the best of both worlds — real experience grounds the learning, while simulated experience accelerates it.

Model Predictive Control (MPC) uses the learned model to plan ahead at each step. You simulate many possible action sequences, evaluate their outcomes under the model, and execute the first action of the best sequence. Then you re-plan at the next step. MPC is popular in robotics because it naturally handles constraints and changing objectives.

AlphaZero and MuZero — Planning with Neural Networks

AlphaZero combines Monte Carlo Tree Search (MCTS) with a neural network that provides a policy prior and value estimate. The MCTS uses these to focus its search on promising moves. The network is trained from self-play — no human games needed. MuZero goes further: it learns the model itself. It does not predict pixels or exact states; it learns a latent model sufficient for planning. MuZero matched AlphaZero on Go, chess, and shogi without knowing the rules, and also mastered Atari games.

Offline RL, Imitation Learning, and Multi-Agent RL

Offline (Batch) RL

Offline RL learns a policy from a fixed dataset of past interactions — no new environment interaction allowed. This is critical for domains where exploration is dangerous or expensive (healthcare, autonomous driving). The challenge is distribution shift: the learned policy wants to take actions not well-represented in the dataset. Conservative Q-Learning (CQL) addresses this by penalizing Q-values for unseen state-action pairs, implementing a pessimism principle. The Decision Transformer reframes RL as sequence modeling — conditioning a transformer on desired returns to generate actions.

Imitation Learning

Behavioral Cloning learns a policy from expert demonstrations using supervised learning: state → action. It is fast but suffers from compounding errors — small mistakes lead to states the expert never visited, where the policy has no guidance. DAgger (Dataset Aggregation) fixes this by iteratively collecting expert labels for states the learned policy actually visits. GAIL (Generative Adversarial Imitation Learning) uses a discriminator to distinguish expert from learned behavior, avoiding the need for an explicit reward function.

Multi-Agent RL

When multiple agents interact, the environment becomes non-stationary from each agent's perspective (other agents are learning too). Cooperative settings (team games) require agents to develop communication protocols and coordinate strategies. Competitive settings (zero-sum games) connect to game theory and Nash equilibria. Emergent behaviors — strategies that no one designed — are one of the most fascinating outcomes. OpenAI Five learned team coordination in Dota 2 that surprised professional players.

The RL Landscape in 2024

RL's biggest impact has been through RLHF (Reinforcement Learning from Human Feedback), which fine-tunes large language models to be helpful and harmless. PPO is the algorithm behind ChatGPT's alignment. Robotics RL is accelerating thanks to sim-to-real transfer — train in simulation, deploy on real robots. Offline RL is gaining traction in healthcare and industrial control where online exploration is impossible. The field is moving from "RL for games" toward "RL for real-world decision-making," and the transition is happening faster than most people expected.
