Online Learning

Chapter 17: Learning Theory & Advanced ML 4 subtopics
TL;DR

Online learning flips the standard ML script: instead of training on a batch and deploying, you learn one example at a time, making predictions before seeing each label. No assumptions about where data comes from — it could even be adversarial. Performance is measured by regret: how much worse you did compared to the best fixed strategy in hindsight. Algorithms like multiplicative weights and online gradient descent achieve sublinear regret, meaning they converge to optimal performance over time. This framework powers ad bidding, spam filters, recommendation engines, and any system that must adapt in real time.

Imagine you run a spam filter. Emails arrive one at a time. For each email, you predict "spam" or "not spam," then the user tells you if you were right. You update your model and move to the next email. You do not get to retrain from scratch on all past emails — you have to adapt on the fly. Worse, spammers are actively trying to fool you, so the data distribution is shifting beneath your feet.

This is online learning. No i.i.d. assumption. No fixed training set. No luxury of multiple passes. You learn as you go, and the goal is to make as few mistakes as possible along the way. It sounds like a harder problem than batch learning, and it is — but the algorithms that emerge are elegant, practical, and surprisingly well-understood theoretically.

Framework

Sequential Prediction

The online learning protocol works in rounds. At each round t = 1, 2, ..., T:

  1. The environment presents an input xₜ
  2. The learner makes a prediction ŷₜ
  3. The environment reveals the true label yₜ (or a loss ℓₜ)
  4. The learner incurs a loss and may update its model

The critical difference from batch learning: the learner must commit to a prediction before seeing the label. There is no validation set, no cross-validation, no training/test split. You are evaluated in real time.

The Adversarial Setting

The adversarial setting is what makes online learning powerful. We make zero statistical assumptions about the data. An adversary could be generating the sequence specifically to trip you up. Despite this, we can still guarantee good performance. That guarantee is called regret.

Regret — The Right Performance Measure

Regret is the difference between your cumulative loss and the cumulative loss of the best fixed strategy in hindsight. Let L_learner be your total loss over T rounds, and L_best be the loss of the best fixed hypothesis you could have chosen if you had seen all the data in advance. Then:

Regret(T) = L_learner - L_best

We want sublinear regret: Regret(T) = o(T). This means the average per-round regret goes to zero. In other words, you eventually perform as well as the best fixed strategy, even though you had to learn online.

import numpy as np

def compute_regret(learner_losses, best_fixed_loss_per_round):
    """Compute regret over time.
    
    Regret(T) = Σ_{t=1}^{T} ℓ_learner(t) - Σ_{t=1}^{T} ℓ_best(t)
    
    Sublinear regret means Regret(T)/T → 0 as T → ∞.
    """
    cumulative_learner = np.cumsum(learner_losses)
    cumulative_best = np.cumsum(best_fixed_loss_per_round)
    regret = cumulative_learner - cumulative_best
    
    T = len(regret)
    avg_regret = regret / np.arange(1, T + 1)
    
    print(f"Total regret after {T} rounds: {regret[-1]:.2f}")
    print(f"Average regret: {avg_regret[-1]:.4f}")
    print(f"Regret growth rate: {'sublinear ✓' if avg_regret[-1] < avg_regret[T//2] else 'linear ✗'}")
    return regret

# Example: random learner vs best fixed strategy
np.random.seed(42)
T = 10000
# Adversary generates losses for 3 actions
losses = np.random.rand(T, 3)
# Best fixed action in hindsight
best_action = np.argmin(losses.sum(axis=0))
best_losses = losses[:, best_action]
# Random learner picks uniformly at random
random_actions = np.random.randint(0, 3, T)
random_losses = losses[np.arange(T), random_actions]

compute_regret(random_losses, best_losses)

Static vs Dynamic Regret

Static regret compares against the single best fixed action over all rounds — the definition above. Dynamic regret compares against a sequence of actions that can change over time. Dynamic regret is much harder to achieve, but more realistic when the environment is non-stationary. Most foundational results focus on static regret because it is achievable against adversaries without additional assumptions.

Algorithms

The Perceptron — Mistake Bound Learning

The Perceptron is the simplest online learning algorithm. For linearly separable data, it makes at most (R/γ)² mistakes, where R is the radius of the data and γ is the margin. This is a mistake bound, not a regret bound, but it is the oldest result in online learning (Rosenblatt, 1958; Novikoff, 1962).

import numpy as np

class OnlinePerceptron:
    """Online perceptron with mistake bound guarantee."""
    
    def __init__(self, d):
        self.w = np.zeros(d)
        self.mistakes = 0
    
    def predict(self, x):
        return 1 if self.w @ x >= 0 else -1
    
    def update(self, x, y):
        """Update only on mistakes."""
        y_hat = self.predict(x)
        if y_hat != y:
            self.w += y * x
            self.mistakes += 1
        return y_hat

# Demo: online learning on linearly separable data
np.random.seed(42)
d = 10
w_true = np.random.randn(d)
w_true /= np.linalg.norm(w_true)

T = 1000
perceptron = OnlinePerceptron(d)
mistakes_over_time = []

for t in range(T):
    x = np.random.randn(d)
    y = 1 if w_true @ x >= 0 else -1
    perceptron.update(x, y)
    mistakes_over_time.append(perceptron.mistakes)

print(f"Total mistakes: {perceptron.mistakes} out of {T}")
print(f"Mistake rate: {perceptron.mistakes/T:.3f}")
# Mistakes plateau quickly — most happen early during learning

Winnow — Multiplicative Updates for Sparse Problems

Winnow is the perceptron's multiplicative cousin. Instead of adding to the weights, Winnow multiplies them by factors of 2 (double on relevant features, halve on irrelevant ones). This makes it much better when only a few features matter. Its mistake bound is O(r log d) where r is the number of relevant features and d is the total dimension — logarithmic in d rather than polynomial. When you have millions of features but only dozens matter, Winnow crushes the perceptron.

Online Gradient Descent (OGD)

Online Gradient Descent generalizes these to convex loss functions. At each round, you predict with your current parameters, observe the loss, and take a gradient step. The simplicity is deceptive — this algorithm achieves O(√T) regret for convex losses, which is optimal.

import numpy as np

class OnlineGradientDescent:
    """Online gradient descent with O(sqrt(T)) regret guarantee."""
    
    def __init__(self, d, lr_schedule='sqrt'):
        self.w = np.zeros(d)
        self.t = 0
        self.lr_schedule = lr_schedule
    
    def learning_rate(self):
        """Decreasing learning rate: η_t = 1/sqrt(t)"""
        if self.lr_schedule == 'sqrt':
            return 1.0 / np.sqrt(self.t + 1)
        return 0.01
    
    def predict(self, x):
        return self.w @ x
    
    def update(self, x, y):
        """Squared loss: ℓ(ŷ, y) = (ŷ - y)², gradient = 2(ŷ - y)x"""
        y_hat = self.predict(x)
        loss = (y_hat - y) ** 2
        grad = 2 * (y_hat - y) * x
        
        eta = self.learning_rate()
        self.w -= eta * grad
        self.t += 1
        return loss

# Simulate online regression
np.random.seed(42)
d, T = 5, 5000
w_true = np.array([1.0, -0.5, 0.3, 0.0, 0.8])

ogd = OnlineGradientDescent(d)
cumulative_loss = 0
losses = []

for t in range(T):
    x = np.random.randn(d)
    y = w_true @ x + 0.1 * np.random.randn()
    loss = ogd.update(x, y)
    cumulative_loss += loss
    losses.append(loss)

# Show average loss decreasing over time
for chunk_end in [100, 500, 1000, 5000]:
    chunk_start = max(0, chunk_end - 100)
    avg = np.mean(losses[chunk_start:chunk_end])
    print(f"Rounds {chunk_start}-{chunk_end}: avg loss = {avg:.4f}")

Follow the Regularized Leader (FTRL)

FTRL takes a different philosophy. Instead of gradient steps, at each round you pick the hypothesis that minimizes total past loss plus a regularization term. The regularizer prevents wild swings between rounds. With L2 regularization, FTRL is equivalent to online gradient descent. With L1 regularization, FTRL produces sparse solutions — and this is exactly the algorithm behind large-scale ad-click prediction at Google and Facebook. When you have billions of features (one per user-ad pair), sparsity is not a luxury, it is a necessity.

class FTRL_L1:
    """Follow the Regularized Leader with L1 regularization.
    
    Used in production at Google for ad CTR prediction.
    Produces sparse weights (most weights exactly zero).
    """
    
    def __init__(self, d, alpha=1.0, beta=1.0, lambda1=1.0):
        self.z = np.zeros(d)  # accumulated gradient info
        self.n = np.zeros(d)  # per-coordinate learning rate info
        self.alpha = alpha
        self.beta = beta
        self.lambda1 = lambda1
    
    def predict(self, x):
        # Compute weights on-the-fly (lazy representation)
        w = np.zeros_like(self.z)
        for i in range(len(w)):
            if abs(self.z[i]) <= self.lambda1:
                w[i] = 0.0  # L1 thresholding → sparsity!
            else:
                sign = -1 if self.z[i] < 0 else 1
                lr_i = (self.beta + np.sqrt(self.n[i])) / self.alpha
                w[i] = -(self.z[i] - sign * self.lambda1) / (lr_i + self.lambda1)
        return w @ x, w
    
    def update(self, x, y):
        y_hat, w = self.predict(x)
        grad = (y_hat - y) * x  # gradient of squared loss
        
        # Per-coordinate learning rate update
        sigma = (np.sqrt(self.n + grad**2) - np.sqrt(self.n)) / self.alpha
        self.z += grad - sigma * w
        self.n += grad**2
        
        sparsity = np.mean(w == 0)
        return (y_hat - y)**2, sparsity

# Demo: FTRL finds sparse solution
np.random.seed(42)
d = 100  # 100 features
w_true = np.zeros(d)
w_true[:5] = [1.0, -0.5, 0.3, 0.8, -0.2]  # only 5 matter

ftrl = FTRL_L1(d, alpha=0.5, lambda1=0.1)
for t in range(5000):
    x = np.random.randn(d)
    y = w_true @ x + 0.1 * np.random.randn()
    loss, sparsity = ftrl.update(x, y)

_, w_learned = ftrl.predict(np.zeros(d))
nonzero = np.sum(w_learned != 0)
print(f"True nonzero features: 5")
print(f"Learned nonzero features: {nonzero}")
print(f"Weight sparsity: {(1 - nonzero/d)*100:.1f}%")

Online Mirror Descent

Online Mirror Descent generalizes OGD by using a different geometry for updates. Instead of Euclidean distance (L2 norm), you can use KL divergence or other Bregman divergences. The choice of geometry should match the problem structure. For probability distributions (like expert weights), KL divergence is natural — and this leads directly to the multiplicative weights algorithm. For sparse problems, unnormalized negative entropy works well. The abstraction unifies many seemingly different algorithms under one roof.

Expert Advice

The Weighted Majority Algorithm

You have N experts giving you advice each day. Each expert says "buy" or "sell." At the end of the day, you see who was right. How should you combine their advice? The Weighted Majority Algorithm is beautifully simple: maintain a weight for each expert. Each round, predict the weighted majority vote. After seeing the outcome, multiply each wrong expert's weight by (1 - ε). Experts who are frequently wrong get exponentially downweighted.

import numpy as np

class WeightedMajority:
    """Weighted Majority Algorithm for expert advice.
    
    Regret bound: mistakes ≤ (1+ε) * best_expert + (ln N)/ε
    """
    
    def __init__(self, n_experts, epsilon=0.1):
        self.n = n_experts
        self.eps = epsilon
        self.weights = np.ones(n_experts)
    
    def predict(self, expert_predictions):
        """Weighted majority vote."""
        weighted = self.weights @ expert_predictions
        return 1 if weighted >= 0.5 * self.weights.sum() else 0
    
    def update(self, expert_predictions, true_label):
        """Penalize wrong experts."""
        wrong = (expert_predictions != true_label)
        self.weights[wrong] *= (1 - self.eps)
        return wrong

# Demo: 10 experts, one is consistently good
np.random.seed(42)
N, T = 10, 1000
wm = WeightedMajority(N, epsilon=0.3)

expert_mistakes = np.zeros(N)
learner_mistakes = 0

for t in range(T):
    true_label = np.random.randint(0, 2)
    
    # Expert 0 is good (90% correct), others are noisy (55% correct)
    preds = np.zeros(N, dtype=int)
    preds[0] = true_label if np.random.rand() < 0.9 else 1 - true_label
    for i in range(1, N):
        preds[i] = true_label if np.random.rand() < 0.55 else 1 - true_label
    
    pred = wm.predict(preds)
    if pred != true_label:
        learner_mistakes += 1
    wm.update(preds, true_label)
    expert_mistakes += (preds != true_label)

best_expert_idx = np.argmin(expert_mistakes)
print(f"Best expert (#{best_expert_idx}): {int(expert_mistakes[best_expert_idx])} mistakes")
print(f"Weighted Majority: {learner_mistakes} mistakes")
print(f"Worst expert: {int(expert_mistakes.max())} mistakes")
print(f"\nFinal weight ratios (normalized):")
w_norm = wm.weights / wm.weights.sum()
print(f"  Expert 0 (good): {w_norm[0]:.4f}")
print(f"  Others (avg):    {w_norm[1:].mean():.4f}")

The Hedge Algorithm

The Hedge algorithm (also called Multiplicative Weights Update) is the continuous-loss version. Instead of binary right/wrong, each expert incurs a real-valued loss. Weights are updated as wᵢ ← wᵢ · exp(-η · lossᵢ), where η is the learning rate. The regret bound is O(√(T log N)) — sublinear in T and logarithmic in the number of experts. This means you can have millions of "experts" (think: all possible simple strategies) and still converge quickly.

import numpy as np

class Hedge:
    """Hedge (Multiplicative Weights Update) algorithm.
    
    Regret bound: O(sqrt(T * ln N))
    """
    
    def __init__(self, n_experts, eta=None, T=None):
        self.n = n_experts
        self.weights = np.ones(n_experts) / n_experts
        # Optimal η = sqrt(ln(N)/T) if T is known
        self.eta = eta or np.sqrt(np.log(n_experts) / (T or 1000))
    
    def get_distribution(self):
        """Return probability distribution over experts."""
        return self.weights / self.weights.sum()
    
    def predict(self, expert_predictions):
        """Predict using weighted combination."""
        p = self.get_distribution()
        return p @ expert_predictions
    
    def update(self, losses):
        """Multiplicative weight update."""
        self.weights *= np.exp(-self.eta * losses)
        self.weights /= self.weights.sum()  # normalize

# Demo: Hedge quickly identifies the best expert
np.random.seed(42)
N, T = 20, 2000
hedge = Hedge(N, T=T)

cumulative_losses = np.zeros(N)
hedge_cumulative = 0

for t in range(T):
    # Expert losses: expert 3 has expected loss 0.2, others ~0.5
    losses = np.random.rand(N) * 0.6 + 0.2  # base losses
    losses[3] *= 0.4  # expert 3 is much better
    
    p = hedge.get_distribution()
    hedge_loss = p @ losses
    hedge_cumulative += hedge_loss
    cumulative_losses += losses
    hedge.update(losses)

best = np.argmin(cumulative_losses)
print(f"Best expert (#{best}): cumulative loss = {cumulative_losses[best]:.1f}")
print(f"Hedge: cumulative loss = {hedge_cumulative:.1f}")
print(f"Regret: {hedge_cumulative - cumulative_losses[best]:.1f}")
print(f"Theoretical bound: O(sqrt(T*ln(N))) = {np.sqrt(T * np.log(N)):.1f}")
print(f"\nFinal probability on expert 3: {hedge.get_distribution()[3]:.4f}")
Multiplicative Weights Are Everywhere

The multiplicative weights framework is one of the most versatile algorithmic ideas in computer science. Beyond online learning, it appears in game theory (computing equilibria), combinatorial optimization (solving LPs approximately), boosting (AdaBoost is multiplicative weights applied to training examples), and even in zero-sum game solvers. Freund and Schapire showed that AdaBoost is equivalent to a game between a booster and a weak learner, solved by multiplicative weights. If you learn one algorithm from online learning, make it this one.

Online Convex Optimization

The General Framework

Online Convex Optimization (OCO) is the modern, general framework that unifies all the algorithms above. At each round, the learner picks a point wₜ from a convex set. The adversary reveals a convex loss function ℓₜ. The learner pays ℓₜ(wₜ). The goal is to minimize regret relative to the best fixed point in hindsight.

Regret Bounds by Loss Geometry

The regret bounds depend on the geometry of the loss functions:

"""Regret bounds for different loss types in OCO."""

# General convex losses + OGD with η_t = 1/sqrt(t):
# Regret = O(sqrt(T))  — you can't do better in general

# Strongly convex losses (like ridge regression) + OGD with η_t = 1/(λt):
# Regret = O(log(T))   — much better! Logarithmic regret.

# Exp-concave losses (like log-loss) + ONS:
# Regret = O(d * log(T))  — logarithmic, but depends on dimension

import numpy as np

def compare_regret_bounds(T_values):
    """Compare regret growth rates for different loss types."""
    print(f"{'T':>8s} | {'sqrt(T)':>10s} | {'log(T)':>10s} | {'Ratio':>8s}")
    print("-" * 45)
    for T in T_values:
        sqrt_regret = np.sqrt(T)
        log_regret = np.log(T)
        print(f"{T:>8d} | {sqrt_regret:>10.1f} | {log_regret:>10.2f} | {sqrt_regret/log_regret:>8.1f}x")

compare_regret_bounds([100, 1000, 10000, 100000, 1000000])
# sqrt(T) regret: 10, 31.6, 100, 316, 1000
# log(T) regret:  4.6, 6.9, 9.2, 11.5, 13.8
# Strong convexity gives MASSIVE regret reduction at scale

AdaGrad — Adaptive Learning Rates from OCO

AdaGrad (Adaptive Gradient) is a direct product of OCO theory. It maintains per-coordinate learning rates that adapt based on the sum of squared past gradients. Features that appear rarely get larger learning rates; frequent features get smaller rates. The regret bound adapts to the actual gradient sequence — if most gradients are sparse, AdaGrad achieves much better regret than the worst-case bound.

import numpy as np

class AdaGrad:
    """AdaGrad: adaptive per-coordinate learning rates.
    
    η_{t,i} = η / sqrt(Σ_{s=1}^{t} g_{s,i}²)
    
    Achieves data-dependent regret that adapts to gradient sparsity.
    """
    
    def __init__(self, d, eta=1.0):
        self.w = np.zeros(d)
        self.G = np.zeros(d)  # sum of squared gradients
        self.eta = eta
    
    def update(self, grad):
        self.G += grad ** 2
        adaptive_lr = self.eta / (np.sqrt(self.G) + 1e-8)
        self.w -= adaptive_lr * grad
        return self.w.copy()

# Compare OGD vs AdaGrad on sparse gradients
np.random.seed(42)
d, T = 100, 5000
w_star = np.zeros(d)
w_star[:3] = [2.0, -1.0, 0.5]  # sparse true weights

ogd_w = np.zeros(d)
adagrad = AdaGrad(d, eta=0.5)
ogd_losses, ada_losses = [], []

for t in range(T):
    x = np.zeros(d)
    # Sparse inputs: only ~3 coordinates active each round
    active = np.random.choice(d, 3, replace=False)
    x[active] = np.random.randn(3)
    
    y = w_star @ x
    
    # OGD
    pred_ogd = ogd_w @ x
    grad_ogd = (pred_ogd - y) * x
    ogd_w -= (1.0 / np.sqrt(t+1)) * grad_ogd
    ogd_losses.append((pred_ogd - y)**2)
    
    # AdaGrad
    pred_ada = adagrad.w @ x
    grad_ada = (pred_ada - y) * x
    adagrad.update(grad_ada)
    ada_losses.append((pred_ada - y)**2)

# Compare convergence
for t_check in [500, 1000, 2000, 5000]:
    window = slice(max(0, t_check-100), t_check)
    print(f"t={t_check}: OGD avg loss={np.mean(ogd_losses[window]):.4f}, "
          f"AdaGrad avg loss={np.mean(ada_losses[window]):.4f}")
Online Learning → Deep Learning Optimizers

The entire lineage of deep learning optimizers traces back to online convex optimization. SGD is online gradient descent. AdaGrad came from OCO regret bounds. RMSProp is a heuristic fix for AdaGrad's aggressive learning rate decay. Adam combines RMSProp momentum with AdaGrad's adaptivity. When you tune an Adam learning rate, you are implicitly navigating the regret landscape of online convex optimization. Understanding the theory helps you understand why certain optimizer hyperparameters matter.
