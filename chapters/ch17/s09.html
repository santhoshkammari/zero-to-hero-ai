Kernel and Information-Theoretic Methods

Chapter 17: Learning Theory & Advanced ML 2 subtopics
TL;DR

Kernel methods let you run linear algorithms in infinite-dimensional feature spaces without ever computing those features — the "kernel trick." The theory lives in Reproducing Kernel Hilbert Spaces (RKHS), and the Representer Theorem guarantees that the optimal solution can always be written in terms of the training data. Information-theoretic methods attack learning from a compression angle: the best model is the shortest description that explains the data (MDL), and the Information Bottleneck finds the optimal tradeoff between compressing inputs and preserving information about outputs. These frameworks provide deep theoretical grounding for regularization, model selection, and representation learning.

Kernel Methods (Theory)

Here's the motivation. You have data that isn't linearly separable. Two classes swirled together in 2D. No straight line can separate them. The classic fix: map the data to a higher-dimensional space where a linear separator exists. A circle in 2D becomes linearly separable in 3D if you add a feature x₁² + x₂². But computing these features explicitly gets expensive fast — polynomial features of degree d on p-dimensional data create O(p^d) features.

The kernel trick sidesteps this entirely. Instead of computing the high-dimensional feature vectors φ(x), you compute only their inner products k(x, x') = ⟨φ(x), φ(x')⟩. Many algorithms (SVMs, ridge regression, PCA) only need inner products, never the features themselves. If you have a function k that computes the inner product cheaply, you never need the explicit feature map. This works even when the feature space is infinite-dimensional.

Reproducing Kernel Hilbert Spaces (RKHS)

The mathematical foundation is the Reproducing Kernel Hilbert Space. An RKHS is a Hilbert space (complete inner product space) of functions where evaluation at any point is a bounded linear functional. Translation: the space has an inner product, and you can evaluate any function in the space at any input point without the operation "blowing up."

Every RKHS has a unique reproducing kernel — a function k(x, x') such that f(x) = ⟨f, k(x, ·)⟩ for every function f in the space. The kernel function at point x acts like a "basis function centered at x." The inner product between two kernel functions gives you the kernel: ⟨k(x, ·), k(x', ·)⟩ = k(x, x'). That's the reproducing property — the kernel reproduces itself under the inner product.

The Mercer theorem guarantees that any continuous, symmetric, positive semi-definite function is a valid kernel — it corresponds to an inner product in some feature space. Common kernels:

# Common kernel functions
import numpy as np

def linear_kernel(x, y):
    """k(x,y) = x·y — feature space = input space"""
    return x @ y

def polynomial_kernel(x, y, degree=3, c=1):
    """k(x,y) = (x·y + c)^d — feature space = all monomials up to degree d"""
    return (x @ y + c) ** degree

def rbf_kernel(x, y, gamma=1.0):
    """k(x,y) = exp(-γ‖x-y‖²) — feature space is infinite-dimensional!"""
    return np.exp(-gamma * np.sum((x - y)**2))

def laplacian_kernel(x, y, gamma=1.0):
    """k(x,y) = exp(-γ‖x-y‖₁) — sharper peaks than RBF"""
    return np.exp(-gamma * np.sum(np.abs(x - y)))

The RBF (Gaussian) kernel deserves special attention. Its feature space is infinite-dimensional — the implicit feature map has infinitely many components. Yet computing the kernel is O(d) where d is the input dimension. You're doing linear algebra in an infinite-dimensional space for the cost of a single distance computation. That's the magic of kernels.

The Representer Theorem

The Representer Theorem is the cornerstone result that makes kernel methods practical. It says: for any regularized empirical risk minimization problem of the form

minimize  Σᵢ L(yᵢ, f(xᵢ)) + λ‖f‖²_H

where the minimization is over all functions f in an RKHS H, the optimal solution has the form:

f*(x) = Σᵢ₌₁ⁿ αᵢ k(xᵢ, x)

The optimal function is a linear combination of kernel functions centered at the training points. This is remarkable: you're optimizing over an infinite-dimensional function space, but the solution is parameterized by just n numbers (one αᵢ per training point). The infinite-dimensional optimization reduces to a finite-dimensional one.

This means you never need to worry about the feature space. Everything reduces to the n × n kernel matrix K where Kᵢⱼ = k(xᵢ, xⱼ). The cost shifts from feature dimension to sample size — kernel methods scale as O(n²) to O(n³) in the number of training points, regardless of the feature space dimension.

Kernel Ridge Regression

Kernel Ridge Regression (KRR) is the cleanest illustration of the Representer Theorem in action. Regular ridge regression minimizes ‖y - Xβ‖² + λ‖β‖² with solution β = (XᵀX + λI)⁻¹Xᵀy. The kernelized version works with the kernel matrix instead:

α = (K + λI)⁻¹y

Predictions on new points: f(x) = Σᵢ αᵢ k(xᵢ, x) = k(x)ᵀ α, where k(x) is the vector of kernel evaluations between x and all training points.

import numpy as np
from sklearn.metrics.pairwise import rbf_kernel

class KernelRidgeRegression:
    def __init__(self, gamma=1.0, lam=1.0):
        self.gamma = gamma
        self.lam = lam
    
    def fit(self, X, y):
        self.X_train = X
        K = rbf_kernel(X, X, gamma=self.gamma)
        n = K.shape[0]
        self.alpha = np.linalg.solve(K + self.lam * np.eye(n), y)
        return self
    
    def predict(self, X):
        K_test = rbf_kernel(X, self.X_train, gamma=self.gamma)
        return K_test @ self.alpha

# Demo: nonlinear regression
np.random.seed(42)
X_train = np.sort(np.random.rand(50, 1) * 4 * np.pi, axis=0)
y_train = np.sin(X_train).ravel() + 0.1 * np.random.randn(50)

krr = KernelRidgeRegression(gamma=0.5, lam=0.01)
krr.fit(X_train, y_train)

X_test = np.linspace(0, 4 * np.pi, 200).reshape(-1, 1)
y_pred = krr.predict(X_test)
print(f"Predictions range: [{y_pred.min():.2f}, {y_pred.max():.2f}]")
# Smooth sinusoidal predictions

The computational cost is O(n³) for the matrix inversion (or O(n²) per conjugate gradient iteration). For n > 10,000, this gets expensive. That's the main limitation of kernel methods — they scale poorly with dataset size. Approximation methods like the Nyström approximation and Random Fourier Features help, but neural networks have largely replaced kernels for big-data problems.

Multiple Kernel Learning

Multiple Kernel Learning (MKL) addresses a practical question: which kernel should you use? Different kernels capture different notions of similarity. Instead of picking one, MKL learns a combination:

k(x, x') = Σₘ μₘ kₘ(x, x')

where the weights μₘ ≥ 0 are learned alongside the model parameters. The optimization is typically formulated as a joint problem: find both the optimal kernel combination and the optimal classifier for that combination. This can be cast as a semi-definite program or solved with alternating optimization.

MKL is useful when you have heterogeneous features — text, images, and metadata for the same items. Each feature type gets its own kernel, and MKL learns how to weight them. It's also a principled way to do hyperparameter selection for the kernel: include RBF kernels with several different bandwidths and let MKL pick the right mix.

Kernels and Neural Networks: The Connection

There's a deep connection between kernels and neural networks. Infinitely wide neural networks converge to Gaussian processes with a specific kernel (the Neural Tangent Kernel). This means kernel methods and neural networks are two points on the same spectrum: kernels are the "infinite width, fixed features" limit, while practical neural networks learn finite, adaptive features. Understanding kernel theory helps you understand why neural networks generalize — the implicit regularization of SGD behaves similarly to the RKHS norm penalty.

Information-Theoretic Learning

We now shift gears from geometry (kernels, feature spaces) to information theory. The central question: what is the relationship between data compression and learning? The key insight is that they're essentially the same thing. A good model is a compressed description of the data. Overfitting means your "compression" is longer than the data itself — you've memorized rather than generalized.

Minimum Description Length (MDL)

The Minimum Description Length principle formalizes Occam's Razor in terms of coding theory. Given data D and a model M, the total description length is:

L(M) + L(D | M)

L(M) is the length (in bits) of describing the model. A more complex model takes more bits. L(D | M) is the length of describing the data given the model — the residuals, the parts the model doesn't explain. A better-fitting model has shorter residuals.

MDL selects the model that minimizes the total description length. This is a natural tradeoff: a very simple model (short L(M)) may leave lots of unexplained data (long L(D|M)). A very complex model (long L(M)) explains the data perfectly (short L(D|M)) but the model description itself is wasteful. The sweet spot is where the model is complex enough to capture the real patterns but not so complex that it memorizes noise.

import numpy as np

def mdl_polynomial_selection(x, y, max_degree=15):
    """Select polynomial degree by MDL principle."""
    n = len(x)
    results = []
    
    for d in range(1, max_degree + 1):
        # Fit polynomial
        coeffs = np.polyfit(x, y, d)
        y_pred = np.polyval(coeffs, x)
        residuals = y - y_pred
        
        # L(D|M): code residuals assuming Gaussian noise
        rss = np.sum(residuals**2)
        sigma2 = rss / n
        if sigma2 < 1e-15:
            sigma2 = 1e-15
        data_cost = 0.5 * n * np.log2(sigma2)  # bits for residuals
        
        # L(M): code the model (d+1 parameters, each ~log2(n)/2 bits)
        model_cost = 0.5 * (d + 1) * np.log2(n)  # BIC-like penalty
        
        total = model_cost + data_cost
        results.append((d, total, model_cost, data_cost))
    
    # Select minimum total description length
    best = min(results, key=lambda r: r[1])
    print(f"MDL selects degree {best[0]}")
    print(f"  Model cost: {best[2]:.1f} bits")
    print(f"  Data cost:  {best[3]:.1f} bits")
    print(f"  Total:      {best[1]:.1f} bits")
    return best[0]

# Demo: true function is degree 3, data has noise
np.random.seed(42)
x = np.linspace(-2, 2, 50)
y = 0.5*x**3 - x + 0.3*np.random.randn(50)
selected_degree = mdl_polynomial_selection(x, y)

MDL has deep connections to Bayesian model selection. The two-part MDL code above is related to BIC (Bayesian Information Criterion). The more sophisticated "normalized maximum likelihood" version of MDL is asymptotically equivalent to Bayesian model evidence. They're different languages for the same idea: the best model is the one that compresses the data most efficiently.

Kolmogorov Complexity

Kolmogorov complexity is the ultimate measure of a string's compressibility: the length of the shortest computer program that produces that string. A string of a million zeros has low Kolmogorov complexity — print("0" * 1000000) is short. A million random bits have high Kolmogorov complexity — there's no program shorter than the string itself.

Kolmogorov complexity is uncomputable — there's no algorithm that computes it for an arbitrary string. But it's conceptually powerful. MDL can be seen as an approximation to Kolmogorov complexity using a restricted class of "programs" (models). The string with the shortest total description (model + data given model) is the most regular, the most compressible, the one with the most learnable structure.

In practice, Kolmogorov complexity shows up as a theoretical benchmark. If the Kolmogorov complexity of a dataset is high relative to its size, the data is essentially random and no model can do better than memorization. If it's low, there's compressible structure that a good model should capture. Normalized compression distances — using gzip or other compressors as approximate Kolmogorov complexity — have been used for clustering, anomaly detection, and even authorship attribution.

Rate-Distortion Theory

Rate-distortion theory is Shannon's framework for lossy compression. Given a source distribution and a distortion measure, it answers: what is the minimum number of bits per symbol needed to describe the source within a given distortion level?

The rate-distortion function R(D) traces this tradeoff. At D = 0 (no distortion allowed), R equals the entropy of the source — you need to encode everything perfectly. As D increases, R decreases — you can use fewer bits by tolerating more error. The shape of R(D) tells you how compressible the source is.

For ML, rate-distortion theory gives a lower bound on how well any model can approximate the data. If your training loss is a distortion measure and your model complexity is a rate, the rate-distortion bound tells you the fundamental tradeoff — no model of a given complexity can do better. This connects to the bias-variance tradeoff: limited model complexity (rate) forces approximation error (distortion).

The Information Bottleneck Method

The Information Bottleneck (IB) method, introduced by Tishby, Pereira, and Bialek (2000), is one of the most elegant frameworks connecting information theory to representation learning. The setup: you have an input variable X, a target variable Y, and you want to find a compressed representation T of X that preserves as much information as possible about Y.

Formally, the IB optimizes:

minimize  I(X; T) - β I(T; Y)

where I(X; T) is the mutual information between the input and the representation (the "rate" — how much of X you're keeping), and I(T; Y) is the mutual information between the representation and the target (the "relevance" — how much useful information T retains about Y). The parameter β controls the tradeoff: large β emphasizes relevance, small β emphasizes compression.

import numpy as np
from scipy.special import softmax

def information_bottleneck(pxy, beta=2.0, n_clusters=3, n_iter=100):
    """
    Information Bottleneck via iterative Blahut-Arimoto-like algorithm.
    pxy: joint distribution P(X,Y) as a 2D array (|X| x |Y|)
    """
    n_x, n_y = pxy.shape
    px = pxy.sum(axis=1)
    py_given_x = pxy / px[:, np.newaxis]  # P(Y|X)
    
    # Initialize encoder P(T|X) randomly
    pt_given_x = np.random.dirichlet(np.ones(n_clusters), size=n_x)
    
    for iteration in range(n_iter):
        # Compute P(T) = Σ_x P(T|X) P(X)
        pt = pt_given_x.T @ px  # (n_clusters,)
        
        # Compute P(Y|T) = Σ_x P(Y|X) P(X|T) P(T)
        # P(X|T) = P(T|X) P(X) / P(T)
        px_given_t = (pt_given_x * px[:, np.newaxis]).T / pt[:, np.newaxis]
        py_given_t = px_given_t @ py_given_x  # (n_clusters, n_y)
        
        # Update encoder: P(T|X) ∝ P(T) exp(-β D_KL(P(Y|X) || P(Y|T)))
        log_pt_given_x = np.zeros((n_x, n_clusters))
        for t in range(n_clusters):
            for x in range(n_x):
                kl = np.sum(py_given_x[x] * 
                           np.log(py_given_x[x] / (py_given_t[t] + 1e-10) + 1e-10))
                log_pt_given_x[x, t] = np.log(pt[t] + 1e-10) - beta * kl
        
        pt_given_x = softmax(log_pt_given_x, axis=1)
    
    return pt_given_x, pt, py_given_t

# Demo with a simple joint distribution
pxy = np.array([[0.1, 0.01], [0.09, 0.02], [0.01, 0.12],
                [0.02, 0.11], [0.08, 0.03], [0.03, 0.10],
                [0.07, 0.04], [0.04, 0.09], [0.02, 0.02]])
pxy = pxy / pxy.sum()  # normalize

encoder, pt, py_t = information_bottleneck(pxy, beta=5.0, n_clusters=2)
print(f"Cluster assignments: {encoder.argmax(axis=1)}")
print(f"Cluster sizes: {pt.round(3)}")

The Information Bottleneck has become famous in deep learning through Shwartz-Ziv and Tishby's (2017) hypothesis: deep networks learn in two phases. First, they fit the training data (increasing I(T; Y)). Then, they compress (decreasing I(X; T)), discarding input information that isn't relevant to the target. This "compression phase" corresponds to generalization — the network is learning to ignore noise. The hypothesis is controversial and the evidence is mixed, but it has inspired a rich line of research connecting information theory to deep learning.

The IB framework also connects to variational autoencoders (VAEs). The VAE objective — reconstruction loss plus KL divergence to a prior — can be viewed as an information bottleneck: the KL term limits I(X; T) while the reconstruction term encourages I(T; Y) (where Y = X for autoencoders). β-VAE explicitly introduces a β parameter to control this tradeoff, directly mirroring the IB objective.

The Deep Unity

Kernel methods and information-theoretic methods seem unrelated, but they're two perspectives on the same question: how do you choose the right function space for learning? Kernels answer it geometrically — pick the RKHS with the right smoothness properties. Information theory answers it through compression — pick the representation that preserves relevant information and discards the rest. The Representer Theorem and the Information Bottleneck are both saying: the optimal solution lives in a much simpler space than you'd expect. That's the recurring miracle of learning theory.
