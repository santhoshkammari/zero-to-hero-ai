Tensor Decomposition

Chapter 17: Learning Theory & Advanced ML 5 subtopics
TL;DR

Matrices are 2D tables. Tensors are the higher-dimensional generalization — 3D cubes, 4D hypercubes, and beyond. Tensor decomposition extends matrix factorization to these multi-way arrays, revealing structure that matrix methods miss entirely. CP decomposition expresses a tensor as a sum of rank-1 components. Tucker decomposition provides a compressed core with factor matrices per mode. Tensor Train handles explosively high-order tensors. These tools power knowledge graph embeddings, neural network compression, multi-way data analysis, and latent variable model learning.

Tensor Fundamentals

In the previous section, we decomposed matrices — 2D arrays. But data is often naturally higher-dimensional. Consider user-movie-time ratings: user × movie × month. Or EEG data: channel × frequency × time. Or a color image: height × width × RGB channel. Flattening these into matrices loses the multi-way structure. You'd be mixing up dimensions that shouldn't be mixed. Tensors let you keep that structure intact.

Tensors as Multilinear Arrays

A tensor is a multi-dimensional array. A scalar is a 0th-order tensor, a vector is 1st-order, a matrix is 2nd-order. A 3rd-order tensor is a "cube" of numbers — index it with three indices X(i, j, k). A 4th-order tensor needs four indices, and so on. The order (or number of modes) is how many indices you need.

Don't confuse this with "tensor" in TensorFlow or PyTorch, where it means "any multi-dimensional array." Here, we're talking about tensors as algebraic objects with specific decomposition theory. The word is overloaded, but the context makes it clear.

Fibers, Slices, and Unfolding

To work with tensors, we need to talk about their parts. A fiber is a 1D fragment obtained by fixing all indices except one — like a row or column of a matrix, but generalized. A mode-1 fiber of a 3rd-order tensor fixes j and k and varies i. A slice is a 2D fragment obtained by fixing all indices except two — a matrix slice of the tensor.

Unfolding (or matricization) reshapes a tensor into a matrix by stacking fibers. The mode-n unfolding X₍ₙ₎ arranges all mode-n fibers as columns of a matrix. This connects tensors back to the matrix world — many tensor algorithms work by unfolding the tensor, doing matrix operations, and folding back.

import numpy as np

# Create a 3rd-order tensor: 4 x 3 x 2
X = np.arange(24).reshape(4, 3, 2)
print(f"Tensor shape: {X.shape}")

# Mode-0 fibers (fix j,k, vary i): each is length 4
fiber_01 = X[:, 0, 0]  # [0, 6, 12, 18]
print(f"Mode-0 fiber (j=0,k=0): {fiber_01}")

# Mode-0 unfolding: reshape to (4, 3*2) = (4, 6)
X_0 = X.reshape(4, -1)
print(f"Mode-0 unfolding shape: {X_0.shape}")

# Mode-1 unfolding: reshape to (3, 4*2) = (3, 8)
X_1 = X.transpose(1, 0, 2).reshape(3, -1)
print(f"Mode-1 unfolding shape: {X_1.shape}")

Tensor Rank and Rank-1 Tensors

A rank-1 tensor is the outer product of vectors — one per mode. For a 3rd-order tensor: X = a ⊗ b ⊗ c, meaning X(i,j,k) = aᵢ bⱼ cₖ. Every entry is a product of one element from each vector. This is the simplest possible tensor structure.

The rank of a tensor is the minimum number of rank-1 tensors you need to sum to exactly reconstruct it. Unlike matrix rank (computable via SVD in polynomial time), tensor rank is NP-hard to compute in general. Even determining whether a tensor has rank ≤ k is NP-hard for k ≥ 3. This fundamental difficulty makes tensor decomposition both more interesting and more challenging than its matrix counterpart.

CP Decomposition (CANDECOMP/PARAFAC)

The CP decomposition (Canonical Polyadic decomposition, independently discovered as CANDECOMP and PARAFAC) expresses a tensor as a sum of R rank-1 components:

X ≈ Σᵣ₌₁ᴿ  λᵣ (aᵣ ⊗ bᵣ ⊗ cᵣ)

Each component is a rank-1 tensor defined by vectors aᵣ, bᵣ, cᵣ (one per mode) and a weight λᵣ. R is the number of components — analogous to the rank k in truncated SVD.

Alternating Least Squares for CP

The standard algorithm for CP decomposition is Alternating Least Squares (ALS). Fix all factor matrices except one, and solve the resulting linear least-squares problem for the free matrix. Rotate through all modes. Each sub-problem has a closed-form solution.

import numpy as np
from numpy.linalg import lstsq

def cp_als(X, R, n_iter=100):
    """CP decomposition via ALS for a 3rd-order tensor."""
    I, J, K = X.shape
    
    # Initialize factor matrices randomly
    A = np.random.randn(I, R)
    B = np.random.randn(J, R)
    C = np.random.randn(K, R)
    
    for iteration in range(n_iter):
        # Update A: unfold X along mode-0, solve X_(0) ≈ A (C⊙B)ᵀ
        # where ⊙ is the Khatri-Rao product (column-wise Kronecker)
        CB = np.array([np.kron(C[k, :], B[j, :]) 
                        for j in range(J) for k in range(K)])
        # Simpler via matricized tensor
        X_0 = X.reshape(I, -1)  # I x (J*K)
        kr_CB = np.column_stack([np.kron(C[:, r], B[:, r]) for r in range(R)])
        A = lstsq(kr_CB, X_0.T, rcond=None)[0].T
        
        # Update B
        X_1 = X.transpose(1, 0, 2).reshape(J, -1)
        kr_CA = np.column_stack([np.kron(C[:, r], A[:, r]) for r in range(R)])
        B = lstsq(kr_CA, X_1.T, rcond=None)[0].T
        
        # Update C
        X_2 = X.transpose(2, 0, 1).reshape(K, -1)
        kr_BA = np.column_stack([np.kron(B[:, r], A[:, r]) for r in range(R)])
        C = lstsq(kr_BA, X_2.T, rcond=None)[0].T
        
        if iteration % 20 == 0:
            # Reconstruct and check error
            X_hat = np.zeros_like(X)
            for r in range(R):
                X_hat += A[:, r:r+1] * B[:, r:r+1].T[np.newaxis, :, :] \
                         if False else np.einsum('i,j,k', A[:,r], B[:,r], C[:,r])
            err = np.linalg.norm(X - X_hat) / np.linalg.norm(X)
            print(f"Iter {iteration}: relative error = {err:.6f}")
    
    return A, B, C

Uniqueness Properties

Here's where CP has a major advantage over matrix SVD. Matrix factorizations are not unique — you can always insert a rotation: X = (UR)(R⁻¹Σ Vᵀ) for any orthogonal R. You need additional constraints (like orthogonality of singular vectors) to pin down a unique decomposition.

CP decomposition, under mild conditions, is essentially unique — unique up to permutation and scaling of the components. This follows from Kruskal's theorem: if the sum of the k-ranks of the factor matrices exceeds 2R + 2, the decomposition is unique. This uniqueness is a gift from the higher-dimensional structure. It means CP components often have genuine physical interpretation — they correspond to real underlying factors, not arbitrary rotations of them.

Rank Determination

Choosing R (the number of components) is tricky. There's no tensor equivalent of the Eckart-Young theorem — the best rank-R approximation may not exist (the infimum of the approximation error may not be achieved). In practice, people try several values of R and use a combination of reconstruction error, core consistency diagnostics, and domain knowledge. Some advocate for Bayesian approaches that learn R from data by placing a sparsity-inducing prior on the component weights.

Tucker Decomposition

Tucker decomposition is more flexible than CP. Instead of a sum of rank-1 components, it decomposes a tensor into a small core tensor multiplied by factor matrices along each mode:

X ≈ G ×₁ A ×₂ B ×₃ C

Here G is an R₁ × R₂ × R₃ core tensor, A is I × R₁, B is J × R₂, C is K × R₃, and ×ₙ denotes the mode-n product (multiplying a matrix along mode n of the tensor). The factor matrices compress each mode, and the core tensor captures the interactions between the compressed modes.

Think of it as a higher-dimensional generalization of UΣVᵀ. In matrix SVD, Σ is diagonal — modes don't interact. In Tucker, the core G can be dense, allowing arbitrary interactions between the factors of different modes. CP is the special case where the core is "super-diagonal" (only entries G(r,r,r) are non-zero).

Higher-Order SVD (HOSVD)

HOSVD computes the Tucker decomposition by performing SVD independently on each mode's unfolding. Take the mode-1 unfolding X₍₁₎, compute its SVD, keep the top R₁ left singular vectors as A. Repeat for modes 2 and 3 to get B and C. The core tensor is then G = X ×₁ Aᵀ ×₂ Bᵀ ×₃ Cᵀ.

import numpy as np

def hosvd(X, ranks):
    """Higher-Order SVD (Tucker decomposition) for a 3rd-order tensor."""
    factors = []
    for mode in range(3):
        # Unfold along this mode
        X_n = np.moveaxis(X, mode, 0).reshape(X.shape[mode], -1)
        # SVD and keep top-r left singular vectors
        U, _, _ = np.linalg.svd(X_n, full_matrices=False)
        factors.append(U[:, :ranks[mode]])
    
    # Compute core tensor: G = X x_1 A^T x_2 B^T x_3 C^T
    G = X.copy()
    for mode in range(3):
        G = np.tensordot(G, factors[mode].T, axes=([0], [1]))
        # Move the contracted axis back to position 0 for next iteration
        G = np.moveaxis(G, -1, 0)
    
    return G, factors

# Demo
X = np.random.randn(20, 15, 10)
G, factors = hosvd(X, ranks=[3, 3, 3])
print(f"Core shape: {G.shape}")       # (3, 3, 3)
print(f"Factor shapes: {[f.shape for f in factors]}")
# Compressed from 20*15*10=3000 to 3*3*3 + 20*3 + 15*3 + 10*3 = 27+60+45+30 = 162 numbers

Higher-Order Orthogonal Iteration (HOOI)

HOSVD is not optimal — it's a one-shot approximation. HOOI refines it iteratively. Starting from the HOSVD factors, HOOI alternates: fix all factor matrices except one, compute the optimal factor for that mode (via SVD of a partially projected tensor), and rotate through all modes. This reduces the approximation error at each step. In practice, HOOI converges in a few iterations and gives noticeably better results than plain HOSVD.

Multilinear Rank

The multilinear rank of a tensor is the tuple (R₁, R₂, R₃) where Rₙ is the rank of the mode-n unfolding. This is different from the CP rank. A tensor can have multilinear rank (3, 3, 3) but CP rank 5. Tucker decomposition targets the multilinear rank; CP decomposition targets the CP rank. Choosing which to use depends on whether you want a compact overall representation (Tucker) or a sum of interpretable components (CP).

Tensor Train (TT) Decomposition

CP and Tucker work well for 3rd or 4th-order tensors, but they struggle with high-order tensors. Tucker decomposition suffers from the curse of dimensionality: a core tensor of order d with ranks r in each mode has r^d entries. For d = 10 and r = 5, that's nearly 10 million entries in the core alone. CP avoids this by constraining the core to be super-diagonal, but the required CP rank can be enormous.

Chain of 3rd-Order Tensors

Tensor Train decomposition solves this by expressing a high-order tensor as a chain of 3rd-order "carriages":

X(i₁, i₂, ..., i_d) = G₁(i₁) · G₂(i₂) · ... · G_d(i_d)

Each Gₖ(iₖ) is an rₖ₋₁ × rₖ matrix (a slice of the kth core tensor, which is 3rd-order: rₖ₋₁ × nₖ × rₖ). The product of all these matrices gives a scalar — the tensor entry. The boundary conditions are r₀ = r_d = 1, making G₁ and G_d vectors.

The total storage is Σₖ rₖ₋₁ · nₖ · rₖ, which grows linearly in d (the number of dimensions), not exponentially. For a 10th-order tensor of size 5 in each mode with TT-ranks all equal to 3, you store roughly 10 × 3 × 5 × 3 = 450 numbers instead of 5^10 ≈ 10 million. That's a compression factor of over 20,000.

TT-Rank

The TT-rank is the tuple (r₁, r₂, ..., r_{d-1}) of ranks connecting the cores. It controls the expressiveness and storage of the TT decomposition. Low TT-ranks mean high compression. The TT-rank is related to the ranks of specific unfoldings of the tensor: rₖ = rank(X₍ₖ₎) where X₍ₖ₎ reshapes the tensor into a matrix by grouping the first k modes versus the last d-k modes.

TT-SVD Algorithm

The TT-SVD algorithm constructs the TT decomposition from left to right using a sequence of SVDs:

import numpy as np

def tt_svd(X, max_rank=None, tol=1e-10):
    """Tensor Train decomposition via sequential SVDs."""
    shape = X.shape
    d = len(shape)
    cores = []
    C = X.reshape(shape[0], -1)  # initial unfolding
    r_prev = 1
    
    for k in range(d - 1):
        C = C.reshape(r_prev * shape[k], -1)
        U, s, Vt = np.linalg.svd(C, full_matrices=False)
        
        # Determine rank (by tolerance or max_rank)
        if tol > 0:
            r = max(1, np.sum(s > tol * s[0]))
        else:
            r = len(s)
        if max_rank is not None:
            r = min(r, max_rank)
        
        # Truncate
        U = U[:, :r]
        s = s[:r]
        Vt = Vt[:r, :]
        
        # Store core: reshape U to (r_prev, shape[k], r)
        cores.append(U.reshape(r_prev, shape[k], r))
        
        # Continue with remainder
        C = np.diag(s) @ Vt
        r_prev = r
    
    # Last core
    cores.append(C.reshape(r_prev, shape[-1], 1))
    
    return cores

# Demo: decompose a 4th-order tensor
X = np.random.randn(5, 6, 4, 3)
cores = tt_svd(X, max_rank=3)
print(f"TT cores: {[c.shape for c in cores]}")

# Reconstruct to verify
def tt_reconstruct(cores):
    result = cores[0]
    for c in cores[1:]:
        result = np.tensordot(result, c, axes=([-1], [0]))
    return result.squeeze()

X_hat = tt_reconstruct(cores)
print(f"Reconstruction error: {np.linalg.norm(X - X_hat) / np.linalg.norm(X):.2e}")

Scalability for High-Order Tensors

The Tensor Train format is the only practical decomposition for truly high-order tensors (d = 10, 20, or more). It's widely used in computational physics (quantum state simulation), numerical PDE solving, and deep learning (compressing weight tensors of large layers). The TT format also supports efficient arithmetic: you can add, multiply, and apply linear operators to TT tensors without ever forming the full tensor, keeping everything in the compressed format.

Applications in ML

Multiway Data Analysis

Any dataset with more than two natural axes benefits from tensor methods. EEG data (subjects × channels × time × frequency), social network data (users × users × time), or sensor networks (location × variable × time) all have multi-way structure. Tensor decomposition discovers patterns that span all modes simultaneously — something matrix methods can't do because they'd have to flatten some modes and lose their structure.

Latent Variable Model Learning

Tensor methods provide provably correct algorithms for learning latent variable models. The key idea (Anandkumar et al., 2014): compute empirical moments (second and third order) of the observed data, form a tensor from these moments, and decompose it. The CP components correspond to the model parameters. This works for mixture models, Hidden Markov Models, topic models, and more. Unlike EM, which can get stuck in local optima, tensor methods find the global optimum (up to estimation error).

Knowledge Graph Embeddings

A knowledge graph has (subject, relation, object) triples — naturally a 3rd-order tensor. CP decomposition of this tensor gives you embeddings for entities and relations. The DistMult and ComplEx models are essentially CP decomposition variants on the knowledge graph tensor. RESCAL uses Tucker decomposition with a per-relation core matrix. These tensor-based methods remain competitive with neural approaches and are far more interpretable.

Neural Network Compression

The weight matrix (or tensor) of a fully connected or convolutional layer can be compressed using tensor decomposition. A 4D convolutional filter tensor can be decomposed in CP or Tucker format, replacing one large layer with a sequence of smaller layers. This can achieve 3–10x compression with minimal accuracy loss. TT decomposition is especially effective for embedding layers, which are enormous matrices that admit very low-rank TT representations.

Higher-Order Statistics

Second-order statistics (covariance matrices) capture pairwise relationships. Third-order cumulants capture three-way interactions. Fourth-order moments are needed for kurtosis and independent component analysis (ICA). All of these are naturally tensors, and their decompositions reveal structure that second-order methods (PCA, factor analysis) miss entirely. Blind source separation, for instance, works by decomposing the fourth-order cumulant tensor of the mixed signals.

When to Use Tensor vs Matrix Methods

If your data is naturally a matrix (users × items, documents × words), matrix factorization is the right tool. If your data has three or more natural modes, tensor decomposition can discover richer structure. The practical test: if flattening your multi-way data into a matrix feels like you're losing important information (e.g., collapsing the time dimension in user-item-time data), that's a signal to try tensor methods. The computational cost is higher, but the payoff is capturing interactions that matrices cannot represent.
