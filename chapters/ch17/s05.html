Sparse & Factorization Methods

TL;DR

Most real-world data is sparse or low-rank — it has far less information in it than its raw size suggests. Sparsity methods (Lasso) find which features actually matter. Matrix and tensor factorization methods (SVD, NMF, CP decomposition) compress data into its essential structure. These techniques power everything from Netflix recommendations to MRI scanners.

Here's a thought experiment. You have 10,000 features describing a patient. How many actually predict their diagnosis? Probably a dozen. You have a 100-million-entry user-movie ratings matrix. What's its actual rank? Maybe 50. The world is ridiculously redundant, and every method in this section exploits that fact.

Sparsity: Most Things Don't Matter

In regression, you're fitting coefficients to features. Ridge regression (L2 penalty) shrinks all coefficients toward zero but never to zero. Lasso (L1 penalty) does something much more interesting — it drives irrelevant coefficients to exactly zero. That's not a minor distinction. It means Lasso does feature selection and model fitting in one shot.

Why does L1 produce exact zeros? Geometrically, the L1 constraint region is a diamond (in 2D). The loss function's contours typically hit the diamond at a corner, where one coordinate is zero. The L2 constraint is a circle — contours hit it on a smooth surface, so both coordinates stay nonzero.

PropertyRidge (L2)Lasso (L1)Elastic Net (L1+L2)
PenaltyΣ β²Σ |β|α Σ |β| + (1−α) Σ β²
Zeros out coefficients?NoYesYes
Correlated featuresKeeps all, shrinks equallyPicks one, drops othersGroups them
Best whenMany small effectsFew features matterGroups of related features
 
from sklearn.linear_model import Lasso, Ridge
from sklearn.datasets import make_regression
import numpy as np

# Generate data: 1000 features, but only 10 actually matter
X, y = make_regression(n_samples=200, n_features=1000,
                       n_informative=10, noise=5, random_state=42)

lasso = Lasso(alpha=0.5).fit(X, y)
ridge = Ridge(alpha=0.5).fit(X, y)

print(f"Lasso nonzero coefficients: {np.sum(lasso.coef_ != 0)}")   # ~10-15
print(f"Ridge nonzero coefficients: {np.sum(ridge.coef_ != 0)}")   # 1000

# Lasso automatically found the ~10 features that matter
important = np.where(lasso.coef_ != 0)[0]
print(f"Selected features: {important}")
When to Pick Which

Use Lasso when you suspect only a handful of features matter and you want the model to tell you which ones. Use Ridge when many features contribute small amounts. Use Elastic Net when features come in correlated groups (e.g., gene expression data). In practice, Elastic Net is the safest default.

Compressed Sensing: Fewer Measurements Than You'd Think

Classical signal processing says you need to sample at 2× the highest frequency to perfectly reconstruct a signal (Nyquist theorem). Compressed sensing says: if the signal is sparse in some basis, you can get away with far fewer measurements.

The MRI scanner in your hospital takes advantage of this. A full MRI scan might take 30 minutes. With compressed sensing, you take a fraction of the measurements, solve an L1-minimization problem, and reconstruct the image in 5 minutes with nearly identical quality. The math guarantees exact recovery under conditions involving something called the Restricted Isometry Property (RIP) — roughly, your measurement matrix must not collapse sparse signals.

Compressed Sensing in One Sentence

If a signal is sparse (or sparse in some transform domain), you can recover it exactly from dramatically fewer measurements than its dimension, by solving an L1 minimization problem. This is why modern MRI is fast and why single-pixel cameras work.

You probably won't implement compressed sensing from scratch, but knowing it exists changes how you think about data collection. If your signal is sparse, you might not need as much data as you think.

Matrix Factorization: The Netflix Trick

Take a matrix of users × movies with ratings. It's mostly empty — any given user has rated maybe 200 out of 50,000 movies. The key insight: this matrix is approximately low-rank. A user's taste can be described by maybe 50 latent factors (loves action, hates romance, likes Nolan, etc.), and a movie can be described by those same factors. So instead of storing millions of ratings, you store two small matrices whose product approximates the original.

SVD — The Swiss Army Knife

Singular Value Decomposition factors any matrix A into three matrices: A = UΣVᵀ. U contains left singular vectors (user patterns), Σ contains singular values (importance of each factor), Vᵀ contains right singular vectors (movie patterns). Keeping only the top-k singular values gives you the best rank-k approximation of A — that's the Eckart-Young theorem.

import numpy as np

# Simulated user-movie ratings (0 = unrated)
ratings = np.array([
    [5, 4, 0, 0, 1],
    [4, 5, 0, 1, 0],
    [0, 0, 5, 4, 0],
    [0, 1, 4, 5, 0],
    [1, 0, 0, 0, 5],
])

# SVD decomposition
U, sigma, Vt = np.linalg.svd(ratings, full_matrices=False)

# Keep top-2 factors (rank-2 approximation)
k = 2
approx = U[:, :k] @ np.diag(sigma[:k]) @ Vt[:k, :]

print("Original:\n", ratings)
print(f"\nRank-{k} approximation:\n", np.round(approx, 1))
# The zeros (unrated items) now have predicted values — that's recommendations!

The Netflix Prize was won in 2009 using matrix factorization as the core technique. The winning team's key insight wasn't a single algorithm but an ensemble — yet matrix factorization variants were the backbone of every competitive submission. In production, you'd use gradient descent-based approaches (like Simon Funk's method) rather than SVD directly, since they handle missing values more naturally.

NMF: Parts-Based Decomposition

Non-negative Matrix Factorization adds one constraint to matrix factorization: all values must be ≥ 0. This seemingly small change produces dramatically different — and more interpretable — results. While SVD factors can have negative values (making them hard to interpret as "parts"), NMF learns additive parts.

Apply NMF to a collection of face images, and you get parts like "left eye," "nose," "right eyebrow." Apply it to a document-term matrix, and you get topics — groups of words that co-occur. It's an alternative to LDA for topic modeling, and often faster.

from sklearn.decomposition import NMF
from sklearn.feature_extraction.text import TfidfVectorizer

docs = [
    "the cat sat on the mat",
    "the dog chased the cat",
    "matrix algebra is fundamental",
    "eigenvalues of a matrix",
    "the dog sat on the rug",
    "linear algebra and matrices",
]

tfidf = TfidfVectorizer().fit_transform(docs)
nmf = NMF(n_components=2, random_state=42).fit(tfidf)

feature_names = TfidfVectorizer().fit(docs).get_feature_names_out()
for i, topic in enumerate(nmf.components_):
    top_words = [feature_names[j] for j in topic.argsort()[-4:]]
    print(f"Topic {i}: {', '.join(top_words)}")
# Topic 0: animals/actions words
# Topic 1: math/algebra words

Use NMF when your data is naturally non-negative (pixel values, word counts, audio spectrograms) and you want interpretable parts. Use SVD/PCA when you need optimal reconstruction or don't care about interpretability.

Tensor Decomposition: Beyond Matrices

Matrices are 2D. But what if your data has three or more dimensions? A user × movie × time tensor. A subject × voxel × frequency tensor in brain imaging. A country × product × year tensor in economics. You can't just flatten these to matrices without losing the multi-way structure.

Tensor decomposition generalizes matrix factorization to higher dimensions. The two workhorses:

MethodIdeaWhen to Use
CP (CANDECOMP/PARAFAC) Decomposes tensor into sum of rank-1 tensors (outer products of vectors) Simple structure, uniqueness guarantees, knowledge graph embeddings
Tucker Core tensor × factor matrices along each mode (like higher-order SVD) Different latent dimensions per mode, compression, signal processing

CP decomposition is particularly elegant: a 3D tensor X ≈ Σᵣ aᵣ ∘ bᵣ ∘ cᵣ, where ∘ is the outer product. Each component is three vectors, one per mode. Unlike matrix decomposition, CP decomposition is essentially unique (under mild conditions) — which means the factors are identifiable, not just arbitrary rotations. This is why it's powerful for scientific applications where you want to discover real latent structure.

Where Tensors Show Up

Knowledge graphs: (subject, relation, object) triples form a 3-way tensor — CP decomposition (e.g., DistMult, ComplEx) is the basis of embedding methods. Neuroscience: EEG data is channels × time × frequency × trials. Chemometrics: samples × wavelengths × time. If you're flattening multi-way data into a matrix, you're probably losing important structure.

For practical work, check out tensorly in Python — it provides CP, Tucker, and tensor train decompositions with NumPy, PyTorch, or TensorFlow backends.

Kernel Methods

You've already seen SVMs draw decision boundaries. But what happens when no straight line can separate your data? You could manually engineer polynomial features, cross terms, maybe throw in some trig functions—and hope you guessed the right ones. Or you could use a kernel and let math do the heavy lifting without ever computing those features explicitly.

That's the kernel trick. It's one of the most elegant ideas in all of machine learning.

The Kernel Trick

Here's the setup. You have data that isn't linearly separable in its original space. The classic fix: map it to a higher-dimensional space where it is separable. A mapping function φ(x) takes your d-dimensional input into some (possibly infinite-dimensional) feature space.

The problem? Computing φ(x) for every data point, then taking dot products in that huge space, is wildly expensive. Sometimes literally impossible—you can't store an infinite-dimensional vector.

The trick: Many algorithms only need dot products between data points, never the transformed features themselves. A kernel function K(x, y) = φ(x)·φ(y) computes that dot product directly—no φ required. You get the power of high-dimensional features at the cost of a simple function evaluation.

Think of it like this: you don't need to fly to the moon to know how far away it is. You can measure from here.

import numpy as np

# The "honest" way: explicitly map to higher dimensions, then dot product
def phi(x):
    """Map 2D -> 6D (all degree-2 terms)"""
    x1, x2 = x
    return np.array([x1**2, x2**2, np.sqrt(2)*x1*x2,
                      np.sqrt(2)*x1, np.sqrt(2)*x2, 1])

# Kernel trick: same result, no mapping needed
def polynomial_kernel(x, y, degree=2, c=1):
    return (np.dot(x, y) + c) ** degree

a, b = np.array([1, 2]), np.array([3, 4])
print(f"Explicit:  {np.dot(phi(a), phi(b)):.4f}")   # 196.0000
print(f"Kernel:    {polynomial_kernel(a, b):.4f}")    # 196.0000
# Same answer. The kernel skipped building those 6D vectors entirely.

Common Kernels

Three kernels cover the vast majority of practical use cases:

KernelFormulaWhen to Use
Linear K(x,y) = x·y Data already separable. High-d sparse data (text). Fast.
Polynomial K(x,y) = (x·y + c)d Feature interactions matter. Image classification. Degree 2-3 typical.
RBF (Gaussian) K(x,y) = exp(−γ‖x−y‖²) The default. Works on almost anything. Maps to infinite dimensions. Tune γ carefully.

When in doubt, start with RBF. It's the Swiss army knife of kernels—implicitly maps to infinite-dimensional space, so it can model any smooth decision boundary given enough data. The γ parameter controls how "local" the kernel is: high γ means each point only influences nearby points (risk of overfitting), low γ means broad influence (risk of underfitting).

from sklearn.svm import SVC
from sklearn.datasets import make_moons
from sklearn.model_selection import GridSearchCV

X, y = make_moons(n_samples=200, noise=0.2, random_state=42)

# RBF is default—just tune C and gamma
param_grid = {'C': [0.1, 1, 10, 100], 'gamma': ['scale', 0.1, 1, 10]}
svm = GridSearchCV(SVC(kernel='rbf'), param_grid, cv=5)
svm.fit(X, y)
print(f"Best: C={svm.best_params_['C']}, γ={svm.best_params_['gamma']}")
print(f"Accuracy: {svm.best_score_:.3f}")

Kernel SVM — Nonlinear Boundaries

A linear SVM finds the maximum-margin hyperplane in the original input space. A kernel SVM does the same thing—but in the feature space that the kernel implicitly defines. The decision boundary that's linear in feature space becomes beautifully nonlinear back in input space.

The math stays the same. The SVM dual formulation only involves dot products xi·xj between training points. Swap every dot product for K(xi, xj) and you're done. The support vectors—the points closest to the boundary—still drive everything. Most of your training data gets ignored at prediction time.

Why this matters: You get the theoretical guarantees of SVMs (maximum margin, generalization bounds) applied to arbitrarily complex decision boundaries. No architecture to design, no activation functions to choose. Just pick a kernel and tune its parameters.

RKHS: The Theory Behind the Curtain

If you read kernel papers, you'll hit the term Reproducing Kernel Hilbert Space (RKHS). Here's the gist without the measure theory:

Every valid kernel function defines a unique Hilbert space (a complete inner product space—think "well-behaved vector space with dot products"). The "reproducing" part means the kernel function reproduces the inner product: evaluating f at point x is the same as taking the inner product of f with K(x, ·). This is the representer theorem—it guarantees that the optimal solution to any regularized loss problem in an RKHS can be written as a linear combination of kernel evaluations at the training points.

In plain English: RKHS theory is why kernel methods work. It's the mathematical foundation that ensures the kernel trick isn't just a clever hack—it's a principled framework. You don't need to master RKHS to use kernels, but knowing the term and its role removes the mystique.

Kernels in Modern ML

Kernels aren't just a classical ML relic. They keep showing up in surprising places:

Gaussian Processes are essentially kernel methods with a Bayesian twist. The kernel (called a covariance function in GP-land) defines the prior over functions. Different kernels encode different assumptions—smooth, periodic, locally varying. See Chapter 14: Gaussian Processes.

The Neural Tangent Kernel (NTK) is a fascinating bridge: it shows that infinitely-wide neural networks trained with gradient descent behave exactly like kernel regression with a specific kernel. This connection has been hugely influential in deep learning theory—it helps explain why overparameterized networks generalize and why lazy training sometimes works.

Attention mechanisms in transformers? They're computing a softmax-weighted kernel between queries and keys. The "efficient attention" literature (Random Fourier Features, Performers) explicitly uses kernel approximations to avoid the quadratic cost.

When Kernels Lose

Kernels have a scaling problem. The kernel matrix K(xi, xj) for all pairs of training points is n×n. Computing it is O(n²); solving the SVM is O(n²) to O(n³). With a million training points, that's a trillion-entry matrix. Not happening.

ScenarioKernelsDeep Learning
Small data (< 10k samples) ✅ Often best choice ⚠️ Needs regularization tricks
Large data (> 100k) ❌ O(n³) kills you ✅ Scales with SGD
Needs uncertainty estimates ✅ GPs give calibrated uncertainty ⚠️ Requires extra work
Raw images/text/audio ❌ Needs hand-crafted features ✅ Learns representations
Theory & guarantees ✅ Strong generalization bounds ⚠️ Theory still catching up

There are approximation methods—Nyström approximation, Random Fourier Features—that bring kernel methods down to O(n·m) where m ≪ n. They work, but at that point you're often trading the elegance that made kernels attractive in the first place.

Practical rule: If your dataset fits in memory and has fewer than ~50k samples, kernel methods (especially SVMs with RBF) are a serious contender. Beyond that, deep learning usually wins—but the kernel perspective can still inform your thinking about similarity, features, and generalization.
