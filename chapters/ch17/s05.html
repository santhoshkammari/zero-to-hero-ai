Meta-Learning

Chapter 17: Learning Theory & Advanced ML 6 subtopics
TL;DR

Meta-learning is learning to learn. Instead of training a model for one task, you train a learning algorithm that can quickly adapt to new tasks from very few examples. MAML finds a weight initialization so good that a few gradient steps on a new task yield strong performance. Prototypical Networks learn an embedding space where classification reduces to nearest-centroid lookup. Matching Networks use attention over support examples. Reptile approximates MAML without second-order gradients. These approaches power few-shot classification, rapid robotics adaptation, and automated hyperparameter optimization — anywhere you need fast adaptation with minimal data.

Think about how you learn. When you see a new animal for the first time — say, a pangolin — you need one or two pictures to recognize it forever. You do not need 10,000 labeled pangolin images. Why? Because you have spent your entire life learning how to learn about animals. You know what features matter (shape, texture, size), how to compare new things to known categories, and how to form concepts quickly.

Machine learning models, by contrast, are terrible at this. Train a classifier on 100 classes, show it a single example of class 101, and it has no idea what to do. Meta-learning fixes this by training models that can learn from a handful of examples — by learning the learning process itself.

The Learning-to-Learn Framework

Tasks, Not Examples

In standard ML, you have a dataset of (input, label) pairs. In meta-learning, you have a dataset of tasks. Each task is itself a small learning problem — say, "classify cats vs. dogs" or "classify ships vs. planes." The meta-learner trains across hundreds of such tasks, extracting transferable knowledge about how to learn.

The standard protocol is N-way K-shot classification. For each training episode, you sample N classes and K examples per class (the support set). The model must classify new examples (the query set) from those N classes, having seen only K examples of each. 5-way 1-shot means: here are 5 classes with 1 example each — now classify this new image.

import numpy as np

def create_meta_learning_episode(all_data, N=5, K=1, Q=15):
    """Create one N-way K-shot episode.
    
    Args:
        all_data: dict mapping class → list of examples
        N: number of classes per episode (ways)
        K: examples per class in support set (shots)
        Q: examples per class in query set
    
    Returns:
        support_set: (N*K examples, labels)
        query_set: (N*Q examples, labels)
    """
    # Sample N random classes
    classes = np.random.choice(list(all_data.keys()), N, replace=False)
    
    support_x, support_y = [], []
    query_x, query_y = [], []
    
    for new_label, cls in enumerate(classes):
        examples = np.array(all_data[cls])
        indices = np.random.choice(len(examples), K + Q, replace=False)
        
        # First K for support, rest for query
        support_x.extend(examples[indices[:K]])
        support_y.extend([new_label] * K)
        query_x.extend(examples[indices[K:K+Q]])
        query_y.extend([new_label] * Q)
    
    return (np.array(support_x), np.array(support_y),
            np.array(query_x), np.array(query_y))

# Simulate: 100 classes, 20 examples each, feature dim = 64
np.random.seed(42)
all_data = {}
for c in range(100):
    center = np.random.randn(64)
    all_data[c] = [center + 0.5 * np.random.randn(64) for _ in range(20)]

sx, sy, qx, qy = create_meta_learning_episode(all_data, N=5, K=1, Q=5)
print(f"Support set: {sx.shape} examples, {len(set(sy))} classes")
print(f"Query set:   {qx.shape} examples to classify")
print(f"Task: classify into {len(set(sy))} new classes with {1} example each")

Meta-Train, Meta-Val, Meta-Test

The classes themselves are split: meta-training classes are used for training episodes, meta-validation for hyperparameter tuning, and meta-test classes (never seen during training) for final evaluation. This ensures the model is evaluated on its ability to learn genuinely new concepts, not memorize the training classes.

MAML — Model-Agnostic Meta-Learning

The Key Idea: Learn a Great Initialization

MAML (Model-Agnostic Meta-Learning, Finn et al. 2017) asks: what if we found model weights θ such that a few gradient steps from θ on any new task give great performance? Instead of learning weights that are good for one task, MAML learns weights that are easy to fine-tune for any task.

The algorithm has two loops. The inner loop takes one or a few gradient steps on a task's support set, producing task-specific weights θ'ᵢ. The outer loop evaluates those adapted weights on the task's query set and updates the initialization θ to minimize the query loss across all tasks. This requires differentiating through the inner gradient steps — a gradient of a gradient (second-order).

import numpy as np

class SimpleMAML:
    """MAML for linear models (illustrative implementation).
    
    In practice, MAML uses neural networks with PyTorch/JAX autograd.
    Here we show the core algorithm with a linear classifier.
    """
    
    def __init__(self, input_dim, n_classes, inner_lr=0.01, outer_lr=0.001):
        self.theta = np.random.randn(input_dim, n_classes) * 0.01
        self.inner_lr = inner_lr
        self.outer_lr = outer_lr
    
    def softmax(self, logits):
        exp = np.exp(logits - logits.max(axis=1, keepdims=True))
        return exp / exp.sum(axis=1, keepdims=True)
    
    def cross_entropy_loss(self, X, y, W):
        """Compute cross-entropy loss and gradient."""
        logits = X @ W
        probs = self.softmax(logits)
        m = len(y)
        loss = -np.mean(np.log(probs[range(m), y] + 1e-10))
        
        # Gradient of cross-entropy w.r.t. W
        grad_logits = probs.copy()
        grad_logits[range(m), y] -= 1
        grad_logits /= m
        grad_W = X.T @ grad_logits
        
        return loss, grad_W
    
    def inner_loop(self, support_x, support_y, n_steps=1):
        """Adapt to a new task with a few gradient steps.
        
        Returns adapted weights (does NOT modify self.theta).
        """
        theta_prime = self.theta.copy()
        
        for _ in range(n_steps):
            loss, grad = self.cross_entropy_loss(support_x, support_y, theta_prime)
            theta_prime = theta_prime - self.inner_lr * grad
        
        return theta_prime
    
    def meta_train_step(self, tasks, inner_steps=1):
        """One MAML outer-loop update.
        
        For each task:
          1. Inner loop: θ' = θ - α∇L_support(θ)
          2. Evaluate: L_query(θ')
        Outer loop: θ ← θ - β∇Σᵢ L_query_i(θ'ᵢ)
        """
        meta_grad = np.zeros_like(self.theta)
        total_loss = 0
        
        for (sx, sy, qx, qy) in tasks:
            # Inner loop: adapt
            theta_adapted = self.inner_loop(sx, sy, inner_steps)
            
            # Evaluate on query set with adapted weights
            query_loss, query_grad = self.cross_entropy_loss(qx, qy, theta_adapted)
            total_loss += query_loss
            
            # Approximate meta-gradient (first-order approximation)
            # Full MAML would compute d(theta_adapted)/d(theta) via chain rule
            meta_grad += query_grad
        
        meta_grad /= len(tasks)
        self.theta -= self.outer_lr * meta_grad
        
        return total_loss / len(tasks)
    
    def evaluate(self, support_x, support_y, query_x, query_y, inner_steps=5):
        """Evaluate on a new task: adapt then test."""
        theta_adapted = self.inner_loop(support_x, support_y, inner_steps)
        logits = query_x @ theta_adapted
        predictions = np.argmax(logits, axis=1)
        accuracy = np.mean(predictions == query_y)
        return accuracy

# Demo: meta-train and evaluate
np.random.seed(42)
input_dim = 64
n_classes = 5
maml = SimpleMAML(input_dim, n_classes, inner_lr=0.1, outer_lr=0.01)

# Meta-training
for epoch in range(200):

    for _ in range(4):  # 4 tasks per batch
        sx, sy, qx, qy = create_meta_learning_episode(all_data, N=5, K=1, Q=5)
        batch_tasks.append((sx, sy, qx, qy))
    
    loss = maml.meta_train_step(batch_tasks, inner_steps=3)
    
    if (epoch + 1) % 50 == 0:
        # Evaluate on fresh tasks
        accs = []
        for _ in range(20):
            sx, sy, qx, qy = create_meta_learning_episode(all_data, N=5, K=1, Q=5)
            acc = maml.evaluate(sx, sy, qx, qy, inner_steps=5)
            accs.append(acc)

              f"5-way 1-shot acc={np.mean(accs):.3f} ± {np.std(accs):.3f}")
Why MAML Works: The Initialization Landscape

MAML does not find weights that are good at any single task. It finds weights that sit in a region of parameter space where the loss landscape is smooth and favorable for many tasks simultaneously. A few gradient steps can "roll downhill" to a good solution for any nearby task. Think of it as finding the center of a mountain range where every valley (task-specific solution) is reachable with a short hike.

Prototypical Networks

Classify by Distance to Class Prototypes

Prototypical Networks (Snell et al. 2017) take a different approach. Instead of learning to fine-tune, they learn an embedding space where classification becomes distance computation. For each class in the support set, compute the prototype — the mean embedding of that class's examples. To classify a query point, embed it and find the nearest prototype.

import numpy as np

class PrototypicalNetwork:
    """Prototypical Networks for few-shot classification.
    
    Learn embedding f_θ such that:
    prototype_c = mean(f_θ(x) for x in support_class_c)
    predict(x) = argmin_c distance(f_θ(x), prototype_c)
    """
    
    def __init__(self, input_dim, embed_dim):
        # Simple linear embedding (in practice, use a CNN)
        self.W = np.random.randn(input_dim, embed_dim) * 0.01
        self.b = np.zeros(embed_dim)
    
    def embed(self, X):
        """Embed inputs into the metric space."""
        # Linear + ReLU (simplified)
        return np.maximum(0, X @ self.W + self.b)
    
    def compute_prototypes(self, support_x, support_y):
        """Compute class prototypes as mean embeddings."""
        embeddings = self.embed(support_x)
        classes = np.unique(support_y)
        prototypes = {}
        for c in classes:
            mask = support_y == c
            prototypes[c] = embeddings[mask].mean(axis=0)
        return prototypes
    
    def predict(self, query_x, prototypes):
        """Classify by distance to nearest prototype."""
        embeddings = self.embed(query_x)
        classes = sorted(prototypes.keys())
        
        # Compute negative squared Euclidean distance (similarity)
        proto_matrix = np.array([prototypes[c] for c in classes])
        
        # distances[i, c] = ||embed(x_i) - prototype_c||²
        distances = np.sum(
            (embeddings[:, None, :] - proto_matrix[None, :, :]) ** 2,
            axis=2
        )
        
        # Softmax over negative distances
        logits = -distances
        probs = np.exp(logits - logits.max(axis=1, keepdims=True))
        probs /= probs.sum(axis=1, keepdims=True)
        
        predictions = np.argmin(distances, axis=1)
        return predictions, probs

# Demo
np.random.seed(42)
proto_net = PrototypicalNetwork(input_dim=64, embed_dim=32)

# Create a 5-way 5-shot task and classify

preds, probs = proto_net.predict(qx, prototypes)

print(f"5-way 5-shot accuracy (untrained): {acc:.3f}")
print(f"(Random chance = {1/5:.3f})")
print(f"\nPrototypes shape: {len(prototypes)} classes × "
      f"{list(prototypes.values())[0].shape[0]} dimensions")
print("Training optimizes the embedding so prototypes separate cleanly.")

Why Prototypical Networks Are Elegant

The beauty of Prototypical Networks is their simplicity. No inner optimization loop like MAML. No attention mechanisms. Just: embed, average, compare distances. This makes them fast at both training and inference. The metric learning objective ensures that examples from the same class cluster tightly and different classes are well-separated. With more shots (more support examples), the prototypes become better estimates of the class center, and accuracy improves gracefully.

Matching Networks

Attention over the Support Set

Matching Networks (Vinyals et al. 2016) use an attention mechanism to classify query points. Instead of computing a single prototype per class, the model computes attention weights over all support examples. The prediction for a query x is a weighted sum of support labels, where weights are determined by similarity in embedding space:

P(y|x, S) = Σᵢ a(x, xᵢ) · yᵢ

where a(x, xᵢ) = softmax(cosine_similarity(f(x), g(xᵢ))) and f, g are embedding networks (possibly with different architectures). Matching Networks also use full context embeddings — each support example's embedding depends on all other support examples via an LSTM, giving the model a global view of the task.

import numpy as np

class MatchingNetwork:
    """Matching Networks for few-shot classification.
    
    P(y|x, S) = Σ_i a(f(x), g(x_i)) * y_i
    where a() is softmax of cosine similarity.
    """
    
    def __init__(self, input_dim, embed_dim):
        self.W = np.random.randn(input_dim, embed_dim) * 0.01
    
    def embed(self, X):
        h = X @ self.W
        # L2 normalize for cosine similarity
        norms = np.linalg.norm(h, axis=1, keepdims=True) + 1e-8
        return h / norms
    
    def predict(self, query_x, support_x, support_y):
        """Classify query using attention over support set."""
        q_embed = self.embed(query_x)   # (Q, embed_dim)
        s_embed = self.embed(support_x)  # (NK, embed_dim)
        
        # Cosine similarity = dot product (since L2-normalized)
        similarity = q_embed @ s_embed.T  # (Q, NK)
        
        # Attention weights via softmax
        attention = np.exp(similarity - similarity.max(axis=1, keepdims=True))
        attention /= attention.sum(axis=1, keepdims=True)
        
        # Weighted vote over support labels
        n_classes = len(np.unique(support_y))
        # One-hot encode support labels
        one_hot = np.zeros((len(support_y), n_classes))
        one_hot[np.arange(len(support_y)), support_y] = 1
        
        # P(y|x) = attention · one_hot labels
        class_probs = attention @ one_hot  # (Q, n_classes)
        predictions = np.argmax(class_probs, axis=1)
        return predictions, class_probs

mn = MatchingNetwork(input_dim=64, embed_dim=32)
sx, sy, qx, qy = create_meta_learning_episode(all_data, N=5, K=5, Q=5)
preds, probs = mn.predict(qx, sx, sy)
print(f"Matching Network 5-way 5-shot accuracy (untrained): {np.mean(preds == qy):.3f}")

Reptile

A Simpler Alternative to MAML

Reptile (Nichol et al. 2018) achieves similar results to MAML with a much simpler algorithm. For each task, run SGD for several steps from the current initialization θ, arriving at adapted weights θ'ᵢ. Then move θ toward θ'ᵢ:

θ ← θ + ε (θ'ᵢ - θ)

That is the entire algorithm. No second-order gradients. No differentiating through the optimization path. Reptile can be shown to approximate MAML's gradient in expectation — both algorithms move toward initializations that are close to the optimal weights for many tasks.

import numpy as np

class Reptile:
    """Reptile meta-learning algorithm.
    
    Simpler than MAML — no second-order gradients needed.
    Just: adapt θ on each task, then move θ toward the adapted weights.
    """
    
    def __init__(self, input_dim, n_classes, inner_lr=0.02, meta_lr=0.1):
        self.theta = np.random.randn(input_dim, n_classes) * 0.01
        self.inner_lr = inner_lr
        self.meta_lr = meta_lr
    
    def inner_loop(self, X, y, n_steps=5):
        """Run SGD for several steps on one task."""
        W = self.theta.copy()
        for _ in range(n_steps):
            logits = X @ W
            exp_l = np.exp(logits - logits.max(axis=1, keepdims=True))
            probs = exp_l / exp_l.sum(axis=1, keepdims=True)
            
            grad_logits = probs.copy()
            grad_logits[range(len(y)), y] -= 1
            grad_logits /= len(y)
            
            grad_W = X.T @ grad_logits
            W -= self.inner_lr * grad_W
        
        return W
    
    def meta_update(self, tasks, inner_steps=5):
        """Reptile outer loop: move θ toward adapted weights."""
        meta_direction = np.zeros_like(self.theta)
        
        for (sx, sy, qx, qy) in tasks:
            # Combine support and query for inner loop
            X = np.concatenate([sx, qx])
            y = np.concatenate([sy, qy])
            
            theta_adapted = self.inner_loop(X, y, inner_steps)
            meta_direction += (theta_adapted - self.theta)
        
        meta_direction /= len(tasks)
        self.theta += self.meta_lr * meta_direction

# Train Reptile
np.random.seed(42)
reptile = Reptile(input_dim=64, n_classes=5, inner_lr=0.05, meta_lr=0.1)

for epoch in range(200):
    tasks = [create_meta_learning_episode(all_data, N=5, K=5, Q=5) for _ in range(4)]
    reptile.meta_update(tasks, inner_steps=10)
    
    if (epoch + 1) % 50 == 0:
        accs = []
        for _ in range(20):
            sx, sy, qx, qy = create_meta_learning_episode(all_data, N=5, K=5, Q=5)
            W = reptile.inner_loop(sx, sy, n_steps=10)
            preds = np.argmax(qx @ W, axis=1)
            accs.append(np.mean(preds == qy))
        print(f"Epoch {epoch+1}: 5-way 5-shot acc = {np.mean(accs):.3f} ± {np.std(accs):.3f}")

Meta-Learning for Hyperparameter Optimization

Learning to Set Learning Rates

Meta-learning is not just for few-shot classification. One of its most practical applications is hyperparameter optimization. If you have trained models on hundreds of similar datasets (e.g., different customer segments), you can meta-learn which hyperparameters work well as a function of dataset properties. This gives you a warm start for new datasets instead of searching from scratch.

More ambitiously, you can meta-learn the entire optimizer. Learned optimizers train an LSTM or transformer to output parameter updates, replacing hand-designed rules like Adam. The meta-objective is the final loss across many different optimization problems. These learned optimizers can discover update rules that outperform Adam on the distribution of problems they were trained on — though they may not generalize to very different problems (a meta-version of no free lunch).

def hyperparameter_meta_learning_concept():
    """Illustrate meta-learning for hyperparameter transfer."""
    
    # Scenario: You've trained models on 50 similar datasets
    # For each, you found the optimal learning rate via grid search
    
    np.random.seed(42)
    n_past_datasets = 50
    
    # Dataset features: (n_samples, n_features, noise_level)
    dataset_features = np.random.rand(n_past_datasets, 3)
    dataset_features[:, 0] *= 10000   # n_samples: 0 to 10000
    dataset_features[:, 1] *= 100     # n_features: 0 to 100
    dataset_features[:, 2] *= 1       # noise: 0 to 1
    
    # "Optimal" learning rates found by grid search (simulated)
    # In reality: smaller datasets → larger LR, more noise → smaller LR
    optimal_lrs = (0.1 / np.sqrt(dataset_features[:, 0] / 1000)
                   * (1 - 0.5 * dataset_features[:, 2]))
    optimal_lrs = np.clip(optimal_lrs, 1e-5, 1.0)
    
    # Meta-learn: fit a simple model from dataset features → optimal LR
    from numpy.linalg import lstsq
    # Log-linear model: log(lr) = w^T features + b
    X_meta = np.column_stack([dataset_features, np.ones(n_past_datasets)])
    log_lrs = np.log(optimal_lrs)
    w_meta, _, _, _ = lstsq(X_meta, log_lrs, rcond=None)
    
    # New dataset arrives: predict learning rate
    new_dataset = np.array([5000, 50, 0.3, 1])  # features + bias
    predicted_lr = np.exp(new_dataset @ w_meta)
    true_optimal = 0.1 / np.sqrt(5000/1000) * (1 - 0.5*0.3)
    
    print(f"New dataset: 5000 samples, 50 features, noise=0.3")
    print(f"Predicted LR: {predicted_lr:.5f}")
    print(f"True optimal: {true_optimal:.5f}")
    print(f"Without meta-learning: you'd grid search from scratch")
    print(f"With meta-learning: start at {predicted_lr:.5f} and fine-tune nearby")

When Meta-Learning Makes Sense

Meta-learning shines when you have many related tasks with limited data per task. If you have one task with abundant data, standard transfer learning (pre-train then fine-tune) works fine. If your tasks are very different from each other, meta-learning may not help — the "meta-knowledge" has nothing transferable to extract. The sweet spot is: a distribution of structurally similar tasks where each task has only a few examples. Drug discovery (many molecules, few assays each), personalization (many users, few interactions each), and robotics (many objects to manipulate, few demonstrations each) are ideal domains.

Comparing Meta-Learning Approaches

def compare_approaches():
    """Summary of meta-learning approaches."""
    approaches = {
        "MAML": {
            "idea": "Learn initialization for fast fine-tuning",
            "inner_loop": "Gradient descent (1-5 steps)",
            "pros": "Model-agnostic, works with any differentiable model",
            "cons": "Second-order gradients are expensive, memory-heavy",
            "best_for": "When you need to adapt any architecture"
        },
        "Prototypical Networks": {
            "idea": "Learn embedding where classes form tight clusters",
            "inner_loop": "None — just compute class means",

Meta-Learning Meets Foundation Models

Large pre-trained models (GPT, CLIP, etc.) are arguably the ultimate meta-learners. They are trained on such diverse data that they develop general-purpose representations enabling few-shot learning via prompting — no gradient updates needed. In-context learning in LLMs (providing examples in the prompt) is a form of meta-learning where the "inner loop" happens implicitly in the forward pass. The line between meta-learning and large-scale pre-training is blurring rapidly, and understanding the formal meta-learning framework helps you reason about why and when in-context learning works.
