Nice to Know

Chapter 11: Natural Language Processing Tooling · Metrics · Augmentation · Tips

These are things that are genuinely useful but don't need their own deep section. You'll encounter all of them in production. Here's what each is, when you'd reach for it, and enough to get started.

HuggingFace Transformers — The Standard Toolkit

HuggingFace standardized NLP tooling. Every model — BERT, GPT-2, T5, LLaMA, Mistral — uses the same API: AutoModel.from_pretrained("model-name"). Same interface for PyTorch and TensorFlow.

Three layers to know: Pipelines (one line, zero config — use for prototyping), Auto classes (AutoTokenizer, AutoModel — use when you need control), and Specific model classes (BertModel, GPT2LMHeadModel — use when you need architecture-specific features).

from transformers import pipeline

# One-line inference for any task
classifier = pipeline("sentiment-analysis")
print(classifier("I love NLP!"))
# [{'label': 'POSITIVE', 'score': 0.9998}]

ner = pipeline("ner", grouped_entities=True)
print(ner("Elon Musk founded SpaceX in California."))

summarizer = pipeline("summarization", model="facebook/bart-large-cnn")
translator = pipeline("translation_en_to_fr", model="Helsinki-NLP/opus-mt-en-fr")
qa = pipeline("question-answering")

When you'd use this: Every NLP project. Start with pipelines for prototyping, switch to Auto classes for production.

HuggingFace Datasets Library

The companion to transformers. Provides thousands of datasets in a unified format with streaming for datasets too large for memory. Uses Apache Arrow under the hood — memory-mapped, not loaded into RAM.

from datasets import load_dataset

dataset = load_dataset("imdb")
# Streaming for huge datasets
large = load_dataset("c4", "en", split="train", streaming=True)

# Preprocessing pattern
from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")
tokenized = dataset["train"].map(
    lambda x: tokenizer(x["text"], padding="max_length", truncation=True, max_length=512),
    batched=True
)

When you'd use this: Loading any standard NLP benchmark, or when you need efficient data preprocessing for fine-tuning.

Model Cards — Read Before You Deploy

Every model on HuggingFace Hub has a model card documenting training data, intended use cases, known biases, and evaluation results. Before using any model in production, check: (1) what data was it trained on, (2) what are the intended use cases, (3) what are the known biases, (4) what's the license. A sentiment model trained on movie reviews will fail on financial text.

When you'd use this: Always. Before deploying any pretrained model.

Evaluation Metrics for NLP

Choosing the right metric matters more than choosing the right model. Here's the decision table:

MetricMeasuresBest ForWeakness
BLEUN-gram precision vs referenceMachine translationPurely surface-level; penalizes valid paraphrases
ROUGEN-gram recall vs referenceSummarizationSame surface-level problem as BLEU
chrFCharacter n-gram F-scoreMT for morphologically rich languagesDoesn't capture semantic similarity
BERTScoreEmbedding cosine similarityAny generation task needing semantic matchingRequires GPU; black box
COMETLearned quality estimationMT (best correlation with human judgment)Requires GPU; not interpretable
Exact Match / F1Span overlapExtractive QAToo strict for open-ended answers
PerplexityModel surprise on held-out textLanguage model comparisonDepends on tokenizer; not comparable across models
 
import evaluate

# BLEU
bleu = evaluate.load("bleu")
result = bleu.compute(predictions=["The cat is on the mat."], references=[["The cat sat on the mat."]])
print(f"BLEU: {result['bleu']:.4f}")

# ROUGE
rouge = evaluate.load("rouge")
result = rouge.compute(predictions=["ML models learn from data."], references=["Machine learning models improve from data."])
print(f"ROUGE-1: {result['rouge1']:.4f}, ROUGE-L: {result['rougeL']:.4f}")

# BERTScore (captures semantic similarity)
bertscore = evaluate.load("bertscore")
result = bertscore.compute(predictions=["The cat rested on the carpet."], references=["The cat sat on the mat."], lang="en")
print(f"BERTScore F1: {result['f1'][0]:.4f}")

When you'd use these: BLEU for MT, ROUGE for summarization, BERTScore when paraphrasing is common, COMET for high-stakes MT evaluation. Always combine with human evaluation for production.

Data Augmentation for NLP

Back-translation: Translate text to another language and back. "The cat sat on the mat" → French → "The cat sat on the carpet." Natural variation while preserving meaning. Most effective augmentation technique for low-resource NLP.

Easy Data Augmentation (EDA): Four simple operations — synonym replacement, random insertion, random swap, random deletion. Applied with 10–20% probability. Embarrassingly simple, surprisingly effective (1–3% accuracy gain on small datasets).

When you'd use this: When you have fewer than 1,000 labeled examples and can't easily get more. Back-translation for quality; EDA for speed.

Explainability for NLP

LIME for text: Randomly removes words and observes how predictions change. Model-agnostic — works with any classifier. Slow (many forward passes per explanation) and sometimes unstable.

Attention visualization: Tempting to use transformer attention weights as explanations, but research shows attention doesn't always correlate with feature importance. Use as one signal, not ground truth.

When you'd use this: When stakeholders ask "why did the model predict this?" — especially in regulated domains (finance, healthcare).

Sentence Segmentation

Splitting text into sentences sounds trivial until you hit abbreviations (Mr., U.S., Fig. 3), URLs, and informal text. Rule-based approaches maintain exception lists. ML approaches (NLTK's Punkt, spaCy's sentencizer) learn boundary patterns from data. In practice, spaCy's default segmentation works for most English text. For specialized domains, train a custom model.

When you'd use this: As a preprocessing step for any document-level task — summarization, translation, sentence-level classification.

Zero-Shot Classification

Classify text without any training data by leveraging a model trained on natural language inference (NLI). The model evaluates whether the text "entails" each candidate label. pipeline("zero-shot-classification") handles this automatically.

from transformers import pipeline

classifier = pipeline("zero-shot-classification")
result = classifier(
    "I need to book a flight to Tokyo next week",
    candidate_labels=["travel", "finance", "food", "technology"]
)
print(f"{result['labels'][0]}: {result['scores'][0]:.3f}")  # travel: 0.96+

When you'd use this: When you have no labeled data and need a quick baseline. Won't match a fine-tuned model, but requires zero training.

The Practical NLP Workflow

The pattern that works in production: (1) Start with a TF-IDF + logistic regression baseline — it takes 5 minutes and costs nothing. (2) Try zero-shot classification with an LLM to see if the task is feasible. (3) If accuracy matters, fine-tune a small transformer (DistilBERT, ~66M params) on your labeled data. (4) Only scale up to larger models if the small one doesn't meet your bar. Most NLP tasks don't need GPT-4.
