Machine Translation

Chapter 11: Natural Language Processing 3 subtopics
TL;DR

Machine translation went from hand-written rules (1950s) to statistical phrase tables (1990s) to neural sequence-to-sequence models with attention (2014+) to massive multilingual transformers covering 200+ languages (today). The attention mechanism was literally invented for translation, and it changed all of deep learning. Modern models like NLLB and mBART handle languages with almost no training data by transferring knowledge across language families. Evaluation uses BLEU (n-gram overlap) and chrF (character-level F-score), though neither fully captures translation quality.

Historical Approaches

Rule-Based Machine Translation

The earliest MT systems (1950s–1980s) used hand-crafted linguistic rules. Linguists wrote thousands of grammar rules, morphological transformations, and bilingual dictionaries. To translate from English to Russian, you'd parse the English sentence into a syntax tree, apply transformation rules to restructure it into Russian syntax, then generate Russian words from the dictionary. This is rule-based MT.

It sort of worked for simple, domain-specific text — weather reports, technical manuals. But language is messy. Idioms don't translate literally. Word order varies wildly across languages. Ambiguity is everywhere ("bank" means riverbank or financial bank?). Every new language pair needed a new set of rules, written by expensive linguists who couldn't agree on the rules anyway. The approach simply couldn't scale.

Statistical Machine Translation

Statistical MT replaced hand-written rules with learned probabilities. The core idea comes from the noisy channel model: assume the foreign sentence f is a "noisy" version of an English sentence e. To translate, find the English sentence that most likely produced the foreign text. Formally: argmax_e P(e|f) = argmax_e P(f|e) · P(e). The term P(f|e) is the translation model (learned from parallel corpora — sentence pairs in both languages). The term P(e) is the language model (learned from monolingual English text, ensuring fluent output).

Alignment models learned which words in the source map to which words in the target. IBM Models 1–5 (yes, they're numbered) progressively handle more complex alignment phenomena: one-to-many mappings, word reordering, fertility (one source word producing multiple target words). These were trained with the EM algorithm on parallel text.

Phrase-based SMT improved word-level alignment by working with phrases — contiguous sequences of words. Instead of translating word by word, the system learns that "in spite of" maps to "malgré" as a unit. This captured local word order and multi-word expressions far better than word-level models. Google Translate used phrase-based SMT from 2006 to 2016, when it switched to neural MT overnight — and users immediately noticed the improvement.

Neural Machine Translation

Encoder-Decoder for MT

The encoder-decoder architecture (Sutskever et al., 2014) was the first successful neural approach to MT. The encoder is an RNN (typically LSTM) that reads the source sentence token by token and compresses it into a fixed-length vector — the "context vector." The decoder is another RNN that takes this context vector and generates the target sentence token by token.

The fatal flaw: everything about a potentially long source sentence must be squeezed into a single fixed-size vector. For short sentences, this works fine. For long sentences, the information bottleneck destroys performance. The decoder literally can't remember the beginning of a 40-word sentence by the time it needs to translate the end.

Attention-Based NMT

Attention (Bahdanau et al., 2015) solved the bottleneck by letting the decoder look back at all encoder hidden states at every generation step. Instead of compressing the entire source into one vector, the decoder computes a weighted sum over all encoder states, with weights that depend on what the decoder is currently trying to generate. When translating a word that corresponds to the third source word, the model "attends to" position 3.

This was revolutionary. Not only did it fix the long-sentence problem, it produced interpretable alignment matrices — you could visualize which source words each target word attended to, and the alignments looked remarkably like what a human translator would produce. Attention became the foundation for the Transformer architecture (Vaswani et al., 2017), which replaced RNNs entirely with self-attention and cross-attention. Every modern NLP model — BERT, GPT, T5, LLaMA — descends from this idea that was invented for translation.

from transformers import pipeline

# Neural MT with HuggingFace
translator_en_de = pipeline("translation_en_to_de", model="Helsinki-NLP/opus-mt-en-de")
translator_en_fr = pipeline("translation_en_to_fr", model="Helsinki-NLP/opus-mt-en-fr")

text = "Attention was originally invented to solve the information bottleneck in machine translation."

de_result = translator_en_de(text)
fr_result = translator_en_fr(text)

print(f"English: {text}")
print(f"German:  {de_result[0]['translation_text']}")
print(f"French:  {fr_result[0]['translation_text']}")

Multilingual Models

Early neural MT required a separate model for every language pair. English→French was one model, English→German was another, French→German was a third. With 100 languages, you'd need 9,900 models (100 × 99). This is absurd.

Multilingual models train a single model on many language pairs simultaneously. The key insight: languages share structure. The concept of "subject-verb-object" exists across languages, even if the order differs. A model that learns to translate English→French can transfer some of that knowledge to English→Spanish because French and Spanish share Latin roots and similar grammar.

mBERT (multilingual BERT) was trained on Wikipedia text in 104 languages. Despite never being explicitly told which language is which, it learned cross-lingual representations — you can fine-tune it on English NER data and it works surprisingly well on German NER with zero German training examples. This zero-shot cross-lingual transfer was one of the most surprising findings in NLP.

mBART extends the multilingual approach to sequence-to-sequence tasks. It's pretrained with a denoising objective (corrupt text, reconstruct it) in 25 languages, then fine-tuned for translation. Because it learned the structure of many languages during pretraining, it needs very few parallel sentence pairs to learn a new language pair.

NLLB (No Language Left Behind), released by Meta in 2022, is the most ambitious multilingual MT model to date. It covers over 200 languages, including many low-resource languages that previous systems ignored entirely. NLLB uses a sparse mixture-of-experts architecture to handle the enormous diversity — different expert sub-networks activate for different language families, so adding Zulu doesn't hurt French performance. For many low-resource language pairs, NLLB improved BLEU scores by over 44% compared to previous state-of-the-art.

from transformers import AutoModelForSeq2SeqLM, AutoTokenizer

# NLLB: translate between 200+ languages
model_name = "facebook/nllb-200-distilled-600M"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

text = "Machine translation has made the world more connected."

# English → Hindi
tokenizer.src_lang = "eng_Latn"
inputs = tokenizer(text, return_tensors="pt")
translated = model.generate(
    **inputs,
    forced_bos_token_id=tokenizer.convert_tokens_to_ids("hin_Deva")
)
print(f"Hindi: {tokenizer.decode(translated[0], skip_special_tokens=True)}")

# English → Swahili
translated = model.generate(
    **inputs,
    forced_bos_token_id=tokenizer.convert_tokens_to_ids("swh_Latn")
)
print(f"Swahili: {tokenizer.decode(translated[0], skip_special_tokens=True)}")

Zero-Shot Translation and Off-Target Translation

Zero-shot translation means translating between a language pair the model was never explicitly trained on. If a multilingual model was trained on English↔French and English↔German, can it translate French→German directly? Often, yes — the shared multilingual representation acts as an interlingua. But the quality is usually lower than the trained directions, and the model sometimes exhibits off-target translation: producing output in the wrong language. The model might translate French→German but accidentally output some English words because English was the dominant pivot language during training.

Low-Resource MT

Most of the world's 7,000+ languages have little to no parallel training data. Low-resource MT tackles this with several strategies: transfer learning from related high-resource languages (training on Spanish helps with Portuguese), back-translation (use a target→source model to generate synthetic parallel data), multilingual pretraining (learn shared representations across many languages), and leveraging monolingual data (abundant even for low-resource languages). NLLB's success on 200+ languages shows this isn't just academic — it's production-ready.

MT Evaluation

BLEU Score

BLEU is the standard automatic metric for MT. It computes n-gram precision: what fraction of the n-grams in the machine translation also appear in the reference? It uses modified precision (each reference n-gram can only be "used" once, preventing cheating by repeating common words) and a brevity penalty (short translations get penalized). The final BLEU score is a geometric mean of 1-gram through 4-gram precision, multiplied by the brevity penalty.

BLEU ranges from 0 to 100. A BLEU of 30+ is generally understandable. A BLEU of 50+ is high quality. Human translators typically score 60–80 against other human references (they don't score 100 because there are many valid translations for any sentence). BLEU is fast, cheap, and reproducible — but it only captures surface-level overlap. Two equally good translations with different word choices will have low BLEU against each other.

import evaluate

bleu = evaluate.load("bleu")

# Single example
prediction = ["The cat is sitting on the mat."]
reference = [["The cat sat on the mat.", "A cat is on the mat."]]

result = bleu.compute(predictions=prediction, references=reference)
print(f"BLEU: {result['bleu']:.4f}")
print(f"Precisions: {[f'{p:.3f}' for p in result['precisions']]}")
print(f"Brevity penalty: {result['brevity_penalty']:.4f}")

chrF

chrF (character n-gram F-score) computes F-score over character-level n-grams instead of word-level. This has several advantages over BLEU: it handles morphologically rich languages better (German, Finnish, Turkish — where word forms vary enormously), it doesn't need tokenization (important for languages without clear word boundaries like Chinese), and it's more robust to minor inflectional differences ("translated" vs "translating" share many character n-grams but zero word n-grams).

chrF++ extends chrF by also including word-level bigrams, combining the best of character-level and word-level matching. Many researchers now report chrF alongside or instead of BLEU, especially for non-English target languages.

COMET and Learned Metrics

COMET (Crosslingual Optimized Metric for Evaluation of Translation) is a learned metric that uses a multilingual encoder to compare the source, machine translation, and reference. Unlike BLEU and chrF, it captures semantic similarity — paraphrases score well even without word overlap. COMET scores correlate significantly better with human judgments than BLEU, which is why the MT community is gradually shifting toward it. The cost: you need a GPU to run the metric, and it's a black box (you can't easily explain why it gave a particular score).

import evaluate

# chrF evaluation
chrf = evaluate.load("chrf")

prediction = ["The cat is sitting on the mat."]
reference = [["The cat sat on the mat."]]

result = chrf.compute(predictions=prediction, references=reference)
print(f"chrF: {result['score']:.2f}")

# Compare BLEU and chrF on morphological variation
pred_morph = ["I am translating this document carefully."]
ref_morph = [["I translated this document with care."]]

bleu_metric = evaluate.load("bleu")
bleu_result = bleu_metric.compute(predictions=pred_morph, references=ref_morph)
chrf_result = chrf.compute(predictions=pred_morph, references=ref_morph)
print(f"\nMorphological test:")
print(f"  BLEU: {bleu_result['bleu']:.4f}  (penalizes word form changes)")
print(f"  chrF: {chrf_result['score']:.2f}   (more forgiving of inflections)")
Which MT Metric Should You Use?

Report BLEU for comparability with older papers — it's the lingua franca of MT evaluation. Add chrF for morphologically rich target languages. Use COMET if you want the most accurate automatic assessment of quality and have GPU resources. For high-stakes deployments, always include human evaluation — have bilingual annotators rate fluency and adequacy on a sample of outputs. No automatic metric is a perfect proxy for translation quality.

The Translation Quality Ladder

Rule-based MT: reliable on narrow domains, terrible everywhere else. Statistical MT (phrase-based): fluent for common patterns, fragile for long sentences. Early neural MT (encoder-decoder): smooth output, but loses track on long inputs. Attention-based NMT: strong across sentence lengths, single language pairs. Multilingual transformers (mBART, NLLB): good quality across hundreds of languages from a single model. Each step was a genuine paradigm shift — not just incremental improvement — driven by better architectures and dramatically more data.
