Recurrent Models: RNN, LSTM, GRU

The sequential bottleneck that motivates attention and transformers

Can't understand why attention was needed without understanding what came before — recurrent loop. Loop-arrow diagrams, hidden state feeding back into itself, powered machine translation, speech recognition, language modeling for two decades. Key question: how does network carry information through sequence, and why does that mechanism break?

RNNs dominated sequence modeling from early 1990s through 2017. Vanilla RNN formalized late 1980s. Long Short-Term Memory (LSTM) arrived 1997, Hochreiter & Schmidhuber. Gated Recurrent Unit (GRU) followed 2014, Cho et al. Together powered MT, speech, language modeling for two decades before transformers swept field.

Why Order Matters

Building tiny weather predictor. Three days of observations: hot, hot, cold. Job: predict tomorrow. Shuffle to cold, hot, hot — prediction should change. Trend reversed. Order carries meaning.

Language works same way. "Dog bites man" vs "Man bites dog" — identical words, swap order, meaning flips. Audio too — same frequencies in different sequence = different word. Stock prices, DNA, sensor readings — all sequential. Each element's identity depends on position relative to neighbors.

Standard feedforward network takes one fixed-size vector, produces one output. No notion of "before" or "after." Could flatten sequence into single long vector — concatenate all three days — but locks you into fixed length. Four days? Hundred? Also throws away temporal structure that makes data meaningful.

Need network that reads inputs one at a time, in order, and remembers what it's seen. That's recurrence.

The Sticky Note — Hidden State as Memory

Weather observations arrive one day at a time. You have sticky note. Each morning: read new temperature, glance at yesterday's sticky note, write new summary combining both. Hand updated note to tomorrow-self. That sticky note = hidden state.

More precisely: hidden state = vector (list of numbers) encoding compressed summary of everything seen so far. Updated every time step. Network's sole mechanism for carrying past information into future.

Entire vanilla RNN captured by single equation:

hₜ = tanh(W_x · xₜ + W_h · hₜ₋₁ + b)

xₜ = input at time step t (today's observation). hₜ₋₁ = hidden state from previous step (yesterday's sticky note). W_x = weight matrix transforming input, W_h = weight matrix transforming previous hidden state, b = bias. tanh squashes result into [-1, 1].

Critical detail: W_x and W_h are same matrices at every time step. Network reuses same parameters for day 1 or day 100. This weight sharing allows RNNs to handle any-length sequences with fixed parameter count.

Walking Through a Vanilla RNN by Hand

Concrete example, smallest possible. Weather station encodes temperatures as numbers: hot = 1.0, mild = 0.5, cold = 0.0. Hidden state = single number (one-dimensional, traceable by hand). Start with h₀ = 0 — blank sticky note.

Suppose trained weights: W_x = 0.5, W_h = 0.8, b = 0. Feed sequence [hot, hot, cold] = [1.0, 1.0, 0.0].

Day 1 (hot, x₁ = 1.0): Combine new input with blank sticky note.

h₁ = tanh(0.5 × 1.0 + 0.8 × 0.0) = tanh(0.5) ≈ 0.46

Sticky note now reads 0.46 — network registered one hot day.

Day 2 (hot, x₂ = 1.0): Another hot day. Previous state carries forward.

h₂ = tanh(0.5 × 1.0 + 0.8 × 0.46) = tanh(0.5 + 0.37) = tanh(0.87) ≈ 0.70

Sticky note now 0.70 — higher. Two consecutive hot days pushed hidden state up. Network encoding "hot streak."

Day 3 (cold, x₃ = 0.0): Temperature drops to zero.

h₃ = tanh(0.5 × 0.0 + 0.8 × 0.70) = tanh(0.56) ≈ 0.51

Hidden state dropped but didn't reset to zero. Memory of hot days still there, fading but present — 0.8 multiplier on previous state kept old information alive. Final hidden state 0.51 feeds into output layer for tomorrow's prediction.

That's entire forward pass of vanilla RNN. One equation, applied repeatedly.

Unrolling Through Time

Drawing RNN as box with arrow looping back looks compact but hides actual computation. Unrolling = copy box once per time step, lay copies left to right, making information flow explicit.

For three-day weather sequence, unrolled view:

  x₁(hot)      x₂(hot)      x₃(cold)
    ↓             ↓             ↓
 [RNN Cell] → [RNN Cell] → [RNN Cell] → h₃ → prediction
  h₀=0  →  h₁=0.46  →  h₂=0.70  →  h₃=0.51

Every box = same cell with same weights. Arrows between = hidden state passed forward (sticky note traveling through time). Unrolled picture is what we compute in forward pass and backpropagate through during training.

In code, unrolling = literally a for loop:

import torch, torch.nn as nn

class VanillaRNN(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        self.W_x = nn.Linear(input_dim, hidden_dim, bias=False)
        self.W_h = nn.Linear(hidden_dim, hidden_dim, bias=False)
        self.b   = nn.Parameter(torch.zeros(hidden_dim))
        self.fc  = nn.Linear(hidden_dim, output_dim)
        self.hidden_dim = hidden_dim

    def forward(self, x):                          # x: (batch, seq_len, input_dim)
        batch, seq_len, _ = x.size()
        h = torch.zeros(batch, self.hidden_dim, device=x.device)
        for t in range(seq_len):                   # the unrolling
            h = torch.tanh(self.W_x(x[:, t]) + self.W_h(h) + self.b)
        return self.fc(h)                          # predict from final hidden state

That for loop is the unrolling. Each iteration applies same weights to new input and carried-forward hidden state. Clean, elegant — and deeply flawed.

Backpropagation Through Time

Training RNN = computing gradients w.r.t. shared weights to update them. Computation graph is unrolled chain of cells, so backpropagation flows backward through each time step. Called Backpropagation Through Time (BPTT).

Key quantity: how does change in hidden state at early step affect loss at end? Chain rule. Gradient of loss w.r.t. hidden state at step k passes through every intermediate step between k and final step T:

∂L/∂hₖ = ∂L/∂hₜ · ∂hₜ/∂hₜ₋₁ · ∂hₜ₋₁/∂hₜ₋₂ · ... · ∂hₖ₊₁/∂hₖ

Each intermediate derivative ∂hⱼ/∂hⱼ₋₁ involves multiplying by weight matrix W_h and derivative of tanh. Full gradient = product of many such terms — one per time step between k and T.

Here's where trouble starts.

The Vanishing Gradient Catastrophe

Vanishing gradient clicked for me when I stopped reading about eigenvalues and did arithmetic on paper.

Derivative of tanh maxes at 1.0, in practice usually 0.1–0.5. Say effective per-step multiplier = 0.9. Sounds close to 1. But multiply repeatedly:

After  5 steps: 0.9⁵  = 0.59
After 10 steps: 0.9¹⁰ = 0.35
After 25 steps: 0.9²⁵ = 0.072
After 50 steps: 0.9⁵⁰ = 0.0052

After 50 steps, gradient shrunk to half a percent. Gone. Network cannot learn that something 50 steps ago matters now. Information might still be in hidden state faintly — but gradient signal needed to learn that relationship has evaporated.

This = vanishing gradient problem. Gradient of loss w.r.t. early hidden states decays exponentially through time.

Flip side: per-step multiplier > 1 (say 1.1) → 1.1⁵⁰ ≈ 117. Gradient explodes, loss = NaN, training collapses. This = exploding gradient problem, fixable with gradient clipping: if gradient norm exceeds threshold, scale it down. Clipping handles explosions but does nothing for vanishing. Can't clip gradient back into existence.

Like game of telephone. One person whispers to next, who whispers to next. After 50 people, original message unrecognizable. Vanilla RNN = playing telephone with gradient signal across time. Longer sequence = worse distortion.

In practice, vanilla RNNs learn dependencies spanning ~10–20 steps. Beyond that = effective amnesia. Not optimizer or learning rate problem — architectural. Multiplicative hidden state update guarantees gradients either vanish or explode. No stable middle ground.

Field lived with this limitation for years. Everyone knew it was a problem. Question: how to fix it.

LSTM — The Conveyor Belt

LSTM equations intimidated me for years. Four lines of sigmas, tanh, element-wise products. What finally clicked: physical metaphor.

Picture conveyor belt in factory. Items ride along belt end to end. At each station, workers can: remove items (forget), place new items on belt (input), inspect belt to make report (output). Belt moves forward steadily. Items stay unless worker actively removes them.

That conveyor belt = LSTM's cell state. Separate vector running alongside hidden state through time. Critical insight: cell state updated using addition, not multiplication. Items placed on or removed from belt, but belt doesn't get squeezed through bottleneck. This lets information (and gradients) travel long distances without decay.

Long Short-Term Memory (Hochreiter & Schmidhuber, 1997) controls conveyor belt with three gates. Building up one at a time, using weather station.

The Forget Gate

Back at weather station: cell state tracking "warm spell." Cold front arrives. Network needs way to say: "warm spell info? Let go. No longer relevant." That's forget gate.

fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)

Forget gate looks at previous hidden state (hₜ₋₁) and current input (xₜ), concatenates, multiplies by learned weight matrix W_f, adds bias, passes through sigmoid (σ). Sigmoid outputs 0–1 per element. 1 = "keep entirely." 0 = "erase completely." Between = partial forgetting.

Gate produces vector same size as cell state. Multiplied element-wise with previous cell state — conveyor belt workers deciding, per item, keep or toss.

Practical detail: forget gate bias typically initialized to 1.0, not 0.0. Jozefowicz et al. (2015) showed this matters enormously. Bias at zero → σ(0) = 0.5, network forgets half its memory from first step. Initialized to 1 → σ(1) ≈ 0.73, network defaults to remembering. Learns to forget later. This one trick = difference between LSTM that works and one that struggles with long-range patterns.

The Input Gate

Network needs to decide what new information to write onto conveyor belt. Two parts.

First, network generates candidate memory — proposal for new information:

C̃ₜ = tanh(W_c · [hₜ₋₁, xₜ] + b_c)

Uses tanh, producing values [-1, 1]. Network composing new sticky note: "today cold after two hot days — temperature drop."

Second, input gate decides how much candidate to actually write:

iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)

Again sigmoid, again 0–1. Input gate = conveyor belt worker deciding: "put this new item on belt? How much?" Routine input → gate stays low. Novel, important input (sudden temperature drop) → gate opens wide.

The Key Line — Additive Cell State

Now comes single most important equation in this chapter. Cell state update:

Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ

⊙ = element-wise multiplication. Read left to right: take old cell state (Cₜ₋₁), multiply element-wise by forget gate (fₜ) — erases stuff we decided to forget. Then add new candidate (C̃ₜ) scaled by input gate (iₜ) — places new items on belt.

That plus sign is everything. Compare to vanilla RNN: hidden state update was tanh(W_h · h + W_x · x) — completely multiplicative. Here, cell state update is additive. When forget gate ≈ 1 and input gate ≈ 0, equation becomes Cₜ ≈ Cₜ₋₁. Cell state passes unchanged. Because it passes unchanged, gradients during backprop also pass unchanged — no exponential decay, no vanishing.

Think of it: vanilla RNN forces every piece of information through narrow doorway (matrix multiplication + tanh) at every step. LSTM provides express lane — cell state — that bypasses doorway. Information on express lane travels far without being squeezed. Gates = on-ramps and off-ramps controlling what enters/exits express lane.

The Output Gate

Cell state carries long-term memory, but we don't always want to expose all of it. Output gate controls what portion becomes visible as hidden state — passed to next layer or used for prediction.

oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)
hₜ = oₜ ⊙ tanh(Cₜ)

Cell state passed through tanh (normalize to [-1, 1]) then filtered by output gate. Weather station example: cell state might remember both temperature trend and season. For today's prediction, maybe trend matters more. Output gate selectively reveals only relevant parts.

All four equations together:

fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)        # forget gate: what to erase
iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)        # input gate: how much new info to write
C̃ₜ = tanh(W_c · [hₜ₋₁, xₜ] + b_c)     # candidate: the new info itself
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ              # cell state update (THE KEY LINE)
oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)        # output gate: what to reveal
hₜ = oₜ ⊙ tanh(Cₜ)                     # hidden state: the visible output

Notice pattern. Every gate has same structure: concatenate previous hidden state + current input, multiply by learned weight matrix, add bias, pass through sigmoid. Differ only in learned weights — each gate independently learns what to care about. Candidate memory = same structure but tanh instead of sigmoid (producing content, not filter).

Peephole Connections and Practical Tips

Standard LSTM: gates base decisions on previous hidden state + current input. Gers, Schmidhuber, Cummins (2000) pointed out: shouldn't gates peek at cell state too? Cell state = where long-term memory lives. Hidden state = filtered view. Gates making decisions based on secondhand summary.

Peephole connections add cell state as additional input to each gate. Empirically improve tasks requiring precise timing or counting — like producing signal after exactly N steps.

In practice, standard LSTMs without peepholes = default. Peepholes = niche optimization. Interview answer: "They let gates look at cell state directly, helps timing-sensitive tasks, rarely used because improvement marginal for most applications."

More practical notes:

Gradient clipping — non-negotiable for recurrent networks. Even with LSTMs, gradients can spike. Max gradient norm 1.0 or 5.0 = standard.

Orthogonal initialization for hidden-to-hidden weight matrix (W_h) maintains gradient norms during backprop. Orthogonal matrix has eigenvalues with magnitude 1 — neither amplifies nor shrinks. Exactly what we want for long-range gradient flow.

Truncated BPTT — compromise for very long sequences. Instead of backpropagating through entire sequence, chop into windows (say 100 steps), backprop within each. Lose ability to learn dependencies longer than window, but save memory and computation. Most real-world LSTM training uses some truncation.

GRU — Same Idea, Fewer Moving Parts

LSTM works. Solved vanishing gradient problem, dominated field for nearly two decades. But lots of machinery — three gates, separate cell state, four weight matrices. In 2014, Cho et al. asked: can we get same benefit with less complexity?

Gated Recurrent Unit merges cell state and hidden state into single vector, reduces three gates to two.

Update gate decides how much old state to keep vs how much new info to write. Combined role of LSTM's forget + input gates:

zₜ = σ(W_z · [hₜ₋₁, xₜ] + b_z)

Reset gate controls how much previous hidden state to use when computing candidate. When reset ≈ 0, candidate computed almost entirely from current input — network starting fresh:

rₜ = σ(W_r · [hₜ₋₁, xₜ] + b_r)

Candidate state uses reset gate to optionally suppress previous hidden state:

h̃ₜ = tanh(W · [rₜ ⊙ hₜ₋₁, xₜ] + b)

Final state = interpolation between old and new:

hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ

Look at last line. zₜ ≈ 0 → keep old hidden state (LSTM equivalent: forget=1, input=0). zₜ ≈ 1 → replace with candidate (forget everything, write new). (1 - z) and z coupled — constrained to sum to 1 — so GRU can't independently control forgetting and updating. LSTM can: forget and input gates independent, so it can simultaneously erase old + write new, or keep old + write new. GRU forces trade-off.

Is coupling a problem? Empirically, most benchmarks: GRUs match LSTMs. Small datasets: GRUs sometimes better (~25% fewer params, less overfitting). Very long-range dependencies + plenty of data: LSTMs occasionally edge ahead. Difference rarely decisive. Start with GRU for speed. Switch to LSTM if gap appears.

Conveyor belt analogy: GRU = factory that merged belt and inspection station into one thing. Instead of separate belt (long-term) and output (short-term), one state serves both purposes. More compact, handles most workloads, costs some flexibility.

Bidirectional RNNs

So far, our weather predictor reads observations left to right — from past to future. That makes sense for prediction: you can't use tomorrow's data to predict tomorrow. But for other tasks, the constraint is artificial.

Consider classifying the sentiment of a weather report: "The forecast was grim, but then the sun broke through." If you only read left to right, by the time you reach "grim" you don't know about the "but" that reverses everything. If you also read right to left, the word "grim" gets context from both directions — the gloom that precedes it and the reversal that follows it.

A bidirectional RNN runs two separate RNNs over the same sequence: one forward (left to right) and one backward (right to left). At each time step, the two hidden states are concatenated to form a richer representation that captures context from both past and future.

Forward:    h₁→  h₂→  h₃→  h₄→
Backward:   ←h₁  ←h₂  ←h₃  ←h₄
Combined:  [h₁→;←h₁]  [h₂→;←h₂]  [h₃→;←h₃]  [h₄→;←h₄]

The combined hidden state at each position has twice the dimension. If each directional RNN has a hidden size of 256, the concatenated output is 512. The two RNNs have separate weights — they learn different things. The forward RNN learns patterns based on what came before. The backward RNN learns patterns based on what comes after. Together they give each position a full-context representation.

Bidirectional RNNs are workhorses for tasks where the entire sequence is available upfront: sentiment analysis, named entity recognition, part-of-speech tagging, and the encoder half of sequence-to-sequence models. They cannot be used for real-time generation — you can't peek at future tokens when you're generating them one at a time.

encoder = nn.LSTM(
    input_size=128,
    hidden_size=256,
    num_layers=2,           # stacked: 2 layers deep
    bidirectional=True,     # forward + backward
    batch_first=True,
    dropout=0.3,            # dropout between layers
)
# Output hidden dim per step: 256 × 2 = 512

Stacking layers — putting one RNN on top of another — is the other common extension. The hidden state of layer l at time t becomes the input to layer l+1 at the same time step. Lower layers tend to capture local patterns (today's temperature), while higher layers capture more abstract patterns (seasonal trends). Google's Neural Machine Translation system (2016) was an 8-layer stacked LSTM. Adding residual connections when going beyond 2–3 layers helps gradient flow — the same principle as in deep feedforward networks.

The Sequential Bottleneck

The LSTM and GRU solved the vanishing gradient problem. That was an enormous achievement. But they left another problem untouched: the sequential bottleneck.

Look back at the unrolled RNN. Computing h₂ requires h₁. Computing h₃ requires h₂. Every step depends on the one before it. You cannot compute them in parallel. For a sequence of length 500, you must perform 500 sequential operations. On a GPU that can do thousands of operations simultaneously, this is heartbreaking — you're using a fraction of the available power.

It gets worse. In a sequence-to-sequence model for translation, the encoder compresses the entire input sentence into a single fixed-length vector — the final hidden state. Everything the decoder needs to know about a 50-word sentence must fit into one vector. That vector becomes a bottleneck, and information gets lost. We'll see in the next section how the attention mechanism alleviates this by letting the decoder look back at every encoder hidden state, not the final one. And in the section after that, we'll see how the transformer eliminates the sequential dependency entirely.

Recurrent models dominated sequence processing from the late 1990s through 2017. Then Vaswani et al. published "Attention Is All You Need," and the era of recurrence began to close. The sequential bottleneck — the inability to parallelize — was the main reason. Attention connects every position to every other position directly, with no sequential chain, and training becomes massively parallel.

But knowing why recurrence was abandoned makes the motivation for attention click instantly. That's why this chapter exists.
