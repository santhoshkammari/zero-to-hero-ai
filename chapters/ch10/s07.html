Nice to Know

Chapter 10: Sequence Models & Attention Interesting but not essential — look these up when you need them
TL;DR

A dozen ideas you'll encounter in papers, blog posts, and production codebases. None of them are required to understand the core chapter material, but recognizing them on sight will save you hours of confused Googling. Skim now, bookmark for later.

Echo State Networks / Reservoir Computing

What if you never trained the recurrent weights at all? In an Echo State Network, you initialize a large recurrent "reservoir" with random fixed weights and only train a linear output layer on top. The reservoir acts like a random nonlinear feature extractor for temporal data. Training is blazingly fast — it's just linear regression on the reservoir states.

The catch: it works surprisingly well for simple time-series forecasting and signal processing, but it can't compete with learned representations on complex language or vision tasks. Look it up when: you need a dirt-cheap baseline for time-series, or you're working on neuromorphic / edge hardware where backprop through time is too expensive.

Clockwork RNNs

A Clockwork RNN splits its hidden state into groups, and each group updates at a different clock rate — some every timestep, some every 2, some every 4, and so on. The fast groups capture short-term dynamics, the slow groups capture long-term trends, and information flows from slow groups to fast groups (but not vice versa).

It's an elegant way to handle multi-scale temporal patterns without the complexity of full LSTM gating. Think of it like having dedicated slow-motion and fast-forward channels in the same network. Look it up when: you're modeling data with clear multi-timescale structure (e.g., music with beat and phrase levels, or sensor data with fast oscillations and slow drift).

IndRNN (Independently Recurrent Neural Network)

Standard RNNs have a full recurrent weight matrix where every hidden neuron connects to every other. IndRNN makes that matrix diagonal — each neuron only recurs with itself. This sounds limiting, but it means you can use ReLU activations (instead of tanh) without exploding gradients, and the network can learn across thousands of timesteps.

You stack multiple IndRNN layers to get cross-neuron interaction, similar to how depthwise separable convolutions factorize spatial and channel mixing. Look it up when: you need an RNN that handles very long sequences and you can't use a Transformer for some reason (memory constraints, streaming, etc.).

Transformer-XL

Transformer-XL introduced segment-level recurrence: when processing the next segment of text, it caches the hidden states from the previous segment and lets the current segment attend to them. Combined with relative positional encoding (so the model knows "3 tokens back" rather than "position 47"), this lets the effective context grow linearly with depth without the quadratic attention cost of processing everything in one shot.

It was the key precursor to XLNet and influenced how modern LLMs handle long contexts. Look it up when: you need to understand how models extend context beyond a single fixed window, or you're reading the XLNet paper.

Reformer

The Reformer attacks quadratic attention with two tricks: locality-sensitive hashing (LSH) to find similar keys without comparing all pairs (reducing attention to O(n log n)), and reversible layers to avoid storing activations for backprop (cutting memory from O(n·L) to O(n)). On paper, it's a clever combination.

In practice, LSH attention is complex to implement, the constant factors are large, and Flash Attention ended up solving the memory problem more cleanly. Reformer is historically important but rarely used today. Look it up when: you see it cited in efficient-attention papers and want to understand the lineage.

Mixture of Experts (MoE) in Transformers

A Mixture of Experts layer replaces the single feed-forward network in a Transformer block with many "expert" FFNs, and a learned router picks the top-k experts for each token. So a model can have 400B total parameters but only activate 12B per token — you scale parameters (and knowledge capacity) without proportionally scaling compute.

Switch Transformer (Google, 2021) pioneered this at scale with top-1 routing. Mixtral (Mistral, 2023) made it practical and open-source with 8 experts, 2 active per token. The tricky parts are load balancing across experts and communication overhead in distributed training. Look it up when: you want to understand how Mixtral, GPT-4 (rumored MoE), or Gemini achieve their parameter counts without proportional inference cost.

Retrieval-Augmented Generation (RAG)

RAG combines a generator (typically a Transformer LM) with a retriever that fetches relevant documents from an external knowledge base at inference time. Instead of memorizing all facts in its weights, the model looks things up. REALM and RETRO baked retrieval into pretraining; practical RAG pipelines today typically bolt a vector database (Pinecone, Weaviate, FAISS) onto an existing LLM.

RAG is how most production LLM applications handle domain-specific or up-to-date knowledge — it's cheaper and more controllable than fine-tuning. Look it up when: you're building any LLM-powered application that needs to answer questions about specific documents, databases, or knowledge that wasn't in the training data.

Prefix Tuning / Prompt Tuning / LoRA

Parameter-efficient fine-tuning (PEFT) methods train a tiny fraction of parameters while keeping the base model frozen. Prefix tuning prepends learnable "virtual tokens" to each layer's input. Prompt tuning only prepends them to the input layer. LoRA (Low-Rank Adaptation) injects small trainable low-rank matrices into the attention weights — typically training less than 1% of total parameters with results close to full fine-tuning.

LoRA is the dominant approach today because it's simple, composable (you can swap LoRA adapters at serving time), and works across model sizes. Look it up when: you need to fine-tune a large model and can't afford to update all parameters, which is basically every real-world fine-tuning scenario in 2024+.

Constitutional AI / RLHF

RLHF (Reinforcement Learning from Human Feedback) is how ChatGPT-style models learn to be helpful, harmless, and honest. The recipe: (1) supervised fine-tuning on demonstrations, (2) train a reward model on human preference comparisons, (3) optimize the LLM against that reward model using PPO. DPO (Direct Preference Optimization) simplifies this by skipping the explicit reward model entirely — it turns preference data directly into a training objective.

Constitutional AI (Anthropic) takes it further by having the model critique and revise its own outputs using a set of written principles, reducing the need for human labeling. Look it up when: you want to understand why LLMs behave the way they do in chat, or you're building an application that needs alignment or safety guardrails.

Knowledge Distillation for Transformers

Knowledge distillation trains a small "student" model to mimic the outputs (and often the internal representations) of a large "teacher" model. DistilBERT is 60% the size of BERT with 97% of its performance. TinyBERT distills attention maps and hidden states layer-by-layer. MiniLM focuses on distilling the self-attention distributions themselves.

The key insight: soft probability distributions from a teacher carry far more information than hard labels — the teacher's uncertainty about wrong answers is itself informative. Look it up when: you need to deploy a model under latency or memory constraints, or you're building a mobile/edge application and can't serve a full-size Transformer.

Attention Rollout / Gradient-weighted Attention

Raw attention weights from a single layer are misleading because they ignore residual connections — information flows through skip connections too, not just through attention. Attention rollout fixes this by recursively multiplying attention matrices across layers, accounting for the residual stream. Gradient-weighted attention goes further by weighting attention by the gradient of the output with respect to the attention weights, highlighting which attention patterns actually matter for the prediction.

Both give a more honest picture of "what the model is looking at" than naive single-layer attention visualization. Look it up when: you're doing interpretability work and need more principled visualization than just plotting one layer's attention heatmap.

Ring Attention

Ring Attention distributes the attention computation for very long sequences across multiple GPUs arranged in a logical ring. Each GPU holds a chunk of the sequence and computes local attention, then passes KV blocks to the next GPU in the ring. By overlapping communication with computation, the entire sequence gets full attention without any single GPU needing to hold the complete KV cache.

It's a systems-level technique, not an approximation — you get exact attention, just distributed. Combined with Flash Attention on each node, it's how models handle million-token contexts. Look it up when: you're working on distributed inference or training for extremely long sequences and need to go beyond what fits on a single GPU.
