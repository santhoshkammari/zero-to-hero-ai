Recurrent Models: RNN, LSTM, GRU

Chapter 10: Sequence Models & Attention Historical foundations · The memory problem · Gates
TL;DR

RNNs process sequences by carrying a hidden state forward through time — but repeated matrix multiplication kills gradients over long ranges. LSTMs fix this with three gates and an additive cell state that lets gradients flow. GRUs simplify LSTMs to two gates with comparable performance. All three are largely historical now — transformers dominate — but you need this foundation to understand why attention was such a breakthrough.

Why Feedforward Nets Can't Handle Sequences

Swap two pixels in a photo and nothing changes. Swap two words in a sentence — "dog the bit man the" — and you get garbage. Sequential data has order, and the meaning of each element depends on what came before it. Language, audio, stock prices, DNA — all sequential.

A feedforward network sees its entire input at once, with no notion of "first" or "last." You could flatten a sequence into one big vector, but that fixes the length (what about variable-length sentences?) and destroys temporal structure entirely. We need something that reads inputs one at a time and remembers.

The RNN: Hidden State as Memory

The core idea is dead simple. At each time step, the network reads one input, combines it with what it remembers from before, and writes an updated memory. That memory is a vector called the hidden state — think of it as a sticky note the network passes to itself at each step.

One equation captures the whole thing:

hₜ = tanh(Wₕ hₜ₋₁ + Wₓ xₜ + b)

xₜ is the input at time t. hₜ₋₁ is the previous hidden state. Wₓ and Wₕ are weight matrices, and they're the same at every time step — this weight sharing is what lets RNNs handle any sequence length with a fixed number of parameters.

Here's a minimal RNN forward pass in PyTorch:

import torch
import torch.nn as nn

class VanillaRNN(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        self.W_x = nn.Linear(input_dim, hidden_dim, bias=False)
        self.W_h = nn.Linear(hidden_dim, hidden_dim, bias=False)
        self.b   = nn.Parameter(torch.zeros(hidden_dim))
        self.fc  = nn.Linear(hidden_dim, output_dim)
        self.hidden_dim = hidden_dim

    def forward(self, x):
        # x: (batch, seq_len, input_dim)
        batch, seq_len, _ = x.size()
        h = torch.zeros(batch, self.hidden_dim, device=x.device)
        for t in range(seq_len):
            h = torch.tanh(self.W_x(x[:, t]) + self.W_h(h) + self.b)
        return self.fc(h)  # classify using final hidden state

That for loop is the entire RNN. Clean, elegant — and deeply flawed.

The Vanishing Gradient Problem

This is the single most important thing to understand about RNNs, because it's the reason everything else in this chapter exists.

During backpropagation through time, gradients flow backward through each time step by multiplying the same weight matrix over and over. Think of it like this:

  • If the largest eigenvalue of that matrix is less than 1 → the gradient shrinks exponentially. By step 50 it's essentially zero. The network can't learn long-range dependencies. This is the vanishing gradient.
  • If the largest eigenvalue is greater than 1 → the gradient grows exponentially. Your loss becomes NaN. This is the exploding gradient (fixable with gradient clipping, but that doesn't help vanishing).
The core issue

Vanilla RNNs use multiplicative updates to the hidden state. Multiplying the same matrix T times either vanishes or explodes — there's no stable middle ground. The network can learn patterns spanning ~10–20 steps, but beyond that it has amnesia. This isn't a training trick problem. It's architectural.

The fix? Stop relying purely on multiplication. Use addition.

LSTM: Gates to the Rescue

The Long Short-Term Memory network (Hochreiter & Schmidhuber, 1997) introduces a separate cell state — a highway that runs through time using addition instead of multiplication. Gradients can flow along this highway without being repeatedly squashed.

Three gates control what enters, exits, and persists in the cell state:

  • Forget gate (fₜ) — what to erase from memory
  • Input gate (iₜ) — what new information to write
  • Output gate (oₜ) — what to expose to the next layer

The equations — don't memorize them, just notice the pattern:

fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)        # forget gate
iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)        # input gate
C̃ₜ = tanh(W_c · [hₜ₋₁, xₜ] + b_c)     # candidate memory
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ              # cell state update ← THE KEY LINE
oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)        # output gate
hₜ = oₜ ⊙ tanh(Cₜ)                     # hidden state
Why this works

Look at the cell state update: Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ. It's an addition. When the forget gate is close to 1 and the input gate is close to 0, the cell state passes through unchanged — gradients flow straight back through time with no decay. That's the whole trick. Addition creates a gradient highway.

All gates use sigmoid (σ) because it outputs values between 0 and 1 — perfect for "how much to let through." The candidate memory uses tanh to produce values in [-1, 1]. Every gate sees the same inputs (previous hidden state + current input) but has its own learned weights.

GRU: Same Idea, Fewer Moving Parts

The Gated Recurrent Unit (Cho et al., 2014) asks: do we really need three gates and a separate cell state? Turns out, two gates and one state work nearly as well.

zₜ = σ(W_z · [hₜ₋₁, xₜ])          # update gate (≈ forget + input combined)
rₜ = σ(W_r · [hₜ₋₁, xₜ])          # reset gate
h̃ₜ = tanh(W · [rₜ ⊙ hₜ₋₁, xₜ])   # candidate state
hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ   # interpolate old and new

The update gate zₜ decides how much of the old state to keep vs. replace — it plays the roles of both the forget and input gates. The reset gate rₜ controls how much of the previous state to use when computing the candidate. No separate cell state. Fewer parameters. Faster training.

Comparison: RNN vs LSTM vs GRU

Vanilla RNN LSTM GRU
Gates 0 3 (forget, input, output) 2 (update, reset)
Separate cell state No Yes No
Parameters (hidden dim n, input dim d) n² + nd + n 4(n² + nd + n) 3(n² + nd + n)
Long-range memory Poor (~10–20 steps) Good (hundreds of steps) Good (hundreds of steps)
Training speed Fastest Slowest Middle
When to use Very short sequences, toy problems Default choice for sequential tasks pre-2017 When LSTM works but you want faster training / less overfitting on small data
Practical note

In most benchmarks, LSTM and GRU perform comparably. The "right" choice usually depends more on your dataset size and compute budget than on architectural superiority. If you're unsure, start with GRU (fewer parameters, faster iteration), and switch to LSTM only if you see a gap.

Bidirectional and Stacked Variants

Two quick extensions that apply to all three architectures:

  • Bidirectional — run two RNNs (or LSTMs/GRUs) over the sequence: one forward, one backward. Concatenate their hidden states at each step. Now every position sees context from both directions. Can't be used for real-time generation (you need the full sequence upfront), but great for classification and encoding.
  • Stacked (deep) — stack 2–4 layers. The hidden state of layer l at time t becomes the input to layer l+1 at time t. Lower layers capture local patterns, higher layers capture abstract meaning. Add residual connections if going beyond 2 layers.
# Both variants in one PyTorch call
rnn = nn.LSTM(
    input_size=128,
    hidden_size=256,
    num_layers=3,         # stacked
    bidirectional=True,   # bidirectional
    batch_first=True,
    dropout=0.3,          # between layers
)
# Output hidden dim = 256 * 2 = 512 (bidirectional doubles it)

Where We Are Now

RNNs, LSTMs, and GRUs dominated sequence modeling from the late 1990s through ~2017. Machine translation, speech recognition, language modeling — all built on recurrent architectures. Google's Neural Machine Translation system (2016) was an 8-layer stacked LSTM.

Then came the transformer. It replaced recurrence entirely with attention — a mechanism that lets every position look at every other position directly, with no sequential bottleneck. Training became massively parallelizable. Long-range dependencies became trivial. That's the next section.

Why learn this at all?

Because you'll encounter LSTMs in legacy codebases, time-series applications, and edge deployment where transformer overhead is too high. And because understanding why RNNs failed at long-range dependencies makes the motivation for attention click instantly.
