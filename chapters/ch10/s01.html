Recurrent Neural Networks

Chapter 10: Sequence Models & Attention 7 subtopics
TL;DR

Recurrent Neural Networks process sequences one step at a time, carrying a hidden state that acts as a compressed memory of everything seen so far. The vanilla RNN applies the same weight matrix at every time step — elegant, but that repeated multiplication causes gradients to vanish (or explode) over long sequences. Tricks like gradient clipping, truncated BPTT, and architectural variants (bidirectional, stacked, Elman vs Jordan) help, but the fundamental vanishing gradient problem is what motivated LSTMs and GRUs, which we cover next.

The Sequential Data Problem

Images are grids, but language is a sequence — order matters. Swap two pixels in a photo and you barely notice. Swap two words in a sentence and you get nonsense ("dog the bit man the" vs "the dog bit the man"). The same is true for audio waveforms, stock prices, sensor readings, and DNA sequences. The data arrives one element at a time, and the meaning of each element depends on what came before it.

A feedforward network can't handle this naturally. You could flatten a sequence into one giant input vector, but that fixes the length — what about sentences of different sizes? And it throws away all temporal structure. The word "bank" means something different after "river" than after "investment." A feedforward net sees both words simultaneously with no notion of which came first.

We need an architecture that processes inputs one at a time, in order, and somehow remembers what it has already seen. That architecture is the Recurrent Neural Network, or RNN.

Hidden State: The Network's Memory

The core idea is simple. At each time step, the network reads one input and updates an internal vector called the hidden state. Think of it as a sticky note the network passes to itself. After reading the first word, it scribbles a summary on the note. After reading the second word, it reads the old note, combines it with the new word, and writes an updated note. By the time it reaches the end of the sentence, that note contains a compressed summary of the entire sequence.

Formally, the hidden state hₜ at time step t is a vector (typically 128 to 1024 dimensions) that encodes everything the network has seen from time steps 1 through t. It is not the output — it is the internal memory. You can tap into it at any time step to produce an output, or you can wait until the end and use the final hidden state as a representation of the whole sequence.

The initial hidden state h₀ is usually a vector of zeros. Some architectures learn it as a parameter instead. Either way, it is the "blank slate" the network starts with before seeing any data.

The Vanilla RNN Equations

The vanilla RNN (sometimes called the Elman RNN) computes the hidden state with one equation:

hₜ = tanh(Wₕ hₜ₋₁ + Wₓ xₜ + b)

Let's unpack every piece. xₜ is the input at time step t — maybe a word embedding vector of dimension d. hₜ₋₁ is the hidden state from the previous step, dimension n. Wₓ is a matrix of shape (n, d) that projects the input into hidden space. Wₕ is a matrix of shape (n, n) that transforms the previous hidden state. b is a bias vector of dimension n. The tanh squashes everything into the range [-1, 1].

The crucial insight: Wₕ, Wₓ, and b are the same at every time step. This is called weight sharing across time. Whether the network is processing the first word or the hundredth, it applies the same transformation. This is what makes RNNs handle variable-length sequences — the number of parameters does not depend on sequence length. It also acts as a form of regularization, similar to how convolutional filters share weights across spatial positions.

Here is a minimal vanilla RNN forward pass from scratch in PyTorch:

import torch
import torch.nn as nn

class VanillaRNN(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        self.hidden_dim = hidden_dim
        # Weight matrices — shared across all time steps
        self.W_x = nn.Linear(input_dim, hidden_dim, bias=False)
        self.W_h = nn.Linear(hidden_dim, hidden_dim, bias=False)
        self.b = nn.Parameter(torch.zeros(hidden_dim))
        self.fc = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        # x shape: (batch, seq_len, input_dim)
        batch_size, seq_len, _ = x.size()
        h = torch.zeros(batch_size, self.hidden_dim, device=x.device)

        for t in range(seq_len):
            h = torch.tanh(self.W_x(x[:, t, :]) + self.W_h(h) + self.b)

        # Use final hidden state for classification
        return self.fc(h)

# Quick smoke test
model = VanillaRNN(input_dim=32, hidden_dim=64, output_dim=10)
dummy = torch.randn(4, 15, 32)  # batch=4, seq_len=15, features=32
out = model(dummy)
print(out.shape)  # torch.Size([4, 10])

Notice the for-loop over time steps. That loop is the heart of the RNN. Each iteration reads one element of the sequence, mixes it with the previous hidden state, and produces a new hidden state. The final h is then projected to an output.

Unfolding Through Time

An RNN looks like a single box with an arrow looping back into itself. That is compact but confusing. To reason about what actually happens during a forward pass, we unfold (or unroll) the network across time. Imagine making T copies of the same box, one per time step, connected left to right by the hidden state. Each copy shares the same weights — they are the same network, just drawn out horizontally.

This unfolded view is not just a visualization trick. It is literally how we compute gradients. Once unfolded, an RNN looks like a very deep feedforward network where each "layer" corresponds to a time step. And we can apply the standard backpropagation algorithm to this unfolded graph. That algorithm has a name.

Backpropagation Through Time (BPTT)

Backpropagation Through Time is just backpropagation applied to the unfolded RNN. The loss at each time step depends on the hidden state at that step, which depends on the hidden state at the previous step, all the way back to h₀. The chain rule propagates gradients backward through every time step.

For a sequence of length T, the gradient of the loss with respect to Wₕ involves a product of T Jacobian matrices — one for each time step. Each Jacobian is the derivative of hₜ with respect to hₜ₋₁. When you multiply T matrices together, the result either explodes or vanishes exponentially depending on the largest eigenvalue of the weight matrix. We will return to this problem shortly.

Truncated BPTT is a practical compromise. Instead of backpropagating through the entire sequence (which might be thousands of steps), you chop it into chunks of length k (commonly 20–50 steps). The forward pass still runs over the full sequence, carrying the hidden state forward. But the backward pass only unrolls k steps at a time. You lose some long-range gradient signal, but training becomes feasible in terms of both memory and compute. Most practical RNN training uses truncated BPTT.

Elman Networks vs Jordan Networks

The architecture we described above — where the hidden state feeds back into itself — is an Elman network. It was proposed by Jeff Elman in 1990. The recurrence is hidden-to-hidden: hₜ depends on hₜ₋₁.

A Jordan network, proposed by Michael Jordan in 1986, uses a different recurrence: the previous output feeds back as input instead of the previous hidden state. So the update becomes hₜ = tanh(Wₓ xₜ + Wᵧ yₜ₋₁ + b), where yₜ₋₁ is the network's output at the previous time step.

The practical difference: Elman networks maintain richer internal memory because the hidden state is never forced through an output bottleneck. Jordan networks can be easier to train in some cases because the output is typically lower-dimensional and more directly supervised. In modern practice, the Elman architecture dominates. When someone says "RNN" without qualification, they almost always mean an Elman-style recurrence.

Bidirectional RNNs

Consider the sentence: "He deposited money in the ___." You would guess "bank." Now consider: "He caught fish by the ___." Also "bank," but a different kind. To disambiguate a word in the middle of a sentence, you need context from both directions. A standard RNN only looks left-to-right, so by the time it reaches a word, it has only seen what came before.

A Bidirectional RNN runs two separate RNNs over the same sequence. One reads left-to-right (forward), producing hidden states h→ₜ. The other reads right-to-left (backward), producing hidden states h←ₜ. At each time step, you concatenate the two: hₜ = [h→ₜ ; h←ₜ]. This doubled hidden state now captures context from the entire sequence at every position.

The catch: bidirectional RNNs require the full sequence upfront. You cannot use them for real-time generation or streaming applications where future tokens have not arrived yet. They are perfect for tasks like classification, named entity recognition, or machine translation encoding — anywhere you can see the whole input before producing output.

Deep (Stacked) RNNs

A single-layer RNN applies one transformation per time step. A stacked RNN (or deep RNN) stacks multiple RNN layers on top of each other. The hidden state of the first layer at time t becomes the input to the second layer at time t, and so on. Typically, 2–4 layers are used. Going deeper helps the network learn hierarchical representations — lower layers might capture local patterns (syntax), while higher layers capture global meaning (semantics).

As you stack layers, training becomes harder. Residual connections help: instead of feeding the output of layer l directly into layer l+1, you add the input back: output = RNNLayer(input) + input. This gives gradients a shortcut path, easing optimization. Google's Neural Machine Translation system famously used an 8-layer stacked LSTM with residual connections.

Here is how you use PyTorch's built-in RNN module, including stacking and bidirectionality:

import torch
import torch.nn as nn

class StackedBiRNN(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim,
                 output_dim, num_layers=2, dropout=0.3):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.rnn = nn.RNN(
            input_size=embed_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            bidirectional=True,
            dropout=dropout,       # dropout between layers (not last)
        )
        # bidirectional doubles the hidden dim
        self.fc = nn.Linear(hidden_dim * 2, output_dim)

    def forward(self, x):
        # x: (batch, seq_len) — token indices
        embedded = self.embedding(x)          # (batch, seq_len, embed_dim)
        output, h_n = self.rnn(embedded)      # output: (batch, seq_len, hidden*2)

        # h_n shape: (num_layers * 2, batch, hidden)
        # Grab last layer's forward and backward final states
        fwd = h_n[-2]   # last layer, forward
        bwd = h_n[-1]   # last layer, backward
        combined = torch.cat([fwd, bwd], dim=1)  # (batch, hidden*2)
        return self.fc(combined)

# Usage
model = StackedBiRNN(vocab_size=5000, embed_dim=128,
                     hidden_dim=256, output_dim=4, num_layers=3)
tokens = torch.randint(0, 5000, (8, 50))  # batch=8, seq_len=50
logits = model(tokens)
print(logits.shape)  # torch.Size([8, 4])
Why batch_first=True?

PyTorch's RNN layers default to shape (seq_len, batch, features), which is confusing if you come from a computer vision background where batch is always first. Setting batch_first=True switches to (batch, seq_len, features). Always set it — your future self will thank you when debugging tensor shapes.

The Vanishing Gradient Problem

This is the core problem of vanilla RNNs, and honestly, the reason this section exists as a stepping stone to LSTMs rather than an endpoint.

Remember BPTT: gradients flow backward through time by repeatedly multiplying the Jacobian of hₜ with respect to hₜ₋₁. For the vanilla RNN, that Jacobian is diag(1 - hₜ²) · Wₕ (the derivative of tanh times the recurrent weight matrix). When you multiply this matrix by itself T times, the result depends on the spectral radius — the largest absolute eigenvalue — of the effective weight matrix.

If the spectral radius is less than 1, the product shrinks exponentially toward zero. Gradients from time step 100 effectively cannot reach time step 1. The network forgets long-range dependencies. This is the vanishing gradient problem.

If the spectral radius is greater than 1, gradients explode exponentially. Your loss becomes NaN after a few updates. This is the exploding gradient problem.

There is no stable middle ground for vanilla RNNs. Tanh helps a little (its derivative is at most 1, which damps the hidden-to-hidden signal), but it is not enough. The network can learn patterns spanning 10–20 time steps, but beyond that, vanilla RNNs effectively have amnesia.

Gradient Clipping

Gradient clipping is a band-aid for exploding gradients, but an effective one. There are two flavors:

Clipping by value: each element of the gradient vector is clamped independently to the range [-threshold, threshold]. Simple, but it can change the direction of the gradient vector.

Clipping by norm: if the global norm of the gradient exceeds a threshold, you rescale the entire vector so its norm equals the threshold. This preserves the direction while capping the magnitude. Almost everyone uses this version in practice. A threshold of 1.0 or 5.0 is common.

# Gradient clipping in PyTorch — two lines
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()

Clipping fixes exploding gradients but does nothing for vanishing gradients. For that, you need architectural changes — LSTMs, GRUs, or attention mechanisms.

Training Techniques for RNNs

Variational Dropout

Standard dropout samples a new random mask at every time step. This is noisy and destroys the hidden state's ability to carry information across time. Variational dropout (Gal & Ghahramani, 2016) fixes this by sampling one dropout mask per sequence and reusing it at every time step. The same hidden units are dropped for the entire sequence, which acts more like a coherent Bayesian approximation and gives much better results in practice.

Zoneout

Zoneout is a regularization technique where, instead of dropping hidden units to zero, you randomly keep the previous hidden state for some units. At each time step, each hidden dimension has a probability p of "zoning out" — ignoring the new update and keeping its old value. This creates a stochastic identity connection through time, which helps gradients flow and acts as a form of implicit residual connection.

Padding and Masking for Variable-Length Sequences

Real datasets have sequences of different lengths. A batch needs tensors of uniform shape, so we pad shorter sequences with zeros (or a special PAD token) to match the longest sequence in the batch. But we don't want the network to treat padding as real data. A mask is a binary tensor that marks which positions are real and which are padding. During the forward pass, we zero out hidden state updates at padded positions. During loss computation, we exclude padded positions from the average.

PyTorch provides pack_padded_sequence and pad_packed_sequence utilities that handle this efficiently. They skip computation at padded positions entirely, saving both time and avoiding numerical noise from processing garbage inputs.

Sorting by Length

When using pack_padded_sequence, your batch must be sorted by sequence length in descending order (longest first). This is a legacy requirement from CuDNN optimization. Modern PyTorch versions support enforce_sorted=False to handle unsorted batches, but sorting is still faster on GPU.

RNN Variants Worth Knowing

The vanilla RNN spawned several creative variants beyond LSTMs and GRUs. Most are niche, but three deserve a brief mention:

Echo State Networks (ESNs) take a radical approach: freeze all recurrent weights at random initialization and only train the output layer. The randomly initialized recurrent layer (called the "reservoir") is large and sparsely connected, creating rich dynamics that the linear output layer learns to read. ESNs train blazingly fast — it is a simple linear regression — and work surprisingly well for time-series prediction. They are part of a broader family called reservoir computing.

Clockwork RNNs partition the hidden state into groups that update at different frequencies. One group updates every time step, another every 2 steps, another every 4, and so on. This hardcoded multi-scale structure lets the network capture patterns at different time scales without needing to learn them, and it naturally helps with long-range dependencies because slow-ticking groups carry information further with fewer multiplications.

IndRNN (Independently Recurrent Neural Network) replaces the dense recurrent weight matrix Wₕ with a diagonal matrix — meaning each hidden neuron only recurs with itself, not with other neurons. Cross-neuron interaction happens through the input-to-hidden weights and stacking multiple layers. This simple change makes it much easier to control the spectral radius (you just bound diagonal elements) and allows RNNs to be trained on sequences thousands of steps long, rivaling LSTMs on some benchmarks.

Where We Stand

The vanilla RNN is a clean, elegant idea: share one set of weights across time, carry memory in a hidden state. It works for short sequences and remains a useful pedagogical tool. But the vanishing gradient problem is not a minor inconvenience — it is a fundamental limitation. No amount of gradient clipping or clever dropout fixes the fact that multiplying the same matrix hundreds of times destroys the gradient signal.

That is why the field moved to gated architectures. LSTMs add explicit gates that control what the hidden state remembers and forgets. GRUs simplify the gating. Both solve the vanishing gradient problem by creating additive paths through time, rather than purely multiplicative ones. We cover those next.
