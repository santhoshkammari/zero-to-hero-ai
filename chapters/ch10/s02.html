LSTM

Chapter 10: Sequence Models & Attention 4 subtopics
TL;DR

Vanilla RNNs forget things fast because gradients vanish over long sequences. LSTMs fix this with a cell state — a dedicated memory highway that runs through every time step, controlled by three learned gates (forget, input, output). The forget gate erases irrelevant memories, the input gate writes new ones, and the output gate decides what to expose. Because the cell state flows with mostly linear operations, gradients travel back through hundreds of steps without shrinking to zero. That single architectural trick — a gated memory highway — turned sequence models from a research curiosity into a production workhorse.

By the end of the last section you saw the brutal reality of vanilla RNNs: they can't remember anything more than about 10–20 steps back. The gradients either vanish into nothing or explode into infinity. In 1997, Sepp Hochreiter and Jürgen Schmidhuber published a fix that would dominate sequence modelling for almost two decades. Their idea was deceptively simple: give the network an explicit, protected memory channel and let it learn what to remember and what to forget. That's the Long Short-Term Memory network — the LSTM.

The Cell State: A Memory Highway

Here's the key innovation and the one thing you need to understand before anything else. Imagine you're on a long road trip, and you carry a notebook. At every stop, you can scribble new notes, erase old ones, or read from it — but the notebook itself travels with you the entire journey, untouched unless you deliberately modify it. That notebook is the cell state, often written as Cₜ.

In a vanilla RNN, all information gets squeezed through the hidden state hₜ, which is rewritten at every step via a tanh activation. That's like being forced to recite your entire memory out loud at every stop, then reconstruct it from what you just said. Lossy doesn't begin to describe it.

The cell state is different. It runs along the top of the network like a conveyor belt. Information on this belt flows from one step to the next through simple element-wise operations — a multiplication here, an addition there. No matrix multiplications squashing things through nonlinearities. This is what gives the LSTM its long-range memory: gradients can flow backward along this belt almost unimpeded. The network then uses three gates to control what gets erased from the belt, what gets added, and what gets read off of it.

The Forget Gate

First question at every time step: what old information is no longer relevant? Back to the notebook analogy — you're now in Paris, so maybe erase your notes about London restaurant reservations. The forget gate handles this.

It looks at the previous hidden state hₜ₋₁ (what you just said) and the current input xₜ (what's happening now), concatenates them, runs them through a sigmoid, and produces a vector of values between 0 and 1 — one value for each dimension of the cell state:

fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)

A value of 1 means "keep this memory completely." A value of 0 means "erase it entirely." Anything in between is a partial erase. When we later multiply this vector element-wise with the old cell state, dimensions near zero get wiped and dimensions near one survive. That's the eraser on your notebook — a learned, differentiable eraser.

The Input Gate

Next question: what new information should we write down? This happens in two parts, because we need to decide both what to write and how much of it to write.

First, we create a set of candidate values — the raw new notes we could write. This uses a tanh activation, which squashes values into the range [-1, 1]:

C̃ₜ = tanh(W_c · [hₜ₋₁, xₜ] + b_c)

Then, the input gate decides how much of each candidate to actually commit to memory. Like the forget gate, it's a sigmoid producing values between 0 and 1:

iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)

Think of it this way: the tanh part drafts a full page of notes, and the sigmoid part highlights which sentences actually matter. Only the highlighted parts get written into the notebook.

Updating the Cell State

Now we combine the erasing and writing into one clean update. The new cell state is:

Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ

That's it. Multiply the old cell state by the forget gate (erase what's irrelevant), then add the new candidate values scaled by the input gate (write what's relevant). The * here means element-wise multiplication, not matrix multiplication. Every dimension of the cell state is independently gated.

This equation is the heart of the entire LSTM. Stare at it. The old cell state Cₜ₋₁ can pass through almost unchanged if fₜ ≈ 1 and iₜ ≈ 0. That linear path is exactly why gradients don't vanish — there's no squashing nonlinearity in the way. The gates themselves are nonlinear, but the cell state update is additive. Addition is gradient-friendly.

The Output Gate

The cell state now holds your updated notebook. But you don't blurt out your entire notebook at every conversation turn. You decide what's relevant to say right now. That's the output gate.

First, a sigmoid decides which parts of the cell state to expose:

oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)

Then we push the cell state through a tanh (to squash values back to [-1, 1]) and multiply element-wise by the output gate:

hₜ = oₜ * tanh(Cₜ)

The result hₜ is the hidden state — what the LSTM "says out loud" at this time step. It's a filtered, curated version of the full memory. This hidden state gets passed to the next time step and optionally to an output layer for predictions. Notice the asymmetry: the cell state is the full internal memory, while the hidden state is the public-facing summary. The network learns to keep far more in the cell state than it ever reveals.

Gate Mechanics: Why Sigmoid and Tanh?

You might wonder why we use sigmoid for gates and tanh for candidate values. It's not arbitrary.

Sigmoid outputs values in [0, 1]. That's a natural range for "how much to let through" — it acts as a soft switch. Zero means block everything, one means let everything pass. When you multiply a signal by a sigmoid output, you're scaling it between "completely off" and "completely on." Perfect for gating.

Tanh outputs values in [-1, 1]. Candidate values need to be able to both increase and decrease the cell state. If candidates were only positive, the cell state could only grow — it would never subtract information. The [-1, 1] range lets the network write both positive and negative updates, keeping the cell state centered and numerically stable.

Every operation between gates and the cell state is element-wise (Hadamard product). Each dimension of the cell state has its own independent gate value. Dimension 42 might be remembering the subject of a sentence while dimension 137 tracks verb tense. They're gated independently, so the network can forget the subject without losing the tense, or vice versa. This fine-grained control is a big part of why LSTMs work so well in practice.

LSTM Forward Pass from Scratch

Let's implement a single LSTM cell to see the gates in action. No magic — just matrix multiplications and activations:

import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-np.clip(x, -500, 500)))

class LSTMCell:
    def __init__(self, input_size, hidden_size):
        self.hidden_size = hidden_size
        scale = 0.1
        combined = input_size + hidden_size
        # One weight matrix for all four gates (efficient)
        self.W = np.random.randn(4 * hidden_size, combined) * scale
        self.b = np.zeros(4 * hidden_size)

    def forward(self, x, h_prev, c_prev):
        combined = np.concatenate([h_prev, x])
        # Compute all four gates in one matrix multiply
        gates = self.W @ combined + self.b
        H = self.hidden_size

        # Split into four gate pre-activations
        f_gate = sigmoid(gates[0:H])       # forget gate
        i_gate = sigmoid(gates[H:2*H])     # input gate
        c_cand = np.tanh(gates[2*H:3*H])   # candidate values
        o_gate = sigmoid(gates[3*H:4*H])   # output gate

        # Update cell state: erase old, write new
        c_new = f_gate * c_prev + i_gate * c_cand

        # Compute hidden state: filtered read of cell state
        h_new = o_gate * np.tanh(c_new)

        return h_new, c_new

# Example usage
input_size, hidden_size, seq_len = 10, 32, 5
cell = LSTMCell(input_size, hidden_size)

h = np.zeros(hidden_size)
c = np.zeros(hidden_size)

# Process a sequence step by step
for t in range(seq_len):
    x = np.random.randn(input_size)
    h, c = cell.forward(x, h, c)
    print(f"Step {t}: h mean={h.mean():.4f}, c mean={c.mean():.4f}")

Notice how we concatenate all four gates into a single large weight matrix W. In practice, every framework does this. One big matrix multiply is much faster on a GPU than four small ones. We split the result after the multiplication.

Using PyTorch's nn.LSTM

In practice, you'll never write the cell from scratch. PyTorch's nn.LSTM handles the math, batching, and GPU optimization. Here's how you'd use it for a simple sequence task:

import torch
import torch.nn as nn

class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim,
                 num_classes, num_layers=2, dropout=0.3):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(
            input_size=embed_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout,
            bidirectional=True  # read sequence both ways
        )
        # bidirectional doubles the hidden dim
        self.classifier = nn.Linear(hidden_dim * 2, num_classes)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # x shape: (batch, seq_len)
        embedded = self.dropout(self.embedding(x))

        # lstm_out: (batch, seq_len, hidden_dim * 2)
        # h_n: (num_layers * 2, batch, hidden_dim)
        lstm_out, (h_n, c_n) = self.lstm(embedded)

        # Use the final hidden states from both directions
        # h_n[-2] is last layer forward, h_n[-1] is last layer backward
        final_hidden = torch.cat([h_n[-2], h_n[-1]], dim=-1)

        logits = self.classifier(self.dropout(final_hidden))
        return logits

# Example
model = LSTMClassifier(
    vocab_size=10000, embed_dim=128,
    hidden_dim=256, num_classes=4
)

# Fake batch: 8 sequences of length 50
batch = torch.randint(0, 10000, (8, 50))
output = model(batch)
print(f"Output shape: {output.shape}")  # (8, 4)

A few things to note. The batch_first=True argument makes the input shape (batch, seq_len, features) instead of the default (seq_len, batch, features). The LSTM returns both the full sequence of hidden states (lstm_out) and the final hidden and cell states (h_n, c_n). For classification, we typically use the final hidden state. For sequence-to-sequence tasks, we'd use the full lstm_out.

Peephole Connections

In the standard LSTM, the gates look at hₜ₋₁ and xₜ to make their decisions. But here's a subtle issue: the gates never directly see the cell state itself. They're deciding what to do with the notebook without actually reading it first.

Peephole connections, introduced by Gers and Schmidhuber in 2000, fix this by letting the gates peek at the cell state. The forget and input gates look at the previous cell state Cₜ₋₁, and the output gate looks at the current cell state Cₜ:

fₜ = σ(W_f · [hₜ₋₁, xₜ] + w_pf ⊙ Cₜ₋₁ + b_f)

iₜ = σ(W_i · [hₜ₋₁, xₜ] + w_pi ⊙ Cₜ₋₁ + b_i)

oₜ = σ(W_o · [hₜ₋₁, xₜ] + w_po ⊙ Cₜ + b_o)

The w_p terms are diagonal weight vectors (not full matrices) — the peephole is element-wise, so each gate dimension peeks at the corresponding cell state dimension. In practice, peepholes help on tasks that require precise timing, like learning to count or generate rhythmic patterns. For most NLP tasks, the improvement is marginal, which is why standard LSTMs without peepholes remain more common.

Coupled Forget-Input Gate

Here's an elegant simplification. Think about it: if you're erasing something from memory, you're probably making room to write something new. And if you're not erasing anything, you probably shouldn't be adding much either. The forget and input decisions are naturally coupled.

The coupled forget-input gate formalizes this by tying the two gates together:

iₜ = 1 − fₜ

Now the cell state update becomes:

Cₜ = fₜ * Cₜ₋₁ + (1 − fₜ) * C̃ₜ

This is a weighted interpolation between the old memory and the new candidate. Whatever fraction you forget, you replace. This reduces the parameter count (one fewer gate's worth of weights) and acts as a form of regularization. The GRU architecture, which we'll cover in the next section, takes this idea even further.

Bidirectional LSTM

A standard LSTM reads left to right. At time step t, it only knows about inputs from steps 0 through t. But in many tasks — sentiment analysis, named entity recognition, machine translation — the meaning of a word depends on what comes after it too. "I went to the bank to fish" vs. "I went to the bank to deposit money." You need the end of the sentence to disambiguate "bank."

A Bidirectional LSTM runs two separate LSTM layers: one reading the sequence forward, one reading it backward. At each time step, the hidden states from both directions are concatenated to give a representation that captures both past and future context:

hₜ = [h⃗ₜ ; h⃖ₜ]

This doubles the hidden dimension of the output, which is why you see hidden_dim * 2 in the PyTorch code above. The forward and backward LSTMs have completely separate parameters — they learn different things. The forward LSTM might learn that "not" modifies the next adjective, while the backward LSTM learns that "but" signals a contrast with a preceding clause.

The tradeoff is real: bidirectional LSTMs can't be used for autoregressive generation (you can't peek at future tokens if they don't exist yet). They're for tasks where you have the full sequence available upfront — classification, tagging, encoding.

Why LSTM Solves the Vanishing Gradient

Let's come back to the fundamental question: why does any of this gating actually fix the vanishing gradient problem?

Recall from the RNN section that gradients vanish because they're repeatedly multiplied by weight matrices and tanh derivatives as they flow backward through time. Each multiplication shrinks the gradient. After 50 steps, the gradient is effectively zero.

Now look at the LSTM's cell state update again:

Cₜ = fₜ * Cₜ₋₁ + iₜ * C̃ₜ

When we backpropagate through this equation, the gradient with respect to Cₜ₋₁ is simply fₜ. Not a weight matrix. Not a tanh derivative. Just the forget gate value. If the network learns to set fₜ ≈ 1 (which sigmoid can easily produce), the gradient flows through that time step with almost no attenuation. Chain this over 100 steps and the gradient arriving at step 1 is roughly the product of 100 forget gate values — each close to 1. Compare that with a vanilla RNN, where you'd multiply 100 weight matrices together.

This is the gradient highway. The cell state provides an unobstructed path for gradient flow, analogous to skip connections in ResNets (which were inspired partly by LSTMs). The network doesn't have to use this highway for every dimension — some dimensions might have small forget gate values, deliberately letting gradients decay for information that's no longer relevant. The network learns which memories to protect and which to let fade. That's the magic: it's not that gradients never vanish, it's that the network can learn to prevent them from vanishing for the specific information that matters.

The Honest Tradeoff

LSTMs are powerful but not free. Each cell has roughly four times the parameters of a simple RNN cell (four gate weight matrices instead of one). They're sequential by nature — you can't compute step t until step t-1 finishes — which means they can't fully exploit GPU parallelism the way convolutional or attention-based models can. For sequences up to a few hundred tokens, LSTMs work great. For sequences of thousands of tokens or more, Transformers (which we'll meet soon) have largely replaced them. But LSTMs remain the right tool when you need low-latency streaming inference — processing one token at a time as it arrives — because their constant memory footprint and fixed per-step cost are hard to beat.

With the LSTM in your toolkit, you can handle sequences hundreds of steps long. But there's still room to simplify. In the next section, we'll look at the GRU — a streamlined variant that merges some of the LSTM's gates and often achieves comparable performance with fewer parameters.
