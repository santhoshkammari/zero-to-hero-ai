Attention Mechanism

Chapter 10: Sequence Models & Attention 7 subtopics
TL;DR

Seq2seq models crush an entire input sentence into one fixed-size vector — a brutal bottleneck. Attention fixes this by letting the decoder peek back at every encoder hidden state at every step, computing a weighted average that highlights the most relevant input words. Bahdanau attention (additive) scores relevance with a small neural network; Luong attention (multiplicative) uses dot products for speed. The resulting attention weights are interpretable — you can literally see which source words the model focused on. Attention isn't just a seq2seq trick: it's a general-purpose information routing mechanism that became the foundation of Transformers.

The Information Bottleneck Problem

Here's the fundamental problem that plagued early seq2seq models. You have an encoder RNN that reads an entire input sentence — maybe 30 words, maybe 80 — and compresses it into a single fixed-size vector. That final hidden state is the only thing the decoder gets to work with. Every nuance, every word, every relationship in the input must survive that compression.

Think about how absurd that is. It's like asking someone to read a full page of text, then summarize everything they learned into a single number. Short sentences? Fine. But try compressing "The government of the Federal Republic of Germany announced on Tuesday that trade negotiations with the European Union regarding agricultural subsidies have been postponed until next quarter" into one vector of 256 floats. The early words get overwritten as the RNN processes later words. By the time you reach the period, "government" is a fading echo buried under layers of matrix multiplications.

This isn't a theoretical worry. Researchers observed it directly: seq2seq translation quality collapsed on sentences longer than about 20 words. The information bottleneck was real and measurable. BLEU scores dropped off a cliff as sentence length grew. The encoder's final hidden state simply couldn't hold enough information.

The decoder needs more than a summary. When it's generating the third output word, it doesn't need the entire input — it needs to know which specific part of the input is relevant right now. When translating "The cat sat on the mat" to French and generating "chat" (cat), the decoder should be looking hard at the word "cat," not at "mat" or "the." The decoder needs to look back at specific positions in the input, differently at each step.

The Core Idea: Dynamic Context Vectors

The attention mechanism solves the bottleneck with an elegant idea: instead of one fixed context vector, compute a different context vector for each decoder time step. Each context vector is a weighted sum of all encoder hidden states. The weights tell the decoder which input words to pay attention to at this particular moment.

Let's say your encoder processes "The cat sat on the mat" and produces hidden states h₁, h₂, h₃, h₄, h₅, h₆ — one for each word. In a vanilla seq2seq model, only h₆ (the last one) gets passed to the decoder. With attention, every h₁ through h₆ stays accessible. At each decoder step, the model computes a relevance score between the current decoder state and every encoder hidden state, normalizes those scores into a probability distribution, and takes a weighted average. The result is a context vector that's custom-built for that specific decoding step.

When generating "chat," the attention weights might be [0.02, 0.85, 0.03, 0.02, 0.01, 0.07] — heavily concentrated on h₂ (the hidden state for "cat"). When generating "assis" (sat), the weights shift to concentrate on h₃. The decoder dynamically routes information from exactly where it's needed.

Bahdanau (Additive) Attention

The first attention mechanism was proposed by Bahdanau, Cho, and Bengio in their 2014 paper. It's called additive attention because the score function adds the decoder state and encoder state together (after linear transformations) before passing through a tanh nonlinearity.

Here's how it works, step by step. At decoder time step i, you have the decoder's current hidden state sᵢ and the full set of encoder hidden states h₁, h₂, ..., hₜ. The score between sᵢ and each encoder state hⱼ is:

score(sᵢ, hⱼ) = vᵀ tanh(W₁sᵢ + W₂hⱼ)

W₁ and W₂ are learned weight matrices that project the decoder state and encoder state into a common space. The tanh squashes the sum. Then v (a learned vector) collapses it to a single scalar — the attention score. This is essentially a tiny one-hidden-layer neural network that answers: "How relevant is encoder position j to the current decoding step i?"

Let's walk through a concrete example. We're translating "The cat sat on the mat" to "Le chat est assis sur le tapis." The encoder (a bidirectional RNN) has produced hidden states h₁ through h₆. The decoder is at step 2, about to generate "chat." Its current hidden state is s₂.

The model computes six scores: score(s₂, h₁), score(s₂, h₂), ..., score(s₂, h₆). Suppose after the matrix multiplications and tanh, these come out as [-1.2, 3.8, 0.1, -0.5, -1.0, 0.3]. The score for h₂ ("cat") is by far the highest. Good — the model has learned that when generating the French word for cat, you should look at the English word "cat."

One subtle detail: Bahdanau attention uses the decoder state from the previous time step (sᵢ₋₁) to compute attention, then feeds the resulting context vector into the current step's RNN cell. This means attention is computed before the RNN update, not after.

import torch
import torch.nn as nn
import torch.nn.functional as F

class BahdanauAttention(nn.Module):
    def __init__(self, encoder_dim, decoder_dim, attention_dim):
        super().__init__()
        self.W1 = nn.Linear(decoder_dim, attention_dim, bias=False)
        self.W2 = nn.Linear(encoder_dim, attention_dim, bias=False)
        self.v = nn.Linear(attention_dim, 1, bias=False)

    def forward(self, decoder_state, encoder_outputs):
        # decoder_state: (batch, decoder_dim)
        # encoder_outputs: (batch, src_len, encoder_dim)

        # Project decoder state and add to each encoder output
        # (batch, 1, attn_dim) + (batch, src_len, attn_dim)
        score = self.v(
            torch.tanh(
                self.W1(decoder_state).unsqueeze(1)
                + self.W2(encoder_outputs)
            )
        )  # (batch, src_len, 1)

        score = score.squeeze(-1)           # (batch, src_len)
        attn_weights = F.softmax(score, dim=-1)  # (batch, src_len)

        # Weighted sum of encoder outputs
        context = torch.bmm(
            attn_weights.unsqueeze(1),       # (batch, 1, src_len)
            encoder_outputs                   # (batch, src_len, encoder_dim)
        ).squeeze(1)                          # (batch, encoder_dim)

        return context, attn_weights

Notice how W₂ can be precomputed for all encoder states once, since they don't change during decoding. Only W₁ needs recomputation at each step (because the decoder state changes). This is a useful optimization.

Context Vector Computation

Let's formalize what happens after scoring. The raw scores eᵢⱼ = score(sᵢ, hⱼ) are just unnormalized numbers — they could be any real value. We need a proper probability distribution, so we apply softmax across all encoder positions:

αᵢⱼ = exp(eᵢⱼ) / Σₖ exp(eᵢₖ)

These attention weights αᵢⱼ are non-negative and sum to 1 over all source positions j. They form a distribution over the input sequence — a soft pointer that says "here's where to look." The context vector for decoder step i is then:

cᵢ = Σⱼ αᵢⱼ hⱼ

This is just a weighted average of the encoder hidden states. If α is peaked on one position, the context vector is essentially that one encoder state. If α is spread out, the context blends information from multiple positions. The decoder then uses this context vector (typically concatenated with its own state or the previous embedding) to predict the next output token.

The beauty is that this entire pipeline — scoring, softmax, weighted sum — is differentiable. Gradients flow straight through, so the model learns what to attend to purely from the translation loss. Nobody hand-codes attention patterns. The model figures out on its own that "chat" should attend to "cat."

Luong (Multiplicative) Attention

A year after Bahdanau, Luong et al. (2015) proposed simpler scoring functions. The key insight: you don't necessarily need a neural network to compute the score. A dot product can work just as well and is much faster.

Luong attention comes in three flavors:

Dot: score(sᵢ, hⱼ) = sᵢᵀ hⱼ. Just the dot product. No learned parameters at all in the scoring function. This only works if sᵢ and hⱼ have the same dimensionality. It's blazingly fast because dot products are highly optimized on GPUs.

General: score(sᵢ, hⱼ) = sᵢᵀ Wₐ hⱼ. A single learned matrix Wₐ sits between the two vectors. This handles different dimensionalities and adds expressiveness. The matrix learns which dimensions of the decoder state should align with which dimensions of the encoder state.

Concat: score(sᵢ, hⱼ) = vᵀ tanh(Wₐ [sᵢ; hⱼ]). Concatenate the two vectors, then use a Bahdanau-style scoring. This is the most expressive but also the slowest.

Another difference from Bahdanau: Luong attention computes attention using the decoder state at the current step (after the RNN update), not the previous step. The context vector is then combined with the current decoder state to produce the prediction. This is a subtle but important architectural difference.

class LuongDotAttention(nn.Module):
    """Luong dot-product attention.
    Requires encoder_dim == decoder_dim.
    """
    def __init__(self, hidden_dim):
        super().__init__()
        # Combines decoder state + context into an attentional hidden state
        self.combine = nn.Linear(hidden_dim * 2, hidden_dim, bias=False)

    def forward(self, decoder_state, encoder_outputs):
        # decoder_state: (batch, hidden_dim)
        # encoder_outputs: (batch, src_len, hidden_dim)

        # Dot-product scores
        scores = torch.bmm(
            encoder_outputs,                     # (batch, src_len, hidden)
            decoder_state.unsqueeze(2)           # (batch, hidden, 1)
        ).squeeze(2)                             # (batch, src_len)

        attn_weights = F.softmax(scores, dim=-1)

        # Context vector via weighted sum
        context = torch.bmm(
            attn_weights.unsqueeze(1),           # (batch, 1, src_len)
            encoder_outputs                       # (batch, src_len, hidden)
        ).squeeze(1)                              # (batch, hidden)

        # Attentional hidden state: combine context + decoder state
        combined = torch.cat([context, decoder_state], dim=-1)
        attentional_state = torch.tanh(self.combine(combined))

        return attentional_state, attn_weights

In practice, the dot and general variants dominate. The dot product is the ancestor of the scaled dot-product attention used in Transformers — with one crucial addition (the scaling factor 1/√dₖ to prevent softmax saturation in high dimensions).

Global vs. Local Attention

Luong's paper also introduced the distinction between global attention and local attention. Everything we've discussed so far is global attention: at each decoder step, you score every encoder position. For a source sentence of length 100, that's 100 scores per decoder step. For length 1000, it's 1000 scores. The cost is O(source_length) per step.

Global attention is expensive for long sequences. Local attention is the compromise. Instead of attending to all positions, you first predict a single alignment position pₜ for the current step. Then you only attend to a window of positions around pₜ — say [pₜ - D, pₜ + D] for some fixed window size D. Only 2D+1 scores to compute instead of the full source length.

How do you predict pₜ? Luong proposed: pₜ = S · sigmoid(vₚᵀ tanh(Wₚ sₜ)), where S is the source length. The sigmoid output is between 0 and 1, scaled to [0, S]. You can also apply a Gaussian bias centered on pₜ to the attention weights so positions farther from pₜ get downweighted even within the window.

Local attention is a middle ground between the hard alignment of traditional statistical MT (which picks exactly one position) and the soft global attention that considers everything. It's particularly useful for tasks where you have a rough prior about alignment being somewhat monotonic — like speech recognition or document summarization.

Visualizing Attention Weights

One of the most exciting properties of attention is interpretability. The attention weights αᵢⱼ form a matrix of shape (output_length × input_length). Plot this matrix as a heatmap and you can literally see what the model is doing.

For English-to-French translation, you'll see a roughly diagonal pattern — word order is similar between the two languages. But you'll also see where the model deviates: French adjectives often follow nouns, so when generating "chat noir" from "black cat," the attention for "noir" points back to "black" (earlier in the English sentence) while "chat" attends to "cat" (later). The model has learned word reordering without being explicitly taught.

For more distant language pairs — like English to Japanese, where word order is dramatically different — the attention matrix looks far from diagonal. You can trace the model's strategy for handling these structural differences directly from the heatmap.

This interpretability was a breakthrough. Before attention, neural translation models were pure black boxes. With attention, you could debug failures: if the model mistranslates a word, check its attention weights. Is it looking at the right source word? If not, the attention mechanism is the problem. If it's attending correctly but still mistranslating, the issue is downstream. This kind of diagnosis was impossible with fixed-context seq2seq models.

Soft Attention vs. Hard Attention

Everything we've described is soft attention. The model computes a continuous weighted average of all positions. Every encoder state contributes a little bit (even if its weight is 0.001). This is wonderful for training because the entire computation is differentiable — gradients flow smoothly through the softmax and the weighted sum. Standard backpropagation works perfectly.

Hard attention takes a different approach: instead of a weighted average, it samples a single position from the attention distribution and uses only that encoder state. The context vector is just hⱼ where j is sampled from the categorical distribution defined by α. This is computationally cheaper at inference time (read one vector instead of summing over all of them) and produces a sparser, more interpretable alignment.

The catch? Sampling is not differentiable. You can't backpropagate through a discrete sampling operation. Hard attention requires reinforcement learning techniques like the REINFORCE algorithm to train. REINFORCE provides unbiased gradient estimates, but they have high variance, making training noisy and unstable. You typically need variance reduction techniques (baselines, multiple samples) to make it work, which adds complexity.

A practical compromise is the straight-through estimator: during the forward pass, use hard (argmax) attention, but during the backward pass, pretend you used soft attention and pass gradients through the softmax weights. It's biased but low-variance, and works surprisingly well in practice. Most modern systems just use soft attention, though — the computational savings of hard attention rarely justify the training headaches.

Cross-Attention vs. Self-Attention

Everything in this section has been cross-attention: the decoder attends to the encoder. The queries come from one sequence (the decoder), and the keys and values come from a different sequence (the encoder). This is the original form of attention, designed to bridge two sequences in tasks like translation, summarization, or image captioning.

There's another flavor: self-attention, where a sequence attends to itself. Every position computes attention over every other position in the same sequence. This lets the model capture dependencies within a single sequence — like understanding that "it" in "The animal didn't cross the street because it was too tired" refers to "animal," not "street." Self-attention is the cornerstone of Transformers, and we'll cover it thoroughly in the next section.

The mechanics are identical. The only difference is where the queries, keys, and values come from. In cross-attention: queries from sequence A, keys and values from sequence B. In self-attention: queries, keys, and values all come from the same sequence. The Transformer decoder uses both — self-attention within the output sequence, then cross-attention to the encoder output.

Monotonic Attention

Monotonic attention adds a constraint: the attention pointer can only move forward (left to right) through the source sequence. At each decoder step, it either stays at the current position or advances — never goes backward. This is motivated by tasks where alignment is naturally monotonic: speech recognition (audio frames map sequentially to phonemes), real-time translation (you process input as it arrives), or text-to-speech.

Standard soft attention can't do streaming because it needs the entire source sequence upfront to compute scores over all positions. Monotonic attention enables online or streaming inference: the decoder can start producing output before the encoder has finished processing input. At each step, the model makes a binary decision — "attend here" or "move forward" — stepping through encoder states one at a time until it decides to stop and emit an output.

Training monotonic attention is tricky because the hard stop/go decisions aren't differentiable. Monotonic Infinite Lookback (MIL) attention and other variants address this by computing the expected attention under the monotonic process, giving you a soft, trainable approximation of the hard monotonic behavior. In practice, monotonic attention has found its niche in simultaneous translation and streaming ASR, where latency matters more than squeezing out the last BLEU point.

Attention as a General Mechanism

Step back and see the bigger picture. Attention isn't just a fix for seq2seq translation. It's a fundamentally new way to route information in neural networks.

Think of attention as a soft lookup table. You have a query (what you're looking for), a set of keys (what's available), and corresponding values (the information stored at each key). The query is compared to all keys, producing a similarity score for each. These scores become weights over the values. The output is a weighted sum of values — you retrieve a blend of information, weighted by relevance.

Attention Is Content-Based Addressing

Traditional computer memory uses addresses — position 0x7F3A holds a value. Attention uses content — "find me the stored information most similar to this query." It's the difference between looking up a word in a dictionary by page number versus by meaning. This content-based addressing makes attention incredibly flexible: the same mechanism works for translation, image captioning, question answering, graph neural networks, and protein folding.

Before attention, information flow in neural networks was fixed by architecture. A CNN has a fixed receptive field. An RNN has a fixed path from position 1 to position T (through T-1 sequential steps). Attention breaks this rigidity. Any position can directly access any other position in a single step. Information no longer needs to squeeze through a narrow pipeline. It flows directly from where it's stored to where it's needed, gated by learned relevance scores.

This is why attention became the dominant paradigm. It solved the bottleneck problem in seq2seq, but more importantly, it gave neural networks a general-purpose mechanism for dynamic, content-based information routing. When the Transformer architecture later replaced RNNs entirely and built everything out of attention, it wasn't a bolt from the blue. It was the logical conclusion of what Bahdanau started in 2014: if attention is the most powerful part of your model, why not make attention all there is?
