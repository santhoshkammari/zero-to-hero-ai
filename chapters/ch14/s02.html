Bayesian Linear Regression

Chapter 14: Probabilistic & Bayesian ML 4 subtopics
TL;DR

Bayesian linear regression puts probability distributions over the weights instead of finding a single best-fit line. The result: you get not one prediction line, but a family of plausible lines with confidence bands that widen where data is sparse. The math stays tractable because Gaussian priors combined with Gaussian likelihoods give a Gaussian posterior in closed form. Evidence approximation (empirical Bayes) lets the data itself choose the right amount of regularization, including which features matter through automatic relevance determination.

Ordinary linear regression gives you one answer: the line of best fit. Ask it to predict the house price at 4,000 square feet — way beyond your training data — and it will confidently extrapolate a number. No hesitation, no warning. That is dangerous. Bayesian linear regression gives you a prediction plus a measure of how much you should trust it. In the region where you have data, the uncertainty bands are tight. Where data is sparse, they fan out. That honesty about uncertainty is exactly what you need in production systems.

The beautiful thing is that for linear regression with Gaussian assumptions, everything has a closed-form solution. No MCMC. No variational inference. Pure linear algebra. This makes it the perfect entry point for understanding Bayesian modeling before moving to more complex models.

Prior on Weights

Gaussian Prior (Zero Mean, Precision Matrix)

In standard linear regression, the model is y = Xw + ε, where ε ~ Normal(0, σ²). The Bayesian version places a prior distribution on the weight vector: w ~ Normal(0, Σ_prior). The most common choice is w ~ Normal(0, α⁻¹I), where α is the precision (inverse variance) of the prior. A large α means a tight prior that strongly believes weights should be near zero — heavy regularization. A small α means a vague prior that lets the data decide.

Why zero mean? Because before seeing any data, we have no reason to prefer positive or negative weights. The zero-mean Gaussian prior says: "Weights are probably small, and equally likely to be positive or negative." This is identical to the assumption behind Ridge regression.

Prior Hyperparameters

The prior has hyperparameters — α (weight precision) and often β (noise precision, where β = 1/σ²). These control the signal-to-noise ratio of the model. A high α/β ratio means you trust the prior more than the data. A low ratio means you let the data speak. Choosing these wisely matters, and later in this section we will see how the data itself can tell you the optimal values.

import numpy as np
import matplotlib.pyplot as plt

# Visualize different priors on weights
np.random.seed(42)
x_plot = np.linspace(-5, 5, 100)

# Sample weight vectors from different priors and plot the resulting lines
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

for ax, alpha, title in zip(axes, [0.1, 1.0, 10.0],
                             ['Vague (α=0.1)', 'Moderate (α=1)', 'Tight (α=10)']):
    for _ in range(20):
        # Sample intercept and slope from N(0, 1/alpha)
        w = np.random.normal(0, 1/np.sqrt(alpha), size=2)
        ax.plot(x_plot, w[0] + w[1]*x_plot, alpha=0.3, color='steelblue')
    ax.set_title(f'Prior: {title}')
    ax.set_ylim(-10, 10)
    ax.set_xlabel('x')

plt.suptitle('Lines sampled from prior — tighter prior = less variation')
plt.tight_layout()
plt.show()
Prior Predictive Checks

Always visualize what your prior implies before fitting. Sample weights from the prior, generate predictions, and ask: "Does this look reasonable?" If your prior produces predictions of house prices in the millions of dollars or negative temperatures, adjust it. This is one of the most underused techniques in Bayesian modeling.

Posterior on Weights

Closed-Form Gaussian Posterior

Here is where the magic happens. With a Gaussian prior and Gaussian likelihood, the posterior over weights is also Gaussian. No approximation needed. The posterior is P(w|X, y) = Normal(w|m_N, S_N), where:

S_N = (αI + βX^TX)⁻¹ — the posterior covariance

m_N = βS_N X^Ty — the posterior mean

Here α is the prior precision, β = 1/σ² is the noise precision, X is the design matrix, and y is the target vector. The posterior mean m_N is the Bayesian estimate of the weights — equivalent to the Ridge regression solution. The posterior covariance S_N tells you how uncertain you are about each weight and how the weights are correlated.

import numpy as np

class BayesianLinearRegression:
    """Bayesian linear regression with Gaussian prior and known noise."""

    def __init__(self, alpha=1.0, beta=25.0):
        self.alpha = alpha  # prior precision
        self.beta = beta    # noise precision (1/sigma^2)

    def fit(self, X, y):
        # Add bias column
        n, d = X.shape
        Phi = np.column_stack([np.ones(n), X])  # design matrix with bias
        d_full = Phi.shape[1]

        # Prior: w ~ N(0, alpha^{-1} I)
        S0_inv = self.alpha * np.eye(d_full)

        # Posterior covariance: S_N = (alpha*I + beta*Phi^T Phi)^{-1}
        self.S_N = np.linalg.inv(S0_inv + self.beta * Phi.T @ Phi)

        # Posterior mean: m_N = beta * S_N @ Phi^T @ y
        self.m_N = self.beta * self.S_N @ Phi.T @ y

        return self

    def predict(self, X_new):
        n = X_new.shape[0]
        Phi_new = np.column_stack([np.ones(n), X_new])

        # Predictive mean
        y_mean = Phi_new @ self.m_N

        # Predictive variance = noise + epistemic uncertainty
        y_var = np.array([
            1/self.beta + Phi_new[i] @ self.S_N @ Phi_new[i]
            for i in range(n)
        ])

        return y_mean, y_var

# Generate synthetic data
np.random.seed(42)
N = 25
X_train = np.random.uniform(-3, 3, (N, 1))
y_train = 1.5 * X_train.ravel() + 0.8 + 0.5 * np.random.randn(N)

# Fit Bayesian model
blr = BayesianLinearRegression(alpha=1.0, beta=4.0)
blr.fit(X_train, y_train)

# Predict on new points
X_test = np.linspace(-5, 5, 100).reshape(-1, 1)
y_mean, y_var = blr.predict(X_test)
y_std = np.sqrt(y_var)

print(f"Posterior weight mean: {blr.m_N}")
print(f"Posterior weight std:  {np.sqrt(np.diag(blr.S_N))}")
# Notice: uncertainty grows as we move away from the training data

Posterior Mean and Covariance

The posterior mean m_N is a precision-weighted blend of the prior mean (zero) and the data. With little data, it stays close to zero (the prior pulls it back). With lots of data, it approaches the OLS solution. The posterior covariance S_N shrinks as you add data — each new observation tightens your uncertainty. Off-diagonal elements of S_N tell you how correlated your uncertainty about different weights is. If two features are correlated, their weight uncertainties will be anti-correlated.

Sequential Bayesian Update

One of the most elegant properties of Bayesian linear regression: you can process data one point at a time. After seeing the first data point, your posterior becomes the prior for the second data point. The final result is identical to processing all data at once. This is called sequential updating, and it makes Bayesian models natural for streaming or online learning scenarios.

import numpy as np

def sequential_bayesian_update(X, y, alpha=1.0, beta=25.0):
    """Update posterior one data point at a time."""
    d = X.shape[1] + 1  # +1 for bias
    S_inv = alpha * np.eye(d)  # prior precision
    m = np.zeros(d)            # prior mean

    history = []  # track posterior evolution

    for i in range(len(X)):
        phi = np.concatenate([[1], X[i]])  # feature vector with bias

        # Rank-1 update of precision matrix
        S_inv_new = S_inv + beta * np.outer(phi, phi)

        # Update mean
        S_new = np.linalg.inv(S_inv_new)
        m_new = S_new @ (S_inv @ m + beta * y[i] * phi)

        S_inv = S_inv_new
        m = m_new
        history.append((m.copy(), S_new.copy()))

    return history

# With 5 points vs 25 points — watch uncertainty shrink
np.random.seed(42)
X = np.random.uniform(-3, 3, (25, 1))
y = 1.5 * X.ravel() + 0.8 + 0.5 * np.random.randn(25)

history = sequential_bayesian_update(X, y, alpha=1.0, beta=4.0)

for n_seen in [1, 5, 10, 25]:
    m, S = history[n_seen - 1]
    print(f"After {n_seen:2d} points: w = [{m[0]:.2f}, {m[1]:.2f}], "
          f"std = [{np.sqrt(S[0,0]):.3f}, {np.sqrt(S[1,1]):.3f}]")

Predictive Distribution

Gaussian Predictive (Mean and Variance)

For a new input x*, the predictive distribution is Gaussian: P(y*|x*, D) = Normal(y*|m_N^T φ(x*), σ²_pred(x*)). The predictive variance has two components: σ²_pred = 1/β + φ(x*)^T S_N φ(x*). The first term (1/β) is the irreducible noise in the data. The second term is the epistemic uncertainty — uncertainty about the weights that decreases with more data.

Epistemic vs Aleatoric Uncertainty

Aleatoric uncertainty is noise inherent in the data-generating process. No amount of data will reduce it — a coin flip is inherently random. Epistemic uncertainty comes from limited knowledge — uncertainty about the model parameters. More data reduces epistemic uncertainty. Bayesian linear regression separates these two sources cleanly. The 1/β term is aleatoric. The φ^T S_N φ term is epistemic.

This separation is crucial in practice. If prediction uncertainty is mostly aleatoric, collecting more data will not help — you need a better sensor or a different measurement approach. If it is mostly epistemic, more data is the answer. Bayesian models give you the diagnostic to tell the difference.

Uncertainty Away From Data

Watch the predictive variance as you move away from training data. It grows because the epistemic component increases. This is the model honestly saying "I have not seen data here, so I am less certain." A standard neural network would extrapolate with full confidence — and potentially give dangerously wrong predictions. This growing uncertainty is a feature of Bayesian methods.

Credible Intervals

A 95% credible interval for a prediction means: "Given the data and our model, there is a 95% probability the true value lies in this interval." This is a direct probability statement, unlike a frequentist confidence interval. For a Gaussian predictive distribution, the 95% credible interval is simply mean ± 1.96 × std. Visualizing these intervals as shaded bands around the prediction line is one of the most useful outputs of Bayesian regression.

Evidence Approximation (Empirical Bayes)

Type-II Maximum Likelihood

So far we treated the hyperparameters α and β as fixed. But how do you choose them? One principled approach: Type-II maximum likelihood, also called empirical Bayes. You maximize the marginal likelihood (evidence) P(y|X, α, β) with respect to α and β. The evidence is the likelihood with weights integrated out, and for Bayesian linear regression it has a closed form.

The evidence naturally balances model fit against model complexity. Too small α (vague prior) lets the model overfit. Too large α (tight prior) underfits. The evidence maximum is the sweet spot. This gives you automatic regularization tuning — no cross-validation needed.

import numpy as np
from scipy.optimize import minimize_scalar

def log_evidence(alpha, beta, X, y):
    """Compute log marginal likelihood for Bayesian linear regression."""
    N, D = X.shape
    Phi = np.column_stack([np.ones(N), X])
    M = Phi.shape[1]

    # Posterior precision and mean
    A = alpha * np.eye(M) + beta * Phi.T @ Phi
    m_N = beta * np.linalg.solve(A, Phi.T @ y)

    # Log evidence (Bishop, eq 3.86)
    E_mN = (beta/2) * np.sum((y - Phi @ m_N)**2) + (alpha/2) * m_N @ m_N
    sign, logdetA = np.linalg.slogdet(A)
    log_ev = (M/2)*np.log(alpha) + (N/2)*np.log(beta) - E_mN \
             - 0.5*logdetA - (N/2)*np.log(2*np.pi)
    return log_ev

# Search for optimal alpha
np.random.seed(42)
N = 50
X = np.random.uniform(-3, 3, (N, 1))
y = 1.5 * X.ravel() + 0.8 + 0.3 * np.random.randn(N)
beta_fixed = 1.0 / 0.3**2  # assume known noise

alphas = np.logspace(-3, 3, 100)
evidences = [log_evidence(a, beta_fixed, X, y) for a in alphas]
best_alpha = alphas[np.argmax(evidences)]
print(f"Optimal alpha (evidence maximization): {best_alpha:.4f}")
# This is the data telling you how much regularization it needs

Automatic Relevance Determination (ARD)

ARD is one of the most powerful ideas in Bayesian linear models. Instead of one shared precision α for all weights, give each weight its own precision: w_j ~ Normal(0, α_j⁻¹). Optimize each α_j by maximizing the evidence. What happens? Irrelevant features get driven to α_j → ∞, which forces their weight to zero. The model automatically discovers which features matter.

This is Bayesian feature selection. No LASSO. No information gain thresholds. No wrapper methods with cross-validation. The evidence optimization naturally prunes irrelevant features by making their priors infinitely tight. Scikit-learn implements this as ARDRegression.

from sklearn.linear_model import ARDRegression, LinearRegression
import numpy as np

# 5 relevant features, 15 irrelevant features
np.random.seed(42)
N = 100
X = np.random.randn(N, 20)
w_true = np.zeros(20)
w_true[:5] = [2.0, -1.5, 1.0, -0.5, 0.3]
y = X @ w_true + 0.5 * np.random.randn(N)

# ARD automatically identifies relevant features
ard = ARDRegression()
ard.fit(X, y)

print("True weights:", np.round(w_true, 2))
print("ARD weights: ", np.round(ard.coef_, 2))
# ARD sets irrelevant feature weights very close to zero
# No manual feature selection needed

Hyperparameter Optimization

Evidence maximization can be done via gradient-based optimization (differentiating the log evidence with respect to hyperparameters) or via iterative update equations. For Bayesian linear regression, there are closed-form iterative updates for α and β that converge quickly. These update equations alternately re-estimate the hyperparameters and the posterior, bootstrapping toward the optimal solution. This is the approach used in Bishop's "Pattern Recognition and Machine Learning" — a foundational reference for this topic.

Bayesian Linear Regression in the Wild

Bayesian linear regression is not a toy. It is used in A/B testing platforms (to get calibrated uncertainty on treatment effects), time series forecasting (Facebook's Prophet has Bayesian bones), Gaussian process regression (which extends Bayesian linear regression to infinite-dimensional feature spaces), and as the output layer in Bayesian neural networks. Understanding it thoroughly pays compound interest.
