Graphical Models & Inference

Chapter 14: Probabilistic & Bayesian ML 5 subtopics
TL;DR

Graphical models use graphs to represent joint probability distributions compactly. Bayesian networks (directed) encode generative stories; Markov random fields (undirected) encode correlations. D-separation tells you what’s conditionally independent. Exact inference (variable elimination, belief propagation) computes posteriors when the graph structure allows it. Before deep learning, graphical models were THE framework for structured prediction.

Before deep learning ate everything, graphical models were the dominant paradigm for structured probabilistic reasoning. They’re less fashionable now, but understanding them is essential: conditional independence, message passing, and factor graphs underpin modern methods from VAEs to attention mechanisms. And in domains like healthcare, causal reasoning, and robotics, graphical models remain irreplaceable.

Bayesian Networks (Directed Graphical Models)

A Bayesian network is a directed acyclic graph (DAG) where each node is a random variable and edges encode conditional dependencies. The joint distribution factors according to the graph:

P(X1, X2, ..., Xn) = ∏ P(Xi | Parents(Xi))

Each node only needs to know its parents. A network with 10 binary variables could require 210 - 1 = 1023 parameters for a full joint table. With the right graph structure, you might need only 20-30. That’s the power of conditional independence assumptions.

# Classic example: Alarm network
# Burglary -> Alarm <- Earthquake
# Alarm -> JohnCalls
# Alarm -> MaryCalls
#
# Joint: P(B,E,A,J,M) = P(B) P(E) P(A|B,E) P(J|A) P(M|A)
# 5 small tables instead of one giant table with 32 entries

# With pgmpy (pip install pgmpy)
from pgmpy.models import BayesianNetwork
from pgmpy.factors.discrete import TabularCPD

model = BayesianNetwork([
    ('Burglary', 'Alarm'), ('Earthquake', 'Alarm'),
    ('Alarm', 'JohnCalls'), ('Alarm', 'MaryCalls')
])

cpd_b = TabularCPD('Burglary', 2, [[0.999], [0.001]])
cpd_e = TabularCPD('Earthquake', 2, [[0.998], [0.002]])
cpd_a = TabularCPD('Alarm', 2,
    [[0.999, 0.71, 0.06, 0.05],
     [0.001, 0.29, 0.94, 0.95]],
    evidence=['Burglary', 'Earthquake'], evidence_card=[2, 2])
cpd_j = TabularCPD('JohnCalls', 2,
    [[0.95, 0.10], [0.05, 0.90]],
    evidence=['Alarm'], evidence_card=[2])
cpd_m = TabularCPD('MaryCalls', 2,
    [[0.99, 0.30], [0.01, 0.70]],
    evidence=['Alarm'], evidence_card=[2])

model.add_cpds(cpd_b, cpd_e, cpd_a, cpd_j, cpd_m)
print(f"Model valid: {model.check_model()}")

Markov Random Fields (Undirected Graphical Models)

Markov random fields (MRFs) use undirected edges. Instead of conditional probability tables, they use potential functions that score how “compatible” neighboring variable assignments are. The joint distribution is a normalized product of potentials:

P(X1, ..., Xn) = (1/Z) ∏ ψ_c(X_c)

where Z is the partition function (normalization constant, often intractable) and each ψc is a potential over a clique of variables.

MRFs are natural for image segmentation (neighboring pixels tend to have the same label), protein structure (nearby amino acids interact), and any problem where you have correlations rather than causal relationships. The Ising model from physics is an MRF.

D-Separation: Reading Independence from the Graph

D-separation is the algorithm for determining whether two variables are conditionally independent given a set of observed variables. Three fundamental patterns:

Chain: A → B → C. A and C are dependent. But given B, they’re independent. Knowing the middle node “blocks” the information flow. Example: exam score → grade → job offer. Knowing the grade makes the exam score irrelevant for predicting the job offer.

Fork: A ← B → C. A and C are dependent (they share a common cause). Given B, they’re independent. Example: sprinkler ← rain → wet grass. If you know it rained, the sprinkler tells you nothing new about the grass.

Collider: A → B ← C. This one is counterintuitive. A and C are independent. But given B, they become dependent! This is the “explaining away” effect. Example: talent → success ← luck. Talent and luck are independent. But if you know someone is successful, learning they’re not talented makes you think they must be lucky.

Why this matters beyond graphical models: Naive Bayes assumes all features are conditionally independent given the class (a star-shaped BN). The attention mechanism in Transformers can be seen as learning conditional dependencies dynamically. Understanding when things are independent — and when they’re not — is fundamental to ML, not just graphical models.

Exact Inference

Given observed evidence, how do you compute the posterior over unobserved variables? For small or tree-structured graphs, exact inference is tractable.

Variable Elimination: Sum out (marginalize) variables one at a time. The order matters — a bad elimination order creates enormous intermediate factors. Finding the optimal order is NP-hard, but good heuristics exist (min-fill, min-weight).

Belief Propagation: Message passing on the graph. Each node sends “messages” to its neighbors summarizing what it knows. On trees, belief propagation is exact and runs in two passes (leaves → root, root → leaves). On graphs with loops, “loopy belief propagation” isn’t exact but often works surprisingly well in practice.

Junction Tree Algorithm: The general-case exact algorithm. Transform the graph into a tree of cliques (a “junction tree”) and run belief propagation on it. Exact but exponential in the treewidth of the graph — for grid-structured MRFs, treewidth is too large for practical use.

# Exact inference with pgmpy
from pgmpy.inference import VariableElimination

# Using the alarm network defined above
infer = VariableElimination(model)

# Query: P(Burglary | JohnCalls=1, MaryCalls=1)
result = infer.query(
    variables=['Burglary'],
    evidence={'JohnCalls': 1, 'MaryCalls': 1}
)
print(result)
# Even though both called, P(Burglary) is still relatively low
# because earthquakes can also cause alarms

When the graph is too large or too densely connected for exact inference, you need the approximate methods in the next section — variational inference and MCMC.

Modern Relevance

Graphical models aren’t a historical curiosity. They’re alive in:

  • Causal inference — DAGs are the language of causality (Pearl, Rubin). If you want to know “what happens if I intervene?” you need a causal graph.
  • Probabilistic programming (Section 6) — PPLs like PyMC and Stan are essentially specifying graphical models in code.
  • VAEs — the encoder-decoder structure is a graphical model with latent variables.
  • Healthcare / clinical trials — regulatory bodies often require interpretable probabilistic models, not black boxes.
  • Knowledge graphs — representing and reasoning over structured knowledge.
