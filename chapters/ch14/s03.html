Graphical Models

Chapter 14: Probabilistic & Bayesian ML 4 subtopics
TL;DR

Graphical models represent joint probability distributions as graphs — nodes are random variables, edges encode dependencies. Directed graphs (Bayesian networks) model causal-style relationships with conditional probability tables. Undirected graphs (Markov random fields) model symmetric associations using potential functions. Factor graphs unify both. These representations make it possible to reason about hundreds of variables efficiently by exploiting the independence structure encoded in the graph.

Imagine modeling the joint probability of 100 binary random variables. The full joint distribution table has 2¹⁰⁰ entries — more than atoms in the universe. That table is useless. But most real systems have structure. Not every variable depends on every other variable. Your headache depends on whether you slept enough, not on the stock price of a company in Tokyo. Graphical models exploit this structure by representing which variables depend on which others as a graph. The graph lets you factorize the joint distribution into small, manageable pieces.

There are two major flavors: directed and undirected. Each has strengths. Directed models are natural when you think in terms of cause and effect. Undirected models are better for symmetric relationships where neither variable "causes" the other. Factor graphs provide a unified view. By the end of this section, you will understand how to read these diagrams, what independence assumptions they encode, and how to learn them from data.

Directed Graphical Models (Bayesian Networks)

DAG Structure

A Bayesian network (also called a belief network or directed graphical model) is a directed acyclic graph (DAG) where each node represents a random variable and each directed edge represents a dependency. The key rule: each node is conditionally independent of its non-descendants, given its parents. This is the local Markov property.

Think of a spam email classifier. The variables might be: {IsSpam, HasLink, MisspelledWords, SenderKnown, ContainsOffer}. IsSpam influences HasLink, MisspelledWords, and ContainsOffer. SenderKnown is independent of IsSpam given the email content. Drawing these relationships as a DAG makes the assumptions explicit.

Conditional Probability Tables (CPTs)

Each node in a Bayesian network has a conditional probability table (or a conditional probability distribution for continuous variables). The CPT for a node specifies P(node | parents). Root nodes (no parents) have a marginal distribution. If a node has two binary parents, its CPT has 2² = 4 rows. The total number of parameters is the sum of CPT sizes across all nodes — far fewer than the full joint distribution when the graph is sparse.

Factorization of Joint Distribution

The graph structure implies a factorization of the joint distribution: P(X₁, ..., Xₙ) = ∏ᵢ P(Xᵢ | Parents(Xᵢ)). This is the fundamental equation of Bayesian networks. Each variable's probability depends only on its parents, not on the full set of other variables. For a sparse graph with at most k parents per node, the joint distribution goes from O(2ⁿ) parameters to O(n × 2ᵏ) — an exponential reduction.

import numpy as np

# Simple Bayesian network: Burglary -> Alarm <- Earthquake
#                          Alarm -> JohnCalls, Alarm -> MaryCalls

# CPTs
P_burglary = 0.001
P_earthquake = 0.002

# P(Alarm | Burglary, Earthquake)
P_alarm = {
    (True, True):  0.95,
    (True, False): 0.94,
    (False, True): 0.29,
    (False, False): 0.001,
}

P_john_calls = {True: 0.90, False: 0.05}
P_mary_calls = {True: 0.70, False: 0.01}

def joint_probability(burglary, earthquake, alarm, john, mary):
    """Compute P(B, E, A, J, M) using the factorization."""
    p = (P_burglary if burglary else 1 - P_burglary)
    p *= (P_earthquake if earthquake else 1 - P_earthquake)
    p_a = P_alarm[(burglary, earthquake)]
    p *= p_a if alarm else (1 - p_a)
    p_j = P_john_calls[alarm]
    p *= p_j if john else (1 - p_j)
    p_m = P_mary_calls[alarm]
    p *= p_m if mary else (1 - p_m)
    return p

# P(Burglary | JohnCalls=True, MaryCalls=True) via enumeration
numerator = 0
denominator = 0
for b in [True, False]:
    for e in [True, False]:
        for a in [True, False]:
            p = joint_probability(b, e, a, True, True)
            denominator += p
            if b:
                numerator += p

print(f"P(Burglary | John calls, Mary calls) = {numerator/denominator:.4f}")
# About 0.284 — both calling is suspicious!

d-Separation

d-separation is the graphical criterion for reading conditional independence from the DAG structure. Two sets of nodes A and B are d-separated by a set C if every path between A and B is "blocked" by C. A path is blocked if it contains a chain X → Z → Y or fork X ← Z → Y where Z is in C, or a collider X → Z ← Y where Z (and its descendants) are not in C.

The collider case is counterintuitive. If Alarm is a collider (Burglary → Alarm ← Earthquake), then Burglary and Earthquake are marginally independent. But once you observe Alarm, they become dependent — knowing about the alarm and that there was no earthquake makes a burglary more likely. This is called "explaining away" and it shows up everywhere in AI reasoning.

Conditional Independence and Plate Notation

Conditional independence is the workhorse of probabilistic reasoning. X ⊥ Y | Z means that knowing Z, learning X gives you no additional information about Y. Graphical models make these statements visual. Plate notation is a compact way to represent repeated structure — if you have N data points that each depend on shared parameters θ, you draw one node for xᵢ inside a "plate" labeled N, instead of drawing N separate nodes.

Structure Learning

Given data, can you learn the graph structure itself? This is structure learning, and it is much harder than parameter learning. Three approaches dominate.

Score-based methods search over possible DAG structures and score each one using criteria like BIC (Bayesian Information Criterion) or MDL (Minimum Description Length). The score balances fit against complexity. Since the space of DAGs is super-exponential in the number of nodes, heuristic search (greedy hill climbing, tabu search) is necessary.

Constraint-based methods like the PC algorithm use statistical independence tests to determine which edges should exist. Start with a fully connected graph, then remove edges between variables that are conditionally independent given some subset. Efficient, but sensitive to errors in independence tests.

Hybrid methods combine both — use constraint-based pruning to narrow the search space, then score-based search over the remaining candidates.

import numpy as np

# Score-based structure learning idea: BIC scoring
def bic_score(X, y, parent_indices):
    """BIC score for a single node given its parents."""
    n = len(y)
    if len(parent_indices) == 0:
        # No parents — just the marginal
        mu = np.mean(y)
        rss = np.sum((y - mu)**2)
        k = 1
    else:
        # Linear regression of node on its parents
        X_parents = X[:, parent_indices]
        X_design = np.column_stack([np.ones(n), X_parents])
        w = np.linalg.lstsq(X_design, y, rcond=None)[0]
        rss = np.sum((y - X_design @ w)**2)
        k = len(parent_indices) + 1

    # BIC = n * log(RSS/n) + k * log(n)
    bic = n * np.log(rss / n + 1e-10) + k * np.log(n)
    return -bic  # higher is better

# Example: node 2 depends on node 0 but not node 1
np.random.seed(42)
n = 500
x0 = np.random.randn(n)
x1 = np.random.randn(n)
x2 = 2.0 * x0 + 0.3 * np.random.randn(n)
X = np.column_stack([x0, x1, x2])

# Score different parent sets for node 2
print(f"Parents=[]    BIC: {bic_score(X, x2, []):.1f}")
print(f"Parents=[0]   BIC: {bic_score(X, x2, [0]):.1f}")
print(f"Parents=[1]   BIC: {bic_score(X, x2, [1]):.1f}")
print(f"Parents=[0,1] BIC: {bic_score(X, x2, [0, 1]):.1f}")
# Parent=[0] wins — correctly identifies x0 as the parent

Parameter Learning

Once you know the structure, learning the CPT parameters is straightforward. With complete data, MLE works — count frequencies from the data. With missing data, use EM (Expectation-Maximization). With a Bayesian approach, place Dirichlet priors on CPT rows and update with observed counts — the conjugacy makes this clean.

Undirected Graphical Models (Markov Random Fields)

Potential Functions and the Partition Function

Markov random fields (MRFs), also called undirected graphical models, use undirected edges. Instead of conditional probabilities, they use potential functions (also called factors) that assign non-negative compatibility scores to configurations of variables. The joint distribution is: P(X) = (1/Z) ∏_c ψ_c(X_c), where ψ_c are potential functions over cliques c, and Z = Σ_X ∏_c ψ_c(X_c) is the partition function — a normalizing constant that ensures probabilities sum to 1.

The partition function Z is the bane of undirected models. Computing it requires summing over all possible configurations — exponential in the number of variables. Much of the research in probabilistic graphical models is about computing or approximating Z.

Gibbs Distribution and Markov Properties

MRFs are often parameterized using an energy function: P(X) = (1/Z) exp(-E(X)). This is the Gibbs distribution (or Boltzmann distribution). Low-energy configurations are more probable. The three Markov properties (pairwise, local, global) all say variations of the same thing: a variable is independent of non-neighbors given its neighbors. The neighborhood structure is read directly from the graph.

Conditional Random Fields (CRFs)

CRFs are discriminative undirected models that model P(Y|X) — the conditional distribution of labels given inputs — rather than the joint P(X,Y). They were the go-to model for sequence labeling tasks (named entity recognition, POS tagging) before deep learning took over. The advantage: CRFs do not need to model the input distribution P(X), which might be complex and irrelevant for prediction.

import numpy as np

# Linear-chain CRF for sequence labeling (simplified forward algorithm)
# States: {0: 'O', 1: 'B-PER', 2: 'I-PER'}
n_states = 3
seq_len = 5

# Transition scores (log potentials)
transition = np.array([
    [1.0, 0.5, -1.0],   # from O
    [-0.5, -1.0, 1.5],  # from B-PER
    [0.3, -1.0, 0.8],   # from I-PER
])

# Emission scores (from a feature function, e.g., neural network)
np.random.seed(42)
emission = np.random.randn(seq_len, n_states)

# Forward algorithm to compute log-partition function
def log_sum_exp(x):
    m = np.max(x)
    return m + np.log(np.sum(np.exp(x - m)))

alpha = emission[0]  # initial scores
for t in range(1, seq_len):
    alpha_new = np.zeros(n_states)
    for j in range(n_states):
        # Score of arriving at state j at time t
        alpha_new[j] = log_sum_exp(alpha + transition[:, j]) + emission[t, j]
    alpha = alpha_new

log_Z = log_sum_exp(alpha)
print(f"Log partition function: {log_Z:.3f}")
# This normalizer is what makes CRF training tractable for sequences

Ising Model and Boltzmann Machines

The Ising model is the simplest MRF: binary variables on a grid with pairwise potentials between neighbors. Originally from physics (modeling magnets), it is a building block for understanding image denoising, texture models, and social network analysis. Boltzmann machines generalize the Ising model by adding hidden variables, creating a two-layer undirected model that can learn complex distributions. Restricted Boltzmann machines (RBMs) — with no within-layer connections — were a key ingredient in the deep learning revival of the 2000s.

Factor Graphs

Unifying Directed and Undirected

Factor graphs provide a unified representation for both Bayesian networks and MRFs. A factor graph is a bipartite graph with two types of nodes: variable nodes (circles) and factor nodes (squares). Each factor connects to the variables it depends on. Any directed or undirected model can be converted to a factor graph, making it the universal language for inference algorithms.

Factor Nodes and Variable Nodes

Variable nodes represent random variables. Factor nodes represent functions (probability tables, potential functions, or any compatibility function). The joint distribution is the product of all factors, divided by Z if needed. The beauty is that inference algorithms (like message passing) can be written once for factor graphs and applied to any model.

Message Passing on Factor Graphs

The sum-product algorithm on factor graphs computes exact marginals by passing messages between variable and factor nodes. Messages from variables to factors aggregate incoming messages. Messages from factors to variables sum out other connected variables. On tree-structured graphs, this converges in a single pass. On graphs with loops, you iterate (loopy belief propagation) — it is not guaranteed to converge, but often works well in practice.

Graphical Models Today

Deep learning has replaced graphical models for many tasks, but the concepts live on. Attention mechanisms are a form of soft message passing. Graph neural networks are message-passing on arbitrary graphs. VAEs have a directed graphical model interpretation. Understanding graphical models gives you a vocabulary for reasoning about dependencies and independence — skills that transfer directly to modern architectures.

Comparing Directed vs Undirected

Moralization

To convert a Bayesian network to an undirected model, you moralize it: connect all parents of each node (marry the parents), then drop the edge directions. Moralization can introduce extra edges, meaning the undirected version may encode fewer independence statements. Some independence relationships in the directed model are lost in translation.

I-Maps and D-Maps

An I-map (independence map) for a distribution P is a graph whose independence assertions are a subset of those true in P — the graph does not claim any false independences, but may miss some true ones. A D-map (dependence map) captures all dependences but may add false independence claims. A perfect map captures both perfectly, but not every distribution has one in a given graphical model family.

Expressiveness Tradeoffs

Directed models can represent v-structures (explaining away) that undirected models cannot. Undirected models can represent certain cyclic dependency patterns that directed models (DAGs) cannot. Neither subsumes the other. Factor graphs can represent both, but the underlying inference algorithms may differ in complexity. The choice between directed and undirected depends on whether your domain has natural causal structure (use directed) or symmetric associations (use undirected).

The Curse of the Partition Function

Directed models have a built-in normalization: the product of conditional probabilities always sums to 1. Undirected models need the partition function Z, which is generally intractable. This is why training MRFs and Boltzmann machines requires approximate methods (contrastive divergence, pseudo-likelihood, MCMC). If you can formulate your problem as a directed model, do it — the math is friendlier.
