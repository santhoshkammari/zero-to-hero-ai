Bayesian Neural Networks

Chapter 14: Probabilistic & Bayesian ML 6 subtopics
TL;DR

Standard neural networks learn a single set of weights — one "best guess." Bayesian Neural Networks (BNNs) learn a distribution over weights, so every prediction comes with a built-in uncertainty estimate. This section covers why that matters, how to actually train a network with distributions instead of point values, practical shortcuts like MC Dropout and deep ensembles, how to decompose uncertainty into things you can fix (get more data) and things you cannot (inherent noise), the surprising theoretical connections to Gaussian processes, and what it takes to run BNNs in production on safety-critical systems.

Foundations

Imagine you train a classifier to detect tumors in medical scans. The model says "92% chance of malignancy." But how confident is the model itself in that 92%? A standard neural network cannot tell you. It learned one set of weights through gradient descent and that is all it knows. If the scan looks nothing like the training data — different imaging device, rare pathology — the model still produces a confident-looking number. It has no mechanism to say "I have never seen anything like this before."

This is the core motivation for Bayesian Neural Networks. Instead of finding a single best weight vector w*, a BNN maintains a full probability distribution over weights: p(w | D), the posterior distribution given training data D. When you make a prediction, you integrate over all plausible weight configurations, and the spread of those predictions tells you how uncertain the model is.

Distributions Over Weights vs Point Estimates

In a standard network, training finds a single point in weight space — the minimum of your loss function. That point might be a good summary of the data, but it throws away information about what other weight configurations also explain the data well. A BNN keeps that information.

Mathematically, you start with a prior p(w) that encodes your beliefs about reasonable weight values before seeing data. After training on dataset D = {(x_i, y_i)}, Bayes' theorem gives you the posterior:

# Bayes' theorem for neural network weights
# p(w | D) = p(D | w) * p(w) / p(D)
#
# p(D | w)  = likelihood — how well weights w explain the data
# p(w)      = prior — what you believed about w before seeing data
# p(D)      = evidence — a normalizing constant (the hard part)
#
# Prediction for new input x*:
# p(y* | x*, D) = ∫ p(y* | x*, w) * p(w | D) dw
#
# This integral averages predictions over ALL plausible weight settings,
# not just the single "best" one.

That integral is the key difference. Instead of predicting with one w*, you average over every w that is plausible given the data. Regions of weight space where many configurations agree give confident predictions. Regions where they disagree signal uncertainty.

Prior Specification for Neural Networks

Choosing a prior p(w) for a neural network is more art than science. The most common choice is an isotropic Gaussian: each weight is drawn independently from N(0, σ²). This acts like L2 regularization — it says "I expect weights to be small." The scale σ controls how much you trust the prior versus the data.

More structured priors exist. You can use a spike-and-slab prior that puts probability mass at exactly zero (encouraging sparsity) and a broad Gaussian elsewhere. Horseshoe priors achieve a similar effect with heavier tails. In practice, most BNN papers use factored Gaussians because the inference math is tractable enough to scale to modern architectures.

Weight Uncertainty

Weight uncertainty captures how much the posterior distribution spreads out for each parameter. A weight with a tight posterior (small variance) is well-determined by the data — the model is confident about what this weight should be. A weight with a wide posterior is poorly constrained — many values are equally plausible.

This per-weight uncertainty propagates through the network into prediction uncertainty. It is what lets a BNN say "I know this weight matters and I know its value" versus "this weight could be anything, so my prediction is unreliable."

BNN vs Standard Neural Networks

Here is the practical scoreboard:

# Standard Neural Network:
#   - Each weight is a single number (point estimate)
#   - Training: minimize loss via SGD → one set of weights
#   - Prediction: one forward pass → one output
#   - Uncertainty: not directly available
#   - Cost: 1x memory, 1x compute per prediction
#
# Bayesian Neural Network:
#   - Each weight is a distribution (mean + variance, at minimum)
#   - Training: approximate posterior inference → distribution over weights
#   - Prediction: multiple forward passes with sampled weights → distribution over outputs
#   - Uncertainty: built in — spread of predictions
#   - Cost: 2x+ memory (storing distribution params), Nx compute (N forward passes)

The tradeoff is clear: BNNs give you uncertainty estimates, better calibration, and natural regularization, but they cost more compute and memory. The rest of this section is about making that tradeoff practical.

Inference Methods for BNNs

The posterior p(w | D) is intractable for any non-trivial neural network. The normalizing constant p(D) requires integrating over all possible weight configurations — an impossibly high-dimensional integral. So every BNN method is really an approximation scheme. The question is which approximation gives the best accuracy-cost tradeoff for your problem.

Bayes by Backprop (Blundell et al.)

The 2015 paper "Weight Uncertainty in Neural Networks" introduced Bayes by Backprop, the method that made BNNs feel practical for the first time. The idea: represent each weight not as a single number but as a Gaussian with learnable mean μ and standard deviation σ. During each forward pass, sample w = μ + σ · ε where ε ~ N(0, 1). This is the same reparameterization trick that makes VAEs work.

The loss function has two terms: the usual data likelihood (how well the sampled weights predict the training data) and a KL divergence term that keeps the learned posterior close to the prior. You minimize this combined loss with standard backpropagation — gradients flow through both μ and σ.

import torch
import torch.nn as nn
import torch.nn.functional as F

class BayesianLinear(nn.Module):
    """A linear layer where weights are Gaussian distributions."""
    def __init__(self, in_features, out_features):
        super().__init__()
        # Learnable parameters of the weight distribution
        self.w_mu = nn.Parameter(torch.randn(out_features, in_features) * 0.1)
        self.w_rho = nn.Parameter(torch.full((out_features, in_features), -3.0))
        # Same for bias
        self.b_mu = nn.Parameter(torch.zeros(out_features))
        self.b_rho = nn.Parameter(torch.full((out_features,), -3.0))

        # Prior: standard Gaussian
        self.prior_mu = 0.0
        self.prior_sigma = 1.0

    def forward(self, x):
        # Convert rho to positive sigma via softplus
        w_sigma = F.softplus(self.w_rho)
        b_sigma = F.softplus(self.b_rho)

        # Reparameterization trick: sample weights
        w = self.w_mu + w_sigma * torch.randn_like(w_sigma)
        b = self.b_mu + b_sigma * torch.randn_like(b_sigma)

        # KL divergence between q(w) and prior p(w)
        self.kl = self._kl_gaussian(self.w_mu, w_sigma) + \
                   self._kl_gaussian(self.b_mu, b_sigma)

        return F.linear(x, w, b)

    def _kl_gaussian(self, mu, sigma):
        """KL(N(mu, sigma^2) || N(prior_mu, prior_sigma^2))"""
        return 0.5 * torch.sum(
            (sigma / self.prior_sigma).pow(2)
            + ((self.prior_mu - mu) / self.prior_sigma).pow(2)
            - 1
            + 2 * torch.log(torch.tensor(self.prior_sigma)) - 2 * torch.log(sigma)
        )

Variational Inference for BNNs

Variational inference (VI) is the general framework behind Bayes by Backprop. You pick a family of tractable distributions q(w; θ) — usually factored Gaussians — and find the θ that makes q closest to the true posterior. "Closest" means minimizing the KL divergence KL(q || p(w|D)), which is equivalent to maximizing the Evidence Lower Bound (ELBO).

The ELBO decomposes cleanly: ELBO = E_q[log p(D|w)] - KL(q(w;θ) || p(w)). The first term says "sampled weights should explain the data." The second says "don't stray too far from the prior." You estimate the expectation with Monte Carlo samples — draw a few weight vectors from q, compute the likelihood for each, average them. One sample per mini-batch is often enough in practice.

💡 Key Insight

The KL term in the ELBO acts as an automatic regularizer. It penalizes posteriors that are very different from the prior, which prevents overfitting without you having to manually tune a weight decay coefficient. The prior is the regularizer.

Stochastic Gradient Langevin Dynamics (SGLD)

Stochastic Gradient Langevin Dynamics takes a different approach: instead of learning a parametric approximation, you collect samples from the posterior by running a modified SGD. The twist is adding calibrated Gaussian noise to each gradient step:

# Standard SGD update:
#   w_{t+1} = w_t - lr * grad_loss(w_t)
#
# SGLD update:
#   w_{t+1} = w_t - lr * grad_loss(w_t) + sqrt(2 * lr) * N(0, I)
#
# The added noise prevents the optimizer from collapsing to a point estimate.
# As learning rate decays, the iterates become samples from the posterior.
# Collect weights from late iterations → ensemble of posterior samples.

The beauty of SGLD is simplicity. You take your existing SGD training loop, add one line of noise injection, and keep snapshots. The downside: convergence is slow, you need many samples to get good posterior coverage, and deciding when the chain has "burned in" (reached the typical set of the posterior) is tricky.

Stochastic Gradient Hamiltonian Monte Carlo (SGHMC)

SGHMC improves on SGLD by adding momentum, the same trick that makes SGD+momentum faster than vanilla SGD. The weights evolve like a particle moving through a potential energy landscape (the loss surface) with friction and random kicks. Momentum lets the sampler traverse low-probability valleys between modes, exploring the posterior more efficiently than SGLD's random walk.

The practical recipe: run SGD with momentum, add calibrated noise, and collect weight snapshots after burn-in. SGHMC reaches good posterior samples faster than SGLD but requires tuning the friction coefficient alongside the learning rate.

Laplace Approximation for NNs

The Laplace approximation is the cheapest way to turn a trained network into a BNN. Train your network normally with SGD to get the MAP estimate w*. Then approximate the posterior as a Gaussian centered at w* with covariance equal to the inverse Hessian of the loss at w*.

Computing the full Hessian is impractical for large networks — it is an N×N matrix where N is the number of parameters. Practical Laplace methods use diagonal, Kronecker-factored (KFAC), or low-rank approximations to the Hessian. The 2021 paper by Daxberger et al. showed that applying Laplace to only the last layer gives surprisingly good uncertainty estimates at minimal cost.

# Laplace approximation — conceptual steps:
# 1. Train network normally → get w* (MAP estimate)
# 2. Compute Hessian H of loss at w*
#    (or approximate: diagonal, KFAC, last-layer only)
# 3. Posterior ≈ N(w*, H^{-1})
# 4. To predict: sample w ~ N(w*, H^{-1}), do forward pass
#    Repeat N times, aggregate predictions
#

Approximate Bayesian Methods

True BNNs are expensive. In many practical settings, you want uncertainty estimates but cannot afford to double your parameter count or run specialized inference algorithms. Enter the approximations that get you 80% of the benefit at 10% of the cost.

Monte Carlo Dropout

In 2016, Yarin Gal showed that dropout — the regularization technique where you randomly zero out neurons during training — is secretly performing approximate variational inference. The insight: if you leave dropout turned on at test time and run the same input through the network multiple times, each forward pass uses a different random subset of neurons. The variation in outputs approximates Bayesian uncertainty.

This is called Monte Carlo Dropout (MC Dropout), and it is the cheapest path to uncertainty in deep learning. You change exactly one thing in your existing code: do not call model.eval() for the dropout layers at test time.

import torch
import torch.nn as nn

class MCDropoutModel(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, drop_rate=0.2):
        super().__init__()
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.fc3 = nn.Linear(hidden_dim, output_dim)
        self.dropout = nn.Dropout(p=drop_rate)

    def forward(self, x):
        x = torch.relu(self.dropout(self.fc1(x)))
        x = torch.relu(self.dropout(self.fc2(x)))
        return self.fc3(x)

def predict_with_uncertainty(model, x, n_samples=50):
    """Run multiple stochastic forward passes with dropout ON."""
    model.train()  # Keep dropout active!
    predictions = torch.stack([model(x) for _ in range(n_samples)])

    mean = predictions.mean(dim=0)       # Best estimate
    std = predictions.std(dim=0)         # Uncertainty estimate
    return mean, std

# Usage: high std → model is uncertain → flag for human review

Deep Ensembles

Deep ensembles (Lakshminarayanan et al., 2017) take a more brute-force approach: train M independent networks from different random initializations, then average their predictions. The disagreement between ensemble members provides an uncertainty estimate. Despite having no Bayesian math anywhere in the method, deep ensembles consistently produce better-calibrated uncertainty than most true BNN methods.

Why does this work? Different random initializations lead SGD to different local minima. Each minimum captures a different "explanation" of the data. Averaging over them is a coarse approximation to integrating over the posterior. Five ensemble members is the standard choice — enough for good uncertainty, not so many that the cost is prohibitive.

import torch
import torch.nn as nn

def train_ensemble(model_class, train_loader, n_models=5, **kwargs):
    """Train an ensemble of independently initialized models."""
    models = []
    for i in range(n_models):
        torch.manual_seed(i * 42)  # Different initialization each time
        model = model_class(**kwargs)
        optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
        for epoch in range(100):
            for x_batch, y_batch in train_loader:
                loss = nn.functional.cross_entropy(model(x_batch), y_batch)
                optimizer.zero_grad()
                loss.backward()
                optimizer.step()
        models.append(model)
    return models

def ensemble_predict(models, x):
    """Average predictions and measure disagreement."""
    with torch.no_grad():
        preds = torch.stack([torch.softmax(m(x), dim=-1) for m in models])
    mean_pred = preds.mean(dim=0)       # Averaged probability
    uncertainty = preds.std(dim=0)       # Disagreement = uncertainty
    return mean_pred, uncertainty

Snapshot Ensembles

Snapshot ensembles (Huang et al., 2017) give you ensemble-like diversity from a single training run. The trick: use a cyclical learning rate schedule that repeatedly raises and lowers the learning rate. At each cycle's end (when the learning rate is low), the network has converged to a local minimum — save a snapshot. After several cycles, you have multiple diverse models without the cost of training from scratch each time.

This exploits the fact that SGD with high learning rates escapes local minima. Each cycle finds a different minimum. Collecting snapshots from different minima gives you the same kind of functional diversity that makes deep ensembles work.

Comparison: MC Dropout vs Ensembles vs True BNNs

# Practical comparison:
#
# Method            | Extra Train Cost | Extra Test Cost | Quality    | Effort
# ------------------|------------------|-----------------|------------|--------
# MC Dropout        | None             | Nx forward pass | Good       | Minimal
# Deep Ensembles    | Mx training      | Mx forward pass | Excellent  | Moderate
# Snapshot Ensemble | ~1x training     | Mx forward pass | Good       | Low
# Bayes by Backprop | ~1.5x training   | Nx forward pass | Good       | High
# SGLD / SGHMC      | ~1x training     | Nx forward pass | Very Good  | Moderate
# Laplace (last-L)  | Negligible       | Nx forward pass | Good       | Low
#
# N = number of MC samples (typically 10-50)
# M = number of ensemble members (typically 3-5)
#
# Rule of thumb: start with MC Dropout. If you need better uncertainty,
# move to deep ensembles. Only go full BNN if you need principled
# uncertainty decomposition or your domain demands it (medical, autonomous).
⚠️ Warning

MC Dropout quality depends heavily on where you place dropout layers and the dropout rate. Dropout only in the last layer gives poor uncertainty. You need dropout throughout the network, including between convolutional layers, which is not standard practice in most architectures. Deep ensembles are more robust to architectural choices.

Uncertainty Quantification

Having uncertainty is only useful if you understand what it means and can act on it. Not all uncertainty is the same. Some uncertainty you can reduce by collecting more data. Some you cannot — it is fundamental noise in the problem. A BNN can tell you which is which.

Epistemic vs Aleatoric Uncertainty in BNNs

Epistemic uncertainty (model uncertainty) comes from not having enough data. If you have seen only 10 examples of a rare disease, your model's weights are poorly constrained in that region of input space. This uncertainty shrinks as you collect more data. In a BNN, it shows up as high variance in the posterior over weights.

Aleatoric uncertainty (data uncertainty) comes from inherent noise in the problem. Even with infinite data, you cannot predict the exact outcome of a coin flip. This uncertainty does not shrink with more data. In a BNN, you can model it by having the network output both a prediction and a noise parameter.

class HeteroscedasticBNN(nn.Module):
    """BNN that outputs both prediction mean and learned noise variance."""
    def __init__(self, input_dim, hidden_dim):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(input_dim, hidden_dim), nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim), nn.ReLU(),
            nn.Dropout(0.1),
        )
        self.mean_head = nn.Linear(hidden_dim, 1)    # Predicted value
        self.var_head = nn.Linear(hidden_dim, 1)      # Predicted log-variance

    def forward(self, x):
        h = self.shared(x)
        mu = self.mean_head(h)
        log_var = self.var_head(h)  # Aleatoric uncertainty (per input)
        return mu, log_var

    def loss(self, x, y, n_samples=5):
        """Negative log-likelihood with learned variance."""
        total = 0
        for _ in range(n_samples):
            mu, log_var = self.forward(x)
            # NLL of Gaussian: 0.5 * (log_var + (y - mu)^2 / exp(log_var))
            total += 0.5 * (log_var + (y - mu).pow(2) / log_var.exp()).mean()
        return total / n_samples

# At test time:
# - Run N forward passes (MC Dropout) → variance of mu's = epistemic uncertainty
# - Average of exp(log_var) across passes = aleatoric uncertainty
# - Total uncertainty = epistemic + aleatoric

Calibration of Uncertainty Estimates

A model is well-calibrated if when it says "80% confident," it is correct about 80% of the time. Calibration is measured with reliability diagrams and the Expected Calibration Error (ECE). Standard neural networks are notoriously overconfident — they output 99% when they should say 70%. BNNs and ensembles tend to be better calibrated out of the box, but you should always check.

Temperature scaling is a simple post-hoc fix: divide logits by a learned temperature T before softmax. T > 1 softens predictions (reduces overconfidence). You fit T on a held-out validation set by minimizing negative log-likelihood. It is a single parameter — hard to overfit.

Out-of-Distribution Detection with BNNs

One of the killer applications of BNN uncertainty: detecting inputs that are far from the training distribution. Train a BNN on chest X-rays. Show it a picture of a cat. The epistemic uncertainty should spike because the model has never seen anything like this.

In practice, you threshold on the predictive entropy or mutual information. High entropy means "I am unsure about all classes equally" — classic OOD signal. Mutual information (the gap between total uncertainty and aleatoric uncertainty) isolates the epistemic component, giving a cleaner OOD signal.

Predictive Uncertainty Decomposition

The total predictive uncertainty decomposes as:

# Total uncertainty = Aleatoric + Epistemic
#
# For regression (Gaussian output):
#   Total variance = E[σ²(x)] + Var[μ(x)]
#     where expectations are over posterior weight samples
#   E[σ²(x)]  = average predicted noise → aleatoric
#   Var[μ(x)]  = spread of predicted means → epistemic
#
# For classification:
#   Total: H[E_q[p(y|x,w)]]       — entropy of average prediction
#   Aleatoric: E_q[H[p(y|x,w)]]   — average entropy of individual predictions
#   Epistemic: Total - Aleatoric    — mutual information I(y; w | x, D)
#
# This decomposition tells you actionable information:
# High epistemic → collect more data like this input
# High aleatoric → this input is inherently ambiguous, more data won't help

Theoretical Connections

BNNs sit at a fascinating intersection of deep learning, statistics, and learning theory. Several surprising theoretical results connect BNNs to seemingly unrelated ideas.

Cold Posterior Effect

The cold posterior effect is an empirical puzzle. Standard Bayesian theory says you should use the posterior p(w|D) as-is. But in practice, BNNs often perform better when you "temper" the posterior by raising it to a power: p(w|D)^(1/T) with T < 1 (cold). This sharpens the posterior, concentrating it around high-likelihood regions.

Why does this happen? Several hypotheses: model misspecification (the likelihood model does not perfectly match the data-generating process), data augmentation breaking the likelihood (augmented examples are not truly independent), or the prior being poorly calibrated. The cold posterior effect is a reminder that "fully Bayesian" does not automatically mean "optimal."

PAC-Bayes Bounds for Neural Networks

PAC-Bayes bounds provide generalization guarantees for stochastic predictors. If your predictor samples weights from a distribution Q, the generalization error is bounded by the training error plus a term proportional to KL(Q || P) / n, where P is a prior and n is the number of training examples. This gives BNNs a theoretical advantage: the KL term in the ELBO directly optimizes this generalization bound.

Recent work has tightened these bounds enough to be non-vacuous for deep networks — a remarkable result given that most classical generalization bounds are useless for overparameterized models.

Lottery Ticket Hypothesis and Bayesian Pruning

The lottery ticket hypothesis says that dense networks contain sparse subnetworks (winning tickets) that can train to full accuracy from their initial weights. BNNs connect naturally to pruning: weights whose posterior has high variance relative to their mean are candidates for removal. They contribute noise but little signal. Bayesian pruning formalizes this — remove weights where the posterior concentrates near zero, as measured by the signal-to-noise ratio |μ|/σ.

BNNs and Neural Network Ensembles Relationship

Deep ensembles approximate BNN inference, but the connection is deeper than it appears. Each ensemble member's weights represent a sample from an implicit posterior — the distribution induced by SGD from random initialization. The 2020 paper by Wilson and Izmailov argued that deep ensembles explore different modes of the posterior, while most variational BNN methods only capture a single mode. This explains why ensembles often outperform variational BNNs: they provide a multimodal approximation for free.

Infinite-Width NNs and Gaussian Processes (Neural Tangent Kernel)

Perhaps the most profound connection: as a BNN's width approaches infinity, its output converges to a Gaussian Process. Neal showed this in 1996 for single-hidden-layer networks with i.i.d. Gaussian priors on weights. The central limit theorem does the work — each output neuron is a sum of many independent random contributions.

The Neural Tangent Kernel (NTK) extends this to training dynamics. An infinitely wide network trained with gradient descent behaves exactly like kernel regression with a specific kernel determined by the architecture. This means you can analyze deep learning with the tools of Gaussian process theory — computing exact posteriors, understanding generalization, and predicting learning curves. The gap between finite-width practical networks and this infinite-width theory is an active area of research.

Scalability and Practical Considerations

The elephant in the room: can you actually run BNNs at the scale of modern deep learning? A ResNet-50 has 25 million parameters. Storing a mean and variance for each doubles that to 50 million. Running 50 forward passes for MC estimation multiplies inference cost by 50x. Here is how practitioners handle this.

Scalability Challenges for Large Architectures

Full BNN inference on large models is currently impractical. The memory cost of storing full weight distributions, the compute cost of multiple forward passes, and the difficulty of posterior inference in high dimensions all work against you. The practical approaches are: (1) apply Bayesian treatment only to the last few layers, (2) use MC Dropout or ensembles instead of true BNNs, (3) use the Laplace approximation as a post-hoc method, or (4) use subnetwork inference — pick a small subset of weights to treat as Bayesian while keeping the rest fixed.

Implementation with PyTorch (Blitz, TyXe)

Blitz (Bayesian Layers in Torch Zoo) wraps standard PyTorch layers with Bayesian versions. You swap nn.Linear for BayesianLinear and the library handles sampling and KL computation. TyXe takes a different approach: you write a normal PyTorch model and TyXe adds Bayesian inference as a wrapper, leveraging Pyro under the hood.

# Using blitz for a Bayesian classifier
from blitz.modules import BayesianLinear
from blitz.utils import variational_estimator

@variational_estimator
class BayesianClassifier(nn.Module):
    def __init__(self, input_dim, hidden_dim, n_classes):
        super().__init__()
        self.bl1 = BayesianLinear(input_dim, hidden_dim)
        self.bl2 = BayesianLinear(hidden_dim, hidden_dim)
        self.bl3 = BayesianLinear(hidden_dim, n_classes)

    def forward(self, x):
        x = torch.relu(self.bl1(x))
        x = torch.relu(self.bl2(x))
        return self.bl3(x)

model = BayesianClassifier(784, 256, 10)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for x_batch, y_batch in train_loader:
    loss = model.sample_elbo(
        inputs=x_batch,
        labels=y_batch,
        criterion=nn.CrossEntropyLoss(),
        sample_nbr=3,               # MC samples per batch
        complexity_cost_weight=1/len(train_loader.dataset)
    )
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

Implementation with TensorFlow Probability

TensorFlow Probability (TFP) provides tfp.layers — drop-in replacements for Keras layers that place distributions over weights. The DenseFlipout layer uses a variance-reduction trick (flipout) that decorrelates gradient estimates across examples in a mini-batch, making training more stable.

import tensorflow as tf
import tensorflow_probability as tfp
tfd = tfp.distributions

def build_bayesian_model(input_shape, n_classes):
    model = tf.keras.Sequential([
        tfp.layers.DenseFlipout(256, activation='relu',
            input_shape=input_shape),
        tfp.layers.DenseFlipout(256, activation='relu'),
        tfp.layers.DenseFlipout(n_classes),
    ])
    return model

model = build_bayesian_model((784,), 10)

# The KL divergence is automatically added to model.losses
model.compile(
    optimizer='adam',
    loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
    metrics=['accuracy'],
    # Scale KL by 1/N where N is dataset size
    experimental_run_tf_function=False
)

# Prediction with uncertainty: run forward pass multiple times
preds = tf.stack([tf.nn.softmax(model(x_test)) for _ in range(50)])
mean_pred = tf.reduce_mean(preds, axis=0)
uncertainty = tf.math.reduce_std(preds, axis=0)

BNNs in Safety-Critical Domains

BNNs find their strongest use case where the cost of an undetected wrong answer is catastrophic. In medical imaging, a BNN can flag uncertain diagnoses for expert review instead of silently producing a confident wrong answer. In autonomous driving, epistemic uncertainty on object detection can trigger a "hand back to human" protocol. In drug discovery, uncertainty over predicted molecular properties determines which compounds are worth the expense of lab testing.

🚨 Critical

Uncertainty estimates are not free from failure. A BNN trained on one hospital's CT scanner may be confidently wrong on scans from a different manufacturer — the distribution shift is in the preprocessing, not the pathology. Always validate uncertainty calibration on data from the actual deployment environment, not the training environment.

The practical deployment pattern: run the BNN, compute uncertainty, apply a threshold. Inputs below the threshold get automated decisions. Inputs above get routed to humans. The threshold is set based on the cost ratio of false positives (unnecessary human review) to false negatives (missed errors). This human-in-the-loop architecture is where BNNs deliver the most real-world value today.
