Classic Generative Models

Generative models feel like magic: neural networks create, not classify. Terms like "latent spaces," "adversarial training," and "evidence lower bound" sound separate. Not separate. Different math attacks on same problem: learn data distribution well enough to sample new data. Next: unpack.

Generative models learn training-data distribution, then sample new examples from it. Idea existed in earlier forms since 1980s. Modern wave started around 2013–2014 with Variational Autoencoders and Generative Adversarial Networks. Then came normalizing flows, energy-based models, and diffusion models. Same problem. Different math angle.

The compression game

Imagine tiny portrait studio. Thousands of 28×28 grayscale headshots. Each image: 784 numbers. Goal: store them efficiently. Many of those 784 numbers are redundant. Faces share structure — two eyes, nose, mouth, similar layout. If you keep essential information that separates one face from another, you can store each face with far fewer numbers.

Core idea behind an autoencoder. Two neural networks glued together. Encoder takes 784-dimensional image and squeezes it to, say, 20 numbers. Decoder takes those 20 numbers and reconstructs original 784-dimensional image. Middle 20-number representation is latent code. Narrow passage is bottleneck.

Training objective: make output match input as closely as possible. Loss is usually mean squared error — average squared difference between original pixels and reconstructed pixels.

Why interesting? Bottleneck tiny. 784 dimensions in, 20 in middle, 784 out. Network cannot memorize inputs; not enough room. It must learn which face features matter — angle, expression, glasses — and drop rest. Bottleneck forces compressed representation. Representation learning.

Vanilla autoencoders

Concrete example. Tiny portrait studio, 4-pixel grayscale images. Each image is vector of 4 numbers between 0 and 1. Encoder compresses to 2 numbers. Decoder expands 2 numbers back to 4.

Input image:     [0.9, 0.1, 0.8, 0.2]   (4 pixels)
                        │
                   ┌────▼────┐
                   │ Encoder  │   (4 → 2)
                   └────┬────┘
                        │
Latent code:         [0.7, -0.3]           (2 numbers)
                        │
                   ┌────▼────┐
                   │ Decoder  │   (2 → 4)
                   └────┬────┘
                        │
Reconstruction:  [0.85, 0.15, 0.78, 0.22] (4 pixels)

Reconstruction is not perfect — bottleneck loses information. But close. Loss, usually mean squared error between input and reconstruction, measures closeness. Training updates encoder and decoder weights to minimize it.

Mental model: encoder summarizes book into tweet. Decoder reads only tweet and rewrites book. If tweet keeps story essence, rewrite stays recognizable. If not, nonsense. Training makes both summarizer and rewriter better.

Important detail: if bottleneck matched input size, or got larger, network could learn identity function — pass everything through unchanged, reconstruct perfectly, learn little. Single-layer linear autoencoder with MSE loss learns same subspace as PCA. Deep nonlinear version learns richer structure. Same spirit: keep dimensions that matter, drop rest.

Denoising and sparse autoencoders

Vanilla autoencoder has problem: laziness. If bottleneck is too generous, or network too strong, model starts memorizing inputs instead of learning structure. Two fixes matter.

Denoising autoencoder corrupts input before encoder — Gaussian noise, randomly zeroed pixels, or both. Loss still compares against clean original. Model cannot memorize corrupted input; it changes every pass. Model must learn underlying clean signal. Portrait studio version: give blurry, scratched photo, ask decoder for clean original. Encoder must learn what face is, not what noisy copy looks like.

Idea came early. BERT's masked language modeling — corrupt words, predict originals — uses same trick on text more than a decade later.

Sparse autoencoder uses different fix. Instead of corrupting input, it adds loss penalty that discourages too many active latent neurons at once. Common choice: KL divergence penalty pushing average hidden activation toward small target like 0.05. Result: for any input, only few latent dimensions fire. Each active dimension carries distinct feature. Like forcing tweet summarizer to use only 5 of 20 words — each word must matter.

Denoising and sparse autoencoders learn better representations than vanilla autoencoders. But same limit remains: latent space is unstructured. Points land wherever optimization puts them. Nearby latent codes need not map to similar images. Random latent points need not decode to valid new images. Decoder reconstructs seen-style codes. Random point usually gives garbage.

That limit pushes us to next idea.

The problem with autoencoder latent spaces

Concrete picture. Train vanilla autoencoder on portrait-studio headshots and inspect 2D latent space. Each face maps to one point, but points form irregular clusters with huge empty gaps. Smiling woman maybe at (3.7, -1.2). Frowning man maybe at (-8.4, 5.6). At (0, 0)? Probably nothing coherent.

We cannot generate new faces from this space because we do not know which coordinates give valid faces and which give noise. We cannot interpolate safely because path between two points may cross dead zones that decode to static. Autoencoder learned compression and reconstruction, not face distribution. It knows individual faces, not face-ness.

Why big deal? Compression helps, but generative models need more. To create new data, latent space must be navigable. Pick point, get something meaningful. Need structure. Need probability.

Variational autoencoders: encoding to distributions

Variational Autoencoder, introduced by Kingma and Welling in 2013, changes one idea and changes everything. Encoder no longer outputs single latent point. It outputs parameters of a probability distribution.

Portrait studio version: feed headshot in. Encoder does not emit one code like [0.7, −0.3]. It emits two vectors: mean μ = [0.7, −0.3] and log-variance log σ² = [−1.0, −0.8]. Together they define Gaussian centered at μ with spread σ. Then we sample latent code z from that distribution and send it to decoder.

Input image → Encoder → μ = [0.7, -0.3], log σ² = [-1.0, -0.8]
                              │
                         Sample z from N(μ, σ²)
                              │
                         z = [0.65, -0.25]  (a random draw)
                              │
                         Decoder → Reconstructed image

Why log-variance, not variance? Variance must stay positive. Log-variance can be any real number, so neural network can output it easily. Recover σ with exp(0.5 × log σ²).

Crucial part: loss has two terms. First is reconstruction loss: make decoder output match input. Second is KL divergence: measure how far each encoding distribution is from standard normal N(0, I). This term penalizes encoder for scattering images across unrelated latent regions. It pulls encoding distributions toward origin and toward each other. Result: overlap and continuity.

Result: structured latent space. Nearby points decode to similar images. Paths between points give smooth transitions. Also, we can sample z from standard normal, feed decoder, and get coherent new portrait not present in training data. VAE did not memorize faces. It learned face-ness.

Portrait studio analogy: vanilla autoencoder was filing system — each photo in arbitrary drawer. VAE is map. Similar photos sit nearby. Walk from "smiling woman" to "smiling man," images morph smoothly. Pick random map location, still find reasonable face.

The reparameterization trick

Mechanical problem: training needs gradients through whole network — reconstruction loss, decoder, latent code z, encoder. But z was sampled. Sampling is random. Derivative of randomness not available.

Reparameterization trick fixes this. Instead of sampling z directly from N(μ, σ²), do this:

First sample ε from standard normal N(0, I). Pure randomness. No model parameters. Then compute z = μ + σ ⊙ ε, where ⊙ means element-wise multiplication.

z is same random variable — same distribution. But now z is deterministic, differentiable function of μ and σ, which come from encoder weights. Randomness lives in ε outside main computation graph. Gradients flow through μ and σ back into encoder.

# The trick in three lines of PyTorch:
std = torch.exp(0.5 * logvar)   # σ = exp(½ log σ²)
eps = torch.randn_like(std)      # ε ~ N(0, I) — no gradients needed
z = mu + std * eps               # z = μ + σ ⊙ ε — differentiable in μ, σ

First reaction often: "Wait, that's it?" Yes. That is it. Trick made VAEs trainable with standard backpropagation. Without it, you need reinforcement-learning-style gradient estimators: noisy, slow. With it, gradients stay clean and low-variance.

The ELBO: why the loss looks the way it does

VAE loss is Evidence Lower Bound, or ELBO. Name sounds heavy. Logic is simple.

Goal: maximize training-data probability under model, log p(x). Problem: log p(x) requires integrating over all latent codes z. Intractable. Cannot compute directly.

Practical move: introduce approximate encoder q(z|x) and derive lower bound on log p(x). Math uses Jensen's inequality. Result:

log p(x) ≥ 𝔼q(z|x)[log p(x|z)] − DKL(q(z|x) ∥ p(z))

That is ELBO. Two terms. Constant tension.

First term, 𝔼q(z|x)[log p(x|z)], is reconstruction loss. Question: for codes encoder produces, how well does decoder rebuild original input? Maximizing it pushes toward perfect reconstruction. In practice, usually mean squared error or binary cross-entropy between input and output.

Second term, DKL(q(z|x) ∥ p(z)), is KL divergence between encoder distribution and prior. Question: how different is learned distribution from simple standard normal prior? Minimizing it pushes encodings toward N(0, I).

For Gaussian encoders with standard normal prior, KL term has closed-form solution. No sampling. No approximation:

DKL = −½ Σj(1 + log σj² − μj² − σj²)

Here j indexes latent dimensions. Computation stays cheap and exact.

These two terms pull opposite ways. Reconstruction term wants different inputs far apart — separate codes make reconstruction easier. KL term wants everything near origin. Training a VAE means balancing fidelity against structure. That tension makes VAE outputs smoother and often blurrier than GAN outputs. KL smoothing costs sharpness.

β-VAE changes one knob. Multiply KL term by scalar β > 1. Higher β pushes harder toward structured latent space and encourages disentanglement — separate dimensions capture factors like rotation, color, size. Cost: blurrier reconstructions. Tradeoff is real. Choosing β is part science, part art.

The adversarial game

In 2014, Ian Goodfellow proposed wild idea: train two networks that fight each other instead of one network that generates data alone.

Generator G takes random noise z, usually from standard normal, and produces fake sample — say, fake portrait. Discriminator D sees real training images and fake images from G, then outputs probability input is real.

Portrait studio version: generator is forger given random noise and command: make face. Discriminator is detective labeling photos "real" or "fake." Better detective forces better forger. Better forger forces sharper detective. Competition drives both.

Formal objective: minimax game.

minG maxD V(D, G) = 𝔼x~pdata[log D(x)] + 𝔼z~pz[log(1 − D(G(z)))]

Unpack objective. D wants to maximize it: push D(x) toward 1 on real data and D(G(z)) toward 0 on fakes. G wants to minimize it: push D(G(z)) toward 1 so fakes look real.

In theory, game has Nash equilibrium: generator matches data distribution and discriminator cannot beat random guessing, so D(x) = 0.5 everywhere. In practice, reaching that point is extremely hard. That is GAN drama.

One practical detail matters. Generator loss is often changed from log(1 − D(G(z))) to −log D(G(z)). Reason: early in training, generator output is obvious garbage. Discriminator says D(G(z)) ≈ 0 with high confidence. Then gradient of log(1 − D(G(z))) becomes tiny, so generator gets almost no learning signal. −log D(G(z)) gives steeper gradients near zero. Generator gets usable signal and starts improving. Small math change. Huge training difference.

Training a GAN: the delicate dance

GAN training feels alien if you know classifiers or regression. No single loss falls steadily. Two losses. Two networks. Oscillation. Discriminator improves at catching fakes, generator adapts, discriminator responds. No natural "done" signal.

Training loop alternates two steps. First, freeze generator and update discriminator: show real images (label 1) and fake images from G (label 0), compute binary cross-entropy loss, update D weights. Second, freeze discriminator and update generator: make fake images, pass them through D, compute loss from how well they fool D, update G weights.

for each training batch:

    # Step 1: Train the detective
    z = sample_noise(batch_size)
    fake = G(z).detach()       # .detach() stops gradients into G
    loss_D = BCE(D(real), 1) + BCE(D(fake), 0)
    update D

    # Step 2: Train the forger
    z = sample_noise(batch_size)
    fake = G(z)                # no detach — G needs gradients through D
    loss_G = BCE(D(fake), 1)   # G wants D to think fakes are real
    update G

.detach() is critical. When training D, gradients must not flow back into G. Only detective improves. When training G, gradients must pass through D because G learns what D finds convincing.

Balance between networks is everything. If D gets too strong too fast, it rejects everything with near-100% confidence. D saturates, gradients to G vanish, G stops learning. If G gets too far ahead, D's gradients become useless too. Both networks must improve at roughly same pace. Common tricks: multiple D steps per G step, label smoothing, input noise for D. Big toolbox built from years of frustration.

Loss curves alone tell you almost nothing. They often oscillate wildly even when training is normal. Reliable monitoring means looking at generated samples or computing FID scores, which compare generated-image distribution with real-image distribution.

Mode collapse and the dirt-pile fix

Signature GAN failure: train on diverse face dataset — men, women, ages, expressions — then inspect samples and every face is same middle-aged woman. Thousand samples. Thousand near-copies.

This is mode collapse. Generator finds one output that fools discriminator reliably and repeats it. Portrait-studio forger learns one fake detective misses, then mass-produces that fake instead of learning full style range.

Reason is subtle. Generator optimizes to fool D right now, not to cover whole data distribution. If one data mode is easier to imitate, G overuses it. D eventually adapts, G jumps to another mode, and system can cycle through modes without covering all at once. GAN objective rewards fooling discriminator, not diversity.

Big fix arrived in 2017: Wasserstein GAN (WGAN). Original GAN loss is closely tied to Jensen-Shannon divergence between real and generated distributions. Problem: when distributions do not overlap — common in high dimensions, especially early — JS divergence becomes constant. Useful gradient disappears. Generator stands on flat field.

Wasserstein distance, or earth mover's distance, fixes this. Think of real distribution as one dirt pile and generated distribution as another. Wasserstein distance measures minimum work — dirt times distance moved — needed to reshape one pile into other. Crucially, this distance stays smooth and meaningful even when piles do not overlap. Gradient still points generator toward real data.

WGAN replaces discriminator with critic that outputs score, not probability. It enforces Lipschitz constraint — first via weight clipping, later via gradient penalty in WGAN-GP — and uses Wasserstein distance as loss. Result: much more stable training, loss curve that tracks sample quality better, and less mode collapse because gradient signal stays informative.

Theme appears again later: smooth, informative gradients fix training problems. Same idea returns in normalizing flows and energy-based models.

Conditional generation: steering the output

So far, models generate random outputs. Sample noise, decode or generate, get whatever model gives. Interesting, not always useful. Sometimes you want specific output: smiling woman, sketch-to-photo conversion, or chosen digit.

Standard fix across model families: feed extra information into network. In Conditional GAN (cGAN), concatenate label y — class label, text embedding, or input image — to generator noise input and discriminator input. G learns image generation conditioned on y. D learns whether image is realistic for category y.

Objective:

minG maxD 𝔼x[log D(x|y)] + 𝔼z[log(1 − D(G(z|y)|y))]

Same idea works for VAEs. Conditional VAE (CVAE) feeds label y into encoder and decoder. Encoder learns q(z|x, y), distribution over latent codes given image and label. Decoder learns p(x|z, y). Then you can fix y = "smiling" and sample different z values to get diverse smiling faces.

Applications are wide. Pix2Pix uses GAN conditioning on input image for paired image-to-image translation: sketch to photo, segmentation map to street scene, day to night. CycleGAN extends this to unpaired translation: horses to zebras, photos to paintings, no matched pairs needed. Simple extra condition opened huge application space.

Portrait studio version: vanilla GAN makes random faces. Conditional GAN takes commissions. "Bearded man in his 40s." Model produces that type.

Normalizing flows: invertible transformations

VAEs and GANs each have annoying weakness. VAEs give tractable approximate likelihood and structured latent space, but outputs blur. GANs give sharp outputs, but no likelihood and unstable training. Question: can one model give exact likelihoods and high-quality samples?

Normalizing flows take different route. Start with simple distribution, usually standard normal, then warp it through series of invertible transformations until it matches data distribution. Because every step is invertible, you can go both directions: noise to data for generation, data to noise for density evaluation.

Math backbone is change of variables formula. If z has known density pz(z) and x = f(z) where f is invertible and differentiable, then:

log px(x) = log pz(f−1(x)) + log |det(∂f−1/∂x)|

Hard part is second term: log-absolute-determinant of Jacobian. For transform f from ℝd to ℝd, Jacobian is d×d matrix. Naive determinant cost is O(d³). For 784-dimensional image, too expensive.

Clever engineering enters here. Coupling layers, from NICE (2014) and refined in RealNVP (2017), are transformations built for tractable Jacobian determinants. Split input into xa and xb. Leave xa unchanged. Transform xb with function conditioned on xa.

y_a = x_a                                     (unchanged)
y_b = x_b ⊙ exp(s(x_a)) + t(x_a)            (scaled and shifted)

Here s() and t() are neural networks that output scale and translation parameters. They can be arbitrarily complex. Coupling layer does not require them to be invertible because they operate only on conditioning half. Inverse stays easy:

x_a = y_a
x_b = (y_b - t(y_a)) ⊙ exp(-s(y_a))

Jacobian here is triangular, so determinant is product of diagonal elements — sum of s(xa) in log space. No expensive matrix operations. Stack many coupling layers, alternate which half gets transformed, and composition stays invertible. Total log-determinant is sum of per-layer log-determinants.

Glow (Kingma and Dhariwal, 2018) added invertible 1×1 convolutions between coupling layers — learnable permutations mixing information across channels before each coupling step. This boosted expressiveness and produced first flow-based model with genuinely convincing face images.

Flows have real tradeoffs. Invertibility constrains architecture. No bottleneck allowed, because that breaks invertibility, so no dimensionality reduction. Models tend to be large. Sample quality improved a lot, but historically trails GANs and later diffusion models on hardest image-generation tasks. But when exact likelihoods matter — anomaly detection, density estimation, lossless compression — flows are hard to beat.

Energy-based models: the landscape view

Previous generative models used specific architecture and specific generation mechanism: encode/decode, adversarial game, invertible transform. Energy-based models strip that down to most general formulation.

Define function Eθ(x) assigning scalar energy to every input x. Low energy means "looks like real data." High energy means "unlikely." Probability of x is:

pθ(x) = exp(−Eθ(x)) / Zθ

Zθ = ∫ exp(−Eθ(x)) dx is partition function, normalizing constant that makes probabilities sum to 1.

Picture energy landscape. Valleys are likely data points. Peaks are unlikely ones. Faces sit in deep valleys. Random noise sits on peaks. Portrait studio headshots occupy one valley.

Big problem: Zθ requires integrating over all possible inputs. For 784-dimensional image, that means integral over ℝ784. Intractable. We can evaluate unnormalized density exp(−E(x)) for given x, but not full probability. We know which valley is deeper, not exact probability relative to whole landscape.

Score matching (Hyvärinen, 2005) sidesteps this. Instead of learning density p(x), learn score function: gradient of log p(x) with respect to x. Score is ∇x log p(x) = −∇x E(x). Key point: partition function Z disappears. It is constant with respect to x, so gradient is zero. We can learn score without computing Z.

But how do you sample if you only know score? Use Langevin dynamics. Start from random point x₀, usually noise. Then iterate:

xt+1 = xt + (ε/2) ∇x log p(xt) + √ε · noise

Each step nudges x toward higher probability by following score, while adding random noise. Over many steps, x moves from peaks down into valleys where real data lives. Like rolling ball downhill on energy landscape, with random bumps preventing traps in shallow dips.

Connection to diffusion models matters. Diffusion models can be viewed as specific way to do score matching across multiple noise levels. Boundary between "EBM with Langevin sampling" and "diffusion model" is blurrier than many presentations imply. Song and Ermon's 2019 score-based generative models made this link explicit and helped start diffusion revolution.

Energy-based models are most flexible generative framework. Energy function E(x) can be any neural network, with no constraints like invertibility or encoder-decoder structure. Cost is slow iterative sampling — often hundreds or thousands of Langevin steps — and tricky training. But framework is elegant and underpins much of modern generative modeling.

The generative model landscape

We covered five generative-model families. Side-by-side view helps. Each makes different tradeoff.

Model Family How It Generates Likelihood? Sample Quality Key Weakness
VAE Sample z ~ N(0,I), decode Approximate (ELBO) Smooth, sometimes blurry Reconstruction-KL tension
GAN Sample z, one forward pass through G None Sharp, realistic Mode collapse, training instability
Normalizing Flow Sample z, apply inverse transforms Exact Good, improving Invertibility limits architecture
Energy-Based Langevin dynamics (iterative) Unnormalized Varies Slow sampling, tricky training
Diffusion (preview) Iterative denoising Approximate Excellent Slow (many denoising steps)

Pattern: no free lunch. Exact likelihoods in flows cost architectural flexibility. Sharp samples in GANs cost training stability and mode coverage. Flexible architectures in EBMs cost efficient sampling. Diffusion models, next section, hit strong compromise by combining ideas from score matching and denoising autoencoders — but they still pay with slow multi-step generation.

Portrait studio analogy still works. VAE forger makes acceptable work, but slightly fuzzy. GAN forger makes photorealistic work, but can get stuck forging same face. Flow forger can tell you exact probability of any face, but needs very specific tools. Energy-based forger is most versatile, but painfully slow. Each model has its place.
