Chapter 14: Probabilistic & Bayesian ML 6 subtopics
TL;DR

Advanced Bayesian topics you should be aware of but probably won’t need daily. Bayesian neural networks for deep learning uncertainty, hierarchical models for grouped data, non-parametric Bayesian methods when you don’t know the model size, and a few more inference tricks. Know they exist so you recognize when you need them.

Bayesian Neural Networks

The idea is straightforward: instead of fixed weights, put distributions on them. Instead of a single prediction, average over all plausible weight configurations. The result: a neural network that knows what it doesn’t know.

The challenge is equally straightforward: a ResNet-50 has 25 million weights. Putting a full posterior distribution on each one is astronomically expensive.

MC Dropout is the pragmatic workaround: keep dropout enabled at test time, run the input through the network multiple times, and treat the variation in outputs as uncertainty. Gal & Ghahramani (2016) showed this approximates a specific type of variational inference. It’s not perfect, but it’s nearly free — you’re already using dropout during training.

import torch
import torch.nn as nn

# MC Dropout: uncertainty from a standard dropout network
class MCDropoutModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(20, 50)
        self.fc2 = nn.Linear(50, 50)
        self.fc3 = nn.Linear(50, 1)
        self.dropout = nn.Dropout(0.1)

    def forward(self, x):
        x = torch.relu(self.dropout(self.fc1(x)))
        x = torch.relu(self.dropout(self.fc2(x)))
        return self.fc3(x)

def predict_with_uncertainty(model, x, n_samples=100):
    model.train()  # keep dropout ON
    preds = torch.stack([model(x) for _ in range(n_samples)])
    mean = preds.mean(dim=0)
    std = preds.std(dim=0)
    return mean, std
# High std = model is uncertain about this input

Bayes by Backprop (Blundell et al., 2015) learns a Gaussian distribution over each weight using the reparameterization trick. More principled than MC Dropout but more expensive and harder to tune.

When you need it: Safety-critical predictions (medical, autonomous driving) where “I don’t know” is better than a wrong confident answer. Out-of-distribution detection. Active learning (query the points the model is most uncertain about).

Hierarchical Bayesian Models

You have data from 50 hospitals. Each has different patient populations, different sample sizes. Two bad options: (1) model each hospital independently (too little data per hospital), (2) pool all data together (ignores real differences between hospitals).

Hierarchical Bayesian models give you the third option: partial pooling. Each hospital has its own parameters, but those parameters are drawn from a shared population distribution. Hospitals with little data get “pulled” toward the population mean (shrinkage). Hospitals with lots of data get to be themselves.

# Hierarchical model in PyMC (conceptual)
# with pm.Model() as hierarchical_model:
#     # Population-level parameters
#     mu_pop = pm.Normal('mu_pop', mu=0, sigma=10)
#     sigma_pop = pm.HalfNormal('sigma_pop', sigma=5)
#
#     # Hospital-level parameters (drawn from population)
#     mu_hospital = pm.Normal('mu_hospital', mu=mu_pop, sigma=sigma_pop,
#                             shape=n_hospitals)
#
#     # Observations
#     y = pm.Normal('y', mu=mu_hospital[hospital_idx], sigma=sigma_obs,
#                   observed=outcomes)

This is one of the most practically useful Bayesian techniques. It’s used in:

  • Clinical trials (multi-site studies)
  • Education (students nested in schools nested in districts)
  • Marketing (user behavior across different segments)
  • Sports analytics (player performance across teams/seasons)

Non-Parametric Bayesian Methods

Standard models fix their complexity upfront: K clusters, D latent dimensions, L mixture components. Non-parametric Bayesian methods let the data determine the complexity.

Dirichlet Process (DP): A distribution over distributions. Use it as a prior for mixture models when you don’t know the number of clusters. The DP mixture model (DPMM) will infer the number of clusters automatically. In the “Chinese Restaurant Process” metaphor: each new customer sits at an existing table with probability proportional to the table’s size, or starts a new table with probability proportional to a concentration parameter α.

Indian Buffet Process (IBP): Like the DP, but for feature allocation rather than clustering. Each data point can have any number of latent features, and the number of features grows with the data. Used in latent factor models when you don’t know how many factors exist.

When you need it: When you genuinely don’t know how many clusters, topics, or latent factors your data has, and you don’t want to do model selection over K. In practice, fitting DP mixtures is slower and more finicky than just running K-means for K=2,3,...,20 and picking the best. Use non-parametric Bayes when the number-of-components question is scientifically important, not just a hyperparameter.

Bayesian Model Comparison

The marginal likelihood P(D|M) provides a principled way to compare models of different complexity. A model that spreads its prior probability thinly over a huge parameter space will have a lower marginal likelihood than a simpler model that concentrates on the right parameters. This is automatic Occam’s razor.

In practice, computing the marginal likelihood exactly is hard. Alternatives:

  • WAIC (Widely Applicable Information Criterion): A Bayesian analogue of AIC, computed from the posterior samples. Implemented in ArviZ.
  • LOO-CV (Leave-One-Out Cross-Validation): Approximated efficiently using Pareto-smoothed importance sampling (PSIS-LOO). The gold standard in the Stan/ArviZ ecosystem.
  • Bayes Factors: Ratios of marginal likelihoods. Powerful but sensitive to prior choice.

Expectation Propagation

EP is an alternative to variational inference that can better handle multimodal posteriors. Where VI minimizes KL(q||p) (mode-seeking, tends to under-cover), EP minimizes KL(p||q) (mass-covering, tends to over-cover). EP approximates each factor in the posterior independently, then combines them.

EP powers the sparse GP approximations in GPflow and was central to the Infer.NET framework (Microsoft). It’s less popular than VI or MCMC in general use but excels for specific model types (GP classification, mixture of Gaussians).

Stein Variational Gradient Descent

SVGD (Liu & Wang, 2016) bridges VI and MCMC. It maintains a set of “particles” (parameter samples) and moves them toward the posterior using a gradient-based update that includes a “repulsion” term to maintain diversity. No parametric assumption on q (unlike VI), no Markov chain convergence issues (unlike MCMC).

SVGD is particularly good for multimodal posteriors where mean-field VI would collapse to one mode. It’s still a research method more than a production tool, but it shows up in Bayesian deep learning and model ensembles.

  • Describe hierarchical Bayesian models and the partial pooling mechanism
  • Know what the Dirichlet Process and Indian Buffet Process are for (unknown number of clusters/features)
  • Use WAIC or LOO-CV for Bayesian model comparison
  • Describe Expectation Propagation and SVGD at a high level and when they improve over standard VI
