Kalman Filters

Chapter 14: Probabilistic & Bayesian ML 5 subtopics
TL;DR

A Kalman filter tracks the hidden state of a system that evolves over time — a robot's position, a missile's trajectory, your phone's GPS coordinates — by combining noisy sensor measurements with a physics-based prediction model. Because both the dynamics and observations are linear with Gaussian noise, the posterior over the hidden state stays Gaussian at every time step, and you get exact closed-form updates. The filter alternates between a predict step (project the state forward) and an update step (correct using the new measurement). When linearity breaks down, the Extended Kalman Filter and Unscented Kalman Filter handle nonlinear dynamics. When Gaussianity breaks down, particle filters take over. This section builds the math, the code, and the intuition for all of them.

Linear Dynamical System

Before you can filter, you need a model of how the world works. A linear dynamical system (also called a state space model) describes two things: how the hidden state evolves over time, and how measurements relate to that state.

State Space Model

The state equation says: the state at time t is a linear transformation of the previous state plus some random noise.

x_t = F · x_{t-1} + w_t, where w_t ~ Normal(0, Q)

The observation equation says: each measurement is a linear transformation of the current state plus measurement noise.

z_t = H · x_t + v_t, where v_t ~ Normal(0, R)

Here, F is the state transition matrix (how the state evolves), H is the observation matrix (how the state maps to measurements), Q is the process noise covariance (uncertainty in the dynamics), and R is the measurement noise covariance (sensor inaccuracy).

import numpy as np

# Example: tracking a 1D object with constant velocity
# State: [position, velocity]
# We measure position only (noisy)

dt = 1.0  # time step

# State transition: x_t = F * x_{t-1}
# position_new = position_old + velocity * dt
# velocity_new = velocity_old (constant velocity assumption)
F = np.array([
    [1, dt],
    [0,  1]
])

# Observation matrix: we only measure position
H = np.array([[1, 0]])

# Process noise: small uncertainty in the dynamics
q = 0.1  # process noise intensity
Q = q * np.array([
    [dt**3/3, dt**2/2],
    [dt**2/2, dt      ]
])

# Measurement noise
R = np.array([[1.0]])  # sensor variance

print("State transition F:\n", F)
print("Observation matrix H:\n", H)
print("Process noise Q:\n", np.round(Q, 3))
print("Measurement noise R:\n", R)

Why Linear + Gaussian Is Special

The magic of this setup: if your prior belief about the state is Gaussian, and both the transition and observation are linear with Gaussian noise, then the posterior belief about the state is also Gaussian. Gaussians are closed under linear transformations and conditioning. This means you only need to track two things — a mean vector and a covariance matrix — instead of a full distribution. That is what makes the Kalman filter exact and efficient.

Filtering: The Predict-Update Cycle

The Kalman filter processes measurements one at a time. At each step, it does two things: predict where the state should be based on the dynamics model, then update that prediction using the new measurement. The result is an optimal estimate — in the sense of minimizing mean squared error — of the hidden state given all measurements so far.

Prediction Step

Project the state estimate and its uncertainty forward in time using the dynamics model:

x̂_t|t-1 = F · x̂_{t-1|t-1} (predicted state mean)

P_t|t-1 = F · P_{t-1|t-1} · Fᵀ + Q (predicted state covariance)

The covariance grows because the process noise Q adds uncertainty. Your belief about where the state is becomes less precise with every prediction step.

Update Step

When a new measurement z_t arrives, compute the Kalman gain — a matrix that determines how much to trust the measurement versus the prediction:

S = H · P_t|t-1 · Hᵀ + R (innovation covariance)

K = P_t|t-1 · Hᵀ · S⁻¹ (Kalman gain)

Then correct the predicted state using the measurement residual (how far off the prediction was):

x̂_t|t = x̂_t|t-1 + K · (z_t − H · x̂_t|t-1) (updated state mean)

P_t|t = (I − K · H) · P_t|t-1 (updated state covariance)

The Kalman gain is a weighted compromise. If the measurement noise R is small (precise sensor), K is large and the filter trusts the measurement. If R is large (noisy sensor), K is small and the filter trusts the prediction more.

import numpy as np

class KalmanFilter:
    """Standard Kalman filter for linear dynamical systems."""

    def __init__(self, F, H, Q, R, x0, P0):
        self.F = F   # state transition
        self.H = H   # observation matrix
        self.Q = Q   # process noise covariance
        self.R = R   # measurement noise covariance
        self.x = x0  # initial state estimate
        self.P = P0  # initial covariance

    def predict(self):
        """Project state and covariance forward."""
        self.x = self.F @ self.x
        self.P = self.F @ self.P @ self.F.T + self.Q
        return self.x.copy(), self.P.copy()

    def update(self, z):
        """Correct prediction using measurement z."""
        # Innovation (measurement residual)
        y = z - self.H @ self.x

        # Innovation covariance
        S = self.H @ self.P @ self.H.T + self.R

        # Kalman gain
        K = self.P @ self.H.T @ np.linalg.inv(S)

        # Updated state and covariance
        self.x = self.x + K @ y
        I = np.eye(len(self.x))
        self.P = (I - K @ self.H) @ self.P
        return self.x.copy(), self.P.copy()

# Simulate a tracking scenario
np.random.seed(42)
T = 50
true_pos = np.zeros(T)
true_vel = 1.5  # constant true velocity
for t in range(1, T):
    true_pos[t] = true_pos[t-1] + true_vel * dt + np.random.normal(0, 0.3)

# Noisy measurements
measurements = true_pos + np.random.normal(0, 1.0, T)

# Run Kalman filter
x0 = np.array([0.0, 0.0])  # start with zero position and velocity
P0 = np.eye(2) * 10.0       # high initial uncertainty
kf = KalmanFilter(F, H, Q, R, x0, P0)

filtered_pos = []
filtered_vel = []
uncertainties = []

for t in range(T):
    kf.predict()
    x_est, P_est = kf.update(np.array([measurements[t]]))
    filtered_pos.append(x_est[0])
    filtered_vel.append(x_est[1])
    uncertainties.append(np.sqrt(P_est[0, 0]))

print(f"True velocity: {true_vel:.2f}")
print(f"Estimated velocity (last): {filtered_vel[-1]:.2f}")
print(f"Position RMSE (raw measurements): {np.sqrt(np.mean((measurements - true_pos)**2)):.3f}")
print(f"Position RMSE (Kalman filtered):  {np.sqrt(np.mean((np.array(filtered_pos) - true_pos)**2)):.3f}")
💡 Intuition

The Kalman filter is doing Bayesian inference, step by step. The prediction step is computing the prior for the next time step. The update step is computing the posterior given the new measurement. Because everything is linear and Gaussian, the posterior is exact — no approximation needed. The Kalman gain is the Bayesian update formula specialized to Gaussians.

Smoothing

The Kalman filter is an online algorithm — it estimates the state at time t using only measurements up to time t. But what if you have the entire sequence of measurements and want the best possible estimate of the state at every time step? That is smoothing: estimate x_t using measurements from time 1 through T, where T > t.

Rauch-Tung-Striebel (RTS) Smoother

The RTS smoother is a backward pass that refines the Kalman filter's forward estimates. After running the Kalman filter forward, you sweep backward from t = T to t = 1, correcting each estimate using future information:

def rts_smoother(F, Q, filtered_means, filtered_covs, predicted_means, predicted_covs):
    """Rauch-Tung-Striebel smoother for fixed-interval smoothing."""
    T = len(filtered_means)
    dim = filtered_means[0].shape[0]

    smoothed_means = [None] * T
    smoothed_covs = [None] * T

    # Initialize with last filtered estimate
    smoothed_means[-1] = filtered_means[-1]
    smoothed_covs[-1] = filtered_covs[-1]

    # Backward pass
    for t in range(T - 2, -1, -1):
        # Smoother gain
        C = filtered_covs[t] @ F.T @ np.linalg.inv(predicted_covs[t + 1])

        # Smoothed estimates
        smoothed_means[t] = (filtered_means[t] +
                             C @ (smoothed_means[t + 1] - predicted_means[t + 1]))
        smoothed_covs[t] = (filtered_covs[t] +
                            C @ (smoothed_covs[t + 1] - predicted_covs[t + 1]) @ C.T)

    return smoothed_means, smoothed_covs

# Run filter again, saving intermediate results
kf2 = KalmanFilter(F, H, Q, R, x0.copy(), P0.copy())
f_means, f_covs = [], []
p_means, p_covs = [], []

for t in range(T):
    x_pred, P_pred = kf2.predict()
    p_means.append(x_pred)
    p_covs.append(P_pred)
    x_filt, P_filt = kf2.update(np.array([measurements[t]]))
    f_means.append(x_filt)
    f_covs.append(P_filt)

s_means, s_covs = rts_smoother(F, Q, f_means, f_covs, p_means, p_covs)

smoothed_pos = [m[0] for m in s_means]
print(f"Position RMSE (filtered):  {np.sqrt(np.mean((np.array(filtered_pos) - true_pos)**2)):.3f}")
print(f"Position RMSE (smoothed):  {np.sqrt(np.mean((np.array(smoothed_pos) - true_pos)**2)):.3f}")

Smoothed estimates are always at least as good as filtered estimates because they use more information. The improvement is most visible at the beginning and middle of the sequence, where the filter had not yet seen all the data.

Extensions

Extended Kalman Filter (EKF)

Real systems are rarely linear. A robot turning corners, a satellite in orbit, a car navigating a curved road — these all have nonlinear dynamics. The Extended Kalman Filter handles this by linearizing the system around the current estimate using a first-order Taylor expansion (the Jacobian). You replace F with the Jacobian of the state transition function and H with the Jacobian of the observation function, then run the standard Kalman equations.

class ExtendedKalmanFilter:
    """EKF for nonlinear systems using Jacobian linearization."""

    def __init__(self, f, h, F_jacobian, H_jacobian, Q, R, x0, P0):
        self.f = f              # nonlinear state transition function
        self.h = h              # nonlinear observation function
        self.F_jac = F_jacobian # Jacobian of f
        self.H_jac = H_jacobian # Jacobian of h
        self.Q = Q
        self.R = R
        self.x = x0
        self.P = P0

    def predict(self):
        F = self.F_jac(self.x)
        self.x = self.f(self.x)
        self.P = F @ self.P @ F.T + self.Q
        return self.x.copy(), self.P.copy()

    def update(self, z):
        H = self.H_jac(self.x)
        y = z - self.h(self.x)
        S = H @ self.P @ H.T + self.R
        K = self.P @ H.T @ np.linalg.inv(S)
        self.x = self.x + K @ y
        I = np.eye(len(self.x))
        self.P = (I - K @ H) @ self.P
        return self.x.copy(), self.P.copy()

# Example: tracking with nonlinear observation (range + bearing)
# State: [x, y, vx, vy], Observation: [range, angle] from origin
def f_nonlinear(x):
    dt = 1.0
    return np.array([x[0] + dt*x[2], x[1] + dt*x[3], x[2], x[3]])

def h_nonlinear(x):
    r = np.sqrt(x[0]**2 + x[1]**2)
    theta = np.arctan2(x[1], x[0])
    return np.array([r, theta])

def F_jac(x):
    dt = 1.0
    return np.array([[1,0,dt,0],[0,1,0,dt],[0,0,1,0],[0,0,0,1]])

def H_jac(x):
    r = np.sqrt(x[0]**2 + x[1]**2) + 1e-10
    return np.array([
        [x[0]/r, x[1]/r, 0, 0],
        [-x[1]/r**2, x[0]/r**2, 0, 0]
    ])

print("EKF initialized for range-bearing tracking")

The EKF works well when the nonlinearity is mild. But when the system is highly nonlinear, the linearization introduces errors that can cause the filter to diverge.

Unscented Kalman Filter (UKF)

The Unscented Kalman Filter avoids linearization entirely. Instead, it picks a carefully chosen set of sigma points around the current mean, passes them through the nonlinear function, and reconstructs the transformed mean and covariance from the output points. This captures the nonlinearity more accurately than a first-order Taylor expansion, with roughly the same computational cost.

from filterpy.kalman import UnscentedKalmanFilter, MerweScaledSigmaPoints
import numpy as np

# UKF using filterpy library
sigmas = MerweScaledSigmaPoints(n=4, alpha=0.1, beta=2.0, kappa=-1)

def fx(x, dt):
    """State transition function."""
    return np.array([x[0] + dt*x[2], x[1] + dt*x[3], x[2], x[3]])

def hx(x):
    """Observation function: range and bearing."""
    return np.array([
        np.sqrt(x[0]**2 + x[1]**2),
        np.arctan2(x[1], x[0])
    ])

ukf = UnscentedKalmanFilter(dim_x=4, dim_z=2, dt=1.0,
                             fx=fx, hx=hx, points=sigmas)
ukf.x = np.array([10.0, 5.0, 1.0, 0.5])
ukf.P *= 5
ukf.Q = np.eye(4) * 0.1
ukf.R = np.diag([0.5, 0.02])  # range noise, angle noise

# One predict-update cycle
ukf.predict()
z = np.array([11.2, 0.45])  # simulated measurement
ukf.update(z)
print(f"UKF state estimate: {np.round(ukf.x, 3)}")

Particle Filters (Sequential Monte Carlo)

When the state space is not even approximately Gaussian — multimodal distributions, hard constraints, highly nonlinear dynamics — both EKF and UKF break down. Particle filters represent the posterior distribution as a weighted set of samples (particles). Each particle is a hypothesis about the current state. At each time step, you propagate particles through the dynamics, weight them by how well they explain the new measurement, and resample to focus particles on high-probability regions.

import numpy as np

def particle_filter(measurements, n_particles=1000):
    """Simple bootstrap particle filter for 1D tracking."""
    T = len(measurements)
    particles = np.random.normal(0, 5, size=n_particles)
    weights = np.ones(n_particles) / n_particles
    estimates = np.zeros(T)

    for t in range(T):
        # Predict: propagate particles through dynamics
        particles = particles + 1.5 + np.random.normal(0, 0.3, n_particles)

        # Update: weight by measurement likelihood
        likelihood = np.exp(-0.5 * ((measurements[t] - particles) / 1.0) ** 2)
        weights = likelihood / (likelihood.sum() + 1e-300)

        # Estimate
        estimates[t] = np.average(particles, weights=weights)

        # Resample (systematic resampling)
        n_eff = 1.0 / np.sum(weights ** 2)
        if n_eff < n_particles / 2:
            cumsum = np.cumsum(weights)
            u = (np.arange(n_particles) + np.random.uniform()) / n_particles
            indices = np.searchsorted(cumsum, u)
            particles = particles[indices]
            weights = np.ones(n_particles) / n_particles

    return estimates

pf_estimates = particle_filter(measurements, n_particles=2000)
print(f"Position RMSE (particle filter): "
      f"{np.sqrt(np.mean((pf_estimates - true_pos)**2)):.3f}")
⚠️ Particle Degeneracy

Without resampling, particle weights concentrate on a few particles over time, and most particles contribute nothing. This is weight degeneracy. Systematic resampling helps, but in high dimensions even resampling struggles. The effective number of particles, N_eff = 1/Σwᵢ², measures how many particles are actually useful. When N_eff drops below N/2, resample.

Applications

Robot Localization

A mobile robot moving through a building uses wheel encoders (odometry) and laser range finders or cameras. The odometry tells it roughly how far it moved (the dynamics model), and the sensors tell it roughly where it is relative to landmarks (the observation model). A Kalman filter or particle filter fuses these two sources of information to maintain a belief about the robot's pose (x, y, heading).

GPS and Navigation

Your phone's GPS chip runs a Kalman filter continuously. The state includes position, velocity, and clock bias. GPS satellite signals give noisy position measurements. The dynamics model assumes smooth motion. The Kalman filter combines these to give you a smooth, accurate position estimate even when individual satellite measurements jump around.

Signal Processing and Finance

Kalman filters are used in speech enhancement (separate speech from noise), econometrics (estimate time-varying parameters in regression models), and algorithmic trading (estimate the "fair value" of an asset as a latent state observed through noisy prices).

# Kalman filter with pykalman for time-varying regression
from pykalman import KalmanFilter as PyKalmanFilter
import numpy as np

np.random.seed(42)
T = 200

# True time-varying coefficient: slowly drifts
true_beta = np.cumsum(np.random.normal(0, 0.02, T)) + 3.0
X = np.random.normal(0, 1, T)
y = true_beta * X + np.random.normal(0, 0.5, T)

# State: beta_t (the coefficient)
# Observation: y_t = X_t * beta_t + noise
# This requires time-varying observation matrix H_t = [[X_t]]
obs_matrices = X.reshape(T, 1, 1)

kf = PyKalmanFilter(
    transition_matrices=np.array([[1.0]]),
    observation_matrices=obs_matrices,
    transition_covariance=np.array([[0.01]]),
    observation_covariance=np.array([[0.25]]),
    initial_state_mean=np.array([3.0]),
    initial_state_covariance=np.array([[1.0]])
)

state_means, state_covs = kf.filter(y.reshape(-1, 1))
print(f"Final true beta: {true_beta[-1]:.3f}")
print(f"Final estimated beta: {state_means[-1, 0]:.3f}")
print(f"Beta tracking RMSE: {np.sqrt(np.mean((state_means[:, 0] - true_beta)**2)):.3f}")
