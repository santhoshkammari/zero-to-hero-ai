Approximate Inference: Variational Methods

Chapter 14: Probabilistic & Bayesian ML 8 subtopics
TL;DR

When exact inference is intractable, variational methods turn the inference problem into an optimization problem. Instead of computing the true posterior, find the closest tractable distribution by maximizing the Evidence Lower Bound (ELBO). Mean field VI uses fully factorized approximations. Stochastic VI scales to massive datasets. The reparameterization trick enables gradient-based optimization and powers Variational Autoencoders. Normalizing flows and the Laplace approximation offer alternative ways to approximate complex posteriors with different accuracy-cost tradeoffs.

Here is the fundamental problem. You have a model with latent variables z, observed data x, and you need the posterior P(z|x). Bayes' theorem says P(z|x) = P(x|z)P(z)/P(x). The denominator P(x) = ∫ P(x|z)P(z) dz requires integrating over all possible values of z. For a deep generative model with millions of latent dimensions, that integral is hopelessly intractable. No analytical solution. No efficient numerical method.

Variational inference takes a radically different approach. Instead of computing the true posterior, define a family of tractable distributions q(z; φ) parameterized by φ. Then find the member of this family closest to the true posterior, where "closest" means minimizing KL divergence. The inference problem becomes an optimization problem — something we know how to do with gradient descent.

Variational Inference (VI)

KL Divergence Minimization

We want to find q(z) that minimizes KL(q(z) || p(z|x)). This KL divergence measures how much information is lost when using q to approximate the true posterior p. The problem: computing this KL requires knowing p(z|x), which is exactly what we cannot compute. The trick is to rearrange the math to get a computable objective.

Evidence Lower Bound (ELBO)

Expanding the KL divergence: KL(q||p) = E_q[log q(z)] - E_q[log p(z|x)]. Substituting Bayes' theorem and rearranging: log p(x) = ELBO + KL(q||p). Since KL is always non-negative, ELBO ≤ log p(x). The ELBO (Evidence Lower Bound) is:

ELBO = E_q[log p(x,z)] - E_q[log q(z)]

Or equivalently: ELBO = E_q[log p(x|z)] - KL(q(z) || p(z))

The first form says: expected joint log-likelihood minus the entropy cost of q. The second form is more intuitive: expected log-likelihood (how well the model explains data) minus a penalty for q deviating from the prior. Maximizing the ELBO simultaneously makes q a good approximation to the posterior and maximizes a lower bound on the model evidence.

import numpy as np
from scipy import stats

# ELBO computation for a simple Gaussian model
# True model: z ~ N(0, 1), x|z ~ N(z, 0.5^2)
# Observed: x = 2.0
# True posterior: N(mu_post, sigma_post^2)

x_obs = 2.0
sigma_x = 0.5
sigma_z = 1.0

# Exact posterior (Gaussian conjugacy)
sigma_post = 1.0 / (1.0/sigma_z**2 + 1.0/sigma_x**2)
mu_post = sigma_post * (x_obs / sigma_x**2)
print(f"True posterior: N({mu_post:.3f}, {sigma_post:.3f})")

# Variational approximation: q(z) = N(mu_q, sigma_q^2)
def compute_elbo(mu_q, log_sigma_q, x_obs, sigma_x, sigma_z):
    sigma_q = np.exp(log_sigma_q)

    # E_q[log p(x|z)] = E_q[-0.5*log(2pi*sigma_x^2) - (x-z)^2/(2*sigma_x^2)]
    expected_ll = -0.5*np.log(2*np.pi*sigma_x**2) \
                  - (1/(2*sigma_x**2)) * (sigma_q**2 + (x_obs - mu_q)**2)

    # KL(q(z) || p(z)) where p(z) = N(0, sigma_z^2)
    kl = 0.5 * (sigma_q**2/sigma_z**2 + mu_q**2/sigma_z**2
                 - 1 - 2*log_sigma_q + 2*np.log(sigma_z))

    return expected_ll - kl

# Gradient ascent to optimize ELBO
mu_q = 0.0
log_sigma_q = 0.0  # sigma_q = 1.0
lr = 0.01

for step in range(2000):
    # Numerical gradients (for clarity; real code uses autograd)
    eps = 1e-5
    elbo = compute_elbo(mu_q, log_sigma_q, x_obs, sigma_x, sigma_z)
    grad_mu = (compute_elbo(mu_q+eps, log_sigma_q, x_obs, sigma_x, sigma_z) - elbo) / eps
    grad_ls = (compute_elbo(mu_q, log_sigma_q+eps, x_obs, sigma_x, sigma_z) - elbo) / eps
    mu_q += lr * grad_mu
    log_sigma_q += lr * grad_ls

print(f"VI approximation: N({mu_q:.3f}, {np.exp(log_sigma_q)**2:.3f})")
print(f"ELBO at optimum: {compute_elbo(mu_q, log_sigma_q, x_obs, sigma_x, sigma_z):.3f}")
# Should match the true posterior closely

Mean Field Variational Inference

Fully Factorized Approximation

Mean field VI assumes the variational distribution fully factorizes: q(z) = ∏ᵢ q_i(z_i). Each latent variable gets its own independent distribution. This is a strong assumption — it ignores all posterior correlations. But it makes the optimization tractable. Each factor q_i can be optimized independently while holding others fixed.

Coordinate Ascent Updates

The optimal form of each factor (holding others fixed) is: log q_j*(z_j) = E_{q_{-j}}[log p(z, x)] + const. You take the log of the joint distribution, average over all variables except z_j, and exponentiate. This gives a closed-form update when the model is in the exponential family. Cycle through all factors, updating each in turn. This is coordinate ascent variational inference (CAVI) — guaranteed to increase the ELBO at each step.

import numpy as np

# Mean field CAVI for a Gaussian mixture model
# Model: pi ~ Dirichlet(alpha), mu_k ~ N(0, sigma_0^2), x_n | z_n ~ N(mu_{z_n}, sigma^2)
# Variational: q(z, mu, pi) = prod_n q(z_n) * prod_k q(mu_k) * q(pi)

def mean_field_gmm(X, K=3, n_iter=50, seed=42):
    np.random.seed(seed)
    N = len(X)

    # Hyperparameters
    alpha_0 = np.ones(K)  # Dirichlet prior
    mu_0, sigma_0 = 0.0, 10.0  # prior on cluster means
    sigma = 1.0  # known observation noise

    # Initialize responsibilities randomly
    r = np.random.dirichlet(np.ones(K), size=N)  # N x K

    elbos = []
    for it in range(n_iter):
        # Update q(mu_k): Gaussian with precision = 1/sigma_0^2 + N_k/sigma^2
        N_k = r.sum(axis=0)  # effective cluster sizes
        x_bar_k = (r.T @ X) / (N_k + 1e-10)  # weighted means

        prec_k = 1/sigma_0**2 + N_k/sigma**2
        mu_k = (N_k * x_bar_k / sigma**2) / prec_k  # posterior means
        sigma_k = 1.0 / prec_k  # posterior variances

        # Update q(pi): Dirichlet with updated counts
        alpha_k = alpha_0 + N_k

        # Update q(z_n): responsibilities
        from scipy.special import digamma
        E_log_pi = digamma(alpha_k) - digamma(alpha_k.sum())
        for n in range(N):
            log_r = E_log_pi - 0.5/sigma**2 * ((X[n] - mu_k)**2 + sigma_k)
            log_r -= log_r.max()
            r[n] = np.exp(log_r)
            r[n] /= r[n].sum()

        if it % 10 == 0:
            print(f"Iter {it:3d}: cluster means = {np.round(mu_k, 2)}, "
                  f"sizes = {np.round(N_k, 1)}")

    return mu_k, r

# Generate mixture data
np.random.seed(42)
X = np.concatenate([
    np.random.normal(-3, 0.8, 100),
    np.random.normal(0, 0.8, 150),
    np.random.normal(4, 0.8, 100)
])

mu_k, r = mean_field_gmm(X, K=3)
print(f"\nFinal cluster means: {np.sort(mu_k).round(2)}")

Underestimates Uncertainty

Mean field VI minimizes KL(q||p), which penalizes q for placing mass where p is low. This makes q "mode-seeking" — it snaps to one mode of the posterior and ignores others. For multimodal posteriors, mean field captures only one peak. Even for unimodal posteriors, the factorized assumption cannot capture correlations, so the posterior variance is underestimated. Keep this in mind: variational methods tend to give overconfident uncertainty estimates.

The KL Direction Matters

KL(q||p) is not the same as KL(p||q). Mean field minimizes KL(q||p), which is zero-avoiding (q tries not to place mass where p is zero). Expectation Propagation minimizes KL(p||q), which is zero-forcing (q covers everywhere p has mass). The choice of KL direction fundamentally affects the approximation quality. Neither is universally better — it depends on whether underestimating or overestimating uncertainty is more dangerous for your application.

Stochastic Variational Inference (SVI)

Natural Gradients and Mini-Batch Updates

CAVI processes the entire dataset at each step — that does not scale. Stochastic variational inference (SVI) uses mini-batches, like stochastic gradient descent for neural networks. The key insight: use natural gradients instead of ordinary gradients. Natural gradients account for the curvature of the parameter space (the Fisher information metric), making updates that are invariant to the parameterization of q. This leads to faster convergence and more stable optimization.

The SVI update for global variational parameters is: φ_{t+1} = (1-ρ_t)φ_t + ρ_t φ̂_t, where φ̂_t is the natural parameter estimated from a mini-batch, and ρ_t is a learning rate following a Robbins-Monro schedule. This turns Bayesian inference into something that looks remarkably like SGD — and it scales to millions of data points.

import numpy as np

# SVI for a simple conjugate model: Gaussian with unknown mean
# Prior: mu ~ N(mu_0, sigma_0^2)
# Likelihood: x_n | mu ~ N(mu, sigma^2)

def svi_gaussian_mean(X, batch_size=32, n_steps=500, seed=42):
    np.random.seed(seed)
    N = len(X)
    sigma = 1.0  # known noise

    # Prior natural parameters
    eta1_prior = 0.0  # mu_0 / sigma_0^2
    eta2_prior = -0.5 / 10.0**2  # -1/(2*sigma_0^2)

    # Initialize variational natural parameters
    eta1 = eta1_prior
    eta2 = eta2_prior

    for t in range(n_steps):
        rho = 1.0 / (t + 10)  # learning rate

        # Sample mini-batch
        batch = np.random.choice(X, size=batch_size, replace=True)

        # Natural parameter updates from mini-batch (scaled to full dataset)
        eta1_batch = eta1_prior + (N/batch_size) * np.sum(batch) / sigma**2
        eta2_batch = eta2_prior + (N/batch_size) * (-0.5 * batch_size / sigma**2)

        # Stochastic natural gradient step
        eta1 = (1 - rho) * eta1 + rho * eta1_batch
        eta2 = (1 - rho) * eta2 + rho * eta2_batch

    # Convert natural parameters to mean/variance
    sigma_q = np.sqrt(-0.5 / eta2)
    mu_q = eta1 * sigma_q**2
    return mu_q, sigma_q

# Test
np.random.seed(42)
X = np.random.normal(3.0, 1.0, 10000)  # true mean = 3.0
mu_q, sigma_q = svi_gaussian_mean(X)
print(f"SVI posterior: N({mu_q:.4f}, {sigma_q:.4f}^2)")
print(f"Exact posterior mean: {np.mean(X):.4f}")
# SVI converges to the correct answer using mini-batches

Black-Box Variational Inference

Score Function Estimator (REINFORCE)

What if your model is not in the exponential family? Black-box VI uses Monte Carlo estimates of the ELBO gradient, requiring only the ability to sample from q and evaluate log p(x,z). The score function estimator (REINFORCE) computes: ∇_φ ELBO ≈ (1/S) Σ_s [f(z_s) × ∇_φ log q(z_s; φ)], where z_s ~ q(z; φ) and f(z_s) = log p(x, z_s) - log q(z_s; φ). This works for any differentiable q, but has high variance.

Reparameterization Trick

The reparameterization trick is the innovation that made modern variational inference practical. Instead of sampling z ~ q(z; φ) directly, write z = g(ε, φ) where ε ~ p(ε) is a fixed noise distribution. For Gaussians: z = μ + σ × ε, where ε ~ N(0,1). Now the gradient of the ELBO can flow through z back to φ because the randomness is in ε, not in z. This gives much lower variance gradients than the score function estimator.

import numpy as np

# Reparameterization trick for a Gaussian variational distribution
def reparameterize(mu, log_sigma, n_samples=1):
    """Sample z = mu + sigma * epsilon, epsilon ~ N(0, I)."""
    sigma = np.exp(log_sigma)
    eps = np.random.randn(n_samples, len(mu))
    z = mu + sigma * eps  # differentiable w.r.t. mu and log_sigma!
    return z, eps

# Example: 2D latent space
mu = np.array([1.0, -0.5])
log_sigma = np.array([-1.0, -0.5])

z_samples, eps = reparameterize(mu, log_sigma, n_samples=1000)
print(f"Sample mean: {z_samples.mean(axis=0).round(3)}")
print(f"Sample std:  {z_samples.std(axis=0).round(3)}")
print(f"Target std:  {np.exp(log_sigma).round(3)}")
# Gradients flow through mu and log_sigma — this enables backprop in VAEs

Automatic Differentiation Variational Inference (ADVI)

ADVI is the "just works" approach to variational inference. It automatically transforms constrained parameters to unconstrained space, uses a Gaussian variational family in the unconstrained space, and optimizes the ELBO using the reparameterization trick and automatic differentiation. PyMC and Stan implement this — you specify the model, call fit(method='advi'), and get an approximate posterior without writing a single update equation.

import pymc as pm
import numpy as np

# ADVI in PyMC — automatic variational inference
np.random.seed(42)
N = 200
true_mu, true_sigma = 5.0, 2.0
data = np.random.normal(true_mu, true_sigma, N)

with pm.Model() as model:
    mu = pm.Normal("mu", mu=0, sigma=10)
    sigma = pm.HalfNormal("sigma", sigma=5)
    obs = pm.Normal("obs", mu=mu, sigma=sigma, observed=data)

    # ADVI: automatic variational inference
    approx = pm.fit(method="advi", n=20000, random_seed=42)
    trace = approx.sample(2000)

print(f"True: mu={true_mu}, sigma={true_sigma}")
print(f"ADVI: mu={trace.posterior['mu'].mean().item():.2f}, "
      f"sigma={trace.posterior['sigma'].mean().item():.2f}")

Variational Autoencoders (VAEs)

Amortized Inference

Standard VI optimizes variational parameters for each data point separately. Amortized inference trains a neural network (the encoder) to predict variational parameters from the input: q(z|x; φ) = N(μ_φ(x), σ²_φ(x)). One forward pass through the encoder gives you the approximate posterior for any input. This is the key insight of Variational Autoencoders (VAEs).

Generative Model and Training

A VAE has two parts. The decoder (generative model) maps latent codes to data: p(x|z; θ). The encoder (recognition model) maps data to approximate posteriors: q(z|x; φ). Training maximizes the ELBO: ELBO = E_{q(z|x)}[log p(x|z)] - KL(q(z|x) || p(z)). The reconstruction term pushes the decoder to be accurate. The KL term pushes the encoder toward the prior p(z) = N(0, I), keeping the latent space smooth and regularized.

import numpy as np

class SimpleVAE:
    """Minimal VAE with linear encoder/decoder for illustration."""

    def __init__(self, input_dim, latent_dim):
        scale = 0.01
        self.W_enc_mu = np.random.randn(input_dim, latent_dim) * scale
        self.W_enc_ls = np.random.randn(input_dim, latent_dim) * scale
        self.W_dec = np.random.randn(latent_dim, input_dim) * scale

    def encode(self, x):
        mu = x @ self.W_enc_mu
        log_sigma = x @ self.W_enc_ls
        return mu, log_sigma

    def decode(self, z):
        return z @ self.W_dec

    def elbo(self, x, n_samples=1):
        mu, log_sigma = self.encode(x)
        sigma = np.exp(log_sigma)

        # Reparameterization trick
        eps = np.random.randn(n_samples, mu.shape[-1])
        z = mu + sigma * eps

        # Reconstruction (Gaussian likelihood)
        x_recon = self.decode(z)
        recon_loss = -0.5 * np.sum((x - x_recon)**2, axis=-1).mean()

        # KL divergence: KL(N(mu, sigma^2) || N(0, 1))
        kl = -0.5 * np.sum(1 + 2*log_sigma - mu**2 - sigma**2)

        return recon_loss - kl

# The ELBO decomposes into reconstruction quality vs latent regularization
vae = SimpleVAE(input_dim=10, latent_dim=2)
x_sample = np.random.randn(10)
print(f"ELBO: {vae.elbo(x_sample):.3f}")

Latent Space Regularization

The KL term in the VAE objective serves as a regularizer on the latent space. Without it, the encoder could map each data point to a unique, isolated location in latent space — perfect reconstruction but useless for generation. The KL penalty ensures the latent space is continuous and smooth. Points that are close in latent space decode to similar outputs. This is what makes VAE latent spaces useful for interpolation, generation, and representation learning.

VAE vs GAN

VAEs optimize a well-defined probabilistic objective (the ELBO) and give you both a generator and an inference network. GANs often produce sharper samples but have no principled way to do inference. VAEs tend to produce blurrier outputs because they average over modes (a consequence of minimizing KL(q||p)). Hybrid approaches like VAE-GAN combine the best of both worlds.

Expectation Propagation

Moment Matching and Iterative Refinement

Expectation Propagation (EP) is an alternative to VI that minimizes KL(p||q) instead of KL(q||p). This makes it "inclusive" — the approximation tries to cover all modes of the posterior rather than snapping to one. EP works by approximating each factor in the model with a simpler factor, iteratively refining these approximations through moment matching. At each step, remove one approximate factor, compute the "cavity distribution," include the true factor, then project back to the approximating family by matching moments.

EP often gives better uncertainty estimates than mean field VI because it captures the overall scale of the posterior. However, it is not guaranteed to converge and can oscillate. It is the method behind many Gaussian process classifiers in practice.

Normalizing Flows

Change of Variables Formula

Mean field VI is limited to simple distributions (often Gaussians). What if you could transform a simple distribution into a complex one while still being able to compute densities? That is the idea behind normalizing flows. Start with z₀ ~ N(0, I), apply a sequence of invertible transformations z_K = f_K ∘ ... ∘ f_1(z₀), and use the change-of-variables formula to compute the density: log q(z_K) = log q(z₀) - Σ_k log |det ∂f_k/∂z_{k-1}|.

Types of Flows

Planar and radial flows are the simplest — each applies a single non-linear transformation. Real NVP and GLOW use affine coupling layers that are efficient to invert and have tractable Jacobian determinants. Autoregressive flows (MAF, IAF) use the autoregressive structure to ensure triangular Jacobians, making determinant computation O(d) instead of O(d³). Continuous normalizing flows use neural ODEs — defining the transformation as the solution to a differential equation — offering maximum flexibility.

import numpy as np

# Planar normalizing flow: z' = z + u * tanh(w^T z + b)
class PlanarFlow:
    def __init__(self, dim):
        self.w = np.random.randn(dim) * 0.1
        self.u = np.random.randn(dim) * 0.1
        self.b = 0.0

    def forward(self, z):
        # z: (batch, dim)
        linear = z @ self.w + self.b  # (batch,)
        h = np.tanh(linear)           # (batch,)
        z_new = z + np.outer(h, self.u)  # (batch, dim)
        return z_new

    def log_det_jacobian(self, z):
        linear = z @ self.w + self.b
        h_prime = 1 - np.tanh(linear)**2  # dtanh/dx
        psi = np.outer(h_prime, self.w)    # (batch, dim)
        det = np.abs(1 + psi @ self.u)     # (batch,)
        return np.log(det + 1e-10)

# Stack multiple flows
dim = 2
n_flows = 5
flows = [PlanarFlow(dim) for _ in range(n_flows)]

# Transform samples from N(0, I)
z = np.random.randn(1000, dim)
log_det_sum = np.zeros(1000)

for flow in flows:
    log_det_sum += flow.log_det_jacobian(z)
    z = flow.forward(z)

print(f"Before flows: mean={np.mean(z, axis=0).round(3)}, std={np.std(z, axis=0).round(3)}")
# After stacking flows, the distribution becomes more complex

Improving Variational Approximations with Flows

Normalizing flows can augment a basic Gaussian variational distribution into something much more expressive. Start with a Gaussian q₀, apply K flow transformations, and get q_K — a complex, multi-modal distribution. This is used in VAEs (e.g., IAF-VAE) and in variational inference for complex posteriors. The tradeoff: more flows give better approximations but increase computational cost.

Laplace Approximation

Second-Order Taylor Expansion at MAP

The Laplace approximation is the oldest and simplest approach to approximate inference. Find the MAP estimate (the posterior mode), then approximate the posterior as a Gaussian centered at the MAP with covariance equal to the inverse Hessian of the negative log posterior. This is a second-order Taylor expansion of the log posterior around its peak.

q(θ) = N(θ|θ_MAP, H⁻¹), where H = -∇²log p(θ|D)|_{θ=θ_MAP}

import numpy as np
from scipy.optimize import minimize

# Laplace approximation for logistic regression
def log_posterior(w, X, y, alpha=1.0):
    """Log posterior for Bayesian logistic regression."""
    logits = X @ w
    log_lik = np.sum(y * logits - np.log(1 + np.exp(logits)))
    log_prior = -0.5 * alpha * np.sum(w**2)
    return log_lik + log_prior

def neg_log_posterior(w, X, y, alpha=1.0):
    return -log_posterior(w, X, y, alpha)

# Generate data
np.random.seed(42)
N, D = 200, 3
X = np.random.randn(N, D)
w_true = np.array([1.0, -2.0, 0.5])
probs = 1 / (1 + np.exp(-X @ w_true))
y = (np.random.rand(N) < probs).astype(float)

# Find MAP estimate
result = minimize(neg_log_posterior, np.zeros(D), args=(X, y, 1.0), method='L-BFGS-B')
w_map = result.x

# Compute Hessian at MAP for Laplace approximation
def compute_hessian(w, X, y, alpha=1.0):
    logits = X @ w
    p = 1 / (1 + np.exp(-logits))
    W_diag = p * (1 - p)
    H = X.T @ np.diag(W_diag) @ X + alpha * np.eye(len(w))
    return H

H = compute_hessian(w_map, X, y)
cov_laplace = np.linalg.inv(H)

print(f"MAP estimate:  {w_map.round(3)}")
print(f"True weights:  {w_true}")
print(f"Posterior std:  {np.sqrt(np.diag(cov_laplace)).round(3)}")
# The Laplace approximation gives us uncertainty estimates for free
# after a single optimization run

Comparison with VI and MCMC

The Laplace approximation is fast (one optimization + one Hessian) but limited to unimodal, roughly Gaussian posteriors. VI is more flexible (especially with flows) but requires iterative optimization of the ELBO. MCMC is asymptotically exact but slow. The tradeoffs form a spectrum:

Laplace (fast, crude) → Mean field VI (moderate, factorized) → Full-rank VI (slower, captures correlations) → Normalizing flows (expensive, flexible) → MCMC (slowest, exact)

Laplace Approximation for Neural Networks

There is a renaissance of the Laplace approximation in deep learning. Train your neural network as usual to get the MAP estimate. Then compute (or approximate) the Hessian of the loss at the trained weights. This gives you a Gaussian distribution over the weights — instant Bayesian neural network. The Hessian is huge (millions of parameters), so approximations like diagonal, KFAC, or low-rank Hessians are used. Libraries like laplace-torch make this practical. It is the cheapest path from a trained network to calibrated uncertainty estimates.

No Free Lunch in Approximate Inference

Every approximation makes tradeoffs. Mean field underestimates uncertainty. The Laplace approximation assumes Gaussianity. Normalizing flows add computational cost. MCMC is slow and hard to diagnose convergence. There is no universally best method. Start with the simplest approach that gives acceptable results (often ADVI or Laplace), then upgrade to more sophisticated methods only if the uncertainty estimates are not reliable enough for your application.
