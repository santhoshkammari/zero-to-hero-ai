Sequential Models: HMMs & Kalman Filters

Chapter 14: Probabilistic & Bayesian ML 5 subtopics
TL;DR

Hidden Markov Models and Kalman Filters are the two classic approaches to sequential Bayesian inference — tracking a hidden state that evolves over time from noisy observations. HMMs handle discrete hidden states; Kalman Filters handle continuous Gaussian states. Both use the same predict-update cycle that is the heartbeat of Bayesian thinking. They built speech recognition and GPS navigation before deep learning existed.

Both HMMs and Kalman Filters solve the same fundamental problem: you can’t observe the true state of the world directly. You only see noisy measurements. You know (approximately) how the world evolves over time. How do you infer the hidden state? The answer is recursive Bayesian estimation: predict forward using the dynamics model, then correct using the observation. Prior → posterior, over and over.

Hidden Markov Models

An HMM has discrete hidden states that transition according to a probability matrix, and each state emits an observation. Formally:

  • Hidden states zt ∈ {1, ..., K} with transition matrix A: Aij = P(zt=j | zt-1=i)
  • Observations xt emitted from each state with emission probability B: P(xt | zt)
  • Initial distribution π: P(z1)

Three fundamental problems:

1. Evaluation (Forward algorithm): What is P(observations | model)? Useful for model comparison. The forward algorithm computes this in O(TK²) by maintaining forward probabilities αt(j) = P(x1:t, zt=j).

2. Decoding (Viterbi algorithm): What is the most likely sequence of hidden states? Uses dynamic programming to find argmax P(z1:T | x1:T). This is the algorithm behind POS tagging and gene finding.

3. Learning (Baum-Welch / EM): Estimate the model parameters (A, B, π) from data. This is EM applied to HMMs: E-step computes expected state occupancies (forward-backward), M-step updates parameters.

from hmmlearn import hmm
import numpy as np

# Fit an HMM to sequential data (e.g., stock market regimes)
np.random.seed(42)

# Simulated daily returns from 2 hidden regimes
# Regime 0: low volatility, slight positive drift
# Regime 1: high volatility, negative drift
model = hmm.GaussianHMM(n_components=2, covariance_type="full", n_iter=100)
model.startprob_ = np.array([0.6, 0.4])
model.transmat_ = np.array([[0.95, 0.05], [0.10, 0.90]])
model.means_ = np.array([[0.05], [-0.02]])
model.covars_ = np.array([[[0.01]], [[0.05]]])

X, states = model.sample(500)

# Fit a NEW model to the observed returns (pretend we don't know the params)
learned = hmm.GaussianHMM(n_components=2, covariance_type="full",
                           n_iter=200, random_state=42)
learned.fit(X)

# Decode: infer the most likely regime at each time step
predicted_states = learned.predict(X)
accuracy = np.mean(predicted_states == states)
print(f"State recovery accuracy: {accuracy:.1%}")
print(f"Learned transition matrix:\n{learned.transmat_.round(3)}")

Heritage: HMMs powered speech recognition from the 1970s through the 2010s (before end-to-end deep learning). They remain the standard approach for gene finding in bioinformatics, and they’re still used in finance for regime detection.

Kalman Filters

Where HMMs handle discrete hidden states, Kalman Filters handle continuous Gaussian states with linear dynamics. The model:

State evolution:    z_t = F z_{t-1} + w_t,    w_t ~ N(0, Q)
Observation:        x_t = H z_t + v_t,         v_t ~ N(0, R)

The predict-update cycle:

  1. Predict: Use the dynamics model to propagate the state forward. The uncertainty grows.
  2. Update: Incorporate the new observation. The uncertainty shrinks. The Kalman gain determines how much to trust the prediction vs. the measurement.

The Kalman Filter is the optimal estimator for linear-Gaussian systems. Every step is a closed-form Bayesian update — the posterior at time t becomes the prior for time t+1.

import numpy as np

class KalmanFilter1D:
    """Minimal 1D Kalman filter for intuition."""
    def __init__(self, F=1.0, H=1.0, Q=0.1, R=1.0, x0=0.0, P0=1.0):
        self.F, self.H, self.Q, self.R = F, H, Q, R
        self.x = x0  # state estimate
        self.P = P0  # state uncertainty

    def predict(self):
        self.x = self.F * self.x
        self.P = self.F * self.P * self.F + self.Q

    def update(self, z):
        K = self.P * self.H / (self.H * self.P * self.H + self.R)  # Kalman gain
        self.x = self.x + K * (z - self.H * self.x)
        self.P = (1 - K * self.H) * self.P
        return self.x, self.P

# Track a noisy 1D position
np.random.seed(42)
true_positions = np.cumsum(np.random.randn(50) * 0.3)  # random walk
measurements = true_positions + np.random.randn(50) * 1.0  # noisy observations

kf = KalmanFilter1D(Q=0.1, R=1.0)
estimates = []
for z in measurements:
    kf.predict()
    est, _ = kf.update(z)
    estimates.append(est)

mse_raw = np.mean((measurements - true_positions)**2)
mse_filtered = np.mean((np.array(estimates) - true_positions)**2)
print(f"Raw measurement MSE: {mse_raw:.3f}")
print(f"Kalman filtered MSE: {mse_filtered:.3f}")
# Kalman filter significantly reduces noise

Extensions for Nonlinear Systems

The standard Kalman Filter only works for linear dynamics. Real systems are rarely linear. Two extensions:

Extended Kalman Filter (EKF): Linearize the dynamics at each step using a first-order Taylor expansion. Simple but can diverge if the nonlinearity is strong. Still widely used in GPS and inertial navigation.

Unscented Kalman Filter (UKF): Propagate a set of carefully chosen “sigma points” through the nonlinear function, then recover the Gaussian approximation. No Jacobians needed, more accurate than EKF for strongly nonlinear systems.

For highly nonlinear systems, particle filters (sequential Monte Carlo) represent the posterior as a set of weighted samples. No Gaussian assumption at all, but computationally expensive.

Where They Live Today

  • Robotics: Kalman filters for sensor fusion (combining GPS, IMU, lidar). Particle filters for robot localization (SLAM).
  • Finance: HMMs for regime detection (bull/bear markets). Kalman filters for estimating unobserved factors.
  • NLP (historical): HMMs for POS tagging, named entity recognition. Replaced by CRFs, then LSTMs, then Transformers — but the conceptual framework persists.
  • Signal processing: Kalman filters are everywhere in communications, radar, and control systems.
The Bayesian thread: Both HMMs and Kalman Filters are recursive Bayesian estimators. Prior (predict) → likelihood (observe) → posterior (update) → becomes prior for next step. If you understand this loop, you understand the core of all sequential inference.
