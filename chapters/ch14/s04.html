Approximate Inference: VI & MCMC

Chapter 14: Probabilistic & Bayesian ML 10 subtopics
TL;DR

When the posterior has no closed form (which is most of the time), you have two families of solutions. Variational inference (VI) turns inference into optimization — find a simple distribution close to the true posterior. Fast, scalable, but biased. MCMC draws correlated samples from the posterior by constructing a clever Markov chain. Asymptotically exact, but slow. VI for speed, MCMC for accuracy. Know both.

Conjugate models give you closed-form posteriors. Everything else — neural networks, mixture models, hierarchical models, basically anything interesting — requires approximation. Two competing philosophies have dominated for decades, and understanding when to use which is a core skill in probabilistic ML.

Variational Inference: Turning Inference into Optimization

The core idea is elegant: you can’t compute the true posterior p(θ|D), so pick a family of simple distributions Q and find the member q(θ) that’s closest to the true posterior. “Closest” measured by KL divergence.

You want to minimize KL(q || p), but computing this requires knowing p(θ|D) — which is the thing you can’t compute. The trick: instead of minimizing KL directly, maximize a lower bound on the log-evidence. This bound is called the ELBO (Evidence Lower BOund):

ELBO(q) = E_q[log p(D, θ)] - E_q[log q(θ)]
         = E_q[log p(D|θ)] - KL(q(θ) || p(θ))
           ├────────────┘     └────────────────┘
           expected likelihood   KL from prior

Maximizing the ELBO simultaneously does two things: make q(θ) explain the data well (first term) while staying close to the prior (second term). Sound familiar? It’s the same regularization-vs-fit tradeoff, just expressed probabilistically.

Why “lower bound”? Because log p(D) = ELBO(q) + KL(q || p). Since KL ≥ 0, ELBO ≤ log p(D). Maximizing ELBO is equivalent to minimizing KL(q || p). The tighter the bound, the better q approximates the true posterior.

Mean-Field Approximation

The simplest variational family: assume all latent variables are independent. The variational distribution factors as q(θ) = ∏ qi(θi). Each factor is optimized in turn using coordinate ascent (CAVI).

This is crude — the true posterior might have strong correlations between parameters. Mean-field misses those entirely. But it’s fast and often gives reasonable point estimates even when the uncertainty estimates are too narrow.

Amortized VI — The VAE Connection

Standard VI optimizes variational parameters for each data point separately. Amortized VI trains a neural network (the “encoder” or “recognition network”) to predict variational parameters from the input data directly. This is exactly how Variational Autoencoders work:

VAE encoder: x → q(z|x) = N(μ(x), σ²(x))   ← amortized VI
VAE decoder: z → p(x|z)                       ← generative model
Training loss: -ELBO = reconstruction loss + KL(q(z|x) || p(z))

Every time you train a VAE, you’re doing amortized variational inference. The encoder learns to perform inference in a single forward pass — no iterative optimization at test time.

Stochastic VI for Scalability

Traditional CAVI requires a full pass over the data per update. Stochastic VI uses minibatches: subsample data, compute a noisy ELBO gradient, take a step. This scales to millions of data points and is the backbone of modern VI. Combine with the reparameterization trick (sample z = μ + σ × ε, where ε ~ N(0,1)) to get low-variance gradient estimates.

VI: Strengths and Weaknesses

Strengths: fast convergence (especially with SGD), deterministic optimization, scales to large datasets, GPU-friendly.

Weaknesses: biased (the true posterior might not be in your variational family). Mean-field tends to underestimate posterior variance. KL(q||p) is mode-seeking — if the posterior is multimodal, VI tends to collapse onto one mode and ignore the rest. You never know how far your approximation is from the truth.

MCMC: Sampling from the Posterior

The alternative philosophy: instead of approximating the posterior with a simple distribution, draw samples from it. With enough samples, you can approximate any expectation. The magic of MCMC is constructing a Markov chain whose stationary distribution is the posterior — run the chain long enough, and the samples will be distributed according to p(θ|D).

Metropolis-Hastings

The foundational MCMC algorithm (1953). At each step:

  1. Propose a new state θ’ from a proposal distribution q(θ’|θ)
  2. Compute the acceptance ratio: α = min(1, p(θ’|D) q(θ|θ’) / p(θ|D) q(θ’|θ))
  3. Accept the proposal with probability α; otherwise stay at θ

The beauty: you only need the posterior up to a constant. The ratio p(θ’|D)/p(θ|D) cancels the evidence. You just need the unnormalized posterior, which is likelihood × prior.

import numpy as np

def metropolis_hastings(log_posterior, initial, n_samples=10000, proposal_std=0.5):
    """Simple MH sampler for 1D posterior."""
    samples = np.zeros(n_samples)
    current = initial
    current_lp = log_posterior(current)
    accepted = 0

    for i in range(n_samples):
        proposed = current + np.random.normal(0, proposal_std)
        proposed_lp = log_posterior(proposed)

        # Log acceptance ratio (symmetric proposal, so q cancels)
        log_alpha = proposed_lp - current_lp

        if np.log(np.random.uniform()) < log_alpha:
            current = proposed
            current_lp = proposed_lp
            accepted += 1

        samples[i] = current

    print(f"Acceptance rate: {accepted/n_samples:.1%}")
    return samples

# Example: sample from a mixture of Gaussians (multimodal!)
def log_posterior(theta):
    """Log of 0.3 * N(-2, 0.5) + 0.7 * N(3, 1)"""
    from scipy.stats import norm
    return np.log(0.3 * norm.pdf(theta, -2, 0.5) +
                  0.7 * norm.pdf(theta, 3, 1) + 1e-300)

samples = metropolis_hastings(log_posterior, initial=0.0, n_samples=50000)
# With enough samples, MH explores both modes

The problem: MH with a random walk proposal is slow in high dimensions. The chain takes tiny steps and needs many samples to explore the space. For 100+ parameters, basic MH is impractical.

Gibbs Sampling

Instead of proposing all parameters at once, Gibbs samples each variable conditioned on all others. If these full conditional distributions have nice forms (they often do in conjugate models), every proposal is accepted. No tuning required.

Gibbs was the workhorse of Bayesian ML for decades (BUGS/JAGS software). It’s still useful for models with lots of conjugacy (e.g., LDA topic models). But it struggles when variables are strongly correlated — the chain gets stuck because each variable can only move along one axis at a time.

Hamiltonian Monte Carlo & NUTS

HMC is the game-changer. Instead of random-walking through parameter space, HMC simulates a physical system where θ is a particle’s position and the negative log-posterior is the “potential energy.” An auxiliary “momentum” variable is introduced, and the particle follows Hamilton’s equations:

dθ/dt = ∂H/∂p = p
dp/dt = -∂H/∂θ = -∇ log p(θ|D)   ← uses the gradient!

The particle rolls along the posterior surface, making large moves in high-probability regions. This is dramatically more efficient than random-walk MH in high dimensions.

NUTS (No-U-Turn Sampler) automates HMC’s two finicky hyperparameters (step size and number of leapfrog steps). It runs the simulation until the trajectory starts to double back, then stops. NUTS is the default sampler in Stan and PyMC — and the reason modern Bayesian inference actually works on real problems.

Expert note: HMC requires the gradient of the log-posterior, which means your model must be differentiable. This is why probabilistic programming languages built on automatic differentiation (PyMC/Aesara, NumPyro/JAX, Stan/AD) are so powerful — they compute gradients for free.

MCMC Diagnostics: Has the Chain Converged?

MCMC doesn’t tell you when it’s done. You have to check. Key diagnostics:

  • Trace plots: Plot parameter values vs. iteration. A well-mixed chain looks like a “hairy caterpillar” — no trends, no getting stuck.
  • R̂ (R-hat): Run multiple chains from different starting points. R̂ measures whether they’ve converged to the same distribution. R̂ < 1.01 is the modern standard (used to be 1.1).
  • Effective Sample Size (ESS): How many independent samples your chain is equivalent to, accounting for autocorrelation. ESS/chain should be at least 100 for reliable estimates, 400+ for tail probabilities.
  • Divergences (HMC/NUTS): Numerical integration errors. Even a single divergence is a warning. If you see divergences, increase target_accept or reparameterize your model.

VI vs. MCMC: The Decision

CriterionVariational InferenceMCMC
SpeedFast (optimization)Slow (sampling)
ScalabilityExcellent (minibatch)Limited (full data per step)
AccuracyBiased approximationAsymptotically exact
Multimodal posteriorsPoor (mode-seeking)Can explore (but slow)
Uncertainty qualityOften underestimatesFaithful (if converged)
Convergence checkELBO plateauR̂, ESS, trace plots
Gradient required?Yes (for SVI)Yes for HMC, no for MH/Gibbs
When to useLarge data, speed matters, exploration-phaseSmall/medium data, accuracy matters, publication
Practical advice: Start with VI to prototype quickly. If the uncertainty estimates matter for your application (medical, safety-critical), validate with MCMC. For production A/B testing or recommendation systems, VI is usually fine. For a scientific paper, reviewers will expect MCMC convergence diagnostics.

Code: Both Approaches on the Same Problem

import pymc as pm
import numpy as np
import arviz as az

# Generate data: linear regression with known parameters
np.random.seed(42)
true_w, true_b, true_sigma = 2.5, -1.0, 0.8
X = np.random.randn(100)
y = true_w * X + true_b + np.random.normal(0, true_sigma, 100)

# --- MCMC (NUTS) ---
with pm.Model() as mcmc_model:
    w = pm.Normal('w', mu=0, sigma=10)
    b = pm.Normal('b', mu=0, sigma=10)
    sigma = pm.HalfNormal('sigma', sigma=5)
    y_obs = pm.Normal('y_obs', mu=w * X + b, sigma=sigma, observed=y)

    trace_mcmc = pm.sample(2000, tune=1000, cores=2, random_seed=42)

print("MCMC results:")
print(az.summary(trace_mcmc, var_names=['w', 'b', 'sigma']))

# --- Variational Inference (ADVI) ---
with pm.Model() as vi_model:
    w = pm.Normal('w', mu=0, sigma=10)
    b = pm.Normal('b', mu=0, sigma=10)
    sigma = pm.HalfNormal('sigma', sigma=5)
    y_obs = pm.Normal('y_obs', mu=w * X + b, sigma=sigma, observed=y)

    approx = pm.fit(method='advi', n=30000, random_seed=42)
    trace_vi = approx.sample(2000)

print("\nVI results:")
print(az.summary(trace_vi, var_names=['w', 'b', 'sigma']))
# Compare: means should be similar, but VI uncertainty is often narrower
