Hidden Markov Models (HMMs)

Chapter 14: Probabilistic & Bayesian ML 3 subtopics
TL;DR

Many real-world systems have hidden states that you cannot observe directly — you only see noisy measurements. Speech recognition sees audio waveforms but needs to recover words. Gene finders see DNA bases but need to identify coding regions. Financial analysts see price movements but want to know the underlying market regime. Hidden Markov Models give you a principled way to reason about these latent sequences. You define transition probabilities between hidden states and emission probabilities that generate observations, then use three classic algorithms — Forward (likelihood), Viterbi (best state sequence), and Baum-Welch (parameter learning) — to solve the inference problem. This section builds all three from scratch and shows you how to use them with hmmlearn and NumPyro.

Model Components

A Hidden Markov Model is a probabilistic model for sequential data where the system moves through a sequence of discrete hidden states over time, and at each time step it emits an observation that depends only on the current hidden state. The "hidden" part means you never see the states directly — you only see the emissions.

Hidden States and Observations

Think about weather prediction from the 1800s. You are stuck inside a cave and cannot see the sky. Every day, your friend brings you a report of how many people carried umbrellas. You want to figure out whether it was sunny, cloudy, or rainy each day based only on umbrella counts. The weather state is hidden. The umbrella count is the observation.

More formally, at each time step t, the system is in one of K discrete hidden states s_t ∈ {1, 2, …, K}. It produces an observation o_t drawn from a distribution that depends on s_t. The states evolve according to a first-order Markov chain: the probability of state s_t depends only on s_{t-1}, not on any earlier history.

Transition Probabilities (A Matrix)

The transition matrix A is a K×K matrix where A[i,j] = P(s_t = j | s_{t-1} = i). Each row sums to 1. Entry A[i,j] tells you: "if the system is in state i right now, what is the probability it moves to state j next?" This matrix encodes the dynamics of the hidden process.

Emission Probabilities (B Matrix)

The emission matrix B defines what you observe in each state. For discrete observations with M possible values, B is a K×M matrix where B[k,m] = P(o_t = m | s_t = k). For continuous observations, B[k] would be a probability density function — often a Gaussian with state-specific mean and variance.

Initial State Distribution (π)

The initial distribution π is a vector of length K where π[k] = P(s_1 = k). It tells you the probability of starting in each state.

import numpy as np

# A simple weather HMM
# Hidden states: 0=Sunny, 1=Rainy
# Observations: 0=Walk, 1=Shop, 2=Clean

# Transition matrix A: P(next_state | current_state)
A = np.array([
    [0.7, 0.3],   # Sunny → {Sunny: 0.7, Rainy: 0.3}
    [0.4, 0.6]    # Rainy → {Sunny: 0.4, Rainy: 0.6}
])

# Emission matrix B: P(observation | state)
B = np.array([
    [0.6, 0.3, 0.1],  # Sunny → {Walk: 0.6, Shop: 0.3, Clean: 0.1}
    [0.1, 0.4, 0.5]   # Rainy → {Walk: 0.1, Shop: 0.4, Clean: 0.5}
])

# Initial distribution
pi = np.array([0.6, 0.4])  # 60% chance of starting sunny

print("Model parameters:")
print(f"States: Sunny, Rainy")
print(f"Observations: Walk, Shop, Clean")
print(f"P(Sunny→Rainy) = {A[0,1]}")
print(f"P(Walk|Sunny)  = {B[0,0]}")

Three Fundamental Problems

In 1966, Leonard Baum and colleagues identified three core problems for HMMs. Every practical application boils down to one of these.

Problem 1: Evaluation — The Forward Algorithm

Given a model (A, B, π) and an observation sequence O = (o₁, o₂, …, o_T), compute P(O | model). Why do you need this? Model comparison. If you have two candidate HMMs — one for English and one for French — feed in a sentence and pick the model that gives it higher probability.

The naive approach sums over all possible state sequences: P(O) = Σ_all_sequences P(O, S). With K states and T time steps, there are K^T sequences. For K=10 and T=100, that is 10^100 terms — absurd.

The forward algorithm uses dynamic programming to compute this in O(K²T) time. Define the forward variable α_t(j) = P(o₁, o₂, …, o_t, s_t = j). You compute it recursively: α_1(j) = π[j] · B[j, o₁], and α_t(j) = [Σᵢ α_{t-1}(i) · A[i,j]] · B[j, o_t]. The total likelihood is Σⱼ α_T(j).

import numpy as np

def forward_algorithm(A, B, pi, observations):
    """Compute P(observations | model) using the forward algorithm."""
    K = len(pi)           # number of states
    T = len(observations) # sequence length

    # Alpha table: alpha[t, k] = P(o_1,...,o_t, s_t=k)
    alpha = np.zeros((T, K))

    # Initialization
    alpha[0] = pi * B[:, observations[0]]

    # Recursion
    for t in range(1, T):
        for j in range(K):
            alpha[t, j] = np.sum(alpha[t-1] * A[:, j]) * B[j, observations[t]]

    # Total probability: sum over final states
    log_likelihood = np.log(np.sum(alpha[-1]))
    return alpha, log_likelihood

# Observation sequence: Walk, Shop, Clean, Walk
obs = np.array([0, 1, 2, 0])
alpha, ll = forward_algorithm(A, B, pi, obs)
print(f"Log-likelihood: {ll:.4f}")
print(f"P(observation sequence): {np.exp(ll):.6f}")
print(f"\nForward variables at each step:")
for t in range(len(obs)):
    print(f"  t={t}: α(Sunny)={alpha[t,0]:.4f}, α(Rainy)={alpha[t,1]:.4f}")

The backward algorithm defines β_t(i) = P(o_{t+1}, …, o_T | s_t = i) and computes it in reverse. Together, the forward and backward variables give you the posterior probability of being in state k at time t: γ_t(k) = α_t(k)·β_t(k) / P(O). This forward-backward algorithm is the E-step of Baum-Welch.

def forward_backward(A, B, pi, observations):
    """Full forward-backward algorithm returning state posteriors."""
    K = len(pi)
    T = len(observations)

    # Forward pass
    alpha = np.zeros((T, K))
    alpha[0] = pi * B[:, observations[0]]
    for t in range(1, T):
        for j in range(K):
            alpha[t, j] = np.sum(alpha[t-1] * A[:, j]) * B[j, observations[t]]

    # Backward pass
    beta = np.zeros((T, K))
    beta[-1] = 1.0  # initialization
    for t in range(T - 2, -1, -1):
        for i in range(K):
            beta[t, i] = np.sum(A[i, :] * B[:, observations[t+1]] * beta[t+1])

    # State posteriors: gamma[t,k] = P(s_t=k | observations)
    gamma = alpha * beta
    gamma /= gamma.sum(axis=1, keepdims=True)

    return gamma, alpha, beta

gamma, _, _ = forward_backward(A, B, pi, obs)
state_names = ["Sunny", "Rainy"]
print("Most likely states (posterior decoding):")
for t in range(len(obs)):
    best = np.argmax(gamma[t])
    print(f"  t={t}: {state_names[best]} (P={gamma[t, best]:.3f})")

Problem 2: Decoding — The Viterbi Algorithm

Given observations, find the single most likely sequence of hidden states: argmax_S P(S | O). This is different from picking the most likely state at each time step independently — the best sequence might not consist of individually best states because transitions matter.

The Viterbi algorithm uses dynamic programming. Define δ_t(j) = max_{s₁,…,s_{t-1}} P(s₁, …, s_{t-1}, s_t = j, o₁, …, o_t). It is the probability of the best path ending in state j at time t. You compute it like the forward algorithm but replace the sum with a max, and keep backpointers to reconstruct the path.

def viterbi(A, B, pi, observations):
    """Find most likely state sequence using the Viterbi algorithm."""
    K = len(pi)
    T = len(observations)

    # Delta table and backpointers
    delta = np.zeros((T, K))
    psi = np.zeros((T, K), dtype=int)

    # Initialization
    delta[0] = pi * B[:, observations[0]]
    psi[0] = 0

    # Recursion
    for t in range(1, T):
        for j in range(K):
            probs = delta[t-1] * A[:, j]
            psi[t, j] = np.argmax(probs)
            delta[t, j] = np.max(probs) * B[j, observations[t]]

    # Backtrack
    path = np.zeros(T, dtype=int)
    path[-1] = np.argmax(delta[-1])
    for t in range(T - 2, -1, -1):
        path[t] = psi[t + 1, path[t + 1]]

    log_prob = np.log(np.max(delta[-1]))
    return path, log_prob

path, log_prob = viterbi(A, B, pi, obs)
obs_names = ["Walk", "Shop", "Clean"]
print(f"Observations: {[obs_names[o] for o in obs]}")
print(f"Viterbi path: {[state_names[s] for s in path]}")
print(f"Log probability of best path: {log_prob:.4f}")
ℹ️ Posterior Decoding vs. Viterbi

Posterior decoding (picking argmax γ_t(k) at each t independently) minimizes the expected number of incorrectly labeled states. Viterbi finds the single most probable sequence. They can give different answers. Viterbi guarantees a valid sequence (respecting transition constraints), while posterior decoding might produce a state sequence that has zero transition probability. Use Viterbi when sequence validity matters.

Problem 3: Learning — The Baum-Welch Algorithm

Given observation sequences but no labels, learn the model parameters A, B, π. This is the Baum-Welch algorithm, which is EM (Expectation-Maximization) applied to HMMs.

The E-step runs forward-backward to compute γ_t(k) and ξ_t(i,j) = P(s_t=i, s_{t+1}=j | O), the probability of transitioning from state i to state j at time t. The M-step re-estimates the parameters: A[i,j] is the expected number of transitions from i to j divided by the expected number of times in state i; B[k,m] is the expected number of times in state k emitting symbol m divided by the expected number of times in state k.

def baum_welch(observations, n_states, n_obs_symbols, n_iter=100, tol=1e-6):
    """Learn HMM parameters from observations using Baum-Welch (EM)."""
    K, M, T = n_states, n_obs_symbols, len(observations)

    # Random initialization
    np.random.seed(42)
    A = np.random.dirichlet(np.ones(K), size=K)
    B = np.random.dirichlet(np.ones(M), size=K)
    pi = np.random.dirichlet(np.ones(K))

    prev_ll = -np.inf
    for iteration in range(n_iter):
        # E-step: forward-backward
        alpha = np.zeros((T, K))
        alpha[0] = pi * B[:, observations[0]]
        for t in range(1, T):
            alpha[t] = (alpha[t-1] @ A) * B[:, observations[t]]

        beta = np.zeros((T, K))
        beta[-1] = 1.0
        for t in range(T-2, -1, -1):
            beta[t] = A @ (B[:, observations[t+1]] * beta[t+1])

        # Compute gamma and xi
        gamma = alpha * beta
        gamma /= gamma.sum(axis=1, keepdims=True) + 1e-300

        xi = np.zeros((T-1, K, K))
        for t in range(T-1):
            numer = (alpha[t, :, None] * A *
                     B[None, :, observations[t+1]] * beta[t+1, None, :])
            xi[t] = numer / (numer.sum() + 1e-300)

        # M-step: re-estimate parameters
        pi = gamma[0]
        A = xi.sum(axis=0) / (gamma[:-1].sum(axis=0)[:, None] + 1e-300)
        for m in range(M):
            mask = (observations == m)
            B[:, m] = gamma[mask].sum(axis=0)
        B /= (gamma.sum(axis=0)[:, None] + 1e-300)

        # Check convergence
        ll = np.log(alpha[-1].sum() + 1e-300)
        if abs(ll - prev_ll) < tol:
            print(f"Converged at iteration {iteration}")
            break
        prev_ll = ll

    return A, B, pi

# Generate synthetic data from the true model
np.random.seed(42)
T = 1000
states = np.zeros(T, dtype=int)
observations_gen = np.zeros(T, dtype=int)
states[0] = np.random.choice(2, p=pi)
observations_gen[0] = np.random.choice(3, p=B[states[0]])
for t in range(1, T):
    states[t] = np.random.choice(2, p=A[states[t-1]])
    observations_gen[t] = np.random.choice(3, p=B[states[t]])

# Learn parameters from observations only
A_learned, B_learned, pi_learned = baum_welch(observations_gen, n_states=2, n_obs_symbols=3)
print(f"\nTrue A:\n{A}")
print(f"Learned A:\n{np.round(A_learned, 3)}")
print(f"\nTrue B:\n{B}")
print(f"Learned B:\n{np.round(B_learned, 3)}")

Applications

Using hmmlearn for Real-World Tasks

In practice, you rarely implement HMMs from scratch. The hmmlearn library provides efficient implementations with Gaussian emissions, GMM emissions, and categorical emissions. Here is a realistic example — detecting market regimes in financial data.

from hmmlearn import hmm
import numpy as np

# Simulate daily stock returns with two regimes:
# Regime 0 (Bull): mean=0.05%, std=1%
# Regime 1 (Bear): mean=-0.03%, std=2.5%
np.random.seed(42)
T = 500
true_states = np.zeros(T, dtype=int)
for t in range(1, T):
    if true_states[t-1] == 0:
        true_states[t] = np.random.choice([0, 1], p=[0.95, 0.05])
    else:
        true_states[t] = np.random.choice([0, 1], p=[0.10, 0.90])

returns = np.where(true_states == 0,
                   np.random.normal(0.0005, 0.01, T),
                   np.random.normal(-0.0003, 0.025, T))

# Fit a 2-state Gaussian HMM
model = hmm.GaussianHMM(n_components=2, covariance_type="full",
                         n_iter=200, random_state=42)
model.fit(returns.reshape(-1, 1))

# Decode the most likely state sequence
predicted_states = model.predict(returns.reshape(-1, 1))

# Accuracy (accounting for label permutation)
acc = max(np.mean(predicted_states == true_states),
          np.mean(predicted_states != true_states))
print(f"State recovery accuracy: {acc:.1%}")
print(f"Learned means: {model.means_.flatten()}")
print(f"Learned transition matrix:\n{np.round(model.transmat_, 3)}")

Part-of-Speech Tagging

In natural language processing, HMMs were the standard approach for part-of-speech (POS) tagging before neural methods took over. Hidden states are POS tags (noun, verb, adjective, …), and observations are words. The transition matrix captures syntactic patterns (a determiner is usually followed by a noun or adjective). The emission matrix captures which words are likely for each tag. Even today, HMMs serve as a baseline and a teaching tool for understanding sequence labeling.

Gene Finding and Speech Recognition

In computational biology, HMMs model the structure of genes. Hidden states represent coding regions, introns, intergenic regions, splice sites. Observations are DNA nucleotides (A, C, G, T). The Viterbi algorithm identifies the most likely annotation of a genome sequence.

In speech recognition, HMMs model the temporal structure of phonemes. Each phoneme is an HMM with states representing the beginning, middle, and end of the sound. Words are built by concatenating phoneme HMMs. Modern speech systems have moved to end-to-end neural networks, but the HMM framework shaped decades of research and many hybrid systems still use HMM-based alignment.

💡 Scaling Tip

In practice, forward/backward computations use the log-sum-exp trick to avoid numerical underflow. When observation sequences are long (hundreds or thousands of steps), raw probabilities become astronomically small. Working in log space and using scipy.special.logsumexp keeps everything numerically stable.

# Bayesian HMM with NumPyro — learning with full uncertainty
import numpyro
import numpyro.distributions as dist
from numpyro.infer import MCMC, NUTS
import jax.numpy as jnp
import jax.random as random

def bayesian_hmm(obs, n_states=2, n_obs=3):
    T = len(obs)
    # Priors on transition and emission matrices (Dirichlet)
    trans_probs = numpyro.sample(
        "trans", dist.Dirichlet(jnp.ones(n_states)).expand([n_states])
    )
    emit_probs = numpyro.sample(
        "emit", dist.Dirichlet(jnp.ones(n_obs)).expand([n_states])
    )
    init_probs = numpyro.sample(
        "init", dist.Dirichlet(jnp.ones(n_states))
    )

    # Use DiscreteHMM distribution for marginalized inference
    transition_dist = dist.Categorical(probs=trans_probs)
    emission_dist = dist.Categorical(probs=emit_probs)
    initial_dist = dist.Categorical(probs=init_probs)

    numpyro.sample(
        "obs",
        dist.DiscreteHMM(initial_dist, transition_dist, emission_dist),
        obs=obs
    )
