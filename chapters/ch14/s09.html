Probabilistic Programming

Chapter 14: Probabilistic & Bayesian ML 3 subtopics
TL;DR

Probabilistic programming flips the ML workflow. Instead of choosing an algorithm first, you write down a generative story for how your data came to be — priors, likelihood, the whole causal process — and the framework figures out how to do inference automatically. You specify the model; the computer solves the math. This section walks through the major frameworks (PyMC, Stan, NumPyro, Pyro, TensorFlow Probability), the complete Bayesian workflow from prior predictive checks to model comparison, and practical patterns for building real models. Every concept comes with runnable code.

Concepts

Generative Model Specification

A generative model is a program that, if you ran it, would produce synthetic data that looks like your real data. You describe the full data-generating process: sample parameters from priors, sample latent variables, then sample observations conditional on everything above. The key insight of probabilistic programming is that this generative program — written in a host language like Python — doubles as a mathematical model specification. The framework reads your code, builds a computation graph, and runs an inference algorithm to compute the posterior over all the random variables you declared.

import pymc as pm
import numpy as np

# The generative story:
# 1. There's some unknown conversion rate θ
# 2. θ was drawn from a Beta(2, 8) prior (we expect ~20% conversion)
# 3. Each visitor independently converts with probability θ
# 4. We observed 47 conversions in 300 visitors

with pm.Model() as ab_test:
    # Step 1-2: prior on conversion rate
    theta = pm.Beta("theta", alpha=2, beta=8)

    # Step 3-4: likelihood of observed data
    obs = pm.Binomial("obs", n=300, p=theta, observed=47)

    # The framework handles inference automatically
    trace = pm.sample(2000, chains=4, random_seed=42)

print(f"Posterior mean: {trace.posterior['theta'].mean().values:.3f}")
print(f"95% credible interval: "
      f"[{np.quantile(trace.posterior['theta'].values, 0.025):.3f}, "
      f"{np.quantile(trace.posterior['theta'].values, 0.975):.3f}]")

Automatic Inference

This is the magic. Once you define the model, the framework chooses and runs an inference algorithm for you. For continuous parameters, it typically uses NUTS (a variant of Hamiltonian Monte Carlo). For discrete parameters, it may use Gibbs sampling or marginalize them out analytically. For very large models, it can switch to variational inference. You, the modeler, focus on getting the model right. The framework handles the computational machinery.

Model Criticism and Checking

Bayesian modeling is not "set and forget." You need to check whether your model is a reasonable description of reality. This means looking at prior predictive distributions (does the model generate plausible data before seeing any observations?), convergence diagnostics (did the sampler explore the posterior properly?), and posterior predictive distributions (can the fitted model reproduce the patterns in your data?). Probabilistic programming frameworks make all of these checks easy because they produce samples at every step.

Frameworks

PyMC (Python)

PyMC is the most popular Python-native probabilistic programming library. It uses PyTensor (formerly Theano) as its computation backend. Models are defined in a with pm.Model() context manager. You declare random variables using distribution classes (pm.Normal, pm.Beta, etc.), and PyMC builds the log-probability graph automatically. Sampling is typically NUTS for continuous parameters, with automatic tuning during warm-up.

import pymc as pm
import arviz as az
import numpy as np

# Bayesian linear regression with PyMC
np.random.seed(42)
N = 200
X = np.random.randn(N, 3)
true_w = np.array([2.0, -1.0, 0.5])
y = X @ true_w + np.random.normal(0, 0.8, N)

with pm.Model() as linear_model:
    # Priors
    w = pm.Normal("w", mu=0, sigma=5, shape=3)
    sigma = pm.HalfNormal("sigma", sigma=5)

    # Likelihood
    mu = pm.math.dot(X, w)
    y_obs = pm.Normal("y_obs", mu=mu, sigma=sigma, observed=y)

    # Inference
    trace = pm.sample(2000, chains=4, random_seed=42)

# Results
summary = az.summary(trace, var_names=["w", "sigma"])
print(summary[["mean", "sd", "hdi_3%", "hdi_97%", "r_hat"]])

Stan (C++/R/Python)

Stan is a standalone probabilistic programming language that compiles models to C++ for speed. It pioneered the use of NUTS for automatic inference and is considered the gold standard for reliability. You write models in Stan's own language, compile them, then call the compiled model from R (rstan), Python (PyStan or CmdStanPy), or the command line. Stan's diagnostics are thorough — it warns you about divergences, maximum treedepth issues, and low effective sample sizes.

# CmdStanPy example: Bayesian logistic regression
import cmdstanpy
import numpy as np

# Write Stan model to file
stan_code = """
data {
  int N;
  int K;
  matrix[N, K] X;
  array[N] int y;
}
parameters {
  vector[K] beta;
  real alpha;
}
model {
  // Priors
  beta ~ normal(0, 5);
  alpha ~ normal(0, 5);

  // Likelihood
  y ~ bernoulli_logit(X * beta + alpha);
}
generated quantities {
  // Posterior predictive
  array[N] int y_rep;
  for (n in 1:N)
    y_rep[n] = bernoulli_logit_rng(X[n] * beta + alpha);
}
"""

NumPyro (JAX)

NumPyro is the JAX-based probabilistic programming library from Uber AI. It provides the same modeling API as Pyro but runs on JAX, which means automatic differentiation, JIT compilation, and GPU/TPU acceleration. NUTS inference in NumPyro is extremely fast — often 10-100x faster than PyMC for the same model — because JAX compiles the entire sampling loop to optimized machine code.

import numpyro
import numpyro.distributions as dist
from numpyro.infer import MCMC, NUTS, Predictive
import jax.numpy as jnp
import jax.random as random

# Bayesian hierarchical model with NumPyro
def hierarchical_model(group_ids, X, y=None):
    n_groups = len(jnp.unique(group_ids))
    n_features = X.shape[1]

    # Hyperpriors: population-level parameters
    mu_beta = numpyro.sample("mu_beta", dist.Normal(0, 5).expand([n_features]))
    sigma_beta = numpyro.sample("sigma_beta",
                                 dist.HalfNormal(2).expand([n_features]))

    # Group-level parameters (non-centered parameterization)
    with numpyro.plate("groups", n_groups):
        z = numpyro.sample("z", dist.Normal(0, 1).expand([n_features]))
    beta = mu_beta + sigma_beta * z  # (n_groups, n_features)

    # Observation noise
    sigma = numpyro.sample("sigma", dist.HalfNormal(5))

    # Likelihood
    mu = jnp.sum(X * beta[group_ids], axis=1)
    numpyro.sample("y", dist.Normal(mu, sigma), obs=y)

# Generate synthetic hierarchical data
np.random.seed(42)
n_groups, n_per_group = 5, 100
N = n_groups * n_per_group
group_ids = np.repeat(np.arange(n_groups), n_per_group)
X = np.random.randn(N, 2)
true_betas = np.random.normal([3, -1], [0.5, 0.3], size=(n_groups, 2))
y = np.sum(X * true_betas[group_ids], axis=1) + np.random.normal(0, 0.5, N)

# Run inference
kernel = NUTS(hierarchical_model)
mcmc = MCMC(kernel, num_warmup=1000, num_samples=2000, num_chains=4)
mcmc.run(random.PRNGKey(0),
         group_ids=jnp.array(group_ids),
         X=jnp.array(X), y=jnp.array(y))
mcmc.print_summary(exclude_deterministic=True)

Pyro (PyTorch) and TensorFlow Probability

Pyro is built on PyTorch and excels at deep probabilistic models — variational autoencoders, deep kernel learning, amortized inference. It is the right choice when your model has neural network components.

TensorFlow Probability (TFP) provides probability distributions, bijectors, and inference tools within the TensorFlow ecosystem. It is particularly strong for production deployment and integration with TensorFlow Serving.

# Pyro: Bayesian neural network
import torch
import pyro
import pyro.distributions as pdist
from pyro.infer import SVI, Trace_ELBO
from pyro.infer.autoguide import AutoDiagonalNormal

def bnn_model(X, y=None):
    n_features = X.shape[1]
    n_hidden = 10

    # Priors on weights
    w1 = pyro.sample("w1", pdist.Normal(
        torch.zeros(n_features, n_hidden),
        torch.ones(n_features, n_hidden)).to_event(2))
    b1 = pyro.sample("b1", pdist.Normal(
        torch.zeros(n_hidden), torch.ones(n_hidden)).to_event(1))
    w2 = pyro.sample("w2", pdist.Normal(
        torch.zeros(n_hidden, 1), torch.ones(n_hidden, 1)).to_event(2))
    b2 = pyro.sample("b2", pdist.Normal(
        torch.zeros(1), torch.ones(1)).to_event(1))

    # Forward pass
    h = torch.tanh(X @ w1 + b1)
    mu = (h @ w2 + b2).squeeze(-1)
    sigma = pyro.sample("sigma", pdist.HalfNormal(1.0))

    with pyro.plate("data", X.shape[0]):
        pyro.sample("y", pdist.Normal(mu, sigma), obs=y)

# Variational inference with auto-guide
guide = AutoDiagonalNormal(bnn_model)
optimizer = pyro.optim.Adam({"lr": 0.01})
svi = SVI(bnn_model, guide, optimizer, loss=Trace_ELBO())

ℹ️ Choosing a Framework

Use PyMC for classic Bayesian models with a gentle learning curve. Use Stan when you need bulletproof diagnostics and maximum MCMC reliability. Use NumPyro when speed matters — it is the fastest option for NUTS inference. Use Pyro when your model includes neural network components. Use TFP when you are already in the TensorFlow ecosystem and need production serving.

Workflow

Probabilistic programming is most powerful when embedded in a disciplined Bayesian workflow. The workflow has six steps, and skipping any of them is how you end up with models that look good in a notebook but fail in production.

Step 1: Model Specification

Write down the generative model. Be explicit about every assumption: what distributions you chose, what independence assumptions you are making, what you think the data-generating process looks like. This is the hypothesis you are testing.

Step 2: Prior Predictive Checks

Before seeing any data, sample from the prior predictive distribution: generate synthetic datasets from your model using only the priors. Do the simulated datasets look remotely plausible? If your prior predictive puts most of the probability on outcomes that are physically impossible (negative heights, conversion rates above 100%), your priors are wrong and you should fix them before running inference.

import pymc as pm
import numpy as np

# Prior predictive check for a simple linear model
np.random.seed(42)
N = 50
X = np.linspace(-2, 2, N)

with pm.Model() as model:
    intercept = pm.Normal("intercept", mu=0, sigma=10)
    slope = pm.Normal("slope", mu=0, sigma=10)
    sigma = pm.HalfNormal("sigma", sigma=5)
    mu = intercept + slope * X
    y = pm.Normal("y", mu=mu, sigma=sigma, shape=N)

    # Sample from priors only — no data involved
    prior_pred = pm.sample_prior_predictive(samples=200, random_seed=42)

# Check: do the prior predictions cover a reasonable range?
y_prior = prior_pred.prior_predictive["y"].values[0]  # (200, 50)
print(f"Prior predictive y range: [{y_prior.min():.1f}, {y_prior.max():.1f}]")
print(f"Prior predictive y std: {y_prior.std():.1f}")
# If this is [-500, 500], your priors are too wide for the problem

Step 3: Inference (MCMC or VI)

Run the sampler. For small to medium models (up to a few thousand parameters), use MCMC (NUTS). For large models or when speed matters, use variational inference. NumPyro can handle thousands of parameters with NUTS in seconds thanks to JAX compilation.

Step 4: Convergence Diagnostics

After sampling, check that the sampler worked. The key diagnostics are:

import pymc as pm
import arviz as az
import numpy as np

# After fitting a model...
np.random.seed(42)
y_data = np.random.normal(5, 2, 100)

with pm.Model() as diag_model:
    mu = pm.Normal("mu", mu=0, sigma=10)
    sigma = pm.HalfNormal("sigma", sigma=10)
    y = pm.Normal("y", mu=mu, sigma=sigma, observed=y_data)
    trace = pm.sample(2000, chains=4, random_seed=42)

# Diagnostic summary
diag = az.summary(trace)
print(diag[["mean", "sd", "r_hat", "ess_bulk", "ess_tail"]])

# Rules of thumb:
# r_hat < 1.01: chains have converged (good)
# ess_bulk > 400: enough effective samples for mean estimates
# ess_tail > 400: enough effective samples for tail quantiles
# No divergent transitions reported by the sampler

# Check divergences (PyMC stores them)
n_divs = trace.sample_stats["diverging"].sum().values
print(f"Number of divergences: {n_divs}")

# Visual check
az.plot_trace(trace)       # trace plots for mixing
az.plot_rank(trace)        # rank plots (uniform = well mixed)

Step 5: Posterior Predictive Checks

Generate synthetic data from the posterior and compare it to the real data. This is the "does the model fit?" test. If the model captures the key patterns — the mean, the spread, the shape of the distribution, any clustering or heterogeneity — the posterior predictive data should look similar to the real data.

with diag_model:
    post_pred = pm.sample_posterior_predictive(trace, random_seed=42)

y_rep = post_pred.posterior_predictive["y"].values
# y_rep shape: (chains, samples, n_obs)

# Compare summary statistics
print(f"Observed mean:  {y_data.mean():.3f}")
print(f"Posterior pred mean: {y_rep.mean():.3f}")
print(f"Observed std:   {y_data.std():.3f}")
print(f"Posterior pred std:  {y_rep.std():.3f}")

# Visual posterior predictive check
az.plot_ppc(az.from_pymc3(posterior_predictive=post_pred, model=diag_model))

Step 6: Model Comparison

When you have multiple candidate models, compare them using information criteria. WAIC (Widely Applicable Information Criterion) and LOO-CV (Leave-One-Out Cross-Validation via Pareto-Smoothed Importance Sampling) are the Bayesian analogues of AIC/BIC. ArviZ computes both directly from the posterior trace.

import pymc as pm
import arviz as az
import numpy as np

np.random.seed(42)
X = np.random.randn(100)
y_data2 = 2 * X + 0.5 * X**2 + np.random.normal(0, 0.5, 100)

# Model 1: Linear
with pm.Model() as model_linear:
    a = pm.Normal("a", 0, 5)
    b = pm.Normal("b", 0, 5)
    sigma = pm.HalfNormal("sigma", 5)
    mu = a + b * X
    y = pm.Normal("y", mu=mu, sigma=sigma, observed=y_data2)
    trace_linear = pm.sample(2000, random_seed=42)

# Model 2: Quadratic
with pm.Model() as model_quad:
    a = pm.Normal("a", 0, 5)
    b = pm.Normal("b", 0, 5)
    c = pm.Normal("c", 0, 5)
    sigma = pm.HalfNormal("sigma", 5)
    mu = a + b * X + c * X**2
    y = pm.Normal("y", mu=mu, sigma=sigma, observed=y_data2)
    trace_quad = pm.sample(2000, random_seed=42)

# Compare using LOO
idata_linear = az.from_pymc3(trace=trace_linear, model=model_linear)
idata_quad = az.from_pymc3(trace=trace_quad, model=model_quad)
comparison = az.compare({"linear": idata_linear, "quadratic": idata_quad})
print(comparison[["rank", "elpd_loo", "p_loo", "weight"]])
💡 The Non-Centered Parameterization

Hierarchical models with tight group-level variances create a "funnel" geometry that NUTS struggles with. The fix is the non-centered parameterization: instead of sampling group parameters directly from Normal(μ, σ), sample a standard normal z and compute the parameter as μ + σ·z. This decouples the group parameters from the variance, smoothing the posterior geometry. Always use this trick for hierarchical models.
