Gaussian Processes

Chapter 14: Probabilistic & Bayesian ML 5 subtopics
TL;DR

A Gaussian Process is a distribution over functions. Instead of fitting a single function to data (like linear regression does), a GP represents your uncertainty over the entire space of possible functions. Before seeing data, the GP encodes your prior beliefs about smoothness, periodicity, and scale through a kernel function. After seeing data, it gives you a posterior distribution over functions — a mean prediction at every point, plus uncertainty bands that widen where data is sparse and narrow where data is dense. GPs are the gold standard for regression with calibrated uncertainty, and they power Bayesian optimization — the method behind hyperparameter tuning in AutoML. This section builds GP regression from the math up, covers the major kernel functions, and shows practical code with scikit-learn and GPyTorch.

Definition and Properties

Distribution Over Functions

A Gaussian Process is a collection of random variables, any finite subset of which has a joint Gaussian distribution. That sounds abstract, so here is the concrete version: imagine you have a function f(x). You do not know what f looks like, but you believe that for any set of input points x₁, x₂, …, xₙ, the corresponding function values [f(x₁), f(x₂), …, f(xₙ)] are jointly Gaussian. A GP is completely specified by two things: a mean function m(x) and a kernel (covariance) function k(x, x').

f(x) ~ GP(m(x), k(x, x'))

The mean function tells you the expected value of f at any point. The kernel function tells you how correlated the function values at two different points are. Points that are close together (according to the kernel) will have similar function values. Points that are far apart will be nearly independent.

Mean Function

In practice, the mean function m(x) is usually set to zero. This is not as restrictive as it sounds — the posterior mean will be pulled toward the data, so the prior mean mainly matters far from observations. If you have strong prior knowledge about the trend, you can set m(x) to a linear function or a polynomial.

Kernel (Covariance) Function

The kernel function k(x, x') is where the real modeling happens. It encodes your assumptions about the function: how smooth it is, how fast it varies, whether it has periodic patterns. The kernel determines the "personality" of the functions the GP generates. Two GPs with different kernels produce wildly different function families.

Marginal and Conditional Gaussians

The power of GPs comes from a property of multivariate Gaussians: conditioning and marginalization have closed-form solutions. If you partition a joint Gaussian into observed and unobserved variables, the conditional distribution of the unobserved given the observed is another Gaussian — and you can compute its mean and covariance exactly. This is what makes GP regression tractable.

import numpy as np
import matplotlib
matplotlib.use('Agg')

# Sample functions from a GP prior
def rbf_kernel(X1, X2, length_scale=1.0, variance=1.0):
    """Squared exponential (RBF) kernel."""
    sqdist = np.sum(X1**2, axis=1).reshape(-1, 1) + \
             np.sum(X2**2, axis=1) - 2 * X1 @ X2.T
    return variance * np.exp(-0.5 * sqdist / length_scale**2)

# 200 test points
X_test = np.linspace(-5, 5, 200).reshape(-1, 1)

# Compute covariance matrix from kernel
K = rbf_kernel(X_test, X_test, length_scale=1.0, variance=1.0)
K += 1e-8 * np.eye(len(K))  # numerical stability

# Draw 5 random functions from the GP prior
np.random.seed(42)
L = np.linalg.cholesky(K)
f_prior = L @ np.random.normal(size=(len(X_test), 5))

print(f"Sampled 5 functions from GP(0, RBF) prior")
print(f"Function values range: [{f_prior.min():.2f}, {f_prior.max():.2f}]")
print(f"Each function is smooth because the RBF kernel enforces smoothness")

GP Regression

Now the useful part. You have some training data — input points X and noisy observations y = f(X) + ε, where ε ~ Normal(0, σ²_n). You want to predict f at new test points X*. GP regression gives you both a prediction and a measure of uncertainty at every test point.

Prior Over Functions

Before seeing data, the GP defines a prior over functions. The prior mean is m(X*) = 0 and the prior covariance is K(X*, X*). Functions drawn from this prior are smooth (if using an RBF kernel) but otherwise unconstrained.

Posterior Predictive Distribution

After observing data, the posterior predictive distribution at test points X* is:

f* | X, y, X* ~ Normal(μ*, Σ*)

where:

μ* = K(X*, X) [K(X, X) + σ²_n I]⁻¹ y

Σ* = K(X*, X*) - K(X*, X) [K(X, X) + σ²_n I]⁻¹ K(X, X*)

The posterior mean μ* is a weighted combination of the training observations, where the weights are determined by the kernel. The posterior covariance Σ* shrinks near training points (the function is well-determined there) and grows far from training points (more uncertainty where you have not seen data).

import numpy as np

def gp_regression(X_train, y_train, X_test, kernel, noise_var=0.1):
    """Gaussian process regression with exact inference."""
    K = kernel(X_train, X_train) + noise_var * np.eye(len(X_train))
    K_s = kernel(X_train, X_test)
    K_ss = kernel(X_test, X_test)

    # Cholesky decomposition for numerical stability
    L = np.linalg.cholesky(K)
    alpha = np.linalg.solve(L.T, np.linalg.solve(L, y_train))

    # Posterior mean
    mu = K_s.T @ alpha

    # Posterior covariance
    v = np.linalg.solve(L, K_s)
    cov = K_ss - v.T @ v

    # Standard deviation (uncertainty)
    std = np.sqrt(np.diag(cov))

    return mu, std, cov

# Generate training data
np.random.seed(42)
X_train = np.sort(np.random.uniform(-5, 5, 20)).reshape(-1, 1)
y_train = np.sin(X_train).ravel() + np.random.normal(0, 0.2, 20)
X_test = np.linspace(-6, 6, 200).reshape(-1, 1)

# Predict
kernel = lambda X1, X2: rbf_kernel(X1, X2, length_scale=1.0, variance=1.0)
mu, std, _ = gp_regression(X_train, y_train, X_test, kernel, noise_var=0.04)

print(f"Predictions at x=0: mean={mu[100]:.3f}, std={std[100]:.3f}")
print(f"Predictions at x=5.5 (extrapolation): mean={mu[192]:.3f}, std={std[192]:.3f}")
print("Notice: uncertainty grows in extrapolation region")

Uncertainty Bands

The posterior standard deviation gives you uncertainty bands around the prediction. A 95% credible interval is μ* ± 1.96·σ*. These bands are narrow near training points and wide in regions with no data. This is the killer feature of GPs: the uncertainty is calibrated. When the GP says it is uncertain, it really does not know, and when it says it is confident, the true function is likely within the bands.

# Using scikit-learn's GP implementation
from sklearn.gaussian_process import GaussianProcessRegressor
from sklearn.gaussian_process.kernels import RBF, WhiteKernel, ConstantKernel

# Define kernel with learnable hyperparameters
kernel_sklearn = (ConstantKernel(1.0, (1e-3, 1e3)) *
                  RBF(length_scale=1.0, length_scale_bounds=(1e-2, 1e2)) +
                  WhiteKernel(noise_level=0.1, noise_level_bounds=(1e-5, 1e1)))

gpr = GaussianProcessRegressor(kernel=kernel_sklearn,
                                n_restarts_optimizer=10,
                                random_state=42)
gpr.fit(X_train, y_train)

# Predict with uncertainty
y_mean, y_std = gpr.predict(X_test, return_std=True)

print(f"Optimized kernel: {gpr.kernel_}")
print(f"Log marginal likelihood: {gpr.log_marginal_likelihood_value_:.3f}")
print(f"Near data (x=0):  mean={y_mean[100]:.3f}, std={y_std[100]:.3f}")
print(f"Far from data (x=5.5): mean={y_mean[192]:.3f}, std={y_std[192]:.3f}")
💡 Noise Modeling

The noise variance σ²_n is critical. Too small and the GP tries to interpolate exactly through noisy observations, producing a wiggly mean function. Too large and it over-smooths, missing real patterns. The standard approach is to include a WhiteKernel in your kernel composition and let marginal likelihood optimization find the right noise level automatically.

Kernel Functions

The kernel is the most important modeling decision in GP regression. It controls what kinds of functions the GP considers likely. Choosing the wrong kernel is like fitting a line to data that follows a sine wave — the math works, but the model is wrong.

Squared Exponential (RBF) Kernel

k(x, x') = σ² exp(−‖x − x'‖² / (2ℓ²))

The squared exponential (also called the radial basis function or RBF) kernel is the default choice. It produces infinitely smooth functions. The length scale ℓ controls how far apart two points can be and still be correlated. Small ℓ means the function can wiggle rapidly. Large ℓ means the function changes slowly. The signal variance σ² controls the overall amplitude.

Matérn Kernels

The Matérn kernel family generalizes the RBF to functions with finite smoothness. The parameter ν controls the differentiability of the functions: ν = 1/2 gives rough (once-differentiable) functions, ν = 3/2 gives moderately smooth functions, ν = 5/2 gives quite smooth functions, and ν → ∞ recovers the RBF. The Matérn-5/2 kernel is often the best default — it is smooth enough for most applications but does not impose the unrealistic infinite smoothness of the RBF.

Periodic Kernel

k(x, x') = σ² exp(−2 sin²(π|x − x'|/p) / ℓ²)

The periodic kernel generates functions that repeat with period p. Use it when you expect your data to have seasonal or cyclic patterns — hourly temperature, weekly sales, annual revenue.

Kernel Composition

Kernels can be added and multiplied to build complex covariance structures. Adding kernels means the function is the sum of independent components — e.g., a smooth trend plus periodic oscillations. Multiplying kernels means the components interact — e.g., a periodic pattern whose amplitude grows over time.

from sklearn.gaussian_process.kernels import (
    RBF, Matern, ExpSineSquared, ConstantKernel, WhiteKernel
)

# Kernel for CO2 data: long-term trend + seasonal + noise
# This is the classic Rasmussen & Williams decomposition
kernel_co2 = (
    # Long-term smooth trend
    ConstantKernel(50.0) * RBF(length_scale=50.0) +
    # Seasonal component (yearly period)
    ConstantKernel(2.0) * ExpSineSquared(length_scale=1.0, periodicity=1.0) *
    RBF(length_scale=10.0) +  # slowly varying amplitude
    # Medium-term irregularities
    ConstantKernel(0.5) * RBF(length_scale=2.0) +
    # Noise
    WhiteKernel(noise_level=0.1)
)

print("CO2 kernel components:")
print("  1. Long-term trend (RBF, ℓ=50 years)")
print("  2. Seasonal cycle (Periodic × RBF for varying amplitude)")
print("  3. Medium-term variation (RBF, ℓ=2 years)")
print("  4. Measurement noise (White)")
print(f"\nFull kernel: {kernel_co2}")

Hyperparameter Learning

Marginal Likelihood Optimization

The kernel has hyperparameters: length scales, signal variances, noise levels. How do you set them? The Bayesian approach is to maximize the marginal likelihood (also called the evidence) — the probability of the observed data given the hyperparameters, with the function f integrated out:

log p(y | X, θ) = −½ yᵀ (K + σ²_n I)⁻¹ y − ½ log|K + σ²_n I| − n/2 log(2π)

This expression has three terms: a data fit term (how well the model explains the observations), a complexity penalty (log determinant — penalizes complex covariance structures), and a normalization constant. The balance between fit and complexity provides automatic Occam's razor — the marginal likelihood naturally favors the simplest model that explains the data.

import numpy as np
from scipy.optimize import minimize
from scipy.linalg import cho_factor, cho_solve

def neg_log_marginal_likelihood(params, X, y):
    """Negative log marginal likelihood for RBF kernel + noise."""
    log_variance, log_length, log_noise = params
    variance = np.exp(log_variance)
    length = np.exp(log_length)
    noise = np.exp(log_noise)

    # Compute kernel matrix
    sqdist = np.sum((X[:, None] - X[None, :]) ** 2, axis=-1)
    K = variance * np.exp(-0.5 * sqdist / length ** 2)
    K_y = K + noise * np.eye(len(X))

    # Cholesky factorization
    L, lower = cho_factor(K_y)
    alpha = cho_solve((L, lower), y)

    # Log marginal likelihood
    n = len(y)
    fit_term = -0.5 * y @ alpha
    complexity_term = -np.sum(np.log(np.diag(L)))  # -0.5 log|K_y|
    constant = -0.5 * n * np.log(2 * np.pi)
    lml = fit_term + complexity_term + constant

    return -lml  # minimize negative

# Optimize hyperparameters
X_flat = X_train.ravel()
result = minimize(neg_log_marginal_likelihood,
                  x0=[0.0, 0.0, -2.0],  # log(variance), log(length), log(noise)
                  args=(X_flat, y_train),
                  method='L-BFGS-B')

opt_var, opt_len, opt_noise = np.exp(result.x)
print(f"Optimized hyperparameters:")
print(f"  Signal variance: {opt_var:.3f}")
print(f"  Length scale: {opt_len:.3f}")
print(f"  Noise variance: {opt_noise:.4f}")

Bayesian Optimization of Hyperparameters

GPs themselves are used to optimize expensive black-box functions — this is Bayesian optimization. The idea: you have an expensive function (like training a neural network with given hyperparameters) and want to find the minimum in as few evaluations as possible. Fit a GP to the (hyperparameters, loss) pairs you have observed so far, then use the GP's posterior to decide where to evaluate next. The acquisition function (Expected Improvement, Upper Confidence Bound, etc.) balances exploration (trying uncertain regions) and exploitation (trying regions with low predicted loss).

from sklearn.gaussian_process import GaussianProcessRegressor
from sklearn.gaussian_process.kernels import Matern
import numpy as np

def bayesian_optimization(objective, bounds, n_initial=5, n_iterations=20):
    """Simple Bayesian optimization with Expected Improvement."""
    dim = len(bounds)

    # Initial random evaluations
    np.random.seed(42)
    X_obs = np.random.uniform(
        [b[0] for b in bounds], [b[1] for b in bounds],
        size=(n_initial, dim)
    )
    y_obs = np.array([objective(x) for x in X_obs])

    gp = GaussianProcessRegressor(
        kernel=Matern(nu=2.5), n_restarts_optimizer=5, random_state=42
    )

    for i in range(n_iterations):
        gp.fit(X_obs, y_obs)
        best_y = y_obs.min()

        # Expected improvement on a grid of candidates
        X_cand = np.random.uniform(
            [b[0] for b in bounds], [b[1] for b in bounds],
            size=(10_000, dim)
        )
        mu, sigma = gp.predict(X_cand, return_std=True)
        sigma = np.maximum(sigma, 1e-8)

        from scipy.stats import norm
        z = (best_y - mu) / sigma
        ei = sigma * (z * norm.cdf(z) + norm.pdf(z))

        # Pick the candidate with highest EI
        next_x = X_cand[np.argmax(ei)]
        next_y = objective(next_x)

        X_obs = np.vstack([X_obs, next_x])
        y_obs = np.append(y_obs, next_y)

    best_idx = np.argmin(y_obs)
    return X_obs[best_idx], y_obs[best_idx]

# Example: optimize Branin function (standard BO benchmark)
def branin(x):
    x1, x2 = x[0], x[1]
    a, b, c = 1, 5.1/(4*np.pi**2), 5/np.pi
    r, s, t = 6, 10, 1/(8*np.pi)
    return a*(x2 - b*x1**2 + c*x1 - r)**2 + s*(1-t)*np.cos(x1) + s

best_x, best_y = bayesian_optimization(
    branin, bounds=[(-5, 10), (0, 15)],
    n_initial=5, n_iterations=25
)
print(f"Best x found: [{best_x[0]:.3f}, {best_x[1]:.3f}]")
print(f"Best y found: {best_y:.4f}")
print(f"True minimum: ~0.3979")
⚠️ Computational Cost

Standard GP regression requires O(N³) time and O(N²) memory for the Cholesky decomposition, where N is the number of training points. This means GPs hit a wall around N = 5,000-10,000 points. For larger datasets, you need approximate methods.

Scalability

Sparse GP Approximations (Inducing Points)

The key idea behind sparse GPs: instead of conditioning on all N training points, pick M ≪ N inducing points and condition on those instead. The inducing points are pseudo-inputs whose locations and function values are learned during optimization. The cost drops from O(N³) to O(NM²), making GPs feasible for datasets with tens or hundreds of thousands of points.

# Sparse GP with GPyTorch (PyTorch-based GP library)
import torch
import gpytorch
from gpytorch.models import ApproximateGP
from gpytorch.variational import (
    CholeskyVariationalDistribution,
    VariationalStrategy
)

class SparseGPModel(ApproximateGP):
    def __init__(self, inducing_points):
        variational_distribution = CholeskyVariationalDistribution(
            inducing_points.size(0)
        )
        variational_strategy = VariationalStrategy(
            self, inducing_points, variational_distribution,
            learn_inducing_locations=True
        )
        super().__init__(variational_strategy)
        self.mean_module = gpytorch.means.ConstantMean()
        self.covar_module = gpytorch.kernels.ScaleKernel(
            gpytorch.kernels.RBFKernel()
        )

    def forward(self, x):
        mean = self.mean_module(x)
        covar = self.covar_module(x)
        return gpytorch.distributions.MultivariateNormal(mean, covar)

Variational Sparse GP

The modern approach to sparse GPs is variational: define a variational distribution over the function values at inducing points and maximize a lower bound on the marginal likelihood (the ELBO). This framework, developed by Titsias (2009) and extended by Hensman et al. (2013), is clean, principled, and compatible with stochastic optimization — enabling mini-batch training for truly large datasets.

Random Fourier Features

An entirely different approach to scaling: approximate the kernel with a finite-dimensional feature map using Bochner's theorem. For a stationary kernel, you can draw random frequencies from the kernel's spectral density and construct features z(x) such that z(x)ᵀz(x') ≈ k(x, x'). Then you do standard Bayesian linear regression in the feature space. This reduces the GP to a parametric model with D features, costing O(ND²) — linear in N.

from sklearn.kernel_approximation import RBFSampler
from sklearn.linear_model import BayesianRidge
import numpy as np

# Random Fourier Features + Bayesian Linear Regression ≈ Sparse GP
np.random.seed(42)
N = 10_000
X_large = np.sort(np.random.uniform(-5, 5, N)).reshape(-1, 1)
y_large = np.sin(X_large.ravel()) + np.random.normal(0, 0.2, N)

# Approximate RBF kernel with 500 random Fourier features
rff = RBFSampler(gamma=0.5, n_components=500, random_state=42)
X_features = rff.fit_transform(X_large)

# Bayesian linear regression in feature space
blr = BayesianRidge()
blr.fit(X_features, y_large)

# Predict with uncertainty
X_test_feat = rff.transform(X_test)
y_pred, y_std = blr.predict(X_test_feat, return_std=True)

print(f"Training points: {N}")
print(f"Random features: 500")
print(f"Prediction RMSE: {np.sqrt(np.mean((y_pred - np.sin(X_test.ravel()))**2)):.4f}")
print(f"Avg uncertainty: {y_std.mean():.4f}")
print("This scales linearly in N — handles millions of points")
ℹ️ When to Use GPs

GPs shine in three scenarios: (1) small datasets where you need calibrated uncertainty, (2) Bayesian optimization for expensive black-box functions, and (3) spatial or temporal modeling where the kernel encodes domain knowledge. For large-scale regression where you do not need uncertainty, neural networks or gradient-boosted trees are more practical. For large-scale regression where you do need uncertainty, sparse GPs or deep kernel learning (GP on neural network features) bridge the gap.
