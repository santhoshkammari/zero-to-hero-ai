Non-Parametric Bayesian Methods

Chapter 14: Probabilistic & Bayesian ML 7 subtopics
TL;DR

Non-parametric Bayesian methods let the model's complexity grow with the data. Instead of choosing "K clusters" or "P features" up front, you let the data decide. The Dirichlet Process handles unbounded clustering, the Indian Buffet Process handles unbounded feature discovery, and Gaussian Processes provide flexible function estimation — all with proper uncertainty. This section demystifies these methods through their generative stories (Chinese Restaurant Process, buffet metaphor) and shows when they beat their fixed-capacity counterparts.

The Problem with Fixing Model Size

Every model you have seen so far forces you to pick a size before seeing the data. K-means needs K. A mixture model needs the number of components. A factor model needs the number of latent dimensions. Pick too few and you underfit. Pick too many and you overfit. You can try cross-validation, but that is expensive and still commits you to one choice.

Non-parametric Bayesian methods side-step this problem entirely. "Non-parametric" is a terrible name — these models have parameters, sometimes infinitely many. The name means "not a fixed, finite number of parameters." The number of effective parameters grows with the data. See 100 data points, use a few clusters. See 10,000 points from a more complex distribution, the model automatically uses more clusters. No manual tuning of K required.

Dirichlet Process

The Dirichlet Process (DP) is the foundation of non-parametric Bayesian clustering. It is a distribution over distributions — a probability measure on the space of probability measures. That sounds abstract, so let us build intuition before the math.

Imagine you are sorting customer behavior into segments. You do not know how many segments exist. The DP says: "Start with zero segments. As you observe each customer, either assign them to an existing segment (with probability proportional to how many customers are already there) or create a brand new segment (with probability proportional to a concentration parameter α)." Rich-get-richer dynamics naturally emerge — big segments attract more members — but new segments can always appear.

Formally, G ~ DP(α, G₀) is a random probability distribution. α is the concentration parameter that controls how many clusters you expect. G₀ is the base distribution — the prior on where new cluster centers come from. Draws from a DP are discrete with probability 1 (they concentrate on a countable set of atoms), even if G₀ is continuous. This discreteness is what creates clustering.

import numpy as np

def stick_breaking_dp(alpha, base_distribution, n_atoms=100):
    """
    Stick-breaking construction of a Dirichlet Process.

    Imagine a stick of length 1. Break off a random fraction.
    That fraction is the weight of cluster 1. From the remaining
    stick, break off another random fraction → weight of cluster 2.
    Continue forever. Most weight concentrates on early clusters.

    v_k ~ Beta(1, alpha)     ← fraction to break off
    w_1 = v_1
    w_k = v_k * prod(1 - v_j for j < k)
    theta_k ~ G_0             ← cluster parameter from base dist
    """
    betas = np.random.beta(1, alpha, size=n_atoms)

    # Compute stick-breaking weights
    weights = np.zeros(n_atoms)
    remaining = 1.0
    for k in range(n_atoms):
        weights[k] = remaining * betas[k]
        remaining *= (1 - betas[k])

    # Sample atom locations from base distribution
    atoms = base_distribution(n_atoms)

    return weights, atoms

# Small alpha → few big clusters (weights concentrate early)
# Large alpha → many small clusters (weights spread out)
weights_small, _ = stick_breaking_dp(alpha=1.0, base_distribution=lambda n: np.random.normal(0, 5, n))
weights_large, _ = stick_breaking_dp(alpha=50.0, base_distribution=lambda n: np.random.normal(0, 5, n))

print(f"alpha=1.0: top 3 weights sum to {np.sort(weights_small)[::-1][:3].sum():.2f}")
print(f"alpha=50:  top 3 weights sum to {np.sort(weights_large)[::-1][:3].sum():.2f}")
# alpha=1.0 → top 3 dominate (e.g., 0.85)
# alpha=50  → top 3 are small (e.g., 0.15)

The stick-breaking construction gives a concrete recipe: break a unit-length stick into pieces using Beta(1, α) random variables. Each piece's length is a cluster weight. Each cluster gets a parameter drawn from G₀. This infinite sequence of weights and parameters is the DP sample — though in practice you truncate to a finite number of atoms since the weights decay rapidly.

Chinese Restaurant Process

The Chinese Restaurant Process (CRP) is the Dirichlet Process viewed from the lens of sequential data assignment. It is a beautiful metaphor that makes the DP's clustering behavior intuitive.

Imagine an infinite Chinese restaurant with infinitely many tables. The first customer sits at the first table. Each subsequent customer either joins an existing table with probability proportional to the number of people already seated there, or starts a new table with probability proportional to α.

import numpy as np

def chinese_restaurant_process(n_customers, alpha):
    """
    Simulate the Chinese Restaurant Process.

    Customer i+1 joins:
      - table k with probability n_k / (i + alpha)
        where n_k = number of people at table k
      - a NEW table with probability alpha / (i + alpha)
    """
    table_assignments = []
    table_counts = []  # number of customers at each table

    for i in range(n_customers):
        if i == 0:
            # First customer always starts a new table
            table_assignments.append(0)
            table_counts.append(1)
        else:
            # Probability of joining each existing table
            probs = np.array(table_counts, dtype=float)
            # Probability of a new table
            probs = np.append(probs, alpha)
            probs /= probs.sum()

            chosen = np.random.choice(len(probs), p=probs)
            table_assignments.append(chosen)

            if chosen == len(table_counts):
                table_counts.append(1)    # New table
            else:
                table_counts[chosen] += 1  # Existing table

    return table_assignments, table_counts

# Simulation
assignments, counts = chinese_restaurant_process(1000, alpha=5.0)
print(f"1000 customers, alpha=5: {len(counts)} tables")
print(f"Table sizes (top 10): {sorted(counts, reverse=True)[:10]}")
# Typical: ~30 tables, power-law distribution of sizes

The CRP has two key properties. First, exchangeability: the probability of any partition depends only on the cluster sizes, not the order in which customers arrived. Second, rich-get-richer: large tables attract more customers. This creates a power-law distribution of cluster sizes — a few large clusters and a long tail of small ones — which matches many real-world phenomena (word frequencies, city sizes, species abundances).

💡 Key Insight

The CRP and stick-breaking are two views of the same mathematical object (the Dirichlet Process). The CRP is easier to reason about sequentially — "what happens when a new data point arrives?" Stick-breaking is easier to implement in variational inference — "pre-compute the weights, then assign." Use whichever view matches your problem.

Indian Buffet Process

The CRP assigns each data point to exactly one cluster. But what if each data point can have multiple latent features? A movie is not one genre — it can be action AND sci-fi AND comedy. The Indian Buffet Process (IBP) extends non-parametric Bayesian thinking to this multi-feature setting.

The metaphor: customers at an Indian buffet walk along an infinitely long buffet line. The first customer tries Poisson(α) dishes. Each subsequent customer tries each previously tasted dish with probability proportional to its popularity, then tries Poisson(α/i) new dishes. The result is a binary matrix Z where Z[n,k] = 1 if customer n tried dish k.

import numpy as np

def indian_buffet_process(n_customers, alpha):
    """
    Simulate the Indian Buffet Process.

    Returns a binary matrix Z of shape (n_customers, K)
    where K is determined by the process.
    Z[i, k] = 1 means customer i has feature k.
    """
    # Customer 1 tries Poisson(alpha) new dishes
    n_new = np.random.poisson(alpha)
    Z = np.zeros((n_customers, n_new), dtype=int)
    Z[0, :] = 1

    for i in range(1, n_customers):
        # For each existing dish, try it with prob = (# who tried it) / (i+1)
        if Z.shape[1] > 0:
            col_sums = Z[:i, :].sum(axis=0)  # popularity of each dish
            probs = col_sums / (i + 1)
            Z[i, :Z.shape[1]] = (np.random.random(Z.shape[1]) < probs).astype(int)

        # Try Poisson(alpha / (i+1)) new dishes
        n_new = np.random.poisson(alpha / (i + 1))
        if n_new > 0:
            new_cols = np.zeros((n_customers, n_new), dtype=int)
            new_cols[i, :] = 1
            Z = np.hstack([Z, new_cols])

    return Z

Z = indian_buffet_process(100, alpha=4.0)
print(f"100 customers discovered {Z.shape[1]} features")
print(f"Average features per customer: {Z.sum(axis=1).mean():.1f}")
print(f"Feature popularity (top 10): {sorted(Z.sum(axis=0), reverse=True)[:10]}")

The IBP is the foundation for non-parametric latent feature models. Instead of specifying "use K latent factors" in a matrix factorization, you let the IBP determine K from the data. This is used in topic modeling (documents have multiple topics), recommendation systems (users have multiple preference dimensions), and computational biology (genes have multiple functions).

Dirichlet Process Mixture Models

The most practical application of the DP is the Dirichlet Process Mixture Model (DPMM) — an infinite mixture model where the number of components is determined by the data. It is the non-parametric replacement for finite Gaussian mixture models.

# Finite Gaussian Mixture Model (you choose K):
#   z_i ~ Categorical(π_1, ..., π_K)          ← cluster assignment
#   x_i | z_i=k ~ Normal(μ_k, Σ_k)           ← observation from cluster k
#
# Dirichlet Process Mixture Model (K is infinite):
#   G ~ DP(α, G_0)                            ← random mixing distribution
#   θ_i ~ G                                    ← draw cluster parameter
#   x_i | θ_i ~ F(θ_i)                        ← observation from that cluster
#
# Equivalently (stick-breaking):
#   π ~ StickBreaking(α)                       ← infinite weight vector
#   μ_k, Σ_k ~ G_0 = NIW(...)                 ← cluster parameters
#   z_i ~ Categorical(π)                       ← assignment (now unbounded)
#   x_i | z_i=k ~ Normal(μ_k, Σ_k)

In practice, you fit DPMMs using either MCMC (collapsed Gibbs sampling, which integrates out the mixing distribution and samples cluster assignments directly) or variational inference (truncated stick-breaking with a maximum number of components, letting the model set unused component weights to near-zero).

import numpy as np
from sklearn.mixture import BayesianGaussianMixture

# Scikit-learn implements a truncated DPMM via variational inference
# Set n_components to an upper bound — the model will use fewer

# Generate data from 4 true clusters
np.random.seed(42)
true_centers = [[-3, -3], [0, 3], [4, -1], [6, 5]]
X = np.vstack([np.random.normal(c, 0.8, size=(100, 2)) for c in true_centers])

# Fit DPMM — set upper bound of 20 components
dpmm = BayesianGaussianMixture(
    n_components=20,                   # Upper bound, not fixed K
    covariance_type='full',
    weight_concentration_prior_type='dirichlet_process',
    weight_concentration_prior=1.0,    # Concentration parameter α
    max_iter=500,
    random_state=42
)
dpmm.fit(X)

# How many components did the model actually use?
effective_components = (dpmm.weights_ > 0.01).sum()
print(f"Set upper bound: 20, effective components: {effective_components}")
# → effective components: 4 (recovered the true number!)

print(f"Component weights: {np.round(dpmm.weights_[:8], 3)}")
# → [0.250, 0.250, 0.248, 0.252, 0.000, 0.000, 0.000, ...] — 4 active, rest ~0
💡 Key Insight

The concentration parameter α controls the expected number of clusters, not the exact number. With α = 1 and n data points, you expect roughly α · log(n) clusters. With n = 1000, that is about 7 clusters. Increasing α gives more clusters. But the data always has the final say — if there are clearly 3 groups, even α = 100 will eventually concentrate on 3 main components (with many tiny ones).

Gaussian Process as Non-Parametric Model

We covered Gaussian Processes in detail in a previous section, but it is worth re-framing them in the non-parametric Bayesian context. A GP is a distribution over functions — an infinite-dimensional object. The function you draw from a GP has a value at every point in the input space, but you only observe it at finitely many points. The posterior GP uses those observations to constrain the function everywhere else.

This is non-parametric in the truest sense: the function is not parameterized by a finite vector. The effective complexity of the function (how wiggly it is) is controlled by the kernel and grows with the data. With more observations, the GP posterior can represent more complex functions.

# Parametric regression: y = w_0 + w_1*x + w_2*x^2 + noise
#   → Fixed model capacity (quadratic). Always 3 parameters.
#   → Cannot capture patterns beyond degree-2 polynomials.
#
# GP regression: f ~ GP(m(x), k(x, x'))
#   → Infinite model capacity (determined by kernel).
#   → n observations → n-dimensional posterior at those points.
#   → More data → more complex patterns captured.
#
# The kernel k(x, x') is the "prior" on function smoothness:
#   - RBF kernel → infinitely smooth functions
#   - Matérn-3/2 → once-differentiable (rougher)
#   - Periodic kernel → repeating patterns
#   - Sum/product of kernels → combine properties
#
# The GP posterior mean is equivalent to kernel ridge regression.
# But the GP also gives you the posterior VARIANCE — uncertainty!

The connection to neural networks is deep: as a single-hidden-layer Bayesian neural network's width goes to infinity, its output converges to a GP. This is Neal's 1996 result, and it means GPs can be seen as the infinite-width limit of BNNs. Modern work on the Neural Tangent Kernel extends this to deep networks and gradient descent dynamics.

Beta Process

The Beta Process (BP) is a continuous analogue of the Indian Buffet Process. While the IBP generates binary feature assignments, the Beta Process generates feature probabilities. Each data point has a probability (between 0 and 1) for each of infinitely many features, rather than a hard yes/no.

Formally, B ~ BP(c, B₀) is a completely random measure on some feature space. Each atom has a location (what the feature is) and a weight between 0 and 1 (how likely it is to be active). The Bernoulli Process draws from the Beta Process: Z_i | B ~ BeP(B), giving binary feature indicators.

# Beta Process and Bernoulli Process:
#
# B ~ BP(c, B_0)
#   B = sum_k p_k * delta_{theta_k}
#   where p_k ~ Beta(c * B_0(theta_k), c * (1 - B_0(theta_k)))
#   and theta_k are feature locations from B_0
#
# For each data point i:
#   z_{ik} ~ Bernoulli(p_k)  — binary feature assignment
#
# The marginal distribution of Z (integrating out B) is the IBP.
#
# Applications:
# - Dictionary learning: B gives feature weights, Z gives which features
#   each data point uses, X ≈ Z * D where D is the dictionary
# - Sparse factor analysis: infinite latent factors, each active for a
#   subset of observations

The Beta Process is used in Bayesian dictionary learning and sparse signal recovery. It provides a principled way to learn an over-complete dictionary (more features than data dimensions) while automatically controlling sparsity through the feature probabilities.

Pitman-Yor Process

The Pitman-Yor Process (PYP) generalizes the Dirichlet Process with an additional "discount" parameter d ∈ [0, 1). Where the DP produces a geometric-like distribution of cluster sizes, the PYP produces heavier-tailed, power-law distributions. This makes it a better fit for phenomena like word frequencies in natural language, where there are many rare words and a few extremely common ones.

import numpy as np

def pitman_yor_restaurant(n_customers, alpha, discount):
    """
    Pitman-Yor generalization of the Chinese Restaurant Process.

    Customer i+1 joins:
      - table k with probability (n_k - discount) / (i + alpha)
      - a NEW table with probability (alpha + K*discount) / (i + alpha)
        where K = current number of tables

    discount=0 → standard CRP (Dirichlet Process)
    discount>0 → heavier tails, more tables, power-law sizes
    """
    table_assignments = []
    table_counts = []

    for i in range(n_customers):
        if i == 0:
            table_assignments.append(0)
            table_counts.append(1)
        else:
            K = len(table_counts)
            # Prob of each existing table
            probs = [max(0, n_k - discount) for n_k in table_counts]
            # Prob of new table
            new_prob = alpha + K * discount
            probs.append(new_prob)

            probs = np.array(probs, dtype=float)
            probs /= probs.sum()

            chosen = np.random.choice(len(probs), p=probs)
            table_assignments.append(chosen)

            if chosen == K:
                table_counts.append(1)
            else:
                table_counts[chosen] += 1

    return table_assignments, table_counts

# Compare DP vs PYP with same α
_, counts_dp = pitman_yor_restaurant(10000, alpha=5.0, discount=0.0)
_, counts_py = pitman_yor_restaurant(10000, alpha=5.0, discount=0.5)

print(f"DP (d=0):   {len(counts_dp)} tables, largest: {max(counts_dp)}")
print(f"PYP (d=0.5): {len(counts_py)} tables, largest: {max(counts_py)}")
# PYP creates more tables with a heavier tail of small tables

The Pitman-Yor Process is the backbone of Bayesian language models like the Hierarchical Pitman-Yor (HPY) language model. In these models, word distributions at each context length (bigram, trigram, etc.) follow a PYP, with longer contexts backing off to shorter ones hierarchically. Before neural language models, HPY models were state-of-the-art for language modeling because they captured the power-law statistics of natural language.

⚠️ Warning

The discount parameter d interacts with α in non-obvious ways. When d > 0, the expected number of clusters grows as O(n^d) — polynomially rather than logarithmically (as with the DP). Setting d too high creates far too many clusters. Start with d = 0 (pure DP) and only increase if you have evidence of power-law behavior in your cluster sizes.

When to Use Non-Parametric Bayesian Methods

Non-parametric Bayesian methods shine in three situations. First, when you genuinely do not know the model size and getting it wrong is costly — exploratory clustering, topic discovery, anomaly detection. Second, when you need a fully Bayesian treatment of model complexity, including uncertainty over the number of components. Third, when your data arrives in a stream and new clusters or features may appear over time — the CRP naturally accommodates new tables for new customers.

They are less appropriate when you have strong prior knowledge about the number of components (use a finite model), when scalability is the primary concern (non-parametric inference is generally slower), or when interpretability requires a fixed-size model. For most large-scale ML applications, the scalability limitation is the binding constraint — which is why you see finite mixture models and fixed-architecture neural networks dominate in practice, even though non-parametric methods are theoretically more elegant.

# Decision guide:
#
# Q: Do you know the number of clusters/features?
#    Yes → Use finite mixture model, K-means, or NMF
#    No  → Consider DPMM or IBP
#
# Q: How much data do you have?
#    <10K points → Non-parametric Bayes works well
#    10K-100K   → Variational inference with truncation
#    >100K      → Consider scalable alternatives (Mini-batch DPMM, spectral methods)
#
# Q: Do cluster sizes follow a power law?
#    Yes → Pitman-Yor Process
#    No  → Dirichlet Process is fine
#
# Q: Does each data point belong to ONE group or MULTIPLE features?
#    One group → DP / CRP
#    Multiple  → IBP / Beta Process
💡 Key Insight

Non-parametric does not mean "no assumptions." You still choose the base distribution G₀, the concentration parameter α, the likelihood model, and (for PYP) the discount parameter. These choices profoundly affect the results. "Non-parametric" means one specific thing: the number of mixture components (or latent features) is not fixed in advance. Every other modeling decision is still yours to make.
