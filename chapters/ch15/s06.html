Modern Generative Breakthroughs (2024–2025)

Chapter 15: Generative Models 5 subtopics
TL;DR

The generative modeling landscape exploded in 2024–2025. Flow Matching and Rectified Flows made diffusion-style generation faster and simpler by learning straight-line paths between noise and data. Consistency Models let you generate in one or two steps instead of hundreds. The Diffusion Transformer (DiT) replaced U-Nets as the backbone of choice. Video generation became practical with temporal attention and spacetime patches. 3D generation took off via Neural Radiance Fields and Gaussian Splatting. And Visual Autoregressive Modeling (VAR) challenged the assumption that autoregressive generation must proceed left-to-right.

Flow Matching & Rectified Flows

Diffusion models work, but they're conceptually heavy — forward noising process, reverse denoising, noise schedules, score functions, SDE vs ODE formulations. What if we could get the same quality with a cleaner framework? That's exactly what Flow Matching delivers.

Probability Flow ODE

Every diffusion model defines a stochastic process that adds noise to data. Song et al. (2021) showed that this stochastic process has a deterministic counterpart — the Probability Flow ODE. Instead of a noisy random walk from data to noise, there exists an ordinary differential equation whose trajectories transform the data distribution into a Gaussian. Running this ODE backward transforms noise back into data.

# The probability flow ODE:
# dx/dt = f(x, t) - 0.5 * g(t)² * ∇_x log p_t(x)
#
# f(x,t): drift coefficient
# g(t): diffusion coefficient
# ∇_x log p_t(x): score function at time t

The score function ∇_x log p_t(x) is what a diffusion model learns. Once you have it, you can generate samples by solving this ODE from t=1 (pure noise) to t=0 (clean data) using any numerical ODE solver. No stochastic sampling needed — the generation is entirely deterministic for a given starting noise.

Continuous Normalizing Flows

A Continuous Normalizing Flow (CNF) defines a time-dependent vector field v_θ(x, t) that transports samples from a simple distribution (Gaussian) to a complex one (your data). The key equation is:

# dx/dt = v_θ(x, t)
# Start: x(0) ~ N(0, I)      (noise)
# End:   x(1) ~ p_data(x)    (data)

This is an ODE defined by a neural network. To generate data, sample noise and integrate forward. To compute log-likelihoods, integrate backward and use the instantaneous change of variables formula. CNFs were proposed by Chen et al. (2018) but were expensive to train because they required solving an ODE at every training step. Flow Matching solves this.

Flow Matching

Flow Matching (Lipman et al., 2023) is a simulation-free training method for CNFs. Instead of solving the full ODE during training, you directly regress on the velocity field at randomly sampled time steps. The trick: define a simple conditional flow that you know the velocity for, then train the network to match it.

import torch
import torch.nn as nn

class FlowMatchingModel(nn.Module):
    """Neural network that predicts velocity v(x, t)."""
    def __init__(self, dim=784, hidden=512, time_emb_dim=128):
        super().__init__()
        self.time_mlp = nn.Sequential(
            nn.Linear(1, time_emb_dim),
            nn.SiLU(),
            nn.Linear(time_emb_dim, time_emb_dim)
        )
        self.net = nn.Sequential(
            nn.Linear(dim + time_emb_dim, hidden),
            nn.SiLU(),
            nn.Linear(hidden, hidden),
            nn.SiLU(),
            nn.Linear(hidden, hidden),
            nn.SiLU(),
            nn.Linear(hidden, dim)
        )

    def forward(self, x, t):
        t_emb = self.time_mlp(t.unsqueeze(-1))
        return self.net(torch.cat([x, t_emb], dim=-1))


def flow_matching_loss(model, x_data):
    """
    Conditional Flow Matching with linear interpolation.
    """
    batch_size = x_data.size(0)
    # Sample random noise and time
    x_noise = torch.randn_like(x_data)
    t = torch.rand(batch_size, device=x_data.device)

    # Linear interpolation: x_t = (1-t)*x_noise + t*x_data
    t_expand = t.unsqueeze(-1)
    x_t = (1 - t_expand) * x_noise + t_expand * x_data

    # Target velocity: d(x_t)/dt = x_data - x_noise
    target_velocity = x_data - x_noise

    # Predict velocity and minimize MSE
    predicted_velocity = model(x_t, t)
    loss = ((predicted_velocity - target_velocity) ** 2).sum(dim=-1).mean()
    return loss

That's it. No noise schedules. No score function estimation. No SDE formulation. You interpolate between noise and data, compute the velocity of that interpolation, and train a network to predict it. The loss function is an MSE between the predicted and true velocity. This makes Flow Matching dramatically simpler to implement than diffusion models while achieving comparable or better results.

Conditional Flow Matching

Conditional Flow Matching conditions the velocity field on the source-target pair. Instead of learning a marginal velocity field directly (which requires knowing the full coupling between noise and data distributions), you decompose it into conditional flows — one per data point. The marginal velocity field emerges as an average over all conditional flows.

This is what makes the training above work. The linear interpolation path x_t = (1-t)·x₀ + t·x₁ is a specific conditional flow. The target velocity x₁ - x₀ is the velocity of this conditional flow. The trained network learns to average over many such conditional flows, recovering the true marginal velocity field.

The optimal transport variant of conditional flow matching pairs noise and data points to minimize transport cost, which yields even straighter paths. But even with random pairing (independent coupling), the learned flow is remarkably effective.

Rectified Flow & the Reflow Procedure

Rectified Flow (Liu et al., 2023) takes Flow Matching further by straightening the learned flow paths. The idea: even though we train with straight-line conditional paths, the learned marginal paths can be curved. Curved paths require many ODE solver steps. Straight paths can be traversed in one step.

def reflow_loss(model, x_data, old_model, n_ode_steps=50):
    """
    Reflow: straighten paths by learning from the old model's ODE
    trajectories.
    Step 1: Generate noise-data pairs using old model's ODE
    Step 2: Train new model on straight lines between those pairs
    """
    batch_size = x_data.size(0)

    # Step 1: Use old model to generate paired samples
    # Run ODE from data (t=1) to noise (t=0)
    x_1 = x_data.clone()
    x_curr = x_1.clone()
    dt = -1.0 / n_ode_steps
    with torch.no_grad():
        for i in range(n_ode_steps):
            t_val = 1.0 + i * dt
            t = torch.full((batch_size,), t_val, device=x_data.device)
            v = old_model(x_curr, t)
            x_curr = x_curr + v * dt
    x_0 = x_curr  # paired noise for this data

    # Step 2: Train on straight-line interpolation between pairs
    t = torch.rand(batch_size, device=x_data.device)
    t_expand = t.unsqueeze(-1)
    x_t = (1 - t_expand) * x_0 + t_expand * x_1

    target_v = x_1 - x_0
    pred_v = model(x_t, t)
    loss = ((pred_v - target_v) ** 2).sum(dim=-1).mean()
    return loss

The Reflow procedure is iterative: train a flow model, use its ODE to create noise-data pairs, train a new model on straight lines between those pairs, repeat. Each iteration makes the paths straighter. After 2–3 rounds, paths are straight enough that a single Euler step produces high-quality samples. This is how models like Stable Diffusion 3 achieve one-step or two-step generation.

Consistency Models

Consistency Models (Song et al., 2023) attack the multi-step problem differently. Instead of straightening paths, they enforce a self-consistency property: any point on the same ODE trajectory should map to the same output. If x_t and x_s are on the same trajectory (at different times), the model should map both to the same clean image x_0.

def consistency_training_loss(model, x_data, noise_schedule,
                               ema_model=None):
    """
    Consistency Training: enforce f(x_t, t) = f(x_s, s)
    for adjacent points on the same trajectory.
    """
    batch_size = x_data.size(0)

    # Sample two adjacent timesteps
    t = torch.rand(batch_size, device=x_data.device) * 0.98 + 0.01
    delta = 0.01  # small gap
    s = (t - delta).clamp(min=0.001)

    noise = torch.randn_like(x_data)

    # Points on the same noising trajectory
    x_t = x_data + noise_schedule(t).unsqueeze(-1) * noise
    x_s = x_data + noise_schedule(s).unsqueeze(-1) * noise

    # Consistency: both should map to same output
    pred_t = model(x_t, t)
    with torch.no_grad():
        target_model = ema_model if ema_model else model
        pred_s = target_model(x_s, s)

    loss = ((pred_t - pred_s) ** 2).sum(dim=-1).mean()
    return loss
💡 Distillation vs Training

Consistency Distillation starts from a pretrained diffusion model and distills it into a consistency model. Consistency Training trains from scratch without a teacher model. Distillation gives better results (you start with a good teacher), but training is more flexible. Both achieve 1–2 step generation with quality approaching 50-step diffusion.

Visual Autoregressive Modeling (VAR)

Visual Autoregressive Modeling (Tian et al., 2024) rethinks autoregressive generation for images. Language models generate tokens left-to-right. For images, VAR generates scale-by-scale — first a tiny 1×1 token map, then 2×2, then 4×4, and so on, each scale conditioned on all coarser scales. This is next-scale prediction instead of next-token prediction.

class VARBlock(nn.Module):
    """Simplified VAR: predict next scale from all previous scales."""
    def __init__(self, dim=512, n_heads=8):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.mlp = nn.Sequential(
            nn.Linear(dim, dim * 4),
            nn.GELU(),
            nn.Linear(dim * 4, dim)
        )
        self.norm1 = nn.LayerNorm(dim)
        self.norm2 = nn.LayerNorm(dim)

    def forward(self, x):
        h = self.norm1(x)
        x = x + self.attn(h, h, h)[0]
        x = x + self.mlp(self.norm2(x))
        return x


# VAR generation loop (pseudocode)
def var_generate(model, tokenizer, class_label,
                 scales=[1, 2, 4, 8, 16, 32]):
    """Generate image scale-by-scale."""
    tokens = []
    for scale in scales:
        n_tokens = scale * scale
        # Predict all tokens at this scale in parallel
        context = torch.cat(tokens) if tokens else torch.zeros(1, 0, 512)
        next_scale_logits = model(context, scale, class_label)
        # Sample tokens for this scale
        new_tokens = tokenizer.sample(next_scale_logits, n_tokens)
        tokens.append(new_tokens)
    return tokenizer.decode(torch.cat(tokens))

VAR achieves remarkable results: on ImageNet 256×256, it outperforms diffusion models in FID while being 20× faster at inference. The coarse-to-fine structure mirrors how humans think about images — first the rough composition, then progressively finer details.

3D Generation

Neural Radiance Fields (NeRF)

Neural Radiance Fields (Mildenhall et al., 2020) represent 3D scenes as a continuous function that maps a 3D position and viewing direction to a color and density. You train this function (an MLP) using only 2D photographs taken from different angles — no 3D supervision needed.

class SimpleNeRF(nn.Module):
    """Simplified NeRF: maps (x,y,z,θ,φ) to (r,g,b,σ)."""
    def __init__(self, pos_dim=60, dir_dim=24, hidden=256):
        super().__init__()
        self.pos_encoder = PositionalEncoding(3, pos_dim // 2)
        self.dir_encoder = PositionalEncoding(2, dir_dim // 2)

        self.density_net = nn.Sequential(
            nn.Linear(pos_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden + 1)  # features + density σ
        )
        self.color_net = nn.Sequential(
            nn.Linear(hidden + dir_dim, hidden // 2), nn.ReLU(),
            nn.Linear(hidden // 2, 3), nn.Sigmoid()  # RGB in [0,1]
        )

    def forward(self, pos, direction):
        pos_enc = self.pos_encoder(pos)
        dir_enc = self.dir_encoder(direction)

        h = self.density_net(pos_enc)
        density = torch.relu(h[..., :1])   # σ >= 0
        features = h[..., 1:]

        color = self.color_net(torch.cat([features, dir_enc], dim=-1))
        return color, density


class PositionalEncoding(nn.Module):
    """Fourier feature encoding for coordinate inputs."""
    def __init__(self, input_dim, n_freqs):
        super().__init__()
        freqs = 2.0 ** torch.arange(n_freqs) * torch.pi
        self.register_buffer('freqs', freqs)
        self.input_dim = input_dim

    def forward(self, x):
        x_proj = x.unsqueeze(-1) * self.freqs
        return torch.cat([x_proj.sin(), x_proj.cos()],
                         dim=-1).flatten(-2)

NeRF renders images via volume rendering: shoot a ray through each pixel, sample points along the ray, query the MLP for color and density at each point, and composite them front-to-back. The rendering is differentiable, so you backpropagate through the entire rendering pipeline to train the MLP from photos alone. The original NeRF was slow (hours to train, seconds per frame to render), but it proved that neural networks can represent 3D scenes with stunning fidelity.

Combining NeRF with diffusion models enables text-to-3D generation. Score Distillation Sampling (SDS) uses a pretrained 2D diffusion model to provide gradient signal for optimizing a NeRF — no 3D training data required. Multi-view diffusion generates consistent images from multiple viewpoints, which are then used to reconstruct a 3D model.

3D Gaussian Splatting

3D Gaussian Splatting (Kerbl et al., 2023) represents scenes as millions of colored 3D Gaussians instead of a neural network. Each Gaussian has a position, covariance (shape), color, and opacity. To render an image, "splat" (project) each Gaussian onto the image plane and blend them by depth order.

# 3D Gaussian Splatting representation (pseudocode)
class GaussianSplatScene(nn.Module):
    """Scene as a collection of optimizable Gaussians."""
    def __init__(self, n_gaussians=100000):

Video Generation

Temporal Attention

Image diffusion generates one frame. Video diffusion generates dozens or hundreds of coherent frames. The key architectural addition is temporal attention: attention layers that operate across the time axis, allowing the model to keep objects, colors, and motion consistent across frames.

class VideoTransformerBlock(nn.Module):
    """Factorized attention: spatial then temporal."""
    def __init__(self, dim=512, n_heads=8, n_frames=16):
        super().__init__()
        self.spatial_attn = nn.MultiheadAttention(
            dim, n_heads, batch_first=True)
        self.temporal_attn = nn.MultiheadAttention(
            dim, n_heads, batch_first=True)
        self.norm_s = nn.LayerNorm(dim)
        self.norm_t = nn.LayerNorm(dim)
        self.n_frames = n_frames

    def forward(self, x):
        # x: (batch * n_frames, n_spatial_tokens, dim)
        B_T, S, D = x.shape
        B = B_T // self.n_frames
        T = self.n_frames

        # Spatial: each frame attends to its own patches
        h = self.norm_s(x)
        x = x + self.spatial_attn(h, h, h)[0]

        # Reshape for temporal: (B*T, S, D) -> (B*S, T, D)
        x = x.view(B, T, S, D).permute(0, 2, 1, 3).reshape(B*S, T, D)

        # Temporal: each spatial position attends across frames
        h = self.norm_t(x)
        x = x + self.temporal_attn(h, h, h)[0]

        # Back to spatial layout: (B*S, T, D) -> (B*T, S, D)
        x = x.view(B, S, T, D).permute(0, 2, 1, 3).reshape(B*T, S, D)
        return x

The factorized design (spatial then temporal) is computationally necessary. Full spatiotemporal attention over all patches in all frames would be quadratic in both space and time — prohibitively expensive for high-resolution video. By factorizing, each attention layer is quadratic in only one dimension.

Video Diffusion & Spacetime Patches

Video Diffusion extends image diffusion to the temporal domain. The key idea behind Sora (OpenAI, 2024) and related work: treat video as a sequence of spacetime patches. A 16-frame, 256×256 video becomes a 3D volume of patches (e.g., 4×16×16 per patch), which are then flattened and processed by a Diffusion Transformer.

class SpacetimePatchEmbed(nn.Module):
    """Convert video to spacetime patch tokens."""
    def __init__(self, frame_size=256, patch_size=16,
                 temporal_patch=4, in_channels=3, embed_dim=768):
        super().__init__()
        self.proj = nn.Conv3d(
            in_channels, embed_dim,
            kernel_size=(temporal_patch, patch_size, patch_size),
            stride=(temporal_patch, patch_size, patch_size)
        )

    def forward(self, video):
        # video: (B, C, T, H, W)
        tokens = self.proj(video)  # (B, D, t, h, w)
        B, D, t, h, w = tokens.shape
        return tokens.flatten(2).permute(0, 2, 1)  # (B, t*h*w, D)
📐 Scaling Is the Main Lever

Video generation models scale dramatically with compute. Sora's quality jump came primarily from scaling the DiT backbone to billions of parameters and training on massive video datasets — not from fundamental architectural innovations. Once you have the right framework (DiT + spacetime patches + flow matching), scaling is the main lever for quality improvements.

Scaling & Architecture

Diffusion Transformer (DiT)

The Diffusion Transformer (DiT) by Peebles and Xie (2023) replaced U-Net with a standard Vision Transformer as the diffusion backbone. U-Nets dominated for years because of their skip connections and multi-scale processing, but DiT showed that a plain Transformer with the right conditioning mechanism achieves better FID at the same compute budget.

class DiTBlock(nn.Module):
    """
    DiT Block with AdaLN-Zero conditioning.
    Adaptive layer norm injects timestep and class info.
    """
    def __init__(self, dim=768, n_heads=12):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.mlp = nn.Sequential(
            nn.Linear(dim, dim * 4), nn.GELU(),
            nn.Linear(dim * 4, dim)
        )
        self.norm1 = nn.LayerNorm(dim, elementwise_affine=False)
        self.norm2 = nn.LayerNorm(dim, elementwise_affine=False)

        # AdaLN-Zero: predict scale, shift, gate from conditioning
        # 6 vectors: γ₁, β₁, α₁ (attn), γ₂, β₂, α₂ (mlp)
        self.adaLN = nn.Sequential(
            nn.SiLU(),
            nn.Linear(dim, 6 * dim)
        )
        # Zero-init so block starts as identity
        nn.init.zeros_(self.adaLN[-1].weight)
        nn.init.zeros_(self.adaLN[-1].bias)

    def forward(self, x, cond):
        """
        x: (B, N, D) patch tokens
        cond: (B, D) conditioning (timestep + class embed)
        """
        params = self.adaLN(cond).unsqueeze(1)  # (B, 1, 6*D)
        gamma1, beta1, alpha1, gamma2, beta2, alpha2 = \
            params.chunk(6, dim=-1)

        # Attention with AdaLN
        h = self.norm1(x) * (1 + gamma1) + beta1
        h = self.attn(h, h, h)[0]
        x = x + alpha1 * h

        # MLP with AdaLN
        h = self.norm2(x) * (1 + gamma2) + beta2
        h = self.mlp(h)
        x = x + alpha2 * h
        return x

AdaLN-Zero is the critical design choice. Standard adaptive layer norm (AdaLN) modulates normalized activations with predicted scale and shift. AdaLN-Zero adds a gating factor (α) initialized to zero, so each DiT block starts as an identity function and gradually learns to contribute. This makes training stable even at very large scales.

MMDiT

MMDiT (Multi-Modal DiT), used in Stable Diffusion 3, extends DiT to handle multiple modalities — text and image tokens — within the same Transformer. Instead of cross-attention from image to text (as in earlier models), MMDiT processes both token types in a shared attention layer but uses separate MLPs for each modality.

class MMDiTBlock(nn.Module):
    """Multi-Modal DiT: shared attention, separate MLPs."""
    def __init__(self, dim=768, n_heads=12):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.img_mlp = nn.Sequential(
            nn.Linear(dim, dim * 4), nn.GELU(),
            nn.Linear(dim * 4, dim))
        self.txt_mlp = nn.Sequential(
            nn.Linear(dim, dim * 4), nn.GELU(),
            nn.Linear(dim * 4, dim))
        self.img_norm1 = nn.LayerNorm(dim)
        self.txt_norm1 = nn.LayerNorm(dim)
        self.img_norm2 = nn.LayerNorm(dim)
        self.txt_norm2 = nn.LayerNorm(dim)

    def forward(self, img_tokens, txt_tokens):
        n_img = img_tokens.size(1)
        # Concat for joint attention
        combined = torch.cat([
            self.img_norm1(img_tokens),
            self.txt_norm1(txt_tokens)
        ], dim=1)
        attn_out = self.attn(combined, combined, combined)[0]
        img_attn, txt_attn = attn_out[:, :n_img], attn_out[:, n_img:]

        img_tokens = img_tokens + img_attn
        txt_tokens = txt_tokens + txt_attn

        img_tokens = img_tokens + self.img_mlp(
            self.img_norm2(img_tokens))
        txt_tokens = txt_tokens + self.txt_mlp(
            self.txt_norm2(txt_tokens))
        return img_tokens, txt_tokens

The joint attention lets text and image tokens attend to each other bidirectionally, enabling stronger text-image alignment than cross-attention. The separate MLPs preserve modality-specific processing. SD3 showed this design outperforms cross-attention at the same parameter count.

Latent Diffusion & Guidance Scale

Latent Diffusion runs the diffusion process in the latent space of a pretrained autoencoder instead of pixel space. A 512×512×3 image becomes a 64×64×4 latent — 48× fewer values. This makes diffusion computationally feasible for high-resolution generation and is the foundation of all Stable Diffusion models.

The guidance scale controls how strongly the model follows a text prompt via classifier-free guidance: at each denoising step, compute both a conditional prediction (with the prompt) and an unconditional prediction (without it), then extrapolate away from the unconditional:

def guided_prediction(model, x_t, t, text_embed, guidance_scale=7.5):
    """Classifier-free guidance at inference time."""
    null_embed = torch.zeros_like(text_embed)
    pred_uncond = model(x_t, t, null_embed)
    pred_cond = model(x_t, t, text_embed)

    # Extrapolate in the direction of the prompt
    guided = pred_uncond + guidance_scale * (pred_cond - pred_uncond)
    return guided

Scale 1.0 means no guidance (pure model output). Scale 7.5 is a common default — it sharpens images and improves text alignment at the cost of some diversity. Higher values (10–20) produce more "on-prompt" but less diverse and sometimes oversaturated results.

Logit-Normal Timestep Sampling

Standard diffusion samples timesteps uniformly during training. Logit-normal sampling (used in SD3 and FLUX) samples more timesteps from the middle of the [0, 1] range where the denoising task is hardest.

def logit_normal_timestep(batch_size, mean=0.0, std=1.0, device='cpu'):
    """
    Sample timesteps with logit-normal distribution.
    Concentrates samples in the middle of [0, 1].
    """
    u = torch.randn(batch_size, device=device) * std + mean
    t = torch.sigmoid(u)  # maps (-inf, inf) -> (0, 1), biased to center
    return t

This seemingly minor change improves training efficiency significantly. Uniform sampling wastes compute on very noisy (t≈1) or nearly clean (t≈0) timesteps where the task is easy. The logit-normal distribution focuses learning capacity where it matters most.

💡 The 2024–2025 Stack

The state-of-the-art generative model in 2024–2025 typically combines: a VAE encoder/decoder for latent space, a DiT or MMDiT backbone, flow matching or rectified flow training, classifier-free guidance, logit-normal timestep sampling, and an EMA target network for stability. Understanding each component and why it's there gives you the ability to read and implement any modern generative model paper.
