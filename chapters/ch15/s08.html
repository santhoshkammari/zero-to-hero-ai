Generative Model Evaluation

Chapter 15: Generative Models 7 subtopics
TL;DR

How do you measure whether a generative model is any good? It's harder than classification (no single accuracy number). FID measures distributional similarity by comparing feature statistics between real and generated images — lower is better. Inception Score checks if generated images are both sharp and diverse. LPIPS measures perceptual similarity between image pairs. CLIP Score measures text-image alignment. No single metric tells the whole story, and there's an inherent tradeoff between fidelity (FID) and text alignment (CLIP Score). Human evaluation remains the gold standard but doesn't scale. AI-generated image detection is the flip side: can we tell when an image is fake?

The Measurement Problem

Classification has accuracy. Regression has MSE. What does generative modeling have? The answer is: a messy collection of imperfect metrics, each measuring a different aspect of quality, none of them sufficient alone.

A good generative model must satisfy multiple criteria simultaneously. The outputs must look realistic (fidelity). The model must produce diverse outputs, not memorize a few examples (diversity). For conditional generation, the outputs must match the input prompt (alignment). And ideally, the model should cover the full data distribution, not collapse to a subset (coverage). No single number captures all of this.

Let's walk through the metrics one by one, understanding what each measures, how it works mechanically, and where it breaks down.

FID: Fréchet Inception Distance

The Fréchet Inception Distance (FID) is the most widely reported metric in generative modeling. It measures how similar the distribution of generated images is to the distribution of real images. Lower FID means more similar — an FID of 0 would mean the distributions are identical.

How FID Works

FID doesn't compare images directly. Instead, it passes both real and generated images through a pretrained Inception-v3 network and extracts features from the pool3 layer (a 2048-dimensional vector per image). It then models both sets of features as multivariate Gaussians and computes the Fréchet distance (also called the Wasserstein-2 distance) between them.

import torch
import numpy as np
from scipy import linalg

def compute_fid(real_features, gen_features):
    """
    Compute FID between two sets of Inception features.

    real_features: (n_real, 2048) numpy array
    gen_features: (n_gen, 2048) numpy array
    Returns: FID score (lower is better)
    """
    # Compute mean and covariance for real images
    mu_real = real_features.mean(axis=0)
    sigma_real = np.cov(real_features, rowvar=False)

    # Compute mean and covariance for generated images
    mu_gen = gen_features.mean(axis=0)
    sigma_gen = np.cov(gen_features, rowvar=False)

    # Fréchet distance between two Gaussians
    diff = mu_real - mu_gen
    # Matrix square root of product of covariances
    covmean, _ = linalg.sqrtm(sigma_real @ sigma_gen, disp=False)

    # Numerical stability: remove imaginary components
    if np.iscomplexobj(covmean):
        covmean = covmean.real

    fid = (diff @ diff +
           np.trace(sigma_real) +
           np.trace(sigma_gen) -
           2 * np.trace(covmean))
    return float(fid)

The formula has three terms. The first, ||μ_real - μ_gen||², measures whether generated images have the same "average" feature profile as real images. The second and third terms measure whether the spread and correlations of features match. Together, they capture both the central tendency and the shape of the feature distribution.

Practical FID Computation

import torch
import torchvision.models as models
import torchvision.transforms as T

class FIDCalculator:
    """End-to-end FID computation pipeline."""
    def __init__(self, device='cuda'):
        # Load Inception-v3 and remove the final classification layer
        self.inception = models.inception_v3(
            weights='IMAGENET1K_V1', transform_input=False)
        self.inception.fc = torch.nn.Identity()  # remove classifier
        self.inception.eval().to(device)
        self.device = device

        self.transform = T.Compose([
            T.Resize(299),  # Inception expects 299x299
            T.CenterCrop(299),
            T.ToTensor(),
            T.Normalize([0.485, 0.456, 0.406],
                       [0.229, 0.224, 0.225])
        ])

    @torch.no_grad()
    def extract_features(self, dataloader):
        """Extract 2048-dim features from a dataset."""
        all_features = []
        for batch in dataloader:
            if isinstance(batch, (list, tuple)):
                batch = batch[0]
            batch = batch.to(self.device)
            features = self.inception(batch)  # (B, 2048)
            all_features.append(features.cpu().numpy())
        return np.concatenate(all_features, axis=0)

    def compute(self, real_loader, gen_loader):
        """Compute FID between real and generated datasets."""
        real_feats = self.extract_features(real_loader)
        gen_feats = self.extract_features(gen_loader)
        return compute_fid(real_feats, gen_feats)
⚠️ FID Pitfalls

FID is sensitive to sample size — you need at least 10,000 images for reliable estimates (50,000 is standard). FID also depends on the Inception model, which was trained on ImageNet. For domains far from natural photos (medical images, art, molecules), Inception features may not capture relevant quality differences. Some researchers use CLIP features or domain-specific encoders instead, calling the metric "FCD" (Fréchet CLIP Distance) or similar.

Inception Score (IS)

The Inception Score was an earlier metric (Salimans et al., 2016) that uses the same Inception network but measures two properties: each image should have a confident class prediction (high quality → low entropy per image), and across all images, the class predictions should be diverse (variety → high entropy across images).

def inception_score(gen_images_probs, splits=10):
    """
    Compute Inception Score from class probabilities.

    gen_images_probs: (n_images, 1000) softmax probabilities
                      from Inception-v3 classifier
    Returns: (mean_IS, std_IS)
    """
    scores = []
    n = len(gen_images_probs)
    split_size = n // splits

    for i in range(splits):
        part = gen_images_probs[i*split_size : (i+1)*split_size]

        # p(y|x): per-image class distribution (should be peaky)
        py_given_x = part

        # p(y): marginal class distribution (should be uniform)
        py = py_given_x.mean(axis=0, keepdims=True)

        # KL divergence: D_KL(p(y|x) || p(y))
        kl_div = py_given_x * (np.log(py_given_x + 1e-10) -
                                np.log(py + 1e-10))
        kl_div = kl_div.sum(axis=1).mean()

        scores.append(np.exp(kl_div))

    return float(np.mean(scores)), float(np.std(scores))

Limitations of IS

IS has serious blind spots. It doesn't compare to real data at all — a model that generates perfect images of only 1000 categories gets a high IS even if those images look nothing like the training data. It can't detect mode dropping within classes (generating only one type of dog). It's capped by the number of ImageNet classes (1000). And it rewards models that generate easily classifiable images, which biases toward certain visual styles.

Today, IS is mostly reported alongside FID for historical comparability, not as a primary metric. If you could only report one number, FID is more informative.

LPIPS: Learned Perceptual Image Patch Similarity

LPIPS (Zhang et al., 2018) measures how perceptually different two specific images are. Unlike FID (which compares distributions), LPIPS compares individual image pairs. It's used for evaluating reconstruction quality (autoencoders, super-resolution) and diversity (higher average LPIPS between generated samples means more diversity).

import torch
import torch.nn as nn
import torchvision.models as models

class LPIPS(nn.Module):
    """
    Learned Perceptual Image Patch Similarity.
    Uses deep features from VGG/AlexNet to measure perceptual distance.
    """
    def __init__(self, backbone='vgg16'):
        super().__init__()
        vgg = models.vgg16(weights='IMAGENET1K_V1').features
        # Extract features at 5 different scales
        self.layers = nn.ModuleList([
            vgg[:4],    # relu1_2: 64 channels
            vgg[4:9],   # relu2_2: 128 channels
            vgg[9:16],  # relu3_3: 256 channels
            vgg[16:23], # relu4_3: 512 channels
            vgg[23:30], # relu5_3: 512 channels
        ])
        # Learned linear weights per channel (trained on human judgments)
        self.weights = nn.ModuleList([
            nn.Conv2d(c, 1, 1, bias=False)
            for c in [64, 128, 256, 512, 512]
        ])
        # Freeze backbone
        for p in self.layers.parameters():
            p.requires_grad = False

    def forward(self, img1, img2):
        """
        Compute perceptual distance between two images.
        Returns: scalar distance (lower = more similar)
        """
        total = 0.0
        x, y = img1, img2
        for layer, weight in zip(self.layers, self.weights):
            x = layer(x)
            y = layer(y)
            # Normalize features
            x_norm = x / (x.norm(dim=1, keepdim=True) + 1e-10)
            y_norm = y / (y.norm(dim=1, keepdim=True) + 1e-10)
            # Weighted spatial average of squared differences
            diff = (x_norm - y_norm) ** 2
            total += weight(diff).mean()
        return total

The key insight of LPIPS: raw pixel differences (L1, L2, SSIM) don't match human perception well. Two images can differ by many pixels (e.g., a slight shift) and look identical to humans, or differ by few pixels (e.g., a changed eye color) and look dramatically different. LPIPS solves this by comparing deep features — which capture semantics, not pixels — and weighting them with coefficients learned from human perceptual judgments.

LPIPS values typically range from 0 to 1. Values below 0.1 indicate images that humans would consider nearly identical. Values above 0.5 indicate images that look significantly different.

CLIP Score: Text-Image Alignment

For text-to-image models, you need to measure not only image quality but also whether the image matches the text prompt. CLIP Score uses OpenAI's CLIP model to compute the cosine similarity between a text embedding and an image embedding.

import torch

class CLIPScoreCalculator:
    """Compute CLIP Score for text-image alignment."""
    def __init__(self, clip_model, clip_processor, device='cuda'):
        self.model = clip_model.eval().to(device)
        self.processor = clip_processor
        self.device = device

    @torch.no_grad()
    def compute_score(self, images, texts):
        """
        Compute CLIP Score between images and text prompts.

        images: list of PIL images
        texts: list of text strings
        Returns: average CLIP score (higher = better alignment)
        """
        # Encode images
        img_inputs = self.processor(
            images=images, return_tensors="pt").to(self.device)
        img_features = self.model.get_image_features(**img_inputs)
        img_features = img_features / img_features.norm(
            dim=-1, keepdim=True)

        # Encode texts
        txt_inputs = self.processor(
            text=texts, return_tensors="pt",
            padding=True).to(self.device)
        txt_features = self.model.get_text_features(**txt_inputs)
        txt_features = txt_features / txt_features.norm(
            dim=-1, keepdim=True)

        # Cosine similarity per pair
        scores = (img_features * txt_features).sum(dim=-1)

        # Scale by 100 (convention in some papers)
        return scores.mean().item() * 100


def clip_score_batch(model, images, texts):
    """
    Quick CLIP score computation.

    Typical values:
    - Random image + text: ~18-22
    - Decent alignment: ~25-28
    - Good alignment: ~28-32
    - Excellent alignment: ~32+
    """
    img_emb = model.encode_image(images)
    txt_emb = model.encode_text(texts)

    img_emb = img_emb / img_emb.norm(dim=-1, keepdim=True)
    txt_emb = txt_emb / txt_emb.norm(dim=-1, keepdim=True)

    return (img_emb * txt_emb).sum(dim=-1).mean().item() * 100

CLIP Score is widely used for evaluating text-to-image models like DALL-E, Stable Diffusion, and Midjourney. However, it has a known blind spot: CLIP itself has biases and failure modes. It might give a high score to an image that contains the right objects but in the wrong spatial arrangement ("a cat on top of a dog" vs "a dog on top of a cat"). Compositional understanding remains a challenge for both CLIP and the models it evaluates.

The FID-CLIP Tradeoff

Here's the uncomfortable truth: improving text alignment (higher CLIP Score) often hurts FID, and vice versa. This happens because stronger guidance toward the text prompt produces images that are more "on-topic" but less natural-looking and less diverse.

# The guidance scale dilemma:
#
# guidance_scale=1.0:  FID ≈ 10 (great), CLIP ≈ 24 (poor alignment)
# guidance_scale=3.0:  FID ≈ 12 (good),  CLIP ≈ 28 (decent)
# guidance_scale=7.5:  FID ≈ 18 (ok),    CLIP ≈ 31 (good)
# guidance_scale=15:   FID ≈ 30 (poor),   CLIP ≈ 33 (great)
#
# There's a Pareto frontier — you can't optimize both simultaneously.

def fid_clip_pareto(model, prompts, real_loader, guidance_scales):
    """Sweep guidance scales to find the Pareto frontier."""
    results = []
    for scale in guidance_scales:
        gen_images = model.generate(prompts, guidance_scale=scale)

        fid = compute_fid_from_images(real_loader, gen_images)
        clip_score = compute_clip_score(gen_images, prompts)

        results.append({
            'guidance_scale': scale,
            'fid': fid,
            'clip_score': clip_score
        })
        print(f"Scale {scale:.1f}: FID={fid:.1f}, CLIP={clip_score:.1f}")

    return results

This tradeoff is fundamental, not a bug. FID rewards naturalness and diversity (looking like the real data distribution). CLIP Score rewards prompt faithfulness (generating exactly what was asked for). A perfectly prompt-faithful image might not look "natural" — it might be oversaturated, oversharpened, or have other artifacts from strong guidance. Papers now typically report both metrics and show the Pareto curve across guidance scales.

📊 Benchmark Numbers (2024)

State-of-the-art FID on ImageNet 256×256: around 1.5–2.0 (DiT-XL/2, SiT). For reference, FID between two random splits of real ImageNet data is about 1.0 — we're approaching the noise floor of the metric itself. CLIP Scores for the best text-to-image models: 31–33. Human preference alignment (ELO ratings from user studies) has become the de facto ranking for commercial models like Midjourney and DALL-E 3.

Human Evaluation Protocols

Automated metrics are proxies. The ground truth for generative model quality is human judgment. But human evaluation is expensive, slow, and hard to reproduce. Standardized protocols help.

# Common human evaluation setups:
#
# 1. Two-Alternative Forced Choice (2AFC)
#    Show two images (from model A and B), ask "which is more realistic?"
#    Result: win rate per model
#
# 2. Mean Opinion Score (MOS)
#    Show one image, ask to rate quality on 1-5 scale
#    Result: average score per model
#
# 3. Text-Image Alignment Rating
#    Show image + prompt, ask "does this match?" on 1-5 scale
#    Result: average alignment score
#
# 4. ELO Rating System
#    Pairwise comparisons, accumulate into ELO scores
#    Used by: Chatbot Arena (for LLMs), ImagenHub (for image gen)

class HumanEvalProtocol:
    """Framework for organizing human evaluation studies."""
    def __init__(self, n_evaluators=5, n_images=500):
        self.n_evaluators = n_evaluators
        self.n_images = n_images

    def two_afc(self, model_a_images, model_b_images):
        """
        Two-Alternative Forced Choice evaluation.

        Best practices:
        - Randomize left/right positioning
        - Include attention checks (obvious quality differences)
        - Require minimum 5 evaluators per comparison
        - Report 95% confidence intervals
        - Use inter-annotator agreement (Krippendorff's alpha)
        """
        comparisons = []
        for i in range(self.n_images):
            # Randomize order to avoid position bias
            if torch.rand(1) > 0.5:
                left, right = model_a_images[i], model_b_images[i]
                order = 'a_left'
            else:
                left, right = model_b_images[i], model_a_images[i]
                order = 'b_left'
            comparisons.append({
                'left': left, 'right': right,
                'order': order, 'prompt': prompts[i]
            })
        return comparisons

    def compute_elo(self, pairwise_results, initial_elo=1000, k=32):
        """
        Compute ELO ratings from pairwise comparison results.
        Same algorithm used in chess ratings.
        """
        elos = {}
        for result in pairwise_results:
            a, b = result['model_a'], result['model_b']
            if a not in elos: elos[a] = initial_elo
            if b not in elos: elos[b] = initial_elo

            expected_a = 1 / (1 + 10 ** ((elos[b] - elos[a]) / 400))
            actual_a = 1.0 if result['winner'] == 'a' else \
                       0.0 if result['winner'] == 'b' else 0.5

            elos[a] += k * (actual_a - expected_a)
            elos[b] += k * ((1 - actual_a) - (1 - expected_a))

        return elos

The ELO system has become particularly popular because it handles multiple models gracefully and is intuitive (higher ELO = better). Chatbot Arena pioneered this for LLMs, and similar platforms now exist for image generation. The downside: you need thousands of comparisons per model pair for stable ratings, and ratings shift as new models enter the pool.

AI-Generated Image Detection & Forensics

The flip side of generating realistic images is detecting them. As models improve, this becomes a critical societal need — for combating misinformation, protecting intellectual property, and maintaining trust in visual media.

Detection Approaches

            features = self.backbone(image)
        return self.classifier(features)


class FrequencyAnalysisDetector:
    """
    Detect AI images via frequency domain analysis.
    Many generators leave artifacts in the Fourier spectrum.
    """
    def analyze(self, image_tensor):
        # Convert to grayscale
        gray = image_tensor.mean(dim=0)

        # 2D FFT
        spectrum = torch.fft.fft2(gray)
        magnitude = torch.log(torch.abs(spectrum) + 1e-10)

        # AI-generated images often show:
        # 1. Grid-like artifacts from upsampling layers
        # 2. Unusual high-frequency rolloff patterns
        # 3. Periodic patterns from conv stride artifacts

        # Azimuthal average of power spectrum
        center = torch.tensor(magnitude.shape) // 2
        Y, X = torch.meshgrid(
            torch.arange(magnitude.shape[0]),
            torch.arange(magnitude.shape[1]), indexing='ij')
        R = torch.sqrt((X - center[1])**2 + (Y - center[0])**2)
        R = R.long()

        radial_profile = torch.zeros(R.max() + 1)
        counts = torch.zeros(R.max() + 1)
        for r in range(R.max() + 1):
            mask = (R == r)
            radial_profile[r] = magnitude[mask].mean()
            counts[r] = mask.sum()

        return radial_profile

Current detection methods work reasonably well against specific generators but struggle with generalization. A detector trained on Stable Diffusion outputs might fail on images from DALL-E 3. The arms race between generators and detectors is ongoing.

Watermarking and Provenance

A complementary approach to detection is proactive watermarking — embedding an invisible signature in generated images at creation time.

# Watermarking approaches for AI-generated content:
#
# 1. Stable Signature (Meta, 2023)
#    Trains the VAE decoder to embed a watermark during generation.
#    The watermark survives JPEG compression, cropping, and resizing.
#    Detection: feed image through a small classifier network.
#
# 2. Tree-Ring Watermarks (Wen et al., 2023)
#    Embeds the watermark in the initial noise pattern of diffusion.
#    By choosing noise with specific Fourier-space patterns,
#    the generated image carries an invisible fingerprint.
#    Detection: invert the diffusion to recover the initial noise
#    and check for the pattern.
#
# 3. C2PA (Coalition for Content Provenance and Authenticity)
#    Industry standard for metadata-based provenance tracking.
#    Embeds cryptographic signatures in image metadata.
#    Limitation: metadata can be stripped, so it's not robust
#    to adversarial removal.

class SimpleImageWatermark(nn.Module):
    """Simplified learned watermarking framework."""
    def __init__(self, msg_len=48, img_channels=3):
        super().__init__()
        # Encoder: embed message into image
        self.encoder = nn.Sequential(
            nn.Conv2d(img_channels + msg_len, 64, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(64, 64, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(64, img_channels, 3, padding=1),
            nn.Tanh()
        )
        # Decoder: extract message from (possibly corrupted) image
        self.decoder = nn.Sequential(
            nn.Conv2d(img_channels, 64, 3, padding=1),
            nn.ReLU(),
            nn.AdaptiveAvgPool2d(16),
            nn.Flatten(),
            nn.Linear(64 * 16 * 16, msg_len)
        )

    def encode(self, image, message):
        """Embed message into image with minimal visual change."""
        # Tile message across spatial dimensions
        B, C, H, W = image.shape
        msg_expanded = message.view(B, -1, 1, 1).expand(-1, -1, H, W)
        combined = torch.cat([image, msg_expanded], dim=1)
        # Small perturbation added to original image
        delta = self.encoder(combined) * 0.01  # keep changes small
        return image + delta

    def decode(self, watermarked_image):
        """Extract message from watermarked (possibly corrupted) image."""
        return torch.sigmoid(self.decoder(watermarked_image))
⚠️ Detection Is Not Solved

No current detection method is reliable enough for high-stakes decisions (legal evidence, content moderation at scale). False positive rates of 5–10% are common, and adversarial attacks (slight image modifications) can fool most detectors. The research community is moving toward a combination approach: passive detection (analyzing the image) + active provenance (watermarks embedded at generation time) + metadata standards (C2PA). All three layers are needed.

Choosing the Right Metrics

There's no single "best" metric. Here's a practical guide for different situations:

# Metric selection guide:
#
# Unconditional image generation (e.g., face generation):
#   Primary: FID (quality + diversity)
#   Secondary: Precision/Recall (fidelity vs coverage)
#   Validation: Human 2AFC studies
#
# Text-to-image generation:
#   Primary: FID + CLIP Score (report the Pareto frontier)
#   Secondary: Human text-alignment ratings
#   Validation: ELO from pairwise human comparisons
#
# Image reconstruction (VAE, super-resolution):
#   Primary: LPIPS (perceptual similarity)
#   Secondary: PSNR, SSIM (pixel-level, for reference)
#   Validation: Human MOS ratings
#
# Video generation:
#   Primary: FVD (Fréchet Video Distance — FID extended to video)
#   Secondary: Frame-level FID, temporal consistency metrics
#   Validation: Human preference studies
#
# Audio generation:
#   Primary: FAD (Fréchet Audio Distance)
#   Secondary: MOS for speech naturalness
#   Validation: Human listening tests (ABX preference)

The field is moving toward multi-metric evaluation dashboards rather than single-number rankings. Papers increasingly report Pareto frontiers (FID vs CLIP at different guidance scales), breakdown metrics (quality/diversity/alignment separately), and human evaluation alongside automated metrics. This is healthier than the "my FID is 0.3 lower" one-upmanship of earlier years.

One final caution: all these metrics measure properties of the output distribution. They don't measure memorization (is the model copying training data?), fairness (does it represent all demographics equally?), or safety (can it generate harmful content?). A model with perfect FID and CLIP Score could still have serious problems along these dimensions. Evaluation is multi-dimensional, and the dimensions we struggle to measure may be the ones that matter most.
