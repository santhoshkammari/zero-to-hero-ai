Diffusion Models

Chapter 15: Generative Models 7 subtopics
TL;DR

Diffusion models generate images by learning to reverse a noise-adding process. You start with real data, gradually corrupt it into pure Gaussian noise across hundreds or thousands of steps, then train a neural network to undo each step. At generation time, you begin with random noise and iteratively denoise it into a crisp image. The approach sidesteps the training instability of GANs and the blurriness of VAEs, producing state-of-the-art image quality. Latent diffusion compresses this process into a lower-dimensional space for efficiency, and classifier-free guidance lets you steer generation with text prompts — which is exactly how Stable Diffusion, DALL·E, and Midjourney work under the hood.

Denoising Diffusion Probabilistic Models (DDPM)

GANs generate sharp images, but training them feels like balancing a pencil on its tip. The generator and discriminator play a minimax game that can collapse, diverge, or mode-drop at any moment. VAEs are more stable, but their outputs look soft — the pixel-level reconstruction loss averages over possibilities and washes out fine detail. By 2020, the field was hungry for a generative approach that was both stable to train and produced high-fidelity samples. Enter diffusion.

The idea behind Denoising Diffusion Probabilistic Models (DDPM) is almost absurdly straightforward. Take a clean image. Add a tiny bit of Gaussian noise. Then add a tiny bit more. And more. Keep going for T steps (usually T = 1000) until the image is pure static — indistinguishable from random noise sampled from a standard normal distribution. That is the forward diffusion process. It requires no learning at all — it is a fixed, predetermined corruption schedule.

Now comes the clever part. Train a neural network to reverse each step: given a noisy image at step t, predict a slightly less noisy version at step t−1. Stack all those tiny denoising steps together, and you can start from pure noise and walk backward all the way to a clean, realistic image. That is the reverse denoising process, and it is the only part that involves learning.

Forward (fixed, no learning):
  x₀ ──→ x₁ ──→ x₂ ──→ ... ──→ x_T
  clean   slightly    more          pure
  image   noisy      noisy         noise

Reverse (learned neural network):
  x_T ──→ x_{T-1} ──→ ... ──→ x₁ ──→ x₀
  pure     slightly              almost   clean
  noise    less noisy            clean    image

The Forward Process and Noise Schedule

At each step t, the forward process adds Gaussian noise controlled by a variance parameter β_t. The noise schedule is the sequence β₁, β₂, ..., β_T, typically with small values (like 0.0001 to 0.02) that increase over time. Given the previous step's image x_{t-1}, the noisy version is:

q(xₜ | xₜ₋₁) = N(xₜ; √(1 - βₜ) · xₜ₋₁, βₜ · I)

You scale the signal down by √(1 − βₜ) and add noise with variance βₜ. One step barely changes the image. But after a thousand steps, the signal is completely buried.

Here is the key insight that makes training practical. You do not need to run all T steps sequentially to get x_t from x₀. Define α_t = 1 − βₜ, and let ᾱ_t = α₁ · α₂ · ... · α_t (the cumulative product). Then you can jump directly to any timestep:

xₜ = √ᾱₜ · x₀ + √(1 - ᾱₜ) · ε    where ε ~ N(0, I)

This is a massive computational win. During training, you pick a random timestep t, sample noise ε, compute x_t in one shot, and ask the network to predict ε. No sequential chain needed.

The Training Objective

The network ε_θ takes a noisy image x_t and the timestep t as inputs, and tries to predict the noise ε that was added. The loss is mean squared error between the predicted noise and the actual noise:

L = E[‖ε - ε_θ(xₜ, t)‖²]

That is the entire training objective. No adversarial games. No KL divergence balancing. One network, one MSE loss, one clear target. This is why diffusion models are so stable to train compared to GANs.

Complete PyTorch DDPM Implementation

import torch
import torch.nn as nn
import torch.nn.functional as F

# --- Noise Schedule ---
def linear_beta_schedule(timesteps, beta_start=1e-4, beta_end=0.02):
    return torch.linspace(beta_start, beta_end, timesteps)

T = 1000
betas = linear_beta_schedule(T)
alphas = 1.0 - betas
alpha_bar = torch.cumprod(alphas, dim=0)  # ᾱ_t

# --- Forward Process: jump to any timestep ---
def q_sample(x0, t, noise=None):
    """Add noise to x0 to get x_t in one step."""
    if noise is None:
        noise = torch.randn_like(x0)
    ab = alpha_bar[t].reshape(-1, 1, 1, 1)  # broadcast for images
    return torch.sqrt(ab) * x0 + torch.sqrt(1.0 - ab) * noise, noise

# --- Simplified U-Net for Noise Prediction ---
class SinusoidalPosEmb(nn.Module):
    """Encode timestep t as a vector using sinusoidal embeddings."""
    def __init__(self, dim):
        super().__init__()
        self.dim = dim

    def forward(self, t):
        half = self.dim // 2
        emb = torch.log(torch.tensor(10000.0)) / (half - 1)
        emb = torch.exp(torch.arange(half, device=t.device) * -emb)
        emb = t[:, None].float() * emb[None, :]
        return torch.cat([emb.sin(), emb.cos()], dim=-1)

class SimpleUNet(nn.Module):
    def __init__(self, in_ch=3, base_ch=64, time_dim=256):
        super().__init__()
        self.time_mlp = nn.Sequential(
            SinusoidalPosEmb(time_dim),
            nn.Linear(time_dim, time_dim),
            nn.GELU(),
        )
        # Encoder
        self.down1 = nn.Conv2d(in_ch, base_ch, 3, padding=1)
        self.down2 = nn.Conv2d(base_ch, base_ch * 2, 3, stride=2, padding=1)
        self.down3 = nn.Conv2d(base_ch * 2, base_ch * 4, 3, stride=2, padding=1)
        # Bottleneck
        self.bot = nn.Conv2d(base_ch * 4, base_ch * 4, 3, padding=1)
        # Time projection
        self.time_proj = nn.Linear(time_dim, base_ch * 4)
        # Decoder
        self.up1 = nn.ConvTranspose2d(base_ch * 4, base_ch * 2, 4, stride=2, padding=1)
        self.up2 = nn.ConvTranspose2d(base_ch * 4, base_ch, 4, stride=2, padding=1)
        self.out = nn.Conv2d(base_ch * 2, in_ch, 1)
        self.act = nn.GELU()

    def forward(self, x, t):
        t_emb = self.time_mlp(t)
        # Encoder path with skip connections
        h1 = self.act(self.down1(x))           # (B, 64, H, W)
        h2 = self.act(self.down2(h1))          # (B, 128, H/2, W/2)
        h3 = self.act(self.down3(h2))          # (B, 256, H/4, W/4)
        # Bottleneck + time conditioning
        h = self.act(self.bot(h3))
        h = h + self.time_proj(t_emb)[:, :, None, None]
        # Decoder with skip connections
        h = self.act(self.up1(h))              # (B, 128, H/2, W/2)
        h = torch.cat([h, h2], dim=1)          # skip connection
        h = self.act(self.up2(h))              # (B, 64, H, W)
        h = torch.cat([h, h1], dim=1)          # skip connection
        return self.out(h)                     # predict noise ε

# --- Training Loop ---
model = SimpleUNet(in_ch=3, base_ch=64)
optimizer = torch.optim.Adam(model.parameters(), lr=2e-4)

for epoch in range(num_epochs):
    for images in dataloader:              # images in [-1, 1]
        t = torch.randint(0, T, (images.shape[0],))  # random timestep
        xt, noise = q_sample(images, t)    # add noise
        pred_noise = model(xt, t)          # predict the noise
        loss = F.mse_loss(pred_noise, noise)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

# --- Sampling (reverse process) ---
@torch.no_grad()
def sample(model, img_shape, device):
    x = torch.randn(img_shape, device=device)  # start from pure noise
    for t in reversed(range(T)):
        t_batch = torch.full((img_shape[0],), t, device=device, dtype=torch.long)
        pred_noise = model(x, t_batch)
        alpha_t = alphas[t]
        ab_t = alpha_bar[t]
        # Reverse step: remove predicted noise
        x = (1 / torch.sqrt(alpha_t)) * (
            x - (betas[t] / torch.sqrt(1 - ab_t)) * pred_noise
        )
        if t > 0:
            x += torch.sqrt(betas[t]) * torch.randn_like(x)  # add stochasticity
    return x
Why predict noise instead of the clean image? You could train the network to predict x₀ directly — the math works out the same. But predicting ε is numerically better behaved. The noise has zero mean and unit variance regardless of the image content, so the network's job is uniform across all timesteps. Predicting x₀ would mean very different magnitudes at t=1 vs t=999.

Score Matching

There is another lens through which to understand diffusion. Forget about iterative noise-adding for a moment. Imagine you have a data distribution p(x) over images. The score function is the gradient of the log-density with respect to the data: ∇ₓ log p(x). This is a vector field. At every point in pixel space, it tells you the direction toward higher probability — toward more realistic images.

Think of a topographic map. The score function is like gravity arrows pointing uphill toward the mountain peaks (the modes of the distribution). If you placed a ball at a random location and let it follow these arrows, it would roll toward a likely image. That is exactly what Langevin dynamics does: start from random noise, and iteratively follow the score plus a small random kick:

x_{t+1} = xₜ + (η/2) · ∇ₓ log p(xₜ) + √η · z    where z ~ N(0, I)

The noise term z prevents the sample from getting stuck and ensures you explore the full distribution, not one mode. With enough steps and a shrinking step size η, this converges to a true sample from p(x).

The problem: we do not know p(x), so we cannot compute the score directly. Denoising Score Matching solves this by training a neural network s_θ(x) to approximate the score. The trick is that if you add known noise to clean data, you can derive the noisy score in closed form and use it as a training target. Sliced score matching reduces the computational cost by projecting the score onto random directions.

Song and Ermon's Noise Conditional Score Network (NCSN) trains a single network s_θ(x, σ) conditioned on different noise levels σ. At high noise, the score landscape is smooth and easy to learn. At low noise, it captures fine details. Sampling starts with high noise and anneals down — coarse to fine.

The beautiful unification came in 2021: Song et al. showed that DDPM and score matching are two views of the same stochastic differential equation (SDE). The forward process is an SDE that adds noise continuously. The reverse process is another SDE that requires the score function. DDPM's ε-prediction and NCSN's score prediction are algebraically equivalent: s_θ(xₜ, t) = −ε_θ(xₜ, t) / √(1 − ᾱₜ). The discrete steps of DDPM are a discretization of this continuous SDE. This SDE framework opened the door to importing decades of numerical ODE/SDE solver research into the sampling pipeline.

Noise Schedules

The noise schedule — the sequence of βₜ values — has an outsized impact on sample quality. Think of it as the curriculum for your denoiser. Too aggressive early on and the network never learns to handle lightly-noised images. Too gentle overall and the endpoint is not fully noisy, so your samples start from the wrong distribution.

The original DDPM used a linear schedule: β increases linearly from 0.0001 to 0.02 across 1000 steps. It works, but it destroys information too quickly in the middle of the process. By step 500, the image is already almost pure noise, wasting the second half of the schedule on near-identical fully-noisy states.

The cosine schedule from Improved DDPM (Nichol & Dhariwal, 2021) fixes this. It defines ᾱₜ using a cosine curve, ensuring a more gradual information loss. The signal-to-noise ratio drops smoothly rather than falling off a cliff. This matters most for low-resolution images (32×32, 64×64) where every pixel counts.

# Cosine schedule: ᾱₜ follows a shifted cosine curve
import math

def cosine_alpha_bar(t, T, s=0.008):
    """Cosine schedule from Improved DDPM."""
    f_t = math.cos(((t / T) + s) / (1 + s) * math.pi / 2) ** 2
    f_0 = math.cos((s / (1 + s)) * math.pi / 2) ** 2
    return f_t / f_0

It helps to think about schedules through the signal-to-noise ratio (SNR): SNR(t) = ᾱₜ / (1 − ᾱₜ). A good schedule has SNR that drops from very high (clean) to very low (noise) in a smooth, monotonic curve. The log-SNR should decrease roughly linearly. Some recent work learns the schedule as part of training, parameterizing β as a small neural network whose output is constrained to be monotonically increasing.

The discrete T-step formulation can also be cast in continuous time (t ∈ [0, 1]), where the noise schedule becomes a continuous function σ(t). This is the natural setting for the SDE framework and lets you decouple the training noise distribution from the number of sampling steps.

Sampling Methods

DDPM sampling takes 1000 denoising steps. Each step runs a full forward pass through a large U-Net. On a modern GPU, that can mean 30+ seconds per image. For a product like Midjourney serving millions of requests, that is a non-starter. The race for faster sampling is one of the most active areas in diffusion research.

DDIM (Denoising Diffusion Implicit Models, Song et al. 2020) was the first big breakthrough. It reinterprets the reverse process as solving an ordinary differential equation (ODE) rather than an SDE. The key difference: DDIM is deterministic. Given the same starting noise, you always get the same image. Because it is an ODE, you can take bigger steps — skip from t=1000 to t=950 to t=900, using 50 or even 20 steps instead of 1000. Quality degrades gracefully, and at 50 steps, DDIM images are nearly as good as 1000-step DDPM images.

DPM-Solver (Lu et al. 2022) pushes this further. It applies high-order ODE solvers (analogous to Runge-Kutta methods) to the diffusion ODE. DPM-Solver++ can produce excellent results in 10-20 steps, making real-time diffusion applications feasible.

Progressive distillation takes a different approach: train a student model to match the output of two teacher steps in a single step. Repeat, and you halve the step count each round. After several rounds, an 8-step or even 4-step model matches the quality of the original 1000-step model. This is how some production systems achieve near-instant generation.

Classifier-Free Guidance

Generating random images is nice, but what you really want is to generate an image that matches a text prompt. The original approach, classifier guidance (Dhariwal & Nichol, 2021), trained a separate image classifier on noisy images, then used its gradients to steer the diffusion process toward a target class. It worked, but required training and maintaining a separate classifier.

Classifier-Free Guidance (CFG) is simpler and more powerful. During training, you randomly drop the conditioning information (e.g., the text prompt) some percentage of the time (typically 10-20%), replacing it with a null/empty token. This teaches the model to generate both conditionally and unconditionally. At inference time, you run the model twice per step — once with the prompt (getting ε_cond) and once without (getting ε_uncond) — then amplify the difference:

ε_guided = ε_uncond + w · (ε_cond - ε_uncond)

The guidance scale w controls the tradeoff. At w=1, you get plain conditional generation. At w=7-15 (typical for Stable Diffusion), the model exaggerates the features that distinguish "conditioned" from "unconditioned," producing images that strongly match the prompt but with less diversity. At w=30+, images become oversaturated and artifacted — the model is trying too hard.

@torch.no_grad()
def sample_with_cfg(model, prompt_emb, null_emb, shape, device, w=7.5):
    """Sampling loop with classifier-free guidance."""
    x = torch.randn(shape, device=device)
    for t in reversed(range(T)):
        t_batch = torch.full((shape[0],), t, device=device, dtype=torch.long)
        # Two forward passes: conditioned and unconditioned
        eps_cond = model(x, t_batch, cond=prompt_emb)
        eps_uncond = model(x, t_batch, cond=null_emb)
        # Guided noise prediction
        eps_guided = eps_uncond + w * (eps_cond - eps_uncond)
        # Standard DDPM/DDIM update using eps_guided
        alpha_t = alphas[t]
        ab_t = alpha_bar[t]
        x = (1 / torch.sqrt(alpha_t)) * (
            x - (betas[t] / torch.sqrt(1 - ab_t)) * eps_guided
        )
        if t > 0:
            x += torch.sqrt(betas[t]) * torch.randn_like(x)
    return x
Negative prompting is a direct consequence of CFG. Instead of using an empty/null embedding for ε_uncond, you substitute the embedding of a "negative prompt" — things you want to avoid. The guidance formula then pushes the output away from the negative prompt and toward the positive one. That is why typing "blurry, low quality" as a negative prompt sharpens outputs.

Latent Diffusion Models

There is a scaling problem. A 512×512 RGB image has 786,432 dimensions. Running a thousand denoising steps through a U-Net in that space is brutally expensive — both in compute and memory. This is what kept diffusion models in the research lab while GANs were already shipping in products.

Latent Diffusion Models (LDM) (Rombach et al. 2022) solve this elegantly. The idea: do not diffuse in pixel space. First, train a separate autoencoder (specifically a VQ-VAE or KL-regularized VAE) that compresses images into a much smaller latent space. A 512×512×3 image becomes, say, a 64×64×4 latent tensor — a 48× reduction in dimensionality. Then run the entire diffusion process in that compact latent space.

Latent Diffusion Architecture:

    Text Prompt
        │
        ▼
  ┌──────────────┐
  │ CLIP Text     │
  │ Encoder       │ ──→ text embeddings (77 × 768)
  └──────────────┘
                           │
                           │ cross-attention
                           ▼
  z_T (noise) ──→ ┌──────────────────┐ ──→ z_0 (clean latent)
                   │   U-Net Denoiser  │
                   │  (in latent space) │
                   └──────────────────┘
                                             │
                                             ▼
                                    ┌─────────────────┐
                                    │ VAE Decoder       │ ──→ 512×512 image
                                    └─────────────────┘

The U-Net backbone is the workhorse. It follows the classic encoder-decoder pattern with skip connections, but adds self-attention layers at lower resolutions (32×32, 16×16, 8×8) and cross-attention layers that attend to the text embeddings. At each cross-attention layer, the image features form the queries, and the text embeddings form the keys and values. This is how language conditions image generation — the text literally tells each spatial location what to produce.

U-Net Architecture for Noise Prediction:

  Input: z_t (64×64×4) + timestep t + text embedding

  ┌─────────────────────────────────────────────┐
  │ Encoder Path           Decoder Path          │
  │                                               │
  │ 64×64 ─── ResBlock ──────── skip ──→ ResBlock │
  │           ↓ downsample              ↑ upsample│
  │ 32×32 ─── ResBlock+Attn ─── skip ──→ Res+Attn│
  │           ↓ downsample              ↑ upsample│
  │ 16×16 ─── ResBlock+Attn ─── skip ──→ Res+Attn│
  │           ↓ downsample              ↑ upsample│
  │  8×8  ─── ResBlock+CrossAttn ──────→ Res+CA  │
  │                    │                          │
  │              ┌─────┴──────┐                   │
  │              │  Bottleneck │                   │
  │              │  Res+Attn   │                   │
  │              └────────────┘                   │
  └─────────────────────────────────────────────┘

  Output: predicted noise ε (64×64×4)

  Attn = Self-Attention    CA/CrossAttn = Cross-Attention to text

The VAE encoder and decoder are trained separately and frozen during diffusion training. This two-stage approach has a huge benefit: the diffusion model never touches pixels directly, so it is fast. And because the VAE is trained for perceptual reconstruction (using a perceptual loss and a patch discriminator), the latent space captures semantically meaningful features rather than raw pixel values.

Stable Diffusion

Stable Diffusion is Latent Diffusion made concrete and released to the world. It was developed by CompVis, Stability AI, and Runway, and its open release in August 2022 democratized high-quality image generation overnight. Let's trace the full pipeline.

The text encoder is OpenAI's CLIP ViT-L/14. Your text prompt gets tokenized into at most 77 tokens, each embedded into 768-dimensional vectors. These 77×768 embeddings are what the U-Net cross-attends to. CLIP was trained on 400 million image-text pairs, so its text embeddings already live in a space aligned with visual concepts.

The U-Net denoiser operates at 64×64×4 latent resolution. It has about 860 million parameters in SD v1.5. Self-attention and cross-attention layers appear at the 32×32, 16×16, and 8×8 resolution levels. Timestep conditioning enters through adaptive group normalization (each ResBlock scales and shifts its features based on the timestep embedding).

The VAE compresses 512×512×3 images to 64×64×4 latents with a downsampling factor of 8. Decoding takes the denoised latent and produces the final image. The KL penalty during VAE training keeps the latent distribution close to a standard normal, which is exactly what the diffusion process assumes.

Img2Img and Inpainting

The Img2Img pipeline starts from an existing image instead of pure noise. You encode the input image into latent space, add noise to a chosen timestep (controlled by a "strength" parameter, 0.0 to 1.0), then denoise with text guidance. Low strength gives results close to the input. High strength gives the model more creative freedom. Inpainting extends this: you provide a mask, and the model only denoises the masked region while keeping the unmasked pixels fixed. The U-Net receives the mask and the masked image as additional input channels.

ControlNet

ControlNet (Zhang et al. 2023) adds spatial conditioning beyond text. Want to generate an image that follows a specific pose, depth map, edge map, or segmentation layout? ControlNet clones the encoder half of the U-Net, trains the clone on paired data (e.g., edge maps → images), and connects it back to the frozen original via zero-initialized convolution layers. The frozen model retains all its generative knowledge, while the trainable copy learns to incorporate the new spatial signal. Because the connection layers start at zero, training begins from a working model rather than random weights.

Personalization: Making Diffusion Your Own

Textual Inversion teaches the model a new concept by optimizing a single text embedding vector. You provide 3-5 images of your subject (say, your dog), and the method finds a new token embedding v* such that "a photo of v*" reconstructs your images. The U-Net stays frozen. Only the embedding is learned. This is lightweight — a few KB — but limited in expressiveness.

DreamBooth fine-tunes the entire U-Net on your subject images, using a special identifier token (like "sks") and a prior preservation loss that prevents the model from forgetting other concepts. Results are much more faithful than Textual Inversion, but you get a full model copy per concept (several GB).

LoRA (Low-Rank Adaptation) offers the sweet spot. Instead of fine-tuning the full weight matrices of the U-Net, you insert small low-rank decomposition matrices (rank 4-64) alongside each attention layer. Only these tiny matrices are trained. A LoRA checkpoint is typically 10-100 MB — small enough to swap at inference time. You can even combine multiple LoRAs (one for style, one for a character) with weighted merging. This is why the community has built massive libraries of LoRA adapters for every conceivable style and subject.

SDXL and Beyond

SDXL (Stability AI, 2023) scales up the architecture significantly. It uses two text encoders (CLIP ViT-L and OpenCLIP ViT-bigG), a 2.6B parameter U-Net (3× larger than SD 1.5), and generates at 1024×1024 base resolution. It also introduces a refiner model — a second diffusion model specialized for the final denoising steps, sharpening fine details. SDXL adds conditioning on the original image size and crop coordinates during training, which eliminates the artifacts that plagued earlier models when training data contained small or cropped images.

The practical takeaway. Diffusion models win on quality and training stability. The cost is inference speed — you need many forward passes to generate one image. But between DDIM, DPM-Solver, progressive distillation, and latent-space tricks, that gap is closing fast. Models like SDXL Turbo can generate quality images in 1-4 steps. The core loop — corrupt with noise, learn to denoise, steer with guidance — is the foundation of nearly every state-of-the-art image, video, and audio generation system shipping today.
