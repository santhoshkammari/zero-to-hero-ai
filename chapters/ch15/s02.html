Generative Adversarial Networks (GANs)

Chapter 15: Generative Models 8 subtopics
TL;DR

A GAN pits two neural networks against each other: a generator that forges fake data, and a discriminator that tries to catch it. This adversarial game drives both networks to improve until the generator produces outputs indistinguishable from real data. Variants like DCGAN add convolutions, WGAN fixes training instability with Wasserstein distance, Conditional GANs add controllability via labels, StyleGAN enables photorealistic face synthesis with fine-grained style control, CycleGAN handles unpaired image translation, and Pix2Pix handles paired translation. Training GANs remains tricky — mode collapse, vanishing gradients, and divergence are constant threats — but a toolkit of tricks (gradient penalty, spectral normalization, progressive training, TTUR) makes them practical.

Vanilla GAN: The Counterfeiter vs. the Detective

Imagine a counterfeiter trying to print fake banknotes and a detective trying to spot them. Every time the detective catches a flaw, the counterfeiter improves. Every time the counterfeiter gets better, the detective sharpens their eye. Neither rests. That ongoing arms race is exactly what a Generative Adversarial Network does with neural networks.

A GAN has two players. The generator G takes random noise z (sampled from a simple distribution like a Gaussian) and produces a fake sample — an image, a sentence, an audio clip. The discriminator D takes a sample (either real from your dataset or fake from G) and outputs a probability: "How likely is it this sample is real?" G wants D to say "real" for fake samples. D wants to correctly classify real as real and fake as fake. They are locked in a minimax game.

  ┌──────────────┐
  │  Random Noise │  z ~ N(0, I)
  │    (latent)   │
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │  Generator G  │  G(z) → fake image
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────┐
  │     Discriminator D       │ ◄── Real images from dataset
  │  D(x) → P(real or fake)  │
  └──────────────────────────┘

The formal objective is the minimax value function:

min_G max_D  V(D, G) = E[log D(x)] + E[log(1 - D(G(z)))]

The first term says: for real data x, D should output values close to 1, making log D(x) close to 0 (its maximum). The second term says: for fake data G(z), D should output values close to 0, making log(1 - D(G(z))) close to 0. Meanwhile, G wants D(G(z)) close to 1, which makes log(1 - D(G(z))) very negative — minimizing V from G's perspective.

In practice, both networks use binary cross-entropy loss. D is trained as a binary classifier with label 1 for real and 0 for fake. G is trained to maximize D's output on fake samples — or equivalently, minimize -log(D(G(z))) instead of log(1 - D(G(z))), because the latter saturates early when G is still bad.

At the theoretical optimum — the Nash equilibrium — the generator produces data indistinguishable from the real distribution, and the discriminator outputs 0.5 for everything (it literally cannot tell). In practice we never quite get there, but the game pushes us close.

The training loop alternates between D steps and G steps. Here is a complete PyTorch implementation for MNIST:

import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

# --- Generator ---
class Generator(nn.Module):
    def __init__(self, latent_dim=100, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, 1024),
            nn.LeakyReLU(0.2),
            nn.Linear(1024, img_dim),
            nn.Tanh(),  # output in [-1, 1] to match normalized images
        )

    def forward(self, z):
        return self.net(z)

# --- Discriminator ---
class Discriminator(nn.Module):
    def __init__(self, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(img_dim, 1024),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(1024, 512),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(256, 1),
            nn.Sigmoid(),
        )

    def forward(self, x):
        return self.net(x)

# --- Training Setup ---
latent_dim = 100
lr = 0.0002
epochs = 100
batch_size = 64
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize([0.5], [0.5]),  # scale to [-1, 1]
])
dataset = datasets.MNIST(root="./data", train=True, transform=transform, download=True)
loader = DataLoader(dataset, batch_size=batch_size, shuffle=True)

G = Generator(latent_dim).to(device)
D = Discriminator().to(device)
opt_G = optim.Adam(G.parameters(), lr=lr, betas=(0.5, 0.999))
opt_D = optim.Adam(D.parameters(), lr=lr, betas=(0.5, 0.999))
criterion = nn.BCELoss()

# --- Training Loop ---
for epoch in range(epochs):
    for real_imgs, _ in loader:
        real_imgs = real_imgs.view(-1, 784).to(device)
        bs = real_imgs.size(0)
        real_labels = torch.ones(bs, 1, device=device)
        fake_labels = torch.zeros(bs, 1, device=device)

        # ---- Train Discriminator ----
        z = torch.randn(bs, latent_dim, device=device)
        fake_imgs = G(z).detach()  # detach so G doesn't get gradients here

        loss_real = criterion(D(real_imgs), real_labels)
        loss_fake = criterion(D(fake_imgs), fake_labels)
        loss_D = (loss_real + loss_fake) / 2

        opt_D.zero_grad()
        loss_D.backward()
        opt_D.step()

        # ---- Train Generator ----
        z = torch.randn(bs, latent_dim, device=device)
        fake_imgs = G(z)
        loss_G = criterion(D(fake_imgs), real_labels)  # G wants D to say "real"

        opt_G.zero_grad()
        loss_G.backward()
        opt_G.step()

    print(f"Epoch {epoch}: D_loss={loss_D:.4f}, G_loss={loss_G:.4f}")

Notice a few things. We use detach() on fake images when training D — otherwise gradients would flow back into G during D's optimization step. We use Adam with beta1=0.5 instead of the default 0.9, which is a well-known trick for GAN stability. And G's loss uses real_labels (ones), because G wants D to think its fakes are real.

DCGAN: Adding Convolutions

The vanilla GAN above uses fully-connected layers, which means it flattens images into vectors and throws away all spatial structure. Deep Convolutional GAN (DCGAN) replaces those linear layers with convolutions, and the results leap from blurry blobs to recognizable faces.

The generator uses transposed convolutions (sometimes misleadingly called "deconvolutions") to upsample from a small spatial feature map to a full-size image. Think of a transposed convolution as the reverse of a regular convolution: where a conv shrinks a feature map, a transposed conv expands it. The discriminator uses regular strided convolutions to downsample.

Generator (DCGAN):
  z (100,)
    → Reshape to (512, 4, 4)
    → ConvTranspose2d → (256, 8, 8)    + BatchNorm + ReLU
    → ConvTranspose2d → (128, 16, 16)  + BatchNorm + ReLU
    → ConvTranspose2d → (64, 32, 32)   + BatchNorm + ReLU
    → ConvTranspose2d → (3, 64, 64)    + Tanh

Discriminator (DCGAN):
  image (3, 64, 64)
    → Conv2d → (64, 32, 32)    + LeakyReLU(0.2)
    → Conv2d → (128, 16, 16)   + BatchNorm + LeakyReLU(0.2)
    → Conv2d → (256, 8, 8)     + BatchNorm + LeakyReLU(0.2)
    → Conv2d → (512, 4, 4)     + BatchNorm + LeakyReLU(0.2)
    → Conv2d → (1, 1, 1)       + Sigmoid

The DCGAN paper established architecture guidelines that became gospel for years. No pooling layers — use strided convolutions for downsampling and transposed convolutions for upsampling. Use batch normalization in both G and D (except D's input layer and G's output layer). Use ReLU in the generator, LeakyReLU in the discriminator. Use Tanh for G's output, Sigmoid for D's output.

One of DCGAN's coolest discoveries was latent space arithmetic. If you find the average z vector for "man with glasses," subtract the average for "man without glasses," and add the average for "woman without glasses," the resulting image is a woman with glasses. The latent space learned meaningful, linear directions for semantic attributes — entirely unsupervised.

z_woman_glasses = z_man_glasses - z_man + z_woman
# Generate: G(z_woman_glasses) → image of woman wearing glasses

WGAN: Fixing the Loss

Vanilla GANs have a dirty secret. When the discriminator gets too good — which happens fast because classification is easier than generation — the loss for G goes flat. The gradients vanish and G stops learning. You are stuck in a situation where D says "everything you make is fake, probability zero" and there is no gradient signal to tell G how to improve.

The Wasserstein GAN (WGAN) fixes this by changing the loss function entirely. Instead of measuring how well D classifies real vs. fake, it measures the Wasserstein distance (also called the Earth Mover's Distance) between the real and generated distributions.

Here is the intuition. Imagine the real data distribution as a pile of dirt, and the generated distribution as another pile. The Earth Mover's Distance is the minimum total work needed to reshape one pile into the other — the cost of moving dirt multiplied by the distance you move it. Even when the two piles don't overlap at all (which causes KL divergence to blow up to infinity), the EMD gives you a smooth, finite distance and a useful gradient.

In WGAN, the discriminator is renamed the critic because it no longer outputs a probability. It removes the sigmoid and outputs an unbounded scalar score. Higher score = more real-looking. The loss is:

L_critic = E[critic(fake)] - E[critic(real)]     # critic minimizes this
L_gen    = -E[critic(fake)]                        # generator minimizes this

For this to work, the critic must be a Lipschitz function — its output cannot change faster than some constant K times its input change. The original WGAN enforced this crudely via weight clipping: after each update, clip all critic weights to a small range like [-0.01, 0.01]. This works but causes problems — the critic tends to learn very simple functions and wastes capacity, or the clipping value is too aggressive.

WGAN-GP (gradient penalty) is the cleaner solution. Instead of clipping weights, you add a penalty term that pushes the gradient norm of the critic toward 1, evaluated at random interpolations between real and fake samples:

# Gradient penalty for WGAN-GP
alpha = torch.rand(batch_size, 1, 1, 1, device=device)
interpolated = alpha * real + (1 - alpha) * fake
interpolated.requires_grad_(True)
critic_interp = critic(interpolated)

gradients = torch.autograd.grad(
    outputs=critic_interp, inputs=interpolated,
    grad_outputs=torch.ones_like(critic_interp),
    create_graph=True
)[0]
grad_norm = gradients.view(batch_size, -1).norm(2, dim=1)
gp = ((grad_norm - 1) ** 2).mean()

loss_critic = critic(fake).mean() - critic(real).mean() + 10 * gp

The lambda=10 for the gradient penalty is remarkably robust — it works well across many settings.

Another approach to enforcing the Lipschitz constraint is spectral normalization. You normalize each weight matrix by its largest singular value (its spectral norm), guaranteeing the Lipschitz constant of each layer is at most 1. It is cheaper than computing gradient penalties and has become the default in many modern GAN architectures.

# PyTorch has built-in spectral normalization
from torch.nn.utils import spectral_norm
layer = spectral_norm(nn.Conv2d(64, 128, 3, padding=1))

Conditional GAN: Controlling the Output

A vanilla GAN generates random samples — you have no control over what comes out. Want a specific digit? Too bad, spin the roulette wheel. A Conditional GAN (cGAN) fixes this by feeding a class label (or any conditioning information) to both G and D.

The generator receives the noise z concatenated with a label embedding. So instead of G(z), you get G(z, y) where y might be "digit 7" or "blonde hair." The discriminator also sees the label: D(x, y). Now D's job is not "is this a real image?" but "is this a real image of class y?" This forces G to produce samples that match the requested class.

class ConditionalGenerator(nn.Module):
    def __init__(self, latent_dim=100, n_classes=10, embed_dim=50, img_dim=784):
        super().__init__()
        self.label_embed = nn.Embedding(n_classes, embed_dim)
        self.net = nn.Sequential(
            nn.Linear(latent_dim + embed_dim, 256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, img_dim),
            nn.Tanh(),
        )

    def forward(self, z, labels):
        label_emb = self.label_embed(labels)
        x = torch.cat([z, label_emb], dim=1)
        return self.net(x)

AC-GAN (Auxiliary Classifier GAN) takes a different approach. Instead of feeding labels to D, the discriminator has two output heads: one for real/fake and one for class prediction. D must classify the image into the correct class on top of detecting fakes. This pushes G to produce class-distinctive features.

The projection discriminator is a more elegant technique. Instead of concatenating the label embedding with intermediate features, you take the inner product between D's learned feature vector and the label embedding. This has better theoretical properties and became the standard approach in BigGAN and other large-scale conditional GANs.

StyleGAN: Photorealistic Face Synthesis

StyleGAN didn't appear overnight — it emerged from NVIDIA's research group, and it completely changed how we think about controlling generated images. When you saw those eerily realistic faces on "This Person Does Not Exist," that was StyleGAN.

The key insight is separating what to generate from how to generate it. Instead of feeding the latent code z directly into the generator, StyleGAN introduces a mapping network: an 8-layer MLP that transforms z into an intermediate latent space w. This disentangles the factors of variation. In z-space, features are tangled — changing one dimension might alter both hair color and face shape. In w-space, each dimension tends to control one attribute more independently.

StyleGAN Architecture:
  z (512,)
    → Mapping Network (8 FC layers) → w (512,)
    → Per-layer style vectors: w → affine → (scale, shift)
    
  Synthesis Network:
    Constant 4×4 input
      → Style injection via AdaIN → Conv → Noise → 4×4
      → Upsample → Style → Conv → Noise → 8×8
      → Upsample → Style → Conv → Noise → 16×16
      → ...
      → Upsample → Style → Conv → Noise → 1024×1024
      → To RGB

The w vector doesn't feed into the network as input. Instead, at each layer, w is transformed into scale and shift parameters through an affine transformation, then applied via Adaptive Instance Normalization (AdaIN). AdaIN first normalizes the feature maps to zero mean and unit variance, then rescales them using the style parameters. Styles injected at early layers (low resolution) control coarse features like pose and face shape. Styles at later layers (high resolution) control fine details like hair texture and skin pores.

Stochastic variation is another trick. Random noise is injected after each convolution, but it only affects fine-grained, spatially varying details — freckles, exact hair placement, skin texture. The noise does not change the identity or pose because those are controlled by the style inputs. This clean separation is what makes StyleGAN outputs so photorealistic.

Style mixing — using one w for early layers and a different w for late layers — lets you combine the coarse structure of one face with the fine details of another. This acts as a regularizer during training (called mixing regularization) and as a creative tool during inference.

StyleGAN2 made several improvements. It removed progressive growing (which caused artifacts at resolution boundaries) and replaced AdaIN with weight demodulation, which achieves the same style modulation by modifying the convolution weights directly rather than normalizing activations. This eliminated the characteristic "water droplet" artifacts. It also introduced path length regularization to encourage smoother mappings from w to images.

StyleGAN3 (alias-free) tackled a subtle problem: the generator was "stuck" to pixel coordinates. Small changes in the latent code would cause texture features to snap to pixel boundaries rather than moving smoothly. StyleGAN3 redesigned the architecture to treat the signal as continuous, enabling smooth equivariance to translation and rotation.

CycleGAN: Unpaired Image Translation

Here is a problem that sounds impossible. You have a collection of horse photos and a separate collection of zebra photos. No horse-zebra pairs — they are completely unmatched. Can you train a network to turn any horse into a zebra? CycleGAN says yes.

The idea rests on cycle consistency. You train two generators: G_AB (horses → zebras) and G_BA (zebras → horses). The key constraint is: if you translate a horse to a zebra and then translate it back, you should get the original horse.

  Horse x ──→ G_AB ──→ Fake Zebra ──→ G_BA ──→ Reconstructed Horse ≈ x
  Zebra y ──→ G_BA ──→ Fake Horse ──→ G_AB ──→ Reconstructed Zebra ≈ y

  Cycle consistency loss = ||G_BA(G_AB(x)) - x||₁ + ||G_AB(G_BA(y)) - y||₁

Each direction also has its own discriminator. D_A distinguishes real horses from fake horses (produced by G_BA). D_B distinguishes real zebras from fake zebras (produced by G_AB). So the total loss combines adversarial losses for both directions plus the cycle consistency loss.

An additional identity loss helps preserve color: if you feed a zebra into G_AB (which is supposed to produce zebras), the output should be unchanged. This prevents the generator from making unnecessary color shifts.

identity_loss = ||G_AB(y) - y||₁ + ||G_BA(x) - x||₁

CycleGAN works for artistic style transfer (photos → Monet paintings), season transfer (summer → winter), and even domain adaptation in medical imaging. The limitation is that it can only handle appearance changes, not geometric transformations — it won't reshape a horse's body into a zebra's body, it will change the texture.

Pix2Pix: Paired Image Translation

When you do have paired data — every satellite image matched with its map, every sketch matched with its photo — you can do much better than CycleGAN. That is Pix2Pix.

The generator is a U-Net: an encoder-decoder with skip connections that copy feature maps from the encoder directly to the matching decoder level. This is critical because image translation tasks need both high-level understanding (what objects are there) and low-level precision (exact edges and colors). The skip connections carry the fine details that the bottleneck would otherwise destroy.

Pix2Pix U-Net Generator:
  Input (256×256)
    Encoder:  ↓64 → ↓128 → ↓256 → ↓512 → ↓512 → ↓512 → ↓512 → ↓512
    Decoder:  ↑512 → ↑512 → ↑512 → ↑512 → ↑256 → ↑128 → ↑64
              (each level concatenated with encoder's skip connection)
    Output: 3 channels, Tanh

The discriminator is the clever part: a PatchGAN. Instead of one scalar "real or fake" for the whole image, the PatchGAN outputs a grid of predictions — each cell classifying whether a 70×70 patch is real or fake. This pushes the generator to produce realistic textures at the local patch level, while the L1 loss handles global structure.

total_loss_G = adversarial_loss(D(fake, input), real_label) + λ * L1_loss(fake, target)
# λ = 100 in the original paper — heavy emphasis on reconstruction

The L1 loss (mean absolute error) is preferred over L2 because L2 tends to produce blurry averages, while L1 preserves sharper edges. The combination of adversarial loss for realism and L1 for faithfulness to the target is what makes Pix2Pix outputs both sharp and accurate.

GAN Training Stability

If you have trained a GAN, you know the pain. The loss curves jump around chaotically. The images look great at epoch 50, then collapse into garbage at epoch 60. The generator discovers one face and produces it a thousand times. Welcome to the hardest part of GANs.

Mode Collapse

Mode collapse is the most common failure mode. The real data distribution has many modes — many types of faces, many digits, many objects. Mode collapse is when the generator finds one output that fools the discriminator and keeps producing that output (or small variations of it), ignoring the rest of the distribution. In MNIST, your GAN might produce beautiful 7s and nothing else.

Why does this happen? The generator is optimizing for D's current state. If producing a particular image always gets high D scores, G will converge on it. D eventually catches on and rejects it, but G might then latch onto a different single mode rather than learning diversity.

Training Divergence and Vanishing Gradients

If D becomes too strong, it classifies everything perfectly with high confidence. The gradient of log(1 - D(G(z))) becomes nearly zero, and G receives no learning signal. If G becomes too strong, D's loss diverges. The delicate balance is hard to maintain.

Practical Tricks

Label smoothing: Instead of training D with hard labels (1 for real, 0 for fake), use soft labels (0.9 for real, 0.1 for fake). This prevents D from becoming overconfident and keeps gradients flowing.

TTUR (Two Time-Scale Update Rule): Use a higher learning rate for D than for G. The theoretical justification is that D needs to be closer to optimal for G's gradients to be meaningful. A common setting is lr_D = 0.0004, lr_G = 0.0001.

Minibatch discrimination: Add a layer to D that computes statistics across the batch. If all generated samples look identical (mode collapse), minibatch discrimination detects this and penalizes it.

Unrolled GANs: Instead of optimizing G against D's current state, unroll a few steps of D's optimization and optimize G against the future D. This prevents G from exploiting D's current weaknesses.

Alternative Loss Functions

LSGAN (Least Squares GAN) replaces binary cross-entropy with mean squared error. D outputs an unbounded score, and the targets are 1 for real and 0 for fake. This avoids the vanishing gradient problem when D is confident and produces more stable training.

# LSGAN losses
loss_D = 0.5 * ((D(real) - 1)**2).mean() + 0.5 * (D(fake)**2).mean()
loss_G = 0.5 * ((D(fake) - 1)**2).mean()

Hinge loss is popular in large-scale GANs like BigGAN and SAGAN:

# Hinge loss
loss_D = -torch.min(D(real) - 1, zero).mean() - torch.min(-D(fake) - 1, zero).mean()
# Equivalently:
loss_D = torch.relu(1 - D(real)).mean() + torch.relu(1 + D(fake)).mean()
loss_G = -D(fake).mean()

GAN Inversion

GAN inversion is the reverse problem: given a real image, find the latent code z (or w) that the generator would map to that image. This enables real image editing with a pre-trained GAN — find the latent code, modify it, regenerate. Methods include direct optimization of z, training an encoder network, or hybrid approaches. StyleGAN's w-space is particularly amenable to inversion because of its disentangled structure.

Evaluation Metrics

How do you know your GAN is good? You cannot trust the loss — it oscillates and doesn't correlate well with image quality. The field developed dedicated metrics.

Fréchet Inception Distance (FID) is the gold standard. You pass real images and generated images through a pre-trained Inception network, extract features, compute the mean and covariance of both feature distributions, and measure the Fréchet distance between them. Lower FID = the generated distribution is closer to the real distribution. FID captures both quality (are individual images good?) and diversity (does the generator cover the full distribution?).

FID = ||μ_real - μ_gen||² + Tr(Σ_real + Σ_gen - 2(Σ_real · Σ_gen)^(1/2))

Inception Score (IS) measures two things: each generated image should be clearly classifiable (low entropy of class prediction), but the overall set should cover many classes (high entropy of marginal class distribution). IS = exp(E[KL(p(y|x) || p(y))]). Higher is better. The limitation is that IS ignores whether images look like real data — it rewards diversity and clarity but not fidelity.

LPIPS (Learned Perceptual Image Patch Similarity) measures perceptual distance between two images by comparing deep features. It is used less as a standalone GAN metric and more for evaluating image reconstruction quality, but it is useful for assessing mode collapse — if all generated samples have low LPIPS distance to each other, the generator lacks diversity.

Key takeaway: GANs are powerful but finicky. The choice of architecture (DCGAN, StyleGAN), loss function (vanilla, Wasserstein, hinge), conditioning mechanism (cGAN, AC-GAN, projection), and stabilization tricks (gradient penalty, spectral normalization, TTUR) all interact. Modern practice has converged on spectral normalization + hinge loss + projection discriminator as a reliable starting point. For image generation specifically, diffusion models have recently surpassed GANs in quality and training stability, but GANs remain faster at inference and dominate real-time applications like video synthesis and interactive editing.
