Autoencoders & VAEs

The core idea is embarrassingly simple: compress input into a small latent vector, then reconstruct the input from that vector. The network that compresses is the encoder. The one that reconstructs is the decoder. The small vector in the middle—the bottleneck—is where all the interesting stuff happens. That bottleneck is representation learning: the network is forced to discover the most informative features just to survive the compression.

Standard Autoencoder

Take an input x of dimension d. Map it through an encoder f to a latent code z of dimension k (where k << d). Map z through a decoder g back to x̂ of dimension d. Minimize the reconstruction loss—typically MSE for continuous data, binary cross-entropy for images normalized to [0, 1].

The bottleneck is what makes this non-trivial. Without it, the network could learn the identity function and call it a day. With it, the encoder must figure out which features actually matter. Think of it as a lossy compression algorithm where the compression strategy is learned from data.

If z has the same dimensionality as x, or the encoder/decoder are too powerful, you get a useless identity mapping. The tension between limited capacity and reconstruction quality is what forces useful representations.

Autoencoders vs. PCA: A single-layer linear autoencoder with MSE loss learns the same subspace as PCA. The deep, nonlinear version learns richer manifolds—same spirit, more expressive.

Denoising Autoencoders

Corrupt the input (Gaussian noise, dropout/masking, salt-and-pepper) and train the autoencoder to reconstruct the clean original. This prevents the network from memorizing inputs and forces it to learn robust features that capture the underlying data distribution, not surface noise.

The idea was ahead of its time—BERT's masked language modeling is essentially the same trick applied to text. Corrupt some tokens, predict the originals, learn great representations as a side effect.

Variational Autoencoders (VAEs)

Standard autoencoders give you an encoder and decoder, but the latent space is a mess. Points are scattered arbitrarily—there's no structure, and you can't meaningfully sample from it to generate new data. This is where VAEs change the game.

The Key Idea

Instead of encoding x to a single point z, encode it to a distribution over z. Specifically, the encoder outputs parameters of a Gaussian: a mean vector μ and a log-variance vector log σ². Then you sample from that distribution to get z, and decode as before.

The magic: you add a KL divergence term to the loss that pushes every encoding distribution toward a standard normal N(0, I). This regularizes the latent space into something smooth and continuous—nearby points decode to similar outputs, and you can sample z ~ N(0, I) at generation time and get coherent outputs.

The Reparameterization Trick

You can't backpropagate through a random sampling operation. The trick: instead of sampling z ~ N(μ, σ²), sample ε ~ N(0, I) and compute z = μ + σ ⊙ ε. The randomness is now in ε (which doesn't depend on parameters), and z is a deterministic, differentiable function of μ and σ. Gradients flow cleanly.

ELBO Loss

The VAE optimizes the Evidence Lower Bound (ELBO):

Loss = Reconstruction Loss + KL Divergence

L(θ, φ; x) = −𝔼qφ(z|x)[log pθ(x|z)] + DKL(qφ(z|x) ∥ p(z))

The first term says "reconstruct well." The second says "keep the latent distributions close to the prior." For Gaussian encoder and standard normal prior, the KL term has a closed-form solution—no sampling needed:

DKL = −½ Σ(1 + log σ² − μ² − σ²)

In practice, these two terms fight each other. Cranking up reconstruction quality tends to spread encodings apart; the KL term pulls them together. Finding the right balance is the art of training VAEs.

Why a Smooth Latent Space Matters

Without KL regularization, the encoder can map different classes to tiny, isolated clusters with dead space between them. Sample from that dead space and you get garbage. The KL term forces overlap, continuity, and smoothness. You get:

  • Interpolation: Walk between two latent codes and get smooth, meaningful transitions in output space.
  • Sampling: Draw z ~ N(0, I) and the decoder produces plausible outputs.
  • Disentanglement: Individual latent dimensions can correspond to interpretable factors (with the right pressure).

VAE in PyTorch

import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20):
        super().__init__()
        # Encoder
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)
        # Decoder
        self.fc3 = nn.Linear(latent_dim, hidden_dim)
        self.fc4 = nn.Linear(hidden_dim, input_dim)

    def encode(self, x):
        h = F.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_logvar(h)

    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)          # σ = exp(0.5 * log σ²)
        eps = torch.randn_like(std)             # ε ~ N(0, I)
        return mu + std * eps                   # z = μ + σ ⊙ ε

    def decode(self, z):
        h = F.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x.view(-1, 784))
        z = self.reparameterize(mu, logvar)
        return self.decode(z), mu, logvar


def vae_loss(recon_x, x, mu, logvar):
    # Reconstruction: binary cross-entropy (per-element, summed)
    bce = F.binary_cross_entropy(recon_x, x.view(-1, 784), reduction='sum')
    # KL divergence: closed-form for Gaussian q vs N(0,I) prior
    kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return bce + kl


# Training loop sketch
model = VAE()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(num_epochs):
    for batch, _ in train_loader:
        recon, mu, logvar = model(batch)
        loss = vae_loss(recon, batch, mu, logvar)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

# Generation: sample from prior and decode
with torch.no_grad():
    z = torch.randn(64, 20)       # sample latent codes
    samples = model.decode(z)      # decode to image space

β-VAE: Disentanglement

The β-VAE (Higgins et al., 2017) introduces a single hyperparameter β that scales the KL term:

L = Reconstruction Loss + β · DKL(q(z|x) ∥ p(z))

When β > 1, you're applying stronger pressure to keep the latent space structured. The result: individual latent dimensions tend to capture independent factors of variation—one dimension for rotation, another for color, another for size. This is disentanglement, and it's genuinely useful for interpretable representations and controlled generation.

The tradeoff: higher β means blurrier reconstructions. You're squeezing more structure out of the latent space at the cost of reconstruction fidelity. In practice, β between 1 and 10 works well depending on the dataset.

VQ-VAE: Discrete Latent Codes

Vector-Quantized VAE (van den Oord et al., 2017) replaces the continuous Gaussian latent space with a discrete codebook. The encoder outputs a continuous vector, which gets snapped to the nearest entry in a learned codebook of K embedding vectors.

Why go discrete? Three reasons:

  • Avoids posterior collapse: VAEs sometimes learn to ignore the latent code entirely (the decoder becomes so powerful it doesn't need z). Discrete codes are harder to ignore.
  • Better for sequences: You can train an autoregressive prior (like a Transformer) over the discrete codes, then sample from that prior to generate. This is exactly how DALL·E 1 worked—VQ-VAE encodes images to discrete tokens, then GPT generates those tokens.
  • Natural for compression: Discrete codes map directly to codebook indices, giving you a genuine compression scheme.

The quantization step isn't differentiable, so VQ-VAE uses a straight-through estimator: gradients from the decoder are passed directly to the encoder, bypassing the quantization. The codebook is updated via exponential moving average or an auxiliary loss.

Practical Applications

Anomaly Detection

Train the autoencoder exclusively on normal/healthy data. At inference, compute the reconstruction error for new inputs. Normal data reconstructs well; anomalies don't. Set a threshold on reconstruction error and you have an anomaly detector. This works remarkably well in manufacturing defect detection, network intrusion detection, and medical imaging.

Representation Learning

The encoder's latent space is a compressed, learned feature space. Use it as input to downstream classifiers, clustering, or retrieval systems. VAE latent spaces are particularly good here because the KL regularization ensures smooth, well-organized representations rather than arbitrary point clouds.

Data Generation

VAEs generate by sampling z ~ N(0, I) and decoding. The samples tend to be blurrier than GANs (because of the reconstruction loss averaging over modes) but VAEs offer something GANs don't: a proper latent space you can interpolate, manipulate, and reason about.

Use Case Variant Why It Works
Anomaly detection Standard AE / VAE High reconstruction error on OOD inputs
Image generation VAE / VQ-VAE Smooth latent space enables sampling
Disentangled features β-VAE KL pressure separates factors of variation
Discrete tokenization VQ-VAE Codebook creates discrete tokens for downstream models
Drug discovery VAE on molecular graphs Smooth latent space enables optimization of molecular properties
Data augmentation VAE / Conditional VAE Generate synthetic training examples with controlled attributes

The Autoencoder Family at a Glance

Variant Latent Space Key Innovation Generation?
Standard AE Continuous, unstructured Bottleneck forces compression No (messy latent space)
