Autoencoders

Chapter 15: Generative Models 6 subtopics
TL;DR

Autoencoders learn to compress data into a small latent code and then reconstruct it. The standard version gives you dimensionality reduction. Add noise and you get robust features. Make the latent space probabilistic (VAE) and you can generate entirely new data by sampling. Variants like β-VAE disentangle factors, VQ-VAE uses discrete codes for high-fidelity generation, and sparse/contractive versions add targeted regularization. Together, they form the foundation of modern generative modeling.

Standard Autoencoder

Here is a question that shows up everywhere in engineering: you have a 784-dimensional vector (a 28×28 MNIST image flattened), but most of those dimensions are redundant. Neighboring pixels are correlated. Backgrounds are mostly zeros. The actual information content — the identity of the digit — could fit in a much smaller space. How do you find that smaller space automatically?

An autoencoder is a neural network trained to copy its input to its output, but with a bottleneck in the middle that forces it to learn a compressed representation. It has two parts: an encoder that maps the input x to a latent code z, and a decoder that maps z back to a reconstruction x̂. The training objective is dead simple — minimize the difference between x and x̂.

  Input (784-d)
       │
  ┌────▼────┐
  │ Encoder  │   fc(784 → 256) → ReLU → fc(256 → 32)
  └────┬────┘
       │
   z (32-d)    ← bottleneck / latent code
       │
  ┌────▼────┐
  │ Decoder  │   fc(32 → 256) → ReLU → fc(256 → 784) → Sigmoid
  └────┬────┘
       │
  Output (784-d)  ≈ Input

The loss function is typically mean squared error (MSE): L = (1/n) Σ (xᵢ − x̂ᵢ)². For binary-ish data like MNIST you can also use binary cross-entropy, which often works better since pixel values are between 0 and 1.

Undercomplete vs Overcomplete. When the bottleneck dimension is smaller than the input, you have an undercomplete autoencoder — the network is forced to learn the most salient features because it cannot memorize everything. When the bottleneck is larger than the input, you have an overcomplete autoencoder. An overcomplete autoencoder can trivially learn the identity function (just pass everything through), so you need additional regularization — sparsity, noise, or some other constraint — to make it learn anything useful.

Here is a complete PyTorch autoencoder for MNIST:

import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

class Autoencoder(nn.Module):
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim),
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, input_dim),
            nn.Sigmoid(),
        )

    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z)

# Training
transform = transforms.ToTensor()
train_set = datasets.MNIST('.', train=True, download=True, transform=transform)
loader = DataLoader(train_set, batch_size=256, shuffle=True)

model = Autoencoder()
optimizer = optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.MSELoss()

for epoch in range(20):
    total_loss = 0
    for imgs, _ in loader:
        x = imgs.view(-1, 784)
        x_hat = model(x)
        loss = criterion(x_hat, x)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    print(f"Epoch {epoch+1}, Loss: {total_loss/len(loader):.4f}")

After training, the 32-dimensional z vectors cluster by digit class — even though you never told the network about labels. That is the magic. The bottleneck forces it to discover the structure in the data on its own.

Denoising Autoencoder

A standard autoencoder can get lazy. If the capacity is high enough, it might memorize input patterns rather than learning generalizable features. The fix: corrupt the input on purpose, then ask the network to recover the clean version.

A denoising autoencoder (DAE) receives a corrupted input x̃ but is trained to reconstruct the original clean x. This forces the model to learn the underlying data distribution rather than surface-level pixel patterns. If you smear ink on a digit and the network can still tell you it is a 7, it must have learned what "7-ness" really means.

Common corruption strategies:

Gaussian noise — add x̃ = x + ε where ε ~ N(0, σ²). Good for continuous data. Masking noise — randomly zero out a fraction of input dimensions (say 50%). This is what BERT does to tokens, by the way. Salt-and-pepper noise — randomly set some pixels to 0 or 1. Simulates dead or saturated sensor pixels.

class DenoisingAutoencoder(nn.Module):
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim),
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, input_dim),
            nn.Sigmoid(),
        )

    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z)

def add_noise(x, noise_factor=0.3):
    noisy = x + noise_factor * torch.randn_like(x)
    return torch.clamp(noisy, 0.0, 1.0)

# Training loop
for epoch in range(20):
    for imgs, _ in loader:
        x_clean = imgs.view(-1, 784)
        x_noisy = add_noise(x_clean)
        x_hat = model(x_noisy)           # feed noisy
        loss = criterion(x_hat, x_clean)  # compare to clean
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

Connection to diffusion models. Denoising autoencoders are deeply connected to score matching — estimating the gradient of the log data density. When you train a DAE, the reconstruction error implicitly estimates this score function. This insight, formalized as denoising score matching, is one of the theoretical foundations of modern diffusion models like DDPM and Stable Diffusion. We will revisit this in a later section.

Variational Autoencoder (VAE)

Here is the fundamental problem with standard autoencoders as generative models: the latent space has no structure. If you take two latent codes z₁ and z₂ that correspond to a "3" and a "7", the point halfway between them might decode to garbage. There is nothing in the training objective that says nearby points in latent space should produce similar outputs. You cannot sample random z vectors and expect meaningful images.

A Variational Autoencoder (VAE) fixes this by making the latent space probabilistic. Instead of encoding x to a single point z, the encoder outputs the parameters of a distribution — a mean μ and a log-variance log σ² — and we sample z from that distribution. We also add a regularization term that pushes these distributions toward a standard normal N(0, I). The result: a smooth, continuous latent space you can actually sample from.

  Input x
     │
  ┌──▼──┐
  │  En  │──→ μ  (mean vector)
  │  co  │
  │  der │──→ log σ²  (log-variance vector)
  └──┬──┘
     │
     ▼
  z = μ + σ · ε    where ε ~ N(0, I)    ← reparameterization trick
     │
  ┌──▼──┐
  │ De   │
  │ co   │
  │ der  │
  └──┬──┘
     │
  Output x̂  ≈  x

The Reparameterization Trick

You cannot backpropagate through a random sampling operation. If z is drawn randomly from N(μ, σ²), there is no gradient path from the loss through z back to μ and σ. The reparameterization trick solves this by rewriting the sampling as z = μ + σ · ε, where ε ~ N(0, I) is a fixed random draw. Now μ and σ are deterministic functions of the input, the randomness is external (ε), and gradients flow cleanly through multiplication and addition.

The ELBO: Loss Function Intuition

The VAE loss has two terms. Think of them as competing pressures. The first is reconstruction loss — how well can the decoder recover x from z? This is the same MSE or BCE you had before. The second is KL divergence — how far is the encoder's distribution q(z|x) from the prior p(z) = N(0, I)?

Together they form the Evidence Lower Bound (ELBO):

L = E[log p(x|z)] − KL(q(z|x) || p(z))

We maximize this (or equivalently minimize the negative). The reconstruction term says "make accurate outputs." The KL term says "keep the latent distributions close to a standard normal." These two forces compete. If you only cared about reconstruction, you would make each q(z|x) a spike (σ → 0) at the optimal z. If you only cared about KL, you would ignore the data and set μ = 0, σ = 1 for everything. The beauty is in the balance.

For Gaussian encoder and standard normal prior, the KL term has a closed-form solution: KL = −0.5 · Σ(1 + log σ² − μ² − σ²). No sampling needed for this part.

Posterior Collapse

In practice, VAEs sometimes learn to ignore the latent code entirely. The decoder becomes so powerful that it generates reasonable outputs without looking at z, and the encoder collapses all q(z|x) to the prior N(0, I). This is called posterior collapse — the KL term goes to zero, which sounds great until you realize it means z carries no information about x.

Two popular fixes. KL annealing: start training with the KL weight at zero (pure reconstruction), then slowly ramp it up to 1 over many epochs. This gives the network time to learn useful latent codes before the KL penalty kicks in. Free bits: set a minimum KL contribution per latent dimension (say λ = 0.25 nats). If a dimension's KL is already below λ, do not penalize it further. This guarantees each dimension carries at least some information.

Complete VAE Implementation

import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=256, latent_dim=32):
        super().__init__()
        # Encoder
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)
        # Decoder
        self.fc3 = nn.Linear(latent_dim, hidden_dim)
        self.fc4 = nn.Linear(hidden_dim, input_dim)

    def encode(self, x):
        h = F.relu(self.fc1(x))
        return self.fc_mu(h), self.fc_logvar(h)

    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)      # σ = exp(0.5 · log σ²)
        eps = torch.randn_like(std)         # ε ~ N(0, I)
        return mu + std * eps               # z = μ + σ · ε

    def decode(self, z):
        h = F.relu(self.fc3(z))
        return torch.sigmoid(self.fc4(h))

    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        x_hat = self.decode(z)
        return x_hat, mu, logvar

def vae_loss(x, x_hat, mu, logvar):
    # Reconstruction: binary cross-entropy (sum over features)
    recon = F.binary_cross_entropy(x_hat, x, reduction='sum')
    # KL divergence: closed form for Gaussian q vs N(0,I) prior
    kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + kl

# Training
model = VAE()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(30):
    total_loss = 0
    for imgs, _ in loader:
        x = imgs.view(-1, 784)
        x_hat, mu, logvar = model(x)
        loss = vae_loss(x, x_hat, mu, logvar)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    print(f"Epoch {epoch+1}, Loss: {total_loss/len(loader.dataset):.2f}")

# Generate new images by sampling from the prior
with torch.no_grad():
    z = torch.randn(16, 32)         # sample from N(0, I)
    samples = model.decode(z)        # decode into images
    samples = samples.view(-1, 1, 28, 28)

Conditional VAE

A Conditional VAE (CVAE) adds label information to both the encoder and decoder. Want to generate specifically a "5"? Concatenate the one-hot label vector to both the encoder input and the decoder input. The model learns to separate content (what digit) from style (how it is written) in the latent space. The encoder becomes q(z|x, y) and the decoder becomes p(x|z, y).

VAE Extensions

Hierarchical VAEs stack multiple latent layers. NVAE (Vahdat & Koltun, 2020) uses a deep hierarchy of latent variables with residual connections and reaches image quality that rivals GANs. VDVAE pushes this further with very deep architectures. The idea: each level captures different scales of detail — coarse structure at the top, fine textures at the bottom.

Importance-Weighted Autoencoders (IWAE) use multiple samples from q(z|x) to get a tighter bound on log p(x). Instead of one z per data point, draw k samples and weight them by importance weights. More samples means a better approximation of the true evidence, which means better gradients.

VAEs have also found significant use in drug discovery. You encode molecular structures (SMILES strings or molecular graphs) into a continuous latent space, then optimize in that space for desired properties like binding affinity or solubility. Decode the optimized z back into a molecule. This approach, pioneered by Gómez-Bombarelli et al. (2018), lets you do gradient-based search over chemical space instead of brute-force enumeration.

β-VAE: Disentangled Representations

Imagine a latent space where one dimension controls rotation, another controls thickness, another controls the slant of a handwritten digit — and changing one does not affect the others. That is a disentangled representation, and it is incredibly useful. You could edit specific properties of generated images. You could generalize to new combinations you have never seen.

A β-VAE (Higgins et al., 2017) achieves this with a trivially simple change to the VAE loss: multiply the KL term by a hyperparameter β:

L = E[log p(x|z)] − β · KL(q(z|x) || p(z))

When β > 1, you are applying extra pressure to make the latent distribution match the isotropic Gaussian prior. This forces the model to use each latent dimension efficiently and independently — which is exactly what disentanglement means. Each dimension has to earn its keep.

The trade-off is real. Higher β gives better disentanglement but worse reconstruction quality. The model has less capacity to encode fine details because the KL penalty is so strong. In practice, you tune β between 1 and 10 depending on your application.

Evaluating disentanglement. Several metrics exist. The DCI metric (Disentanglement, Completeness, Informativeness) measures whether each latent dimension captures exactly one generative factor. The β-VAE metric trains a linear classifier to detect which factor was changed from pairs of latent codes. The FactorVAE metric is similar but uses a majority vote classifier, making it more robust to correlations between factors. None of these is perfect — measuring disentanglement rigorously remains an open problem.

VQ-VAE: Discrete Latent Codes

Continuous latent spaces are elegant, but sometimes you want discrete representations. Language is discrete. Musical notes are discrete. Object categories are discrete. A Vector Quantized VAE (VQ-VAE) (van den Oord et al., 2017) replaces the continuous bottleneck with a learned codebook of embedding vectors — a fixed-size dictionary of latent codes.

Here is how it works. The encoder produces a continuous vector ze(x). You then find the closest codebook entry: zq = argmin_k ||ze(x) − eₖ||. That discrete code index k is your latent representation, and the codebook vector eₖ goes to the decoder. Think of it like color quantization — you have millions of colors but you snap each pixel to the nearest entry in a 256-color palette.

  Input x
     │
  ┌──▼──┐
  │Encoder│ → ze (continuous)
  └──┬──┘
     │
     ▼
  ┌─────────────────────┐
  │     Codebook         │
  │  e₁  e₂  e₃ ... eK  │   Find nearest: k* = argmin ||ze - eₖ||
  └─────────┬───────────┘
            │
     zq = e_{k*}  (discrete → looked up)
            │
  ┌─────────▼──────────┐
  │      Decoder        │ → x̂
  └────────────────────┘

The Gradient Problem and Straight-Through Estimator

There is a problem: argmin is not differentiable. You cannot compute gradients through a nearest-neighbor lookup. The straight-through estimator handles this by copying the gradient from zq directly to ze during backpropagation. In the forward pass, zq = codebook_lookup(ze). In the backward pass, we pretend zq = ze and let gradients flow straight through. It sounds like a hack. It works remarkably well.

In PyTorch, the trick looks like this: zq = ze + (zq - ze).detach(). The forward pass uses zq. The backward pass sees ze because the detach kills the gradient through the discrete lookup.

Training Losses

VQ-VAE has three loss terms. First, the reconstruction loss between x and x̂. Second, the codebook loss: ||sg[ze] − eₖ||² moves the codebook vectors toward the encoder outputs (sg means stop-gradient — freeze the encoder for this term). Third, the commitment loss: β · ||ze − sg[eₖ]||² pulls the encoder outputs toward the codebook vectors (freeze the codebook this time). The commitment loss prevents the encoder from wildly jumping around and ignoring the codebook.

class VectorQuantizer(nn.Module):
    def __init__(self, num_embeddings=512, embedding_dim=64, commitment_cost=0.25):
        super().__init__()
        self.embedding = nn.Embedding(num_embeddings, embedding_dim)
        self.embedding.weight.data.uniform_(-1/num_embeddings, 1/num_embeddings)
        self.commitment_cost = commitment_cost

    def forward(self, ze):
        # ze shape: (B, D) — encoder output
        # Compute distances to all codebook entries
        distances = (
            ze.pow(2).sum(1, keepdim=True)
            - 2 * ze @ self.embedding.weight.t()
            + self.embedding.weight.pow(2).sum(1)
        )
        # Nearest codebook entry
        indices = distances.argmin(dim=1)
        zq = self.embedding(indices)

        # Losses
        codebook_loss = F.mse_loss(zq, ze.detach())    # move codebook → encoder
        commitment_loss = F.mse_loss(ze, zq.detach())   # move encoder → codebook
        vq_loss = codebook_loss + self.commitment_cost * commitment_loss

        # Straight-through estimator
        zq_st = ze + (zq - ze).detach()
        return zq_st, vq_loss, indices

VQ-VAE-2 and Applications

VQ-VAE-2 (Razavi et al., 2019) extends this with a hierarchical structure — a top-level codebook for global structure and a bottom-level codebook for local details. Combined with a powerful autoregressive prior (like PixelCNN) trained on the discrete codes, VQ-VAE-2 generates images competitive with BigGAN in quality.

The discrete latent space is particularly natural for audio generation. Jukebox (OpenAI, 2020) uses hierarchical VQ-VAEs to generate music with singing. The idea: compress raw audio into a small sequence of discrete tokens, then model that sequence autoregressively. This is also the backbone of neural audio codecs like SoundStream and EnCodec that power modern speech synthesis.

Autoencoder Variants

Sparse Autoencoder

Instead of restricting the bottleneck size, you can keep it large but force most neurons to stay inactive. A sparse autoencoder adds an L1 penalty on the latent activations: L = reconstruction_loss + λ · ||z||₁. The L1 norm pushes most components of z toward zero, so only a handful of neurons fire for any given input. Each input activates a different sparse subset, creating a distributed code that is both overcomplete and information-efficient. This is similar to how biological neurons exhibit sparse firing patterns.

Sparse autoencoders have recently become important for mechanistic interpretability — researchers train them on transformer activations to decompose the dense internal representations into sparse, human-interpretable features.

Contractive Autoencoder

A contractive autoencoder (CAE) adds a penalty on the Frobenius norm of the encoder's Jacobian matrix: L = reconstruction_loss + λ · ||∂h/∂x||²_F. In plain English: if you wiggle the input a tiny bit, the latent code should not change much. This makes the learned representation robust to small perturbations of the input — the encoder contracts the local neighborhood around each data point. It is mathematically related to denoising autoencoders; both resist noise, but through different mechanisms.

Autoencoders for Anomaly Detection

This is one of the most practical applications. Train an autoencoder on normal data only — healthy machine sounds, non-fraudulent transactions, clean medical scans. At test time, compute the reconstruction error. Normal data reconstructs well (low error) because the model has seen similar patterns. Anomalous data reconstructs poorly (high error) because it does not fit the learned manifold. Set a threshold on reconstruction error and you have an anomaly detector.

This approach is popular in manufacturing (detecting defective products from images), cybersecurity (detecting network intrusions), and healthcare (flagging unusual ECG patterns). The beauty is that you do not need labeled anomalies for training — you only need examples of normal.

The autoencoder family tree. Standard autoencoders give you compression. Denoising autoencoders give you robust features. VAEs give you a generative model with a smooth latent space. β-VAEs give you disentangled factors. VQ-VAEs give you discrete codes for high-fidelity generation. Sparse and contractive variants give you regularized, interpretable representations. Each variant answers a different question about the data, but they all share the same skeleton: encoder, bottleneck, decoder, reconstruction loss plus some form of regularization. Master that skeleton and the variants are all variations on a theme.
