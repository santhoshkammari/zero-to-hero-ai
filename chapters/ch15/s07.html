Audio & Music Generation

Chapter 15: Generative Models 5 subtopics
TL;DR

Audio generation has two hard problems: raw audio is absurdly high-dimensional (44,100 samples per second), and humans are perceptually sensitive to tiny glitches. Modern systems solve this by working in compressed representations. Neural audio codecs like EnCodec compress waveforms into discrete tokens. Autoregressive models (MusicLM, MusicGen) generate those tokens sequentially. Diffusion models operate on mel-spectrograms or latent spaces (AudioLDM) for text-to-speech and sound effects. Tortoise TTS and XTTS combine these approaches for voice cloning. The key insight across all methods: don't generate raw waveforms — generate compact representations and decode them.

Why Audio Is Hard

A single second of CD-quality audio contains 44,100 floating-point samples. A 3-minute song has nearly 8 million. Compare that to a 256×256 image — about 196,000 values. Audio is 40× larger per second of content, and temporal coherence matters at multiple scales: individual wave cycles (microseconds), phonemes (milliseconds), words (hundreds of milliseconds), and musical phrases (seconds).

The human auditory system is also unforgiving. A single popped sample in audio is immediately audible. A single wrong pixel in an image is invisible. This means audio generation needs both extreme temporal precision and long-range structure. No single architecture handles both scales well, so modern systems use a pipeline: compress audio to a compact representation, generate in that space, then decode back to a waveform.

Neural Audio Codecs

EnCodec and SoundStream

A neural audio codec compresses raw audio into a sequence of discrete tokens, similar to how a VQ-VAE tokenizes images. EnCodec (Meta, 2022) and SoundStream (Google, 2021) are the two dominant codecs. Both use an encoder-decoder architecture with Residual Vector Quantization (RVQ) in the bottleneck.

import torch
import torch.nn as nn

class ResidualVectorQuantizer(nn.Module):
    """
    RVQ: multiple codebook layers, each encoding the residual
    from the previous layer. This achieves variable bitrate —
    more codebooks = higher quality.
    """
    def __init__(self, n_codebooks=8, codebook_size=1024,
                 codebook_dim=128):
        super().__init__()
        self.n_codebooks = n_codebooks
        self.codebooks = nn.ModuleList([
            VectorQuantizer(codebook_size, codebook_dim)
            for _ in range(n_codebooks)
        ])

    def forward(self, z):
        """
        z: (batch, time, dim) continuous latent
        Returns: quantized, codes, commitment_loss
        """
        residual = z
        quantized = torch.zeros_like(z)
        all_codes = []
        total_loss = 0.0

        for i, vq in enumerate(self.codebooks):
            # Quantize the residual
            q_i, codes_i, loss_i = vq(residual)
            quantized = quantized + q_i
            residual = residual - q_i.detach()
            all_codes.append(codes_i)
            total_loss += loss_i

        # all_codes: list of (batch, time) integer tensors
        return quantized, all_codes, total_loss


class VectorQuantizer(nn.Module):
    """Single-codebook vector quantizer."""
    def __init__(self, n_codes=1024, dim=128):
        super().__init__()
        self.embedding = nn.Embedding(n_codes, dim)
        nn.init.uniform_(self.embedding.weight, -1/n_codes, 1/n_codes)

    def forward(self, z):
        # z: (batch, time, dim)
        # Find nearest codebook entry for each time step
        dists = torch.cdist(z, self.embedding.weight)  # (B, T, n_codes)
        codes = dists.argmin(dim=-1)                     # (B, T)
        quantized = self.embedding(codes)                # (B, T, dim)

        # Straight-through estimator for gradients
        commitment_loss = ((z.detach() - quantized) ** 2).mean() + \
                         ((z - quantized.detach()) ** 2).mean()
        quantized = z + (quantized - z).detach()

        return quantized, codes, commitment_loss

The key idea behind RVQ: the first codebook captures the coarse structure of the audio (pitch, rhythm). Each subsequent codebook captures finer residual details (timbre, texture). You can drop the last few codebooks for a lower bitrate with graceful degradation — the audio gets slightly muddier but doesn't break. EnCodec compresses 24kHz audio to 1.5–24 kbps (compared to 1,411 kbps for uncompressed CD audio), with perceptually transparent quality at 6 kbps using 8 codebook layers.

class EnCodecEncoder(nn.Module):
    """Simplified EnCodec encoder: waveform -> latent codes."""
    def __init__(self, in_channels=1, hidden=64, latent_dim=128,
                 strides=[8, 5, 4, 2]):
        super().__init__()
        layers = []
        ch = in_channels
        for s in strides:
            layers.extend([
                nn.Conv1d(ch, hidden, kernel_size=2*s, stride=s,
                          padding=s//2),
                nn.ELU(),
                # Residual blocks for capacity
                ResidualBlock1d(hidden),
                ResidualBlock1d(hidden),
            ])
            ch = hidden
            hidden *= 2
        layers.append(nn.Conv1d(ch, latent_dim, kernel_size=7,
                               padding=3))
        self.encoder = nn.Sequential(*layers)
        # Total downsampling: 8*5*4*2 = 320x
        # 24kHz input -> 75 latent frames/sec

    def forward(self, waveform):
        # waveform: (batch, 1, n_samples)
        return self.encoder(waveform)  # (batch, latent_dim, n_frames)


class ResidualBlock1d(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.block = nn.Sequential(
            nn.ELU(),
            nn.Conv1d(channels, channels, 3, padding=1),
            nn.ELU(),
            nn.Conv1d(channels, channels, 1)
        )
    def forward(self, x):
        return x + self.block(x)

The encoder downsamples by 320×, turning 24,000 samples per second into 75 latent frames per second. Each frame is quantized into a stack of codebook indices. This is the representation that downstream generative models work with — 75 discrete tokens per second instead of 24,000 continuous samples.

📊 Codec Comparison

EnCodec and SoundStream are architecturally similar (convolutional encoder-decoder + RVQ). The main differences are in training losses: EnCodec uses a multi-scale STFT discriminator plus a multi-period discriminator, while SoundStream introduced the multi-scale discriminator approach. Both achieve near-transparent quality at 6 kbps. DAC (Descript Audio Codec) later improved on both with better quantizer initialization and improved discriminator training.

Autoregressive Audio Generation

MusicLM

MusicLM (Google, 2023) generates music from text descriptions by treating music generation as a token prediction problem. It uses a hierarchical approach: first generate "semantic tokens" (high-level musical content from a MuLan audio embedding), then generate "acoustic tokens" (EnCodec/SoundStream codes) conditioned on the semantic tokens.

# MusicLM generation pipeline (conceptual)
class MusicLMPipeline:
    def __init__(self, semantic_model, acoustic_model, codec):
        self.semantic_model = semantic_model   # text -> semantic tokens
        self.acoustic_model = acoustic_model   # semantic -> acoustic tokens
        self.codec = codec                      # acoustic tokens -> audio

    def generate(self, text_prompt, duration_sec=30):
        # Stage 1: Generate semantic tokens from text
        # These capture high-level musical structure (melody, rhythm)
        text_embedding = self.encode_text(text_prompt)
        n_semantic = duration_sec * 25  # ~25 semantic tokens/sec
        semantic_tokens = self.semantic_model.generate(
            text_embedding, max_len=n_semantic)

        # Stage 2: Generate acoustic tokens from semantic tokens
        # These capture fine audio details (timbre, texture)
        acoustic_tokens = self.acoustic_model.generate(
            semantic_tokens, n_codebooks=8)

        # Stage 3: Decode to waveform
        waveform = self.codec.decode(acoustic_tokens)
        return waveform

The semantic-acoustic split is crucial. Semantic tokens change slowly (25/sec) and capture "what" the music is. Acoustic tokens are dense (75/sec × 8 codebooks) and capture "how" it sounds. This two-stage approach makes the generation tractable — predicting 750 acoustic tokens per second autoregressively would be painfully slow.

MusicGen

MusicGen (Meta, 2023) simplifies the multi-stage approach by introducing clever codebook interleaving patterns. Instead of separate models for different codebook levels, MusicGen uses a single Transformer that generates all codebook levels with a delay pattern.

class CodebookDelayPattern:
    """
    MusicGen's delay pattern for parallel codebook generation.

    Instead of generating all 8 codebooks sequentially (slow),
    or all at once (can't condition on previous codebooks),
    delay each codebook by its index:

    Time step:  0  1  2  3  4  5  6  ...
    Codebook 0: c0 c1 c2 c3 c4 c5 c6
    Codebook 1: -- c0 c1 c2 c3 c4 c5
    Codebook 2: -- -- c0 c1 c2 c3 c4
    ...
    Codebook 7: -- -- -- -- -- -- -- c0 c1

    At each step, the model predicts one token per codebook,
    and each codebook can see previous tokens from all codebooks.
    """
    def __init__(self, n_codebooks=8):
        self.n_codebooks = n_codebooks

    def apply_delays(self, codes):
        """
        codes: (batch, n_codebooks, time)
        Returns delayed pattern for training.
        """
        B, K, T = codes.shape
        delayed = torch.full((B, K, T + K - 1), -1, dtype=codes.dtype,
                             device=codes.device)
        for k in range(K):
            delayed[:, k, k:k+T] = codes[:, k, :]
        return delayed

    def remove_delays(self, delayed_codes):
        """Remove delays to get aligned codes for decoding."""
        B, K, T_delayed = delayed_codes.shape
        T = T_delayed - K + 1
        codes = torch.zeros(B, K, T, dtype=delayed_codes.dtype,
                           device=delayed_codes.device)
        for k in range(K):
            codes[:, k, :] = delayed_codes[:, k, k:k+T]
        return codes

This delay pattern is MusicGen's key contribution. It allows a single autoregressive pass to generate all codebook levels with minimal quality loss compared to the multi-stage approach. The model generates roughly one "column" of codebook tokens per step, with each codebook slightly delayed so it can condition on the previous codebook's output from the prior time step.

class MusicGenTransformer(nn.Module):
    """Simplified MusicGen: single-model music generation."""
    def __init__(self, n_codebooks=8, codebook_size=1024,
                 dim=1024, n_layers=24, n_heads=16):
        super().__init__()
        self.n_codebooks = n_codebooks
        # Separate embedding per codebook
        self.embeddings = nn.ModuleList([
            nn.Embedding(codebook_size + 1, dim)  # +1 for delay padding
            for _ in range(n_codebooks)
        ])
        self.transformer = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(
                d_model=dim, nhead=n_heads, dim_feedforward=dim*4,
                batch_first=True, activation='gelu'),
            num_layers=n_layers
        )
        # Separate prediction heads per codebook
        self.heads = nn.ModuleList([
            nn.Linear(dim, codebook_size)
            for _ in range(n_codebooks)
        ])

    def forward(self, delayed_codes, text_conditioning):
        """
        delayed_codes: (batch, n_codebooks, time) with delay pattern
        text_conditioning: (batch, text_len, dim)
        """
        B, K, T = delayed_codes.shape
        # Sum embeddings across codebooks for each time step
        token_emb = sum(
            self.embeddings[k](delayed_codes[:, k, :].clamp(min=0))
            for k in range(K)
        )  # (B, T, dim)

        # Prepend text conditioning
        x = torch.cat([text_conditioning, token_emb], dim=1)

        # Causal transformer
        mask = nn.Transformer.generate_square_subsequent_mask(
            x.size(1), device=x.device)
        h = self.transformer(x, mask=mask)

        # Predict each codebook separately
        audio_h = h[:, text_conditioning.size(1):, :]
        logits = [self.heads[k](audio_h) for k in range(K)]
        return torch.stack(logits, dim=1)  # (B, K, T, codebook_size)
💡 Text Conditioning

MusicGen conditions on text using a frozen T5 encoder. The text embeddings are prepended to the audio token sequence as a prefix, and the causal attention mask allows all audio tokens to attend to all text tokens. This is the same "prefix-LM" conditioning used in many language models. No cross-attention needed — the standard causal Transformer does the job.

Diffusion-Based Text-to-Speech

Mel-Spectrogram Diffusion

Most modern TTS systems don't generate waveforms directly. They generate mel-spectrograms — a 2D time-frequency representation that captures the perceptually relevant features of audio. A mel-spectrogram is essentially a heatmap: time on the x-axis, frequency (on a perceptual mel scale) on the y-axis, and energy as the color. A neural vocoder then converts this spectrogram back to a waveform.

class MelSpectrogramDiffusion(nn.Module):
    """Diffusion model that generates mel-spectrograms from text."""
    def __init__(self, n_mels=80, hidden=256, text_dim=512):
        super().__init__()
        # U-Net operating on mel-spectrogram "images"
        # Treat (n_mels, time) as a 2D signal
        self.unet = UNet1D(
            in_channels=n_mels,
            model_channels=hidden,
            cond_dim=text_dim
        )
        self.n_mels = n_mels

    def diffusion_loss(self, mel, text_embed):
        """Standard DDPM loss on mel-spectrograms."""
        batch_size = mel.size(0)
        # mel: (batch, n_mels, time_frames)
        t = torch.randint(0, 1000, (batch_size,), device=mel.device)

        noise = torch.randn_like(mel)
        alpha_bar = self.get_alpha_bar(t)
        noisy_mel = alpha_bar.sqrt().view(-1,1,1) * mel + \
                    (1 - alpha_bar).sqrt().view(-1,1,1) * noise

        predicted_noise = self.unet(noisy_mel, t, text_embed)
        loss = ((predicted_noise - noise) ** 2).mean()
        return loss

    def get_alpha_bar(self, t):
        """Cosine noise schedule."""
        s = 0.008
        steps = t.float() / 1000
        alpha_bar = torch.cos((steps + s) / (1 + s) * torch.pi / 2) ** 2
        return alpha_bar

The advantage of operating on mel-spectrograms: they are 100–300× smaller than raw waveforms (typically 80 frequency bins × 100 frames per second), and the 2D structure lets you reuse image-generation techniques directly. The disadvantage: you need a separate vocoder (like HiFi-GAN or BigVGAN) to convert back to audio, which adds complexity and potential artifacts.

Tortoise TTS and XTTS

Tortoise TTS (Betker, 2023) combines autoregressive and diffusion approaches for high-quality text-to-speech with voice cloning. The pipeline has three stages:

# Tortoise TTS pipeline (conceptual)
class TortoiseTTS:
    """Three-stage TTS: AR codes -> diffusion mel -> vocoder audio."""
    def __init__(self):
        self.ar_model = None       # Autoregressive: text -> speech codes
        self.diffusion = None      # Diffusion: codes -> mel-spectrogram
        self.vocoder = None        # Vocoder: mel -> waveform
        self.clvp = None           # Ranking model for best sample

    def generate(self, text, reference_audio, n_candidates=4):
        # Encode reference audio for voice cloning
        voice_embedding = self.encode_voice(reference_audio)

        # Stage 1: AR model generates multiple candidate speech codes
        candidates = []
        for _ in range(n_candidates):
            codes = self.ar_model.generate(
                text, voice_embedding, temperature=0.8)
            candidates.append(codes)

        # Rank candidates using CLVP (contrastive model)
        best_codes = self.clvp.rank(candidates, text)

        # Stage 2: Diffusion model refines codes into mel-spectrogram
        mel = self.diffusion.generate(best_codes, voice_embedding)

        # Stage 3: Vocoder converts mel to waveform
        waveform = self.vocoder(mel)
        return waveform

XTTS (Coqui, 2023) builds on Tortoise with improvements for multilingual support and faster inference. It uses a GPT-like backbone for the autoregressive stage and supports zero-shot voice cloning from as little as 6 seconds of reference audio. The key advances: a VQ-VAE for more efficient speech tokenization, language-specific conditioning tokens, and latent diffusion instead of mel-spectrogram diffusion for faster generation.

AudioLDM: Latent Diffusion for Audio

AudioLDM (Liu et al., 2023) applies the Stable Diffusion recipe to audio: encode audio into a latent space with a VAE, run diffusion in that latent space, then decode back to a mel-spectrogram. The text conditioning comes from a CLAP model (Contrastive Language-Audio Pretraining) — the audio equivalent of CLIP.

class AudioLDM(nn.Module):
    """Latent Diffusion Model for audio generation."""
    def __init__(self, vae, unet, clap_encoder):
        super().__init__()
        self.vae = vae             # Audio VAE: mel <-> latent
        self.unet = unet           # Diffusion in latent space
        self.clap = clap_encoder   # Text -> audio-aligned embeddings

    def training_step(self, mel_spectrogram, text):
        # Encode mel to latent space
        with torch.no_grad():
            z = self.vae.encode(mel_spectrogram)  # (B, C, T_latent)
            text_embed = self.clap.encode_text(text)

        # Standard diffusion loss in latent space
        t = torch.randint(0, 1000, (z.size(0),), device=z.device)
        noise = torch.randn_like(z)
        z_noisy = self.add_noise(z, noise, t)

        noise_pred = self.unet(z_noisy, t, text_embed)
        loss = ((noise_pred - noise) ** 2).mean()
        return loss

    @torch.no_grad()
    def generate(self, text, n_steps=50, guidance_scale=3.5):
        text_embed = self.clap.encode_text(text)
        null_embed = torch.zeros_like(text_embed)

        # Start from noise in latent space
        z = torch.randn(1, 8, 256, device=text_embed.device)

        for t in reversed(range(0, 1000, 1000 // n_steps)):
            t_tensor = torch.tensor([t], device=z.device)
            # Classifier-free guidance
            uncond = self.unet(z, t_tensor, null_embed)
            cond = self.unet(z, t_tensor, text_embed)
            noise_pred = uncond + guidance_scale * (cond - uncond)

            z = self.ddim_step(z, noise_pred, t)

        # Decode latent to mel, then mel to audio
        mel = self.vae.decode(z)
        return mel
📊 Why Latent Space?

Working in latent space reduces the diffusion input from 80×1000 (mel bins × time frames for 10 seconds) to approximately 8×256 — a 39× reduction. This makes the diffusion process faster and lets the model focus on semantic audio structure rather than low-level spectral details. The VAE handles the fine details during decoding. AudioLDM2 extended this with a "language of audio" representation that handles speech, music, and sound effects in a unified framework.

Putting It Together: The Audio Generation Stack

The modern audio generation landscape looks like this:

# The audio generation toolchain (2024)
#
# Text-to-Speech:
#   Text -> [GPT/AR model] -> speech tokens
#   Speech tokens -> [Diffusion] -> mel-spectrogram
#   Mel-spectrogram -> [Vocoder] -> waveform
#
# Text-to-Music:
#   Text -> [T5 encoder] -> text embeddings
#   Text embeddings -> [Transformer + delay pattern] -> codec tokens
#   Codec tokens -> [EnCodec decoder] -> waveform
#
# Text-to-Sound:
#   Text -> [CLAP encoder] -> audio-aligned embeddings
#   Embeddings -> [Latent Diffusion] -> latent codes
#   Latent codes -> [VAE decoder] -> mel-spectrogram
#   Mel-spectrogram -> [Vocoder] -> waveform

Every path involves the same principle: compress audio to a manageable representation, generate in that compressed space, decode back to audio. The representation varies (discrete tokens for autoregressive models, continuous latents for diffusion), but the compress-generate-decode pipeline is universal.

The field is converging rapidly. Models like Voicebox (Meta), NaturalSpeech 3 (Microsoft), and Stable Audio (Stability AI) are blurring the lines between these categories, combining autoregressive and diffusion components, using flow matching instead of DDPM, and scaling to longer outputs. The common thread: neural audio codecs as the bridge between raw audio and generative models.
