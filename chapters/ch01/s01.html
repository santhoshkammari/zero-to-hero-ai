Getting Started with Python

Chapter 1: Python & Programming Foundations 3 subtopics

TL;DR

Programming is the act of writing precise instructions that a computer can follow. Python is a high-level, interpreted language that reads almost like English, which is why it dominates data science and AI. You install it, type commands into a REPL or save them in .py files, and learn a handful of syntax rules — indentation, variables, comments — to start building real programs.

What is Programming

How Computers Execute Instructions

Imagine you hire someone who follows directions with absolute precision but has zero common sense. You can't say "make me a sandwich." You have to say: "Open the bread bag. Remove two slices. Place them on the counter. Open the peanut butter jar…" That's programming. A program is a sequence of unambiguous instructions that a computer executes one after another.

At the hardware level, a computer's processor — the CPU (Central Processing Unit) — only understands electrical signals that represent ones and zeros. Every instruction boils down to moving numbers between memory locations, adding them, comparing them, or jumping to a different instruction. That's it. Every video game, every web search, every AI model is built on those four basic operations, repeated billions of times per second.

When you write Python code, you're writing something humans can read. That human-readable text gets translated into the ones and zeros the CPU actually runs. The piece of software that does this translation is called an interpreter (for Python) or a compiler (for languages like C). We'll get to that distinction in a moment.

High-Level vs Low-Level Languages

A low-level language is one that's close to what the hardware speaks. Assembly language, for instance, maps almost directly to CPU instructions. You control every register, every memory address. It's powerful, but writing a simple "hello world" might take ten lines of cryptic code.

A high-level language is one that's close to what humans speak. Python, JavaScript, and Ruby are high-level. They hide the details of memory management and CPU registers behind friendly syntax. When you write print("hello") in Python, dozens of low-level operations happen underneath — allocating memory for the string, sending bytes to your terminal — but you don't see any of it.

The trade-off? Low-level languages give you precise control and speed. High-level languages give you productivity and readability. For data science and AI, where you're prototyping ideas fast and working with libraries that handle the heavy number-crunching in optimized C code under the hood, high-level is the right choice.

Interpreted vs Compiled Languages

A compiled language like C or Rust converts your entire source code into machine code before you run it. You run a compiler, it produces an executable file, and you run that file. The program runs fast because the translation happened ahead of time. But if there's a bug on line 500, you don't find out until you try to compile the whole thing.

An interpreted language like Python translates your code line by line, at the moment it runs. There's no separate "compile" step. You write code, you run it immediately. If line 10 has a bug, lines 1–9 still execute. This makes development faster — you can test ideas interactively — but the program itself runs slower because translation happens at runtime.

Python specifically uses a hybrid approach. Your .py file is first compiled to bytecode — an intermediate representation stored in .pyc files — and then the Python interpreter executes that bytecode. But as a developer, this is invisible to you. You write code and run it. No build step.

Key Insight: Python's "slowness" rarely matters in data science because the heavy computation is done by libraries like NumPy and TensorFlow, which are written in C and C++ underneath. Python acts as the conductor; the orchestra members speak machine code.

Why Python for Data Science and AI

Python won the data science world for three reasons. First, readability. Code is read far more often than it's written, and Python enforces clean, readable structure. Second, the ecosystem. Libraries like NumPy, Pandas, Scikit-learn, TensorFlow, and PyTorch give you world-class tools without reinventing anything. Third, community. When you hit a wall, millions of people have hit the same wall and posted solutions online.

What Python cannot do well: it's not the right choice for systems that need raw speed, like operating system kernels, game engines, or embedded devices with tiny amounts of memory. For those, you'd reach for C, C++, or Rust.

Installing Python

Python Versions

Python 2 and Python 3 are effectively different languages. Python 2 reached end-of-life in January 2020. When we say "Python" in this book, we mean Python 3 — specifically 3.9 or later. You can check your version like this:

# In your terminal (not inside Python)
# Run:
#   python3 --version
# Expected output: Python 3.11.x (or similar)

import sys
print(sys.version)
# Example output: 3.11.4 (main, Jun  9 2023, 07:30:00)

Installing on Windows, macOS, Linux

On Windows, download the installer from python.org. Check the box that says "Add Python to PATH" — this is critical. Without it, your terminal won't know where to find Python.

On macOS, Python 2 used to come pre-installed (older versions). Install Python 3 via the official installer or through Homebrew: brew install python3.

On Linux, most distributions include Python 3. If not: sudo apt install python3 on Debian/Ubuntu, or sudo dnf install python3 on Fedora.

After installation, open a terminal and type python3 --version. If you see a version number, you're set. If you get "command not found," the installation didn't add Python to your system's PATH — the list of directories your terminal searches for executables.

The Python Interpreter (REPL)

Type python3 in your terminal and hit Enter. You'll see a prompt that looks like >>>. This is the REPL — Read, Eval, Print, Loop. It reads what you type, evaluates it, prints the result, and loops back for more input.

>>> 2 + 3
5
>>> "hello" + " " + "world"
'hello world'
>>> len("Python")
6

The REPL is your scratchpad. Use it to test ideas quickly. It's not where you write real programs, but it's where you figure things out. Think of it as a calculator that speaks Python.

Running Python Scripts from Terminal

For actual programs, you save code in a file ending with .py and run it from the terminal:

# Save this as hello.py
name = "World"
print(f"Hello, {name}!")
print(f"The name has {len(name)} characters")

Then in your terminal: python3 hello.py. The interpreter reads the file top to bottom and executes each line in order.

IDEs and Text Editors

You can write Python in Notepad if you want, but an IDE (Integrated Development Environment) makes life dramatically easier. VS Code is free, lightweight, and has excellent Python support through extensions — it highlights syntax errors, auto-completes function names, and lets you run code with a button click. PyCharm is heavier but has built-in support for virtual environments, debugging, and testing. For data science specifically, Jupyter Notebooks let you mix code, output, and documentation in one interactive document — we'll cover those later.

Basic Syntax

Indentation and Code Blocks

Most languages use curly braces {} to group lines of code together. Python uses indentation — whitespace at the start of a line. This isn't a style preference; it's a rule enforced by the language.

# This is correct — the indented lines belong to the if block
temperature = 35
if temperature > 30:
    print("It's hot outside")
    print("Drink water")
print("This line runs regardless")

# This would cause an IndentationError:
# if temperature > 30:
# print("It's hot outside")   # not indented!

Use 4 spaces per indent level. Not tabs, not 2 spaces — 4 spaces. This is the universal Python convention (PEP 8). Most editors convert tab presses into 4 spaces automatically.

Comments

A comment is a note for humans that Python ignores completely. Single-line comments start with #:

# This calculates the area of a circle
radius = 5
area = 3.14159 * radius ** 2  # pi * r^2

For multi-line comments, Python doesn't have a dedicated syntax. You use triple-quoted strings, which Python evaluates (as a string) but discards if not assigned to anything:

"""
This function will eventually calculate compound interest.
For now it's a placeholder.
TODO: Add error handling for negative rates.
"""

Print Function and Output

The print() function sends text to your terminal. It's your primary way to see what your program is doing.

print("Hello")                    # Hello
print("Age:", 25)                  # Age: 25
print("One", "Two", "Three")      # One Two Three (space-separated)
print("Items:", 3, sep="→")       # Items:→3
print("No newline", end=" ")      # Stays on same line
print("...continues here")

Notice that print() accepts multiple arguments separated by commas. By default it puts a space between them (sep=" ") and adds a newline at the end (end="\n"). You can change both.

Input Function and User Input

The input() function pauses your program and waits for the user to type something, then returns what they typed as a string — always a string, even if they type a number.

name = input("What is your name? ")
print(f"Hello, {name}!")

# If you need a number, you must convert it yourself
age_str = input("How old are you? ")
age = int(age_str)               # convert string to integer
print(f"In 10 years, you'll be {age + 10}")
Watch Out: If the user types "twenty" instead of "20" and you call int() on it, your program crashes with a ValueError. We'll learn how to handle that gracefully in the Error Handling section.

Variables and Assignment

A variable is a name that points to a value stored in memory. You create one with the = operator — this is called assignment, not "equals." It means "make this name refer to this value."

message = "Hello, Python"   # message now points to this string
count = 42                   # count now points to the integer 42
pi = 3.14159                 # pi now points to this float

# Variables can be reassigned to different types
x = 10          # x is an integer
x = "ten"       # now x is a string — Python doesn't complain

# Multiple assignment in one line
a, b, c = 1, 2, 3
print(a, b, c)  # 1 2 3

# Swap two variables — Python makes this elegant
a, b = b, a
print(a, b)     # 2 1

Naming Conventions

Python uses snake_case for variable and function names — all lowercase with underscores between words. This isn't enforced by the language, but it's enforced by the community. Break this convention and experienced Python developers will squint at your code.

# Good — snake_case
user_name = "Alice"
max_retry_count = 3
total_price = 49.99

# Bad — these work but violate convention
userName = "Alice"       # this is camelCase (used in Java/JavaScript)
MaxRetryCount = 3        # this is PascalCase (used for class names in Python)
TOTAL_PRICE = 49.99      # ALL_CAPS is reserved for constants

Variable names must start with a letter or underscore, can contain letters, digits, and underscores, and are case-sensitive (age and Age are different variables).

Reserved Keywords

Python reserves about 35 words for its own use. You cannot use them as variable names. Here are the ones you'll encounter most often:

import keyword
print(keyword.kwlist)
# ['False', 'None', 'True', 'and', 'as', 'assert', 'async', 'await',
#  'break', 'class', 'continue', 'def', 'del', 'elif', 'else',
#  'except', 'finally', 'for', 'from', 'global', 'if', 'import',
#  'in', 'is', 'lambda', 'nonlocal', 'not', 'or', 'pass', 'raise',
#  'return', 'try', 'while', 'with', 'yield']
Common Mistake: Using list, dict, str, or type as variable names. These aren't keywords — Python won't stop you — but they shadow built-in functions. If you write list = [1, 2, 3], you can no longer call list() to convert things to lists until you restart your interpreter. Name your variables descriptively: scores = [1, 2, 3] instead.
