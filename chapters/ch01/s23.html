Algorithms and Data Structures for ML

Chapter 1: Python & Programming Foundations 9 subtopics
TL;DR

ML isn't exempt from computer science fundamentals — knowing when to use a hash map vs. a tree, understanding why quicksort averages O(n log n), and being able to implement core algorithms from scratch makes you a better ML engineer. This section covers the essential data structures (hash maps, trees, heaps) and algorithms (searching, sorting) with their complexities, then builds three ML algorithms from scratch: K-Means clustering, gradient descent for linear regression, and a decision tree. These implementations cement your understanding of what libraries like scikit-learn do under the hood.

Why Algorithms Matter for ML

Here's a scenario. You've trained a recommendation model that returns the top 100 candidate items for a user. Now you need to re-rank them by a score, filter out items the user has already seen (from a set of 10,000 viewed items), and return the top 10. If you use a linear scan for the filtering step, that's 100 × 10,000 = 1,000,000 comparisons per request. Use a hash set and it drops to 100 lookups, each O(1). At 1,000 requests per second, you've saved a billion unnecessary comparisons per second.

Algorithms and data structures aren't academic exercises. They're the difference between an ML system that scales and one that falls over. Let's build the toolkit.

Searching: Binary Search and Linear Search

Linear search walks through a collection one element at a time. It's O(n) — works on any data, but doesn't scale.

Binary search works on sorted data by repeatedly cutting the search space in half. It's O(log n) — searching 1 billion items takes at most 30 comparisons.

def linear_search(arr: list, target) -> int:
    """O(n) — check every element."""
    for i, val in enumerate(arr):
        if val == target:
            return i
    return -1

def binary_search(arr: list, target) -> int:
    """O(log n) — requires sorted input."""
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

# Example: finding a threshold in sorted model scores
scores = sorted([0.1, 0.3, 0.45, 0.67, 0.72, 0.89, 0.95])
idx = binary_search(scores, 0.67)
print(f"Found 0.67 at index {idx}")  # Found 0.67 at index 3

Binary search is everywhere in ML. Finding optimal thresholds for classification, locating insertion points in sorted arrays, bisecting hyperparameter ranges — all variations on the same O(log n) idea. Python's bisect module gives you a production-quality implementation:

import bisect

sorted_scores = [0.1, 0.3, 0.45, 0.67, 0.72, 0.89, 0.95]
# Find where 0.5 would be inserted to maintain sorted order
insert_pos = bisect.bisect_left(sorted_scores, 0.5)
print(f"0.5 would go at index {insert_pos}")  # index 3

# Count how many scores are >= 0.7 (threshold for "positive" class)
threshold_pos = bisect.bisect_left(sorted_scores, 0.7)
num_positive = len(sorted_scores) - threshold_pos
print(f"{num_positive} scores >= 0.7")  # 3 scores >= 0.7

Sorting: Quicksort, Merge Sort, Bubble Sort

Sorting is the most studied problem in computer science. You need to understand three algorithms to cover the essentials.

Bubble Sort — O(n²), the Baseline

Bubble sort repeatedly walks through the list, swapping adjacent elements that are out of order. It's O(n²) and terrible for large data, but it's dead easy to understand and implement. It's the "hello world" of sorting algorithms.

def bubble_sort(arr: list) -> list:
    """O(n²) — only for tiny arrays or teaching."""
    arr = arr.copy()
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # Already sorted — early exit
            break
    return arr

Merge Sort — O(n log n), Guaranteed

Merge sort splits the array in half, recursively sorts each half, and merges the sorted halves. It's always O(n log n) — no worst-case degradation. The tradeoff: it uses O(n) extra memory for the merge step.

def merge_sort(arr: list) -> list:
    """O(n log n) time, O(n) space — stable sort."""
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)

def merge(left: list, right: list) -> list:
    """Merge two sorted lists into one sorted list."""
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # <= makes it stable
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

Quicksort — O(n log n) Average, In-Place

Quicksort picks a pivot, partitions the array so that smaller elements go left and larger go right, then recursively sorts each partition. Average case is O(n log n), worst case is O(n²) (when the pivot is always the smallest or largest element). In practice, with random pivots, it's one of the fastest sorting algorithms.

import random

def quicksort(arr: list) -> list:
    """O(n log n) average — fast in practice."""
    if len(arr) <= 1:
        return arr

    pivot = arr[random.randint(0, len(arr) - 1)]  # Random pivot avoids worst case
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]

    return quicksort(left) + middle + quicksort(right)
Which Sort Does Python Use?

Python's built-in sorted() and list.sort() use Timsort — a hybrid of merge sort and insertion sort. It's O(n log n) worst case, stable, and optimized for real-world data that's often partially sorted. Always use the built-in sort in production. Implement your own only for learning or when you need a custom sorting strategy that the key function can't express.

Hash Maps: Implementation and Complexity

A hash map (dictionary in Python) maps keys to values with O(1) average lookup, insertion, and deletion. It's the most important data structure in programming. Period.

Under the hood, a hash map works by computing a hash of the key, using that hash to determine a bucket (array index), and storing the key-value pair in that bucket. When two keys map to the same bucket (a collision), the implementation handles it — Python uses open addressing with probing.

class SimpleHashMap:
    """Minimal hash map to show how it works."""
    def __init__(self, capacity: int = 16):
        self.capacity = capacity
        self.buckets: list[list[tuple]] = [[] for _ in range(capacity)]
        self.size = 0

    def _hash(self, key) -> int:
        return hash(key) % self.capacity

    def put(self, key, value):
        bucket_idx = self._hash(key)
        bucket = self.buckets[bucket_idx]
        # Update existing key
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        # New key
        bucket.append((key, value))
        self.size += 1

    def get(self, key, default=None):
        bucket_idx = self._hash(key)
        for k, v in self.buckets[bucket_idx]:
            if k == key:
                return v
        return default

# Usage
feature_cache = SimpleHashMap()
feature_cache.put("user_123", [0.1, 0.5, 0.3])
print(feature_cache.get("user_123"))  # [0.1, 0.5, 0.3]

In ML, hash maps are everywhere: feature lookups, vocabulary mappings, caching predictions, counting word frequencies, storing model configurations. Know their complexity: O(1) average, O(n) worst case (when every key collides into one bucket — extremely rare with good hash functions).

Trees and Tries

A binary search tree (BST) organizes data so that for every node, all values in the left subtree are smaller and all values in the right subtree are larger. This gives O(log n) search, insert, and delete — if the tree is balanced.

class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

class BST:
    def __init__(self):
        self.root = None

    def insert(self, val):
        self.root = self._insert(self.root, val)

    def _insert(self, node, val):
        if node is None:
            return TreeNode(val)
        if val < node.val:
            node.left = self._insert(node.left, val)
        elif val > node.val:
            node.right = self._insert(node.right, val)
        return node

    def search(self, val) -> bool:
        return self._search(self.root, val)

    def _search(self, node, val) -> bool:
        if node is None:
            return False
        if val == node.val:
            return True
        elif val < node.val:
            return self._search(node.left, val)
        else:
            return self._search(node.right, val)

    def inorder(self) -> list:
        """Returns elements in sorted order."""
        result = []
        self._inorder(self.root, result)
        return result

    def _inorder(self, node, result):
        if node:
            self._inorder(node.left, result)
            result.append(node.val)
            self._inorder(node.right, result)

A trie (prefix tree) is a specialized tree for string lookups. Each path from root to leaf represents a string. Tries give O(k) lookup where k is the string length — independent of how many strings are stored. In NLP, tries power autocomplete, spell checking, and tokenizer vocabulary lookups.

class TrieNode:
    def __init__(self):
        self.children: dict[str, "TrieNode"] = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self._find_node(word)
        return node is not None and node.is_end

    def starts_with(self, prefix: str) -> bool:
        return self._find_node(prefix) is not None

    def _find_node(self, prefix: str):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return None
            node = node.children[char]
        return node

# NLP use case: vocabulary lookup
vocab = Trie()
for word in ["attention", "attention_mask", "attention_weights", "embedding"]:
    vocab.insert(word)

print(vocab.search("attention"))       # True
print(vocab.starts_with("atten"))      # True
print(vocab.search("transformer"))     # False

Heaps and Priority Queues

A heap is a tree-based structure where the parent is always smaller (min-heap) or larger (max-heap) than its children. The key operation: extract the minimum (or maximum) element in O(log n) and insert in O(log n). Python's heapq module implements a min-heap.

import heapq

# Use case: Top-K predictions — keep only the K highest scores
def top_k_predictions(scores: list[tuple[str, float]], k: int) -> list[tuple[str, float]]:
    """Return the k items with highest scores using a min-heap.

    We maintain a min-heap of size k. For each new score,
    if it's larger than the heap's minimum, we replace the minimum.
    This is O(n log k) — much better than sorting O(n log n) when k << n.
    """
    heap = []
    for label, score in scores:
        if len(heap) < k:
            heapq.heappush(heap, (score, label))
        elif score > heap[0][0]:
            heapq.heapreplace(heap, (score, label))

    # Return in descending order
    return [(label, score) for score, label in sorted(heap, reverse=True)]

predictions = [
    ("cat", 0.85), ("dog", 0.92), ("bird", 0.43),
    ("fish", 0.67), ("horse", 0.78), ("snake", 0.31),
]
print(top_k_predictions(predictions, k=3))
# [('dog', 0.92), ('cat', 0.85), ('horse', 0.78)]

Priority queues are essential in ML for beam search in sequence models, Dijkstra's algorithm in graph neural networks, and maintaining top-K candidate lists in retrieval systems.

ML Algorithm Implementations from Scratch

Understanding what happens inside sklearn.cluster.KMeans() or sklearn.linear_model.LinearRegression() makes you a better practitioner. Let's build three core algorithms from scratch.

K-Means Clustering (NumPy)

K-Means partitions data into K clusters by alternating between assigning points to the nearest centroid and updating centroids to be the mean of their assigned points.

import numpy as np

def kmeans(X: np.ndarray, k: int, max_iters: int = 100, tol: float = 1e-4):
    """K-Means clustering from scratch.

    Args:
        X: Data matrix, shape (n_samples, n_features)
        k: Number of clusters
        max_iters: Maximum iterations
        tol: Convergence tolerance for centroid movement
    """
    n_samples, n_features = X.shape

    # Step 1: Initialize centroids by randomly picking k data points
    indices = np.random.choice(n_samples, k, replace=False)
    centroids = X[indices].copy()

    for iteration in range(max_iters):
        # Step 2: Assign each point to nearest centroid
        # distances[i, j] = distance from point i to centroid j
        distances = np.linalg.norm(X[:, np.newaxis] - centroids, axis=2)
        labels = np.argmin(distances, axis=1)

        # Step 3: Update centroids to mean of assigned points
        new_centroids = np.array([
            X[labels == j].mean(axis=0) if np.any(labels == j) else centroids[j]
            for j in range(k)
        ])

        # Check convergence
        shift = np.linalg.norm(new_centroids - centroids)
        centroids = new_centroids
        if shift < tol:
            print(f"Converged at iteration {iteration}")
            break

    return labels, centroids

# Demo
np.random.seed(42)
cluster1 = np.random.randn(50, 2) + np.array([0, 0])
cluster2 = np.random.randn(50, 2) + np.array([5, 5])
cluster3 = np.random.randn(50, 2) + np.array([10, 0])
X = np.vstack([cluster1, cluster2, cluster3])

labels, centroids = kmeans(X, k=3)
print(f"Centroids:\n{centroids}")

Gradient Descent for Linear Regression

Gradient descent finds the parameters that minimize a loss function by iteratively moving in the direction of steepest descent. For linear regression, the loss is mean squared error.

import numpy as np

def linear_regression_gd(
    X: np.ndarray, y: np.ndarray, lr: float = 0.01, epochs: int = 1000
):
    """Train linear regression with gradient descent.

    Model: y_hat = X @ w + b
    Loss:  MSE = (1/n) * sum((y - y_hat)^2)
    """
    n_samples, n_features = X.shape

    # Initialize weights and bias to zeros
    w = np.zeros(n_features)
    b = 0.0
    losses = []

    for epoch in range(epochs):
        # Forward pass: compute predictions
        y_hat = X @ w + b

        # Compute loss (MSE)
        error = y_hat - y
        loss = np.mean(error ** 2)
        losses.append(loss)

        # Backward pass: compute gradients
        dw = (2 / n_samples) * (X.T @ error)  # gradient w.r.t weights
        db = (2 / n_samples) * np.sum(error)   # gradient w.r.t bias

        # Update parameters
        w -= lr * dw
        b -= lr * db

        if epoch % 200 == 0:
            print(f"Epoch {epoch}: loss = {loss:.4f}")

    return w, b, losses

# Demo: y = 3*x1 + 5*x2 + 7 + noise
np.random.seed(42)
X = np.random.randn(200, 2)
y = 3 * X[:, 0] + 5 * X[:, 1] + 7 + np.random.randn(200) * 0.1

w, b, losses = linear_regression_gd(X, y, lr=0.05, epochs=1000)
print(f"Learned weights: {w}")  # Close to [3, 5]
print(f"Learned bias: {b:.2f}")  # Close to 7

Decision Tree (CART)

A decision tree recursively splits data on the feature and threshold that best separates the target classes. The CART algorithm uses the Gini impurity to measure "best."

import numpy as np
from collections import Counter

def gini_impurity(y: np.ndarray) -> float:
    """Gini impurity: 1 - sum(p_i^2) for each class."""
    counts = Counter(y)
    n = len(y)
    return 1.0 - sum((count / n) ** 2 for count in counts.values())

def best_split(X: np.ndarray, y: np.ndarray):
    """Find the feature and threshold that minimizes weighted Gini impurity."""
    n_samples, n_features = X.shape
    best_gini = float("inf")
    best_feature = None
    best_threshold = None

    for feature_idx in range(n_features):
        thresholds = np.unique(X[:, feature_idx])
        for threshold in thresholds:
            left_mask = X[:, feature_idx] <= threshold
            right_mask = ~left_mask

            if left_mask.sum() == 0 or right_mask.sum() == 0:
                continue

            # Weighted Gini impurity
            n_left = left_mask.sum()
            n_right = right_mask.sum()
            gini = (
                (n_left / n_samples) * gini_impurity(y[left_mask])
                + (n_right / n_samples) * gini_impurity(y[right_mask])
            )

            if gini < best_gini:
                best_gini = gini
                best_feature = feature_idx
                best_threshold = threshold

    return best_feature, best_threshold

class DecisionTreeNode:
    def __init__(self, feature=None, threshold=None, left=None, right=None, value=None):
        self.feature = feature
        self.threshold = threshold
        self.left = left
        self.right = right
        self.value = value  # Leaf prediction (majority class)

def build_tree(X: np.ndarray, y: np.ndarray, depth: int = 0, max_depth: int = 5):
    """Recursively build a CART decision tree."""
    # Base cases: pure node or max depth reached
    if len(np.unique(y)) == 1:
        return DecisionTreeNode(value=y[0])
    if depth >= max_depth or len(y) < 2:
        return DecisionTreeNode(value=Counter(y).most_common(1)[0][0])

    feature, threshold = best_split(X, y)
    if feature is None:
        return DecisionTreeNode(value=Counter(y).most_common(1)[0][0])

    left_mask = X[:, feature] <= threshold
    left = build_tree(X[left_mask], y[left_mask], depth + 1, max_depth)
    right = build_tree(X[~left_mask], y[~left_mask], depth + 1, max_depth)

    return DecisionTreeNode(feature=feature, threshold=threshold, left=left, right=right)

def predict_one(node: DecisionTreeNode, x: np.ndarray):
    if node.value is not None:
        return node.value
    if x[node.feature] <= node.threshold:
        return predict_one(node.left, x)
    return predict_one(node.right, x)

def predict(tree: DecisionTreeNode, X: np.ndarray) -> np.ndarray:
    return np.array([predict_one(tree, x) for x in X])
Scratch Implementations vs Production Libraries

These implementations are for learning. In production, always use scikit-learn, XGBoost, or similar optimized libraries. They handle edge cases (empty clusters, numerical precision, missing values), use optimized C/Cython code, and are battle-tested on millions of datasets. The point of building from scratch is to understand the algorithm well enough to debug it when the library version behaves unexpectedly.
