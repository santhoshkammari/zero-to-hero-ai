Functions

Chapter 1: Python & Programming Foundations 4 subtopics

TL;DR

Functions are reusable blocks of code that take inputs, do something, and return outputs. They let you name a piece of logic, avoid repeating yourself, and break complex problems into manageable pieces. Python supports flexible argument passing (positional, keyword, defaults, *args, **kwargs), lexical scoping with the LEGB rule, closures that capture surrounding state, and higher-order patterns like map, filter, and lambda functions.

Function Basics

Imagine you're computing the average of a list of numbers in five different places in your code. Without functions, you'd copy-paste the same three lines five times. Then when you discover a bug — say, you forgot to handle the empty-list case — you'd have to fix it in all five places. Functions solve this. You write the logic once, give it a name, and call that name whenever you need it.

Defining Functions

You define a function with the def keyword, followed by the function name, parentheses with parameters, and a colon. The indented block below is the function body.

def greet(name):
    """Print a personalized greeting."""
    print(f"Hello, {name}! Welcome aboard.")

# Calling the function
greet("Alice")   # Hello, Alice! Welcome aboard.
greet("Bob")     # Hello, Bob! Welcome aboard.

The function doesn't run when you define it. It only runs when you call it. Think of def as saving a recipe. Calling the function is actually cooking the dish.

Function Parameters and Arguments

A parameter is the variable name in the function definition. An argument is the actual value you pass when calling the function. People often use these words interchangeably, but the distinction helps when reading documentation.

def add(x, y):    # x and y are parameters
    return x + y

result = add(3, 5)  # 3 and 5 are arguments
print(result)        # 8

Return Values

The return statement sends a value back to the caller and immediately exits the function. If a function has no return statement (or just return with no value), it returns None.

def square(n):
    return n ** 2

result = square(7)
print(result)  # 49

# Functions without return give you None
def say_hello():
    print("Hello!")

value = say_hello()  # prints "Hello!"
print(value)         # None

# return exits immediately — code after it doesn't run
def check_age(age):
    if age < 0:
        return "Invalid age"
    if age >= 18:
        return "Adult"
    return "Minor"

print(check_age(25))   # Adult
print(check_age(-1))   # Invalid age

Docstrings

A docstring is a string literal placed as the very first line inside a function. It documents what the function does, what it expects, and what it returns. It's not a comment — Python stores it as an attribute you can access at runtime.

def calculate_bmi(weight_kg, height_m):
    """
    Calculate Body Mass Index.

    Parameters:
        weight_kg: Body weight in kilograms.
        height_m: Height in meters.

    Returns:
        BMI as a float, rounded to one decimal place.
    """
    return round(weight_kg / (height_m ** 2), 1)

# Access the docstring programmatically
print(calculate_bmi.__doc__)

# help() uses the docstring
help(calculate_bmi)

Multiple Return Values

Python functions can return multiple values by separating them with commas. Under the hood, this creates a tuple which you can unpack on the receiving end.

def analyze_scores(scores):
    """Return basic statistics for a list of scores."""
    mean = sum(scores) / len(scores)
    minimum = min(scores)
    maximum = max(scores)
    return mean, minimum, maximum

# Unpack into separate variables
avg, lo, hi = analyze_scores([85, 92, 78, 95, 88])
print(f"Average: {avg}, Range: {lo}-{hi}")
# Average: 87.6, Range: 78-95

# Or capture as a single tuple
stats = analyze_scores([85, 92, 78, 95, 88])
print(stats)        # (87.6, 78, 95)
print(stats[0])     # 87.6

Arguments and Parameters

Python has a surprisingly flexible system for passing data into functions. Understanding it prevents a whole category of confusing bugs and lets you write functions that adapt to different calling patterns.

Positional Arguments

The simplest kind. Arguments are matched to parameters by position — first argument goes to first parameter, second to second, and so on.

def power(base, exponent):
    return base ** exponent

print(power(2, 10))   # 1024 — 2^10
print(power(10, 2))   # 100  — 10^2, order matters!

Keyword Arguments

You can also pass arguments by name, which makes the call self-documenting and lets you use any order.

def create_user(name, age, role):
    return {"name": name, "age": age, "role": role}

# Keyword arguments — order doesn't matter
user = create_user(role="admin", name="Alice", age=30)
print(user)  # {'name': 'Alice', 'age': 30, 'role': 'admin'}

# Mix positional and keyword — positional must come first
user = create_user("Bob", role="user", age=25)

Default Parameter Values

You can give parameters default values. If the caller doesn't provide an argument, the default kicks in.

def train_model(data, epochs=10, learning_rate=0.001, verbose=True):
    """Simulate training with configurable hyperparameters."""
    if verbose:
        print(f"Training for {epochs} epochs at lr={learning_rate}")
    # ... training logic ...

# Use all defaults
train_model(my_data)

# Override some defaults
train_model(my_data, epochs=50)
train_model(my_data, learning_rate=0.01, verbose=False)
Common Mistake: Using a mutable object (list, dict) as a default parameter value. The default is created once when the function is defined, and shared across all calls. 
# WRONG — the default list accumulates across calls
def add_item(item, items=[]):
    items.append(item)
    return items

print(add_item("a"))  # ['a']
print(add_item("b"))  # ['a', 'b'] — wait, where did 'a' come from?!

# CORRECT — use None as the default, create a new list inside
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

print(add_item("a"))  # ['a']
print(add_item("b"))  # ['b'] — fresh list each time

*args — Variable Positional Arguments

Sometimes you want a function that accepts any number of positional arguments. The *args syntax collects them into a tuple.

def total(*args):
    """Sum any number of values."""
    print(f"Received {len(args)} arguments: {args}")
    return sum(args)

print(total(1, 2, 3))         # Received 3 arguments: (1, 2, 3) → 6
print(total(10, 20, 30, 40))  # Received 4 arguments: (10, 20, 30, 40) → 100

# Mix regular parameters with *args
def average(first, *rest):
    all_values = (first,) + rest
    return sum(all_values) / len(all_values)

print(average(10, 20, 30))  # 20.0

**kwargs — Variable Keyword Arguments

Similarly, **kwargs collects any extra keyword arguments into a dictionary.

def build_profile(name, **kwargs):
    """Build a user profile from arbitrary attributes."""
    profile = {"name": name}
    profile.update(kwargs)
    return profile

user = build_profile("Alice", age=30, city="NYC", role="engineer")
print(user)
# {'name': 'Alice', 'age': 30, 'city': 'NYC', 'role': 'engineer'}

# The real power: passing config dictionaries
def configure_model(**hyperparams):
    defaults = {"lr": 0.001, "epochs": 10, "batch_size": 32}
    defaults.update(hyperparams)
    return defaults

config = configure_model(epochs=50, lr=0.01)
print(config)  # {'lr': 0.01, 'epochs': 50, 'batch_size': 32}

Argument Unpacking

The reverse of *args and **kwargs — you can unpack a list or dictionary into function arguments using * and **.

def train(model, epochs, lr):
    print(f"Training {model} for {epochs} epochs at lr={lr}")

# Unpack a list as positional arguments
args = ["ResNet", 50, 0.001]
train(*args)  # same as train("ResNet", 50, 0.001)

# Unpack a dictionary as keyword arguments
config = {"model": "ResNet", "epochs": 50, "lr": 0.001}
train(**config)  # same as train(model="ResNet", epochs=50, lr=0.001)

Scope and Closures

When you create a variable inside a function, where does it live? Can code outside the function see it? What if there's a variable with the same name outside? These questions are about scope — the rules that determine which variables are visible where.

Local and Global Scope

Variables created inside a function are local — they exist only during the function call and are destroyed when the function returns. Variables created at the top level of your script are global — they exist for the entire program.

message = "I am global"   # global scope

def show_message():
    message = "I am local"  # local scope — different variable!
    print(message)

show_message()     # I am local
print(message)     # I am global — unchanged

The local message inside the function shadows the global one but doesn't affect it. They're completely separate variables that happen to share a name.

The LEGB Rule

When Python encounters a variable name, it searches for it in this order: Local → Enclosing → Global → Built-in. The first match wins.

x = "global"        # G — global scope

def outer():
    x = "enclosing"  # E — enclosing scope

    def inner():
        x = "local"  # L — local scope
        print(x)     # looks up: L → found "local"

    inner()
    print(x)         # looks up: L (nothing) → E → found "enclosing"

outer()
print(x)             # looks up: L (nothing) → E (nothing) → G → found "global"

# B — built-in scope: functions like print, len, range
# print = "oops"    # Don't do this — shadows the built-in!

global and nonlocal Keywords

If you need to modify a variable from an outer scope (not just read it), you must declare your intent explicitly with global or nonlocal.

# global — modify a module-level variable
counter = 0

def increment():
    global counter    # without this, Python creates a new local 'counter'
    counter += 1

increment()
increment()
print(counter)  # 2

# nonlocal — modify a variable from an enclosing function
def make_counter():
    count = 0
    def increment():
        nonlocal count   # refers to 'count' in make_counter
        count += 1
        return count
    return increment

counter = make_counter()
print(counter())  # 1
print(counter())  # 2
print(counter())  # 3
Watch Out: Using global is generally a code smell. It creates hidden dependencies between your functions and module-level state, making code hard to test and reason about. Prefer passing values as arguments and returning results. The make_counter pattern above is a cleaner alternative — it uses closures.

Closures

A closure is a function that remembers the variables from the scope where it was created, even after that scope has finished executing. The make_counter example above is a closure — the inner increment function carries a reference to the count variable from make_counter, even after make_counter has returned.

# Closure for configuration
def make_multiplier(factor):
    def multiply(x):
        return x * factor    # 'factor' is captured from the enclosing scope
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))   # 10
print(triple(5))   # 15

# Each closure has its own copy of 'factor'
print(double.__closure__[0].cell_contents)  # 2
print(triple.__closure__[0].cell_contents)  # 3

# Practical example: creating custom loss functions
def make_weighted_loss(class_weights):
    """Create a loss function with specific class weights baked in."""
    def loss_fn(predictions, targets):
        errors = [(p - t) ** 2 * class_weights.get(t, 1.0)
                  for p, t in zip(predictions, targets)]
        return sum(errors) / len(errors)
    return loss_fn

# Configure once, use many times
weighted_loss = make_weighted_loss({0: 1.0, 1: 5.0})  # class 1 weighted 5x

Closures cannot modify their captured variables unless you use nonlocal. They also keep the captured variables alive in memory, which is usually fine but can cause memory leaks if the closure captures large objects you expected to be garbage collected.

Advanced Function Concepts

First-Class Functions

In Python, functions are first-class objects. That means a function is a value, like a number or a string. You can assign it to a variable, store it in a list, pass it as an argument to another function, and return it from a function.

def shout(text):
    return text.upper()

def whisper(text):
    return text.lower()

# Assign function to a variable (no parentheses — not calling it)
transform = shout
print(transform("hello"))  # HELLO

# Store functions in a data structure
processors = [shout, whisper, str.title]
for fn in processors:
    print(fn("hello world"))
# HELLO WORLD
# hello world
# Hello World

Higher-Order Functions

A higher-order function is a function that takes another function as an argument, returns a function, or both. You've already seen one — make_multiplier returns a function. Here's one that takes a function as input:

def apply_to_all(func, data):
    """Apply a function to every element and return results."""
    return [func(item) for item in data]

numbers = [1, -3, 5, -7, 9]
print(apply_to_all(abs, numbers))        # [1, 3, 5, 7, 9]
print(apply_to_all(str, numbers))        # ['1', '-3', '5', '-7', '9']

# Sorting with a key function — sort by absolute value
print(sorted(numbers, key=abs))          # [1, -3, 5, -7, 9]

# Sorting complex objects
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]
by_score = sorted(students, key=lambda s: s[1], reverse=True)
print(by_score)  # [('Bob', 92), ('Alice', 85), ('Charlie', 78)]

Lambda Functions

A lambda is an anonymous function — a function without a name, defined in a single expression. Use them for short, throwaway operations, especially as arguments to higher-order functions.

# Regular function
def square(x):
    return x ** 2

# Lambda equivalent
square = lambda x: x ** 2

# Lambdas are most useful inline
numbers = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x ** 2, numbers))
print(squared)  # [1, 4, 9, 16, 25]

# Filtering with lambda
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens)    # [2, 4]

# Sorting with lambda
pairs = [(1, "b"), (3, "a"), (2, "c")]
pairs.sort(key=lambda p: p[1])  # sort by second element
print(pairs)    # [(3, 'a'), (1, 'b'), (2, 'c')]
Key Insight: Lambdas can only contain a single expression — no statements, no assignments, no multi-line logic. If your lambda is getting complicated, write a regular function. Assigning a lambda to a variable (like square = lambda x: x ** 2) defeats the purpose — use def instead. Lambdas are meant to be anonymous.

map(), filter(), reduce()

These three built-in functions (well, reduce was moved to functools) apply a function across a sequence. They come from the functional programming tradition.

# map() — apply a function to every element
temperatures_c = [0, 20, 37, 100]
temperatures_f = list(map(lambda c: c * 9/5 + 32, temperatures_c))
print(temperatures_f)  # [32.0, 68.0, 98.6, 212.0]

# filter() — keep elements where the function returns True
scores = [45, 82, 67, 91, 55, 73, 88]
passing = list(filter(lambda s: s >= 70, scores))
print(passing)  # [82, 91, 73, 88]

# reduce() — combine all elements into a single value
from functools import reduce
numbers = [1, 2, 3, 4, 5]
product = reduce(lambda a, b: a * b, numbers)
print(product)  # 120 (1 * 2 * 3 * 4 * 5)

# How reduce works step by step:
# Step 1: a=1, b=2 → 2
# Step 2: a=2, b=3 → 6
# Step 3: a=6, b=4 → 24
# Step 4: a=24, b=5 → 120

In practice, list comprehensions often replace map() and filter() in Python because they're more readable. [c * 9/5 + 32 for c in temps] reads better than list(map(lambda c: c * 9/5 + 32, temps)). But you'll see map and filter in existing codebases and in functional-style data processing pipelines.

Recursion

Recursion is when a function calls itself. Every recursive function needs a base case — a condition where it returns without calling itself — otherwise it would call itself forever.

# Factorial: n! = n × (n-1) × (n-2) × ... × 1
def factorial(n):
    if n <= 1:          # base case
        return 1
    return n * factorial(n - 1)   # recursive case

print(factorial(5))   # 120 (5 × 4 × 3 × 2 × 1)

# How the calls unfold:
# factorial(5) → 5 * factorial(4)
# factorial(4) → 4 * factorial(3)
# factorial(3) → 3 * factorial(2)
# factorial(2) → 2 * factorial(1)
# factorial(1) → 1 (base case)
# Then it unwinds: 2*1=2, 3*2=6, 4*6=24, 5*24=120

# A more practical example: flatten a nested list
def flatten(nested):
    result = []
    for item in nested:
        if isinstance(item, list):
            result.extend(flatten(item))  # recurse into sublists
        else:
            result.append(item)
    return result

data = [1, [2, 3], [4, [5, 6]], 7]
print(flatten(data))  # [1, 2, 3, 4, 5, 6, 7]
Watch Out: Python has a default recursion limit of 1000 calls. Deep recursion hits this limit and crashes with RecursionError. For problems requiring deep recursion, convert to an iterative solution using a loop and an explicit stack. You can check the limit with sys.getrecursionlimit() and increase it with sys.setrecursionlimit(), but this is usually a sign that recursion isn't the right approach.

Memoization

Memoization is caching the results of function calls so you don't compute the same thing twice. It's essential for recursive functions that have overlapping subproblems — like the Fibonacci sequence, where fib(5) calls fib(4) and fib(3), but fib(4) also calls fib(3), resulting in redundant work.

# Without memoization — exponentially slow
def fib_slow(n):
    if n <= 1:
        return n
    return fib_slow(n - 1) + fib_slow(n - 2)

# fib_slow(35) takes seconds. fib_slow(50) takes forever.

# With memoization — using functools.lru_cache
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(100))  # 354224848179261915075 — computed instantly

# Manual memoization with a dictionary
def fib_memo(n, cache={}):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    cache[n] = fib_memo(n - 1, cache) + fib_memo(n - 2, cache)
    return cache[n]
Key Insight: The @lru_cache decorator turns any pure function (same inputs always produce same outputs) into a memoized version with one line of code. The @ syntax is a decorator — a topic we'll cover in a later section. For now, know that it wraps your function with caching logic. LRU stands for "Least Recently Used" — when the cache is full, it evicts the entries that haven't been accessed recently.
