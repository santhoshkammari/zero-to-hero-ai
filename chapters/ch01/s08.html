Decorators

Chapter 1: Python & Programming Foundations 2 subtopics

TL;DR

Decorators let you wrap a function (or class) with extra behavior without changing its source code. They are the backbone of cross-cutting concerns like logging, timing, caching, authentication, and input validation. Once you understand that functions are objects you can pass around, decorators become a natural and powerful abstraction. You will encounter them constantly in Flask, FastAPI, PyTorch, and every major Python framework.

Why Decorators Exist

You just wrote 50 API endpoint functions. Now your team lead says: "We need to log every call with its arguments and execution time." Without decorators, you would copy-paste timing and logging code into all 50 functions. Change the log format? Edit 50 places. Miss one? Bug.

Decorators solve this. You write the logging logic once, slap a @log_calls on each function, and you are done. The original function stays clean. The cross-cutting concern lives in one place.

Decorator Fundamentals

Functions as First-Class Objects

Before we build decorators, we need one foundational concept: in Python, functions are objects. You can assign them to variables, store them in lists, and pass them as arguments:

def greet(name):
    return f"Hello, {name}!"

# Assign function to a variable (no parentheses = no call)
say_hello = greet
print(say_hello("Alice"))  # Hello, Alice!

# Pass function as an argument
def apply_twice(func, value):
    return func(func(value))

def add_exclaim(text):
    return text + "!"

print(apply_twice(add_exclaim, "wow"))  # wow!!

# Functions defined inside functions (closures)
def make_multiplier(factor):
    def multiply(x):
        return x * factor  # 'factor' is captured from enclosing scope
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))  # 10
print(triple(5))  # 15

That make_multiplier function returns a new function. The inner function "remembers" the factor value even after make_multiplier has finished. This is called a closure — and it is the mechanism that makes decorators work.

Building a Decorator from Scratch

A decorator is a function that takes a function, wraps it with additional behavior, and returns the wrapper:

def shout(func):
    """Decorator that uppercases the return value."""
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result.upper()
    return wrapper

# Manual decoration
def greet(name):
    return f"hello, {name}"

greet = shout(greet)  # Replace greet with the wrapped version
print(greet("Alice"))  # HELLO, ALICE

That greet = shout(greet) line is what the @ syntax does automatically:

# The @ syntax — identical to the manual version above
@shout
def greet(name):
    return f"hello, {name}"

print(greet("Alice"))  # HELLO, ALICE

The @ syntax is pure syntactic sugar. @shout above def greet means "after defining greet, pass it to shout() and rebind the name greet to whatever shout returns."

Why *args and **kwargs Matter

The wrapper must accept any arguments the original function might receive. That is why we use *args, **kwargs — they capture all positional and keyword arguments, regardless of the decorated function's signature.

def log_calls(func):
    def wrapper(*args, **kwargs):
        arg_str = ", ".join(
            [repr(a) for a in args] +
            [f"{k}={v!r}" for k, v in kwargs.items()]
        )
        print(f"→ {func.__name__}({arg_str})")
        result = func(*args, **kwargs)
        print(f"← {func.__name__} returned {result!r}")
        return result
    return wrapper

@log_calls
def add(a, b):
    return a + b

@log_calls
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

add(3, 4)
# → add(3, 4)
# ← add returned 7

greet("Alice", greeting="Hi")
# → greet('Alice', greeting='Hi')
# ← greet returned 'Hi, Alice!'

Preserving Function Metadata with functools.wraps

There is a subtle problem. After decoration, the function's __name__ and __doc__ attributes point to the wrapper, not the original function. This breaks documentation tools and debugging:

def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@bad_decorator
def my_function():
    """This is my function."""
    pass

print(my_function.__name__)  # wrapper  ← wrong!
print(my_function.__doc__)   # None     ← lost!

The fix is functools.wraps:

import functools

def good_decorator(func):
    @functools.wraps(func)  # Copy metadata from func to wrapper
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@good_decorator
def my_function():
    """This is my function."""
    pass

print(my_function.__name__)  # my_function ✓
print(my_function.__doc__)   # This is my function. ✓
Critical: Always use @functools.wraps(func) in your wrapper. Without it, stack traces show "wrapper" instead of the actual function name, help() shows wrong docs, and tools like Sphinx or pytest can behave unpredictably. This is a non-negotiable best practice.

Advanced Decorators

Decorators with Arguments

What if you want your decorator to be configurable? For example, a retry decorator where you specify the number of attempts. You need one more layer of nesting — a decorator factory:

import functools
import time

def retry(max_attempts=3, delay=1.0):
    """Decorator factory: retry a function on failure."""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_exception = None
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_exception = e
                    print(f"Attempt {attempt}/{max_attempts} failed: {e}")
                    if attempt < max_attempts:
                        time.sleep(delay)
            raise last_exception
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.5)
def fetch_data(url):
    """Simulate flaky network call."""
    import random
    if random.random() < 0.7:
        raise ConnectionError("Network timeout")
    return {"data": "success"}

# fetch_data("https://api.example.com")
# Attempt 1/3 failed: Network timeout
# Attempt 2/3 failed: Network timeout
# {'data': 'success'}  (if third attempt succeeds)

The pattern is three levels deep: retry() returns decorator, which takes the function and returns wrapper. It looks intimidating at first, but the structure is always the same.

Stacking Multiple Decorators

You can apply multiple decorators to a single function. They are applied bottom-up (closest to the function first):

import functools
import time

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"⏱ {func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

def validate_positive(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        for arg in args:
            if isinstance(arg, (int, float)) and arg < 0:
                raise ValueError(f"Expected positive, got {arg}")
        return func(*args, **kwargs)
    return wrapper

@timer                  # Applied second (outermost)
@validate_positive      # Applied first (innermost)
def compute(x, y):
    time.sleep(0.1)
    return x ** y

print(compute(2, 10))  # ⏱ compute took 0.10xxs \n 1024
# compute(-1, 5)       # ValueError: Expected positive, got -1

Think of it as timer(validate_positive(compute)). The call flows outward: validation first, then timing.

Key Insight: Decorator stacking is like function composition in math: f(g(x)). The bottom decorator wraps first, so it is "closest" to the original function. Understanding this order is critical when the decorators interact — for example, authentication should run before logging.

Class Decorators

Decorators are not limited to functions. You can decorate entire classes to add functionality:

def add_repr(cls):
    """Class decorator: auto-generate __repr__ from __init__ params."""
    import inspect
    params = list(inspect.signature(cls.__init__).parameters.keys())[1:]  # skip self

    def __repr__(self):
        values = ", ".join(f"{p}={getattr(self, p)!r}" for p in params)
        return f"{cls.__name__}({values})"

    cls.__repr__ = __repr__
    return cls

@add_repr
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(3, 4)
print(p)  # Point(x=3, y=4)

You can also use a class as a decorator (the class implements __call__):

class CountCalls:
    """Class-based decorator: count how many times a function is called."""
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"{self.func.__name__} called {self.count} time(s)")
        return self.func(*args, **kwargs)

@CountCalls
def say_hello(name):
    return f"Hello, {name}!"

say_hello("Alice")  # say_hello called 1 time(s)
say_hello("Bob")    # say_hello called 2 time(s)
print(say_hello.count)  # 2 — access state on the decorator

Common Built-in Decorators

Python comes with several built-in decorators you have already seen or will use constantly:

class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):
        """Getter — accessed like an attribute."""
        return self._radius

    @radius.setter
    def radius(self, value):
        """Setter — validated attribute assignment."""
        if value <= 0:
            raise ValueError("Radius must be positive")
        self._radius = value

    @property
    def area(self):
        """Computed property — no stored value."""
        import math
        return math.pi * self._radius ** 2

    @staticmethod
    def is_valid_radius(value):
        """No self or cls needed."""
        return isinstance(value, (int, float)) and value > 0

    @classmethod
    def unit_circle(cls):
        """Alternative constructor."""
        return cls(radius=1)

c = Circle.unit_circle()
print(c.area)            # 3.14159...
c.radius = 5
print(c.area)            # 78.5398...
print(Circle.is_valid_radius(-1))  # False

Practical Use Cases

Here are three production-quality decorators you will use in real projects:

import functools
import time

# 1. TIMING — measure function execution time
def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"⏱ {func.__name__}: {elapsed:.4f}s")
        return result
    return wrapper

# 2. CACHING — memoize expensive computations
def memoize(func):
    cache = {}
    @functools.wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    wrapper.cache = cache  # Expose cache for inspection
    return wrapper

@memoize
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(50))  # 12586269025 — instant with memoization
print(len(fibonacci.cache))  # 51 cached values

# Python has a built-in version:
@functools.lru_cache(maxsize=128)
def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

# 3. LOGGING — structured function call logging
import logging

def log_call(level=logging.INFO):
    def decorator(func):
        logger = logging.getLogger(func.__module__)
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            logger.log(level, f"Calling {func.__name__}")
            try:
                result = func(*args, **kwargs)
                logger.log(level, f"{func.__name__} returned successfully")
                return result
            except Exception as e:
                logger.exception(f"{func.__name__} raised {type(e).__name__}")
                raise
        return wrapper
    return decorator

@timer
@log_call(level=logging.DEBUG)
def train_model(epochs):
    time.sleep(0.1)  # Simulate training
    return {"accuracy": 0.95}
Key Insight: Python's functools.lru_cache is a battle-tested memoization decorator. Use it instead of rolling your own. For more control, functools.cache (Python 3.9+) is an unbounded version. These are indispensable for caching expensive computations in data pipelines — feature extraction, API calls, database queries.

Putting It All Together

Let's build a mini framework that uses decorators to register and validate API-like functions — a simplified version of what Flask and FastAPI do:

import functools
import time
import json

class APIFramework:
    """Miniature decorator-based API framework."""
    def __init__(self):
        self.routes = {}

    def route(self, path, methods=("GET",)):
        """Register a function as an API route handler."""
        def decorator(func):
            @functools.wraps(func)
            def wrapper(*args, **kwargs):
                start = time.perf_counter()
                try:
                    result = func(*args, **kwargs)
                    elapsed = time.perf_counter() - start
                    return {
                        "status": 200,
                        "data": result,
                        "time_ms": round(elapsed * 1000, 2)
                    }
                except Exception as e:
                    return {"status": 500, "error": str(e)}
            self.routes[path] = {
                "handler": wrapper,
                "methods": methods
            }
            return wrapper
        return decorator

    def handle(self, path, **kwargs):
        """Simulate handling a request."""
        if path not in self.routes:
            return {"status": 404, "error": "Not found"}
        return self.routes[path]["handler"](**kwargs)

app = APIFramework()

@app.route("/users", methods=("GET",))
def list_users():
    return [{"name": "Alice"}, {"name": "Bob"}]

@app.route("/add", methods=("POST",))
def add_numbers(a, b):
    return {"sum": a + b}

# Simulate requests
print(json.dumps(app.handle("/users"), indent=2))
print(json.dumps(app.handle("/add", a=3, b=7), indent=2))
print(json.dumps(app.handle("/missing"), indent=2))

This is 40 lines of code, yet it demonstrates route registration, automatic timing, error handling, and a request dispatcher — all powered by decorators. Flask's @app.route() works on exactly this principle.
