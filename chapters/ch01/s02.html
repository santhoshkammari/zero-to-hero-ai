Data Types and Variables

Chapter 1: Python & Programming Foundations 4 subtopics

TL;DR

Every value in Python has a type — integer, float, string, boolean, or None — and the type determines what operations you can do with it. Python figures out types automatically (dynamic typing), but understanding them is essential because type mismatches are one of the most common sources of bugs. Strings are immutable sequences of characters with powerful built-in methods. Booleans drive all decision-making in your code.

Numeric Types

Integers

An integer (type int) is a whole number — no decimal point. In Python, integers have unlimited precision. That means you can work with numbers as large as your computer's memory allows, unlike most languages that cap out at around 9 quintillion.

count = 42
negative = -17
huge = 10 ** 100   # a 1 followed by 100 zeros — no overflow, no problem
print(type(count))  # <class 'int'>

# Underscores for readability (Python 3.6+)
population = 7_900_000_000
print(population)   # 7900000000

Floating Point Numbers

A float is a number with a decimal point. Under the hood, Python uses 64-bit IEEE 754 double-precision representation. That's a fancy way of saying: floats are approximations. They can represent very large and very small numbers, but not all decimal values exactly.

temperature = 98.6
tiny = 0.000001
scientific = 2.5e10     # 2.5 × 10^10 = 25,000,000,000

# The infamous floating-point surprise
print(0.1 + 0.2)        # 0.30000000000000004, not 0.3!
print(0.1 + 0.2 == 0.3) # False!
Watch Out: Never compare floats with ==. Instead, check if they're "close enough." Python provides math.isclose() for this: 
import math
print(math.isclose(0.1 + 0.2, 0.3))  # True
This float imprecision isn't a Python bug — it's how all computers store decimal numbers in binary. It matters enormously in ML when you're computing loss values or gradients.

Complex Numbers

Python has built-in support for complex numbers — numbers with a real part and an imaginary part. The imaginary part is denoted with j (not i like in math textbooks).

z = 3 + 4j
print(z.real)    # 3.0
print(z.imag)    # 4.0
print(abs(z))    # 5.0 — the magnitude (distance from origin)

# You'll encounter these in signal processing and certain linear algebra operations
# For most ML work, you won't use them directly

Arithmetic Operators

Python gives you the standard math operations, plus a few extras that come up constantly in programming:

a, b = 17, 5

print(a + b)    # 22   — addition
print(a - b)    # 12   — subtraction
print(a * b)    # 85   — multiplication
print(a / b)    # 3.4  — true division (always returns float)
print(a // b)   # 3    — floor division (rounds down to integer)
print(a % b)    # 2    — modulo (remainder after division)
print(a ** b)   # 1419857 — exponentiation (17 to the power of 5)

# Floor division and modulo are used everywhere in ML
# Example: splitting data into batches
total_samples = 1000
batch_size = 32
num_full_batches = total_samples // batch_size   # 31
leftover_samples = total_samples % batch_size    # 8

Operator Precedence

Python follows standard math order: exponentiation first, then multiplication/division, then addition/subtraction. When in doubt, use parentheses — they make your intent explicit.

# Without parentheses — relies on precedence rules
result = 2 + 3 * 4 ** 2    # 2 + 3 * 16 = 2 + 48 = 50

# With parentheses — intent is crystal clear
result = 2 + (3 * (4 ** 2))  # same: 50
result = (2 + 3) * 4 ** 2    # different: 5 * 16 = 80

Type Conversion

You can convert between numeric types explicitly using int(), float(), and str(). This is called type casting.

# float to int — truncates (does NOT round)
print(int(3.7))     # 3
print(int(-3.7))    # -3

# int to float
print(float(42))    # 42.0

# string to number (if the string contains a valid number)
print(int("100"))     # 100
print(float("3.14"))  # 3.14

# number to string
print(str(42))        # "42"
print(str(3.14))      # "3.14"

# This fails — Python won't guess what you mean
# int("3.14")   → ValueError
# int("hello")  → ValueError
Common Mistake: Confusing int() truncation with rounding. int(3.9) gives you 3, not 4. If you want rounding, use round(3.9) which gives 4. And round(2.5) gives 2, not 3 — Python uses banker's rounding (rounds to the nearest even number), which is more statistically fair.

Strings

String Creation

A string (type str) is a sequence of characters. You can create strings with single quotes, double quotes, or triple quotes — they're all equivalent, except triple quotes can span multiple lines.

single = 'Hello'
double = "Hello"              # identical to single quotes
contains_apostrophe = "It's"  # use double quotes when string has '
contains_quotes = 'She said "hi"'  # use single quotes when string has "

multiline = """This string
spans multiple
lines."""
print(multiline)
# This string
# spans multiple
# lines.

String Indexing and Slicing

Every character in a string has a position number called an index. Python counts from zero — the first character is at index 0, not 1. You can also count from the end using negative indices.

word = "Python"
#        P  y  t  h  o  n
# index: 0  1  2  3  4  5
# neg:  -6 -5 -4 -3 -2 -1

print(word[0])     # 'P'  — first character
print(word[-1])    # 'n'  — last character
print(word[2])     # 't'  — third character

# Slicing: word[start:stop] — includes start, excludes stop
print(word[0:3])   # 'Pyt'  — characters 0, 1, 2
print(word[2:5])   # 'tho'
print(word[:3])    # 'Pyt'  — from beginning to index 3
print(word[3:])    # 'hon'  — from index 3 to end
print(word[::2])   # 'Pto'  — every 2nd character
print(word[::-1])  # 'nohtyP' — reversed string
Key Insight: Slicing uses the "fence post" model. Think of indices as pointing between characters, like fence posts between fence panels. word[2:5] grabs everything between post 2 and post 5 — that's characters at positions 2, 3, 4. This is why the stop index is excluded.

String Methods

Strings come loaded with methods — functions attached to the string object. Here are the ones you'll use daily:

text = "  Hello, World!  "

# Case transformation
print(text.upper())          # '  HELLO, WORLD!  '
print(text.lower())          # '  hello, world!  '

# Whitespace removal
print(text.strip())          # 'Hello, World!' — both sides
print(text.lstrip())         # 'Hello, World!  ' — left only
print(text.rstrip())         # '  Hello, World!' — right only

# Splitting and joining
csv_line = "Alice,28,Engineer"
parts = csv_line.split(",")  # ['Alice', '28', 'Engineer']
rejoined = " | ".join(parts) # 'Alice | 28 | Engineer'

# Searching
print("World" in text)       # True
print(text.find("World"))    # 9 (index where it starts)
print(text.find("Mars"))     # -1 (not found)

# Replacing
print(text.replace("World", "Python"))  # '  Hello, Python!  '

# Chaining methods
clean = "  messy DATA  ".strip().lower().replace(" ", "_")
print(clean)  # 'messy_data'

String Formatting

You'll constantly need to build strings that include variable values. Python gives you three ways to do this. Use f-strings — they're the most readable and the most modern.

name = "Alice"
age = 28
accuracy = 0.9534

# f-strings (Python 3.6+) — the best option
print(f"Name: {name}, Age: {age}")
print(f"Accuracy: {accuracy:.2%}")    # 'Accuracy: 95.34%'
print(f"Pi is approximately {3.14159:.3f}")  # 'Pi is approximately 3.142'

# .format() method — older but still common in existing code
print("Name: {}, Age: {}".format(name, age))

# %-formatting — oldest style, avoid in new code
print("Name: %s, Age: %d" % (name, age))

Escape Characters and Raw Strings

Some characters have special meaning inside strings. You use a backslash \ to include them:

print("Line one\nLine two")     # \n = newline
print("Column1\tColumn2")       # \t = tab
print("She said \"hello\"")     # \" = literal quote
print("C:\\Users\\Documents")   # \\ = literal backslash

# Raw strings ignore escape characters — useful for file paths and regex
path = r"C:\Users\new_folder\test"
print(path)  # C:\Users\new_folder\test (no escape interpretation)

String Immutability

Strings in Python are immutable — once created, you cannot change individual characters. Every operation that seems to "modify" a string actually creates a brand new string.

word = "Hello"
# word[0] = "J"    # TypeError: 'str' object does not support item assignment

# Instead, create a new string
word = "J" + word[1:]
print(word)  # 'Jello'

This matters for performance when you're building strings in a loop. Concatenating with + inside a loop creates a new string every iteration. For large-scale string building, use "".join(list_of_strings) instead.

Booleans and None

Boolean Values

A boolean (type bool) has exactly two possible values: True and False. That's it. Booleans are the foundation of all decision-making in your code — every if statement, every while loop, ultimately evaluates to a boolean.

is_raining = True
has_umbrella = False
print(type(is_raining))  # <class 'bool'>

# Booleans are secretly integers: True is 1, False is 0
print(True + True)    # 2
print(False * 100)    # 0
print(sum([True, False, True, True]))  # 3 — counts True values!

Comparison Operators

Comparison operators compare two values and return a boolean:

x = 10
print(x == 10)    # True  — equal to
print(x != 5)     # True  — not equal to
print(x > 5)      # True  — greater than
print(x < 5)      # False — less than
print(x >= 10)    # True  — greater than or equal to
print(x <= 9)     # False — less than or equal to

# You can chain comparisons — this is uniquely Pythonic
age = 25
print(18 <= age <= 65)  # True — is age between 18 and 65?
# This is equivalent to: 18 <= age and age <= 65

Logical Operators

Logical operators combine booleans. Python uses English words instead of symbols: and, or, not.

age = 25
has_id = True

# and — both must be True
can_enter = age >= 18 and has_id
print(can_enter)  # True

# or — at least one must be True
is_weekend = False
is_holiday = True
day_off = is_weekend or is_holiday
print(day_off)  # True

# not — flips the boolean
print(not True)   # False
print(not False)  # True

# Combining all three
has_ticket = True
is_vip = False
can_attend = has_ticket and (is_vip or age >= 18)
print(can_attend)  # True

Truthy and Falsy Values

Here's something that catches people off guard. In Python, every value has a boolean interpretation. When you use a non-boolean value in a boolean context (like an if statement), Python converts it automatically.

These values are falsy — they evaluate to False:

# All of these are falsy:
bool(0)        # False — zero
bool(0.0)      # False — zero float
bool("")       # False — empty string
bool([])       # False — empty list
bool({})       # False — empty dict
bool(None)     # False — None

# Everything else is truthy:
bool(42)       # True — any non-zero number
bool("hello")  # True — any non-empty string
bool([1, 2])   # True — any non-empty list
Key Insight: This is why you see Python code like if my_list: instead of if len(my_list) > 0:. The shorter form is idiomatic Python. An empty list is falsy, a non-empty list is truthy. Same goes for strings, dicts, and sets.

None Type and Its Uses

None is Python's way of saying "this variable has no value" or "nothing here." It's not zero. It's not an empty string. It's the absence of a value. There's exactly one None object in all of Python.

# Functions that don't explicitly return something return None
def greet(name):
    print(f"Hello, {name}")

result = greet("Alice")  # prints "Hello, Alice"
print(result)            # None

# Use None as a placeholder or default
user_input = None   # we haven't received input yet
middle_name = None  # not everyone has one

Identity Operators

The is operator checks if two variables point to the exact same object in memory, not whether they have equal values. This is different from ==.

# Always use 'is' to check for None
x = None
print(x is None)      # True
print(x == None)       # True, but don't do this — 'is' is correct

# The difference between == and is
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)    # True — same values
print(a is b)    # False — different objects in memory

c = a            # c points to the SAME object as a
print(a is c)    # True — same object
Watch Out: Always use is None and is not None, never == None. A class can override == to return True when compared to None, which would be a bug. The is operator cannot be overridden.

Type System

Dynamic Typing

Python is dynamically typed, which means you don't declare what type a variable is. The type is determined at runtime by whatever value the variable holds at that moment. You can reassign a variable to a completely different type.

x = 42          # x is an int
x = "hello"     # now x is a str — no error
x = [1, 2, 3]   # now x is a list — still no error

This is the opposite of static typing, used in languages like Java or C, where you must declare int x = 42; and then x = "hello" would be a compile-time error.

Dynamic typing makes Python fast to write. The downside? Type-related bugs hide until runtime. You won't discover that you accidentally passed a string where a number was expected until that line actually executes.

type() and isinstance()

The type() function tells you exactly what type a value is. The isinstance() function checks whether a value is a specific type (or one of several types) — and it respects inheritance.

print(type(42))          # <class 'int'>
print(type(3.14))        # <class 'float'>
print(type("hello"))     # <class 'str'>
print(type(True))        # <class 'bool'>

# isinstance is preferred for type checking
print(isinstance(42, int))           # True
print(isinstance(42, (int, float)))  # True — is it int OR float?
print(isinstance(True, int))         # True — bool is a subclass of int!

# Practical use: handling mixed input
def double(value):
    if isinstance(value, (int, float)):
        return value * 2
    elif isinstance(value, str):
        return value + value
    else:
        raise TypeError(f"Cannot double {type(value)}")

Type Hints and Annotations

Type hints (added in Python 3.5) let you annotate what types your functions expect and return. Python does not enforce these — they're documentation for humans and tools. But they're incredibly valuable as your projects grow.

def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """Calculate Body Mass Index."""
    return weight_kg / (height_m ** 2)

# The annotations don't prevent wrong usage at runtime
# calculate_bmi("heavy", "tall")  # Python won't stop this!

# But tools like mypy (a static type checker) will catch the error
# before you run the code

# Common type hints you'll see
from typing import List, Dict, Optional, Tuple

def process_scores(scores: List[float]) -> Dict[str, float]:
    return {
        "mean": sum(scores) / len(scores),
        "max": max(scores),
        "min": min(scores),
    }

# Optional means "this type OR None"
def find_user(user_id: int) -> Optional[str]:
    users = {1: "Alice", 2: "Bob"}
    return users.get(user_id)  # returns None if not found

Type hints cannot enforce types at runtime. They also cannot express complex constraints like "a list with exactly 3 elements" or "an integer between 0 and 100." For that, you'd need runtime validation logic.
