Data Structures

Chapter 1: Python & Programming Foundations 4 subtopics

TL;DR

Data structures are containers that organize values so you can access, modify, and search through them efficiently. Python gives you four core ones: lists (ordered, mutable), tuples (ordered, immutable), dictionaries (key-value pairs), and sets (unique elements, no order). Choosing the right one is about understanding what operations you need — fast lookups, ordered sequences, or guaranteed uniqueness.

Lists

Real-world problem: you have a collection of items — test scores, file names, pixel values — and you need to keep them in order, access them by position, and add or remove items as your program runs. That's a list.

Creating Lists

A list is an ordered, mutable collection of values enclosed in square brackets. "Ordered" means items stay in the order you put them. "Mutable" means you can change the list after creating it.

scores = [85, 92, 78, 95, 88]
names = ["Alice", "Bob", "Charlie"]
mixed = [42, "hello", 3.14, True]  # lists can hold different types
empty = []

# list() constructor — converts other iterables to lists
letters = list("Python")  # ['P', 'y', 't', 'h', 'o', 'n']
numbers = list(range(5))  # [0, 1, 2, 3, 4]

Indexing and Slicing

Lists use the same indexing and slicing rules as strings — zero-based, negative indices from the end, and the [start:stop:step] pattern.

colors = ["red", "green", "blue", "yellow", "purple"]

print(colors[0])      # 'red'
print(colors[-1])     # 'purple'
print(colors[1:3])    # ['green', 'blue']
print(colors[::2])    # ['red', 'blue', 'purple'] — every 2nd item

# Unlike strings, you CAN modify list elements
colors[0] = "crimson"
print(colors)  # ['crimson', 'green', 'blue', 'yellow', 'purple']

# Slice assignment — replace a range of elements
colors[1:3] = ["lime", "navy"]
print(colors)  # ['crimson', 'lime', 'navy', 'yellow', 'purple']

List Methods

Lists come with a rich set of methods. These modify the list in place — they change the original list rather than returning a new one.

tasks = ["write code", "test code"]

# Adding elements
tasks.append("deploy")               # adds to the end
tasks.insert(1, "review code")       # inserts at index 1
tasks.extend(["monitor", "iterate"]) # adds multiple items from another list

print(tasks)
# ['write code', 'review code', 'test code', 'deploy', 'monitor', 'iterate']

# Removing elements
tasks.remove("monitor")   # removes first occurrence by value
last = tasks.pop()         # removes and returns last item ('iterate')
second = tasks.pop(1)     # removes and returns item at index 1 ('review code')

# Sorting
numbers = [3, 1, 4, 1, 5, 9, 2, 6]
numbers.sort()             # sorts in place: [1, 1, 2, 3, 4, 5, 6, 9]
numbers.sort(reverse=True) # descending: [9, 6, 5, 4, 3, 2, 1, 1]

# sorted() returns a NEW list, leaving the original unchanged
original = [3, 1, 4, 1, 5]
ordered = sorted(original)
print(original)  # [3, 1, 4, 1, 5] — untouched
print(ordered)   # [1, 1, 3, 4, 5]

# Reversing
numbers.reverse()  # reverses in place

List Mutability

Because lists are mutable, assigning a list to a new variable doesn't create a copy — both names point to the same list in memory. Changing one changes the other.

a = [1, 2, 3]
b = a           # b is NOT a copy — it's the same list
b.append(4)
print(a)        # [1, 2, 3, 4] — surprise! a changed too

# This is called aliasing, and it's a frequent source of bugs

Nested Lists

Lists can contain other lists. This is how you represent grids, matrices, and tables before reaching for NumPy.

# A 3x3 grid (matrix)
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
print(matrix[0])      # [1, 2, 3] — first row
print(matrix[0][1])   # 2 — first row, second column
print(matrix[2][2])   # 9 — third row, third column

# Iterating over a matrix
for row in matrix:
    for value in row:
        print(value, end=" ")
    print()  # newline after each row

List Copying: Shallow vs Deep

A shallow copy creates a new list, but the elements inside still point to the same objects. For simple lists (numbers, strings), this is fine. For nested lists, it's a trap.

import copy

# Shallow copy — three ways to do it
original = [1, 2, [3, 4]]
shallow1 = original.copy()
shallow2 = original[:]
shallow3 = list(original)

# The outer list is a new object...
shallow1.append(5)
print(original)   # [1, 2, [3, 4]] — original unaffected

# ...but the inner list is SHARED
shallow1[2].append(99)
print(original)   # [1, 2, [3, 4, 99]] — original IS affected!

# Deep copy — recursively copies everything
original = [1, 2, [3, 4]]
deep = copy.deepcopy(original)
deep[2].append(99)
print(original)   # [1, 2, [3, 4]] — original safe
Common Mistake: Forgetting that .copy() and slicing ([:]) only make shallow copies. If your list contains other lists, dicts, or objects, changes to those nested objects will propagate back to the original. Use copy.deepcopy() when you need true independence.

List Membership

The in operator checks whether a value exists in a list. It scans the entire list, so for very large lists, this is slow — O(n) time. If you need fast membership checks, use a set instead.

fruits = ["apple", "banana", "cherry"]
print("banana" in fruits)      # True
print("mango" not in fruits)   # True

Tuples

A tuple looks like a list but uses parentheses instead of brackets, and it's immutable — once created, you cannot add, remove, or change elements. That sounds limiting, but immutability is a feature, not a bug.

Creating Tuples

point = (3, 4)
rgb = (255, 128, 0)
single = (42,)     # note the comma — without it, (42) is just the number 42
empty = ()

# Parentheses are optional in many contexts
coordinates = 10, 20, 30   # this is a tuple too
print(type(coordinates))    # <class 'tuple'>

Tuple Immutability

point = (3, 4)
# point[0] = 5    # TypeError: 'tuple' object does not support item assignment

# You CAN access elements with indexing, though
print(point[0])    # 3
print(point[-1])   # 4

Tuple Unpacking

Unpacking lets you assign each element of a tuple to a separate variable in one clean line. This is one of Python's most elegant features.

# Basic unpacking
x, y = (3, 4)
print(x)  # 3
print(y)  # 4

# Functions that return multiple values actually return tuples
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([4, 2, 9, 1, 7])
print(lo, hi)  # 1 9

# Star unpacking — grab "the rest"
first, *middle, last = [1, 2, 3, 4, 5]
print(first)    # 1
print(middle)   # [2, 3, 4]
print(last)     # 5

# Swap variables — this is tuple unpacking under the hood
a, b = 1, 2
a, b = b, a     # a=2, b=1

Named Tuples

When you have tuples where each position has a specific meaning — like (x, y) coordinates or (name, age, city) records — named tuples let you access fields by name instead of position.

from collections import namedtuple

Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
print(p.x)     # 3 — much clearer than p[0]
print(p.y)     # 4
print(p[0])    # 3 — still works by index

# Named tuples are still immutable
# p.x = 5     # AttributeError

When to Use Tuples vs Lists

Use tuples when the data shouldn't change — function return values, dictionary keys, coordinates, database records. Use lists when you need to add, remove, or sort items. Tuples are also slightly faster and use less memory than lists, which matters when you have millions of them.

Dictionaries

Problem: you have data where each value is associated with a unique label. Student names mapped to grades. Configuration settings mapped to values. Words mapped to definitions. You need to look things up by label instantly, not by scanning through a list. That's a dictionary.

Creating Dictionaries

A dictionary (type dict) stores key-value pairs. Keys must be unique and immutable (strings, numbers, tuples). Values can be anything.

student = {
    "name": "Alice",
    "age": 22,
    "grades": [85, 92, 78]
}

# dict() constructor
config = dict(learning_rate=0.001, epochs=100, batch_size=32)

empty = {}  # empty dictionary

Accessing and Modifying Values

student = {"name": "Alice", "age": 22, "major": "CS"}

# Access by key
print(student["name"])     # 'Alice'

# student["gpa"]           # KeyError! — key doesn't exist

# .get() returns None (or a default) instead of crashing
print(student.get("gpa"))          # None
print(student.get("gpa", 0.0))    # 0.0 — custom default

# Add or update a key
student["gpa"] = 3.8           # adds new key
student["age"] = 23            # updates existing key

# Remove a key
del student["major"]
popped = student.pop("age")   # removes and returns the value (23)
Key Insight: Dictionary lookups by key are O(1) — constant time, regardless of dictionary size. This is because dictionaries use a hash table internally. A list lookup by value is O(n) — you have to scan through every element. If you're doing frequent lookups, dictionaries are dramatically faster.

Dictionary Methods

settings = {"color": "blue", "size": 12, "bold": True}

# Get all keys, values, or key-value pairs
print(list(settings.keys()))    # ['color', 'size', 'bold']
print(list(settings.values()))  # ['blue', 12, True]
print(list(settings.items()))   # [('color', 'blue'), ('size', 12), ('bold', True)]

# Update — merge another dictionary into this one
settings.update({"size": 14, "font": "Arial"})
print(settings)
# {'color': 'blue', 'size': 14, 'bold': True, 'font': 'Arial'}

# Check if a key exists
print("color" in settings)     # True
print("height" in settings)    # False

Iterating Over Dictionaries

scores = {"Alice": 95, "Bob": 87, "Charlie": 92}

# Loop over keys (default)
for name in scores:
    print(name)

# Loop over values
for score in scores.values():
    print(score)

# Loop over both — this is the most common pattern
for name, score in scores.items():
    print(f"{name}: {score}")
# Alice: 95
# Bob: 87
# Charlie: 92

Nested Dictionaries

Dictionaries can contain other dictionaries. This is how you represent structured, hierarchical data — and it's the format that maps directly to JSON, the data exchange format used by virtually every API.

model_config = {
    "architecture": {
        "type": "transformer",
        "layers": 12,
        "attention_heads": 8
    },
    "training": {
        "learning_rate": 0.001,
        "batch_size": 32,
        "epochs": 100
    }
}

# Access nested values
lr = model_config["training"]["learning_rate"]
print(lr)  # 0.001

Dictionary Comprehensions

Just like list comprehensions, you can create dictionaries in a single expression:

# Square of each number
squares = {n: n**2 for n in range(6)}
print(squares)  # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# Invert a dictionary (swap keys and values)
grades = {"Alice": "A", "Bob": "B", "Charlie": "A"}
grade_to_students = {}
for name, grade in grades.items():
    grade_to_students.setdefault(grade, []).append(name)
print(grade_to_students)  # {'A': ['Alice', 'Charlie'], 'B': ['Bob']}

defaultdict and OrderedDict

The collections module provides specialized dictionary variants. defaultdict automatically creates a default value for missing keys, eliminating the "check if key exists before updating" pattern.

from collections import defaultdict

# Count word frequencies
words = ["apple", "banana", "apple", "cherry", "banana", "apple"]

# Without defaultdict — tedious
counts_manual = {}
for word in words:
    if word in counts_manual:
        counts_manual[word] += 1
    else:
        counts_manual[word] = 1

# With defaultdict — clean
counts = defaultdict(int)  # missing keys default to int() which is 0
for word in words:
    counts[word] += 1

print(dict(counts))  # {'apple': 3, 'banana': 2, 'cherry': 1}

# Group items
from collections import defaultdict
students_by_grade = defaultdict(list)
for name, grade in [("Alice", "A"), ("Bob", "B"), ("Charlie", "A")]:
    students_by_grade[grade].append(name)
print(dict(students_by_grade))  # {'A': ['Alice', 'Charlie'], 'B': ['Bob']}

Sets

A set is an unordered collection of unique elements. If you add a duplicate, the set ignores it. Sets are optimized for two things: checking whether an item exists (membership testing) and mathematical set operations like union and intersection.

Creating Sets

colors = {"red", "green", "blue"}
print(type(colors))  # <class 'set'>

# Duplicates are automatically removed
numbers = {1, 2, 2, 3, 3, 3}
print(numbers)  # {1, 2, 3}

# set() from a list — great for deduplication
raw_data = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4]
unique = set(raw_data)
print(unique)  # {1, 2, 3, 4}

# Empty set — must use set(), not {} (which creates an empty dict)
empty = set()

Set Operations

Sets support the mathematical operations you'd expect from set theory:

python_devs = {"Alice", "Bob", "Charlie", "Diana"}
ml_devs = {"Charlie", "Diana", "Eve", "Frank"}

# Union — everyone in either set
print(python_devs | ml_devs)
# {'Alice', 'Bob', 'Charlie', 'Diana', 'Eve', 'Frank'}

# Intersection — people in both sets
print(python_devs & ml_devs)
# {'Charlie', 'Diana'}

# Difference — in python_devs but NOT in ml_devs
print(python_devs - ml_devs)
# {'Alice', 'Bob'}

# Symmetric difference — in one but not both
print(python_devs ^ ml_devs)
# {'Alice', 'Bob', 'Eve', 'Frank'}

Set Methods

tags = {"python", "ml"}

tags.add("deep-learning")      # add one element
tags.discard("java")           # remove if present, no error if absent
tags.remove("ml")              # remove, KeyError if absent
# tags.remove("java")          # KeyError!

print(tags)  # {'python', 'deep-learning'}

Frozen Sets

A frozenset is an immutable set. You can't add or remove elements. Because they're immutable, they can be used as dictionary keys or elements of another set — regular sets cannot.

fs = frozenset([1, 2, 3])
# fs.add(4)  # AttributeError — immutable

# Use as dictionary key
feature_sets = {
    frozenset(["age", "income"]): 0.85,
    frozenset(["age", "income", "education"]): 0.91,
}
print(feature_sets[frozenset(["age", "income"])])  # 0.85

Use Cases for Sets

Two patterns come up constantly. Deduplication: convert a list to a set and back. Fast membership testing: checking x in my_set is O(1), while x in my_list is O(n).

# Deduplication while preserving order (Python 3.7+)
data = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5]
unique_ordered = list(dict.fromkeys(data))
print(unique_ordered)  # [3, 1, 4, 5, 9, 2, 6]

# Fast membership testing
valid_commands = {"start", "stop", "pause", "resume"}
user_input = "stop"
if user_input in valid_commands:   # O(1) lookup
    print(f"Executing: {user_input}")

# Finding common elements between datasets
train_ids = set(range(0, 800))
test_ids = set(range(750, 1000))
leaked = train_ids & test_ids
print(f"Data leakage: {len(leaked)} samples overlap")
# Data leakage: 50 samples overlap
Watch Out: Sets are unordered. You cannot access elements by index (my_set[0] is an error), and the order you see when printing may differ between runs. If you need both uniqueness and order, use dict.fromkeys() as shown above.
