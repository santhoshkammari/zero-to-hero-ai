File I/O

Chapter 1: Python & Programming Foundations 3 subtopics

TL;DR

Every ML project reads data in and writes results out — CSV datasets, JSON configs, model checkpoints, log files. Python gives you open() for raw file access, dedicated modules for structured formats like CSV and JSON, and the pathlib/os modules for navigating the file system. Understanding file I/O is the first step toward building data pipelines that ingest real-world data instead of toy arrays.

Why File I/O Matters

Your model is only as good as the data you feed it. And that data lives in files — CSV exports from databases, JSON from APIs, text corpora for NLP, binary blobs for images. Before you can call model.fit(X, y), you need to load X and y from somewhere. After training, you need to save weights, metrics, and predictions. File I/O is the bridge between the filesystem and your code.

Reading and Writing Files

The open() Function

The open() function is your gateway to files. It returns a file object that you use to read or write:

# Basic signature
f = open("data.txt", mode="r", encoding="utf-8")
content = f.read()
f.close()  # Always close when done!

File Modes

The mode parameter controls what you can do with the file:

# Text modes
"r"   # Read (default) — file must exist
"w"   # Write — creates new or TRUNCATES existing (deletes content!)
"a"   # Append — creates new or adds to end of existing
"x"   # Exclusive create — fails if file already exists

# Binary modes (add 'b')
"rb"  # Read binary (images, pickled objects, model weights)
"wb"  # Write binary

# Combined modes
"r+"  # Read and write (file must exist)
"w+"  # Write and read (truncates first)
Critical: Mode "w" destroys existing file content immediately on open — before you write a single byte. If you meant to append, use "a". This mistake has destroyed many a dataset. Double-check your mode.

Reading Files

# First, create a sample file to work with
with open("sample.txt", "w") as f:
    f.write("Line 1: Hello World\n")
    f.write("Line 2: Python is great\n")
    f.write("Line 3: File I/O is essential\n")

# Method 1: read() — entire file as one string
with open("sample.txt", "r") as f:
    content = f.read()
    print(content)
    print(f"Total characters: {len(content)}")

# Method 2: readline() — one line at a time
with open("sample.txt", "r") as f:
    first_line = f.readline()   # "Line 1: Hello World\n"
    second_line = f.readline()  # "Line 2: Python is great\n"
    print(first_line.strip())

# Method 3: readlines() — all lines as a list
with open("sample.txt", "r") as f:
    lines = f.readlines()
    print(lines)
    # ['Line 1: Hello World\n', 'Line 2: Python is great\n', ...]

# Method 4: iterate (BEST for large files — memory efficient)
with open("sample.txt", "r") as f:
    for line in f:
        print(line.strip())
Key Insight: Method 4 (iterating the file object directly) is the most memory-efficient approach. Python reads one line at a time from the operating system's buffer. For a 10 GB log file, methods 1 and 3 would try to load everything into RAM. Method 4 uses constant memory regardless of file size.

Writing Files

# write() — write a string
with open("output.txt", "w") as f:
    f.write("First line\n")
    f.write("Second line\n")
    chars_written = f.write("Third line\n")  # Returns number of chars
    print(f"Wrote {chars_written} characters")

# writelines() — write a list of strings (no newlines added!)
lines = ["Alpha\n", "Beta\n", "Gamma\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)

# Append mode — add to existing file
with open("output.txt", "a") as f:
    f.write("Delta\n")  # Added after Gamma

# print() can write to files too
with open("output.txt", "w") as f:
    print("Hello", "World", sep=", ", file=f)
    print("Line two", file=f)

Context Managers: The with Statement

The with statement ensures the file is closed automatically, even if an exception occurs. This is not optional — always use it:

# WITHOUT with — risky
f = open("data.txt", "r")
try:
    content = f.read()
finally:
    f.close()  # Must remember to close!

# WITH with — clean and safe
with open("data.txt", "r") as f:
    content = f.read()
# File is automatically closed here, even if an error occurred

# Multiple files at once
with open("input.txt", "r") as src, open("output.txt", "w") as dst:
    for line in src:
        dst.write(line.upper())

import os

# Relative path — relative to current working directory
rel_path = "data/train.csv"

print(os.getcwd())

# Join paths safely (handles OS-specific separators)
path = os.path.join("data", "raw", "train.csv")
print(path)  # data/raw/train.csv (Linux/Mac)

# Check if path exists
print(os.path.exists("data/train.csv"))
print(os.path.isfile("data/train.csv"))
print(os.path.isdir("data"))

Working with Different Formats

CSV Files

CSV (Comma-Separated Values) is the workhorse format of data science. Python's csv module handles the parsing edge cases — quoted fields, escaped commas, different delimiters:

import csv

# Writing CSV
data = [
    ["name", "age", "score"],
    ["Alice", 30, 95.5],
    ["Bob", 25, 87.3],
    ["Charlie", 35, 91.0],
]

with open("students.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerows(data)

# Reading CSV as lists
with open("students.csv", "r") as f:
    reader = csv.reader(f)
    header = next(reader)  # First row is header
    for row in reader:
        name, age, score = row
        print(f"{name}: age={age}, score={score}")

# Reading CSV as dictionaries (PREFERRED)
with open("students.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row)
        # {'name': 'Alice', 'age': '30', 'score': '95.5'}
        # Note: all values are strings!

# Writing CSV from dictionaries
records = [
    {"name": "Diana", "age": 28, "score": 93.2},
    {"name": "Eve", "age": 32, "score": 88.7},
]

with open("more_students.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "age", "score"])
    writer.writeheader()
    writer.writerows(records)
Watch Out: CSV readers return all values as strings. You must convert numeric columns yourself: int(row['age']), float(row['score']). This is a common source of subtle bugs. For real data work, pandas handles this automatically with pd.read_csv().

JSON Files

JSON is the universal exchange format for APIs, configs, and structured data. Python's json module handles serialization (Python → JSON) and deserialization (JSON → Python):

import json

# Python dict → JSON string
config = {
    "model": "random_forest",
    "hyperparameters": {
        "n_estimators": 100,
        "max_depth": 10,
        "learning_rate": 0.01
    },
    "features": ["age", "income", "score"],
    "normalize": True,
    "dropout": None  # Becomes null in JSON
}

# Write to file (pretty-printed)
with open("config.json", "w") as f:
    json.dump(config, f, indent=2)

# Read from file
with open("config.json", "r") as f:
    loaded = json.load(f)
    print(loaded["hyperparameters"]["max_depth"])  # 10

# Convert to/from strings
json_str = json.dumps(config, indent=2)
print(json_str)

parsed = json.loads(json_str)
print(parsed["model"])  # random_forest

# Handling types that JSON doesn't support natively
from datetime import datetime

class CustomEncoder(json.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, datetime):
            return obj.isoformat()
        if isinstance(obj, set):
            return list(obj)
        return super().default(obj)

data = {
    "timestamp": datetime.now(),
    "tags": {"ml", "python", "data"},
}

json_str = json.dumps(data, cls=CustomEncoder, indent=2)
print(json_str)
# {"timestamp": "2024-01-15T10:30:00.123456", "tags": ["ml", "data", "python"]}
Key Insight: json.dump() writes directly to a file. json.dumps() returns a string (the 's' stands for 'string'). Same for json.load() (file) vs json.loads() (string). This naming convention trips up beginners, so memorize it early.

Binary Files and Pickle

Binary files store raw bytes — images, model weights, serialized Python objects:

# Reading/writing raw bytes
with open("data.bin", "wb") as f:
    f.write(b"\x00\x01\x02\x03\xff")

with open("data.bin", "rb") as f:
    data = f.read()
    print(data)        # b'\x00\x01\x02\x03\xff'
    print(list(data))  # [0, 1, 2, 3, 255]

The pickle module serializes any Python object to bytes and back. It is how scikit-learn saves trained models:

import pickle

# Save a complex Python object
model_state = {
    "weights": [0.5, -0.3, 0.8, 0.1],
    "bias": 0.02,
    "metadata": {
        "epochs": 50,
        "accuracy": 0.93,
        "features": ["age", "income", "score"]
    }
}

# Serialize to file
with open("model.pkl", "wb") as f:
    pickle.dump(model_state, f)

# Deserialize from file
with open("model.pkl", "rb") as f:
    loaded_state = pickle.load(f)

print(loaded_state["metadata"]["accuracy"])  # 0.93
print(loaded_state["weights"])  # [0.5, -0.3, 0.8, 0.1]
Critical: Never unpickle data from untrusted sources. pickle.load() can execute arbitrary code during deserialization. A malicious pickle file can delete files, install malware, or exfiltrate data. For data exchange with external systems, use JSON. Pickle is only for saving/loading your own objects.

File System Operations

The os Module

import os

# List directory contents
files = os.listdir(".")
print(files[:5])  # First 5 items

# Create directories
os.makedirs("output/models/v1", exist_ok=True)  # Creates parents
# exist_ok=True prevents error if directory already exists

# Walk a directory tree
for dirpath, dirnames, filenames in os.walk("data"):
    for fname in filenames:
        full_path = os.path.join(dirpath, fname)
        size = os.path.getsize(full_path)
        print(f"{full_path} ({size} bytes)")

# Path operations
path = "/home/user/projects/data/train.csv"
print(os.path.dirname(path))   # /home/user/projects/data
print(os.path.basename(path))  # train.csv
print(os.path.splitext(path))  # ('/home/.../train', '.csv')

# Environment variables
home = os.environ.get("HOME", "/default/path")
print(home)

The pathlib Module (Modern Python)

pathlib provides an object-oriented interface for file paths. It is the modern, Pythonic way to work with the filesystem:

from pathlib import Path

# Create Path objects
project = Path(".")
data_dir = project / "data" / "raw"  # / operator joins paths!
print(data_dir)  # data/raw

# Path properties
p = Path("/home/user/projects/data/train.csv")
print(p.name)       # train.csv
print(p.stem)       # train
print(p.suffix)     # .csv
print(p.parent)     # /home/user/projects/data
print(p.parts)      # ('/', 'home', 'user', 'projects', 'data', 'train.csv')

# Check existence
print(p.exists())
print(p.is_file())
print(p.is_dir())

# Create directories
output = Path("output/results/v1")
output.mkdir(parents=True, exist_ok=True)

# Read and write (no need for open()!)
config_path = Path("config.json")
text = config_path.read_text(encoding="utf-8")
config_path.write_text('{"key": "value"}', encoding="utf-8")

# Binary read/write
model_path = Path("model.bin")
# model_path.write_bytes(b"\x00\x01\x02")
# data = model_path.read_bytes()

# Glob — find files by pattern
for csv_file in Path("data").glob("**/*.csv"):
    print(f"Found: {csv_file}")

# Iterate directory
for item in Path(".").iterdir():
    kind = "DIR " if item.is_dir() else "FILE"
    print(f"  {kind} {item.name}")
Key Insight: pathlib.Path is better than os.path for almost everything. Paths are objects, not strings, so you get autocompletion, type safety, and the clean / join syntax. Most modern Python code (including FastAPI, PyTorch Lightning, and Hydra) uses pathlib exclusively. Adopt it from day one.

The shutil Module

shutil handles high-level file operations — copying, moving, and removing files and directory trees:

import shutil
from pathlib import Path

# Create test files
Path("source").mkdir(exist_ok=True)
Path("source/data.txt").write_text("hello")
Path("source/notes.txt").write_text("world")

# Copy a single file
shutil.copy("source/data.txt", "backup_data.txt")
# copy2 preserves metadata (timestamps, permissions)
shutil.copy2("source/data.txt", "backup_data2.txt")

# Copy entire directory tree
shutil.copytree("source", "source_backup")

# Move (rename) a file or directory
shutil.move("backup_data.txt", "archive/data.txt")

# Remove a directory tree (DANGEROUS — no undo!)
shutil.rmtree("source_backup")

# Disk usage
usage = shutil.disk_usage(".")
print(f"Total: {usage.total // (1024**3)} GB")
print(f"Used:  {usage.used // (1024**3)} GB")
print(f"Free:  {usage.free // (1024**3)} GB")
Critical: shutil.rmtree() recursively deletes everything in the given path with no confirmation and no recycle bin. A typo like shutil.rmtree("/") is catastrophic. Always validate paths before deletion, and consider using a "trash" pattern (move to a trash directory first) instead of permanent deletion.

The glob Module

import glob

# Find all CSV files in data directory

# Recursive search with **

# Multiple patterns
import itertools
code_files = list(itertools.chain(
    glob.glob("**/*.py", recursive=True),
    glob.glob("**/*.json", recursive=True),
))

Putting It All Together: ML Data Pipeline

Let's build a realistic data loading and saving pipeline that ties together everything we have covered:

import csv
import json
from pathlib import Path
import pickle
from datetime import datetime

class DataPipeline:
    """Load, process, and save ML experiment data."""

    def __init__(self, project_dir):
        self.root = Path(project_dir)
        self.data_dir = self.root / "data"
        self.output_dir = self.root / "output"
        self.model_dir = self.root / "models"

        # Create directories
        for d in [self.data_dir, self.output_dir, self.model_dir]:
            d.mkdir(parents=True, exist_ok=True)

    def load_csv(self, filename):
        """Load CSV data as list of dicts."""
        filepath = self.data_dir / filename
        if not filepath.exists():
            raise FileNotFoundError(f"Data file not found: {filepath}")

        with open(filepath, "r", encoding="utf-8") as f:
            reader = csv.DictReader(f)
            records = list(reader)

        print(f"Loaded {len(records)} records from {filename}")
        return records

    def load_config(self, filename="config.json"):
        """Load experiment configuration."""
        filepath = self.root / filename
        try:
            with open(filepath, "r") as f:
                config = json.load(f)
            print(f"Config loaded: {list(config.keys())}")
            return config
        except FileNotFoundError:
            print(f"No config found, using defaults")
            return {"epochs": 10, "lr": 0.001}

    def save_results(self, results, filename="results.json"):
        """Save experiment results as JSON."""
        filepath = self.output_dir / filename
        results["saved_at"] = datetime.now().isoformat()

        with open(filepath, "w") as f:
            json.dump(results, f, indent=2, default=str)

        print(f"Results saved to {filepath}")

    def save_model(self, model_state, version="v1"):
        """Save model state with pickle."""
        filepath = self.model_dir / f"model_{version}.pkl"
        with open(filepath, "wb") as f:
            pickle.dump(model_state, f)

        size_kb = filepath.stat().st_size / 1024
        print(f"Model saved to {filepath} ({size_kb:.1f} KB)")

    def load_model(self, version="v1"):
        """Load model state from pickle."""
        filepath = self.model_dir / f"model_{version}.pkl"
        with open(filepath, "rb") as f:
            state = pickle.load(f)
        print(f"Model loaded from {filepath}")
        return state

    def list_data_files(self, pattern="*"):
        """List available data files."""
        files = sorted(self.data_dir.glob(pattern))
        for f in files:
            size = f.stat().st_size
            print(f"  {f.name:30s} {size:>10,} bytes")
        return files

    def cleanup_old_models(self, keep_latest=3):
        """Remove old model files, keeping the N most recent."""
        models = sorted(
            self.model_dir.glob("model_*.pkl"),
            key=lambda p: p.stat().st_mtime,
            reverse=True
        )
        for old_model in models[keep_latest:]:
            old_model.unlink()
            print(f"Removed old model: {old_model.name}")

# Usage example
pipeline = DataPipeline("my_experiment")

# Create sample data
sample_csv = pipeline.data_dir / "train.csv"
with open(sample_csv, "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["feature1", "feature2", "label"])
    for i in range(100):
        writer.writerow([i * 0.1, i * 0.2, i % 2])

# Run the pipeline
data = pipeline.load_csv("train.csv")
config = pipeline.load_config()

# Simulate training
model_state = {"weights": [0.5, -0.3], "bias": 0.1, "epochs": 10}
pipeline.save_model(model_state, version="v1")

results = {"accuracy": 0.95, "loss": 0.05, "epochs": 10}
pipeline.save_results(results)

pipeline.list_data_files("*.csv")
Key Insight: This pipeline pattern — a class that owns directory structure, handles serialization, and manages versions — is what production ML projects look like. Tools like MLflow, Weights & Biases, and DVC all build on this foundation. Understanding the raw file operations underneath helps you debug and customize those tools when needed.
