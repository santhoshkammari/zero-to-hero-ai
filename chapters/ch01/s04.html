Control Flow

Chapter 1: Python & Programming Foundations 3 subtopics

TL;DR

Control flow is how your program decides what to do next. Conditional statements (if/elif/else) let you branch based on conditions. Loops (for and while) let you repeat actions. Iterators are the mechanism underneath loops — objects that produce values one at a time. Master these three and you can express any algorithm.

Conditional Statements

Every interesting program needs to make decisions. Should we approve the loan? Is this email spam? Did the model's accuracy improve? Conditional statements are how you tell Python: "if this condition is true, do this; otherwise, do that."

if Statement

The simplest form. If the condition evaluates to True, the indented block runs. If it evaluates to False, the block is skipped entirely.

temperature = 38

if temperature > 37:
    print("Fever detected")
    print("Recommend rest and fluids")

# Both print statements run because temperature (38) is greater than 37.
# If temperature were 36, neither would run.

if-else Statement

When you need to handle both outcomes — when the condition is true and when it's not:

score = 72
passing_threshold = 70

if score >= passing_threshold:
    print(f"Score {score}: PASS")
    status = "passed"
else:
    print(f"Score {score}: FAIL")
    status = "failed"

print(f"Student {status} the exam")

if-elif-else Chain

When you have more than two branches, you chain conditions with elif (short for "else if"). Python checks each condition top-to-bottom and runs the first one that's true, then skips the rest.

score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"      # this one matches
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Score {score} → Grade {grade}")  # Score 85 → Grade B
Key Insight: The order of conditions matters. If you checked score >= 70 first, a score of 85 would match it (85 is indeed ≥ 70) and you'd get grade "C" incorrectly. Always put the most specific conditions first.

Nested Conditionals

You can nest if statements inside other if statements. This works, but deep nesting makes code hard to read. More than two levels deep is a signal to refactor.

age = 25
has_license = True
has_insurance = True

if age >= 18:
    if has_license:
        if has_insurance:
            print("Approved to drive")
        else:
            print("Need insurance")
    else:
        print("Need a license")
else:
    print("Too young to drive")

# Better approach — flatten with 'and'
if age >= 18 and has_license and has_insurance:
    print("Approved to drive")
elif age < 18:
    print("Too young to drive")
elif not has_license:
    print("Need a license")
else:
    print("Need insurance")

Ternary Operator (Conditional Expression)

For simple if-else assignments, Python has a one-liner called the ternary operator:

# Full form
age = 20
if age >= 18:
    status = "adult"
else:
    status = "minor"

# Ternary form — same logic, one line
status = "adult" if age >= 18 else "minor"

# Useful in f-strings and function arguments
print(f"You are {'eligible' if age >= 18 else 'not eligible'}")

# Don't nest ternaries — it becomes unreadable
# bad: x = "a" if condition1 else "b" if condition2 else "c"

Match-Case (Python 3.10+)

Structural pattern matching is Python's version of a switch statement, but much more powerful. It matches values against patterns.

# Basic value matching
def handle_command(command):
    match command:
        case "start":
            print("Starting...")
        case "stop":
            print("Stopping...")
        case "pause" | "wait":        # multiple patterns with |
            print("Pausing...")
        case _:                        # _ is the wildcard (default)
            print(f"Unknown command: {command}")

handle_command("start")   # Starting...
handle_command("wait")    # Pausing...
handle_command("fly")     # Unknown command: fly

# Pattern matching shines with structured data
def describe_point(point):
    match point:
        case (0, 0):
            return "Origin"
        case (x, 0):
            return f"On x-axis at x={x}"
        case (0, y):
            return f"On y-axis at y={y}"
        case (x, y):
            return f"Point at ({x}, {y})"

print(describe_point((0, 0)))    # Origin
print(describe_point((3, 0)))    # On x-axis at x=3
print(describe_point((2, 5)))    # Point at (2, 5)

Match-case cannot replace complex conditional logic with computed boolean expressions. It's designed for matching structures and values, not for evaluating arbitrary conditions like if x > 10 and y < 20.

Loops

Loops let you repeat a block of code. Every time you need to process items in a collection, retry an operation, or keep going until a condition changes, you're using a loop. Python has two kinds: for loops that iterate over sequences, and while loops that run as long as a condition holds.

for Loop

A for loop iterates over a sequence — a list, string, tuple, range, dictionary, or any iterable — executing the body once for each item.

# Iterating over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(f"I like {fruit}")
# I like apple
# I like banana
# I like cherry

# Iterating over a string
for char in "Python":
    print(char, end=" ")
# P y t h o n

range() Function

The range() function generates a sequence of numbers. It's the standard way to loop a specific number of times.

# range(stop) — 0 to stop-1
for i in range(5):
    print(i, end=" ")   # 0 1 2 3 4

# range(start, stop) — start to stop-1
for i in range(2, 6):
    print(i, end=" ")   # 2 3 4 5

# range(start, stop, step) — with custom increment
for i in range(0, 20, 5):
    print(i, end=" ")   # 0 5 10 15

# Counting backwards
for i in range(5, 0, -1):
    print(i, end=" ")   # 5 4 3 2 1

while Loop

A while loop keeps running as long as its condition is True. Use it when you don't know in advance how many iterations you'll need.

# Count until we hit a threshold
total = 0
count = 0
while total < 100:
    total += 17
    count += 1
print(f"Added 17 exactly {count} times to reach {total}")
# Added 17 exactly 6 times to reach 102

# Simulate training until convergence
loss = 10.0
epoch = 0
while loss > 0.01:
    loss *= 0.7    # each epoch reduces loss by 30%
    epoch += 1
print(f"Converged after {epoch} epochs, final loss: {loss:.4f}")
# Converged after 19 epochs, final loss: 0.0093
Common Mistake: Infinite loops. If the condition never becomes False, the loop runs forever. Always make sure something inside the loop changes the condition. 
# This runs forever — don't do this
# x = 1
# while x > 0:
#     x += 1    # x only gets bigger, never reaches 0

# Always have a safety exit
max_iterations = 10000
i = 0
while some_condition and i < max_iterations:
    # ... your logic ...
    i += 1

break and continue

break immediately exits the loop. continue skips the rest of the current iteration and jumps to the next one.

# break — stop searching once you find what you need
data = [4, 7, 2, 9, 1, 8, 3]
target = 9
for i, value in enumerate(data):
    if value == target:
        print(f"Found {target} at index {i}")
        break
# Found 9 at index 3

# continue — skip certain items
scores = [85, -1, 92, -1, 78, 95]  # -1 means missing data
valid_scores = []
for score in scores:
    if score < 0:
        continue   # skip invalid entries
    valid_scores.append(score)
print(valid_scores)  # [85, 92, 78, 95]

else Clause on Loops

This is a Python feature that confuses almost everyone at first. A loop can have an else block that runs if the loop completed without hitting a break.

# Search for a value — else runs if NOT found
haystack = [2, 4, 6, 8, 10]
needle = 7

for item in haystack:
    if item == needle:
        print(f"Found {needle}!")
        break
else:
    # This runs because we never broke out of the loop
    print(f"{needle} not found in the list")
# 7 not found in the list
Key Insight: Think of the else on a loop as "no break." It runs when the loop finishes naturally. If break fires, the else block is skipped. This is useful for search patterns where you need to know if you found something or exhausted all options.

Nested Loops

A loop inside a loop. The inner loop runs completely for each iteration of the outer loop. This gives you the ability to work with grids, pairs, and combinations.

# Multiplication table
for row in range(1, 4):
    for col in range(1, 4):
        print(f"{row * col:4}", end="")
    print()
#    1   2   3
#    2   4   6
#    3   6   9

# All pairs from two lists
colors = ["red", "blue"]
sizes = ["S", "M", "L"]
for color in colors:
    for size in sizes:
        print(f"{color}-{size}", end="  ")
    print()
# red-S  red-M  red-L
# blue-S  blue-M  blue-L

enumerate() Function

When you need both the item and its index, don't use range(len(...)). Use enumerate() — it gives you both in a clean tuple.

languages = ["Python", "JavaScript", "Rust", "Go"]

# Don't do this
for i in range(len(languages)):
    print(f"{i}: {languages[i]}")

# Do this instead
for i, lang in enumerate(languages):
    print(f"{i}: {lang}")

# Start counting from a different number
for i, lang in enumerate(languages, start=1):
    print(f"{i}. {lang}")
# 1. Python
# 2. JavaScript
# 3. Rust
# 4. Go

zip() Function

zip() takes two or more sequences and pairs up their elements. It stops when the shortest sequence runs out.

names = ["Alice", "Bob", "Charlie"]
scores = [95, 87, 92]

for name, score in zip(names, scores):
    print(f"{name}: {score}")
# Alice: 95
# Bob: 87
# Charlie: 92

# Create a dictionary from two lists
grade_book = dict(zip(names, scores))
print(grade_book)  # {'Alice': 95, 'Bob': 87, 'Charlie': 92}

# Zip three lists
subjects = ["Math", "English", "Science"]
for name, score, subject in zip(names, scores, subjects):
    print(f"{name} scored {score} in {subject}")
Watch Out: zip() silently drops leftover elements from longer sequences. If names has 3 items and scores has 5, you only get 3 pairs. Use itertools.zip_longest() if you want to keep all elements and fill missing ones with a default.

Iterators and Iterables

Here's what's actually happening under the hood when you write a for loop. Understanding this mechanism is important because generators, file objects, and many ML data pipelines are built on it.

Iterable vs Iterator

An iterable is any object that can produce an iterator. Lists, strings, tuples, dictionaries, sets, and files are all iterables. An iterator is an object that produces values one at a time using the __next__ method and remembers where it is between calls.

Think of an iterable as a book and an iterator as a bookmark. The book contains all the pages; the bookmark keeps track of which page you're on.

# A list is an iterable
colors = ["red", "green", "blue"]

# Create an iterator from the iterable
color_iter = iter(colors)

# Pull values one at a time
print(next(color_iter))  # 'red'
print(next(color_iter))  # 'green'
print(next(color_iter))  # 'blue'
# print(next(color_iter))  # StopIteration exception — no more items

The Iterator Protocol

When Python executes for item in collection, here's what it actually does:

# This for loop:
for color in colors:
    print(color)

# Is equivalent to this:
_iter = iter(colors)       # calls colors.__iter__()
while True:
    try:
        color = next(_iter)  # calls _iter.__next__()
        print(color)
    except StopIteration:
        break

An object becomes iterable by implementing the __iter__ method (which returns an iterator). An iterator implements __next__ (which returns the next value) and raises StopIteration when exhausted.

iter() and next() Built-in Functions

iter() gets an iterator from an iterable. next() pulls the next value from an iterator. You can also give next() a default value to return instead of raising StopIteration:

numbers = iter([10, 20, 30])

print(next(numbers, "done"))  # 10
print(next(numbers, "done"))  # 20
print(next(numbers, "done"))  # 30
print(next(numbers, "done"))  # 'done' — instead of StopIteration

StopIteration Exception

StopIteration is not an error — it's a signal. It tells the for loop "there are no more items." The loop catches it silently and exits. If you're manually calling next(), you need to handle it yourself (or use the default value approach above).

# Iterators are one-shot — once exhausted, they're done
data = iter([1, 2, 3])
print(list(data))   # [1, 2, 3]
print(list(data))   # [] — empty! The iterator is exhausted

# Iterables can create fresh iterators as many times as you want
data = [1, 2, 3]   # this is the iterable (list)
print(list(iter(data)))  # [1, 2, 3]
print(list(iter(data)))  # [1, 2, 3] — fresh iterator each time
Key Insight: This one-shot nature of iterators is why you can only loop through a file object once without seeking back to the beginning. It's also why generators (which we'll cover later) can only be consumed once. If you need to iterate multiple times, collect the values into a list first — but be aware that this loads everything into memory.
