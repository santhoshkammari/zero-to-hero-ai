List Comprehensions and Generators

Chapter 1: Python & Programming Foundations 2 subtopics

TL;DR

List comprehensions let you build new lists in one readable expression instead of multi-line loops. Generators produce items one at a time, so you can process a billion rows without blowing up your RAM. Together they are the backbone of writing Pythonic data pipelines. Master these two and your code will be shorter, faster, and far easier for teammates to read.

Why This Matters

Imagine you have ten million log lines and you need to extract every line that contains the word ERROR. You could write a for loop, append to a list, and return it. That works. But it is verbose, and if the file is huge, you just loaded every matching line into memory at once.

Python gives you two elegant tools for this exact problem. List comprehensions collapse the loop-plus-append pattern into a single expression. Generators go further — they produce items lazily, one at a time, so your memory footprint stays constant no matter how large the data source is.

These are not cosmetic features. In data science and ML, you will constantly transform, filter, and chain data. Comprehensions and generators are the building blocks for doing that cleanly.

List Comprehensions

Basic Syntax

A list comprehension is an expression that creates a new list by applying an operation to each item in an iterable. The general pattern is:

[expression for item in iterable]

Compare the traditional loop approach with the comprehension approach:

# Traditional loop
squares = []
for x in range(10):
    squares.append(x ** 2)

# List comprehension — same result, one line
squares = [x ** 2 for x in range(10)]
print(squares)  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

Read it left to right: "give me x ** 2 for every x in range(10)." The expression comes first, then the loop clause. That is the entire mental model.

Key Insight: List comprehensions are not just syntactic sugar. CPython optimizes them internally with a dedicated LIST_APPEND bytecode instruction, which is faster than calling list.append() in a regular loop. On typical workloads you will see a 10–30% speed improvement.

Conditional Filtering

You can attach an if clause to keep only items that satisfy a condition:

# Only even squares
even_squares = [x ** 2 for x in range(20) if x % 2 == 0]
print(even_squares)  # [0, 4, 16, 36, 64, 100, 144, 196, 256, 324]

# Filter strings by length
words = ["Python", "AI", "ML", "deep learning", "data"]
short_words = [w for w in words if len(w) <= 4]
print(short_words)  # ['AI', 'ML', 'data']

You can also use an if/else expression (note: this goes in the expression part, not the filter part):

# Label each number
labels = ["even" if x % 2 == 0 else "odd" for x in range(6)]
print(labels)  # ['even', 'odd', 'even', 'odd', 'even', 'odd']
Watch Out: The position of if changes its meaning. After the for it filters. Before the for (as part of a ternary expression) it transforms. Mixing them up is a common bug.

Nested Comprehensions

You can nest multiple for clauses. The order follows the same order you would write nested loops:

# Flatten a matrix (list of lists)
matrix = [[1, 2, 3],
          [4, 5, 6],
          [7, 8, 9]]

flat = [num for row in matrix for num in row]
print(flat)  # [1, 2, 3, 4, 5, 6, 7, 8, 9]

# Equivalent nested loop
flat = []
for row in matrix:
    for num in row:
        flat.append(num)

You can also build nested structures — for example, transposing a matrix:

# Transpose: swap rows and columns
transposed = [[row[i] for row in matrix] for i in range(3)]
print(transposed)
# [[1, 4, 7], [2, 5, 8], [3, 6, 9]]
Watch Out: If a comprehension requires more than two levels of nesting, it becomes hard to read. At that point, switch to a regular loop or break the logic into helper functions. Readability beats cleverness.

Set and Dictionary Comprehensions

The same syntax works with curly braces for sets and dictionaries:

# Set comprehension — unique lengths
words = ["hello", "world", "hi", "hey", "hello"]
unique_lengths = {len(w) for w in words}
print(unique_lengths)  # {2, 3, 5}

# Dictionary comprehension — word to length mapping
word_lengths = {w: len(w) for w in words}
print(word_lengths)
# {'hello': 5, 'world': 5, 'hi': 2, 'hey': 3}

# Invert a dictionary
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
print(inverted)  # {1: 'a', 2: 'b', 3: 'c'}

A set comprehension automatically deduplicates. A dictionary comprehension gives you key-value pairs. Both are extremely common in data preprocessing — for example, building lookup tables from raw data.

Generators

Generator Functions and the yield Keyword

A generator function looks like a normal function except it uses yield instead of return. When you call it, Python does not execute the body immediately. Instead it returns a generator object. Each time you ask for the next value, execution resumes right where it left off.

def count_up_to(n):
    """Yield numbers from 1 to n, one at a time."""
    i = 1
    while i <= n:
        yield i
        i += 1

counter = count_up_to(5)
print(type(counter))  # <class 'generator'>
print(next(counter))  # 1
print(next(counter))  # 2
print(next(counter))  # 3

# Iterate the rest with a for loop
for num in counter:
    print(num, end=" ")  # 4 5

Think of yield as a pause button. The function remembers its local variables, its position in the code, everything. When you call next() again, it resumes exactly where it paused.

Generator Expressions

Just like list comprehensions use square brackets, generator expressions use parentheses:

# List comprehension — builds entire list in memory
sum_of_squares = sum([x ** 2 for x in range(1_000_000)])

# Generator expression — computes one value at a time
sum_of_squares = sum(x ** 2 for x in range(1_000_000))

The second version never creates a million-element list. It feeds values to sum() one by one. Same answer, fraction of the memory.

Key Insight: When you pass a generator expression as the sole argument to a function, you can drop the extra parentheses: sum(x**2 for x in range(n)). This is not a tuple — it is a generator.

Lazy Evaluation and Memory Efficiency

Lazy evaluation means values are computed only when requested, not upfront. This is the killer feature of generators. Consider reading a massive file:

def read_large_file(filepath):
    """Yield one line at a time from a huge file."""
    with open(filepath, "r") as f:
        for line in f:
            yield line.strip()

# Process 10 GB of logs without loading them all into memory
for line in read_large_file("server.log"):
    if "ERROR" in line:
        print(line)

At any point in time, only one line lives in memory. The file could be 10 GB or 100 GB — your memory usage stays constant. This is why generators are essential in ML data pipelines where datasets often exceed available RAM.

import sys

# Compare memory: list vs generator
big_list = [x for x in range(1_000_000)]
big_gen  = (x for x in range(1_000_000))

print(sys.getsizeof(big_list))  # ~8,448,728 bytes (8 MB)
print(sys.getsizeof(big_gen))   # 200 bytes (constant!)

Chaining Generators

The real power shows when you chain generators into a pipeline. Each stage does one transformation, and data flows through lazily:

def read_lines(path):
    with open(path) as f:
        for line in f:
            yield line.strip()

def filter_errors(lines):
    for line in lines:
        if "ERROR" in line:
            yield line

def extract_timestamp(lines):
    for line in lines:
        # Assume format: "2024-01-15 10:30:00 ERROR ..."
        yield line[:19]

# Pipeline: read → filter → extract
pipeline = extract_timestamp(filter_errors(read_lines("app.log")))

for timestamp in pipeline:
    print(timestamp)

Each function is small, testable, and reusable. No intermediate lists are created. Data flows through the chain one item at a time. This pattern is the backbone of production data processing.

The send() and close() Methods

Generators are not one-way streets. You can send values into a running generator using send():

def running_average():
    """Compute running average, accepting new values via send()."""
    total = 0
    count = 0
    average = None
    while True:
        value = yield average
        if value is not None:
            total += value
            count += 1
            average = total / count

avg = running_average()
next(avg)          # Prime the generator (advance to first yield)

print(avg.send(10))  # 10.0
print(avg.send(20))  # 15.0
print(avg.send(30))  # 20.0

avg.close()  # Shut down the generator

The send() method resumes the generator and delivers a value that becomes the result of the yield expression. The close() method throws a GeneratorExit exception inside the generator, giving it a chance to clean up resources.

Key Insight: You must call next() once before the first send(). This is called "priming" the generator — it advances execution to the first yield. Without priming, you will get a TypeError.

The itertools Module

Python's itertools module provides a toolkit of generator-based utilities that are fast and memory-efficient. Think of it as the standard library for iterator algebra.

import itertools

# chain — concatenate multiple iterables
combined = list(itertools.chain([1, 2], [3, 4], [5]))
print(combined)  # [1, 2, 3, 4, 5]

# product — cartesian product (like nested loops)
pairs = list(itertools.product("AB", [1, 2]))
print(pairs)  # [('A', 1), ('A', 2), ('B', 1), ('B', 2)]

# combinations — all unique k-length subsets
combos = list(itertools.combinations([1, 2, 3, 4], 2))
print(combos)  # [(1,2), (1,3), (1,4), (2,3), (2,4), (3,4)]

# permutations — order matters
perms = list(itertools.permutations("ABC", 2))
print(perms)  # [('A','B'), ('A','C'), ('B','A'), ...]

# islice — lazy slicing (works on any iterator)
first_five = list(itertools.islice(range(1_000_000), 5))
print(first_five)  # [0, 1, 2, 3, 4]

# groupby — group consecutive elements
data = [("fruit", "apple"), ("fruit", "banana"),
        ("veggie", "carrot"), ("fruit", "date")]
for key, group in itertools.groupby(data, key=lambda x: x[0]):
    print(key, list(group))
# fruit [('fruit', 'apple'), ('fruit', 'banana')]
# veggie [('veggie', 'carrot')]
# fruit [('fruit', 'date')]
Watch Out: itertools.groupby only groups consecutive items with the same key. If your data is not sorted by the grouping key, sort it first. Otherwise you will get fragmented groups (as shown above with the second "fruit" group).

Here is a practical ML example — generating all hyperparameter combinations for a grid search:

import itertools

learning_rates = [0.001, 0.01, 0.1]
batch_sizes = [16, 32, 64]
optimizers = ["adam", "sgd"]

# All combinations of hyperparameters
grid = list(itertools.product(learning_rates, batch_sizes, optimizers))
print(f"Total experiments: {len(grid)}")  # 18

for lr, bs, opt in grid[:3]:
    print(f"  lr={lr}, batch_size={bs}, optimizer={opt}")

Putting It All Together: A Data Pipeline

Let's combine comprehensions and generators to build a realistic data processing pipeline — loading CSV-like data, cleaning it, and computing statistics:

import itertools

# Simulated raw data (imagine this comes from a file)
raw_records = [
    "Alice,85,Math",
    "Bob,,Science",        # missing score
    "Charlie,92,Math",
    "Diana,78,Science",
    "Eve,95,Math",
    "Frank,invalid,Math",  # bad data
    "",                     # empty line
    "Grace,88,Science",
]

def parse_records(lines):
    """Yield (name, score, subject) tuples from raw lines."""
    for line in lines:
        if not line.strip():
            continue
        parts = line.split(",")
        if len(parts) == 3:
            name, score_str, subject = parts
            try:
                score = int(score_str)
                yield (name, score, subject)
            except ValueError:
                continue  # skip bad data silently

def filter_by_subject(records, subject):
    """Yield only records matching the given subject."""
    for name, score, subj in records:
        if subj == subject:
            yield (name, score, subj)

# Build the pipeline
all_records = parse_records(raw_records)
math_records = filter_by_subject(all_records, "Math")

# Comprehension to extract scores
math_scores = [score for name, score, subj in math_records]
print(f"Math scores: {math_scores}")          # [85, 92, 95]
print(f"Average: {sum(math_scores)/len(math_scores):.1f}")  # 90.7
Key Insight: Notice the pattern — generators for pipeline stages (lazy, memory-friendly), comprehensions for the final collection step (when you need the whole list). This is idiomatic Python. You will see this exact approach in pandas internals, Spark pipelines, and ML data loaders.
