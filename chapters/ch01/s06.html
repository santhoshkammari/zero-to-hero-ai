NumPy

Chapter 1: Python & Programming Foundations 9 concept areas

NumPy is the foundation everything else sits on — Pandas, scikit-learn, PyTorch. Every tensor, every DataFrame column, every fit() call eventually bottoms out in a NumPy ndarray. Two concepts separate people who read ML code fluently from people who don't: broadcasting and vectorization. Once those click, the rest is just API.

Arrays in 30 Seconds

You create arrays with a handful of constructors. No table needed — just know they exist:

import numpy as np

np.array([1, 2, 3])          # from existing data
np.zeros((3, 4))              # all zeros — buffers, accumulators
np.ones((2, 5))               # all ones — bias init, masks
np.arange(0, 10, 0.5)         # evenly spaced by step — like range() but for floats
np.linspace(0, 1, 50)         # evenly spaced by count — plotting grids, schedules

And three attributes you'll check constantly: .shape (the dimensions), .dtype (the element type), .ndim (how many axes). When something goes wrong in NumPy, the answer is almost always in one of those three.

Indexing & Slicing

Basic slicing works like Python lists, extended to multiple dimensions: arr[0], arr[1:5], arr[:, 0] (all rows, first column). If you know Python lists, this is just more commas.

Boolean masking is the one to internalize. It shows up everywhere in ML — filtering samples, selecting features, applying thresholds:

# You have predictions and you want only the confident ones
predictions = model.predict(X)          # shape (1000,)
confident = predictions[predictions > 0.9]  # only high-confidence predictions

# Select all samples belonging to class 1
X_class1 = X[y == 1]   # y is your label array — this grabs matching rows

Fancy indexing lets you cherry-pick specific rows: arr[[0, 3, 5]]. Useful for grabbing a specific batch of samples by index.

One thing to keep straight: basic slicing returns a view (modifying the slice modifies the original). Boolean and fancy indexing return copies. If you're ever unsure: np.shares_memory(a, b).

Broadcasting — The Mental Model

Here's the thing about broadcasting — once it clicks, you stop writing loops entirely. Everything becomes an operation on shapes.

The intuition: when you subtract the mean from every row of a matrix, NumPy doesn't loop. It "broadcasts" the mean vector across all rows simultaneously. Understanding how it decides what to broadcast is the key to reading any ML codebase.

There are three rules, and they're simpler than they sound:

Rule 1: Pad with 1s. If the arrays have different numbers of dimensions, NumPy pads the smaller shape with 1s on the left. So a shape (3,) operating against (5, 3) first becomes (1, 3).

Rule 2: Stretch the 1s. Any dimension of size 1 stretches to match the other array. That (1, 3) stretches to (5, 3). No data is copied — NumPy just pretends the row is repeated 5 times.

Rule 3: If it doesn't fit, it breaks. If two dimensions differ and neither is 1, you get a ValueError. No guessing, no silent nonsense.

The canonical example — centering a feature matrix:

X = np.random.randn(500, 10)   # 500 samples, 10 features
centered = X - X.mean(axis=0)  # X.mean(axis=0) has shape (10,)

# What just happened:
# X shape:         (500, 10)
# mean shape:      (10,)       → Rule 1 pads to (1, 10)
#                               → Rule 2 stretches to (500, 10)
# Result: (500, 10) — each column now has zero mean. No loop. One line.

The gotcha nobody warns you about: shapes like (100, 1) + (1, 200) silently create a (100, 200) array. Both dimensions are 1, so both stretch. That's 20,000 elements from two small arrays. Scale those to (10000, 1) + (1, 50000) and you've just allocated 2 GB.

Debug rule: If you're ever confused about a broadcasting operation, print the shapes. print(a.shape, b.shape). Always print the shapes.

Vectorization — Why NumPy Is Fast

The mental model is simple: Python loops are slow because every iteration goes through the interpreter — type checking, reference counting, dispatch. NumPy operations happen in compiled C, operating on entire arrays at once. One call into C replaces thousands of Python iterations.

Rule of thumb: if you're writing a for loop over array elements, you're probably doing it wrong.

# ❌ The loop way — 3x the code, 100x slower
def normalize_loop(X):
    result = np.empty_like(X)
    for i in range(X.shape[0]):
        row_mean = X[i].mean()
        row_std = X[i].std()
        result[i] = (X[i] - row_mean) / (row_std + 1e-8)
    return result

# ✅ The vectorized way — one shot
def normalize_vectorized(X):
    mean = X.mean(axis=1, keepdims=True)
    std = X.std(axis=1, keepdims=True)
    return (X - mean) / (std + 1e-8)

Same result. The vectorized version is cleaner and faster — typically 50-100× on large arrays. The speed gap only grows with array size because the C code can use SIMD instructions and cache-friendly access patterns that Python loops can never touch.

A real example — cosine similarity between all pairs of vectors. This shows up in retrieval, recommendation, and embedding analysis:

def cosine_similarity_matrix(A, B):
    """A: (n, d), B: (m, d) → (n, m) similarity matrix"""
    A_norm = A / np.linalg.norm(A, axis=1, keepdims=True)
    B_norm = B / np.linalg.norm(B, axis=1, keepdims=True)
    return A_norm @ B_norm.T  # matmul handles the pairwise dot products

# No loops. Handles 10k × 10k comparisons in under a second.

Reshaping

.reshape(), .ravel(), .flatten(), .T — these change how you view the data, not the data itself.

The key insight: reshape gives you a view, not a copy. Modifying the reshaped array modifies the original. This is fast (zero-cost) but can bite you if you don't expect it. .flatten() always copies if you need independence.

np.newaxis (or just None) is the trick for adding dimensions, and it's critical for making broadcasting work when shapes don't quite line up:

# You have a 1D array of biases, shape (128,)
# Model expects (1, 128) for broadcasting across a batch
bias = np.zeros(128)
bias_row = bias[np.newaxis, :]    # shape (1, 128) — ready to broadcast

# Reshaping for model input: (n_samples, n_features) → (n_samples, 1, n_features)
X = np.random.randn(64, 784)
X_seq = X[:, np.newaxis, :]       # shape (64, 1, 784) — added a "sequence" dimension

Use -1 in reshape to auto-compute one dimension: arr.reshape(-1, 3) means "whatever number of rows gives me 3 columns."

dtypes

ML models almost always use float32. NumPy defaults to float64, which wastes memory for no accuracy gain in training. Cast early.

dtypeSizePrecisionWhen to Use
float648 bytes~15 decimal digitsNumPy default. Scientific computing, loss where precision matters. Avoid on GPU.
float324 bytes~7 decimal digitsML standard. PyTorch/TensorFlow default. Always cast to this before training.
int648 bytesExact integersNumPy default int. Large indices, counts.
int324 bytesExact integersClass labels, token IDs. Sufficient for most index arrays.
float162 bytes~3 decimal digitsMixed-precision training (AMP). Limited range — use with care.
uint81 byte0–255Image pixels, quantized model weights.
bool1 byteTrue/FalseMasks, filter conditions.

The one method to remember: X = X.astype(np.float32). A (100000, 768) embedding table in float64 is 587 MB; in float32 it's 293 MB. That difference compounds across every operation.

Nice to Know

np.einsum — Einstein summation. Express any tensor contraction in a short string: np.einsum('ij,jk->ik', A, B) is matmul. Becomes essential for attention scores ('bqd,bkd->bqk') and batch operations. Learn it when you hit your first transformer.

np.memmap — memory-mapped arrays that load from disk on demand. Your escape hatch when datasets exceed RAM.

np.random.default_rng(seed=42) — the modern way to do reproducible random. Always seed your Generator in experiments. Avoid the legacy np.random.seed() global state.

Structured arrays — heterogeneous dtypes per column in a single ndarray. Niche, but useful for C struct interop without Pandas.

  • Rewrite a Python loop over array elements as a single vectorized NumPy expression

  • Reshape arrays for model input using .reshape(), np.newaxis, and -1
  • Cast arrays to float32 before training and explain why it matters
  • Debug shape mismatches by printing .shape at every step
