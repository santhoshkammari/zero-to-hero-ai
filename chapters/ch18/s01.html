Fairness & Bias Detection

Chapter 18: Responsible AI and Ethics Section 1
TL;DR

Fairness has 6+ mathematical definitions that contradict each other — you literally cannot satisfy all of them simultaneously when base rates differ across groups (this is a proven theorem, not a limitation of current tools). Your job isn't to "be fair." It's to choose which fairness definition fits your context, measure it, document the tradeoff, and defend that choice. Here are the tools.

Bias in Data

Bias doesn't start with your model. It starts with the data you feed it. Here's a field guide to the six patterns you'll encounter most often:

Bias TypeWhat HappensClassic Example
HistoricalData faithfully reflects an unfair worldAmazon hiring model learned "male = good candidate" from a decade of male-dominated hiring
RepresentationSome groups are underrepresented in training dataImageNet skewed toward US/Western Europe; dermatology datasets skewed toward lighter skin
MeasurementFeatures are noisy proxies contaminated by the bias you're trying to avoid"Prior arrests" as proxy for criminality — reflects policing patterns, not crime rates
SamplingData collection systematically excludes populationsSpeech recognition trained on podcast data underrepresents elderly, non-native speakers
LabelHuman annotations carry prejudiceToxicity classifiers flag African American English as more "toxic" than Standard American English
AggregationPooling across subgroups hides differencesDiabetes models that ignore HbA1c-to-risk variation across ethnic groups

The practical move: before you train anything, audit your dataset for representation and label distribution across sensitive groups.

import pandas as pd
import numpy as np

def audit_representation(df, sensitive_col, label_col):
    """Flag underrepresented groups and label rate disparities.
    Run this BEFORE training — not after a PR disaster."""

    # Group proportions
    group_counts = df[sensitive_col].value_counts(normalize=True)
    print("=== Group Proportions ===")
    print(group_counts.round(3).to_string())

    # Positive label rate per group
    label_rates = df.groupby(sensitive_col)[label_col].mean()
    print("\n=== Positive Label Rate by Group ===")
    print(label_rates.round(3).to_string())

    # Flag groups under 10% representation
    underrep = group_counts[group_counts < 0.10]
    if len(underrep) > 0:
        print(f"\n⚠️  Underrepresented (<10%): {list(underrep.index)}")

    # Flag label rate disparities > 2x
    max_rate, min_rate = label_rates.max(), label_rates.min()
    if min_rate > 0 and max_rate / min_rate > 2.0:
        print(f"\n⚠️  Label rate disparity: {max_rate/min_rate:.1f}x "
              f"(highest: {label_rates.idxmax()}, lowest: {label_rates.idxmin()})")

    return {"group_proportions": group_counts, "label_rates": label_rates}

Bias in Models

Proxy Variables: Why "Fairness Through Unawareness" Fails

The most common first instinct: "just remove the sensitive attribute." This doesn't work. Zip code correlates with race. Name correlates with gender. Purchase history correlates with income. These proxy variables carry the signal you removed, often with enough fidelity that the model barely notices the protected attribute is gone.

import pandas as pd
from sklearn.ensemble import GradientBoostingClassifier

def detect_proxy_variables(df, sensitive_col, feature_cols, threshold=0.15):
    """Train a model to predict the sensitive attribute from other features.
    High accuracy = your features are proxies."""

    X = pd.get_dummies(df[feature_cols], drop_first=True)
    y = df[sensitive_col]

    model = GradientBoostingClassifier(n_estimators=100, max_depth=3, random_state=42)
    model.fit(X, y)

    proxy_score = model.score(X, y)
    print(f"Proxy detection score: {proxy_score:.3f}")
    print(f"(Baseline = {y.value_counts(normalize=True).max():.3f})")

    if proxy_score > y.value_counts(normalize=True).max() + threshold:
        # Which features carry the most signal?
        importances = pd.Series(
            model.feature_importances_, index=X.columns
        ).sort_values(ascending=False)
        print(f"\n⚠️  Top proxy features:")
        print(importances.head(5).to_string())
    else:
        print("✓ No strong proxies detected.")

    return proxy_score
The key insight: If a simple model can predict the sensitive attribute from your remaining features with accuracy well above the majority-class baseline, you have proxies. Removing the protected column was theater, not fairness.

Embedding Bias

Word embeddings encode societal stereotypes directly in geometry: "man" is to "computer programmer" as "woman" is to "homemaker." These associations propagate into downstream models. If you're doing NLP, assume your embeddings are biased and apply debiasing (covered in the NLP chapters) or use embedding-specific fairness metrics.

Amplification

Models don't just reflect bias — they amplify it. A dataset where 60% of cooking images show women produces a model that assigns 80%+ cooking predictions to women. The feedback loop: biased predictions → biased data collection → even more biased predictions. Monitor for amplification by comparing your model's demographic distribution of outcomes against the training data's distribution.

Fairness Metrics — The Hard Part

This is where it gets real. Each metric below defines "fair" differently. You need to understand them all because your choice of metric is a value judgment — it decides who benefits and who is harmed.

Metric What It Requires When to Use Limitation
Demographic Parity P(Ŷ=1 | G=a) = P(Ŷ=1 | G=b) — equal positive prediction rates across groups When you want equal outcome rates regardless of qualifications (e.g., diversity hiring targets) Ignores actual qualifications; can require selecting less-qualified candidates from one group
Equalized Odds Equal TPR and FPR across groups When both false positives and false negatives carry real costs (criminal justice, medical screening) Hard to satisfy exactly; requires post-processing that may reduce overall accuracy
Equal Opportunity Equal TPR across groups (relaxed version of Equalized Odds) When the primary harm is denying a benefit to qualified individuals (loan approvals, college admissions) Says nothing about false positive rates — one group may bear a higher false accusation burden
Predictive Parity Equal precision across groups — P(Y=1 | Ŷ=1, G=a) = P(Y=1 | Ŷ=1, G=b) When you want the "positive" label to mean the same thing regardless of group (risk scores) Can coexist with wildly different FPR/FNR across groups
Individual Fairness Similar individuals get similar predictions — requires a domain-specific similarity metric When group-level metrics aren't granular enough (you care about individual treatment) Defining "similar" is the whole problem; no universal solution
Counterfactual Fairness Prediction wouldn't change if the individual had been in a different group (causal reasoning) When you can build a causal model of the domain (research settings, well-understood processes) Requires a causal DAG — if your causal model is wrong, your fairness guarantee is meaningless

The Impossibility Theorem

Here's the mathematical gut punch. Chouldechova (2017) and Kleinberg et al. (2016) proved that when base rates differ between groups (e.g., actual recidivism rates differ by race), you cannot simultaneously satisfy:

  1. Calibration — a score of 0.7 means 70% probability for every group
  2. Equal false positive rates across groups
  3. Equal false negative rates across groups

This isn't an engineering limitation — it's a mathematical impossibility. With different base rates, satisfying any two of these forces a violation of the third.

Why this matters in practice: the COMPAS debate. ProPublica showed COMPAS had different FPR across racial groups. Northpointe showed it was calibrated. Both were right. The impossibility theorem means they had to disagree on at least one metric. The real question was never "is COMPAS fair?" but "which fairness property matters most in criminal sentencing?" — and that's a value judgment, not a technical one.

You must choose. There is no "just be fair" setting. Every fairness intervention picks winners and losers. Your job is to make that choice explicitly, document it, and defend it — not to pretend a neutral option exists.

Measuring Fairness with Fairlearn

from fairlearn.metrics import MetricFrame, selection_rate, false_positive_rate, false_negative_rate
from sklearn.metrics import accuracy_score, precision_score, recall_score

def fairness_audit(y_true, y_pred, sensitive_features):
    """Compute multiple fairness metrics simultaneously.
    This is your go-to audit function."""

    metrics = {
        "accuracy": accuracy_score,
        "selection_rate": selection_rate,           # Demographic Parity
        "precision": precision_score,               # Predictive Parity
        "recall": recall_score,                     # Equal Opportunity
        "false_positive_rate": false_positive_rate,  # Equalized Odds (part 1)
        "false_negative_rate": false_negative_rate,  # Equalized Odds (part 2)
    }

    mf = MetricFrame(
        metrics=metrics,
        y_true=y_true,
        y_pred=y_pred,
        sensitive_features=sensitive_features
    )

    print("=== Overall Metrics ===")
    print(mf.overall.to_string())

    print("\n=== By Group ===")
    print(mf.by_group.to_string())

    print("\n=== Disparities (max - min across groups) ===")
    print(mf.difference().to_string())

    # EEOC 80% rule check (selection rate ratio)
    rates = mf.by_group["selection_rate"]
    ratio = rates.min() / rates.max() if rates.max() > 0 else 0
    print(f"\n80% Rule: min/max selection rate = {ratio:.3f} "
          f"({'✓ PASS' if ratio >= 0.8 else '⚠️  FAIL'})")

    return mf

# Usage:

Debiasing Techniques

Three intervention points, each with different tradeoffs:

Stage What It Does Pros Cons Tools
Pre-processing Transform data before training to remove bias signal Model-agnostic; fixes the root cause May distort useful signal; information loss AIF360 Reweighing, DisparateImpactRemover
In-processing Add fairness constraints during training Jointly optimizes accuracy + fairness Requires model modification; slower training Fairlearn ExponentiatedGradient, AIF360 AdversarialDebiasing
Post-processing Adjust predictions after training to satisfy fairness constraints Works with any model; no retraining needed Can degrade accuracy significantly; feels like a patch Fairlearn ThresholdOptimizer, AIF360 CalibratedEqOdds

Pre-processing: Reweighing

from fairlearn.reductions import ExponentiatedGradient, DemographicParity
from sklearn.linear_model import LogisticRegression

# Simple reweighing approach: compute sample weights that
# equalize positive label rates across groups
def compute_reweighing_weights(df, sensitive_col, label_col):
    """Assign sample weights to counteract representation bias."""
    overall_rate = df[label_col].mean()

    weights = []
    for _, row in df.iterrows():
        group = row[sensitive_col]
        label = row[label_col]
        group_mask = df[sensitive_col] == group
        group_rate = df.loc[group_mask, label_col].mean()
        group_size = group_mask.mean()

        # Weight = expected / observed
        expected = overall_rate if label == 1 else (1 - overall_rate)
        observed = group_rate if label == 1 else (1 - group_rate)
        weights.append(expected / observed if observed > 0 else 1.0)

    return weights

Legal Foundations (The Numbers You Need)

Disparate treatment = intentionally using a protected attribute. Disparate impact = facially neutral policy that disproportionately harms a protected group. Your model can have zero race features and still create disparate impact through proxies.

EEOC 80% Rule (Four-Fifths Rule): if the selection rate for any protected group is less than 80% of the rate for the highest group, there's prima facie evidence of adverse impact. This is the number auditors actually check. The code in the MetricFrame section above computes this directly.

EU AI Act: Classifies AI systems by risk tier. High-risk systems (hiring, credit, criminal justice, healthcare) must conduct conformity assessments including bias audits. Covered in detail in the Governance section (s06).

Intersectional Fairness

Checking fairness for "gender" and "race" separately can hide compounding effects. A model might be fair for women overall and fair for Black applicants overall, while being deeply unfair to Black women specifically. Crenshaw's intersectionality framework applies directly to ML auditing.

from fairlearn.metrics import MetricFrame
from sklearn.metrics import accuracy_score, recall_score

def intersectional_audit(y_true, y_pred, df, sensitive_cols):
    """Audit fairness at the intersection of multiple sensitive attributes.
    This catches disparities that single-axis audits miss."""

    # Create intersection column
    intersection = df[sensitive_cols].astype(str).agg(" × ".join, axis=1)

    mf = MetricFrame(
        metrics={
            "accuracy": accuracy_score,
            "recall": recall_score,
        },
        y_true=y_true,
        y_pred=y_pred,
        sensitive_features=intersection
    )

    print("=== Intersectional Fairness Audit ===")
    print(mf.by_group.sort_values("accuracy").to_string())

    # Flag groups performing > 10% below the best group
    best_acc = mf.by_group["accuracy"].max()
    flagged = mf.by_group[mf.by_group["accuracy"] < best_acc - 0.10]
    if len(flagged) > 0:
        print(f"\n⚠️  Groups >10% below best accuracy ({best_acc:.3f}):")
        print(flagged.to_string())

    return mf

# Usage:
# intersectional_audit(y_test, y_pred, df_test, ["gender", "race"])
Practical tip: Intersectional subgroups can get small fast (e.g., "Black + female + age 60+" may be 12 samples in your test set). Small sample sizes make metrics unreliable. Report confidence intervals, not just point estimates, for small subgroups. Fairlearn's MetricFrame supports this with control_features for hierarchical analysis.
