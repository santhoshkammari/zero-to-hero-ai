TL;DR

Backpropagation = chain rule, run backward through neural network's computational graph. Question: for each weight, how much blame for error? One backward pass gives answer, whether network has nine parameters or nine billion. Each node computes local derivative. Chain rule multiplies them, gives each weight blame signal. Signal = gradient. Optimizer needs that direction. Backprop makes deep learning possible. Without this efficiency, deep networks would be impossible to train.

Why Backprop Matters

You can write loss.backward() in PyTorch, watch loss drop, never know what happens between loss and weight update. Fine until something breaks. Then need know why gradient is zero, exploding, or wrong sign. Here deep dive.

Backpropagation computes gradient of loss with respect to every parameter in neural network. Seppo Linnainmaa described reverse-mode automatic differentiation in 1970. Paul Werbos applied it to neural networks in his 1974 PhD thesis. Rumelhart, Hinton, and Williams popularized it in 1986. Every trained neural network since — every ResNet, every Transformer, every GPT — uses it.

Brute Force Fails

Situation: neural network with millions of parameters. Feed input. Get output. Output wrong. Loss gives one number — how wrong. Now hard question: which weights should change, and by how much?

Obvious approach: wiggle each weight one at time. Increase w₁ slightly, rerun forward pass, see whether loss drops. Reset w₁. Repeat for w₂, w₃, rest. This is numerical differentiation — approximate gradient by finite differences. Network with n parameters needs n + 1 forward passes for one gradient vector.

GPT-3 has 175 billion parameters. That means 175 billion forward passes per training step. One forward pass takes seconds on massive GPU cluster. Math breaks. Universe too short.

Backpropagation computes gradient of loss with respect to every weight in network in two passes — one forward, one backward. Total cost ≈ 2× one forward pass, whether network has nine parameters or nine billion. Not minor optimization. Reason deep learning exists.

Forward Pass: Build Path

Before backprop, need path to backprop through. Build tiny network. Trace every computation by hand. Task: predict whether customer buys product (1 = yes, 0 = no) from two features: website browse time (x₁) and items viewed (x₂).

Two inputs. Two hidden neurons with ReLU activation. One output neuron with sigmoid. Binary cross-entropy loss. Small enough for head. Large enough to show patterns that scale.

Inputs:  x₁ = 1.0 (browsing time, normalized)
         x₂ = 0.5 (items viewed, normalized)
Target:  y = 1  (did buy)

Hidden layer weights and biases:
  w₁₁ = 0.3   w₁₂ = 0.7   b₁ = 0.1   (→ hidden neuron h₁)
  w₂₁ = −0.2  w₂₂ = 0.5   b₂ = 0.0   (→ hidden neuron h₂)

Output layer weights and bias:
  v₁ = 0.6   v₂ = −0.4   b₃ = 0.1

Nine parameters total. Six weights. Three biases. Push input through.

First, each hidden neuron computes weighted sum of inputs and adds bias. This is pre-activation — raw value before activation function.

z₁ = w₁₁·x₁ + w₁₂·x₂ + b₁ = 0.3(1.0) + 0.7(0.5) + 0.1 = 0.75
z₂ = w₂₁·x₁ + w₂₂·x₂ + b₂ = −0.2(1.0) + 0.5(0.5) + 0.0 = 0.05

Now ReLU. Positive values pass unchanged. Negative values clamp to zero. Both z₁ and z₂ are positive, so both pass unchanged.

h₁ = ReLU(0.75) = 0.75
h₂ = ReLU(0.05) = 0.05

Output neuron takes hidden activations, computes weighted sum, applies sigmoid. Sigmoid squashes any real number into range (0, 1). We read that as probability.

z₃ = v₁·h₁ + v₂·h₂ + b₃ = 0.6(0.75) + (−0.4)(0.05) + 0.1 = 0.53
ŷ  = σ(z₃) = 1 / (1 + e⁻⁰·⁵³) ≈ 0.6295

Finally, binary cross-entropy loss measures gap between prediction and truth. When target is 1, formula simplifies to −log(ŷ):

L = −[y·log(ŷ) + (1−y)·log(1−ŷ)] = −log(0.6295) ≈ 0.4629

Network predicted 0.63 when answer was 1. Not terrible. Still wrong. Loss = 0.4629. Every intermediate value — z₁, z₂, h₁, h₂, z₃, ŷ — matters in backward pass. Forward pass gives prediction and raw material for derivatives.

Chain Rule — Signal Through Stages

Before going backward through network, chain rule must feel automatic. Everything below repeats it.

Imagine factory line with three stations. Station A shapes metal. Station B paints it. Station C adds label. Defective product comes out. Question: if raw material improves slightly, how much does final product improve?

Chain rule says: multiply local effects at each stage. If Station A amplifies change by 2, Station B preserves it with factor 1, Station C dampens it with factor 0.5, total amplification = 2 × 1 × 0.5 = 1. Small input improvement gives equal output improvement.

Math version: for three nested functions — output = f(g(h(x))) — chain rule says:

d(output)/dx = f'(g(h(x))) · g'(h(x)) · h'(x)

Each function contributes local derivative — how much it amplifies or dampens what passes through. Total derivative = product of local amplifications. Product = gradient.

Neural network is composition of functions. Weighted sum is one function. ReLU is another. Sigmoid is another. Loss is another. Chain rule tells how tiny change in any weight ripples through later layers and changes final loss.

That is backpropagation. Start at loss. Compute local derivative at each operation. Multiply backward. Each weight gets own gradient — own blame signal.

We keep returning to this assembly line. Gradients vanish when one station has near-zero amplification, and multiplying that through kills upstream signal. Gradients explode when factors stay above one and product runs wild. Chain rule is product. Product obeys its factors.

Backward Pass: Trace Blame

Walk backward through example. We have L ≈ 0.4629. Need ∂L/∂w for all nine parameters. Start at loss. Move backward. Apply chain rule at each operation.

Loss to Output Pre-activation

For binary cross-entropy with sigmoid, clean simplification appears. Instead of computing ∂L/∂ŷ and ∂ŷ/∂z₃ separately then multiplying, product collapses to:

∂L/∂z₃ = ŷ − y = 0.6295 − 1.0 = −0.3705

First time, this looks suspiciously clean. But expand derivatives of −log(σ(z)), apply chain rule, and sigmoid derivative σ(z)(1−σ(z)) cancels with terms from cross-entropy derivative. Math works. Very satisfying cancellation.

Sign matters. Negative means loss drops if z₃ increases. Makes sense: ŷ = 0.63 is too low because target is 1, so z₃ must rise to push sigmoid output toward 1. This number — −0.3705 — is error signal that ripples backward through whole network.

Gradients for Output Layer

Output pre-activation was z₃ = v₁·h₁ + v₂·h₂ + b₃. To get each weight gradient, multiply upstream error signal by local derivative — for linear combination, local derivative is input connected to weight:

∂L/∂v₁ = (∂L/∂z₃) × h₁ = (−0.3705)(0.75)  = −0.2779
∂L/∂v₂ = (∂L/∂z₃) × h₂ = (−0.3705)(0.05)  = −0.0185
∂L/∂b₃ = (∂L/∂z₃) × 1  =                     −0.3705

Look at magnitudes. Weight v₁ connects to h₁ = 0.75, strong activation, so v₁ gets large gradient. Strong influence, strong blame. Weight v₂ connects to h₂ = 0.05, barely active, so v₂ gets tiny gradient. That neuron barely affected prediction. Math matches intuition.

Propagating Error to Hidden Layer

To get gradients for hidden-layer weights, first need error signal at each hidden neuron — how much did h₁ and h₂ contribute to loss? Weights between layers route upstream error signal:

∂L/∂h₁ = (∂L/∂z₃) × v₁ = (−0.3705)(0.6)   = −0.2223
∂L/∂h₂ = (∂L/∂z₃) × v₂ = (−0.3705)(−0.4)  = +0.1482

Interesting part: h₁ gets negative gradient, so it should increase. h₂ gets positive gradient, so it should decrease. Why? v₂ is negative. Negative weight flips blame sign. Network structure shapes gradient flow. Weights route signals forward and blame backward.

Through ReLU Gate

ReLU derivative is 1 if input was positive, 0 if input was negative or zero. Gate behavior — open or shut. Both z₁ = 0.75 and z₂ = 0.05 were positive, so gate stays open and gradient passes unchanged:

∂L/∂z₁ = (∂L/∂h₁) × ReLU'(z₁) = (−0.2223) × 1 = −0.2223
∂L/∂z₂ = (∂L/∂h₂) × ReLU'(z₂) = (+0.1482) × 1 = +0.1482
⚠️ The Dying ReLU Problem

If z₂ had been negative — say, −0.3 — ReLU would output 0, derivative would be 0. Then ∂L/∂z₂ = 0, and every weight feeding into that neuron gets zero gradient. Neuron dead. It contributes nothing, backprop assigns zero blame, zero blame means zero update. Neuron never recovers. This is dying ReLU problem. In assembly-line terms, one station's amplification is permanently zero, so no upstream signal gets through. Leaky ReLU and similar variants keep gate slightly open for negative inputs.

Gradients for Hidden Layer

Same pattern: upstream error signal × input to weight:

∂L/∂w₁₁ = (∂L/∂z₁) × x₁ = (−0.2223)(1.0) = −0.2223
∂L/∂w₁₂ = (∂L/∂z₁) × x₂ = (−0.2223)(0.5) = −0.1112
∂L/∂b₁  = (∂L/∂z₁) × 1   =                  −0.2223

∂L/∂w₂₁ = (∂L/∂z₂) × x₁ = (+0.1482)(1.0) = +0.1482
∂L/∂w₂₂ = (∂L/∂z₂) × x₂ = (+0.1482)(0.5) = +0.0741
∂L/∂b₂  = (∂L/∂z₂) × 1   =                  +0.1482

Now every parameter has gradient. One backward pass. Nine gradients. Full picture:

Parameter   Value    Gradient    What it means
─────────   ─────    ────────    ──────────────────────────
w₁₁         0.300    −0.2223     increase it (reduces loss)
w₁₂         0.700    −0.1112     increase it
b₁          0.100    −0.2223     increase it
w₂₁        −0.200    +0.1482     decrease it (reduces loss)
w₂₂         0.500    +0.0741     decrease it
b₂          0.000    +0.1482     decrease it
v₁          0.600    −0.2779     increase it
v₂         −0.400    −0.0185     increase it (make less negative)
b₃          0.100    −0.3705     increase it

Gradient points in direction of steepest ascent. To reduce loss, optimizer subtracts some fraction — learning rate — from each gradient. That is gradient descent. That is optimizer's job, not backprop's. Backprop only computed these nine numbers in one backward pass.

Universal Pattern

After this example, pattern should be clear. At every node in backward pass, same thing happened:

gradient for a weight = (upstream error signal) × (local input to that weight)
gradient for an input  = (upstream error signal) × (weight connecting to that input)

First line — upstream signal × local input — is only formula to remember for backward pass of linear layer. It shows up everywhere because derivative of z = w·x + b with respect to w is x, and derivative with respect to x is w. Chain rule packs downstream blame into upstream signal. Local derivative tells how much this weight or input contributed.

Activation functions add own local derivative multiplier. ReLU multiplies by 1 or 0. Sigmoid by σ(z)(1−σ(z)). Tanh by 1−tanh²(z). Each one shapes error signal as it passes — amplifies, damps, or kills it.

Once pattern clicks, backward pass through any layer — convolutional, recurrent, attention, normalization — is same game. Compute local derivatives. Multiply by upstream signal. Pass result upstream. Every architecture follows these rules.

Get gradient for every weight. Cost is O(n) — one backward pass, all gradients. That efficiency makes deep learning feasible.

Three Ways to Compute Derivatives

Before PyTorch implementation, need know why it chose this approach. Three fundamentally different ways exist to compute derivative. Tradeoffs matter at scale.

Numerical differentiation: wiggle each input by tiny ε, measure output change, divide. Formula: f'(x) ≈ (f(x+ε) − f(x)) / ε. Easy to implement. Works for any function, differentiable or not. But needs one forward pass per parameter and stays approximate — ε too small gives floating-point error, ε too large gives bad approximation. Main practical use: gradient checking. Write custom backward pass, compare analytical gradients to numerical ones. Little else.

Symbolic differentiation is what Wolfram Alpha does. Given sin(x²), it applies rules and produces 2x·cos(x²). It manipulates symbols, not numbers. Result is exact, closed-form formula. Problem: expression swell. Deeply nested functions can double expression size at each chain-rule step. A 50-layer network would create derivative expression huge enough to choke computer algebra system. Great for single equations. Bad for programs.

Automatic differentiation (AD) is sweet spot deep learning frameworks use. It does not produce expressions. It produces values. During forward pass, it records each operation (multiply, add, relu, etc.) in trace. During backward pass, it replays trace in reverse, applying chain rule to actual numbers at each step. No expression swell because no expression, only numbers multiplied at each node. Exact to machine precision because it uses analytical local derivatives, not finite differences. Handles arbitrary program structure: loops, conditionals, recursion. If Python executes it, autograd can usually differentiate through it.

Computational Graphs and Autograd

Write y = torch.relu(x @ W + b) in PyTorch, and more happens than visible computation. PyTorch builds computational graph — directed acyclic graph (DAG) where each node is operation (matrix multiply, add, relu) and each edge is tensor flow between operations. Every tensor with requires_grad=True joins this graph. Tensor stores reference to operation that created it — .grad_fn.

This graph is tape in tape-based automatic differentiation. PyTorch records forward pass as it runs, building tape on fly. That is dynamic computational graph — graph rebuilds every forward pass. If code takes different if branch for different inputs, graph changes too. Old TensorFlow (1.x) used static graph defined once, then run, so normal Python control flow did not work. PyTorch won people over because dynamic graphs keep normal Python control flow and still preserve gradients.

Call loss.backward(), and PyTorch walks graph in reverse topological order — from loss node back to leaf parameters. At each node, it does vector-Jacobian product (VJP): take incoming gradient vector from downstream, multiply by local Jacobian, pass result upstream. Framework never builds full Jacobian matrix — too large for operations like matrix multiplication. Instead, each operation has custom backward function that computes VJP efficiently. Result for each leaf parameter accumulates in .grad.

Key idea for vector-Jacobian product: vector flowing backward is gradient of loss with respect to node's output. Jacobian says how that output depends on node inputs. Multiply them, get gradient of loss with respect to node inputs. Chain rule as matrix-vector product. One node at time.

Forward Mode vs. Reverse Mode

Automatic differentiation has two modes. Choice changes efficiency a lot.

Forward mode propagates derivatives from inputs to outputs. Pick one input parameter, seed derivative 1, trace that derivative forward through every operation to every output. End result: derivative of all outputs with respect to one input. Need different input? Start over. For function with n inputs and m outputs, forward mode costs n forward passes for full gradient.

Reverse mode propagates derivatives from outputs to inputs. Start with one output — loss — seed derivative 1, trace backward through every operation to every input. End result: derivative of that one output with respect to all inputs. For n inputs and m outputs, reverse mode costs m backward passes.

Neural networks have millions of input parameters and one scalar loss. Forward mode needs millions of passes. Reverse mode needs one. Gap is not 2× or 10×. Gap is millions.

Reverse-mode automatic differentiation is backpropagation. Same algorithm. Name "backpropagation" came from neural network community. Name "reverse-mode AD" came from numerical computing community. Same insight: many inputs, few outputs, so work backward.

Forward mode still matters — ideal for few inputs and many outputs, like Jacobian of physics simulation with three control knobs and million mesh points. But for neural network training, reverse mode wins.

When Signal Dies: Vanishing and Exploding Gradients

Chain rule computes gradient as product of local derivatives. Products are fragile. Small factor shrinks whole product. Many small factors make product vanish.

Back to assembly line. Each station has amplification factor — local derivative. If Station A, B, C, and D each have factor 0.25, signal reaching first station is 0.25⁴ = 0.004. 99.6% gone after four layers. After ten layers? 0.25¹⁰ ≈ 0.000001. Early layers get almost no gradient. They cannot learn. This is vanishing gradient problem.

Why early deep learning suffered: sigmoid activation has maximum derivative 0.25. That happens only at input zero, and derivative gets smaller away from zero. If every layer uses sigmoid, gradient shrinks by at least factor 4 at each layer. Hochreiter documented this in 1991. Core reason deep networks stayed hard to train for decades.

Mirror problem is exploding gradients. If local derivatives stay above 1, product grows exponentially. After 50 layers with factor 2, gradient is 2⁵⁰ ≈ 10¹⁵. Weight updates become huge. Loss oscillates. Training diverges. Common in recurrent neural networks, where backpropagation through time (BPTT) unrolls same recurrence across hundreds of time steps — "depth" becomes sequence length, and same weight matrix multiplies into gradient product at each step.

Fixes that unlocked deep learning all attack this product-of-many-numbers problem:

ReLU (2011, popularized by Glorot et al.) has derivative exactly 1 for positive inputs. Unlike sigmoid's maximum 0.25, ReLU does not systematically shrink gradient. Chain-rule product stays closer to 1 across many layers. This swap — sigmoid to ReLU — was major turning point.

Careful weight initialization keeps activation and gradient variance stable across layers. Xavier initialization (Glorot, 2010) targets sigmoid/tanh. He initialization (2015) targets ReLU. Both set initial weight scale from fan-in and fan-out, so expected signal size neither grows nor shrinks as it moves through network.

Residual connections (He et al., 2015) add block input directly to output: output = block(x) + x. During backprop, gradient flows through addition unchanged — derivative of x + something with respect to x is always 1, no matter what block does. This creates "gradient highway" that bypasses block. Even if block gradients vanish, residual path keeps signal alive. That is how ResNets train hundreds of layers while plain networks collapse.

Batch normalization (Ioffe and Szegedy, 2015) normalizes each layer's inputs to zero mean and unit variance. This keeps activations out of saturated sigmoid/tanh regions where derivatives become tiny.

Gradient clipping caps gradient norm at threshold during training. If gradient vector norm exceeds threshold, whole vector gets scaled down. This does not fix vanishing gradients, but it tames exploding ones. Standard for RNNs and Transformers.

PyTorch Autograd in Practice

Theory done. Now actual PyTorch code. Computational graph, VJPs, reverse-mode traversal stay hidden. Here is visible part:

import torch
import torch.nn as nn

model = nn.Sequential(
    nn.Linear(2, 2),
    nn.ReLU(),
    nn.Linear(2, 1),
    nn.Sigmoid()
)
loss_fn = nn.BCELoss()

x = torch.tensor([[1.0, 0.5]])
y = torch.tensor([[1.0]])

y_hat = model(x)          # forward pass — builds the computational graph
loss = loss_fn(y_hat, y)   # compute loss — extends the graph

loss.backward()            # backward pass — computes ALL gradients

for name, param in model.named_parameters():
    print(f"{name:15s}  grad_norm = {param.grad.norm().item():.4f}")

model(x) runs forward pass and records tape. loss.backward() walks tape backward and fills .grad on every parameter with requires_grad=True (default inside nn.Module). Then optimizer reads gradients and updates weights.

Complete training loop has three core lines:

optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for epoch in range(1000):
    y_hat = model(x)
    loss = loss_fn(y_hat, y)

    optimizer.zero_grad()    # clear old gradients
    loss.backward()          # compute new gradients
    optimizer.step()         # update weights using those gradients

zero_grad(), backward(), step(). Every training loop has this skeleton. Reason zero_grad() matters comes next.

During inference, gradients not needed. Turn off graph construction. Save memory. Run faster:

with torch.no_grad():
    predictions = model(test_inputs)

# Or, for an entire function:
@torch.inference_mode()
def predict(model, inputs):
    return model(inputs)

Details That Bite: Accumulation, Detaching, Checkpointing

Gradient Accumulation — Common Backprop Bug

PyTorch accumulates gradients by default. Each call to .backward() adds to existing .grad tensors instead of replacing them. Forget zero_grad(), gradients grow every batch. Weight updates become huge. Loss explodes. Common backprop bug.

# BUG: gradients accumulate and grow every iteration
for batch in dataloader:
    loss = compute_loss(model, batch)
    loss.backward()
    optimizer.step()

# FIX: reset gradients before each backward pass
for batch in dataloader:
    optimizer.zero_grad()
    loss = compute_loss(model, batch)
    loss.backward()
    optimizer.step()

Accumulation is not design flaw. Deliberate feature. It lets training simulate larger batch sizes when GPU memory is tight. Compute gradients on four small batches, accumulate them, then take one optimizer step. Effective batch size = 4× actual batch size:

accumulation_steps = 4
optimizer.zero_grad()

for i, batch in enumerate(dataloader):
    loss = compute_loss(model, batch) / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()

Detaching Tensors from Graph

.detach() creates tensor that shares same data but is disconnected from computational graph. Gradients do not flow through it. Needed when computed value must not affect gradient computation — target networks in reinforcement learning, teacher models in distillation, stop-gradient tricks in self-supervised learning:

# RL: target Q-values must NOT influence gradient computation
with torch.no_grad():
    target_q = target_network(next_state).max(dim=1).values
loss = F.mse_loss(current_q, target_q.detach())

# Self-supervised learning (SimSiam): stop gradient on one branch
z1 = projector(encoder(x1))
z2 = projector(encoder(x2)).detach()
loss = -F.cosine_similarity(z1, z2).mean()

Difference between .detach() and torch.no_grad(): detach() disconnects one specific tensor while rest of graph stays intact. torch.no_grad() prevents graph construction entirely for all operations inside context manager. Use no_grad() for inference. Use detach() when one part needs gradients and another part does not.

Gradient Checkpointing — Trade Compute for Memory

During forward pass, PyTorch stores every intermediate activation because backward pass needs them for gradients. Network with 100 layers means 100 layers of tensors in memory. In large models with long sequences, this memory cost can exceed model weights.

Gradient checkpointing fights this by storing only subset of activations — checkpoints. During backward pass, when missing activation is needed, PyTorch reruns forward pass from nearest checkpoint to recompute it. Memory drops from O(n) to O(√n) at cost of roughly one extra forward pass. In practice, about 30% more compute for 60% less activation memory.

from torch.utils.checkpoint import checkpoint

class DeepModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.block1 = HeavyBlock()
        self.block2 = HeavyBlock()
        self.block3 = HeavyBlock()

    def forward(self, x):
        x = checkpoint(self.block1, x, use_reentrant=False)
        x = checkpoint(self.block2, x, use_reentrant=False)
        x = self.block3(x)
        return x

Standard for LLM fine-tuning and deep-model training on limited hardware. Tradeoff is usually worth it.

What Backprop Does NOT Do

Backprop is powerful, but scope is narrower than many think. Three things often get misattributed to it.

Backprop does not update weights. It computes gradients, nothing more. Actual update — subtract lr × gradient, maintain momentum, adapt per-parameter learning rates — belongs to optimizer. SGD, Adam, AdaGrad, LAMB — separate algorithms that consume backprop output. Backprop does not care which optimizer runs.

Backprop does not find global minimum. Neural-network loss landscape is non-convex, full of local minima, saddle points, and flat plateaus. Backprop gives gradient at current location — direction of steepest local descent. Whether that leads to good solution depends on initialization, learning-rate schedule, optimizer choice, and geometry of loss surface. We have high-level stories about wide minima generalizing better, but no complete explanation for why SGD finds such good solutions so often.

Backprop does not choose architecture. It works on any differentiable computational graph — feed-forward, convolutional, recurrent, attention-based, graph networks. It computes gradients for whatever structure exists. Whether that structure fits problem is separate question. Backprop has no opinion.

The Three-Step Dance

Forward pass: input → prediction → loss. What network currently does.
Backward pass (backprop): loss → gradients for every parameter. How each weight contributed to error.
Optimizer step: update weights using gradients. How to fix error.

Backprop is step 2. Whole job.
