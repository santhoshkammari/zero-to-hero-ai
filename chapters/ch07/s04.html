Backpropagation

Chapter 7: Deep Learning Foundations Section 4 of 7
TL;DR

Training a neural network means: run input forward, measure error, figure out which weights caused the error, nudge them. Backpropagation is the "figure out which weights caused the error" part. It's the chain rule from calculus, applied backwards through the network. That's literally all it is. Each layer computes a local gradient вҖ” how much its output changes when you wiggle its input вҖ” and these local gradients get multiplied together to tell every single weight in the network exactly how much it contributed to the final loss. No weight gets special treatment. No layer is exempt. The gradient flows backwards from the loss through every operation, and every weight gets its own personalized "blame signal." The algorithm is elegant, efficient, and the reason deep learning works at all.

The Core Insight

Here's the situation. You have a neural network with millions of parameters. You feed in an input, it produces an output, and that output is wrong. The loss function gives you a single number quantifying how wrong. Now you need to answer the hardest question in all of machine learning: which of these millions of weights should change, and by how much?

You could try changing each weight one at a time, re-running the forward pass, and seeing if the loss goes down. That's numerical differentiation. For a network with n parameters, you'd need n forward passes to get one gradient vector. GPT-3 has 175 billion parameters. That's 175 billion forward passes per training step. Not happening.

Backpropagation computes the gradient of the loss with respect to every weight in the network in just two passes вҖ” one forward, one backward. It does this by exploiting the chain rule of calculus and the fact that a neural network is a composition of differentiable functions. Each layer computes a simple local derivative, and the backward pass multiplies these local derivatives together in the right order. The result: every weight gets its gradient, and the total cost is roughly 2Г— one forward pass regardless of how many parameters you have.

That efficiency is not a minor detail. It's the reason deep learning exists. Without backprop (or something equivalently efficient), training networks with more than a few hundred parameters would be computationally infeasible. The algorithm was popularized by Rumelhart, Hinton, and Williams in 1986, and every neural network trained since then вҖ” every ResNet, every Transformer, every GPT вҖ” uses it.

Forward Pass: A Concrete Example

Before we can backpropagate, we need something to backpropagate through. Let's build a tiny network and trace every computation. Two inputs, two hidden neurons with ReLU, one output neuron with sigmoid, binary cross-entropy loss. Small enough to compute by hand, large enough to contain every pattern that scales to real networks.

The Chain Rule вҖ” Why It Works

Before we go backward through the network, let's make sure the chain rule is bone-deep intuitive, because everything else is just applying it repeatedly.

Suppose you have a composition of three functions: output = f(g(h(x))). You want to know: if I nudge x by a tiny amount Оө, how much does output change? The chain rule says:

d(output)/d(x) = f'(g(h(x))) В· g'(h(x)) В· h'(x)

In plain English: multiply the local derivatives at each stage. First, h amplifies or dampens the nudge by h'(x). Then g amplifies or dampens what it receives by g'(h(x)). Then f does the same by f'(g(h(x))). The total effect is the product. If h doubles the nudge, g triples it, and f halves it, the total amplification is 2 Г— 3 Г— 0.5 = 3.

A neural network is a composition of functions. Each layer is one function. The chain rule tells you exactly how a tiny change in any weight ripples through all subsequent layers to affect the final loss. That's it. That's backpropagation.

You start at the loss, compute the local gradient at each layer, and multiply backwards. Each weight gets its own gradient вҖ” its own "blame signal" for the error. Weights that contributed a lot to the error get large gradients. Weights that barely influenced the output get tiny gradients. Weights behind a dead ReLU (which outputs zero, so its derivative is zero) get exactly zero gradient вҖ” the chain rule multiplies by zero and the blame signal dies.

The Assembly Line Analogy

Imagine a factory with five workstations, each performing one operation on a product. A defective product comes off the end of the line. You need to figure out which station screwed up.

Station 5 added a crooked label вҖ” small contribution to the defect. Station 4 applied the paint evenly вҖ” no contribution. Station 3 drilled a hole off-center вҖ” big contribution. Station 2 cut the metal to spec вҖ” no contribution. Station 1 selected slightly impure raw material вҖ” moderate contribution.

Each station gets blame proportional to its contribution to the defect. That blame is the gradient. The process of tracing backwards from the defect through each station is backpropagation. And the mathematical tool that makes the blame-assignment precise вҖ” that lets you multiply local contributions to get the total effect вҖ” is the chain rule.

The key insight that makes this efficient: you don't need to trace a separate path for every station. You compute the blame at the end (station 5), pass it backward to station 4 (multiplying by station 5's local contribution), then to station 3, and so on. One backward pass gives you blame for every station. That's why backprop is O(n) and not O(nВІ).

Backward Pass: Following the Gradient

Let's continue the concrete example. We have the loss L = 0.4629 and we need вҲӮL/вҲӮw for every weight and bias. We work backward through the computational graph, applying the chain rule at every node.

Step 1 вҖ” From Loss to Output Pre-activation

For binary cross-entropy with sigmoid, there's a famously clean combined derivative. Instead of computing вҲӮL/вҲӮЕ· and then вҲӮЕ·/вҲӮzвӮғ separately, the product simplifies to:

вҲӮL/вҲӮzвӮғ = Е· вҲ’ y = 0.6295 вҲ’ 1.0 = вҲ’0.3705

This is negative вҖ” the loss decreases if zвӮғ increases. Makes sense: Е· = 0.63 is too low (target is 1), so we need zвӮғ to be larger to push the sigmoid output closer to 1. This single number is the "error signal" that will propagate backward through the entire network.

Step 2 вҖ” Gradients for Output Layer Weights

The output pre-activation is zвӮғ = vвӮҒВ·hвӮҒ + vвӮӮВ·hвӮӮ + bвӮғ. Each weight's gradient is the upstream gradient times the local derivative:

вҲӮL/вҲӮvвӮҒ = вҲӮL/вҲӮzвӮғ В· вҲӮzвӮғ/вҲӮvвӮҒ = (вҲ’0.3705) Г— hвӮҒ = (вҲ’0.3705)(0.75)  = вҲ’0.2779
вҲӮL/вҲӮvвӮӮ = вҲӮL/вҲӮzвӮғ В· вҲӮzвӮғ/вҲӮvвӮӮ = (вҲ’0.3705) Г— hвӮӮ = (вҲ’0.3705)(0.05)  = вҲ’0.0185
вҲӮL/вҲӮbвӮғ = вҲӮL/вҲӮzвӮғ В· вҲӮzвӮғ/вҲӮbвӮғ = (вҲ’0.3705) Г— 1  =                    вҲ’0.3705

Notice the pattern: the gradient for a weight is always (upstream error signal) Г— (input to that weight). Weight vвӮҒ connects to hвӮҒ = 0.75, so its gradient is large. Weight vвӮӮ connects to hвӮӮ = 0.05, so its gradient is tiny вҖ” hвӮӮ barely contributed to the output, so vвӮӮ barely contributed to the error.

Step 3 вҖ” Propagate Error to Hidden Layer

We're not done. We also need gradients for the hidden layer weights. To get those, we first need the error signal at each hidden neuron вҖ” how much did hвӮҒ and hвӮӮ contribute to the loss?

вҲӮL/вҲӮhвӮҒ = вҲӮL/вҲӮzвӮғ В· вҲӮzвӮғ/вҲӮhвӮҒ = (вҲ’0.3705) Г— vвӮҒ = (вҲ’0.3705)(0.6)   = вҲ’0.2223
вҲӮL/вҲӮhвӮӮ = вҲӮL/вҲӮzвӮғ В· вҲӮzвӮғ/вҲӮhвӮӮ = (вҲ’0.3705) Г— vвӮӮ = (вҲ’0.3705)(вҲ’0.4)  = +0.1482

The error signal splits at the hidden layer. hвӮҒ has a negative gradient (we want it to increase), and hвӮӮ has a positive gradient (we want it to decrease). The weights between layers act as routing coefficients for the error signal вҖ” vвӮӮ is negative, so it flips the sign of the blame.

Step 4 вҖ” Through the ReLU

ReLU's derivative is either 1 (input was positive) or 0 (input was negative). Both zвӮҒ = 0.75 and zвӮӮ = 0.05 were positive, so ReLU passes the gradient through unchanged:

вҲӮL/вҲӮzвӮҒ = вҲӮL/вҲӮhвӮҒ В· ReLU'(zвӮҒ) = (вҲ’0.2223) Г— 1 = вҲ’0.2223
вҲӮL/вҲӮzвӮӮ = вҲӮL/вҲӮhвӮӮ В· ReLU'(zвӮӮ) = (+0.1482) Г— 1 = +0.1482
вҡ пёҸ Dead Neurons

If zвӮӮ had been negative (say, вҲ’0.3), then ReLU would have output 0, and ReLU'(zвӮӮ) = 0. The gradient вҲӮL/вҲӮzвӮӮ would be zero, and every weight feeding into hidden neuron 2 would get zero gradient. That neuron is dead вҖ” it contributes nothing to the output, and backprop assigns it zero blame, so it never learns. This is the "dying ReLU" problem, and it's why Leaky ReLU and other variants exist.

Step 5 вҖ” Gradients for Hidden Layer Weights

Same pattern as before: (upstream error signal) Г— (input to that weight).

вҲӮL/вҲӮwвӮҒвӮҒ = вҲӮL/вҲӮzвӮҒ В· xвӮҒ = (вҲ’0.2223)(1.0) = вҲ’0.2223
вҲӮL/вҲӮwвӮҒвӮӮ = вҲӮL/вҲӮzвӮҒ В· xвӮӮ = (вҲ’0.2223)(0.5) = вҲ’0.1112
вҲӮL/вҲӮbвӮҒ  = вҲӮL/вҲӮzвӮҒ В· 1   =                  вҲ’0.2223

вҲӮL/вҲӮwвӮӮвӮҒ = вҲӮL/вҲӮzвӮӮ В· xвӮҒ = (+0.1482)(1.0) = +0.1482
вҲӮL/вҲӮwвӮӮвӮӮ = вҲӮL/вҲӮzвӮӮ В· xвӮӮ = (+0.1482)(0.5) = +0.0741
вҲӮL/вҲӮbвӮӮ  = вҲӮL/вҲӮzвӮӮ В· 1   =                  +0.1482

The gradients tell us the direction of steepest ascent. To minimize the loss, the optimizer subtracts some fraction (the learning rate) of each gradient from the corresponding weight. That's gradient descent вҖ” but that's the optimizer's job, not backprop's. Backprop's job was computing these nine numbers, and it did it in one backward pass.

Computational Graphs and Autograd

When you write y = torch.relu(x @ W + b) in PyTorch, it doesn't just compute the result. It builds a computational graph вҖ” a directed acyclic graph (DAG) where each node is an operation (matmul, add, relu) and each edge is a tensor flowing between operations. The graph records every operation that happened during the forward pass, along with the information needed to compute derivatives.

When you call loss.backward(), PyTorch walks this graph in reverse topological order вҖ” from the loss back to the inputs. At each node, it applies the chain rule: take the incoming gradient (from downstream), multiply by the local Jacobian (derivative of this operation with respect to its inputs), and pass the result upstream. This is reverse-mode automatic differentiation, and it's what every deep learning framework implements under the hood.

The reason we use reverse-mode (and not forward-mode) is efficiency. Reverse-mode computes the gradient of one scalar (the loss) with respect to all parameters in a single backward pass. Forward-mode computes the derivative of all outputs with respect to one parameter per pass. Since we have one loss scalar and millions of parameters, reverse-mode wins by a factor of millions. That's the whole story on forward-mode vs reverse-mode вҖ” reverse-mode is backprop.

Autograd Is Not Symbolic Differentiation

Symbolic differentiation (what Wolfram Alpha does) takes an expression like sin(xВІ) and produces the expression 2xВ·cos(xВІ). It manipulates symbols. Automatic differentiation doesn't produce expressions вҖ” it produces values. It evaluates the derivative at a specific point by tracing the computation and applying the chain rule to actual numbers. This is why autograd works on arbitrary Python code with control flow, loops, and conditionals. There's no symbolic expression to simplify. There's just a recorded trace of operations and numbers.

PyTorch Autograd in Practice

The computational graph is an implementation detail you rarely think about. In practice, backprop in PyTorch looks like this:

import torch
import torch.nn as nn

# в”Җв”Җ A simple network в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
model = nn.Sequential(
    nn.Linear(2, 2),    # hidden layer: 2 inputs вҶ’ 2 neurons
    nn.ReLU(),
    nn.Linear(2, 1),    # output layer: 2 вҶ’ 1
    nn.Sigmoid()
)
loss_fn = nn.BCELoss()

# в”Җв”Җ Forward pass в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
x = torch.tensor([[1.0, 0.5]])       # input
y = torch.tensor([[1.0]])            # target
y_hat = model(x)                     # forward: builds the graph
loss = loss_fn(y_hat, y)             # compute loss

# в”Җв”Җ Backward pass в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
loss.backward()                      # compute ALL gradients

# в”Җв”Җ Every parameter now has a .grad в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
for name, param in model.named_parameters():
    print(f"{name:20s} shape={str(list(param.shape)):10s} "
          f"grad_norm={param.grad.norm().item():.4f}")

That's the whole pattern. model(x) runs the forward pass and records the graph. loss.backward() walks the graph backward and populates .grad on every parameter that has requires_grad=True (which is the default for nn.Parameter). After that, the optimizer reads those gradients and updates the weights.

A complete training step:

optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for epoch in range(1000):
    # Forward
    y_hat = model(x)
    loss = loss_fn(y_hat, y)

    # Backward
    optimizer.zero_grad()    # CRITICAL: clear old gradients first
    loss.backward()          # compute new gradients

    # Update
    optimizer.step()         # nudge weights in the direction that reduces loss

Three lines in the loop. That's it. zero_grad(), backward(), step(). Every training loop you'll ever write has this skeleton.

For inference, you don't need gradients. Disable graph construction to save memory and compute:

# During inference вҖ” no graph, no gradients
with torch.no_grad():
    predictions = model(test_inputs)

# Equivalent: decorate the whole function
@torch.inference_mode()
def predict(model, inputs):
    return model(inputs)

Key Practical Details

Gradient Accumulation

This is the most common backprop bug in PyTorch: gradients accumulate by default. Each call to .backward() adds to the existing .grad tensors rather than replacing them. If you forget zero_grad(), your gradients grow with every batch, your weight updates become enormous, and your loss explodes.

# BUG: gradients grow every iteration
for batch in dataloader:
    loss = compute_loss(model, batch)
    loss.backward()          # accumulates onto existing .grad!
    optimizer.step()

# FIX: zero gradients before each backward pass
for batch in dataloader:
    optimizer.zero_grad()    # reset .grad to zero
    loss = compute_loss(model, batch)
    loss.backward()
    optimizer.step()

But accumulation isn't a bug вҖ” it's a feature. It lets you simulate larger batch sizes when you don't have enough GPU memory:

# Gradient accumulation: effective batch size = 4 Г— actual batch size
accumulation_steps = 4
optimizer.zero_grad()

for i, batch in enumerate(dataloader):
    loss = compute_loss(model, batch) / accumulation_steps
    loss.backward()                    # accumulate gradients

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()               # update with accumulated gradient
        optimizer.zero_grad()          # reset for next accumulation cycle

Detaching from the Graph

.detach() creates a tensor that shares the same data but is disconnected from the computational graph. Gradients will not flow through a detached tensor. This is essential in several scenarios:

# Target networks in reinforcement learning (DQN, etc.)
# The target should NOT influence gradient computation
with torch.no_grad():
    target_q = target_network(next_state).max(dim=1).values
loss = F.mse_loss(current_q, target_q.detach())

# Teacher-student distillation вҖ” teacher gradients are frozen
teacher_logits = teacher_model(x).detach()
student_logits = student_model(x)
loss = F.kl_div(student_logits.log_softmax(dim=-1),
                teacher_logits.softmax(dim=-1))

# Stopping gradient in part of a network (e.g., SimSiam, BYOL)
z1 = projector(encoder(x1))
z2 = projector(encoder(x2)).detach()  # stop-gradient on one branch
loss = -F.cosine_similarity(z1, z2).mean()

Gradient Checkpointing

During the forward pass, PyTorch stores every intermediate activation (every hвӮҒ, zвӮғ, etc.) because the backward pass needs them to compute gradients. For a deep network, this memory cost is enormous вҖ” it scales linearly with depth. A 100-layer ResNet stores 100 layers worth of activations.

Gradient checkpointing trades compute for memory. Instead of storing all activations, you store only a few "checkpoints." During the backward pass, when you need an activation that wasn't stored, you recompute it by re-running the forward pass from the nearest checkpoint. This reduces memory from O(n) to O(вҲҡn) at the cost of one extra forward pass.

from torch.utils.checkpoint import checkpoint

class DeepModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.block1 = HeavyBlock()
        self.block2 = HeavyBlock()
        self.block3 = HeavyBlock()

    def forward(self, x):
        # Activations for block1 and block2 are recomputed during backward
        x = checkpoint(self.block1, x, use_reentrant=False)
        x = checkpoint(self.block2, x, use_reentrant=False)
        x = self.block3(x)  # last block stores activations normally
        return x

This is how people train models that would otherwise not fit in GPU memory. LLM fine-tuning with gradient checkpointing is standard practice вҖ” you're trading ~30% more compute for ~60% less activation memory.

What Backpropagation Does NOT Do

Backprop is powerful, but its scope is narrow. Three things people frequently misattribute to backprop:

Backprop does not update weights. It computes gradients вҖ” that's all. The actual weight update is the optimizer's job. SGD subtracts lr Г— gradient. Adam maintains per-parameter momentum and adaptive learning rates. These are separate algorithms that consume backprop's output. Backprop doesn't know or care which optimizer you use.

Backprop does not guarantee finding the global minimum. The loss landscape of a neural network is non-convex вҖ” it's riddled with local minima, saddle points, and flat regions. Backprop gives you the gradient at your current location, which points toward the steepest local descent. Following it will decrease the loss locally, but there's no guarantee you'll reach the best possible loss. Whether you converge to a good solution depends on initialization, learning rate, optimizer, and the geometry of the loss surface вҖ” not on backprop itself.

Backprop does not tell you which architecture to use. It works on any differentiable computation graph вҖ” feed-forward networks, convnets, transformers, graph networks, whatever. It computes gradients for the architecture you give it. Whether that architecture is well-suited to your problem is an entirely separate question.

The Full Training Pipeline

Forward pass: input вҶ’ prediction вҶ’ loss. (What the network currently does.)
Backward pass (backprop): loss вҶ’ gradients for all parameters. (How each parameter contributed to the error.)
Optimizer step: update each parameter using its gradient. (How to fix the error.)

Backprop is step 2. That's its entire job. Steps 1 and 3 are separate algorithms, covered in their own sections.

What You Should Now Be Able To Do

  • Explain backpropagation as "the chain rule applied in reverse through a computational graph" and mean it вҖ” not as a memorized phrase, but as a deep understanding of why multiplying local derivatives gives you the gradient for any weight in the network.
  • Trace a forward pass through a small network by hand вҖ” compute weighted sums, apply activations, compute the loss вҖ” and know exactly which intermediate values are needed for the backward pass.
  • Compute the backward pass by hand: start with вҲӮL/вҲӮoutput, multiply by local derivatives at each layer, and arrive at вҲӮL/вҲӮw for every weight. Verify that the gradient's sign makes intuitive sense (negative means "increase this weight to reduce loss").
  • Explain why backprop is O(n) вҖ” one backward pass gives gradients for all n parameters вҖ” and why this efficiency is what makes deep learning computationally feasible.
  • Write the zero_grad() вҶ’ backward() вҶ’ step() training loop in PyTorch and explain what each call does, including why zero_grad() is necessary.
  • Use .detach() and torch.no_grad() correctly вҖ” know when gradient flow should be stopped and why (target networks, inference, stop-gradient tricks).
  • Explain gradient checkpointing as a compute-memory tradeoff and know when to reach for it (large models, limited GPU memory, long sequences).
  • Clearly distinguish what backprop does (computes gradients) from what it doesn't do (update weights, find global minima, choose architectures).
