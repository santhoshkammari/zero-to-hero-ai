Activation Functions

Chapter 7: Deep Learning Foundations Section 2 of 7

Why Nonlinearity Matters

Stack two linear layers: y = W₂(W₁x). Expand it: y = (W₂W₁)x = Wx. You just collapsed two layers into one matrix multiply. Stack a hundred linear layers and you still get a single matrix multiply. The proof is one line of algebra and the implication is devastating: without a nonlinear activation between layers, depth is an illusion. Your 50-layer network computes exactly the same family of functions as a single layer.

Activation functions break this collapse. By inserting a nonlinear function after each layer's linear transformation, you make each layer's output something the next layer cannot replicate with a simple matrix multiply. This is what gives deep networks their power — each layer can carve a new nonlinear boundary in representation space, and the composition of these boundaries can approximate arbitrarily complex functions.

ReLU

The Rectified Linear Unit is the activation function that made deep learning practical:

ReLU(x) = max(0, x)

Three properties explain its dominance. First, no vanishing gradient for positive inputs — the derivative is exactly 1 when x > 0, so gradients flow through unchanged. Compare this to sigmoid where the maximum derivative is 0.25. Second, sparse activation — for any given input, roughly half the neurons output zero, which acts as implicit regularization and makes representations easier to interpret. Third, computational cheapness — it's a comparison and a branch, no exponentials, no divisions.

import torch
import torch.nn.functional as F

x = torch.linspace(-3, 3, 7)
print(x)           # tensor([-3., -2., -1.,  0.,  1.,  2.,  3.])
print(F.relu(x))   # tensor([ 0.,  0.,  0.,  0.,  1.,  2.,  3.])

The Dying ReLU Problem

If a neuron's weights get pushed into a region where its pre-activation is always negative — across every training example — the output is permanently zero. The gradient is also zero, so the weights never update. The neuron is dead. It contributes nothing for the rest of training.

This isn't rare. With a bad weight initialization or a too-large learning rate, you can kill 20–40% of your neurons in the first few epochs. The standard fix:

Leaky ReLU lets a small gradient through for negative inputs: f(x) = x if x > 0 else αx, where α is typically 0.01. Dead neurons can now recover because they still have a nonzero gradient. PReLU (learned α) and ELU (exponential curve for negatives) exist, but Leaky ReLU with α=0.01 is the standard insurance policy.

print(F.leaky_relu(x, negative_slope=0.01))
# tensor([-0.0300, -0.0200, -0.0100,  0.0000,  1.0000,  2.0000,  3.0000])

Sigmoid

The sigmoid (logistic) function squashes inputs to (0, 1):

σ(x) = 1 / (1 + e⁻ˣ)

For hidden layers, sigmoid is a bad choice. The maximum gradient is 0.25 (at x=0), and it drops to near-zero for |x| > 4. Chain five layers of this and your gradient is 0.25⁵ ≈ 0.001 — the vanishing gradient problem. Add that outputs are always positive (never zero-centered), which forces weight gradients to zig-zag during optimization.

Where sigmoid does belong: the output layer of a binary classifier. You need a probability between 0 and 1, sigmoid gives you exactly that. Paired with binary cross-entropy loss, the gradients simplify beautifully to (predicted − actual). Nowhere else.

Tanh

tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)

Tanh squashes inputs to (−1, 1). It's literally a rescaled sigmoid: tanh(x) = 2σ(2x) − 1. The key improvement is zero-centered output, which eliminates the zig-zag gradient problem. The maximum derivative is 1.0 (vs. sigmoid's 0.25), so gradients propagate more effectively near the origin.

But tanh still saturates at the extremes. For |x| > 3, the derivative is negligible and the vanishing gradient problem returns. You'll still encounter tanh inside LSTM and GRU gates (where bounded outputs are a design requirement), but for new architectures, it's mostly historical.

GELU

The Gaussian Error Linear Unit is the default activation in transformers — BERT, GPT, ViT all use it.

GELU(x) = x · Φ(x)    where Φ(x) is the CDF of the standard normal distribution

The intuition: GELU gates each input by how likely it is to be positive under a Gaussian distribution. Large positive values pass through nearly unchanged (Φ(x) ≈ 1). Large negative values get zeroed out (Φ(x) ≈ 0). But crucially, small negative values get a small nonzero output, weighted by their probability of being positive. This is smoother than ReLU's hard cutoff at zero, which helps optimization in the high-dimensional loss landscapes that transformers navigate.

print(F.gelu(x))
# tensor([-0.0036, -0.0454, -0.1587,  0.0000,  0.8413,  1.9546,  2.9964])
# Note: small negative inputs produce small negative outputs

In practice, the fast approximation GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))) is often used. You don't need to memorize this — your framework handles it. What matters: use nn.GELU() over nn.ReLU() when building transformer architectures.

SiLU / Swish

SiLU(x) = x · σ(x)

Swish (renamed SiLU in PyTorch) is a self-gated activation: the input gates itself through its own sigmoid. Like GELU, it's smooth and allows small negative values through. The shape is strikingly similar to GELU — the two functions differ by less than 0.1 across most of their range.

Discovered via automated search (Google Brain, 2017), used in EfficientNet and some diffusion model backbones. The practical difference between GELU and SiLU is negligible in most experiments — use whichever your architecture's reference implementation specifies.

Softmax

Softmax is not a per-neuron activation — it operates across an entire vector of logits to produce a probability distribution:

softmax(zᵢ) = eᶻⁱ / Σⱼ eᶻʲ

Every output is in (0, 1) and they sum to exactly 1. This makes softmax the standard final activation for multi-class classification. Your network outputs raw logits, softmax converts them to probabilities, and cross-entropy loss measures how far those probabilities are from the true labels.

Temperature Scaling

Divide the logits by a temperature parameter T before applying softmax:

logits = torch.tensor([2.0, 1.0, 0.1])

# T = 1.0 (default)
print(F.softmax(logits / 1.0, dim=0))   # tensor([0.6590, 0.2424, 0.0986])

# T = 0.5 (sharper — more confident)
print(F.softmax(logits / 0.5, dim=0))   # tensor([0.8360, 0.1131, 0.0508])

# T = 2.0 (smoother — more uniform)
print(F.softmax(logits / 2.0, dim=0))   # tensor([0.4566, 0.2784, 0.2650])

Low temperature sharpens toward the argmax. High temperature flattens toward uniform. Shows up in LLM generation (controlling "creativity"), knowledge distillation (soft targets at T=20), and calibration (post-hoc temperature tuning for overconfident models).

The Decision Rule

This should take you about five seconds:

Hidden layers:
 ├── Default              → ReLU (or Leaky ReLU for insurance)
 └── Transformer blocks   → GELU

Output layer:
 ├── Binary classification  → Sigmoid
 ├── Multi-class            → Softmax
 └── Regression             → None (linear output)

That covers 99% of cases. If a paper you're reproducing uses SiLU, use SiLU. If you're debugging dead neurons, swap to Leaky ReLU. But don't waste time A/B testing activation functions when your data pipeline has bugs — the activation function is almost never the bottleneck.
