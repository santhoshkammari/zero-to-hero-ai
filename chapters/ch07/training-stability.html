Section Overview

Deep networks don't train by default — they vanish, explode, or stagnate. Section traces why from first principles, then builds four pillars of modern deep learning: proper initialization, normalization layers, residual connections, gradient clipping. Follows single toy network through each failure mode and fix. By end, you diagnose broken training runs from gradient norms alone.

The Core Problem: Why Deep Networks Break

First time 40-layer network produces NaN on batch 12 — no idea which layer is responsible — gradient flow stops being someone else's problem. Here's that dive.

Culprit: chain rule. Backprop computes gradient for each weight by multiplying local gradients from every layer between loss and that weight. Weight in layer 1 of 50-layer network → gradient is product of ~50 terms. Each term includes activation derivative and weights in that layer. Terms consistently below 1 → product collapses to zero. Terms consistently above 1 → product explodes to infinity.

That's entire story. Everything else — initialization, normalization, residual connections, clipping — exists to keep per-layer multipliers near 1. Not shrinking, not growing. Stable.

Running example: 20-layer network classifying handwritten digits — small enough to watch every gradient by hand.

Vanishing Gradients

Every layer uses sigmoid activation. Sigmoid squashes input to (0, 1), derivative is σ'(x) = σ(x) · (1 − σ(x)). Maximum derivative: 0.25 at x = 0. That's the ceiling. Once neuron saturates (output near 0 or 1), local derivative drops toward zero.

Trace backprop: gradient for layer 1's weights multiplied by sigmoid derivative at every layer. Best case — every neuron at zero input, every derivative at maximum 0.25 — gradient shrinks by 0.25 per layer.

# Best-case gradient magnitude with sigmoid across depth
for depth in [10, 20, 50]:
    grad = 0.25 ** depth
    print(f"After {depth:2d} layers: 0.25^{depth} = {grad:.2e}")

# After 10 layers: 0.25^10 = 9.54e-07
# After 20 layers: 0.25^20 = 9.09e-13
# After 50 layers: 0.25^50 = 7.89e-31

After 10 layers: gradient is one millionth of original. After 20: less than trillionth. After 50: float32 rounds to exactly zero. Remember — 0.25 is best case. Real neurons saturate often, actual numbers worse.

First time computing 0.25²⁰ = 10⁻¹³ feels unreal. But that's literally what happens. Early layers receive gradients so small weights barely move from random start. Last few layers near loss train fine — healthy gradients. Result: network where top floors furnished, foundation still scaffolding. Penthouse on sand.

How You Spot It

Symptom is deceptive. Loss plateaus early, looks like convergence. But model learned nothing useful — more like it gave up. Giveaway: check gradient norms per layer. Last layers show healthy values (10⁻² to 10⁰). First layers show 10⁻⁹ or smaller. That ratio — orders of magnitude difference between early and late layers — fingerprint of vanishing gradients.

⚠️ The Deceptive Plateau

Flat loss curve doesn't always mean convergence. Training deep network with sigmoid/tanh and loss stops improving early → print gradient norms for first few layers before concluding model learned what it can. Might discover early layers never started learning.

Exploding Gradients

Same chain rule, opposite failure. Per-layer multipliers consistently >1 → product detonates. Happens when weight matrices have large eigenvalues amplifying gradient at each layer.

# Gradient amplification across layers
for factor in [1.5, 2.0, 3.0]:
    result = factor ** 50
    print(f"{factor}^50 = {result:.2e}")

# 1.5^50 = 6.38e+08
# 2.0^50 = 1.13e+15
# 3.0^50 = 7.18e+23

Per-layer multiplier of 1.5 — not far from 1 — means first layer's gradient is 638 million times larger than output gradient. Weight updates become enormous, weights jump to absurd values, next forward pass produces inf or NaN. Training over in one step.

Back to digit classifier. Weights initialized too large, or using RNN where same weight matrix multiplied at every time step. In RNNs this is literal matrix exponentiation — spectral radius >1 means gradients grow exponentially with sequence length. 200-token sentence = multiplying same matrix 200 times. Math is unforgiving.

How You Spot It

Exploding gradients are much louder. Loss decreasing normally — maybe looking promising — then suddenly spikes to NaN or inf. No warning, no gradual degradation. One moment learning, next moment dead. Log weight values → see them balloon to astronomical magnitudes in batch right before crash.

💡 Quick Diagnostic

Loss suddenly jumps to NaN? Likely exploding gradients. Loss plateaus, early layers barely update? Likely vanishing gradients. Root cause same — chain rule amplifying deviations from 1 — but they fail differently and need different fixes.

Weight Initialization: The First Line of Defense

Most direct way to keep per-layer multipliers near 1: start with right weights. Wrong initialization doesn't give slow start — gives death sentence.

The Symmetry Problem

Before scale, even more basic failure. Initialize all weights to zero → every neuron computes same weighted sum, produces same output, receives same gradient, gets same update. After one step: still identical. After million steps: still identical. Paid for 512 neurons, got computational power of one.

This is symmetry problem. Identical neurons have no mechanism to become different. Random initialization breaks symmetry — each neuron starts with different weights, specializes during training, learns different feature. Randomness isn't compromise. It's requirement.

Scale Is Everything

So initialize randomly. But from what distribution? Weights from N(0, 10) — activations explode. By layer 5: float overflow. Weights from N(0, 0.001) — activations collapse to zero. By layer 5: every neuron outputs same tiny value, gradients vanish.

Trace through 20-layer network. Single neuron with n inputs: output before activation is z = w₁x₁ + w₂x₂ + ... + wₙxₙ. Weights and inputs independent with mean zero → variance of z is n · Var(w) · Var(x). Factor of n is key — output variance depends on layer width. Var(w) too large → variance grows each layer. Too small → shrinks. Need Var(w) calibrated so activation variance stays constant layer to layer. Not growing, not shrinking. Stable.

Xavier / Glorot Initialization (2010)

Xavier Glorot and Yoshua Bengio worked out the math. Derivation assumes roughly linear activations (valid for sigmoid/tanh near zero, where properly initialized neurons start). To preserve variance in both forward and backward pass:

Var(W) = 2 / (fan_in + fan_out)

fan_in = number of inputs, fan_out = number of outputs. The 2 comes from averaging forward-pass requirement (1/fan_in) with backward-pass requirement (1/fan_out). In practice: sample from N(0, 2/(fan_in + fan_out)) or equivalent uniform.

Works well for sigmoid and tanh — but has blind spot.

He / Kaiming Initialization (2015)

ReLU breaks Xavier derivation. ReLU zeros all negative inputs, killing ~half activations per layer. That halving means Xavier's variance too small by factor of 2 — activations shrink layer by layer, signals decay significantly by layer 20.

Kaiming He's fix: double variance, use only fan_in.

Var(W) = 2 / fan_in

Factor of 2 compensates exactly for ReLU zeroing negative half of distribution. With He init + ReLU: activation variance stays constant across depth. 20-layer digit classifier has signal reaching input to output, gradients flowing all way back.

💡 The Decision Rule
ActivationInitializationWhy
ReLU / Leaky ReLUHe (Kaiming)Compensates for ReLU killing ~50% of inputs
Sigmoid / TanhXavier (Glorot)Assumes near-linear behavior around zero
GELU / SiLUHe worksShaped similarly to ReLU in practice

In PyTorch

PyTorch's nn.Linear defaults to Kaiming uniform. So does nn.Conv2d. For most ReLU networks, defaults already correct — testament to how settled this is. When you need explicit control:

import torch.nn as nn

# He / Kaiming — the default choice for ReLU networks
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='relu')

# Xavier / Glorot — for sigmoid or tanh networks
nn.init.xavier_normal_(layer.weight)

# Apply custom init across an entire model
def init_weights(m):
    if isinstance(m, (nn.Linear, nn.Conv2d)):
        nn.init.kaiming_normal_(m.weight, nonlinearity='relu')
        if m.bias is not None:
            nn.init.zeros_(m.bias)

model.apply(init_weights)

Biases initialized to zero. Don't cause symmetry problem because weight matrices already different — each neuron's unique weights produce unique gradients regardless of shared bias.

Normalization Layers: Keeping Activations in Check

Proper initialization sets good starting point, but it's one-time thing. As training progresses, activation distributions shift — nicely centered distribution in layer 7 at step 0 might be wildly different at step 10,000. Normalization layers fix this dynamically, re-centering and re-scaling activations every forward pass.

Batch Normalization

Batch normalization introduced by Ioffe and Szegedy (2015), changed deep learning overnight. Mechanics are deceptively straightforward. For each feature: compute mean and variance across all examples in mini-batch, subtract mean, divide by std (plus tiny ε), then apply learned scale (γ) and shift (β).

# Batch norm forward pass (simplified)
# x has shape [batch_size, features]
mean = x.mean(dim=0)                    # mean per feature across batch
var  = x.var(dim=0)                     # variance per feature across batch
x_hat = (x - mean) / (var + 1e-5).sqrt()  # normalize
out = gamma * x_hat + beta              # learned scale and shift

Learned γ and β are crucial. Without them, normalization forces every layer's activations to mean=0, variance=1 — stripping representational power. γ and β let network learn optimal scale/shift while normalization ensures stable baseline.

Subtlety for production: during training, batch norm uses current mini-batch statistics. During inference, uses running averages accumulated during training (exponential moving average). Model's inference behavior doesn't depend on what else is in batch — matters more than it sounds.

Original paper claimed batch norm works by reducing internal covariate shift — shifting of layer input distributions as earlier weights change. This explanation contested. Later research suggests real benefit: smoothing loss landscape, making optimization surface better-conditioned so gradients more predictable and larger learning rates feasible. Debate unresolved, but batch norm works. Makes training faster, allows higher learning rates, reduces sensitivity to initialization.

Where Batch Norm Breaks Down

Batch norm has dependency that becomes liability: needs a batch. Batch size of 1 — common during inference or with huge models fitting one example per GPU — batch statistics meaningless (mean/variance of single sample tells nothing). Also struggles with variable-length sequences in NLP: tokens at position 50 might exist in only few sequences, making statistics unreliable.

This motivated different approach.

Layer Normalization

Layer normalization flips the axis. Instead of statistics across batch (across examples, per feature), computes statistics across features (within single example). Each sample normalized independently — no batch dependency, no issues with batch size 1, no variable-length problems.

# Layer norm forward pass (simplified)
# x has shape [batch_size, features]
mean = x.mean(dim=-1, keepdim=True)        # mean across features, per sample
var  = x.var(dim=-1, keepdim=True)         # variance across features, per sample
x_hat = (x - mean) / (var + 1e-5).sqrt()
out = gamma * x_hat + beta

Why every transformer uses layer norm. Language models have variable lengths, batch size often 1 during inference, training distributed across GPUs where synchronizing batch stats is expensive. Layer norm sidesteps all of this.

Pre-Norm vs. Post-Norm: Where You Put It Matters

Original transformer (2017) placed layer norm after residual connection — Post-LayerNorm: x' = LayerNorm(x + Sublayer(x)). Unstable for very deep models. Gradients can explode because unnormalized residual flows through sublayer before being tamed.

Pre-LayerNorm places normalization before sublayer: x' = x + Sublayer(LayerNorm(x)). Dramatically more stable. Residual path stays clean (no normalization disrupting it), sublayer always receives well-behaved input. GPT-3, Llama, PaLM, most modern LLMs use Pre-LayerNorm.

Newer variant: RMSNorm drops mean subtraction, normalizes using only root mean square of activations. Computationally cheaper, empirically performs as well. Llama 2 and many recent LLMs use RMSNorm — sign that mean centering in standard layer norm may not be pulling its weight.

ArchitectureNormalizationWhy
CNNsBatch NormFixed spatial dims, large batches, well-tested
Transformers / LLMsLayer Norm (Pre-LN) or RMSNormVariable lengths, batch-size independent
Small-batch / inferenceLayer Norm or Group NormBatch stats unreliable with few samples

Residual Connections: The Gradient Highway

Proper initialization and normalization keep signals healthy in theory, but deeper problem exists. In 2015, Kaiming He's team observed paradox: 56-layer plain CNN performed worse than 20-layer one — not on test set (that would be overfitting), but on training set. More layers made optimization harder. Called this degradation problem.

Think about it: 56-layer network should be at least as good as 20-layer. Worst case, extra 36 layers learn identity (pass input unchanged), get same performance. But optimizer couldn't find those identity mappings. Learning "do nothing" turns out surprisingly difficult for neural network.

Fix was elegant. Instead of learning desired output directly — y = F(x) — restructure as y = F(x) + x. Block only needs to learn residual: difference between desired output and input. If ideal behavior is identity, F(x) needs to be zero — pushing weights toward zero is something optimizers are good at.

The Gradient Perspective

Gradient story is where residual connections shine. Without skip connection: gradient through block is ∂F(x)/∂x — depends entirely on F. If F poorly behaved, gradient vanishes or explodes. With skip connection: gradient becomes ∂F(x)/∂x + 1. That "+ 1" is gradient through identity path (∂x/∂x = 1), always there regardless of what F does.

# Without residual: gradient depends entirely on F
# ∂y/∂x = ∂F(x)/∂x                    → can vanish

# With residual: y = F(x) + x
# ∂y/∂x = ∂F(x)/∂x + 1               → always ≥ 1
# The +1 creates a "gradient highway" — information flows back
# even if the learned block has vanishing gradients

Why ResNets train 152 layers (and 1000+ with proper normalization) where plain networks collapse at ~20. Skip connections create highway for gradients to flow backward without being multiplied through dozens of nonlinear transformations. Every modern deep architecture — ResNets, transformers, U-Nets — uses this.

Back to digit classifier: with residual connections, even if sigmoid activations in middle layers have vanishing derivatives, gradient bypasses them through skip paths. Early layers receive learning signal. Network trains.

Gradient Clipping: The Safety Net

Initialization, normalization, and residual connections are preventive. Gradient clipping is reactive — catches explosions that slip through. Think of it as circuit breaker: rather fix wiring, but want breaker in case something goes wrong.

Norm-Based Clipping

Most common approach: compute total L2 norm of all gradients. If exceeds threshold (max_norm), scale all gradients down proportionally so norm equals max_norm. Gradient direction preserved — only magnitude capped.

# After loss.backward(), before optimizer.step()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()

max_norm=1.0 is standard starting point for RNNs and transformers. Deep CNNs: 0.5–5.0 common. If clipping activates every step → threshold too aggressive, artificially limiting learning. If never activates → not doing anything (might be fine — safety net, not tuning knob).

Value-Based Clipping

Alternative: clamp each individual gradient element to range, say [-1, 1]. Simpler but cruder — distorts gradient direction because different elements get clipped differently. Norm-based clipping preferred in nearly every modern setting.

Adaptive Gradient Clipping (AGC)

More recent idea from NF-Net paper (2021): instead of fixed global threshold, compare each layer's gradient norm to parameter norm. Intuition: layer with large weights tolerates larger gradient updates than one with small weights. AGC clips only when ratio exceeds factor λ (typically 0.01–0.05). Critical for training very deep normalizer-free networks without batch norm.

Dead Neurons: The ReLU Tradeoff

ReLU solved vanishing gradients for active neurons — but introduced new failure. Neuron's input becomes permanently negative (large weight update or bad init) → ReLU outputs zero, gradient zero, neuron never recovers. Dead. In large networks, can lose 10–20% of neurons without noticing — remaining compensate. In smaller networks or with aggressive learning rates, dead neurons cripple capacity.

Detection straightforward: after forward pass, check fraction of activations exactly zero. Consistently above 50% in a layer (more than expected ~50% from ReLU's negative half) → dying neurons.

Leaky ReLU fixes this: outputs small slope (typically 0.01x) for negative inputs instead of zero. Neuron still has gradient when inactive, can recover. GELU — used in BERT and most modern transformers — takes smoother approach, gradually transitioning between linear and near-zero. Neither universally better than ReLU, but both eliminate dead neuron problem entirely.

Putting It All Together: The Stability Diagnostic

Difference between someone who debugs broken training and someone who randomly tries hyperparameters: knowing what to measure. Function I keep in every project:

def gradient_health_check(model):
    """Print per-layer gradient norms after a backward pass."""
    norms = {}
    for name, param in model.named_parameters():
        if param.grad is not None:
            norm = param.grad.norm().item()
            flag = ""
            if norm < 1e-7:
                flag = " ← vanishing!"
            elif norm > 1e3:
                flag = " ← exploding!"
            norms[name] = norm
            print(f"{name:<40s} {norm:>12.2e}{flag}")

    if norms:
        ratio = max(norms.values()) / (min(norms.values()) + 1e-12)
        print(f"\nMax/min gradient ratio: {ratio:.1f}x")
        if ratio > 100:
            print("⚠ Gradient flow problem: >100x difference across layers")

Ratio between largest and smallest gradient norms tells whole story. Healthy network: ratio stays within 1–2 orders of magnitude. Exceeds 100× → gradient flow problem. Fix depends on direction: vanishing → check activations, add residual connections; exploding → check initialization, add gradient clipping.

⚠️ The Connection Triangle

Activation functions, weight initialization, and gradient behavior = three vertices of single triangle. Deeply coupled:

Sigmoid + large random init = saturated neurons = vanishing gradients. Guaranteed.
ReLU + He init = stable variance = stable gradients. Reliable.
ReLU + large random init = exploding activations = NaN within a few batches.
Any activation + BatchNorm + residual connections = dramatically reduced sensitivity to everything above.

When training fails, check all three vertices. Answer almost always points directly to fix.

Learning Rate Warmup

One more piece, initially dismissed but now essential for large models: learning rate warmup. At training start, all weights random, gradients noisy and large. Full learning rate immediately = flooring accelerator on ice — might be fine, might spin out. Warmup ramps learning rate linearly from near-zero to target over first few hundred/thousand steps. Gives optimizer time to find reasonable loss landscape region before aggressive steps. Nearly every large transformer uses warmup. Cheap insurance against early-training instability.

The Modern Recipe

He initialization + ReLU/GELU + normalization layers + residual connections + gradient clipping + learning rate warmup. That's the stack. Each piece addresses specific failure: initialization prevents signal collapse at step 0, normalization prevents drift during training, residual connections guarantee gradient flow at any depth, clipping catches remaining edge cases, warmup smooths critical early phase.

None of these were part of original neural network recipe. Each invented because someone's training broke, they figured out why, engineered a fix. Stack we have now = product of 30+ years of watching networks fail and asking "why."

Next time you see NaN in loss, or plateau that doesn't make sense — don't reach for random hyperparameter changes. Check gradient norms, look at activation–initialization–normalization triangle. You'll know exactly where to intervene.
