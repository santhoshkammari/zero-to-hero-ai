Weight Initialization

Chapter 7: Deep Learning Foundations 6 subtopics
TL;DR

How you set a neural network's weights before training begins determines whether the network learns at all. Zero initialization kills learning by making every neuron identical. Random initialization works but the scale matters enormously — too small and signals vanish, too large and they explode. Xavier initialization preserves variance for sigmoid/tanh networks, He initialization does the same for ReLU networks, and modern tricks like BatchNorm reduce (but never eliminate) the importance of getting initialization right.

The Zero Initialization Problem

Symmetry Breaking

Let's start with the most intuitive mistake. You have a neural network. Every weight needs a starting value before training begins. The simplest idea: set them all to zero. Every weight is 0. Clean. Symmetric. And completely broken.

Here's why. Consider a hidden layer with three neurons, all connected to the same inputs. If every weight is zero, every neuron computes the exact same weighted sum — zero. Every neuron produces the exact same output. During backpropagation, every neuron receives the exact same gradient. Every weight gets the exact same update. After one step of training, all three neurons are still identical. After a thousand steps, still identical. You've paid for three neurons but you're getting the work of one.

This is the symmetry problem. When neurons in the same layer start identical, they stay identical forever. The network can never learn to have one neuron detect edges, another detect textures, and a third detect colors. They're all locked in step, computing the same function for eternity.

Identical Gradients

Let's make this concrete with a tiny network. Two inputs, one hidden layer with two neurons, one output. All weights initialized to zero.

# Forward pass with zero initialization
import torch
import torch.nn as nn

model = nn.Sequential(
    nn.Linear(2, 2, bias=False),
    nn.ReLU(),
    nn.Linear(2, 1, bias=False)
)

# Set all weights to zero
for p in model.parameters():
    nn.init.zeros_(p)

x = torch.tensor([[1.0, 2.0]])
y = model(x)
print(y)  # tensor([[0.]])  — dead output

# Even after backprop, both hidden neurons get identical gradients
y.backward()
print(model[0].weight.grad)
# tensor([[0., 0.],
#         [0., 0.]])  — no signal at all

With zero weights, the output is zero. The loss gradient with respect to a zero weight depends on the activation from the previous layer — which is also zero. So the gradient is zero. Nothing updates. The network is brain-dead from birth. Even if you use biases (which break the strict zero-gradient issue), the symmetry problem remains: every neuron in a layer still computes the same function and receives the same gradient signal. You need different starting points so that neurons can specialize.

Random Initialization

Small Random Values

The fix is simple in concept: initialize weights to small random values. Each neuron starts slightly different from its neighbors, so each computes a slightly different function, receives a slightly different gradient, and evolves in a different direction. Symmetry is broken.

# Naive random initialization
layer = nn.Linear(256, 256, bias=False)
layer.weight.data = torch.randn(256, 256) * 0.01  # small random values

But "small random values" raises an immediate question: how small? This turns out to be the central problem of initialization, and getting it wrong is just as deadly as using zeros.

If the values are too small (say, multiply by 0.001), then after several layers, the activations shrink exponentially toward zero. By layer 10, your signal is effectively gone. Gradients during backpropagation also vanish — this is the vanishing gradient problem triggered by bad initialization. If the values are too large (say, multiply by 1.0), activations explode exponentially. By layer 10, you're dealing with astronomical numbers, and your gradients blow up. The network diverges and loss becomes NaN.

The sweet spot: you want the variance of activations to stay roughly the same from layer to layer. Not shrinking, not growing. Stable signal propagation through the entire depth of the network.

Uniform vs Normal Distribution

You can draw random weights from either a uniform distribution (every value in a range is equally likely) or a normal distribution (bell curve centered at zero). Both work. The key is matching the variance of the distribution to the layer size, not the shape of the distribution itself. In practice, normal initialization is more common because it naturally concentrates values near zero while allowing occasional larger values, which provides a good mix of weak and strong initial connections.

Xavier/Glorot Initialization

Variance Preservation

In 2010, Xavier Glorot and Yoshua Bengio asked a precise question: what variance should the weights have so that the variance of activations stays constant across layers? Their answer became Xavier initialization (also called Glorot initialization), and the derivation is surprisingly intuitive.

Consider a single neuron that computes y = w₁x₁ + w₂x₂ + … + wₙxₙ, where n is the number of inputs (the fan-in). Assume the weights and inputs are independent, both have zero mean, and the weights all share the same variance Var(w). Then by the rules of variance for sums of independent random variables:

Var(y) = n · Var(w) · Var(x)

We want Var(y) = Var(x) — the output variance equals the input variance. That means we need:

n · Var(w) = 1
Var(w) = 1/n = 1/fan_in

That's the core insight. If a layer has 256 input connections, each weight should have variance 1/256. The more inputs, the smaller each weight needs to be, because all those small contributions add up.

Fan-in and Fan-out

There's a subtlety. The derivation above preserves variance in the forward pass. But gradients flow backward through the same weights. If you run the same analysis for the backward pass, you get a different requirement:

Var(w) = 1/fan_out

where fan-out is the number of neurons in the next layer. You can't satisfy both simultaneously (unless fan-in equals fan-out), so Glorot and Bengio proposed a compromise — average the two:

Var(w) = 2 / (fan_in + fan_out)

For a normal distribution, this means sampling from N(0, 2/(fan_in + fan_out)). For a uniform distribution, you compute the range from the variance: U(-limit, limit) where limit = √(6 / (fan_in + fan_out)).

# Xavier initialization in PyTorch
layer = nn.Linear(512, 256)

# Normal variant
nn.init.xavier_normal_(layer.weight)
# Samples from N(0, 2/(512+256)) = N(0, 0.0026)

# Uniform variant
nn.init.xavier_uniform_(layer.weight)
# Samples from U(-sqrt(6/768), sqrt(6/768)) = U(-0.088, 0.088)

For Sigmoid/Tanh

Xavier initialization was derived under the assumption that activations are linear — or more precisely, that the activation function operates in its linear regime near zero. This assumption holds well for sigmoid and tanh activations, which are approximately linear around zero (tanh(x) ≈ x for small x). When inputs are small and centered, these activations don't squash the signal, and Xavier's variance math holds.

But this assumption breaks catastrophically for ReLU. The ReLU function kills half its inputs (everything negative becomes zero), which means the effective variance is halved at each layer. Xavier doesn't account for that, and a Xavier-initialized ReLU network will see activations slowly die out in deep networks.

He/Kaiming Initialization

ReLU-Aware Scaling

In 2015, Kaiming He and colleagues fixed this gap. Their key observation: ReLU sets all negative activations to zero, so on average, half the neurons in each layer produce zero output. This halves the variance at every layer. To compensate, you need to double the weight variance compared to Xavier.

Variance = 2/fan_in

The derivation mirrors Xavier's but accounts for ReLU. Starting from the same setup, y = w₁x₁ + … + wₙxₙ, we pass y through ReLU. Since ReLU zeros out negative values and the distribution is symmetric around zero, exactly half the variance survives:

Var(ReLU(y)) = ½ · Var(y) = ½ · n · Var(w) · Var(x)

Setting Var(ReLU(y)) = Var(x) to preserve signal strength:

½ · n · Var(w) = 1
Var(w) = 2/n = 2/fan_in

That's He initialization (also called Kaiming initialization). The only difference from Xavier is the factor of 2 in the numerator, compensating for ReLU's habit of killing half the signal.

For ReLU Networks

# He initialization in PyTorch
layer = nn.Linear(512, 256)

# Normal variant (most common)
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='relu')
# Samples from N(0, 2/512) = N(0, 0.0039)

# Uniform variant
nn.init.kaiming_uniform_(layer.weight, mode='fan_in', nonlinearity='relu')

# For Leaky ReLU (accounts for the non-zero negative slope)
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='leaky_relu')

A practical rule of thumb: if you're using ReLU or its variants (Leaky ReLU, ELU, GELU), use He. If you're using sigmoid or tanh, use Xavier. If you're not sure, use He — it's more robust across activation functions.

Let's see the difference empirically. We'll push a signal through a 20-layer network with each initialization strategy and watch what happens to the activation magnitudes:

import torch
import torch.nn as nn
import torch.nn.init as init

def measure_activations(init_fn, depth=20, width=512):
    """Track activation variance through a deep network."""
    x = torch.randn(64, width)
    variances = []

    for i in range(depth):
        layer = nn.Linear(width, width, bias=False)
        init_fn(layer.weight)
        x = torch.relu(layer(x))
        variances.append(x.var().item())

    return variances

# Compare initialization strategies
strategies = {
    'zeros':       lambda w: init.zeros_(w),
    'small_rand':  lambda w: init.normal_(w, 0, 0.01),
    'large_rand':  lambda w: init.normal_(w, 0, 1.0),
    'xavier':      lambda w: init.xavier_normal_(w),
    'he':          lambda w: init.kaiming_normal_(w, nonlinearity='relu'),
}

for name, fn in strategies.items():
    torch.manual_seed(42)
    vars = measure_activations(fn)
    print(f"{name:12s} | Layer 1: {vars[0]:.6f} | Layer 10: {vars[9]:.6f} | Layer 20: {vars[19]:.6f}")

# Typical output:
# zeros        | Layer 1: 0.000000 | Layer 10: 0.000000 | Layer 20: 0.000000
# small_rand   | Layer 1: 0.000003 | Layer 10: 0.000000 | Layer 20: 0.000000
# large_rand   | Layer 1: 261.8945 | Layer 10:      NaN | Layer 20:      NaN
# xavier       | Layer 1: 0.384521 | Layer 10: 0.005127 | Layer 20: 0.000019
# he           | Layer 1: 0.978634 | Layer 10: 0.912847 | Layer 20: 0.854213

The results tell the whole story. Zeros: dead. Small random: vanishes by layer 10. Large random: explodes before layer 10. Xavier: slowly decays (because it doesn't account for ReLU). He: stable through all 20 layers. This is why initialization strategy matters — it's the difference between a network that trains and one that doesn't.

Other Strategies

Orthogonal Initialization

Orthogonal initialization constructs a weight matrix whose rows (or columns) are mutually orthogonal — perpendicular to each other in high-dimensional space. Mathematically, you generate a random matrix, compute its QR decomposition or SVD, and use the orthogonal factor. The result: the weight matrix preserves the norm of any vector it multiplies, preventing both explosion and vanishing in a single step. This is especially useful in recurrent neural networks (RNNs), where the same weight matrix is applied repeatedly across time steps. Without orthogonal initialization, RNNs are notoriously prone to vanishing or exploding gradients over long sequences.

# Orthogonal initialization
layer = nn.Linear(256, 256)
nn.init.orthogonal_(layer.weight)
# Weight matrix W satisfies W^T · W ≈ I (identity)

LSUV

Layer-Sequential Unit-Variance (LSUV) initialization takes a pragmatic approach. Instead of deriving the correct variance mathematically, it measures it empirically. The algorithm works layer by layer: initialize weights with orthogonal initialization, push a mini-batch through the network, measure the actual output variance of each layer, and rescale the weights until the variance is 1.0. It's brute-force but effective, and it works regardless of the activation function or architecture. The tradeoff: it requires a forward pass over real data before training begins, adding a small startup cost.

Fixup Initialization

Fixup initialization was designed specifically for residual networks (ResNets) that don't use BatchNorm. In a ResNet, the output of each block is added to a skip connection: y = F(x) + x. If the residual branch F(x) has too-large variance, the additions compound and activations explode as the network gets deeper. Fixup addresses this by scaling down the initialization of the last layer in each residual block by a factor of L^(-1/(2m-2)), where L is the number of residual blocks and m is the number of layers per block. It also adds learnable scalar biases and multipliers that give the network extra flexibility to adjust signal magnitude during training.

Initialization & Normalization Interaction

BN Reducing Initialization Sensitivity

Batch Normalization (BN) normalizes each layer's activations to zero mean and unit variance during training. At first glance, this seems like it should make initialization irrelevant — if BN is going to normalize everything anyway, who cares what the weights start at?

There's truth to this. BN dramatically reduces sensitivity to initialization. A network with BN will often train successfully with Xavier, He, or even naive random initialization, where the same network without BN would fail with anything but the right strategy. BN acts as a safety net, correcting the signal magnitude at every layer.

# BatchNorm makes initialization less critical
def build_model(use_bn=False, init_fn=None):
    layers = []
    for _ in range(20):
        linear = nn.Linear(256, 256)
        if init_fn:
            init_fn(linear.weight)
        layers.append(linear)
        if use_bn:
            layers.append(nn.BatchNorm1d(256))
        layers.append(nn.ReLU())
    layers.append(nn.Linear(256, 10))
    return nn.Sequential(*layers)

# Without BN: only He init converges
# With BN: Xavier, He, and even small random all converge
# But He + BN still converges fastest

When Initialization Still Matters

But BN doesn't make initialization completely irrelevant. Three situations where init still matters, even with BN in place.

First, the first few training steps. BN normalizes based on batch statistics, but the very first forward pass uses whatever activations the initial weights produce. Bad initialization means the first few gradient updates are noisy and unstable. Good initialization means training starts moving in a useful direction immediately. This can matter for convergence speed — networks with He initialization reach the same final accuracy faster than networks with bad-but-BN-rescued initialization.

Second, small batch sizes. BN estimates mean and variance from each mini-batch. With small batches (8 or 16 samples), these estimates are noisy. The smaller the batch, the less BN can compensate for bad initialization, because the normalization statistics themselves are unreliable.

Third, architectures without BN. Transformers typically use Layer Normalization instead of Batch Normalization, and the dynamics are different. Vision Transformers, GPT-style models, and many modern architectures are quite sensitive to initialization. The recent trend toward normalization-free networks (like NFNets) has also renewed interest in careful initialization, since there's no normalization layer to catch your mistakes.

The honest takeaway: use He initialization for ReLU networks, Xavier for sigmoid/tanh, and let BatchNorm handle the rest if you have it. But understand the math behind both strategies, because when you're debugging a network that won't train, initialization is one of the first things to check — and knowing why He works tells you when it won't.
