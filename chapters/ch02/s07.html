Home/Ch 2/Information Theory
TL;DR

Information theory gives ML its native language. Entropy measures uncertainty; cross-entropy measures how badly your model approximates reality (and is exactly the classification loss you already use). KL divergence quantifies the gap between two distributions, powering VAEs, knowledge distillation, and policy optimization. Mutual information tells you which features actually matter. The data processing inequality explains why deep networks must learn efficient representations. If you only remember one thing: minimizing cross-entropy loss is minimizing KL divergence from the true data distribution.

Entropy

Before we talk about formulas, let's talk about surprise. Imagine you flip a fair coin. Heads or tails — you're roughly equally surprised either way. Now imagine a coin that lands heads 99% of the time. When it lands heads, you shrug. When it lands tails, you're genuinely shocked. Entropy captures this idea: it's the average surprise you experience when observing outcomes from a random process.

Claude Shannon formalized this in 1948. For a discrete random variable X with possible outcomes x₁, x₂, …, xₙ and probabilities p(xᵢ), the entropy is:

H(X) = -Σ p(xᵢ) log₂ p(xᵢ)

Why log base 2? Because Shannon was thinking about communication channels that transmit binary digits. One bit is the information content of a single fair coin flip. A fair coin has entropy H = 1 bit. That biased coin (99% heads) has H ≈ 0.08 bits — almost no surprise on average, because you can almost always predict the outcome. A fair six-sided die has H ≈ 2.58 bits. More possible outcomes, more surprise, more bits needed to encode the result.

The key intuition: entropy is maximized when all outcomes are equally likely (maximum uncertainty), and minimized when one outcome is certain (zero uncertainty). This shows up everywhere in ML.

Differential Entropy

When your random variable is continuous rather than discrete, you replace the sum with an integral to get differential entropy: h(X) = -∫ p(x) log p(x) dx. A Gaussian with variance σ² has differential entropy ½ log(2πeσ²). Unlike discrete entropy, differential entropy can be negative — a very peaked distribution concentrates so much probability in a tiny region that the "average surprise" dips below zero. This quirk rarely causes practical problems, but it's worth knowing so you don't panic when you see a negative value.

Conditional and Joint Entropy

Conditional entropy H(Y|X) measures how much uncertainty remains about Y after you observe X. If X completely determines Y, then H(Y|X) = 0 — no leftover surprise. If X tells you nothing about Y, then H(Y|X) = H(Y). This is exactly the concept behind decision tree splitting: you pick the feature X that minimizes H(Y|X), because that's the feature that reduces your uncertainty about the label Y the most.

Joint entropy H(X, Y) measures the total uncertainty in both variables together. The chain rule ties everything together elegantly: H(X, Y) = H(X) + H(Y|X). The total uncertainty is the uncertainty in X plus whatever uncertainty remains in Y after you know X. This decomposition is not just mathematically pretty — it's the foundation of how we think about information flow in pipelines.

Key Insight

In decision trees, the "information gain" used for splitting is literally H(Y) - H(Y|X) — the reduction in entropy of the target variable Y when you split on feature X. ID3, C4.5, and similar algorithms are doing information theory at every node.

Entropy also appears as a regularizer. In reinforcement learning, adding an entropy bonus to the policy's objective encourages exploration — a policy with higher entropy tries a wider range of actions rather than collapsing to a single deterministic choice too early. This is the core idea behind Soft Actor-Critic (SAC) and entropy-regularized policy gradient methods.

Cross-Entropy and KL Divergence

Here's where information theory becomes the backbone of modern ML training. Suppose you have a true distribution P (the real data) and a model distribution Q (your neural network's predictions). Cross-entropy measures how many bits you'd need, on average, to encode samples from P using a code optimized for Q:

H(P, Q) = -Σ p(x) log q(x)

If Q perfectly matches P, cross-entropy equals the entropy of P — you're using the optimal code. If Q is a bad approximation, cross-entropy is higher — you're wasting bits. This is exactly why cross-entropy is the standard classification loss function. When you call nn.CrossEntropyLoss() in PyTorch, you're asking: "How many extra bits does my model waste compared to a perfect encoding of the labels?"

For binary classification with true label y ∈ {0, 1} and predicted probability q, the binary cross-entropy loss is -[y log(q) + (1-y) log(1-q)]. That formula you've seen a hundred times? Pure information theory.

KL Divergence

KL divergence (Kullback-Leibler divergence) measures the "extra bits" — the overhead of using Q instead of P:

D_KL(P ‖ Q) = Σ p(x) log [p(x) / q(x)]

The fundamental relationship connecting these three quantities is: H(P, Q) = H(P) + D_KL(P ‖ Q). Since H(P) is a constant (the true data distribution doesn't change during training), minimizing cross-entropy is identical to minimizing KL divergence. This is not a coincidence — it's why cross-entropy loss works.

KL divergence has two important properties. First, it's non-negative — you can never do better than the true distribution's own entropy. This is called Gibbs' inequality. Second, it's asymmetric: D_KL(P ‖ Q) ≠ D_KL(Q ‖ P). This asymmetry matters in practice. D_KL(P ‖ Q) — called the "forward KL" — penalizes Q heavily wherever P has mass but Q doesn't (Q must cover all of P). D_KL(Q ‖ P) — the "reverse KL" — penalizes Q for placing mass where P doesn't (Q avoids regions P considers unlikely). Variational inference uses the reverse KL, which tends to produce distributions that are too narrow rather than too broad.

Watch Out

KL divergence is not a true distance metric — it violates symmetry and the triangle inequality. Don't say "the KL distance between P and Q." Say "the KL divergence from P to Q" to make the direction explicit. The direction matters for your model's behavior.

ML Applications of KL Divergence

KL divergence is the workhorse behind several major ML techniques. In Variational Autoencoders (VAEs), the loss function (the ELBO) includes a KL term that pushes the encoder's approximate posterior toward a standard Gaussian prior. In knowledge distillation, you minimize KL divergence between the teacher's soft outputs and the student's predictions. In policy optimization (PPO, TRPO), a KL penalty or constraint prevents the new policy from straying too far from the old one, keeping training stable.

The Jensen-Shannon divergence (JSD) fixes the asymmetry problem: JSD(P ‖ Q) = ½ D_KL(P ‖ M) + ½ D_KL(Q ‖ M), where M = ½(P + Q). It's symmetric, bounded between 0 and 1 (with log base 2), and its square root is a proper metric. GANs originally used a loss function closely related to JSD — the discriminator's loss implicitly estimates the Jensen-Shannon divergence between real and generated data.

import numpy as np

# True distribution (e.g., one-hot label smoothed slightly)
P = np.array([0.9, 0.05, 0.05])

# Model's predicted distribution
Q = np.array([0.7, 0.2, 0.1])

# Cross-entropy H(P, Q)
cross_entropy = -np.sum(P * np.log(Q))
print(f"Cross-entropy H(P,Q): {cross_entropy:.4f} nats")

# Entropy H(P)
entropy_P = -np.sum(P * np.log(P))
print(f"Entropy H(P): {entropy_P:.4f} nats")

# KL divergence D_KL(P || Q)
kl_div = np.sum(P * np.log(P / Q))
print(f"KL divergence D_KL(P||Q): {kl_div:.4f} nats")

# Verify the relationship: H(P,Q) = H(P) + D_KL(P||Q)
print(f"H(P) + D_KL: {entropy_P + kl_div:.4f} nats")

# Jensen-Shannon divergence
M = 0.5 * (P + Q)
jsd = 0.5 * np.sum(P * np.log(P / M)) + 0.5 * np.sum(Q * np.log(Q / M))
print(f"JSD(P||Q): {jsd:.4f} nats")

Mutual Information

Sometimes you don't want to know how well Q approximates P. You want to know how much two variables share. Mutual information I(X; Y) quantifies exactly this: how much does knowing X reduce your uncertainty about Y?

I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = H(X) + H(Y) - H(X, Y)

The symmetry is beautiful: X tells you as much about Y as Y tells you about X. If X and Y are independent, I(X; Y) = 0 — knowing one tells you nothing about the other. If X completely determines Y, I(X; Y) = H(Y) — you've eliminated all uncertainty.

Think of it like a Venn diagram. H(X) and H(Y) are two overlapping circles of uncertainty. The overlap is mutual information — the shared knowledge. Joint entropy H(X, Y) is the total area of both circles combined. Conditional entropy H(Y|X) is the part of Y's circle that doesn't overlap with X.

ML Uses of Mutual Information

For feature selection, mutual information beats correlation because it captures nonlinear relationships. A feature might have zero Pearson correlation with the target but high mutual information — think of Y = X² where X is symmetric around zero. Scikit-learn's mutual_info_classif and mutual_info_regression use this directly.

In InfoGAN, the generator is trained to maximize mutual information between a subset of latent codes and the generated output, encouraging those codes to capture interpretable, disentangled factors of variation. In contrastive learning, the InfoNCE loss (used in SimCLR, CPC, and CLIP) is a lower bound on mutual information between different views of the same data point. When you train CLIP to align image and text embeddings, you're maximizing mutual information between visual and linguistic representations.

The Information Bottleneck method formalizes a fundamental tradeoff: compress the input X into a representation T that retains as much information as possible about the target Y while discarding everything else. Formally, you minimize I(X; T) - β · I(T; Y), where β controls the tradeoff between compression and prediction. Some researchers argue this is what deep networks learn to do naturally — each layer progressively strips away irrelevant details while preserving the signal needed for the task.

import numpy as np
from sklearn.feature_selection import mutual_info_classif
from sklearn.datasets import make_classification

# Create a dataset with some informative and some useless features
X, y = make_classification(
    n_samples=1000, n_features=10,
    n_informative=3, n_redundant=2,
    n_useless=5, random_state=42
)

# Estimate mutual information between each feature and the target
mi_scores = mutual_info_classif(X, y, random_state=42)

print("Mutual Information scores per feature:")
for i, score in enumerate(mi_scores):
    label = "informative" if score > 0.1 else "weak/useless"
    print(f"  Feature {i}: MI = {score:.3f} ({label})")

# Select top-k features
k = 3
top_features = np.argsort(mi_scores)[::-1][:k]
print(f"\nTop {k} features by MI: {top_features}")

Data Processing Inequality

Here's one of the most profound results in information theory, and it takes just one line to state. If X → Y → Z form a Markov chain (Z depends on X only through Y), then:

I(X; Z) ≤ I(X; Y)

In words: processing data can only destroy information, never create it. Every transformation, every function, every layer in your neural network can only lose information about the original input. If your first convolutional layer throws away some detail from the raw pixels, no subsequent layer can recover it.

This sounds depressing, but it's actually liberating. A good neural network is not trying to preserve all information — it's trying to throw away the right information. The pixels of a cat photo contain information about the lighting, camera sensor noise, JPEG compression artifacts, and the cat's breed. A classifier should retain the breed information and discard the rest. The data processing inequality tells you that each layer must be a strategic bottleneck, keeping what matters and forgetting what doesn't.

This connects to rate-distortion theory, which quantifies the fundamental tradeoff between compression and fidelity. Given a distortion budget (how much error you'll tolerate), rate-distortion theory tells you the minimum number of bits needed to represent your data. Lossy compression codecs (JPEG, MP3) operate near the rate-distortion bound. Similarly, a neural network's hidden layers are performing lossy compression of the input, preserving just enough to keep prediction error low. The information bottleneck view of deep learning argues that training has two phases: first the network learns to fit the data (increasing mutual information with the target), then it compresses (decreasing mutual information with the input).

Key Insight

The data processing inequality explains why feature engineering matters. If you feed raw, noisy data into a pipeline, every downstream component is bounded by the information in that noisy input. Cleaning data and extracting good features up front raises the ceiling on what any model can achieve.

Fisher Information

Shannon entropy and its friends measure properties of distributions. Fisher information measures something different: how sensitive a distribution is to changes in its parameters. Formally, for a distribution p(x; θ) parameterized by θ, the Fisher information is:

I(θ) = E[(∂/∂θ log p(x; θ))²]

This is the expected squared gradient of the log-likelihood — or equivalently, the negative expected second derivative (the curvature) of the log-likelihood. High Fisher information means the log-likelihood surface has sharp curvature near θ: the data is very informative about the parameter value, and small changes in θ produce large changes in the likelihood.

The Cramér-Rao bound (which you may have seen in the statistics section) says that no unbiased estimator of θ can have variance lower than 1/I(θ). More Fisher information means tighter bounds on estimation accuracy. This is the theoretical limit on how well you can learn a parameter from data.

In deep learning, Fisher information shows up in natural gradient descent. Ordinary gradient descent treats parameter space as Euclidean — a step of size 0.01 in any direction is "the same." But parameter space isn't flat. A small change to a weight in one layer might barely affect the output, while the same-sized change in another layer could be catastrophic. The Fisher information matrix (the multivariate extension) captures this geometry. Natural gradient descent multiplies the gradient by the inverse of the Fisher information matrix, so the effective step size is measured in terms of how much the model's output distribution changes — not how much the raw parameters change. This is the idea behind K-FAC and similar second-order optimizers. Elastic Weight Consolidation (EWC), a technique for mitigating catastrophic forgetting in continual learning, uses the diagonal of the Fisher information matrix to identify which parameters are "important" for previously learned tasks and penalizes changes to them.

Connections to ML: The Bigger Picture

Let's step back and see how all these pieces fit together, because information theory isn't just a chapter in a math textbook — it's the theoretical language that ML speaks, often without practitioners realizing it.

We've already seen that minimizing cross-entropy loss is equivalent to minimizing KL divergence from the true data distribution. This isn't a loose analogy; it's a mathematical identity. Every time you train a classifier with cross-entropy loss, you're doing information-theoretic optimization.

Regularization as Compression

The Minimum Description Length (MDL) principle says: the best model is the one that compresses the data most efficiently. A model's total description cost is the bits needed to describe the model itself plus the bits needed to describe the data given the model. A complex model (many parameters) is expensive to describe but encodes the data cheaply. A simple model is cheap to describe but may encode the data poorly. MDL says to minimize the sum. This is exactly the bias-variance tradeoff, stated in information-theoretic terms. L1/L2 regularization, dropout, and pruning all reduce model complexity — they're all compressing the model description.

Variational Inference

Variational inference reframes Bayesian computation as optimization: find the distribution q(z) in some tractable family that minimizes D_KL(q(z) ‖ p(z|x)), the KL divergence from the approximate posterior to the true posterior. Since the true posterior is intractable, we maximize the Evidence Lower Bound (ELBO) instead, which is equivalent. This powers VAEs, and recent extensions (normalizing flows, diffusion models) can be understood as making q(z) more flexible to reduce this KL gap.

Information-Theoretic Generalization

Recent theoretical work uses mutual information to bound generalization error. The intuition: if a learning algorithm's output hypothesis has high mutual information with the specific training set it saw, it has likely memorized training-set-specific details and will generalize poorly. Formally, the expected generalization gap is bounded by the mutual information between the training data and the learned model parameters. This gives a clean, assumption-light explanation for why techniques that reduce memorization (data augmentation, dropout, noise injection) improve generalization.

import numpy as np
from scipy.stats import entropy as sp_entropy

def estimate_entropy_discrete(labels):
    """Estimate entropy from observed labels (plug-in estimator)."""
    _, counts = np.unique(labels, return_counts=True)
    probs = counts / counts.sum()
    return sp_entropy(probs, base=2)  # in bits

def information_gain(parent_labels, left_labels, right_labels):
    """Compute information gain for a binary split (decision tree style)."""
    n = len(parent_labels)
    n_left, n_right = len(left_labels), len(right_labels)

    h_parent = estimate_entropy_discrete(parent_labels)
    h_left = estimate_entropy_discrete(left_labels)
    h_right = estimate_entropy_discrete(right_labels)

    # Weighted average of child entropies
    h_children = (n_left / n) * h_left + (n_right / n) * h_right
    return h_parent - h_children

# Example: splitting animals by "has_wings" to predict "can_fly"
labels = np.array([1, 1, 1, 0, 0, 0, 0, 0])  # 3 flyers, 5 non-flyers
left = np.array([1, 1, 1, 0])     # has_wings group
right = np.array([0, 0, 0, 0])    # no_wings group

ig = information_gain(labels, left, right)
print(f"Parent entropy: {estimate_entropy_discrete(labels):.3f} bits")
print(f"Information gain from split: {ig:.3f} bits")
print(f"Remaining entropy: {estimate_entropy_discrete(labels) - ig:.3f} bits")
Key Insight

Information theory provides the "right" language for ML because learning is compression. A model that generalizes well has found a compact description of the patterns in data — keeping the signal, discarding the noise. Entropy, cross-entropy, KL divergence, and mutual information aren't just mathematical tools; they're the natural units of measurement for what machine learning systems actually do.
