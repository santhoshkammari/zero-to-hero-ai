TL;DR

Linear algebra runs machine learning. Every neural network forward pass is matrix multiplication. PCA, SVD, and eigendecomposition drive dimensionality reduction, recommender systems, and feature extraction. Understand how matrices transform space — stretch, rotate, project — and you understand what ML models do to data. We build from scalars to graph Laplacians with one running example: tiny movie recommender. Two analogies — telescope and shadow — carry story from vectors through SVD, with NumPy code you can run today.

Can ship ML code without understanding linear algebra — call np.linalg.svd, get numbers, move on. But gap between calling API and knowing what output means geometrically will catch up with you. Interview or debugging at 2 a.m.

Linear algebra is language for data. Every vector is data point. Every matrix is transformation. Every decomposition is better view of that transformation. It shows up everywhere: neural network forward passes are matrix multiplication, linear regression is geometric projection, recommender systems are matrix factorization, and attention in transformers is sequence of matrix products with softmax in middle. Understand what matrices do to space, algorithms make more sense.

We build from scalar to decompositions behind production ML. One running example threads through every section: tiny movie recommender with three users and three movies. End result: SVD predicts missing rating, path from scalars to singular values clicks.

Vectors: Where Everything Starts

The Data Hierarchy: Scalars, Vectors, Matrices, Tensors

A scalar is single number. One movie rating: 4 stars. One pixel intensity. One weight in neural network. Scalar.

A vector is ordered list of scalars. Suppose three movies — Alien, Titanic, and Toy Story — and user Alice rates them [5, 2, 4]. That list is Alice's rating vector. It lives in three-dimensional space, one axis per movie. Each user gets vector. Each movie gets vector. This is running example. We keep building on it.

A matrix is grid of numbers — stack vectors and get one. If Alice, Bob, and Carol each rate three movies, we get 3×3 matrix. Each row is user. Each column is movie. Whole dataset in one object. A tensor generalizes this to more dimensions. Color image is 3D tensor (height × width × 3 color channels). Batch of 32 images is 4D tensor. In code, all are NumPy arrays with different numbers of dimensions.

import numpy as np

# Our running example: 3 users × 3 movies
# Movies: Alien, Titanic, Toy Story
alice = np.array([5, 2, 4])   # loves sci-fi and animation
bob   = np.array([4, 1, 5])   # loves sci-fi and animation
carol = np.array([1, 5, 2])   # loves romance

# Stack into a matrix: each row is a user
R = np.array([alice, bob, carol])
print(R)
# [[5 2 4]
#  [4 1 5]
#  [1 5 2]]

Notice structure already. Alice and Bob have similar taste — both rate Alien high, Titanic low. Carol is opposite. No math yet, pattern is visible. Linear algebra makes pattern precise.

The Dot Product: Measuring Similarity

dot product multiplies matching elements and sums results. Most important single operation in ML. Reason: it measures how much two vectors agree. Same direction gives large positive value. Perpendicular gives zero. Opposite directions give large negative value.

Compute dot product of Alice and Bob, then Alice and Carol.

# Dot product: element-wise multiply and sum
alice_bob = np.dot(alice, bob)    # 5*4 + 2*1 + 4*5 = 42
alice_carol = np.dot(alice, carol) # 5*1 + 2*5 + 4*2 = 23

print(f"Alice · Bob:   {alice_bob}")    # 42 — high agreement
print(f"Alice · Carol: {alice_carol}")  # 23 — less agreement

Alice and Bob agree more than Alice and Carol. Dot product catches it. But problem: if Alice used 1-to-10 instead of 1-to-5, all dot products would double even though her preferences stay same. Vector magnitude contaminates similarity.

Enter cosine similarity. It divides dot product by product of vector magnitudes: cos(θ) = (a · b) / (||a|| × ||b||). This removes magnitude and leaves pure directional agreement, number between −1 and +1. Word embeddings like Word2Vec use cosine similarity to show that "king" and "queen" point in similar directions even when vector lengths differ.

# Cosine similarity removes magnitude effects
from numpy.linalg import norm

cos_ab = np.dot(alice, bob) / (norm(alice) * norm(bob))
cos_ac = np.dot(alice, carol) / (norm(alice) * norm(carol))

print(f"cos(Alice, Bob):   {cos_ab:.4f}")   # ~0.937 — very similar
print(f"cos(Alice, Carol): {cos_ac:.4f}")    # ~0.610 — less similar

Norms: Measuring Size and the Geometry of Regularization

A norm measures vector size. The L2 norm (Euclidean norm) is straight-line distance from origin: ||v||₂ = √(v₁² + v₂² + ... + vₙ²). The L1 norm (Manhattan distance) sums absolute values: ||v||₁ = |v₁| + |v₂| + ... + |vₙ|. The L∞ norm is largest absolute component.

Why norms matter? Regularization. Interview-relevant geometry. L2 regularization (Ridge) penalizes L2 norm of weight vector. Set of vectors with L2 norm ≤ 1 is smooth circle (2D) or sphere (higher dimensions). Optimizer slides smoothly, so weights shrink but rarely hit zero. L1 regularization (Lasso) penalizes L1 norm. L1 "ball" is diamond with sharp corners on axes. Hit corner and some weights become exactly zero. That gives sparsity — feature selection for free. L2 does not.

w = np.array([0.5, -1.2, 0.01, 3.0])

l1 = np.linalg.norm(w, ord=1)       # 4.71 — sum of absolute values
l2 = np.linalg.norm(w, ord=2)       # 3.27 — Euclidean length
linf = np.linalg.norm(w, ord=np.inf) # 3.0 — largest component

print(f"L1: {l1:.2f}, L2: {l2:.2f}, L∞: {linf:.2f}")

A unit vector has L2 norm 1. Make one by dividing vector by its norm — normalization. Batch normalization and layer normalization in deep learning are richer versions of same idea: keep magnitudes controlled so signals do not explode or vanish through network.

Orthogonality

Two vectors are orthogonal if dot product is zero — right angles, no contribution to each other. Orthogonal features are statistically uncorrelated, ideal for ML. PCA explicitly produces orthogonal components. Angle between vectors comes from dot product: θ = arccos((a · b) / (||a|| × ||b||)).

Limitation: Where Single Vectors Fall Short

We can now measure similarity between two users. But one vector cannot capture relationships among all users and movies at once. Alice's vector shows her preferences, not movie-to-movie relations or hidden taste groups. For that, move from vectors to full grid. We need matrices.

Matrices: The Language of Transformation

A Matrix Is a Grid — and a Function

A matrix is rectangular grid of numbers with m rows and n columns. Our ratings table is 3×3 matrix. In ML, dataset is usually m×n: m samples, n features. Deeper idea: matrix is also function. Multiply matrix by vector, get new vector. Matrix transforms input into output. Every matrix is machine that eats vectors and outputs vectors.

# Our ratings matrix R is both a table and a transformation
R = np.array([
    [5, 2, 4],  # Alice
    [4, 1, 5],  # Bob
    [1, 5, 2],  # Carol
])

# What does multiplying R by a vector mean?
# If v represents "movie qualities," then R @ v gives "user scores"
movie_quality = np.array([1, 0, 0])  # only Alien matters
user_scores = R @ movie_quality
print(user_scores)  # [5, 4, 1] — each user's Alien rating

Matrix Multiplication: THE Operation

Matrix multiplication is core operation in ML. It is not element-wise. To multiply m×n matrix A by n×p matrix B, take dot product of each row of A with each column of B, producing m×p result. Inner dimensions must match.

Neural network fully connected layer is one matrix multiplication and one vector addition: y = Wx + b. Here x is input (n×1), W is weight matrix (k×n), and b is bias (k×1). Stack several with nonlinearities and you get deep neural network. Transformer forward pass is carefully arranged sequence of matrix multiplications.

# A single neural network layer: y = Wx + b
W = np.random.randn(3, 4)                    # 4 inputs → 3 outputs
x = np.array([1.0, 0.5, -1.0, 2.0])         # input vector
b = np.array([0.1, -0.1, 0.0])              # bias

y = W @ x + b   # @ is matrix multiplication in Python 3.5+
print(f"Layer output: {y}")

# Batch: process 100 samples at once
X = np.random.randn(100, 4)
Y = X @ W.T + b   # (100, 3) — all samples transformed simultaneously
print(f"Batch output shape: {Y.shape}")

Transpose, Identity, and Special Matrices

transpose flips matrix across diagonal: rows become columns, columns become rows. m×n becomes n×m. In NumPy: A.T. Transposes show up everywhere — normal equation for regression is x̂ = (AᵀA)⁻¹Aᵀb.

identity matrix I has ones on diagonal and zeros elsewhere. Multiply by I, nothing changes — matrix version of 1. Symmetric matrices equal own transpose: A = Aᵀ. Covariance matrices are always symmetric, giving clean decomposition properties for PCA.

An orthogonal matrix Q has orthonormal columns. Its inverse equals transpose: Q⁻¹ = Qᵀ. Orthogonal matrices represent rotations and reflections — they preserve lengths and angles. They are building blocks of SVD.

Limitation: We Have the Grid, But What Does It Do?

We have ratings table as matrix and can multiply it. Deeper question remains: what does multiplication by this matrix do to space? Stretch? Rotate? Flatten? Geometry of matrix transformations separates API callers from people who know what APIs do.

What Matrices Actually Do to Space

Stretching, Rotating, Projecting

Think of matrix as machine that deforms fabric of space. Some stretch space — some directions move farther from origin. Some rotate. Some project — flatten 3D object onto 2D surface, like shadow of statue on wall. Remember shadow. We use it in regression and PCA.

Concrete example. Matrix [[2, 0], [0, 0.5]] stretches x-axis by 2 and squishes y-axis to half. Apply it to unit circle, get ellipse — wide, not tall. Rotation matrix like [[cos θ, -sin θ], [sin θ, cos θ]] spins every point by angle θ without changing distances. Projection matrix collapses dimension completely.

import numpy as np

# A stretching matrix
S = np.array([[2, 0], [0, 0.5]])

# Apply to a vector
v = np.array([1, 1])
print(S @ v)  # [2, 0.5] — stretched horizontally, squished vertically

# A rotation matrix (45 degrees)
theta = np.pi / 4
rot = np.array([[np.cos(theta), -np.sin(theta)],
                [np.sin(theta),  np.cos(theta)]])
print(rot @ v)  # [0, 1.414] — rotated 45° counterclockwise

# A projection matrix (onto x-axis)
P = np.array([[1, 0], [0, 0]])
print(P @ v)  # [1, 0] — shadow on x-axis

The Determinant: How Much Space Gets Scaled

determinant of square matrix is single number telling scaling factor of transformation. For 2×2 matrix [[a, b], [c, d]], it is ad - bc. Absolute value tells how much area (2D) or volume (3D) changes. Determinant 2 doubles area. Determinant −1 flips orientation, like mirror, while keeping area.

Determinant 0 matters most. It means matrix squishes space to lower dimension — 2D region collapses to line, line collapses to point. Once space is flattened, cannot unflatten it. So determinant 0 means matrix is not invertible: cannot undo squish.

A = np.array([[3, 1], [2, 4]])
print(f"det(A) = {np.linalg.det(A):.1f}")  # 10.0 — scales area by 10x

# Singular matrix: row 2 is 2× row 1
B = np.array([[1, 2], [2, 4]])
print(f"det(B) = {np.linalg.det(B):.1f}")  # 0.0 — squishes to a line

In probability, determinant of covariance matrix appears in normalization constant of multivariate Gaussian. Geometrically, it measures volume of uncertainty ellipsoid.

Never Invert a Matrix in Production Code

If you need to solve Ax = b, do NOT compute A⁻¹ and multiply. Use np.linalg.solve(A, b). It uses LU decomposition underneath. Faster. More numerically stable. Matrix inversion is O(n³) and amplifies floating-point error. Nearly singular matrices make np.linalg.inv produce garbage.

Positive Definite Matrices

A symmetric matrix is positive definite if xᵀAx > 0 for every nonzero vector x. It is positive semi-definite (PSD) if xᵀAx ≥ 0. Geometrically, positive definite matrix never flips or collapses any direction — pure stretch. Covariance matrices are always PSD. Kernel matrices (Gram matrices) must be PSD for kernel to be valid. If Hessian is positive definite at a point, point is local minimum — foundation of second-order optimization.

Systems of Equations and the Least Squares Trick

Ax = b: The Setup

Any system of linear equations can be written as Ax = b. A is coefficient matrix. x is unknown vector. b is right-hand side. In linear regression, rows of A are data points, b is observed outputs, and x is weight vector you want.

If A is square and invertible, there is one exact solution. If equations outnumber unknowns — usual ML case — system is overdetermined and no exact solution exists. We need best approximate solution.

Least Squares: The Shadow of Reality

least squares solution is x̂ that minimizes ||Ax - b||² — total squared error. Shadow analogy fits here. b is point in high-dimensional space. Column space of A is flat surface inside that space. Least squares solution Ax̂ is point on that surface closest to b. Geometrically, it is projection — shadow of b dropped onto column space.

Residual b - Ax̂ is perpendicular to column space. That gives normal equation: AᵀAx̂ = Aᵀb. “Normal” means perpendicular. Equation guarantees error is orthogonal to every feature — model extracted everything it can from data.

import numpy as np

# 3 users rate a "sci-fi affinity" feature and "romance affinity" feature
# We want to predict their overall satisfaction score
A = np.array([
    [5, 2],   # Alice: high sci-fi, low romance
    [4, 1],   # Bob: high sci-fi, very low romance
    [1, 5],   # Carol: low sci-fi, high romance
])
b = np.array([9, 8, 6])  # satisfaction scores

# Least squares via normal equation
x_hat = np.linalg.solve(A.T @ A, A.T @ b)
print(f"Weights: {x_hat.round(3)}")

# Predictions = shadow of b onto column space of A
predictions = A @ x_hat
residual = b - predictions
print(f"Predictions: {predictions.round(2)}")
print(f"Residual: {residual.round(2)}")

# Verify: residual is orthogonal to column space
print(f"A^T @ residual ≈ 0: {(A.T @ residual).round(10)}")

This is one of ML's cleanest results. Linear regression is not curve fitting. It is geometric projection. Predicted values are shadow of reality cast onto subspace of possible predictions. Every linear model fit does this under hood.

Why "Normal" Equation?

“Normal” means perpendicular. Normal equation makes residual vector perpendicular to column space of A. Same statement: error is orthogonal to all features. Equation exists because perpendicularity gives closest point.

Vector Spaces: The Stage Where Data Lives

Span, Basis, Dimension

A linear combination of vectors is any weighted sum: c₁v₁ + c₂v₂ + ... + cₙvₙ. span is every point reachable by linear combinations. If vectors span all of ℝⁿ, you can reach any point in n-dimensional space. Vectors are linearly independent if none can be written as combination of others — each adds new direction.

A basis is minimal set of linearly independent vectors that spans space. Number of vectors in any basis is dimension. ℝ³ has dimension 3. ML insight: data may live in 100-dimensional space but vary along only 5 directions. Effective dimension is 5. Other 95 dimensions are noise or redundancy. PCA finds that basis.

Column Space, Null Space, Rank

column space of matrix is span of its columns — all possible outputs Ax. Linear model predictions live there. null space is all vectors x such that Ax = 0 — inputs matrix kills completely. If null space has anything besides zero vector, matrix loses information.

rank is dimension of column space — number of linearly independent columns. Rank tells how much information matrix carries. Rank-deficient matrix has redundant columns. In ML, low-rank data matrix means features are correlated and compressible. Rank-nullity theorem ties it together: rank + nullity = number of columns. Information kept + information destroyed = total information.

Back to movie example. Our 3×3 ratings matrix has rank at most 3. If Alice and Bob's taste vectors are proportional, effective rank drops to 2 — two taste dimensions, not three. Recommender systems exploit this low-rank structure.

Limitation: We Know the Stage. Now We Need to Find the Best Directions on It.

We can talk about spaces and dimensions, but not yet which directions matter most. If data spreads across 100 dimensions but most action lives along 3, which 3? That is where eigenvalues and eigenvectors enter.

Eigenvalues and Eigenvectors: The Directions That Matter

The Key Idea

We know matrices transform space. They stretch, rotate, project. But which directions survive best? Which directions does matrix treat as special?

An eigenvector of matrix A is nonzero vector v that, after multiplication by A, only gets scaled — not rotated. Scaling factor is eigenvalue λ. Equation: Av = λv. Most vectors change direction under matrix multiplication. Eigenvectors do not. They are natural axes of transformation.

Think telescope. From most angles galaxy looks blurry. From few special angles, structure snaps into focus — spiral arms, core, halo. Eigenvectors are those angles. They show transformation from clearest view. Remember telescope. It returns in PCA and SVD.

Why Eigenvectors Matter for ML

For covariance matrix, eigenvectors point along directions of maximum variance. Eigenvalues tell how much variance sits along each direction. Largest eigenvalue marks direction where data spreads most. That is PCA in one line: find covariance eigenvectors, keep biggest ones, project data onto them.

import numpy as np

# Covariance-like symmetric matrix
A = np.array([[4, 2],
              [2, 3]])

# Use eigh for symmetric matrices — faster, more stable, sorted
eigenvalues, eigenvectors = np.linalg.eigh(A)
print(f"Eigenvalues: {eigenvalues.round(3)}")
print(f"Eigenvectors:\n{eigenvectors.round(3)}")

# Verify: Av = λv for each pair
for i in range(len(eigenvalues)):
    Av = A @ eigenvectors[:, i]
    lv = eigenvalues[i] * eigenvectors[:, i]
    print(f"Av = {Av.round(4)}, λv = {lv.round(4)}, match: {np.allclose(Av, lv)}")

Computation and the Spectral Theorem

To find eigenvalues analytically, solve det(A - λI) = 0, characteristic equation. For n×n matrix, this gives degree-n polynomial. Nobody solves these by hand past 3×3. Numerical algorithms like QR iteration do it. np.linalg.eig calls them.

spectral theorem says every real symmetric matrix decomposes as A = QΛQᵀ, where Q is orthogonal and Λ is diagonal. All eigenvalues are real. All eigenvectors are orthogonal. That is why PCA works cleanly — covariance matrices are symmetric, so eigenvectors form orthogonal coordinate system. Use np.linalg.eigh for symmetric matrices. Faster. More stable.

Matrix with n linearly independent eigenvectors can be diagonalized: A = PDP⁻¹. Then powers become easy: Aⁿ = PDⁿP⁻¹. Markov chain steady-state analysis uses this trick — raise transition matrix to large power by diagonalizing it.

Limitation

Eigendecomposition needs square matrices. Ratings matrix is often rectangular. We need decomposition that works for any shape. That is SVD. First: use eigendecomposition where it shines — PCA.

PCA: Finding the Best Angle

The Telescope, Calibrated

Remember telescope? PCA calibrates it. Data is scattered across many dimensions, and many axes are noise. PCA rotates coordinate system until axes align with directions of maximum variance — where data spreads. Then it drops axes that do not matter.

Walk through PCA step by step with movie recommender. Familiar data makes mechanics click.

Step-by-Step with the Movie Recommender

Step 1: Center the data. Subtract mean of each feature (movie) so data sits at origin. PCA covariance formula assumes centered data.

Step 2: Compute the covariance matrix. This shows how each movie pair co-varies across users. Large positive entry means one movie gets rated high when other gets rated high.

Step 3: Eigendecompose the covariance matrix. Eigenvectors are principal components — telescope angles. Eigenvalues tell how much variance each direction captures.

Step 4: Project the data onto the top eigenvectors. This is dimensionality reduction — keep informative directions, drop rest.

import numpy as np

# Our running example: 3 users × 3 movies
R = np.array([
    [5, 2, 4],  # Alice
    [4, 1, 5],  # Bob
    [1, 5, 2],  # Carol
], dtype=float)

# Step 1: Center (subtract column means)
means = R.mean(axis=0)
R_centered = R - means
print(f"Means: {means.round(2)}")
print(f"Centered:\n{R_centered.round(2)}")

# Step 2: Covariance matrix (3×3, one entry per movie pair)
cov = (R_centered.T @ R_centered) / (R.shape[0] - 1)
print(f"Covariance matrix:\n{cov.round(2)}")

# Step 3: Eigendecomposition
eigenvalues, eigenvectors = np.linalg.eigh(cov)
# eigh returns sorted ascending; we want descending
idx = np.argsort(eigenvalues)[::-1]
eigenvalues = eigenvalues[idx]
eigenvectors = eigenvectors[:, idx]

print(f"Eigenvalues: {eigenvalues.round(3)}")
print(f"Explained variance ratio: {(eigenvalues / eigenvalues.sum()).round(3)}")

# Step 4: Project onto top 2 components
k = 2
W = eigenvectors[:, :k]  # top-k eigenvectors
R_projected = R_centered @ W
print(f"Projected data (3 users in 2D):\n{R_projected.round(3)}")

First eigenvalue captures most variance — taste axis separating sci-fi lovers from romance lovers. Second captures what remains. Third is tiny. We compressed three dimensions into two with little loss. In production, same logic takes 768 dimensions to 50.

Scree Plot and Choosing k

scree plot graphs eigenvalues from largest to smallest. Look for elbow — point where values drop hard. Components before elbow carry signal; after elbow carry noise. explained variance ratio of each component is its eigenvalue divided by total eigenvalue sum. Common rule: keep enough components to explain 95% of variance.

PCA has two equivalent views. variance maximization: find direction that maximizes projected variance. That gives first eigenvector. reconstruction error: find k-dimensional subspace minimizing average squared distance between points and projections. Same answer. Good sign math is consistent.

PCA Variants

Kernel PCA uses kernel trick to capture nonlinear structure — PCA in implicit high-dimensional feature space without explicit coordinates. Incremental PCA processes batches for data too large for memory. Sparse PCA adds L1 regularization so each component loads on few features, improving interpretability. Probabilistic PCA treats PCA as latent variable model, enabling likelihood computation and handling missing data.

import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# Larger example: 500 samples, 10 features with injected correlations
np.random.seed(42)
X = np.random.randn(500, 10)
X[:, 1] = X[:, 0] * 0.8 + np.random.randn(500) * 0.3
X[:, 2] = X[:, 0] * 0.6 + np.random.randn(500) * 0.5

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

pca = PCA()
pca.fit(X_scaled)

print("Explained variance ratio:", pca.explained_variance_ratio_.round(3))
print("Cumulative:", np.cumsum(pca.explained_variance_ratio_).round(3))

# Keep 95% of variance
pca_95 = PCA(n_components=0.95, svd_solver='full')
X_reduced = pca_95.fit_transform(X_scaled)
print(f"Reduced: {X_scaled.shape[1]}D → {X_reduced.shape[1]}D")

# Reconstruction error
X_recon = pca_95.inverse_transform(X_reduced)
error = np.mean((X_scaled - X_recon) ** 2)
print(f"Reconstruction error: {error:.4f}")

PCA works elegantly: directions of maximum variance also minimize reconstruction error. Proof is not hard once set up. Still feels like small miracle.

PCA Under the Hood: SVD, Not Eigendecomposition

Scikit-learn's PCA does not compute covariance matrix then eigenvalues. It computes SVD of centered data matrix directly. More numerically stable — no squaring data matrix. Also handles features > samples. Right singular vectors are principal components. Connection: if X = UΣVᵀ, then XᵀX = VΣ²Vᵀ, so eigenvectors of XᵀX are right singular vectors of X.

SVD: The Swiss Army Knife

The Decomposition That Works on Anything

Eigendecomposition needs square matrices. Ratings matrix is often rectangular. Need more general tool. Singular Value Decomposition (SVD) factors any m×n matrix A into A = UΣVᵀ. Think three steps: rotate (Vᵀ), scale (Σ), rotate again (U). Every matrix transformation is rotation, then stretch along axes, then another rotation.

U is m×m orthogonal (left singular vectors). Σ is m×n diagonal with non-negative singular values, sorted largest to smallest. V is n×n orthogonal (right singular vectors). Singular values are square roots of eigenvalues of AᵀA. Right singular vectors are eigenvectors of AᵀA. SVD generalizes eigendecomposition to non-square matrices.

Back to telescope: eigenvectors found best angles for square transformations. SVD finds best angles for any transformation, even between spaces of different dimensions.

Low-Rank Approximation and the Netflix Prize

SVD earns “Swiss Army knife” name here. Keep top k singular values, zero rest, and you get matrix approximation. Eckart-Young theorem says this is best possible rank-k approximation — nothing else of rank k gets closer in Frobenius or spectral norm. If 1000×1000 matrix has 10 important singular values, you can store about 20,000 numbers instead of 1,000,000.

This powered Netflix Prize. In 2006, Netflix offered $1 million for 10% recommendation boost. Simon Funk described method: SVD on ratings matrix — factor user×movie matrix into user-factor and movie-factor matrices, then multiply back to predict missing ratings. Key idea: if Alice and Bob both love sci-fi, and Alice rated Interstellar but Bob did not, latent “sci-fi” factor helps predict Bob's rating.

SVD clicked for me when I saw it predict missing ratings. Here is running example.

import numpy as np

# Our 3×3 ratings matrix. But now Carol hasn't rated Toy Story.
# We'll put 0 as a placeholder and see if SVD can fill it in.
R = np.array([
    [5, 2, 4],  # Alice: Alien=5, Titanic=2, ToyStory=4
    [4, 1, 5],  # Bob:   Alien=4, Titanic=1, ToyStory=5
    [1, 5, 0],  # Carol: Alien=1, Titanic=5, ToyStory=?
], dtype=float)

# Full SVD
U, sigma, Vt = np.linalg.svd(R, full_matrices=False)
print(f"Singular values: {sigma.round(2)}")

# Low-rank approximation: keep top 2 components
k = 2
approx = U[:, :k] @ np.diag(sigma[:k]) @ Vt[:k, :]
print(f"\nRank-{k} approximation:\n{approx.round(1)}")
print(f"\nPredicted Carol's Toy Story rating: {approx[2, 2]:.1f}")
# Carol doesn't like sci-fi, so we'd expect a low-ish prediction

Rank-2 approximation fills missing entry with prediction. Carol's taste pattern differs from Alice and Bob — low sci-fi, high romance — so SVD predicts modest Toy Story rating. Same idea scales to millions of users and items.

Randomized SVD for Scale

Full SVD costs O(mn × min(m,n)), too expensive for matrices with millions of rows and columns. Randomized SVD uses random projections to approximate top-k singular vectors in O(mn × k) time. Idea: multiply matrix by random skinny matrix to get sketch, compute SVD of much smaller sketch, project back. Scikit-learn uses it in TruncatedSVD and PCA(svd_solver='randomized'). For datasets with more than ~1000 features, it is usually what you want.

from sklearn.decomposition import TruncatedSVD
import numpy as np

# Simulate a large sparse ratings matrix
np.random.seed(42)
large_R = np.random.rand(10000, 500)

# Randomized SVD: much faster than full SVD
svd = TruncatedSVD(n_components=20, algorithm='randomized', random_state=42)
R_reduced = svd.fit_transform(large_R)
print(f"Reduced: {large_R.shape} → {R_reduced.shape}")
print(f"Explained variance: {svd.explained_variance_ratio_.sum():.3f}")

Matrix Decompositions: The Right Tool

Cholesky: The Square Root of a Matrix

Cholesky decomposition factors positive definite matrix as A = LLᵀ, where L is lower triangular. It is about twice as fast as general LU and acts like matrix square root. Main use: sampling from multivariate Gaussians. Given mean μ and covariance Σ, compute Cholesky factor L, sample standard normals z, then compute x = μ + Lz. That is how np.random.multivariate_normal works internally. Gaussian processes rely on Cholesky for marginal likelihood computation.

import numpy as np

# Sampling from a multivariate Gaussian via Cholesky
mean = np.array([1.0, 2.0])
cov = np.array([[2.0, 0.8],
                [0.8, 1.0]])

L = np.linalg.cholesky(cov)
z = np.random.randn(2)
sample = mean + L @ z
print(f"Sample: {sample.round(3)}")

# Verify with many samples
samples = mean + (L @ np.random.randn(2, 10000)).T
print(f"Empirical mean: {samples.mean(axis=0).round(2)}")
print(f"Empirical cov:\n{np.cov(samples.T).round(2)}")

QR: The Stable Least Squares Solver

QR decomposition factors A into orthogonal Q and upper triangular R: A = QR. Standard method uses Householder reflections, which are numerically stable. Main ML use: least squares. Instead of normal equation AᵀAx = Aᵀb — which squares condition number — decompose A = QR and solve Rx = Qᵀb. R is triangular, so back-substitution is fast. np.linalg.lstsq uses this under hood.

import numpy as np

# Least squares via QR vs. normal equation
A = np.random.randn(100, 5)
true_w = np.array([1, -2, 3, 0.5, -1])
b = A @ true_w + np.random.randn(100) * 0.1

# QR approach (numerically stable)
Q, R = np.linalg.qr(A)
x_qr = np.linalg.solve(R, Q.T @ b)

# Normal equation (less stable)
x_ne = np.linalg.solve(A.T @ A, A.T @ b)

print(f"QR solution:     {x_qr.round(3)}")
print(f"Normal eq:       {x_ne.round(3)}")
print(f"True weights:    {true_w}")

LU: General Purpose Solving

LU decomposition factors matrix into lower triangular L and upper triangular U: A = LU (with pivoting: PA = LU). It is Gaussian elimination packaged for efficiency. np.linalg.solve uses it internally. It is workhorse for solving Ax = b when A is square and well-conditioned.

When to Use Which

Choose decomposition by matrix structure. Cholesky works only for positive definite matrices but is fastest. QR handles any matrix and preserves condition number — default for least squares. LU is general-purpose for square systems. SVD handles everything, including rank-deficient matrices, but is slowest.

For numerical stability: normal equation squares condition number — dangerous. QR preserves it. SVD handles even rank-deficient matrices well. On well-conditioned problems, all methods agree. On ill-conditioned problems, only SVD stays reliable.

Choosing a Least Squares Solver

Small, well-conditioned: normal equation. General use: QR. Ill-conditioned or rank-deficient: SVD via np.linalg.lstsq. Scikit-learn's LinearRegression uses SVD by default — sensible, because you rarely know condition number ahead of time.

Condition Number: Why Your Model Produces NaN

Definition and Intuition

condition number measures how sensitive solution of Ax = b is to small perturbations. It is ratio of largest to smallest singular value: κ(A) = σ_max / σ_min. Near 1 means well-behaved. 10⁸ means solution may lose 8 digits of accuracy.

Once I spent two days chasing NaN in production model. Features looked right. Labels looked right. Architecture looked right. Learning rate looked fine. Problem: one feature ranged 0 to 1, another 0 to 1,000,000. Condition number was about 10¹². Hyperparameter tuning could not fix that. Five minutes of feature scaling did.

Feature Scaling Is Not Optional

When features live on wildly different scales, matrices become ill-conditioned. Feature scaling (standardization, normalization) is not optional best practice — it is numerical necessity. Batch normalization in deep learning serves same purpose: keep activations well-conditioned so gradients do not explode or vanish.

import numpy as np

# Ill-conditioned: one feature is huge, another tiny
X_bad = np.column_stack([
    np.random.randn(100) * 1e6,   # income in dollars
    np.random.randn(100) * 0.001   # some tiny feature
])
print(f"Condition number (bad): {np.linalg.cond(X_bad):.0e}")

# After standardization
X_good = (X_bad - X_bad.mean(axis=0)) / X_bad.std(axis=0)
print(f"Condition number (good): {np.linalg.cond(X_good):.1f}")
NaN Debugging Checklist

When model outputs NaN: (1) Check feature scales — standardize everything. (2) Compute np.linalg.cond(X) — above 10⁶ means trouble. (3) Lower learning rate — large steps amplify numerical error. (4) Add small epsilon (1e-8) to every denominator in custom loss functions. (5) Check for log(0) or division by zero. (6) If using custom matrix ops, replace np.linalg.inv with np.linalg.solve.

Covariance, Correlation, and Mahalanobis

The Covariance Matrix

covariance matrix Σ captures pairwise linear relationships between features. Entry (i,j) is covariance between features i and j. Diagonal entries are variances. It is always symmetric and positive semi-definite. For centered data X, covariance matrix is (1/(n-1))XᵀX.

In movie example, covariance matrix shows how movie pairs co-vary across users. Large positive entry between Alien and Toy Story means users who rate Alien high also rate Toy Story high. Negative entry between Alien and Titanic means they move opposite ways. Eigenvectors of this covariance matrix are principal components — directions of maximum variance in taste space.

Correlation vs. Covariance

correlation matrix is covariance matrix of standardized features. Each entry lies between −1 and +1, removing scale effects. PCA on correlation matrix equals PCA on standardized data — usually what you want, otherwise feature with biggest units dominates.

Whitening

whitening transform decorrelates features and normalizes variances so covariance matrix becomes identity. Compute it with eigendecomposition: X_white = Λ^(-1/2) Qᵀ X, where Q and Λ come from covariance eigendecomposition. Whitening is preprocessing for ICA (Independent Component Analysis) and conceptually related to batch normalization.

Mahalanobis Distance: The Right Ruler

Euclidean distance treats all dimensions equally. Wrong when features are correlated or have different variances. Mahalanobis distance from point x to distribution with mean μ and covariance Σ is √((x - μ)ᵀ Σ⁻¹ (x - μ)). It is Euclidean distance after whitening — stretch axes until data looks spherical, then measure distance in corrected space.

This matters for anomaly detection. Point may be close in Euclidean distance but far in Mahalanobis distance once correlations are accounted for. Gaussian classifiers, outlier detection, and Gaussian mixture models rely on Mahalanobis distance.

import numpy as np

# Two-feature data with strong correlation
np.random.seed(42)
mean = np.array([0, 0])
cov = np.array([[1.0, 0.9],
                [0.9, 1.0]])
X = np.random.multivariate_normal(mean, cov, 200)

# A test point: close in Euclidean but far along the thin axis
point = np.array([2, -1])

# Euclidean distance from mean
eucl = np.linalg.norm(point - mean)

# Mahalanobis distance from mean
diff = point - mean
cov_inv = np.linalg.inv(cov)  # small matrix, inv is OK here
mahal = np.sqrt(diff @ cov_inv @ diff)

print(f"Euclidean distance: {eucl:.2f}")
print(f"Mahalanobis distance: {mahal:.2f}")
# Mahalanobis is much larger — this point is a true outlier

Graph Laplacian and Spectral Methods

Turning Graphs into Matrices

Not all data comes as neat feature matrices. Social networks, citation graphs, molecular structures — these are graphs. Linear algebra reaches here too: turn graph into matrix, decompose it, extract structure.

adjacency matrix A of graph has A[i,j] = 1 if edge connects nodes i and j (or weight for weighted graphs). degree matrix D is diagonal, with D[i,i] = degree of node i. graph Laplacian is L = D - A. Looks arbitrary. Encodes deep structure.

Spectral Clustering

Eigenvectors of Laplacian reveal cluster structure. Smallest eigenvalue is always 0 (with eigenvector of all ones). Second-smallest eigenvector — Fiedler vector — gives optimal two-way split. Nodes with positive Fiedler values go in one cluster, negative in other. Spectral clustering computes k smallest eigenvectors, treats each node's eigenvector entries as coordinates, then runs k-means there.

Why work? Laplacian eigenvectors minimize quantity called graph cut — they find splits with few edges crossing between groups. Same telescope idea, now on graphs: eigenvectors show angles where community structure is clearest.

import numpy as np
from scipy.sparse.csgraph import laplacian

# A small graph with two clear clusters
# Nodes 0,1,2 are connected; nodes 3,4,5 are connected
# One weak link between 2 and 3
A = np.array([
    [0, 1, 1, 0, 0, 0],
    [1, 0, 1, 0, 0, 0],
    [1, 1, 0, 1, 0, 0],  # node 2 bridges the clusters
    [0, 0, 1, 0, 1, 1],  # node 3 bridges the clusters
    [0, 0, 0, 1, 0, 1],
    [0, 0, 0, 1, 1, 0],
])

L = laplacian(A)
eigenvalues, eigenvectors = np.linalg.eigh(L)
fiedler = eigenvectors[:, 1]  # second-smallest eigenvector

print(f"Fiedler vector: {fiedler.round(3)}")
print(f"Cluster assignment: {['A' if x < 0 else 'B' for x in fiedler]}")

PageRank: Power Iteration in Action

How find dominant eigenvector without computing all eigenvectors? Power iteration: start with random vector, multiply by matrix again and again, normalize each step. Vector converges to eigenvector with largest eigenvalue.

Google's original PageRank is power iteration on transition matrix of web graph. Each iteration refines page importance scores. It converges because transition matrix has dominant eigenvalue 1 (Perron-Frobenius theorem for stochastic matrices). Web has billions of pages, so full eigendecomposition is impossible. Power iteration still scales with sparse matrix-vector products.

import numpy as np

# Tiny web graph: 4 pages, links as edges
# Page 0 → 1, 2; Page 1 → 2; Page 2 → 0; Page 3 → 0, 2
n = 4
links = {0: [1, 2], 1: [2], 2: [0], 3: [0, 2]}

# Build transition matrix
M = np.zeros((n, n))
for src, dests in links.items():
    for dst in dests:
        M[dst, src] = 1.0 / len(dests)

# Power iteration
v = np.ones(n) / n
for i in range(50):
    v = M @ v
    v /= v.sum()

print(f"PageRank scores: {v.round(4)}")
# Page 0 and 2 have highest scores — most linked to

NMF: When You Need Parts, Not Wholes

The Problem with PCA and SVD

PCA and SVD can produce components with negative values. Mathematically fine. Sometimes nonsense. Face cannot have “negative nose.” Document cannot have “negative topic.” When data is inherently non-negative — pixel intensities, word counts, ratings — use decomposition that respects that constraint.

Non-Negative Matrix Factorization

NMF decomposes non-negative matrix V into two non-negative matrices: V ≈ WH. Both W and H stay non-negative. Constraint forces decomposition to learn additive parts. In face recognition, NMF learns pieces like “nose,” “eye,” “mouth” that add up to faces. PCA learns holistic features — whole eigenfaces — that cancel through subtraction. NMF parts are more interpretable.

In topic modeling, NMF on term-document matrix produces topics (columns of W) as non-negative mixtures of words, and document representations (rows of H) as non-negative mixtures of topics. It is popular alternative to LDA that is often faster and gives comparable results.

import numpy as np
from sklearn.decomposition import NMF

# Term-document matrix: 4 documents, 6 words
# Each entry = word count (non-negative)
V = np.array([
    [5, 0, 3, 0, 2, 0],  # doc about topic A
    [4, 0, 4, 0, 1, 0],  # doc about topic A
    [0, 3, 0, 5, 0, 4],  # doc about topic B
    [0, 4, 0, 3, 0, 5],  # doc about topic B
], dtype=float)

nmf = NMF(n_components=2, random_state=42)
W = nmf.fit_transform(V)  # document-topic matrix
H = nmf.components_        # topic-word matrix

print("Document-topic weights:\n", W.round(2))
print("Topic-word weights:\n", H.round(2))
print("Reconstruction:\n", (W @ H).round(1))

Notice result: NMF found two clean topics. Topic 1 loads on words 0, 2, 4. Topic 2 loads on words 1, 3, 5. Documents 0-1 load on topic 1; documents 2-3 on topic 2. Because everything is non-negative, parts stay additive and interpretable in way PCA components do not.

Tradeoff: NMF lacks SVD's optimality guarantee (Eckart-Young). Solution depends on initialization and is not unique. For many applications, interpretability matters more than mathematical optimality.
