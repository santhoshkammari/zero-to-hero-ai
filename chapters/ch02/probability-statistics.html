The Language of Uncertainty

Start with three emails in inbox. One from friend. One newsletter. One sells discount pharmaceuticals. We want classifier labeling each spam or not-spam. Need language for uncertain outcomes. Language = probability.

Sample Space, Events, and Probabilities

sample space Ω = set of all possible outcomes. For one email: {spam, not-spam}. For three emails: every label combination — eight total. Event = any subset of sample space: "at least one email is spam" is event. "all three are spam" is another.

Probability assigns number from 0 to 1 to each event. Three axioms run whole show: probabilities non-negative, probability of whole sample space is 1, probabilities of mutually exclusive events add. Done. Every theorem here — Bayes' theorem, law of total probability, rest — comes from those three rules. Big structure. Tiny foundation.

If roughly 45% of email is spam, assign P(spam) = 0.45 for random email. But what does number mean? Depends on who answers.

Two Philosophies: Frequentist and Bayesian

Frequentist: P(spam) = 0.45 means if you drew huge number of random emails, about 45% would be spam. Probability = long-run frequency. One-off event probabilities do not fit strict frequentist view; specific email either is spam or isn't.

Bayesian: P(spam) = 0.45 means given what I know now, belief this email is spam is 0.45. Probability = state of knowledge, not physical frequency. Lets you reason about one-off events, update beliefs with evidence, put distributions over parameters.

In ML, split matters. MLE is frequentist — find parameters that best explain observed data. MAP and full Bayesian inference are Bayesian — add priors, produce parameter distributions. Most practitioners use whatever builds better model. I lean Bayesian for regularization and uncertainty. I still use MLE every day. Math stays same.

Back to spam filter. Three emails. Features observed: contains "free," has attachment, sender in contacts. Now we need partial information — know something, update belief. That is conditional probability.

Conditional Probability — Where the Mistakes Live

Suppose email contains "free." Does belief about spam change? Yes. Emails with "free" are more likely spam. Conditional probability formalizes this: P(A|B) = probability of A given B happened.

Definition: P(A|B) = P(A ∩ B) / P(B). Read it like this: among cases where B happened, what fraction also had A? Denominator P(B) shrinks universe to outcomes where B is true. Simple ratio. Core tool of probabilistic reasoning.

For spam filter: P(spam | "free") = P(spam AND "free") / P("free"). If 40% of all emails are spam-with-"free" and 50% contain "free," then P(spam | "free") = 0.40 / 0.50 = 0.80. Knowing "free" pushes spam belief from 0.45 to 0.80. Conditioning.

The Trap: P(A|B) ≠ P(B|A)

I mixed up P(A|B) and P(B|A) for years. Not definition problem. Gut-swap problem. Common trap. Name: prosecutor's fallacy. It has sent innocent people to prison.

P(contains "free" | spam) is high — spammers love "free." But P(spam | contains "free") asks different question. Legitimate emails use it too — free trials, free shipping, "feel free to reach out." Probability of feature given class = likelihood. Probability of class given feature = posterior. Not same. Bayes' theorem converts between them.

In ML terms: P(data | model) = likelihood — easy from model. P(model | data) = posterior — what you want. Every training run wrestles with this split.

The swap test: See P(A|B)? Pause. Ask: "Am I secretly thinking P(B|A)?" This error breaks more probabilistic reasoning than anything else. Shows up in diagnoses, courts, spam filters, model evaluation.

Independence and Conditional Independence

Events A and B are independent if knowing one tells nothing about other: P(A|B) = P(A), equivalently P(A ∩ B) = P(A) · P(B). Two coin flips are independent. First flip landing heads tells nothing about second.

ML uses independence assumptions hard. Naive Bayes — engine behind many early spam filters — assumes features are conditionally independent given class label. That means: P(has "free", has attachment, not in contacts | spam) = P(has "free" | spam) · P(has attachment | spam) · P(not in contacts | spam). Wild assumption. "free" and attachments are clearly correlated in spam. Naive Bayes still works surprisingly well because classification boundary can be decent even when probability estimates are garbage.

Conditional independence is subtler: A and B are conditionally independent given C if P(A, B | C) = P(A|C) · P(B|C). Two symptoms may be correlated overall but conditionally independent given disease. Bayesian networks are built on this — factor huge joint distributions into small tractable pieces.

The Chain Rule and Autoregressive Models

Chain rule of probability breaks any joint distribution into product of conditionals: P(A, B, C) = P(A) · P(B|A) · P(C|A, B). For n variables: P(x₁, x₂, ..., xₙ) = P(x₁) · P(x₂|x₁) · P(x₃|x₁,x₂) · ... · P(xₙ|x₁,...,xₙ₋₁). Not assumption. Mathematical identity. Always true.

GPT works this way. It factors text-sequence probability as: P(token₁) · P(token₂|token₁) · P(token₃|token₁,token₂) · ... Each token prediction conditions on previous tokens. Chain rule guarantees factorization is exact — no information lost. Model job: estimate each conditional well. WaveNet does same for audio. PixelCNN for images. Autoregressive models = chain rule turned into architecture.

Spam filter could use chain rule too. Instead of assuming independent features, model P(has "free", has attachment, not in contacts | spam) = P(has "free" | spam) · P(has attachment | has "free", spam) · P(not in contacts | has "free", has attachment, spam). More accurate. But needs exponentially more data to estimate conditionals. Naive Bayes assumption is wrong, but it keeps data requirements manageable. Every modeling choice is tradeoff.

Bayes' Theorem — The One Equation

We built to this. Know P(A|B) = P(A ∩ B) / P(B). Also know P(B|A) = P(A ∩ B) / P(A). Both share P(A ∩ B) in numerator. Set equal. Rearrange:

P(A|B) = P(B|A) · P(A) / P(B)

That is Bayes' theorem. It flips conditional — turns P(B|A) into P(A|B). In ML notation, with parameters θ and data D:

P(θ|D) = P(D|θ) · P(θ) / P(D)

The Courtroom Analogy

Think courtroom. Prior = "innocent until proven guilty" before evidence. Likelihood = testimony: if defendant did crime, how probable is this evidence? Posterior = jury verdict: after evidence, how confident are we in guilt? Bayes' theorem is rational update from starting assumption to verdict, weighted by evidence strength.

With little evidence, prior dominates — jury should lean innocence. With overwhelming evidence, likelihood dominates — data overwhelms prior. Same behavior we want in ML: cautious with small datasets, data-driven with large ones.

Same pattern in weather. Prior = yesterday's forecast. Evidence = today's satellite data. Posterior = updated forecast. Rain probability jumping from 30% to 70% after new radar? Bayes' theorem working.

The Medical Test Example (Base Rate Fallacy)

This example made Bayes click for me. Result shocked me first time.

Disease affects 1 in 1,000 people. Test has 99% sensitivity (P(positive|disease) = 0.99) and 95% specificity (P(negative|no disease) = 0.95). You test positive. Probability you actually have disease?

Most people — doctors too, studies show — guess 95% or 99%. Real answer: about 2%.

import numpy as np

# Bayes' theorem: the medical test that humbles everyone
prevalence = 0.001           # P(disease) — the base rate
sensitivity = 0.99           # P(positive | disease) — true positive rate
specificity = 0.95           # P(negative | no disease)
false_positive_rate = 1 - specificity  # P(positive | no disease) = 0.05

# Law of total probability: P(positive)
p_positive = (sensitivity * prevalence) + (false_positive_rate * (1 - prevalence))

# Bayes' theorem: P(disease | positive)
p_disease_given_positive = (sensitivity * prevalence) / p_positive

print(f"P(positive)         = {p_positive:.5f}")
print(f"P(disease|positive) = {p_disease_given_positive:.4f}")
# P(disease|positive) ≈ 0.0194 — less than 2%!

# The counting approach makes it vivid:
n = 100_000
sick = int(n * prevalence)                        # 100 actually sick
healthy = n - sick                                 # 99,900 healthy
true_positives = int(sick * sensitivity)           # 99 caught correctly
false_positives = int(healthy * false_positive_rate)  # 4,995 false alarms

print(f"\nOut of {n:,} people:")
print(f"  Actually sick who test positive:  {true_positives}")
print(f"  Healthy who test positive:        {false_positives}")
print(f"  Total positive tests:             {true_positives + false_positives}")
print(f"  Chance you're actually sick:      {true_positives/(true_positives+false_positives):.1%}")

Intuition: disease is so rare that 5% false-positive rate on huge healthy population creates far more positive tests than 99% true-positive rate on tiny sick population. False alarms swamp real cases. Base rate matters enormously. Ignoring it is base rate fallacy.

Back to spam filter. If only 0.1% of emails are spam in some corporate inbox, even filter with 99% precision can generate mountain of false positives. That is why class imbalance wrecks naive accuracy metrics. Model predicting "not spam" always hits 99.9% accuracy. Precision and recall exist to fight this — Bayesian reasoning applied to classifier evaluation.

The Law of Total Probability

Used this above without naming it. If events B₁, B₂, ..., Bₙ partition sample space — mutually exclusive and exhaustive — then for any event A:

P(A) = P(A|B₁)·P(B₁) + P(A|B₂)·P(B₂) + ... + P(A|Bₙ)·P(Bₙ)

We computed P(positive) by summing two cases: sick and healthy. That is law of total probability. In ML, it appears whenever you marginalize: P(y) = Σₓ P(y|x)·P(x). Sum out latent variable in EM, VAEs, mixture models? Same law.

Random Variables — Turning Events into Numbers

Spam filter currently thinks in events: "this email is spam." But computation — optimization, differentiation, training — needs numbers. Random variable is bridge: function assigning number to each outcome. X = 1 if spam, X = 0 if not-spam. Now math works.

Discrete vs. Continuous

Discrete random variables take countable values: number of spam emails in batch, number of heads in 10 coin flips. Continuous random variables take uncountably many values in interval: spam probability from logistic regression (any number in [0, 1]), or loss on one training example (any non-negative real number).

PMF, PDF, CDF

Discrete random variable is described by probability mass function (PMF): P(X = x) for each value x. Values are non-negative and sum to one. Spam/not-spam variable has two-point PMF: P(X=1) = 0.45, P(X=0) = 0.55.

Continuous random variable is described by probability density function (PDF): f(x). Key point: PDF is not probability. It can be greater than 1. Probabilities come from integrating PDF over interval: P(a < X < b) = ∫f(x)dx from a to b. Probability of one exact value is zero for continuous variable. Weird. Necessary.

Cumulative distribution function (CDF) works for both types: F(x) = P(X ≤ x). Starts at 0, ends at 1, never decreases. CDF is underrated — tells what fraction lies below threshold, powers percentiles, enables inverse-CDF sampling.

Joint, Marginal, and Conditional Distributions

With two random variables X and Y, joint distribution P(X, Y) describes every (x, y) pair. Marginal P(X) sums out Y: P(X=x) = Σ_y P(X=x, Y=y). Conditional P(Y|X) gives Y's distribution for specific X.

This maps straight to ML's basic split. Discriminative models learn P(Y|X) — given features, predict label. Logistic regression, neural networks, SVMs. Generative models learn joint P(X, Y), so they model how data is generated. Naive Bayes, GANs, VAEs, diffusion models. Generative path lets you sample new data and detect outliers, but it requires modeling P(X) — whole input distribution — which is harder.

The Distribution Toolkit

Every ML model either assumes distribution or implicitly defines one. Gaussian noise assumption gives MSE. Bernoulli assumption gives cross-entropy. Choosing distribution is modeling decision with real consequences. Knowing options separates hyperparameter tuner from model designer.

The Gaussian — Why It's Everywhere

Gaussian (normal) distribution is parameterized by mean μ and variance σ²: f(x) = (1/√(2πσ²)) · exp(-(x-μ)²/(2σ²)). Symmetric. Bell-shaped. Everywhere. Not coincidence — Central Limit Theorem says sums of many independent effects converge to Gaussian no matter original distribution.

ML uses Gaussians constantly. Neural network weights often start from N(0, σ²) with carefully chosen σ (He initialization uses σ² = 2/n_in). Assume Gaussian noise on regression targets, you get MSE loss — not design taste, mathematical derivation we see in MLE section. VAEs force latent space Gaussian so sampling stays smooth. Batch normalization pushes activations toward N(0, 1). Gaussian processes place Gaussian prior over whole functions.

68-95-99.7 rule: ~68% within 1σ of mean, ~95% within 2σ, ~99.7% within 3σ. Quick outlier check: beyond 3σ is suspicious. Beyond 5σ is almost certainly bad data or different distribution.

The Multivariate Gaussian and Mahalanobis Distance

In d dimensions, multivariate Gaussian is parameterized by mean vector μ and covariance matrix Σ (d×d, symmetric positive definite). PDF: f(x) = (1/√((2π)ᵈ·det(Σ))) · exp(-½(x-μ)ᵀΣ⁻¹(x-μ)).

Σ controls probability-cloud shape. Diagonal → dimensions uncorrelated, cloud axis-aligned. Off-diagonal entries → correlations, cloud tilts. Eigenvectors of Σ are principal axes; eigenvalues are variance along each axis. That is what PCA computes: eigenvectors of sample covariance matrix, then project data onto max-variance directions.

Exponent (x-μ)ᵀΣ⁻¹(x-μ) is Mahalanobis distance — distance metric accounting for correlations and different scales. Euclidean distance treats all dimensions same. Mahalanobis stretches and rotates space until Gaussian becomes sphere, then measures Euclidean distance there. Right metric for outlier detection in correlated data.

The Beta Distribution — Modeling Probabilities

Beta distribution Beta(α, β) lives on [0, 1]. Perfect for modeling probability itself. Probability that spam filter's true accuracy is 0.85? That question needs distribution on [0, 1]. Beta is tool.

Shape depends on α and β. α = β = 1 gives uniform — total ignorance. α > 1 and β > 1 gives bell shape. Think α as pseudo-counts of successes, β as pseudo-counts of failures. Beta(10, 2) says "seen something like 10 successes and 2 failures" — peak near 0.83.

Beautiful part: Beta is conjugate prior for Bernoulli and Binomial likelihoods. Start with Beta(α, β), observe k successes in n trials, posterior is Beta(α+k, β+n-k). No integrals. No MCMC. Add counts. Done. We use this for spam filter soon.

The Dirichlet — Multivariate Beta

Dirichlet distribution Dir(α₁,...,αK) generalizes Beta to K dimensions. It produces vectors on probability simplex — non-negative, sum to one. It is conjugate prior for Categorical and Multinomial likelihoods. All αk > 1 favors uniform-ish distributions. All αk < 1 favors sparse distributions. In Latent Dirichlet Allocation (LDA), Dirichlet prior over topic-word distributions literally gives model its name.

Bernoulli, Binomial, Categorical, Poisson — and Your Loss Functions

Back to spam filter. Each email is Bernoulli trial: spam (1) with probability p, not-spam (0) with probability 1-p. Bernoulli distribution is simplest: P(X=x) = pˣ(1-p)¹⁻ˣ for x ∈ {0, 1}.

Now good part. Take negative log: -[y·log(p) + (1-y)·log(1-p)]. Familiar? That is binary cross-entropy loss. Loss used in every binary classifier is negative log-likelihood of Bernoulli. Not invented. Derived. MLE section makes derivation explicit.

Categorical distribution generalizes Bernoulli to K classes. Softmax output of neural network defines Categorical, and its negative log-likelihood is categorical cross-entropy loss. Binomial counts successes in n Bernoulli trials. Poisson models event counts at constant rate and shows up in NLP word-count models.

The Rest of the Family

Exponential models time between events — memoryless, used in survival analysis. Gamma generalizes it and serves as conjugate prior for Poisson rate. Chi-squared is Gamma special case — sum of squared standard normals — behind goodness-of-fit tests. Student-t looks like Gaussian with heavier tails; it approaches Gaussian as degrees of freedom grow and is standard for small-sample hypothesis tests. Uniform is maximum-entropy when range is all you know.

Expectation, Variance, and Friends

Expected Value

Expected value E[X] is weighted average of all possible values, weighted by probabilities. Discrete: E[X] = Σ x·P(X=x). Continuous: E[X] = ∫ x·f(x) dx. Think center of mass of distribution.

Linearity of expectation is one of probability's strongest tools: E[aX + bY] = a·E[X] + b·E[Y]. Holds always, even if X and Y are dependent. That is why expected mini-batch loss equals average of individual expected losses. No independence assumption needed.

Variance and Standard Deviation

Variance measures spread: Var(X) = E[(X - E[X])²] = E[X²] - (E[X])². Standard deviation σ = √Var(X) lives in same units as X. Key properties: Var(aX + b) = a²·Var(X) — scaling squares, shifts disappear. For independent variables: Var(X + Y) = Var(X) + Var(Y). That is why averaging n independent samples cuts variance by factor n — variance of sample mean is σ²/n.

Covariance, Correlation, and the Covariance Matrix

Covariance measures how two variables move together: Cov(X, Y) = E[(X-E[X])(Y-E[Y])] = E[XY] - E[X]·E[Y]. Positive: rise together. Negative: one rises, other falls. Zero: no linear relationship — nonlinear dependence can still exist.

Correlation normalizes to [-1, 1]: ρ(X,Y) = Cov(X,Y)/(σ_X·σ_Y). Scale-invariant measure of linear association.

For random vector X = [X₁,...,Xd], covariance matrix Σ has entry (i,j) = Cov(Xᵢ, Xⱼ). Symmetric. Positive semi-definite. Diagonal entries are variances. PCA decomposes this matrix: eigenvectors give principal components, eigenvalues give variance explained. When you whiten data by multiplying by Σ^(-½), dimensions become uncorrelated with unit variance — common preprocessing that helps optimization converge faster.

The Two Theorems That Make ML Work

Law of Large Numbers → Monte Carlo

Law of Large Numbers says sample mean converges to true mean as n grows. Take n i.i.d. samples from any distribution with mean μ. As n → ∞, average (1/n)·Σ Xᵢ → μ with probability 1.

This justifies using averages as estimators. Also why more training data helps: empirical risk (average training loss) converges to true risk (expected loss on data distribution). It is also foundation of Monte Carlo methods — if integral is intractable, sample and average. MCMC, importance sampling, every simulation-based method in ML stands on this.

Central Limit Theorem → Why SGD Works

Central Limit Theorem (CLT) goes deeper. Take n i.i.d. samples from any distribution with mean μ and finite variance σ². Distribution of sample mean is approximately Gaussian with mean μ and variance σ²/n, no matter original distribution. As n grows, approximation tightens.

CLT explains why Gaussians appear everywhere — heights, measurement errors, test scores are sums of many small independent effects. ML consequence is direct: gradients in mini-batch SGD are averages over batch_size samples. By CLT, stochastic gradient is approximately Gaussian around true gradient. Noise variance drops as 1/batch_size. Double batch size, halve gradient-noise variance. Returns diminish. Some noise even helps escape sharp minima.

import numpy as np

# CLT in action: averages of wildly non-Gaussian data become Gaussian
np.random.seed(42)
n_experiments = 10_000

# Source: heavily skewed Exponential (definitely not Gaussian)
lam = 2.0
true_mean = 1 / lam   # 0.5
true_var = 1 / lam**2  # 0.25

for batch_size in [1, 5, 30, 100]:
    means = np.array([
        np.mean(np.random.exponential(1/lam, batch_size))
        for _ in range(n_experiments)
    ])
    clt_predicted_std = np.sqrt(true_var / batch_size)
    print(f"batch_size={batch_size:3d}  "
          f"mean={np.mean(means):.4f} (true={true_mean:.4f})  "
          f"std={np.std(means):.4f} (CLT={clt_predicted_std:.4f})")
    # As batch_size grows, std shrinks and distribution becomes more Gaussian

MLE and MAP — Where Loss Functions Come From

Section where everything connects. Every loss function you use has probabilistic derivation. Understand that derivation, and you can design right loss for new problem instead of guessing.

Maximum Likelihood Estimation (MLE)

MLE asks: what parameter values make observed data most probable? Given data D = {x₁,...,xₙ} assumed i.i.d. from distribution with parameter θ, likelihood is L(θ) = P(D|θ) = Π P(xᵢ|θ). MLE finds θ* = argmax L(θ).

Products of small numbers underflow, so we use log-likelihood: log L(θ) = Σ log P(xᵢ|θ). Products become sums. Maximizing log-likelihood is equivalent — log is monotonic — and numerically stable.

Now punchline. What happens when you write negative log-likelihood for specific distributional assumptions?

AssumptionNegative Log-LikelihoodYou Get
Gaussian noise on targetsΣ (yᵢ - ŷᵢ)² / (2σ²) + constMean Squared Error
Bernoulli output-Σ [yᵢ·log(pᵢ) + (1-yᵢ)·log(1-pᵢ)]Binary Cross-Entropy
Categorical output-Σ log(p_yᵢ)Categorical Cross-Entropy

MSE is not design taste — it appears when regression targets have Gaussian noise. Cross-entropy is not design taste — it appears when outputs are Bernoulli or Categorical. Every time you call loss = nn.CrossEntropyLoss(), you are doing MLE under Categorical assumption. Worth burning into memory.

See it concretely with spam filter. Each email has true label y ∈ {0,1}; model outputs spam probability p. Bernoulli log-likelihood for one email is y·log(p) + (1-y)·log(1-p). Sum over emails, negate, get binary cross-entropy. Minimizing it equals maximizing likelihood that model generated observed spam labels. That's where loss comes from.

Code: MLE Estimation

import numpy as np

np.random.seed(42)

# MLE for Gaussian: estimates converge as sample size grows
true_mu, true_sigma = 5.0, 2.0

for n in [10, 50, 200, 1000, 10000]:
    data = np.random.normal(true_mu, true_sigma, n)
    mu_mle = np.mean(data)                        # MLE for μ = sample mean
    sigma2_mle = np.mean((data - mu_mle)**2)      # MLE for σ² uses 1/n (biased)
    print(f"n={n:5d}  μ̂={mu_mle:.4f} (true={true_mu})  "
          f"σ̂²={sigma2_mle:.4f} (true={true_sigma**2:.1f})")

# Visualize: log-likelihood peaks at the sample mean
data = np.random.normal(true_mu, true_sigma, 100)

def gaussian_log_lik(mu, data, sigma=true_sigma):
    n = len(data)
    return -n/2 * np.log(2*np.pi*sigma**2) - np.sum((data-mu)**2)/(2*sigma**2)

mu_grid = np.linspace(2, 8, 200)
ll_values = [gaussian_log_lik(mu, data) for mu in mu_grid]
best_mu = mu_grid[np.argmax(ll_values)]
print(f"\nGrid search peak: μ={best_mu:.3f}, sample mean: {np.mean(data):.3f}")
# They match — the MLE for the Gaussian mean IS the sample mean

MAP: Add a Prior, Get Regularization

MAP maximizes posterior instead of likelihood: θ_MAP = argmax P(θ|D) = argmax [P(D|θ)·P(θ)]. Take log: argmax [log P(D|θ) + log P(θ)]. First term = log-likelihood — data fit. Second = log-prior — regularization.

Famous connections come from here:

Prior on WeightsLog-Prior PenaltyRegularization
Gaussian: P(w) ∝ exp(-w²/(2σ²))-Σ wᵢ² / (2σ²)L2 (weight decay)
Laplace: P(w) ∝ exp(-|w|/b)-Σ |wᵢ| / bL1 (sparsity)

Gaussian prior says "weights are probably small" and penalizes large values quadratically — that is L2 regularization. Laplace prior penalizes linearly, pushing many weights to exactly zero — that is L1 regularization and why LASSO gives sparse solutions. Every time you add weight_decay=0.01 to optimizer, you make Bayesian statement: "weights have Gaussian prior with this variance." Regularization strength λ is inversely proportional to prior variance σ².

MLE vs. MAP vs. Full Bayesian

Three levels of inference: MLE gives one point estimate — most likely parameters. MAP gives one point estimate with regularization — most likely parameters under prior. Full Bayesian inference gives whole posterior distribution over parameters — not one answer, weighted distribution of answers. Most deep learning uses MLE or MAP. Full Bayesian methods (Gaussian processes, Bayesian neural networks) shine in small-data settings where uncertainty matters.

Fisher Information and Cramér-Rao (Brief)

Fisher information I(θ) = E[(∂/∂θ log P(X|θ))²] measures how much information data carries about θ. High Fisher information means log-likelihood has sharp curvature — data is informative. Cramér-Rao bound says no unbiased estimator can have variance below 1/I(θ). MLE is asymptotically efficient — it hits this bound for large n. One reason MLE dominates: provably optimal in large-sample limit.

Bayesian Inference — The Full Picture

The Workflow: Prior → Likelihood → Posterior

Bayesian workflow always has three steps. (1) Choose prior P(θ): beliefs before data. (2) Choose likelihood P(D|θ): how data is generated. (3) Apply Bayes' theorem to get posterior P(θ|D). If new data arrives, current posterior becomes next prior. Sequential updating is one of Bayesian framework's big strengths.

For spam filter: believe spam rate is around 50% but uncertain. Use Beta(2, 2) prior. Observe 100 emails, 45 spam. Posterior becomes Beta(2+45, 2+55) = Beta(47, 57). New posterior mean: 47/104 ≈ 0.452. As more data arrives, uncertainty shrinks and estimate moves toward true rate. Bayesian learning.

Conjugate Priors

Prior is conjugate to likelihood if posterior has same form as prior. These cases are analytically tractable — no approximation needed.

LikelihoodConjugate PriorPosteriorUpdate Rule
Bernoulli / BinomialBeta(α, β)Beta(α+k, β+n-k)Add successes to α, failures to β
Gaussian (known σ²)Gaussian(μ₀, σ₀²)Gaussian(μₙ, σₙ²)Precision-weighted average of prior and data
PoissonGamma(α, β)Gamma(α+Σxᵢ, β+n)Add total count to α, sample size to β
Categorical / MultinomialDirichlet(α₁,...,αK)Dirichlet(α₁+c₁,...,αK+cK)Add observed counts per category

Posterior Predictive Distribution

After P(θ|D), predict new data by integrating out parameter: P(x_new|D) = ∫ P(x_new|θ)·P(θ|D) dθ. That is posterior predictive. Unlike plug-in predictions using one θ, it accounts for parameter uncertainty — if θ is uncertain, predictions widen. Bayesian models give calibrated uncertainty because they know what they don't know.

Code: Beta-Binomial Bayesian Updating

import numpy as np
from scipy.stats import beta as beta_dist

# Bayesian spam filter: estimating the true spam rate
# True rate (unknown to us): 0.45
true_spam_rate = 0.45

# Prior: Beta(2, 2) — we mildly believe spam rate is ~50%
alpha, beta_param = 2.0, 2.0
print(f"Prior: Beta({alpha:.0f}, {beta_param:.0f}), mean = {alpha/(alpha+beta_param):.3f}\n")

np.random.seed(42)
batch_sizes = [5, 10, 20, 50, 100, 500]
total_seen = 0

for batch in batch_sizes:
    emails = np.random.binomial(1, true_spam_rate, batch)
    spam_count = emails.sum()
    not_spam = batch - spam_count
    total_seen += batch

    alpha += spam_count
    beta_param += not_spam

    mean = alpha / (alpha + beta_param)
    ci_low, ci_high = beta_dist.ppf([0.025, 0.975], alpha, beta_param)

    print(f"After {total_seen:4d} emails ({spam_count:2d} spam in batch): "
          f"Beta({alpha:.0f}, {beta_param:.0f})  "
          f"mean={mean:.3f}  95% CI=[{ci_low:.3f}, {ci_high:.3f}]")

print(f"\nTrue rate: {true_spam_rate}  |  Final estimate: {alpha/(alpha+beta_param):.3f}")
print("The credible interval tightens as evidence accumulates.")

MCMC: When Conjugacy Isn't Available

Most real models lack conjugate priors. Posterior becomes complicated unnormalized density you cannot integrate analytically. Markov Chain Monte Carlo (MCMC) handles this: build Markov chain whose stationary distribution is posterior, run long enough, collected samples approximate posterior.

Metropolis-Hastings is simplest MCMC. Each step: propose new parameter value, compute acceptance ratio (ratio of posterior densities), accept or reject probabilistically. Higher posterior probability? Always accept. Lower? Accept sometimes — keeps chain from getting stuck. Accepted samples form approximate posterior.

Hamiltonian Monte Carlo (HMC) uses gradients to make smarter proposals — it travels along posterior surface instead of stumbling randomly. HMC explores high-dimensional posteriors much more efficiently. PyMC and Stan use HMC variants (NUTS — No-U-Turn Sampler) underneath.

Variational Inference → ELBO → VAEs

Variational inference took me three tries to understand. Version that finally clicked:

Variational Inference (VI) takes different path from MCMC: instead of sampling from posterior, approximate it with simpler distribution q(θ; φ) and optimize φ so q is as close as possible to true posterior.

"Close" is measured by KL divergence: KL(q || p). Minimizing it directly needs true posterior — exact thing we want to find. Trick: maximize Evidence Lower Bound (ELBO) instead:

ELBO = E_q[log P(D|θ)] - KL(q(θ) || P(θ))

First term: approximate posterior should explain data. Second term: approximate posterior should not drift too far from prior. Maximizing ELBO is equivalent to minimizing KL divergence to true posterior.

VAE loss function is exactly this. Encoder outputs q(z|x) — approximate posterior over latent variables. Decoder outputs P(x|z) — likelihood. Loss = reconstruction loss (negative log-likelihood) + KL divergence between encoder output and standard Gaussian prior. That is negative ELBO. VAE loss is not ad hoc — it is variational inference with specific approximate-posterior family.

Hypothesis Testing and A/B Testing

Null and Alternative Hypotheses

You trained two spam filters. Model B looks 2% better on test set. Real improvement, or luck from test split? Hypothesis testing formalizes that question.

Null hypothesis H₀ is boring story: no real difference, 2% is noise. Alternative hypothesis H₁: real difference exists. You never prove H₁. You either reject H₀ — strong evidence against no-difference — or fail to reject it — not enough evidence.

p-values — What They Are and What They Are NOT

p-value is probability of seeing data at least as extreme as observed, assuming H₀ is true. Small p-value means "this result would be very unlikely if no real effect existed" — evidence against H₀.

What p-values are NOT:
  • NOT probability that H₀ is true. p-value 0.03 does not mean "3% chance models are same."
  • NOT probability result happened by chance.
  • NOT measure of effect size. Tiny meaningless difference can give tiny p-value with enough data.
p-value is P(data | H₀), not P(H₀ | data). Mixing these repeats conditional-probability swap from earlier. Getting P(H₀ | data) needs Bayes' theorem and prior on H₀.

Type I/II Errors and Power

H₀ True (no real difference)H₀ False (real difference)
Reject H₀Type I Error (false positive, rate = α)Correct! (Power = 1 - β)
Fail to reject H₀Correct!Type II Error (false negative, rate = β)

Statistical power (1 - β) is probability of detecting real effect when it exists. It depends on effect size, sample size, significance level, and variance. Run power analysis before experiments — underpowered studies waste resources and give unreliable conclusions.

Paired t-Tests for Model Comparison

When comparing two models on same test set, use paired t-test. For each sample, compute performance difference. Paired test accounts for sample difficulty — some examples are hard for both models. It tests whether differences are systematically non-zero.

Multiple Testing Correction

Run 20 tests at α = 0.05 and expect one false positive even with no real effects. Bonferroni correction: divide α by number of tests. Simple. Conservative. Benjamini-Hochberg: controls false discovery rate (FDR) instead — less conservative, standard when you expect some true positives among many tests.

A/B Testing: Frequentist vs. Bayesian

Frequentist A/B testing: fix sample size, run experiment, compute p-value. Binary outcome: significant or not. You cannot peek early without inflating error rates. Bayesian A/B testing: compute posterior probability that variant B is better. Check anytime, because you are updating belief, not running fixed-sample procedure. Bayesian approach gives P(B is better | data) directly — much more useful than p-value.

Code: Paired t-Test for Model Comparison

import numpy as np
from scipy import stats

np.random.seed(42)
n_test = 200

# Two spam filters on the same test set
difficulty = np.random.uniform(0, 1, n_test)
model_a_correct = (difficulty + np.random.normal(0, 0.15, n_test) < 0.82).astype(int)
model_b_correct = (difficulty + np.random.normal(0, 0.15, n_test) < 0.78).astype(int)

print(f"Model A accuracy: {model_a_correct.mean():.3f}")
print(f"Model B accuracy: {model_b_correct.mean():.3f}")
print(f"Raw difference:   {model_a_correct.mean() - model_b_correct.mean():.3f}\n")

# Paired t-test on per-sample differences
differences = model_a_correct - model_b_correct
t_stat, p_value = stats.ttest_rel(model_a_correct, model_b_correct)

print(f"Paired t-test:")
print(f"  Mean difference: {differences.mean():.4f}")
print(f"  t-statistic:     {t_stat:.3f}")
print(f"  p-value:         {p_value:.4f}")

if p_value < 0.05:
    print("  → Reject H₀: difference is statistically significant at α=0.05")
else:
    print("  → Fail to reject H₀: no significant difference detected")

# Effect size
cohens_d = differences.mean() / differences.std()
label = 'small' if abs(cohens_d) < 0.5 else 'medium' if abs(cohens_d) < 0.8 else 'large'
print(f"  Cohen's d:       {cohens_d:.3f} ({label})")

Descriptive Statistics and Resampling

Summary Statistics

You know these: mean (average), median (middle value, robust to outliers), mode (most frequent). Variance and standard deviation measure spread. Skewness measures asymmetry — income distributions are right-skewed, so mean exceeds median. Kurtosis measures tail heaviness — high kurtosis means more outliers than Gaussian. In practice, always check median with mean. Big gap means skewed distribution and misleading mean.

Bootstrap Confidence Intervals

Bootstrap is one of statistics' most direct tools. Want uncertainty for any statistic — any statistic — without distributional assumptions? Resample data with replacement many times. Compute statistic on each resample. Look at resulting distribution. 2.5th and 97.5th percentiles give 95% confidence interval. Works for means, medians, AUC, F1, any strange metric you can compute.

import numpy as np

np.random.seed(42)

# Bootstrap CI for our spam filter's accuracy
n_test = 500
correct = np.random.binomial(1, 0.85, n_test)  # ~85% accuracy
observed_acc = correct.mean()

n_bootstrap = 10_000
boot_accs = np.array([
    np.random.choice(correct, size=n_test, replace=True).mean()
    for _ in range(n_bootstrap)
])

ci_low, ci_high = np.percentile(boot_accs, [2.5, 97.5])

print(f"Observed accuracy:  {observed_acc:.4f}")
print(f"Bootstrap mean:     {boot_accs.mean():.4f}")
print(f"Bootstrap std:      {boot_accs.std():.4f}")
print(f"95% Bootstrap CI:   [{ci_low:.4f}, {ci_high:.4f}]")
print(f"\nWe're 95% confident the true accuracy is between {ci_low:.1%} and {ci_high:.1%}.")

Permutation Tests

Permutation tests assess significance by randomly shuffling labels. Want to know whether two models differ? Randomly swap which predictions belong to Model A vs. B many times. If observed difference lands in top 5% of shuffled distribution, reject null at α = 0.05. Like bootstrap, permutation tests are nonparametric — no distributional assumptions.

Concentration Inequalities

These bounds tell how far random variable can stray from expected value. They are theoretical backbone of generalization in learning theory — and they show up in ML interviews when talk turns to "why does model generalize?"

Markov's inequality: For non-negative X, P(X ≥ a) ≤ E[X]/a. Very loose. Needs almost nothing. Chebyshev's inequality: P(|X - μ| ≥ kσ) ≤ 1/k². Uses variance for tighter bound. Still distribution-free.

Hoeffding's inequality is workhorse. For n independent bounded random variables in [a, b]: P(|X̄ - μ| ≥ t) ≤ 2·exp(-2nt²/(b-a)²). Probability that sample mean deviates from true mean drops exponentially with n.

This gives PAC (Probably Approximately Correct) learning bounds directly: with high probability — "probably" — empirical risk stays close to true risk — "approximately correct." Hoeffding tells exactly how many samples you need for chosen confidence. Sub-Gaussian random variables extend exponential tail bounds to unbounded distributions — and many ML quantities (bounded losses, Gaussian noise) are sub-Gaussian.

Gaussian Mixture Models

Gaussian Mixture Model assumes data comes from K Gaussian components. Generative story: pick component k with probability πk (mixing weight), then sample x from N(μk, Σk). Density is P(x) = Σk πk · N(x|μk, Σk). You do not know which component generated each point — that hidden choice is latent variable.

GMMs are fit with Expectation-Maximization (EM). Two alternating steps:

E-step: For each data point, compute posterior probability — responsibility — that it belongs to each component: rᵢk = πk·N(xᵢ|μk, Σk) / Σⱼ πⱼ·N(xᵢ|μⱼ, Σⱼ). Bayes' theorem here: prior is mixing weight, likelihood is Gaussian density, posterior is responsibility.

M-step: Update each component's parameters using responsibilities as soft weights: μk = (Σᵢ rᵢk·xᵢ) / (Σᵢ rᵢk), and similarly for Σk and πk.

EM is guaranteed to increase or preserve log-likelihood each iteration — it never gets worse. But it can land in local optima, so run it multiple times with different initializations.

GMMs give soft clustering: each point gets membership probability for each cluster. K-means is special case of GMM where all covariances are equal and isotropic, then you take hard assignments — argmax instead of soft responsibilities. If you need non-spherical clusters or assignment uncertainty, use GMMs.

Causal Inference

Correlation does not imply causation. Heard it thousand times. Still traps people. Ice cream sales correlate with drowning deaths — hot weather is confounder causing both. Spam filter might learn emails from certain domain correlate with spam, but blocking that domain may kill legitimate emails because some third factor drives correlation.

Simpson's paradox is sharp example: trend inside subgroups reverses when you combine them. Treatment can look better overall but worse in every subgroup because group sizes and baselines differ. Confounding in action. Fix: condition on confounder, not blindly aggregate.

Directed Acyclic Graphs (DAGs) formalize causal relationships. Judea Pearl's do-calculus separates observing from intervening: P(Y | do(X=x)) ≠ P(Y | X=x) because "doing" cuts confounder's influence on X. Models trained on observational data learn correlations, not causal effects. Model that learns "lighter → lung cancer" will make terrible interventions. As ML moves into healthcare, policy, and other high-stakes domains, causal reasoning becomes essential.
