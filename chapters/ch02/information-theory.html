Entropy kept showing up in loss functions, KL divergence in VAE papers, information gain in decision tree internals. Information theory not side topic. It was language my tools used all along.

Information theory started in 1948 when Claude Shannon published "A Mathematical Theory of Communication." Goal: transmit messages reliably over noisy wire. Result: mathematical framework for uncertainty, surprise, and cost of being wrong. Machine learning needs exactly those ideas.

The Bigger Picture: Why ML Speaks Information Theory

Surprise and Entropy

Start concrete. Build tiny weather app for city with sun 364 days each year. Every morning, predict "sunny." Most days bring no surprise. Same weather as yesterday and day before. One rainy day hits hard. That outcome carries lot of information because it was unlikely.

Core insight: unlikely events carry more information than likely ones. Shannon formalized surprise for single event with probability p as:

surprise(x) = -log₂(p(x))

When p = 1 (certainty), surprise is zero. Outcome already known. When p is tiny, surprise is large. Negative sign stays because log of number between 0 and 1 is negative, and surprise should be positive.

Entropy is average surprise across all possible outcomes, weighted by each outcome's probability:

H(X) = -Σ p(xᵢ) log₂ p(xᵢ)

Back to weather app. City with sunshine 99% of time and rain 1% has entropy about 0.08 bits. Average surprise stays tiny, so weather prediction needs barely any information. City with sun half time and rain half time has entropy 1 bit: maximum uncertainty for two outcomes. City with three equally likely states (sunny, rainy, cloudy) reaches about 1.58 bits. More outcomes mean more uncertainty and more bits to describe result.

Pattern: entropy is highest when outcomes are equally likely and lowest when one outcome is certain. Think filing cabinet. Cabinet with one labeled folder and everything inside has zero entropy. Cabinet with documents scattered across dozens of folders has high entropy. More mess means more information needed to say where anything is.

Why Logarithms?

Log base caused trouble for years. Why use log at all? Why not some other function that grows for small numbers and shrinks for big ones?

Reason: logarithms make information additive. Flip two independent fair coins, total information should be 1 bit + 1 bit = 2 bits, not some strange product. Joint probabilities of independent events multiply (P(A and B) = P(A) × P(B)). Logs turn that multiplication into addition. Whole reason.

Base picks unit. Log base 2 gives bits, log base e (natural log) gives nats, log base 10 gives hartleys. ML usually uses nats because natural logs work well with calculus and gradients. Base choice is unit conversion, like Celsius vs Fahrenheit. Underlying physics stays same.

Differential Entropy

When random variable is continuous, replace sum with integral: h(X) = -∫ p(x) log p(x) dx. Gaussian with variance σ² has differential entropy ½ log(2πeσ²). Wider Gaussians have higher entropy because more spread means more uncertainty about where sample lands.

Oddity: unlike discrete entropy, differential entropy can be negative. Very peaked distribution packs so much probability into tiny region that average surprise drops below zero. Strange first time you compute it, but practical trouble is rare. Quirk of continuous formulation, not sign of failure.

Conditional and Joint Entropy

Back to weather app. Add barometer reading. Conditional entropy H(weather | barometer) measures uncertainty left in weather after reading barometer. If barometer predicts weather perfectly, conditional entropy is zero. No surprise left. If barometer is broken and random, barometer tells you nothing, so H(weather | barometer) = H(weather).

Joint entropy H(weather, barometer) captures total uncertainty in both variables together. Chain rule connects them: H(X, Y) = H(X) + H(Y|X). Total uncertainty equals uncertainty in X plus uncertainty left in Y once X is known. This decomposition drives how decision trees choose splits.

Entropy in the Wild

Here entropy becomes concrete. In decision trees, information gain at each split node is H(Y) - H(Y|X): reduction in target entropy after splitting on feature X. ID3, C4.5, and descendants run information theory at every node. Feature with highest information gain cuts uncertainty about label most. Tree picks that one.

Entropy also acts as regularizer in reinforcement learning. In Soft Actor-Critic (SAC), entropy bonus is added to policy objective. Higher-entropy policy tries wider range of actions instead of collapsing early into one deterministic choice. Entropy term measures how much policy explores, and algorithm balances reward with exploration.

import numpy as np

# Our weather app: three cities with different predictability
# City A: almost always sunny
city_a = np.array([0.99, 0.005, 0.005])
# City B: equal chance of anything
city_b = np.array([1/3, 1/3, 1/3])
# City C: usually sunny, sometimes rainy, rarely cloudy
city_c = np.array([0.7, 0.2, 0.1])

def entropy(probs):
    # Filter zeros to avoid log(0)
    probs = probs[probs > 0]
    return -np.sum(probs * np.log2(probs))

for name, p in [("Predictable", city_a), ("Chaotic", city_b), ("Realistic", city_c)]:
    print(f"{name} city: H = {entropy(p):.3f} bits")
# Predictable city: H = 0.091 bits
# Chaotic city:     H = 1.585 bits
# Realistic city:   H = 1.157 bits

Predictable city needs barely any bits to describe weather. Chaotic city needs most. Filing-cabinet analogy holds: messier distribution needs more information to pin down outcome.

Cross-Entropy: Measuring How Wrong Your Model Is

Entropy tells us inherent uncertainty in distribution. ML asks different question: true distribution P comes from real data, model distribution Q comes from neural network predictions. How wrong is Q?

Imagine weather app predicts sunny 90%, rainy 8%, cloudy 2%. Real weather is sunny 70%, rainy 20%, cloudy 10%. Model is too confident about sunshine and underestimates rain. Cross-entropy measures mismatch:

H(P, Q) = -Σ p(x) log q(x)

Think of it this way. Entropy H(P) gives minimum average bits needed to encode weather with perfect model. Cross-entropy H(P, Q) gives bits you actually need when using imperfect model Q. Difference is waste: bits spent on being wrong.

If Q matches P perfectly, cross-entropy equals entropy. Encoding is optimal. If Q is poor approximation, cross-entropy rises. Like tourist using phrasebook for wrong country. Some communication happens, but effort is much higher than for local speaker.

Key connection: when you call nn.CrossEntropyLoss() in PyTorch, you compute exactly this. One-hot label is P (truth), softmax output is Q (model belief). Loss measures extra bits wasted because model does not match reality. For binary classification, it reduces to -[y log(q) + (1-y) log(1-q)]. That formula is pure information theory.

Another key connection: minimizing cross-entropy loss is equivalent to maximum likelihood estimation. Both ask: make Q match P as closely as possible. One comes from information theory, one from statistics. Both land on same optimization problem.

KL Divergence: The Exact Cost of Being Wrong

Cross-entropy bundles two parts: inherent uncertainty in data and penalty for being wrong. KL divergence (Kullback-Leibler divergence) isolates second part: pure overhead from using Q instead of P:

D_KL(P ‖ Q) = Σ p(x) log [p(x) / q(x)]

Fundamental relationship is H(P, Q) = H(P) + D_KL(P ‖ Q). Cross-entropy equals true entropy plus KL divergence. Since H(P) is constant during training, minimizing cross-entropy is mathematically identical to minimizing KL divergence. Not coincidence. Not approximation. Identity. Reason cross-entropy loss works.

KL divergence is always non-negative. You can never beat true distribution's own entropy. This fact, called Gibbs' inequality, means best possible cross-entropy loss is H(P), reached only when Q = P exactly.

Forward vs Reverse KL

Subtle part: forward and reverse KL look similar, but difference matters.

KL divergence is asymmetric: D_KL(P ‖ Q) ≠ D_KL(Q ‖ P). Not minor detail. Direction choice changes model behavior.

Forward KL, D_KL(P ‖ Q), penalizes Q hard wherever P has mass but Q does not. Model logic: cover everything data might produce. Result is mode-covering. Q spreads to blanket all of P, even across low-density regions between modes. Maximum likelihood estimation does this.

Reverse KL, D_KL(Q ‖ P), penalizes Q for placing mass where P is near zero. Model logic: do not put probability where data would not go. Result is mode-seeking. Q locks onto one mode of P and ignores others. Variational inference does this.

Picture target distribution P with two peaks, like two clusters of weather patterns. If Q can only be one Gaussian, forward KL stretches Q across both peaks: covers both, fits neither well. Reverse KL collapses Q onto one peak: fits one well, ignores other. Neither is universally right. Choice depends on whether missing mode or being imprecise everywhere hurts more.

This is why variational inference often produces posteriors that are too narrow. It uses reverse KL. Approximate posterior would rather be confidently wrong about some modes than place probability where true posterior is low.

Let's see this concretely with our weather app:

import numpy as np

# True weather distribution (two distinct seasonal patterns blended)
P = np.array([0.4, 0.35, 0.25])  # sunny, rainy, cloudy

# Model A: spreads probability broadly (mode-covering behavior)
Q_cover = np.array([0.33, 0.34, 0.33])

# Model B: locks onto the dominant mode (mode-seeking behavior)
Q_seek = np.array([0.7, 0.2, 0.1])

def kl_divergence(p, q):
    # Only where p > 0
    mask = p > 0
    return np.sum(p[mask] * np.log(p[mask] / q[mask]))

print(f"Forward KL(P || Q_cover): {kl_divergence(P, Q_cover):.4f} nats")
print(f"Forward KL(P || Q_seek):  {kl_divergence(P, Q_seek):.4f} nats")
print(f"Reverse KL(Q_cover || P): {kl_divergence(Q_cover, P):.4f} nats")
print(f"Reverse KL(Q_seek || P):  {kl_divergence(Q_seek, P):.4f} nats")
# Notice: the two directions rank the models differently

KL divergence is not distance metric. It violates symmetry and triangle inequality. Say "KL divergence from P to Q," not "KL distance between P and Q." Direction encodes design choice about model behavior.

Where does KL divergence show up in practice? Everywhere. In VAEs, ELBO loss includes KL term that pushes encoder's approximate posterior toward standard Gaussian prior. In knowledge distillation, you minimize KL divergence between teacher soft outputs and student predictions. In PPO and TRPO, KL penalty keeps new policy from drifting too far from old one. Same leash each time: do not let Q wander too far from reference.

Jensen-Shannon Divergence

KL divergence asymmetry can be problem. Jensen-Shannon divergence (JSD) fixes it by averaging both directions through midpoint distribution M = ½(P + Q):

JSD(P ‖ Q) = ½ D_KL(P ‖ M) + ½ D_KL(Q ‖ M)

JSD is symmetric, bounded between 0 and 1 with log base 2, and its square root is proper metric. Original GAN discriminator loss implicitly estimates JSD between real and generated data. Elegant, but practical problem appears early in training: if real and generated distributions do not overlap, JSD saturates and gives no useful gradient. Wasserstein GANs were built for this. Wasserstein distance still gives meaningful gradients when distributions are disjoint.

Phrasebook analogy extends here. KL divergence asks how much harder it is for you, speaking model-Q-language, to communicate in country-P. JSD seats both speakers at one table and measures total miscommunication in both directions. Wasserstein distance asks different question: how much physical effort does it take to rearrange one language into other?

Mutual Information: What Do These Variables Share?

So far, model Q was compared to truth P. Sometimes question changes. You have two variables, like barometer readings and tomorrow's weather. How much does one tell you about other?

Mutual information I(X; Y) answers that question: how much does knowing X reduce uncertainty about Y?

I(X; Y) = H(Y) - H(Y|X)

That is formula. Start with uncertainty about Y. Subtract uncertainty left after observing X. Difference is information X gives about Y. Equivalent form is I(X; Y) = H(X) + H(Y) - H(X, Y). Symmetry is real: X tells as much about Y as Y tells about X.

If X and Y are independent, I(X; Y) = 0. Knowing one tells you nothing about other. Broken barometer has zero mutual information with weather. If X completely determines Y, then I(X; Y) = H(Y). All uncertainty disappears. Perfect barometer has mutual information equal to weather's entropy.

Venn diagram gives clear picture. Draw two overlapping circles: H(X) and H(Y). Overlap is mutual information: shared uncertainty. Total area of both circles is joint entropy H(X, Y). Part of Y that does not overlap with X is conditional entropy H(Y|X). Information-theoretic relationships become statements about circle areas.

Feature Selection That Actually Works

Why mutual information matters in practical ML: Pearson correlation measures linear relationships. If Y = X² and X is symmetric around zero, Pearson correlation between X and Y is exactly zero. It misses relationship completely. Mutual information sees full picture because it captures any statistical dependency, linear or not.

Scikit-learn's mutual_info_classif and mutual_info_regression use this directly. They estimate mutual information between each feature and target, giving feature-importance score that still works for nonlinear relationships where correlation fails silently.

import numpy as np
from sklearn.feature_selection import mutual_info_classif

# Feature X1: linearly related to target
# Feature X2: nonlinearly related (X2 determines target, but nonlinearly)
# Feature X3: pure noise
np.random.seed(42)
n = 1000
X1 = np.random.randn(n)
X2 = np.random.randn(n)
X3 = np.random.randn(n)

# Target depends on X1 linearly and X2 quadratically
target = 2*X1 + 3*(X2**2) + 0.5*np.random.randn(n)
y = (target > np.median(target)).astype(int)

X = np.column_stack([X1, X2, X3])
mi = mutual_info_classif(X, y, random_state=42)

# Pearson correlation for comparison
corr = [np.abs(np.corrcoef(X[:, i], y)[0, 1]) for i in range(3)]

print("Feature | Mutual Info | |Correlation|")
for i, name in enumerate(["Linear", "Nonlinear", "Noise"]):
    print(f"  {name:10s} | {mi[i]:.3f}       | {corr[i]:.3f}")
# MI catches the nonlinear feature. Correlation misses it.

Estimation is hard part. Exact mutual information needs true joint distribution, and we never have that. In practice, k-nearest-neighbor estimators (Kraskov method) approximate it from samples. Works reasonably well in low dimensions, then struggles as dimensionality grows. Curse of dimensionality hits MI estimation hard.

Contrastive Learning and InfoNCE

Modern contrastive learning (SimCLR, CPC, CLIP) has clean information-theoretic interpretation. InfoNCE loss, introduced by van den Oord et al. in 2018, is lower bound on mutual information between two views of same data point. When CLIP aligns image and text embeddings, it maximizes mutual information between visual and linguistic representations of same concept.

In InfoGAN, generator maximizes mutual information between subset of latent codes and generated output. Result: those latent codes learn to control interpretable, disentangled factors like rotation, thickness, and style. High mutual information means code actually determines variation.

The Data Processing Inequality

One profound result fits in one line. If X → Y → Z form chain, so Z depends on X only through Y, then:

I(X; Z) ≤ I(X; Y)

Processing data can only destroy information, never create it. Every transformation, function, and neural-network layer can only lose information about original input. If preprocessing throws away detail from raw signal, no downstream model can recover it. Ever.

This sounds bleak, but it explains deep learning. Good neural network does not try to preserve all information. It tries to throw away right information. Photo of dog contains breed, lighting, camera sensor noise, JPEG compression artifacts, and exact background-wall shade. Breed classifier should keep breed information and discard rest aggressively.

Think of filing cabinet again. Massive cabinet holds every document ever generated: raw input. Job is not building bigger cabinet. Job is building smaller one that keeps documents needed for task and shreds rest. Each neural-network layer is clerk deciding what to keep and what to shred. Data processing inequality says clerk can never un-shred document.

Practical consequence: feature engineering sets ceiling. If noisy, badly processed data enters pipeline, data processing inequality guarantees no model can exceed information inside that input. Clean data and extract good features early to raise ceiling for everything downstream. Fancy architecture cannot recover information destroyed before arrival.

Skip connections in ResNets make sense through this lens. Direct path from earlier layers to later ones lets information bypass potentially destructive transformations. They raise information floor of deep networks.

The Information Bottleneck

Naftali Tishby and colleagues formalized key idea: goal of learned representation is to compress input while preserving what matters about target. Given input X, representation T, and target Y, information bottleneck objective minimizes:

I(X; T) - β · I(T; Y)

First term says: compress, reduce how much T remembers about X. Second says: keep parts that predict Y. Parameter β controls tradeoff. High β means prioritize prediction accuracy. Low β means prioritize compression.

In 2017, Shwartz-Ziv and Tishby proposed two distinct phases in deep-network training, visible on information plane plotting I(T; X) vs I(T; Y) for each layer over training. First comes fitting phase: mutual information with target Y rises as network learns prediction. Then comes compression phase: mutual information with input X falls as network forgets irrelevant details.

Open question: is this compression phase universal? Evidence is strong for networks with saturating activations like tanh, where gradient noise naturally drives compression. ReLU networks are murkier: some studies find compression, others do not. Information bottleneck remains powerful lens for understanding deep learning, but universality debate is real and still ongoing.

What is not debatable: good representations compress. Embedding that preserves everything is useless because it is input. Embedding that throws away everything is also useless because it is constant. Sweet spot is keeping what task needs and discarding rest. Information bottleneck formalizes that tradeoff.

Fisher Information

Everything so far — entropy, cross-entropy, KL divergence, mutual information — measures properties of distributions. Fisher information asks different question: how sensitive is distribution to changes in its parameters?

I(θ) = E[(∂/∂θ log p(x; θ))²]

This is expected squared gradient of log-likelihood. Equivalent form is negative expected second derivative: curvature of log-likelihood surface. High Fisher information at parameter value θ means likelihood landscape has sharp curvature there. Data is very informative about θ, and small changes in θ cause large changes in how data looks.

Crude visualization: stand on hilly landscape. Fisher information tells you how steep hills are around you. If terrain is sharply curved, meaning high Fisher information, you can pinpoint position precisely because small step changes altitude noticeably. If terrain is flat, meaning low Fisher information, you can wander far without view changing. Data then tells you little about exact position.

Cramér-Rao bound makes this precise: no unbiased estimator of θ can have variance lower than 1/I(θ). More Fisher information means tighter bounds on parameter accuracy. This is fundamental limit on learning from data. Problem sets it, not weakness in algorithm.

In deep learning, Fisher information powers natural gradient descent. Ordinary gradient descent treats parameter space as flat, so step size 0.01 means same thing everywhere. Parameter space is not flat. Small weight change in one layer might barely affect output, while same change elsewhere might be catastrophic. Fisher information matrix captures that curved geometry. Natural gradient descent adjusts gradient by inverse Fisher information matrix, so step sizes are measured by change in model output distribution, not raw parameter movement. This drives K-FAC and similar second-order optimizers.

Elastic Weight Consolidation (EWC) pushes idea further. When training on task B after task A, EWC uses diagonal of Fisher information matrix from task A to identify important parameters. High Fisher information means data was very sensitive to that parameter. Important parameters get penalty for changing during task B training: L_EWC = L_B + (λ/2) Σ Fᵢ(θᵢ - θ*ᵢ)². Parameters old task did not need much, with low Fᵢ, can change freely. Parameters old task depended on, with high Fᵢ, stay pinned. Result is principled defense against catastrophic forgetting, grounded in information geometry.

The Bigger Picture: Why ML Speaks Information Theory

We started with surprise: unlikely events carry more information than likely ones. From there came entropy, average surprise; cross-entropy, cost of wrong model; and KL divergence, pure waste from being wrong. Everyday loss function turned out to be information-theoretic quantity. Then came mutual information, shared knowledge between variables; data processing inequality, which says destroyed information stays destroyed; information bottleneck, where good representations compress intelligently; and Fisher information, which measures sensitivity to parameter changes.

These are not separate tools sharing old math. They are faces of one insight: learning is compression. Model that generalizes well finds compact description of patterns in data, keeping signal and discarding noise.

Minimum Description Length (MDL) principle makes this explicit: best model compresses data most efficiently. Total cost is bits for model description plus bits for data given model. Complex model with many parameters costs more to describe but may encode data cheaply. Simple model is cheap to describe but may encode data poorly. MDL says minimize sum. This is bias-variance tradeoff stated in bits. L1/L2 regularization, dropout, and pruning all reduce model description length.

Variational inference turns KL divergence into computational tool. Find distribution q(z) that minimizes D_KL(q(z) ‖ p(z|x)), making approximate posterior as close as possible to true posterior. True posterior is intractable, so we maximize Evidence Lower Bound (ELBO) instead. Math is equivalent. This powers VAEs. Normalizing flows and diffusion models can be viewed as making q(z) more flexible to shrink KL gap.

Recent theoretical work uses mutual information to bound generalization error. Intuition: if learning algorithm output has high mutual information with specific training set, it has memorized training-specific noise and will generalize poorly. Data augmentation, dropout, and noise injection help partly because they reduce this memorization, lowering mutual information between learned model and particular training examples.

Next time you see KL divergence term in loss function, hear "information gain" in decision tree, or wonder why cross-entropy loss plateaus above zero, do not just nod and move on. Recognize information-theoretic machinery underneath. Keep solid mental model of what it is doing and why.

import numpy as np
from scipy.stats import entropy as sp_entropy

def entropy_bits(probs):
    """Entropy in bits."""
    return sp_entropy(probs, base=2)

def cross_entropy(p, q):
    """Cross-entropy H(P, Q) in nats."""
    return -np.sum(p * np.log(q))

def kl_div(p, q):
    """KL divergence D_KL(P || Q) in nats."""
    mask = p > 0
    return np.sum(p[mask] * np.log(p[mask] / q[mask]))

def jsd(p, q):
    """Jensen-Shannon divergence in nats."""
    m = 0.5 * (p + q)
    return 0.5 * kl_div(p, m) + 0.5 * kl_div(q, m)

def info_gain(parent, left, right):
    """Information gain for a binary split (decision tree)."""
    n = len(parent)
    h_parent = entropy_bits(np.bincount(parent) / n)
    h_left = entropy_bits(np.bincount(left) / len(left))
    h_right = entropy_bits(np.bincount(right) / len(right))
    return h_parent - (len(left)/n * h_left + len(right)/n * h_right)

# --- Demo: the full information theory toolkit ---

P = np.array([0.7, 0.2, 0.1])    # true weather distribution
Q = np.array([0.5, 0.3, 0.2])    # model's prediction

print("=== Core Quantities ===")
print(f"Entropy H(P):           {sp_entropy(P):.4f} nats")
print(f"Cross-entropy H(P,Q):   {cross_entropy(P, Q):.4f} nats")
print(f"KL divergence D(P||Q):  {kl_div(P, Q):.4f} nats")
print(f"Verify H(P,Q) = H(P) + KL: {sp_entropy(P) + kl_div(P, Q):.4f} nats")
print(f"JSD(P, Q):              {jsd(P, Q):.4f} nats")

print("\n=== Decision Tree Split ===")
labels = np.array([0, 0, 0, 0, 0, 1, 1, 1])  # 5 sunny, 3 rainy
left = np.array([1, 1, 1, 0])   # high-humidity group
right = np.array([0, 0, 0, 0])  # low-humidity group
print(f"Information gain: {info_gain(labels, left, right):.3f} bits")
