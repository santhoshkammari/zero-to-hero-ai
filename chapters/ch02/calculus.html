TL;DR

Neural network learns by nudging parameters toward smaller loss. Direction comes from derivatives. Chain rule sends that signal backward through whole network — backpropagation. Gradients point uphill on loss surface; we walk downhill. Taylor expansions justify gradient descent: trust local linear loss approximation. Reverse-mode automatic differentiation makes this tractable, so PyTorch can differentiate through millions of parameters in one backward pass. Section builds all that from one tiny example.

The Nudge Test: What Is a Derivative?

Imagine house-price model from square footage. One weight, w. Prediction is: predicted_price = w × square_footage. Start with w = 100; prediction is way off. Loss — gap between prediction and reality — might be 5,000.

Now nudge w. Increase from 100 to 100.001. Run prediction again. Loss changes to 4,998. Nudge of 0.001 in w caused change of −2 in loss. Ratio of those changes — output wobble per unit input wobble — is derivative. Formally, if loss is function L(w), derivative is L'(w) = lim(h→0) [L(w+h) − L(w)] / h. Geometrically: slope of tangent line at current position.

Call this nudge test. Poke one knob, watch output. Derivative measures output sensitivity to poke. Large positive derivative means "turn knob up, loss shoots up — turn it down." Negative derivative means opposite. Derivative near zero means loss barely notices this knob right now.

Nudge test is why ML learns. Training means minimizing loss function that depends on millions of parameters. Derivative of loss with respect to each parameter tells exact adjustment direction. Without derivatives, training is blind — random knob flailing, hoping loss goes down.

Problem: model has one weight. Real network has millions. Nudge test, one parameter at time, takes forever. Need faster tools.

Differentiation Rules That Actually Matter

No need to return to limit definition every time. Handful of rules covers almost everything. Power rule says: if f(x) = x^n, then f'(x) = n × x^(n−1). Derivative of x³ is 3x². Derivative of x (x¹) is 1. Derivative of constant is zero — constants do not move when you nudge input. Back to house-price model: if loss is L(w) = (w − 150)², power rule (plus chain rule, coming next) gives L'(w) = 2(w − 150). At w = 100, that is −100 — strong signal to increase w.

Product rule handles multiplication of two functions: if f(x) = g(x) × h(x), then f'(x) = g'(x)×h(x) + g(x)×h'(x). Differentiate one factor at time and add results. Quotient rule handles division, but in practice you can rewrite quotient as product with negative exponent and use product rule instead. These rules are vocabulary of differentiation — small, learnable, enough to break apart nearly any ML function.

But these rules alone handle single-layer functions. House-price model is one multiplication. Neural network is dozens of nested functions, layer after layer. To differentiate through nesting, need something stronger.

The Chain Rule — The Single Most Important Idea

For long time, backpropagation looked like separate clever algorithm. It is chain rule. Same chain rule from introductory calculus. Realizing that changes everything.

Build toward it. Extend house model to two steps. First compute raw score: z = w × square_footage. Then squash through activation function for final output: a = sigmoid(z). Loss depends on a, which depends on z, which depends on w. Functions nested inside functions — composition.

Chain rule tells how to differentiate composition. If y = f(g(x)), then dy/dx = f'(g(x)) × g'(x). Differentiate outer function evaluated at inner function, then multiply by derivative of inner function. In Leibniz notation: dy/dx = (dy/du) × (du/dx), where u = g(x). Ask: "how sensitive is final output to my knob?" Break question into hops. How sensitive is output to intermediate value? How sensitive is intermediate value to knob? Multiply sensitivities together.

For two-step model: dL/dw = dL/da × da/dz × dz/dw. Three factors. Each is local derivative, easy to compute. dL/da tells how loss reacts to activation output. da/dz tells how activation reacts to input. dz/dw tells how raw score reacts to weight. Chain rule multiplies local sensitivities into global sensitivity — how loss at very end reacts to weight at very beginning.

Now scale up. Neural network with 50 layers is 50 nested functions, with loss function on top. Chain rule lets you peel them apart one layer at time, multiplying local derivatives as you go. That multiplication — from output back to each parameter — is backpropagation. Nothing more. Nothing less. Chain rule, applied systematically from end back to beginning. Formal details come later; essence is here.

Limitation is subtle but important. Chain rule gives exact derivatives, but says nothing about behavior when you multiply dozens together. Question is: what happens when you chain 50 or 100 local derivatives? That causes some of deep learning's worst headaches. We get there soon.

Partial Derivatives and the Gradient Vector

House-price model has limped along with one weight. Time for second feature: number of bedrooms. Now prediction is predicted_price = w₁ × square_footage + w₂ × bedrooms. Two knobs to tune. Loss depends on both: L(w₁, w₂).

Partial derivative is nudge test applied to one knob while all others stay fixed. ∂L/∂w₁ means: nudge w₁ a tiny amount, freeze w₂, measure loss change. Compute it like ordinary derivative, treating other variable as constant. If L(w₁, w₂) = (w₁×1200 + w₂×3 − 350000)², then ∂L/∂w₁ = 2×1200×(w₁×1200 + w₂×3 − 350000). Partial derivative for w₂ is similar, but with 3 instead of 1200. Notice w₁'s partial derivative is much larger — loss is far more sensitive to square-footage weight than bedroom weight. That fits intuition.

Now stack partial derivatives into vector: ∇L = [∂L/∂w₁, ∂L/∂w₂]. That vector is gradient. It points in direction of steepest ascent of loss function — direction that increases loss fastest. Its magnitude tells how steep slope is.

Key insight for optimization: to minimize loss, walk opposite gradient. If gradient points northeast, step southwest. That is gradient descent in one sentence. Directional derivative tells rate of change in any direction — dot product of gradient with chosen direction vector. Gradient direction gives maximum directional derivative. It is steepest way uphill, so negative gradient is steepest way downhill.

Think hiking in fog. You cannot see valley floor, but you can feel slope under your feet. Gradient is that slope. Step downhill, feel new slope, step again. Eventually you reach bottom. Analogy returns later — loss terrain is more interesting than simple valley.

But gradient here has two components. Real network has millions. Computing gradient means computing millions of partial derivatives. Brutally expensive if done separately. Chain rule rescues us, but first we need case where output is not one number.

Jacobians and Hessians: When One Number Isn't Enough

So far, loss has been one number — scalar. Gradient is vector of partial derivatives, one per parameter. But what about layer that takes vector in and produces vector out? Softmax layer, for example, takes vector of raw scores and produces vector of probabilities. Every output depends on every input. One number cannot capture that. Need matrix.

Jacobian matrix is that matrix. If function f maps n inputs to m outputs, Jacobian is m×n matrix where entry (i,j) is ∂f_i/∂x_j — how much output i changes when you nudge input j. Each row is gradient of one output component. During backpropagation, when gradients must move backward through vector-valued layer, you multiply by Jacobian (more precisely, its transpose). In practice, frameworks never build full Jacobian explicitly — they compute vector-Jacobian products (VJPs), giving exact row you need without materializing whole matrix. That engineering choice makes deep learning at scale possible.

Hessian matrix captures something else: curvature. Entry (i,j) is ∂²L/(∂w_i ∂w_j) — second-order partial derivative. Gradient tells slope of terrain; Hessian tells how slope itself changes. Is ground curving into bowl (local minimum)? Ridge (saddle point)? Dome (local maximum)? Eigenvalues of Hessian answer this: all positive means bowl, all negative means dome, mix means saddle point.

Hessians are intimidating for good reason. They are n×n for n parameters, so model with 100 million parameters has Hessian with 10¹⁶ entries. You cannot store it, much less invert it. That is why deep learning mostly uses first-order methods — gradient is O(n), and that is all you can afford. Second-order information (curvature) is better in theory, giving direction and suitable step size. But computational cost makes it luxury. Approximations like L-BFGS and Kronecker-factored methods capture some curvature cheaply and work well for smaller models. For modern deep networks, gradient alone must be enough.

The Functions You'll Meet Everywhere

Before backpropagation through real network, know derivatives of functions networks are built from. Only handful matter. Each has story.

Sigmoid

Sigmoid function, σ(z) = 1/(1 + exp(−z)), squashes any real number into range (0, 1). For years it was default activation function — output looks like probability. Its derivative has elegant property: σ'(z) = σ(z) × (1 − σ(z)). Derivative is expressed entirely in terms of function value already computed during forward pass. Computationally convenient — backward pass is nearly free.

But numbers hurt. Maximum value of σ(z) × (1 − σ(z)) is 0.25, at z = 0. For any other input, derivative is smaller. Now stack 20 sigmoid layers. Gradient flowing backward is product of 20 numbers, each at most 0.25. That product is 0.25²⁰ ≈ 10⁻¹². Gradient effectively disappears. This is vanishing gradient problem, and reason sigmoid fell out of favor for hidden layers. It still appears at output layer for binary classification, where probability output is exactly what you want.

ReLU

ReLU (Rectified Linear Unit) is f(z) = max(0, z). Its derivative is 1 when z > 0 and 0 when z < 0. (At z = 0 exactly, derivative is technically undefined, but implementations pick 0 or 1 — it does not matter because hitting exactly zero has probability zero for continuous inputs.) ReLU was fix for vanishing gradients. When neuron is active (z > 0), gradient passes through unattenuated — multiplied by 1. No shrinkage.

Tradeoff: if neuron's input stays negative, gradient is always zero and neuron stops learning. Dead neuron. This is dying ReLU problem. Variants like Leaky ReLU (small positive slope, say 0.01, for negative inputs) keep small gradient flowing even for inactive neurons, keeping them alive. In hiking analogy, ReLU makes trail either flat or sloped — never gradual leveling-off that sigmoid creates.

Tanh

Tanh maps inputs to (−1, 1), centered at zero. Its derivative is tanh'(z) = 1 − tanh(z)². Like sigmoid, it saturates at extreme values and causes vanishing gradients. But because output is zero-centered — negative inputs give negative outputs, positive inputs give positive outputs — it helps gradient flow compared to sigmoid, whose output is always positive. Tanh was go-to activation before ReLU took over. You still find it inside LSTM and GRU gates, where range (−1, 1) serves specific purpose.

Softmax

Softmax function converts vector of raw scores (logits) into probability distribution. For vector z of length K: softmax(z)_i = exp(z_i) / Σ_j exp(z_j). Every output is positive, and outputs sum to one. Jacobian of softmax is dense K×K matrix: ∂softmax_i/∂z_j = softmax_i × (δ_ij − softmax_j), where δ_ij is 1 when i = j and 0 otherwise. In practice, you almost never compute this Jacobian directly, because softmax is usually paired with cross-entropy loss, and combined gradient collapses into something beautifully simple. We derive that later — one of most satisfying results in ML calculus.

One practical hazard: computing exp(z_i) for large z_i causes overflow. Log-sum-exp trick fixes this by subtracting max before exponentiating: log(Σ exp(z_i)) = max(z) + log(Σ exp(z_i − max(z))). After subtraction, largest exponent is zero, so nothing overflows. Every production framework does this internally.

Integrals: The Other Half of Calculus

Derivatives tell rates of change. Integrals tell accumulation — adding infinitely many infinitely small pieces. Definite integral ∫[a,b] f(x) dx computes area under curve f(x) between a and b. If derivatives are speedometer, integrals are odometer.

ML uses integrals constantly, even when not written out. Every probability density function (PDF) must integrate to 1 over entire domain — that makes it valid distribution. Probability that random variable lands in some range is integral of PDF over that range. Expected value of function g(X) under distribution p(x) is E[g(X)] = ∫ g(x) p(x) dx. Loss functions in probabilistic models are expectations. Marginal distributions require integrating out variables. Bayesian inference requires integrating over entire parameter space to compute evidence.

Fundamental Theorem of Calculus connects both halves: differentiation and integration are inverse operations. If you can find function whose derivative equals integrand, you can compute integral exactly by evaluating function at endpoints. When closed-form antiderivative exists, integration is straightforward. When it does not — and in ML, often does not — use Monte Carlo approximation: replace integral with average over random samples. This is workhorse behind variational inference, reinforcement learning policy gradients, and much of probabilistic ML.

In higher dimensions, multiple integrals compute volumes and probabilities over joint distributions. To get marginal p(x) from joint p(x, y), integrate out y: p(x) = ∫ p(x, y) dy. In Bayesian inference, computing posterior means integrating over parameter space that may have millions of dimensions. Cost grows exponentially with dimension — curse of dimensionality. That challenge drives variational inference and Markov Chain Monte Carlo methods. Derivatives dominate day-to-day neural-network training, while integrals dominate theory of probabilistic models. Both halves matter, for different reasons.

Taylor Series → Gradient Descent: Calculus Produces an Algorithm

All pieces come together here. Calculus hands us algorithm.

Taylor series approximates function as sum of polynomial terms built from derivatives at point: f(x) ≈ f(a) + f'(a)(x−a) + f''(a)(x−a)²/2! + ... More terms give better approximation near a. Here we need only first term — linear approximation.

Take loss function L(w). Stand at current weights w and ask where to step next. First-order Taylor expansion says: L(w + δ) ≈ L(w) + ∇L(w) · δ. Loss at nearby point is approximately current loss plus dot product of gradient with step. We want δ that makes L(w + δ) as small as possible. Dot product ∇L(w) · δ is most negative when δ points opposite gradient. So set δ = −α × ∇L(w), where α is small positive number called learning rate.

Update rule becomes: w_new = w − α × ∇L(w). That is gradient descent. Derived directly from linear approximation of loss.

Learning rate α controls how far we trust linear approximation. Too large, and step goes beyond valid region — loss may increase, training may oscillate or diverge. Too small, and progress crawls, wasting thousands of iterations. Think of Taylor expansion as tangent plane touching loss surface at one point. Plane guides few nearby steps, but wander too far and real surface curves away. Learning rate keeps you inside zone where approximation holds.

import numpy as np

def gradient_descent(grad_f, x0, lr=0.01, steps=100):
    x = x0.copy()
    for _ in range(steps):
        x = x - lr * grad_f(x)
    return x

# Our house price model: L(w1, w2) = (w1*1200 + w2*3 - 350000)^2
def grad_L(w):
    residual = w[0]*1200 + w[1]*3 - 350000
    return np.array([2*1200*residual, 2*3*residual])

w_start = np.array([0.0, 0.0])
w_final = gradient_descent(grad_L, w_start, lr=1e-10, steps=5000)
print(f"Learned weights: w1={w_final[0]:.1f}, w2={w_final[1]:.1f}")
# w1 dominates because the loss is far more sensitive to the square footage weight

Second-order Taylor expansion adds curvature term: L(w + δ) ≈ L(w) + ∇L · δ + ½ δᵀHδ, where H is Hessian. Setting gradient of this quadratic to zero gives Newton's method: δ = −H⁻¹ × ∇L. Newton's method accounts for curvature and converges much faster — quadratically near optimum, versus gradient descent's linear convergence. But inverting Hessian costs O(n³) for n parameters. For network with 100 million parameters, that is 10²⁴ operations. Unthinkable. That is why deep learning stops at first-order methods. Gradient is O(n) — all you can afford at scale. Quasi-Newton methods like L-BFGS approximate Hessian cheaply and work beautifully for smaller models, like logistic regression or small neural networks.

Convex Functions: The Easy Case

Function is convex if line segment connecting any two points on graph lies above graph. Picture bowl. Convex functions make optimization almost trivial: every local minimum is global minimum. Only one bowl, and gradient descent will find bottom. Linear regression's squared loss is convex. Logistic regression's loss is convex. That is why these models train reliably — no traps.

Neural-network losses are not convex. Loss landscape has countless saddle points, flat regions, and local minima. Yet gradient descent still finds good solutions. Convexity explains why classical ML problems are easier: they are convex. Neural networks work despite non-convexity, and reasons why — wide minima, high-dimensional geometry, stochastic noise — remain active research. Large networks still find good solutions reliably. No complete answer yet.

Constrained Optimization

Lagrange multipliers handle optimization with constraints. When you want to minimize f(x) subject to g(x) = 0, introduce new variable λ and find points where ∇f = λ × ∇g. Geometric picture: at optimum, gradient of objective is parallel to gradient of constraint. If not parallel, you could slide along constraint and still improve objective. In ML, Lagrange multipliers are backbone of support vector machine derivation — whole SVM dual formulation falls out of this framework. Extension to inequality constraints gives Karush-Kuhn-Tucker (KKT) conditions, adding rule: either constraint is active and multiplier is positive, or constraint is slack and multiplier is zero.

Backpropagation: The Chain Rule, Industrialized

Chain Rule Through Layers

Backpropagation is algorithm for computing gradient of loss with respect to every parameter in neural network. Despite mystique, it is chain rule applied systematically — nothing more. Consider network with L layers. Forward pass computes chain: z₁ = W₁x + b₁, a₁ = activation(z₁), z₂ = W₂a₁ + b₂, a₂ = activation(z₂), and so on up to loss. Each layer is function whose output feeds next.

Backward pass starts at loss and works in reverse. At each layer, compute two things: local gradient with respect to layer's parameters (store for update step), and local gradient with respect to layer's input (pass backward to previous layer). Incoming gradient from above gets multiplied by each local gradient — chain rule doing its job. When you reach bottom, every parameter has gradient.

Remarkable part is cost. One forward pass computes loss, one backward pass computes all gradients. Backward pass costs roughly twice forward pass — regardless of parameter count. For network with 100 million parameters, that gives 100 million gradients in one shot. Compare with nudge test, which would require 100 million separate forward passes.

Computational Graphs

Modern frameworks do not think of networks only as layer sequences. They treat forward pass as computational graph: directed acyclic graph where each node is elementary operation (add, multiply, exp, matrix multiply) and edges carry tensors. During forward pass, framework records every operation in this graph. During backward pass, it traverses graph in reverse, applying chain rule at each node.

Original TensorFlow built graph once and reused it — static graph. You had to define entire computation upfront, which made debugging painful. PyTorch took different approach: dynamic graphs, rebuilt from scratch on every forward pass. That means you can use normal Python if-statements and for-loops, and graph adapts. You can print intermediate values. You can set breakpoints. Dynamic graphs made prototyping much easier, so they became standard — even TensorFlow later added eager execution mode. Flexibility of dynamic graphs is one reason PyTorch dominates research.

When Gradients Go Wrong: Vanishing and Exploding

Remember hiking analogy — follow slope downhill to find valley. Now make terrain strange. In some directions, slope gets flatter and flatter as you walk, until you can barely feel it under your feet. You get stuck on almost-flat plateau, unable to tell which way is downhill. That is vanishing gradient problem.

When network has many layers, gradient at each layer is product of local derivatives from every layer above it. If each local derivative is less than 1, product shrinks exponentially. Sigmoid activations are worst offender — maximum derivative is 0.25. Twenty sigmoid layers give gradient multiplied by 0.25²⁰ ≈ 10⁻¹². Early layers receive signal so faint they cannot learn. Loss flatlining after many epochs often traces back to vanishing gradients.

Fixes are well-established. ReLU activations pass gradients through at full strength (multiplied by 1) when active. Residual connections add shortcut that lets gradients skip layers entirely — highway for gradient signal. Careful initialization (He initialization for ReLU, Xavier for tanh) sets initial weights so activation variance stays roughly constant across layers, preventing gradients from shrinking or growing from start.

Mirror-image problem is exploding gradients: if local derivatives exceed 1, product grows exponentially. Weights get enormous updates, loss spikes, training collapses. Standard fix is gradient clipping: if gradient norm exceeds threshold, scale it down proportionally before update. This matters especially in recurrent neural networks, where backpropagation through time (BPTT) unrolls network across every timestep. Sequence of length 100 means gradient passes through 100 copies of same weight matrix — 100 matrix multiplications. If largest eigenvalue of that weight matrix exceeds 1, gradient explodes. If it is less than 1, gradient vanishes. That razor-thin balance is why vanilla RNNs struggle with long sequences, and why LSTMs and GRUs were invented — their gating mechanisms explicitly control gradient flow, opening and closing gates to let information and gradients through selectively.

Automatic Differentiation: How the Machine Actually Does It

For years, PyTorch looked like symbolic math under hood — rewrite loss formula, differentiate on paper. Wrong. Truth is more clever.

Computer can compute derivatives three ways. Symbolic differentiation manipulates expressions using rules (power rule, chain rule) to produce new derivative expression. SymPy and Mathematica do this. Result is exact, but expressions can explode in size — derivative may be far more complex than original function. Approach breaks down for functions defined by programs with loops and conditionals.

Numerical differentiation uses finite differences: f'(x) ≈ [f(x+h) − f(x−h)] / (2h). Dead simple. Works for any function you can evaluate. Needs no special implementation. But for n parameters, you need 2n function evaluations — two per parameter. For model with million parameters, that is two million forward passes. Unacceptable for training. Excellent for testing, though: numerical gradients can verify analytical gradients, a technique called gradient checking.

import numpy as np

def gradient_check(f, analytical_grad, x, h=1e-5):
    """Verify analytical gradients against numerical approximation."""
    numerical = np.zeros_like(x)
    for i in range(len(x)):
        x_plus, x_minus = x.copy(), x.copy()
        x_plus[i] += h; x_minus[i] -= h
        numerical[i] = (f(x_plus) - f(x_minus)) / (2 * h)
    
    analytical = analytical_grad(x)
    diff = np.linalg.norm(numerical - analytical) / (np.linalg.norm(numerical) + np.linalg.norm(analytical) + 1e-8)
    print(f"Relative difference: {diff:.2e}")  # Should be < 1e-5
    return diff

# Verify: f(x,y) = x^2 + 3xy. Gradient: [2x+3y, 3x]
f = lambda x: x[0]**2 + 3*x[0]*x[1]
grad_f = lambda x: np.array([2*x[0] + 3*x[1], 3*x[0]])
gradient_check(f, grad_f, np.array([1.0, 2.0]))

Forward-Mode vs Reverse-Mode

Automatic differentiation (AD) is third approach, and one that powers deep learning. It does not manipulate expressions and does not use finite differences. Instead, it decomposes function into sequence of elementary operations — add, multiply, exp, log — each with known derivative. Then it applies chain rule through sequence. Result is exact (up to floating-point precision) and efficient.

Forward-mode AD propagates derivatives from inputs to outputs. For each input, you track value and derivative through every operation. To get derivative with respect to all n inputs, you need n forward passes. Efficient when you have few inputs and many outputs.

Reverse-mode AD propagates derivatives backward from outputs to inputs. One backward pass gives derivative of output with respect to every input — all of them, in one shot. If you have many inputs and one output — exactly ML situation, with millions of parameters and one scalar loss — reverse mode is spectacularly efficient.

This asymmetry is why deep learning works at current scale. One forward pass computes loss. One backward pass computes every gradient. Backward pass costs roughly 2–4× forward pass, regardless of parameter count. Network with billion parameters gets full gradient in one backward pass. This is reverse-mode AD. This is what loss.backward() does in PyTorch. And this is why backpropagation is identical to reverse-mode AD applied to computational graph.

import torch

x = torch.tensor([1.0, 2.0], requires_grad=True)

# Forward pass — PyTorch silently builds the computational graph
loss = x[0]**2 + 3*x[0]*x[1]

# Backward pass — reverse-mode AD through the recorded graph
loss.backward()

print(f"Gradient: {x.grad}")  # tensor([8., 3.]) — both gradients in one pass
# Analytical: ∂L/∂x₁ = 2(1) + 3(2) = 8, ∂L/∂x₂ = 3(1) = 3
import torch
import torch.nn as nn

# Back to our running example, now as a proper neural network
model = nn.Sequential(
    nn.Linear(2, 8),    # 2 features → 8 hidden units
    nn.ReLU(),
    nn.Linear(8, 1)     # 8 hidden → 1 output (price)
)

# Fake batch: 4 houses, 2 features each (sq_ft normalized, bedrooms)
x = torch.randn(4, 2)
y = torch.randn(4, 1)

pred = model(x)
loss = ((pred - y)**2).mean()
loss.backward()

# Every parameter now has a .grad — computed via reverse-mode AD
for name, param in model.named_parameters():
    print(f"{name}: shape={param.shape}, grad_norm={param.grad.norm():.4f}")

The Softmax + Cross-Entropy Miracle

This derivation is central to classification, so it gets its own section. Ending is one of best in ML.

Take network that outputs raw logits z. Softmax converts them to predicted probabilities: ŷ_i = exp(z_i) / Σ_j exp(z_j). True label is one-hot vector y — all zeros except 1 at correct class k. Cross-entropy loss is L = −Σ_i y_i × log(ŷ_i). Since y is one-hot, this collapses to L = −log(ŷ_k): negative log-probability assigned to correct class. If model is confident and correct, ŷ_k is close to 1 and loss is close to 0. If model puts low probability on correct class, loss is large.

Now gradient ∂L/∂z_i. You need derivatives through log and softmax. This involves softmax Jacobian, dense K×K matrix — sounds messy. But something beautiful happens. For correct class: ∂L/∂z_k = ŷ_k − 1. For every other class: ∂L/∂z_i = ŷ_i. Combine with one-hot vector: ∇_z L = ŷ − y.

Gradient of cross-entropy loss with respect to logits is predicted probability minus true label. That is it. Jacobian, logarithm, summation — everything cancels into ŷ − y.

This elegance is not accident — cross-entropy loss was designed to pair with softmax. And beauty is not only point; numerical stability is. Computing log(softmax(z)) naively is catastrophically unstable — softmax can produce values near zero, and log of near-zero is negative infinity. Frameworks provide combined log_softmax or cross_entropy that uses log-sum-exp trick internally for stability. Always pass raw logits to loss function. Never compute softmax yourself and then take log. Combined implementation gives elegant ŷ − y gradient and numerical stability in one package.

Advanced Territory

L2 Regularization as Weight Decay

L2 regularization adds penalty (λ/2) × ||w||² to loss. Take derivative: gradient of regularized loss is ∇L(w) + λw. Update rule becomes w_new = w − α(∇L(w) + λw) = (1 − αλ)w − α∇L(w). Factor (1 − αλ) shrinks every weight slightly before each update — hence weight decay. It discourages any single weight from growing too large, acting as soft constraint toward simpler models. For standard SGD, L2 regularization and weight decay are mathematically identical. For adaptive optimizers like Adam, they differ — reason AdamW was introduced to decouple them.

OLS Closed-Form Solution

House-price model, in simplest linear form, has exact solution. Ordinary least squares loss is ||Xw − y||². Take gradient: ∇L = 2XᵀXw − 2Xᵀy. Set to zero: XᵀXw = Xᵀy. If XᵀX is invertible, solution is w = (XᵀX)⁻¹Xᵀy — normal equation. No gradient descent needed; one matrix inversion gives answer. But inverting XᵀX costs O(d³) for d features. When d is large, gradient descent with O(nd) per-step cost is more practical. Closed-form solutions are elegant; iterative methods scale.

Neural ODEs

Residual networks compute h_{t+1} = h_t + f(h_t), which is Euler step for ordinary differential equation dh/dt = f(h). Neural ODEs take this literally — parameterize continuous transformation and solve it with numerical ODE solver. Backward pass uses adjoint method, solving another ODE in reverse to compute gradients with constant memory cost regardless of depth. Neural ODEs enable continuous-depth networks, continuous normalizing flows, and time-series models with irregular sampling intervals.

Variational Calculus

Variational calculus extends optimization from finding optimal numbers to finding optimal functions. In ML, it appears in variational inference: approximate intractable posterior p(z|x) with simpler distribution q(z) by minimizing KL divergence, or equivalently maximizing evidence lower bound (ELBO). Variational autoencoders (VAEs) use this, optimizing encoder and decoder jointly with reparameterization trick to backpropagate through random sampling.

Information Geometry

Information geometry treats probability distributions as points on curved surface (manifold), with Fisher information matrix as metric. Natural gradient descent uses this metric to take steps that respect geometry of distribution space instead of flat Euclidean space, making optimization invariant to model parameterization. Theoretically beautiful, but computationally expensive — Fisher matrix is as big as Hessian.
