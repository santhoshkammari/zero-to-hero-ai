TL;DR

First reaction to most topics: "I'll never need this." Then paper cites Itô's lemma, interview asks about pseudoinverse. Wish I spent 30 minutes earlier. Not syllabus — field guide. Enough to recognize what you're looking at. Enough to know where to dig when it matters.

Linear Algebra

Tensor Decomposition (CP, Tucker)

Matrices handle 2D data. Lots of data refuses 2D: user × item × time in recommendation systems, spatial × spectral × temporal in remote sensing, NN layer weights when stacking attention heads. Flattening 3-way array to matrix loses structure that may matter.

Tensor decompositions are higher-dimensional SVD analog. CP decomposition (CANDECOMP/PARAFAC) writes tensor as sum of rank-one outer products — SVD's more ambitious cousin. Tucker decomposition more general: produces compressed core tensor multiplied by factor matrices along each mode — more flexibility, more parameters.

In practice: compressing convolutional layers (decomposing 4D weight tensor), multi-relational knowledge graph embeddings, signal processing where data is naturally multi-way. Algorithms iterative, decompositions generally not unique, rank selection as unsettled as for matrices — meaning not very.

Kronecker Product

Kronecker product A ⊗ B builds block matrix by replacing each entry aᵢⱼ with scaled matrix aᵢⱼB. Natural algebraic structure for operations acting simultaneously on two spaces — sounds abstract until you see it everywhere.

In GNNs, message-passing often involves adjacency structure ⊗ feature transformations. Multi-task learning: task and input covariance relationship naturally Kroneckerian. Vectorizing matrix equations: vec(AXB) = (BT ⊗ A) vec(X) turns matrix problem into vector problem. K-FAC exploits Kronecker structure to make Fisher matrix inversion tractable.

Matrix Square Root

Matrix square root S of positive semidefinite matrix A satisfies S² = A. Multiple such S exist — want unique symmetric positive semidefinite one, computed via eigendecomposition: diagonalize A, take sqrt of eigenvalues, rotate back.

Appears in neural style transfer: matching sqrt of feature covariance (Gram matrix) between content and style images transfers artistic texture. Also in natural gradient: update direction requires inverse square root of Fisher matrix — expensive enough that implementations approximate it. K-FAC uses Kronecker structure to make approximation cheap enough to run.

Pseudoinverse (Moore-Penrose)

Regular inverse requires square, full-rank matrix. Real problems rarely cooperate on either count. Moore-Penrose pseudoinverse A⁺ handles any shape and rank: gives minimum-norm least-squares solution to Ax = b. If A is full-rank and square, A⁺ = regular inverse — useful special case worth remembering for interviews.

Computed from SVD: invert nonzero singular values, leave zero ones. Intuition: invert "informative" directions of A, ignore directions where A provides no information. Appears in linear regression theory (normal equations), over- and under-parameterized model analysis, initialization schemes needing controlled pseudoinverse-like mappings.

Random Matrix Theory

Surprising fact: large matrix of i.i.d. random entries has eigenvalue distribution that's not random-looking — follows precise deterministic law called Marchenko-Pastur distribution. Bulk eigenvalue spectrum shape depends only on aspect ratio, not specific entries. This is random matrix theory.

Why care? Eigenvalue spectrum of randomly initialized weight matrix should look like Marchenko-Pastur. After training it often doesn't — few eigenvalues "spike" out of bulk, corresponding to directions network actually learned. Diagnostic: compare empirical eigenvalue spectrum to Marchenko-Pastur prediction for same-shape random matrix. Eigenvalues outside bulk = learned structure; inside bulk = noise. Some initialization schemes and implicit regularization analyses build directly on this.

Calculus

Neural ODEs

Residual network update: ht+1 = ht + f(ht). Make layers thinner and more numerous, discrete update approaches continuous ODE: dh/dt = f(h, t, θ). That's Neural ODEs — parameterize derivative of hidden state with NN, use black-box ODE solver to integrate forward.

Memory efficiency genuine. Standard backprop through ODE solver requires storing all intermediate states. Instead, adjoint method solves second ODE backwards to compute gradients — constant memory regardless of solver steps. Cost: slower, less predictable forward passes. Cited in continuous-time dynamics, latent SDEs, irregularly sampled time series where discrete-layer assumption is awkward.

Variational Calculus

Standard calculus: what value of x minimizes f(x)? Variational calculus: what function q(x) minimizes functional F[q]? Functional takes function as input, returns scalar. Setup: F[q] = ∫ L(q(x), q'(x), x) dx; optimal q satisfies Euler-Lagrange equation — "derivative equals zero" condition for functionals.

Most important ML application: variational inference. Writing ELBO, asking "what distribution q(z) minimizes KL(q||p)?" — that's variational calculus. Connection not always explicit in deep learning tutorials, but it's there. Physics-informed NNs, optimal control, normalizing flow theory all use same machinery.

Stochastic Calculus / SDEs

ODEs describe deterministic dynamics. Stochastic differential equations add noise: dX = f(X, t) dt + g(X, t) dW, where dW is infinitesimal Wiener process increment. Brownian motion is nowhere differentiable — breaks ordinary calculus. Itô's lemma replaces chain rule here — has extra second-derivative term from roughness of noise.

Stopped being niche when diffusion models arrived. Score-based models like DDPM and continuous-time extensions are literally SDEs. Forward process adds noise via specific SDE; reverse process removes noise via related SDE involving score function ∇x log p(x). Want to understand why diffusion works mathematically — not just recipe but why — read stochastic calculus.

Implicit Differentiation

Most functions explicit: given x, compute y = f(x). Sometimes y defined as fixed point of g(x, y) = 0 — solve numerically, want gradients through that solve. Implicit differentiation computes dy/dx = −(∂g/∂y)⁻¹ (∂g/∂x) without re-solving or unrolling solver.

Deep equilibrium models define output layer as fixed point of transformation, backpropagate via implicit differentiation. iMAML and other meta-learning algorithms differentiate through inner optimization loops this way — avoids memory explosion from unrolling hundreds of gradient steps. How you backpropagate through "run gradient descent until convergence."

Probability & Statistics

Copulas

Pearson correlation measures only linear dependence — misleading when marginals are non-Gaussian. Two variables can have zero correlation but strong dependence. Correlation says nothing about tail behavior, which is often where things go wrong.

Copulas separate marginals from dependence structure. Sklar's theorem: any joint distribution = copula applied to marginals — copula captures co-movement, stripped of individual shapes. Gaussian copula: dependence structure of multivariate Gaussian with arbitrary marginals. Clayton copula: strong lower-tail dependence — models things failing together when both bad. 2008 financial crisis partly a story of Gaussian copulas for mortgage defaults missing correlated tail events.

Sufficient Statistics

Collecting data to estimate parameter — how much do you need to keep? Sufficient statistic T(X) captures everything about θ in data: distribution of X given T(X) doesn't depend on θ. Mean and variance sufficient for Gaussian. Sum of Bernoulli sample sufficient for success probability.

Matters for two reasons. First: tells you when to safely compress data without losing estimation power — useful for streaming or distributed settings. Second: connects to exponential family — every exponential family has finite-dimensional sufficient statistic, reason that family is mathematically tractable. Underlies Rao-Blackwell theorem: improve any unbiased estimator by conditioning on sufficient statistic.

Exponential Family

Gaussian, Bernoulli, Poisson, Gamma, Beta, Dirichlet — unrelated-looking until written in common form. Every exponential family member: p(x | η) = h(x) exp(η · T(x) − A(η)), where η are natural parameters, T(x) sufficient statistic, A(η) log-partition function normalizing everything. Pattern unlocked: gradients, conjugate priors, MLE all follow same template.

Log-partition function A(η) convex; gradient gives mean of sufficient statistic: E[T(x)] = ∇A(η). Fitting exponential family = moment matching: match model's expected sufficient statistics to empirical ones. GLMs, variational inference with structured approximations, natural gradient theory all live here. Most useful unifying framework in classical statistics.

Survival Analysis

Survival analysis handles time-to-event outcomes: churn, bearing failure, patient relapse. Often can't observe outcome for everyone — study ends, customer still active, patient still alive. This is censoring. Ignoring censored observations biases every estimate.

Methods handle this gracefully. Kaplan-Meier estimator: nonparametric survival curve accounting for censoring at each time step. Cox proportional hazards model: estimates covariate effects on relative event risk without assuming baseline hazard shape — semiparametric, elegant, practically useful. Deep survival models like DeepSurv graft NN onto Cox framework. Building churn or predictive maintenance models without accounting for censoring? Estimates wrong in systematic direction.

Causal Discovery (PC, FCI)

Most ML models learn X and Y are associated, not that X causes Y. Fine sometimes. But for understanding mechanisms, designing interventions, or building models that generalize under distribution shift — correlation not enough. Causal discovery algorithms learn actual directed causal graph from observational data — no experiments.

PC algorithm: start with fully connected undirected graph, remove edges where conditional independence found, orient edges via orientation rules. FCI: extends PC to handle latent confounders. Neither gives certainty — gives Markov equivalence class of graphs consistent with conditional independence tests. Can sometimes identify impossible causal directions even without unique true graph. Useful for ruling out wrong stories.

Optimization

Mirror Descent

Gradient descent assumes Euclidean parameter space and Euclidean distance as right measure of movement. Not always appropriate. Parameters on probability simplex: L2 gradient step leaves simplex, projection required. Projection wasteful, breaks natural geometry.

Mirror descent replaces Euclidean distance with Bregman divergence matching constraint set geometry. For simplex, natural choice is KL divergence — gives exponentiated gradient algorithm, stays on simplex without projection. Foundation of online learning theory; clarifies why different algorithms suit different geometries. Online mirror descent in regret analyses = this.

Proximal Operators

Gradient descent needs smooth objective. Add L1 regularization — creates exactly-zero weights — and objective has non-differentiable term. Subgradients exist but converge slowly. Proximal operators: split into smooth and non-smooth parts, gradient step on smooth, proximal step on non-smooth.

Proximal operator of g: proxg(v) = argmin_x [g(x) + ½‖x − v‖²]. For L1: this is soft thresholding — push values toward zero by λ, set to exactly zero if within λ of origin. Why LASSO produces sparse solutions. ISTA and FISTA (accelerated) alternate gradient steps on smooth loss with proximal steps on regularizer. Right tool for combining smooth losses with structured sparsity penalties.

Frank-Wolfe

Constrained optimization usually: project unconstrained gradient step onto feasible set. Projection can be expensive — matrix norm balls, trace norm constraints, complex polytopes may require solving convex program each step.

Frank-Wolfe algorithm (conditional gradient) avoids projection. Finds constraint set vertex most aligned with negative gradient — linear minimization, often much cheaper. Moves toward that vertex via convex combination. Iterates naturally feasible at every step. Cost: sublinear convergence in general. But where projection expensive and constraint vertices easily computable — nuclear norm balls in matrix completion — Frank-Wolfe often wins in practice.

Alternating Minimization

Some optimization problems factor into two variable groups where fixing one makes the other easy. Alternating minimization exploits this: fix one, optimize other, swap, repeat. Not directly minimizing joint objective, but works on surprising number of problems.

Non-negative matrix factorization: given W and H, solving for H with W fixed is least-squares, vice versa. EM alternates E-step (compute expected sufficient statistics) and M-step (maximize expected log-likelihood). K-means alternates assignment and centroid recomputation. None guaranteed global optima; all converge to local optima reliably — in practice often good enough. Pattern worth recognizing: suggests clean implementation for problems that look joint and hard.

Information Theory

Rate-Distortion Theory

Lossless compression floor: can't compress below Shannon entropy. Accept reconstruction error, go further. Rate-distortion theory characterizes this tradeoff: for distortion level D, minimum bits R(D) needed to encode source. R(D) is rate-distortion function — fundamental limit for lossy compression.

Matters for ML because representation learning is implicitly compression. Information bottleneck frames learning as finding compressed representation Z of input X retaining max information about Y — rate-distortion with semantic distortion measure. Neural compression codecs explicitly target rate-distortion bound. When someone claims "compact" or "efficient" representation, rate-distortion theory makes it precise.

Channel Capacity

Shannon's noisy channel coding theorem: for any noisy channel, maximum rate C (channel capacity) exists. Below C: transmit with arbitrarily low error probability, no matter how noisy. Above C: errors unavoidable. Boundary sharp.

For ML: information-theoretic generalization bounds, capacity of NNs as communication channels, communication-efficient distributed learning with bandwidth-limited gradient transmission. Also genuinely beautiful result that justifies information theory as its own discipline.

Rényi Entropy

Shannon entropy is one member of parametric family. Rényi entropy of order α: H_α(X) = (1/(1−α)) log Σ p_i^α. α → 1: Shannon entropy. α = 0: log of support size. α = 2: collision entropy (probability two independent draws match). α → ∞: min-entropy, determined by most likely outcome.

Min-entropy: conservative uncertainty measure — "how uncertain is worst case?" not "how uncertain is average case?" Right measure for cryptographic and privacy applications where adversary picks best attack. Differential privacy's composition theorems and moments accountant use Rényi divergences to track privacy budget across adaptive queries. Implementing or analyzing differential privacy? Rényi divergences appear repeatedly.

f-Divergences

KL divergence: most used distribution discrepancy measure, but one choice among many. f-divergence D_f(P‖Q) = E_Q[f(dP/dQ)] defined by convex f with f(1) = 0. f(t) = t log t: forward KL. f(t) = −log t: reverse KL. f(t) = (t−1)²: χ². f(t) = |t−1|/2: total variation. All measure distributional mismatch but penalize different discrepancy types.

Became practical when GANs arrived. Original GAN minimizes Jensen-Shannon divergence (an f-divergence). f-GAN shows any f-divergence recoverable as minimax game objective. Different choices → different training dynamics and mode-dropping behavior. Understanding f-divergences: unified view of why GAN variants exist and what tradeoffs they make — not ad hoc modifications.

Numerical Computing

Mixed Precision Beyond fp16

fp16: halve memory, roughly same accuracy with careful gradient scaling. Industry went further. INT8 inference: weights in 8-bit integers — quantization rounds to discrete grid, accuracy loss small with careful calibration. INT8 often faster due to higher hardware throughput than fp16.

More recent: NF4 (4-bit NormalFloat), introduced with QLoRA. NF4 designed for approximately normal weight distributions: places 16 quantization levels non-uniformly, denser near zero where most weights cluster. Better reconstruction than uniform 4-bit. QLoRA: NF4 for frozen base weights, full precision for trainable LoRA adapters — fine-tune 65B models on single consumer GPU. Trend clear: go as low in bit-width as possible without measurable task degradation. NF4 + INT8 are current practical endpoints.

Kahan Summation

Floating-point addition not associative. Naïve loop summing 10⁶ numbers accumulates rounding error — running sum grows large, small addends rounded away. Usually fine for ML workloads. For gradient accumulators, long-sequence statistics, or summing many small numbers: silently degrades precision in hard-to-debug ways.

Kahan summation fixes this: keep compensation variable c tracking rounding error from last addition. Each step: adjust new term by c before adding, compute new c from what got lost. Result: near full-precision sums over millions of additions, cost of handful of extra ops per step. Disproportionately clever for how simple it is to implement.

Matrix-Free Methods

Hessian of NN loss has as many rows and columns as parameters. 100M parameters = 10¹⁶-element matrix — can't store it. But Hessian-vector products cheap: forward + backward pass gives gradient; second backward pass through gradient gives Hv for any v. Key operation for matrix-free methods.

Conjugate gradient solves linear systems using only matrix-vector products, never matrix itself. Lanczos iteration computes extreme eigenvalues and eigenvectors same way. Randomized SVD estimates top singular vectors via few matrix-vector products. Essential for second-order optimization (K-FAC, Hessian-free), loss landscape analysis (Hessian spectrum), large-scale kernel methods where kernel matrix is too large to materialize. Abstraction: "I have operator, not matrix" — powerful.

Structured Matrices

Toeplitz matrix: constant values along each diagonal — (i,j) entry depends only on i−j. Structure appears in any shift-invariant linear operation — exactly what convolution is. Circulant matrix: Toeplitz with periodic boundary conditions, last row wraps to first. Key fact: circulant matrices diagonalized by DFT — any circulant multiplication done in O(n log n) via FFT.

Why frequency-domain convolution is faster for large kernels. Why state space models like S4 process sequences efficiently. Why convolutional layer eigenvalue analysis has clean spectral form. Broader class of structured matrices — low-displacement-rank, butterfly, others — captures linear transforms computable in O(n log n) instead of O(n²). Active neural architecture research: replace dense weight matrices with structured ones, retain expressivity, gain efficiency.

The Deeper Waters

Everything above: detours from first year or two of serious ML work. What follows is further out. Shows up when reading theory papers carefully, working in differential geometry or physics-inspired ML, or when senior researcher says something you want to actually understand. Not suggesting study these now. Suggesting: when you hear the words, don't dismiss them as out of reach.

Measure Theory

ML-course probability is informal about what "probability space" means. Measure theory makes it rigorous: probability space = set Ω, σ-algebra of measurable events, probability measure assigning [0,1] values. σ-algebra exists because not every subset of Ω can be consistently assigned a probability — obvious once you've seen the paradoxes it prevents.

Payoff: Lebesgue integration, generalizing Riemann integral to complicated discontinuous functions. Expectations in probability are Lebesgue integrals. "Almost surely" — as in "converges almost surely" — means "except on set of measure zero." Don't need measure theory to do ML. But first time you read formal convergence proof and every step cites measure theory, you'll want at least the vocabulary.

Functional Analysis & RKHS

Linear algebra: finite-dimensional vector spaces. Functional analysis extends same ideas — inner products, norms, orthogonality, projections — to infinite-dimensional function spaces. Hilbert space: inner product space that's complete (no convergent sequences with missing limits). Square-integrable functions form Hilbert space. So do kernel-defined function spaces.

Reproducing kernel Hilbert space (RKHS): Hilbert space where point evaluation is continuous linear functional — evaluation has representer in space. Mathematical foundation for kernel trick: instead of explicit infinite-dimensional feature space, work with kernel k(x, x') = ⟨φ(x), φ(x')⟩ computing inner products implicitly. Representer theorem: solution to any regularized kernel learning problem = finite linear combination of kernels at training points — why infinite-dimensional features produce finite solutions. Want to understand kernel methods deeper than "dot products in big space"? RKHS is language.

Topology & Topological Data Analysis

Topology studies properties preserved under continuous deformation — stretching and bending, not tearing or gluing. Coffee mug and donut topologically equivalent (one hole each); sphere and torus are not. Topological data analysis (TDA): track how "shape" of point cloud changes as scale varies — recording when components appear/merge, loops form/fill, voids appear/close.

This is persistent homology — produces persistence diagram summarizing multi-scale topological structure. ML applications: loss landscape analysis (are minima connected? connected manifold of good solutions explaining why different training runs find similar models?), mode connectivity in NNs, feature engineering for data where shape matters. Still research-level tool, not production. But topology language increasingly appearing in NN theory papers.

Group Theory & Geometric Deep Learning

Group: set with associative binary operation, identity, and inverses. Integers under addition form group. Sphere rotations form group. Permutations of n elements form group. Groups formalize symmetry — symmetry is everywhere in ML.

CNNs equivariant to translations: translate input, feature maps translate correspondingly. Equivariance hard-codes translation symmetry into architecture instead of learning from data. Geometric deep learning generalizes: given symmetry group G, build NNs equivariant to G. E(3)-equivariant GNNs: equivariant to 3D rotations, reflections, translations — essential for molecular property prediction where properties don't depend on orientation. Lie groups: continuous groups with differentiable structure, enabling gradient-based optimization over group. Framework makes architecture choices in geometric and physics-inspired models transparent.

Information Geometry

Probability distributions form manifold — smooth curved space. Natural geometry given by Fisher information matrix, acting as Riemannian metric: defines distance between nearby distributions invariant to reparameterization. This is information geometry — mathematical foundation for why natural gradient matters.

Gradient descent moves in parameter space, treating all directions equally. Nearby Euclidean parameters can be very different distributions; nearby distributions can be very different parameters. Natural gradient moves in distribution space: premultiply gradient by inverse Fisher matrix, correcting for statistical manifold curvature. Converges faster than ordinary gradient descent in parameter-invariant sense. K-FAC approximates natural gradient cheaply. Amari's information geometry: one of the better bridges between differential geometry and statistics.

Optimal Transport

Two piles of dirt. Reshape one into other. Optimal transport: minimum total work required (mass × distance moved). Minimum cost = Wasserstein distance (Earth Mover's Distance) between distributions. Unlike KL divergence, Wasserstein defined even when distributions have non-overlapping support — matters enormously in practice.

Prominent in ML via Wasserstein GANs — replaced Jensen-Shannon divergence with Wasserstein for smoother gradients and more stable training. Broader uses: distribution alignment in domain adaptation, comparing point clouds in 3D shape analysis, training set selection, geometrically-aware loss functions. Sinkhorn algorithm computes approximate OT efficiently via entropy regularization — practical at ML scale.

Category Theory

Category theory: mathematics of structure and composition. Category = objects + morphisms (arrows) + composition rules. Functor: maps between categories preserving structure. Natural transformation: maps between functors. Captures compositional structure of mathematical objects, abstracted from specific details.

Furthest from daily ML practice, but not decorative. "Backprop as Functor" formulates reverse-mode AD as functor between categories — clean compositional account of why backprop works. Categorical thinking appears in differentiable programming framework design and understanding algebraic structure of model composition. Category theory at "read Wikipedia several times" level probably fine for most ML work. But knowing it exists and what it claims: worth more than knowing nothing.

Interview Gotchas

These are questions where answer feels obvious, you say it, it's wrong.

Eigenvalues, Zero, and Nilpotent Matrices

All eigenvalues zero doesn't mean matrix is zero. Nilpotent matrix has all zero eigenvalues but is itself nonzero — classic example: strictly upper-triangular matrix. Zero eigenvalues describe characteristic polynomial, not matrix entries. Don't confuse "all eigenvalues zero" with "matrix is zero."

Zero Covariance Is Not Independence

Zero covariance = no linear relationship. Two variables can have zero covariance and wildly dependent distributions — classic: X uniform on [−1, 1], Y = X². Zero covariance because positive and negative parts cancel; knowing X tells you Y exactly. Everyone says they know this. Gets wrong in the moment.

Jensen's Inequality and E[f(X)] ≠ f(E[X])

For nonlinear f, E[f(X)] ≠ f(E[X]). For convex f, E[f(X)] ≥ f(E[X]) — Jensen's inequality. Concave f: flips. Comes up constantly: expected loss under stochastic training ≠ loss at expected parameters; ensemble expected prediction ≠ prediction at mean weights; expected perplexity ≠ perplexity at expected distribution. Paper "lower bounds" something using Jensen? This is what they're doing.

Medical Tests and Base Rates

Test for rare disease is 99% accurate (1% false positive, 1% false negative). You test positive. Probability you have disease? Most say 99%. Depends entirely on base rate. One in 10,000 have it: most positives are false positives — even very accurate test. Bayes' theorem. Failure: ignoring prior. ML lesson: when classifier says "positive," ask how rare positive class is before trusting it.

PCA and Uniform Variance

PCA finds directions of maximum variance. If all features have equal variance and all directions equally informative, PCA finds... something, but not meaningful. Principal components sensitive to noise, not signal. Assumes genuine variance structure exists. Isotropic data — variance uniform in all directions — PCA output essentially arbitrary. Dimensionality reduction will hurt, not help.

The Pseudoinverse of a Full-Rank Square Matrix

If A is square and full-rank, A⁺ = A⁻¹. Pseudoinverse reduces to ordinary inverse exactly when ordinary inverse exists. Common interview question: "what does formula reduce to in this special case?" Knowing A⁺ = A⁻¹ when inverse exists signals understanding pseudoinverse, not just memorizing definition.

The Softmax Jacobian

Jacobian of softmax σ(z) has specific structure. Diagonal entries: σᵢ(1 − σᵢ). Off-diagonal: −σᵢσⱼ. Matrix form: J = diag(σ) − σσᵀ. Symmetric, rank-deficient (rank n−1), positive semidefinite. Rank deficiency: softmax outputs sum to one, so one direction where Jacobian is identically zero. Deriving cross-entropy gradient updates by hand or debugging custom softmax? This structure tells you exactly what to expect.
