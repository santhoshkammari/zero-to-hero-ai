Linear Algebra

Chapter 2: Mathematical Foundations 19 subtopics
TL;DR

Linear algebra is the operating system of machine learning. Every neural network forward pass is matrix multiplication. Every backpropagation step is matrix calculus. PCA, SVD, and eigendecomposition are the workhorses behind dimensionality reduction, recommender systems, and feature extraction. If you understand how matrices transform space — stretching, rotating, projecting — you understand what ML models actually do to your data. This section covers vectors, matrices, decompositions, and their direct applications in ML, with NumPy code you can run today.

Every ML model is really matrix multiplication in disguise. A neural network? Stack a bunch of matrix multiplications with nonlinearities sandwiched between them. Linear regression? One matrix equation. PCA? Eigendecomposition of a matrix. Recommender systems? Matrix factorization. Even attention in transformers boils down to matrix products. If you take one thing from this chapter, let it be this: the better you understand how matrices transform data, the more intuitive every ML algorithm becomes.

We are going to build up from the smallest building block — a single number — all the way to decompositions that power production ML systems. Every concept comes with a concrete ML example. No hand-waving.

Vectors

Scalars, Vectors, Matrices, and Tensors — The Data Hierarchy

Let us start at the bottom. A scalar is a single number — the temperature outside, one pixel intensity, a single weight in your model. A vector is an ordered list of scalars. Think of a single data point: a house with 3 bedrooms, 1500 square feet, built in 1990 is the vector [3, 1500, 1990]. A matrix is a grid of numbers — your entire dataset of 1000 houses with 3 features is a 1000×3 matrix. A tensor is the generalization to any number of dimensions. A color image is a 3D tensor: height × width × 3 color channels. A batch of 32 color images is a 4D tensor: 32 × height × width × 3.

In code, these map directly to NumPy arrays of different dimensions. PyTorch and TensorFlow tensors behave the same way. The entire deep learning stack is built on operations over these structures.

Vector Notation and Representation

A vector is typically written as a column of numbers. In math notation, a vector v in ℝⁿ has n components. In ML, we usually think of vectors as column vectors by default — a matrix with n rows and 1 column. NumPy does not distinguish between row and column vectors unless you explicitly reshape, which is a common source of bugs when you are starting out.

Vector Addition and Scalar Multiplication

Adding two vectors means adding them element by element. Multiplying a vector by a scalar means scaling every element. These two operations together give you linear combinations — and linear combinations are the single most important idea in this entire section. When a neural network computes a weighted sum of inputs, it is forming a linear combination.

Dot Product (Inner Product) — Similarity!

The dot product of two vectors multiplies corresponding elements and sums the results. Sounds simple, but this operation is everywhere. It measures how much two vectors "agree" with each other. If two vectors point in the same direction, their dot product is large and positive. If they are perpendicular, it is zero. If they point in opposite directions, it is large and negative.

This is why cosine similarity uses the dot product. Cosine similarity is defined as the dot product of two vectors divided by the product of their magnitudes: cos(θ) = (a · b) / (||a|| × ||b||). The division normalizes out the length, so you are purely measuring directional agreement. Word embeddings like Word2Vec use cosine similarity to find semantically similar words — "king" and "queen" have high cosine similarity because they point in similar directions in embedding space.

import numpy as np

a = np.array([1, 2, 3])
b = np.array([4, 5, 6])

# Dot product
dot = np.dot(a, b)  # 1*4 + 2*5 + 3*6 = 32

# Cosine similarity
cos_sim = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
print(f"Dot product: {dot}")        # 32
print(f"Cosine similarity: {cos_sim:.4f}")  # 0.9746

Vector Norm (L1, L2, Lp, L∞) — Regularization!

A norm measures the "size" of a vector. The L2 norm (Euclidean norm) is the straight-line distance from the origin: the square root of the sum of squared components. The L1 norm (Manhattan distance) is the sum of absolute values. The L∞ norm is simply the largest absolute component value.

Why do you care? Regularization. L2 regularization (Ridge) penalizes the L2 norm of weight vectors, pushing weights to be small but not exactly zero. L1 regularization (Lasso) penalizes the L1 norm, which drives some weights to exactly zero — giving you feature selection for free. The geometry explains why: the L1 "ball" has corners on the axes, making it more likely that the optimal solution hits a corner where some weights are zero.

w = np.array([0.5, -1.2, 0.01, 3.0])

l1_norm = np.linalg.norm(w, ord=1)    # |0.5|+|1.2|+|0.01|+|3.0| = 4.71
l2_norm = np.linalg.norm(w, ord=2)    # sqrt(0.25+1.44+0.0001+9) = 3.27
linf_norm = np.linalg.norm(w, ord=np.inf)  # 3.0

print(f"L1: {l1_norm:.2f}, L2: {l2_norm:.2f}, L-inf: {linf_norm:.2f}")

Unit Vectors and Normalization

A unit vector has a norm of exactly 1. You get one by dividing a vector by its norm — this is called normalization. Batch normalization and layer normalization in deep learning are more sophisticated versions of this same idea: rescale your data so that magnitudes do not explode or vanish as they flow through the network.

Orthogonality and Angle Between Vectors

Two vectors are orthogonal if their dot product is zero — they are at right angles to each other. Orthogonal features are statistically uncorrelated, which is ideal for ML. PCA explicitly produces orthogonal components. The angle between any two vectors comes from the dot product formula: θ = arccos((a · b) / (||a|| × ||b||)).

Cross Product (3D)

The cross product is defined only in 3D. It gives a vector perpendicular to both inputs, with magnitude equal to the area of the parallelogram they form. It shows up less in ML directly, but it matters in computer vision and 3D graphics — anywhere you need surface normals or rotation axes.

Linear Combinations, Span, and Linear Independence

A linear combination of vectors is any sum of those vectors scaled by constants: c₁v₁ + c₂v₂ + ... + cₙvₙ. The span of a set of vectors is all possible linear combinations you can form. If a set of vectors spans all of ℝⁿ, you can reach any point. Vectors are linearly independent if none of them can be written as a linear combination of the others.

In ML terms: if your features are linearly dependent, you have redundant information. Multicollinearity in regression is exactly this problem — your feature vectors are (nearly) linearly dependent, so the model cannot tell which feature is actually contributing. Dimensionality reduction techniques remove this redundancy by finding a smaller set of independent directions.

Vectors in ML Are Not Just Arrows

Mathematically, a vector is an arrow in space. But in ML, a vector is an embedding — a learned representation. A word, an image, a user profile, a product listing — all become vectors. The entire field of representation learning is about finding the right vectors so that geometric operations (dot products, distances, projections) correspond to meaningful relationships.

Matrices

Matrix Notation and Dimensions

A matrix is a rectangular array of numbers with m rows and n columns, written as an m×n matrix. In ML, your dataset is typically an m×n matrix where m is the number of samples and n is the number of features. The weight matrix in a neural network layer mapping n inputs to k outputs is k×n.

Matrix Addition and Scalar Multiplication

Same idea as vectors — element-wise operations. These work only when dimensions match. NumPy's broadcasting rules let you add a vector to a matrix by automatically repeating the vector across rows or columns, which is how bias vectors get added in neural network layers.

Matrix Multiplication — The Most Important Operation in ML

This is it. The big one. Matrix multiplication is not element-wise. To multiply an m×n matrix A by an n×p matrix B, you take the dot product of each row of A with each column of B, producing an m×p result. The inner dimensions must match.

A single fully connected layer in a neural network is literally: y = Wx + b. Here, x is your input vector (n×1), W is your weight matrix (k×n), and b is a bias vector (k×1). The output y is a k×1 vector. That is one matrix multiplication and one vector addition. Stack multiple of these with activation functions between them, and you have a deep neural network.

import numpy as np

# A single neural network layer: y = Wx + b
# 4 inputs, 3 outputs
W = np.random.randn(3, 4)   # weight matrix
x = np.array([1.0, 0.5, -1.0, 2.0])  # input
b = np.array([0.1, -0.1, 0.0])        # bias

y = W @ x + b   # @ is matrix multiplication in Python 3.5+
print(f"Output: {y}")

# Batch processing: multiply entire dataset at once
X = np.random.randn(100, 4)  # 100 samples, 4 features
Y = X @ W.T + b              # 100 samples, 3 outputs — all at once
print(f"Batch output shape: {Y.shape}")  # (100, 3)

Transpose

The transpose of a matrix flips rows and columns: the (i,j) element becomes the (j,i) element. An m×n matrix becomes n×m. The transpose is everywhere in ML formulas — the normal equation for linear regression is x = (AᵀA)⁻¹Aᵀb. In NumPy, use A.T.

Identity, Diagonal, and Symmetric Matrices

The identity matrix I has ones on the diagonal and zeros elsewhere. Multiplying by I changes nothing — it is the matrix equivalent of multiplying by 1. Diagonal matrices have nonzero values only on the diagonal and are fast to compute with. Symmetric matrices equal their own transpose: A = Aᵀ. Covariance matrices are always symmetric, which gives them nice properties for decomposition.

Matrix Inverse and Invertibility

The inverse of a square matrix A is the matrix A⁻¹ such that AA⁻¹ = I. Not every matrix has an inverse. A matrix is invertible (nonsingular) if and only if its determinant is nonzero, its columns are linearly independent, and its rank equals its size. In ML, you rarely compute inverses directly — it is numerically unstable and slow. Use decompositions instead.

Orthogonal Matrices

An orthogonal matrix Q has columns that are orthonormal unit vectors. Its inverse equals its transpose: Q⁻¹ = Qᵀ. Orthogonal matrices represent rotations and reflections — they preserve lengths and angles. They show up in SVD, QR decomposition, and as the eigenvectors of symmetric matrices.

Positive Definite and Semi-Definite Matrices

A symmetric matrix is positive definite if xᵀAx > 0 for every nonzero vector x. It is positive semi-definite (PSD) if xᵀAx ≥ 0. Covariance matrices are always PSD. Kernel matrices (Gram matrices) must be PSD for a kernel function to be valid. If your optimization landscape has a positive definite Hessian at a point, that point is a local minimum — this is how second-order optimization methods work.

Never Invert a Matrix in Production Code

If you need to solve Ax = b, do NOT compute A⁻¹ and multiply. Use np.linalg.solve(A, b) instead. It uses LU decomposition under the hood and is both faster and more numerically stable. Matrix inversion is O(n³) and amplifies floating-point errors. This is one of the most common numerical mistakes in ML implementations.

Systems of Linear Equations

Matrix Form Ax = b

Any system of linear equations can be written as Ax = b, where A is the coefficient matrix, x is the unknown vector, and b is the right-hand side. Linear regression is solving exactly this kind of system. You have data points (rows of A), observed outputs (b), and you want to find weights (x) that make the equations approximately true.

Gaussian Elimination and Row Echelon Form

Gaussian elimination systematically zeros out entries below the diagonal by adding scaled rows to other rows, producing row echelon form. From there, back-substitution gives the solution. This is the classic direct solver and what np.linalg.solve uses (via LU decomposition, which is Gaussian elimination organized for efficiency).

Existence and Uniqueness of Solutions

A system Ax = b either has no solution (inconsistent), exactly one solution (A is full rank and square), or infinitely many solutions (underdetermined). In ML, your system is almost always overdetermined — more equations than unknowns — so no exact solution exists. Instead, you find the least squares solution, the x that minimizes ||Ax - b||².

Homogeneous Systems

When b = 0, the system Ax = 0 is homogeneous. It always has at least the trivial solution x = 0. Nontrivial solutions exist when A is not full rank. The set of all solutions is the null space of A. In PCA, finding eigenvectors involves solving a homogeneous system: (A - λI)v = 0.

Vector Spaces

Definition and Subspaces

A vector space is a collection of vectors that is closed under addition and scalar multiplication. ℝⁿ is the most common example. A subspace is a vector space sitting inside a larger one — it must contain the zero vector and be closed under the same operations. The set of all linear combinations of your training data vectors forms a subspace. ML models that use linear methods can only produce outputs within this subspace.

Basis and Dimension

A basis is a minimal set of linearly independent vectors that spans the space. The dimension of a space is the number of vectors in any basis. ℝ³ has dimension 3. If your data lives in a 100-dimensional space but only varies along 5 directions, the effective dimension is 5. PCA finds exactly this — the basis for the subspace where your data actually lives.

Column Space, Row Space, and Null Space

The column space of a matrix is the span of its columns — it is all possible outputs Ax. The row space is the span of its rows. The null space (kernel) is all vectors x such that Ax = 0. These spaces are deeply connected. The dimension of the column space equals the dimension of the row space (both equal the rank), and the rank plus the null space dimension equals the number of columns (the rank-nullity theorem).

Rank of a Matrix

The rank is the number of linearly independent rows (or columns). It tells you how much information the matrix carries. A rank-deficient matrix has redundant rows or columns. In ML, a low-rank data matrix means your features are correlated and can be compressed. Low-rank approximation (via SVD) is the mathematical foundation of dimensionality reduction.

Determinants

Definition and Computation

The determinant of a square matrix is a single number that captures how the matrix scales space. For a 2×2 matrix [[a,b],[c,d]], the determinant is ad - bc. For larger matrices, you compute it recursively using cofactor expansion — expand along a row or column, alternating signs.

Geometric Interpretation

The absolute value of the determinant tells you the factor by which the matrix scales areas (2D) or volumes (3D). A determinant of 2 means the matrix doubles all areas. A determinant of 0 means the matrix collapses space to a lower dimension — it squishes a 2D region into a line or a point. This is why a zero determinant means the matrix is not invertible: you cannot undo a squish.

Determinant and Invertibility

A matrix is invertible if and only if its determinant is nonzero. In practice, you rarely compute determinants to check invertibility — you check the rank or condition number instead. But the determinant shows up in probability: the normalization constant of a multivariate Gaussian involves the determinant of the covariance matrix.

import numpy as np

A = np.array([[3, 1], [2, 4]])
det = np.linalg.det(A)
print(f"Determinant: {det:.1f}")  # 10.0 — matrix scales area by 10x

# Singular matrix (determinant = 0)
B = np.array([[1, 2], [2, 4]])   # row 2 is 2× row 1
print(f"Singular det: {np.linalg.det(B):.1f}")  # 0.0

Eigenvalues and Eigenvectors

Definition: Av = λv

An eigenvector of a matrix A is a nonzero vector v that, when multiplied by A, only gets scaled — not rotated. The scaling factor is the eigenvalue λ. In equation form: Av = λv. Most vectors change direction when you multiply by a matrix. Eigenvectors are the special ones that keep pointing the same way.

Why does this matter for ML? Consider a covariance matrix. Its eigenvectors point in the directions of maximum variance in your data. Its eigenvalues tell you how much variance exists along each direction. This is PCA in a nutshell: find the eigenvectors of the covariance matrix, keep the ones with the largest eigenvalues, and project your data onto them.

Characteristic Equation and Computation

To find eigenvalues, you solve det(A - λI) = 0 — the characteristic equation. This gives a polynomial of degree n for an n×n matrix. In practice, nobody solves polynomials for matrices bigger than 3×3. Numerical algorithms like QR iteration are used instead, which is what np.linalg.eig calls.

Eigendecomposition and Diagonalization

If a matrix has n linearly independent eigenvectors, it can be decomposed as A = PDP⁻¹, where P is the matrix of eigenvectors and D is a diagonal matrix of eigenvalues. This is eigendecomposition (or spectral decomposition for symmetric matrices). Diagonalization makes computation easy: A^k = PD^kP⁻¹, and raising a diagonal matrix to a power just raises each diagonal entry to that power.

Symmetric Matrices and PCA

Symmetric matrices (like covariance matrices) have a special property: their eigenvalues are always real, and their eigenvectors are always orthogonal. This means PCA always produces orthogonal principal components — your new features are guaranteed to be uncorrelated.

import numpy as np

# Eigendecomposition of a covariance matrix
data = np.random.randn(1000, 3)  # 1000 samples, 3 features
cov_matrix = np.cov(data.T)

eigenvalues, eigenvectors = np.linalg.eigh(cov_matrix)  # eigh for symmetric
# Sort by descending eigenvalue
idx = np.argsort(eigenvalues)[::-1]
eigenvalues = eigenvalues[idx]
eigenvectors = eigenvectors[:, idx]

print(f"Eigenvalues: {eigenvalues}")
print(f"Variance explained: {eigenvalues / eigenvalues.sum()}")
# First eigenvector is the direction of maximum variance — the first principal component
eig vs eigh — Know the Difference

Use np.linalg.eigh for symmetric (Hermitian) matrices — it is faster and guarantees real eigenvalues and orthogonal eigenvectors. Use np.linalg.eig for general matrices, but beware that eigenvalues may be complex. Since covariance matrices are always symmetric, always use eigh for PCA.

Singular Value Decomposition (SVD)

Definition: A = UΣVᵀ

SVD decomposes any matrix (not just square ones!) into three matrices: A = UΣVᵀ. U is an orthogonal matrix of left singular vectors, Σ is a diagonal matrix of singular values (always non-negative, sorted in decreasing order), and Vᵀ is an orthogonal matrix of right singular vectors. Think of it as decomposing any linear transformation into a rotation, a scaling, and another rotation.

Relationship to Eigendecomposition

The singular values of A are the square roots of the eigenvalues of AᵀA. The right singular vectors are the eigenvectors of AᵀA, and the left singular vectors are the eigenvectors of AAᵀ. SVD is the generalization of eigendecomposition to non-square matrices.

Low-Rank Approximation

Here is the power of SVD: you can approximate a matrix by keeping only the top k singular values and their corresponding vectors. This gives the best rank-k approximation (by the Eckart-Young theorem, covered below). If a 1000×1000 matrix has only 10 significant singular values, you can represent it with roughly 10 × (1000 + 1000) = 20,000 numbers instead of 1,000,000. This is how latent semantic analysis, image compression, and collaborative filtering work.

import numpy as np

# SVD of a user-item ratings matrix (recommender systems!)
# 5 users, 4 items
ratings = np.array([
    [5, 4, 0, 1],
    [4, 5, 1, 0],
    [0, 1, 4, 5],
    [1, 0, 5, 4],
    [5, 4, 1, 0]
])

U, sigma, Vt = np.linalg.svd(ratings, full_matrices=False)
print(f"Singular values: {sigma.round(2)}")

# Low-rank approximation (keep top 2 components)
k = 2
approx = U[:, :k] @ np.diag(sigma[:k]) @ Vt[:k, :]
print(f"Rank-{k} approximation:\n{approx.round(1)}")
# Fills in the zeros with predicted ratings!

Projections and Least Squares

Vector Projections

The projection of vector b onto vector a gives the component of b in the direction of a. The formula is proj_a(b) = (a·b / a·a) × a. The residual b - proj_a(b) is perpendicular to a. This decomposition — splitting a vector into a component along a direction and a perpendicular error — is the geometric heart of regression.

Projection onto Subspaces and the Normal Equation

Linear regression asks: given data matrix A and observations b, find x that minimizes ||Ax - b||². Geometrically, you are projecting b onto the column space of A. The closest point in the column space is Ax̂, and the residual b - Ax̂ is perpendicular to the column space. Setting up this perpendicularity condition gives the normal equation: AᵀAx̂ = Aᵀb.

This is one of the most beautiful results in all of ML. Linear regression is not curve fitting — it is a geometric projection. The predicted values are the shadow of reality cast onto the subspace of possible predictions.

Why "Normal" Equation?

"Normal" means perpendicular. The equation ensures the residual vector is perpendicular (normal) to the column space of A. This is the same as saying the error is orthogonal to all features — the model has extracted everything it can from the data.

Matrix Calculus Essentials

Gradient of Scalar w.r.t. Vector

If you have a scalar loss function L that depends on a parameter vector w, the gradient ∇L is a vector of partial derivatives — one for each component of w. The gradient points in the direction of steepest increase of L. Gradient descent moves in the opposite direction: w ← w - α∇L. Every neural network training step computes this gradient.

Jacobian and Hessian

The Jacobian generalizes the gradient to vector-valued functions. If f: ℝⁿ → ℝᵐ, the Jacobian is an m×n matrix of partial derivatives. Backpropagation chains Jacobians together — that is the chain rule for vector functions. The Hessian is the matrix of second partial derivatives of a scalar function. It tells you about curvature: whether you are at a minimum, maximum, or saddle point. Second-order optimization methods (Newton's method, L-BFGS) use the Hessian, while first-order methods (SGD, Adam) ignore it for speed.

Common Matrix Derivatives

Some derivatives you will see constantly: the gradient of wᵀx with respect to w is x. The gradient of ||Ax - b||² with respect to x is 2Aᵀ(Ax - b). Setting this to zero gives the normal equation. The gradient of xᵀAx with respect to x is (A + Aᵀ)x, which simplifies to 2Ax when A is symmetric. These formulas are the building blocks of deriving ML loss function gradients.

Condition Number and Numerical Stability

The condition number of a matrix measures how sensitive the solution of Ax = b is to small changes in A or b. It is defined as the ratio of the largest to smallest singular value: κ(A) = σ_max / σ_min. A condition number near 1 is great. A condition number of 10⁸ means your solution could have 8 fewer digits of accuracy than your input.

This is why your model sometimes outputs NaN. When features have vastly different scales — one ranges from 0 to 1 and another from 0 to 1,000,000 — the resulting matrices become ill-conditioned. Feature scaling (standardization, normalization) is not just a best practice — it is a numerical necessity. Batch normalization in deep learning also helps by keeping intermediate activations well-conditioned.

NaN Debugging Checklist

If your model produces NaN during training: (1) Check for features with very different scales — standardize them. (2) Lower the learning rate — large steps can hit numerically unstable regions. (3) Add a small epsilon to denominators in custom loss functions. (4) Check for log(0) or division by zero. (5) Inspect the condition number of your data matrix with np.linalg.cond(X).

Graph Laplacian and Spectral Clustering

Graphs encode relationships: users in a social network, words in a document, pixels in an image. The graph Laplacian is a matrix L = D - A, where D is the diagonal degree matrix and A is the adjacency matrix. Its eigenvectors reveal the cluster structure of the graph.

Spectral clustering works by computing the smallest eigenvectors of the Laplacian (or the normalized Laplacian) and then running k-means on those eigenvector coordinates. The reason this works is that the second-smallest eigenvector (the Fiedler vector) provides the optimal two-way partition of the graph. This connects linear algebra directly to graph theory and clustering — matrix structure captures community structure.

Power Iteration Method

How do you find the dominant eigenvalue (the largest one) without computing all of them? Power iteration: start with a random vector v, repeatedly multiply by A and normalize. The vector converges to the eigenvector with the largest eigenvalue, and the scaling factor converges to that eigenvalue. It is simple, memory-efficient, and works for very large sparse matrices where full eigendecomposition is impossible.

Google's original PageRank algorithm is power iteration on the web graph's transition matrix. Each iteration refines the importance scores of web pages. It converges because the transition matrix has a dominant eigenvalue of 1 (guaranteed by the Perron-Frobenius theorem for stochastic matrices).

Eckart-Young Theorem

The Eckart-Young theorem states that the best rank-k approximation to a matrix (in the Frobenius or spectral norm) is obtained by taking the SVD and keeping only the top k singular values. "Best" means it minimizes the difference between the original and the approximation. No other rank-k matrix can do better.

This theorem is the theoretical justification for truncated SVD, PCA, and every low-rank approximation technique used in ML. When you reduce a 768-dimensional BERT embedding to 50 dimensions, Eckart-Young guarantees you are preserving as much information as possible given that budget.

Randomized SVD

Full SVD of an m×n matrix costs O(mn × min(m,n)), which is impractical when your matrix has millions of rows and columns. Randomized SVD uses random projections to approximate the top-k singular vectors in O(mn × k) time. The key idea: multiply the matrix by a random matrix to get a sketch, compute the SVD of the much smaller sketch, and project back.

Scikit-learn uses randomized SVD under the hood in TruncatedSVD and PCA when svd_solver='randomized'. For most practical purposes, the approximation error is negligible while the speedup is orders of magnitude. If you are running PCA on a dataset with more than ~1000 features, randomized SVD is almost certainly what you want.

Non-Negative Matrix Factorization (NMF)

NMF decomposes a non-negative matrix V into two non-negative matrices: V ≈ WH. The non-negativity constraint is what makes NMF special — the parts can only be added, never subtracted. This produces parts-based representations: in face recognition, NMF learns features like "nose," "eye," and "mouth" that combine additively to form faces. PCA, by contrast, produces holistic features that can cancel each other out.

In topic modeling, NMF applied to a term-document matrix produces topics (columns of W) as non-negative mixtures of words, and document representations (rows of H) as non-negative mixtures of topics. It is a popular alternative to LDA that is often faster and gives comparable results. Scikit-learn provides NMF directly in sklearn.decomposition.

PCA Comprehensive

Step-by-Step Procedure

PCA in five steps: (1) Center the data by subtracting the mean of each feature. (2) Compute the covariance matrix. (3) Find the eigenvalues and eigenvectors of the covariance matrix. (4) Sort eigenvectors by descending eigenvalue. (5) Project the data onto the top k eigenvectors. That is it. You have reduced your data from n dimensions to k dimensions, preserving as much variance as possible.

Centering and Standardization

Centering (subtracting means) is mandatory for PCA to work correctly — the covariance matrix formula assumes centered data. Standardization (dividing by standard deviation) is optional but important when features have different scales. Without standardization, PCA is dominated by the highest-variance features, which might just be the ones measured in the largest units.

Scree Plot and Explained Variance

A scree plot graphs eigenvalues in descending order. You look for the "elbow" — the point where eigenvalues drop off sharply. Components before the elbow capture signal; those after capture noise. The explained variance ratio of each component is its eigenvalue divided by the sum of all eigenvalues. A common heuristic is to keep enough components to explain 95% of the variance.

Variance Maximization and Reconstruction Error

PCA can be derived two equivalent ways. The variance maximization view says: find the direction that maximizes the variance of the projected data. That is the first eigenvector. The reconstruction error view says: find the k-dimensional subspace that minimizes the average squared distance between data points and their projections. These two formulations produce the same answer — a satisfying duality.

PCA vs Factor Analysis

PCA assumes that the important structure in data is captured by directions of maximum variance. Factor analysis assumes the data is generated by a small number of latent factors plus noise. When the noise is isotropic (equal in all directions), they give similar results. When noise varies across features, factor analysis can outperform PCA. In practice, PCA is used far more often because it is simpler and faster.

Kernel PCA, Incremental PCA, Probabilistic PCA, and Sparse PCA

Kernel PCA applies the kernel trick to capture nonlinear structure. It computes PCA in a high-dimensional feature space without ever explicitly computing the coordinates there. Incremental PCA processes data in batches, making it feasible for datasets that do not fit in memory. Probabilistic PCA casts PCA as a latent variable model, enabling it to handle missing data and provide a proper likelihood. Sparse PCA adds L1 regularization so that each component loads on only a few features, improving interpretability at the cost of optimality.

import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# Generate sample data: 500 samples, 10 features
np.random.seed(42)
X = np.random.randn(500, 10)
# Inject correlations: features 0-2 share an underlying factor
X[:, 1] = X[:, 0] * 0.8 + np.random.randn(500) * 0.3
X[:, 2] = X[:, 0] * 0.6 + np.random.randn(500) * 0.5

# Standardize
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# PCA
pca = PCA()
pca.fit(X_scaled)

print("Explained variance ratio:", pca.explained_variance_ratio_.round(3))
print("Cumulative:", np.cumsum(pca.explained_variance_ratio_).round(3))

# Keep enough for 95% variance
pca_95 = PCA(n_components=0.95, svd_solver='full')
X_reduced = pca_95.fit_transform(X_scaled)
print(f"Reduced from {X_scaled.shape[1]} to {X_reduced.shape[1]} dimensions")

# Reconstruction
X_reconstructed = pca_95.inverse_transform(X_reduced)
recon_error = np.mean((X_scaled - X_reconstructed) ** 2)
print(f"Reconstruction error: {recon_error:.4f}")
PCA Under the Hood: SVD, Not Eigendecomposition

Scikit-learn's PCA does not compute the covariance matrix and find its eigenvalues. It computes SVD of the centered data matrix directly. This is more numerically stable and handles the case where the number of features exceeds the number of samples. The right singular vectors of the data are the principal components.

Covariance Matrix

Definition and Properties

The covariance matrix Σ of a dataset captures pairwise linear relationships between features. The (i,j) entry is the covariance between features i and j. Diagonal entries are variances. It is always symmetric and positive semi-definite. For centered data X (mean-subtracted), the covariance matrix is (1/n)XᵀX.

Correlation Matrix

The correlation matrix is the covariance matrix of standardized features. Each entry is between -1 and 1. It tells you the same story as covariance but removes the effect of feature scales. Running PCA on the correlation matrix is equivalent to running PCA on standardized data.

Whitening Transform

The whitening transform decorrelates features and normalizes variances so that the covariance matrix becomes the identity. You compute it using the eigendecomposition of the covariance matrix: X_white = (eigenvalues)^(-1/2) × eigenvectors^T × X. Whitening is used as a preprocessing step in some algorithms (like ICA) and is related to batch normalization in deep learning.

Mahalanobis Distance

The Mahalanobis distance between a point x and a distribution with mean μ and covariance Σ is √((x-μ)ᵀ Σ⁻¹ (x-μ)). It is the Euclidean distance after whitening — it accounts for correlations and different variances. It is used in anomaly detection, Gaussian classifiers, and outlier detection. A point might be close in Euclidean distance but far in Mahalanobis distance if it lies away from the principal axes of the data distribution.

Cholesky Decomposition

Definition: A = LLᵀ

The Cholesky decomposition factors a positive definite matrix into the product of a lower triangular matrix and its transpose: A = LLᵀ. It is twice as fast as the general LU decomposition and numerically stable. It is the matrix equivalent of taking a square root.

Efficient Sampling from Multivariate Gaussians

To sample from a multivariate Gaussian with mean μ and covariance Σ, you do not need to find eigenvectors. Compute the Cholesky factor L of Σ, sample standard normal z, and compute x = μ + Lz. The resulting x has the correct distribution. This is how np.random.multivariate_normal works internally.

Applications in Gaussian Processes

Gaussian processes (GPs) rely heavily on Cholesky decomposition. The GP marginal likelihood involves the log-determinant and inverse of the kernel matrix, both of which can be computed efficiently via Cholesky. For an n×n kernel matrix, Cholesky brings the cost from O(n³) for a general inverse to O(n³/3) — a factor of 3 speedup that matters when n is in the thousands.

import numpy as np

# Sampling from a multivariate Gaussian using Cholesky
mean = np.array([1.0, 2.0, 3.0])
cov = np.array([
    [2.0, 0.5, 0.3],
    [0.5, 1.0, 0.2],
    [0.3, 0.2, 1.5]
])

L = np.linalg.cholesky(cov)      # lower triangular
z = np.random.randn(3)           # standard normal sample
sample = mean + L @ z            # transformed sample
print(f"Sample: {sample.round(3)}")

# Verify: generate many samples and check covariance
samples = mean + (L @ np.random.randn(3, 10000)).T
print(f"Empirical mean: {samples.mean(axis=0).round(2)}")
print(f"Empirical cov:\n{np.cov(samples.T).round(2)}")

QR Decomposition

Gram-Schmidt Process

QR decomposition factors a matrix into an orthogonal matrix Q and an upper triangular matrix R: A = QR. The classical method is the Gram-Schmidt process: take each column, subtract its projections onto all previous orthogonalized columns, and normalize. The result is an orthonormal set of vectors. However, classical Gram-Schmidt is numerically unstable — use modified Gram-Schmidt or Householder reflections in practice.

Householder Reflections

Householder reflections zero out below-diagonal entries one column at a time using orthogonal transformations. They are the standard method used by LAPACK (and therefore NumPy) for QR decomposition. They are unconditionally numerically stable, unlike Gram-Schmidt, which can lose orthogonality for ill-conditioned matrices.

Least Squares via QR

QR decomposition provides the most numerically stable way to solve least squares problems. Instead of solving the normal equation AᵀAx = Aᵀb (which squares the condition number), decompose A = QR and solve Rx = Qᵀb. Since R is triangular, back-substitution is trivial. This is what np.linalg.lstsq uses internally.

import numpy as np

# Solving least squares with QR (numerically stable)
A = np.random.randn(100, 5)   # 100 data points, 5 features
b = A @ np.array([1, -2, 3, 0.5, -1]) + np.random.randn(100) * 0.1

Q, R = np.linalg.qr(A)
x_qr = np.linalg.solve(R, Q.T @ b)  # Rx = Qᵀb

# Compare with direct least squares
x_lstsq, _, _, _ = np.linalg.lstsq(A, b, rcond=None)

print(f"QR solution:     {x_qr.round(3)}")
print(f"lstsq solution:  {x_lstsq.round(3)}")
print(f"True weights:    [1, -2, 3, 0.5, -1]")

Numerical Stability Compared

There are three ways to solve least squares: normal equation, QR, and SVD. The normal equation is fastest but squares the condition number — if κ(A) = 10⁴, then κ(AᵀA) = 10⁸. QR preserves the original condition number. SVD is the most robust and can handle rank-deficient matrices gracefully. The tradeoff is speed versus stability. For well-conditioned problems, all three give the same answer. For ill-conditioned problems, only SVD remains reliable.

Choosing a Least Squares Solver

For small well-conditioned problems: normal equation (fastest). For most practical cases: QR decomposition (good balance). For ill-conditioned or rank-deficient problems: SVD via np.linalg.lstsq (most robust). Scikit-learn's LinearRegression uses SVD by default — a sensible choice since you rarely know your data's condition number in advance.

That covers the essential linear algebra toolkit for machine learning. Every concept here connects directly to something you will build or debug. Vectors represent data and embeddings. Matrix multiplication implements neural network layers. Eigendecomposition powers PCA. SVD enables dimensionality reduction and recommender systems. Projections underpin regression. And numerical stability determines whether your model trains successfully or explodes into NaN.

The core insight is this: ML models transform data by moving it through different vector spaces. Linear algebra gives you the language and tools to understand, design, and debug those transformations. When you see a model architecture diagram, see matrices. When you see a loss function, see geometry. When you see a training failure, check the condition numbers. Linear algebra is not a prerequisite you grind through and forget — it is the lens through which everything else comes into focus.
