## Key Decisions
1. **Societal Impact removed as standalone** — The original s01 was ~90% philosophical essays and case studies that don't help an ML engineer build better systems. Key examples preserved as brief references in Fairness and Nice to Know.
2. **Sustainability → Nice to Know** — CodeCarbon, Chinchilla scaling, and carbon tracking are awareness-level for most ML engineers. LoRA/QLoRA already covered in LLM chapters. Didn't warrant a full section.
3. **Human-in-the-Loop split** — Active learning details → Nice to Know (it's an ML technique, not an ethics topic). Human oversight workflows → Safety section (that's where engineers need it).
4. **Security kept deep** — Adversarial attacks, prompt injection, and model stealing are real threats every ML engineer in production will face. Full code examples retained.
5. **Fairness metrics table** — Replaced verbose per-metric prose with a comparison table + the impossibility theorem explanation. Engineers need to quickly compare metrics, not read paragraphs about each one.
6. **Production guardrails as code** — The multi-layer guardrail class is the most practically useful artifact in the safety section. Engineers can adapt it directly.
