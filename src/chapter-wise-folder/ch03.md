# Chapter 03 — Data Fundamentals

## Status: ✅ v3 COMPLETE — Variable-depth, concept-driven, conversational tone

## Author Intent (cumulative — keep adding every session)
1. **Data is 80% of ML** — this chapter should make that crystal clear. Cleaning and feature engineering get the deepest treatment because that's where real improvement lives.
2. **Variable depth per concept** — EDA and cleaning/engineering go DEEP. Data types and versioning are brief. Format matches importance.
3. **Workflow over reference** — EDA is taught as a sequential workflow, not a list of functions. The reader should be able to follow the steps on any new dataset.
4. **Decision frameworks** — missing data handling, outlier treatment, encoding choice, scaler choice — all presented as "when to use what" tables.
5. **Production-relevant** — preprocessing pipelines, data validation, drift detection — the things that bite in real systems.
6. **Conversational tone** — Brandon Rohrer style. Not a textbook. Honest about tradeoffs.
